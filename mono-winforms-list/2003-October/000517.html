Andreas,

 

Try running ldconfig as root. I had to do this last time that I upgraded
rpms. Also make sure that the path to your wine .dll.so files is in
/etc/ldconfig.so.conf. This is the file that ldconfig reads when it runs.

 

I hope this helps.

 

-Scott
