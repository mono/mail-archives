I am pretty sure that this is a bug in the propertygrid control you are using. It overrides PropertyGrid.SelectedObject and wraps the object in a custom one to filter the properties. However it keeps settings the base.SelectedObject with the same object. The PropertyGrid will not refresh unless you are setting a different object and I think we have unit tests for that and that this is the behavior both on <a href="http://MS.NET">MS.NET</a> and Mono. And this comment confirms what I think - <a href="http://www.codeproject.com/KB/cs/FilteredPropertyGrid.aspx?msg=1422328#xx1422328xx">http://www.codeproject.com/KB/cs/FilteredPropertyGrid.aspx?msg=1422328#xx1422328xx</a><br>

<br>So you can fix this by changing the source code to set SelectedObject to null before setting it with the wrapper object to force a reset. Also just so you know you can also filter the objects properties by using the BrowsableAttribute and also by implementing the ICustomTypeDescriptor interface.<br>
