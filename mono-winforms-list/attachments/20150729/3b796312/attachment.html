<div dir="ltr">Hello Peter,<div><br></div><div>Currently I'm also building a multi platform desktop application.</div><div>I develop with VS, import it in a virtual Linux Mint via Git and build it with Monodevelop.</div><div><br></div><div>I use Monodevelop and Google to fix the list of build failures Monodevelop comes up with.</div><div>Besides that I try to use as little Dependencies as I can and only use Libraries which are crossplatform.</div><div><br></div><div>So, 1 codebase -> several deployments.</div><div><br></div><div>Personally I don't mind the Designer files VS creates, since these files only initiate and set the properties of the controls you create when you use the visual designer is VS.</div><div><div>The .resx files (also) contain the images you use in your application (b.t.w. I've noticed that it's better to avoid using Icon files but instead use png files and convert them to ico via code).</div></div><div><br></div><div>Monodevelop doesn't have a WinForm designer so you will be creating (some sort of) Designer files yourself anyway to initiatie and set the controls you need.</div><div><br></div><div>I have noticed that some Control properties set in VS don't act like they should in Monodevelop, many of which have been submitted as bugs.</div><div>In many cases initiating those Controls and setting the properties yourself (via code instead of using the Visual Designer) solve the problems (but not all of them, some properties simply don't work so I choose to ommit them instead of creating hacks around them ...).</div><div><br></div><div>So you can also stick to develop a winform app without a Visual Designer ;)</div><div><br></div><div>Keep in mind that you will have some issues regarding:</div><div>- visual refreshing on Unix systems (controls simply won't refresh themselve like they do on Windows, so sometimes you don't see the controls at all, which looks pretty awful. I see it in more applications on Unix systems ...)</div><div>- The controls used via Monodevelop are based on winXP which make them look a buit outdated</div><div>- Some controls used via Monodevelop have different base (not mutable) properties then WIndows, which leads to different spacing requirements ... (e.g. ComboBox).  </div><div><br></div><div>Since I only have the requirement that the software should work on Unix and that the graphical issues are not that important, I stick with my current development strategy.<br></div><div><br></div><div>I think it's very cool that I only have to write the code once and deploy it on different OSses, but the graphical experience is a bit dissapointing. </div><div><br></div><div>I'm hoping that future mono updates wil solve these graphical issues automagically, so the software works and looks as stunning as it does on a windows system :-)</div><div><br></div><div>I don't mind helping out in that area ;)</div><div><br></div><div>If not, I guess, HTML5 instead of WinForm will be a nice GUI replacement :-)</div><div><br></div><div>cheers,</div><div><br></div><div>Marcel Burger </div></div><div class="gmail_extra"><br><div class="gmail_quote">On Tue, Jul 28, 2015 at 5:23 PM, Stifu <span dir="ltr"><<a href="mailto:stifu@free.fr" target="_blank">stifu@free.fr</a>></span> wrote:<br><blockquote class="gmail_quote" style="margin:0 0 0 .8ex;border-left:1px #ccc solid;padding-left:1ex">Hi,<br>
