You can't.

The monocharge files are only a last-try solution to fix your mono-install
if it's broken, it's not supported and it will likely fail.

If you still want to try, I suggest reverting your mono install to 1.2.5,
find the assemblies you want to update (only) in the monocharge archive, and
put them manually in your gac.

If you're only interested in
System.Windows.Forms.dll/System.Drawing.dll/libgdiplus, it might actually
work (with some luck), but if it doesn't you'll have to use SVN and compile
it yourself.
