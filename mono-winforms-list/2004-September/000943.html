Hey guys,

Jackson suggested I look into making the radio button look more like the
Windows version. I've done a bit of work on it so far and have made it
look _mostly_ the windows version.

I've got it all working and looking properly (mainly tweaks to the line
thicknesses and some pixel offsets for text). I made sure the radio
background was filled in with white, it was previously transparent. I
even looked at getting the FlatStyle.Flat to render properly (think I
got it right).

All looks well (if you ignore the AA issues with cairo), but I did get
stuck on one item, the FlatStyle.Popup property.

The way the radio button renders now it actually calls DrawButton() in
the theme engine, but passes in a ButtonState enumeration that indicates
it's a radio button. My problem is that there doesn't seem to be a way
to pass in the FlatStyle.Popup state into the DrawButton() method using
the current setup. The ButtonState enum doesn't seem to hold enough
values for us to indicate that the radio is to be rendered like a flat
radio when no hover, and then like a normal radio when hovered.

Some help with that one would be much appreciated.

Managed.Windows.Forms) and a screenshot of the winforms radio sample in
taken winxp.

screenshot to the winforms/radiobutton folder in the winforms module.

files using cvs).
