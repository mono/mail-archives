It's not strictly required that they have the same targetsite or
stacktrace.  anyone relying on that information is likely to be broken
between minor releases of .net anyway, if microsoft moves code around.  

We should be as compatible as possible, and in many cases I'd agree with
you that the code should be changed to accommodate that.  In this case,
though, I think it's unnecessary.  The use of InternalBorderStyle
consolidates all the various BorderStyle get/set implementations into
one spot, and is good for that reason.  If we find at some point that a
subclass's implementation of BorderStyle differs from the usual case, we
can discuss about duplicating the code.

Chris
