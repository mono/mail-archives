	I would like to see files converted back to UNIX style.
When we finished all the stubs, I did convert all files to UNIX style, but
since people have checked in files with DOS newlines. As a visual studio
users and someone who touches a lot of the SWF files, I am probably public
enemy #1.
	If we do the conversion, we need to freeze SWF development while the
conversion is done, and everyone must checkout fresh code afterwards.
	Another problem is keeping it UNIX style. We should require (us)
windows programmers to run dos2unix/unix2dos before checkin. 
	We all should also run Monostyle on our code before checkin.

	When we switch to UNIX style newlines, someone should do s full
checkout, do the conversion, then check back in, with no other changes. the
sytem.drawing namespeaces should be done at the same time.

	A month ago only two of us were working on SWF, but that has sure
changed. How many people are working on SWF?

	If you are currently writing SWF or System drawing code, send me an
email, so I will have an up to date list.
