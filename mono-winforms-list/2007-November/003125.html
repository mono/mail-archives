I found out that Carbon 64-bit UI will not be supported in Leopard in 
favor of Cocoa. The current WindowsForms MacOSX port is entirely written 
in Carbon. Apple is obviosuly pushing everyone to Cocoa. It seems like the 
MacOSX port should use Cocoa for the windowing functions instead of Carbon 
because of the 64 bit issue. There is Cocoa# which could be leveraged. 
Leveraging Cocoa# would follow the "eating your own dogfood" approach 
making it more stable in the process.

I just subscribed to this list. Sorry if this might have been posted 
before.

Thoughts?
