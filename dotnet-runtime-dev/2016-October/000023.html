> Assuming that safe Span<T> stackalloc exists, the verifier (and compiler) must track Spans that aren’t allowed to be returned.
 
As an aside, there are some very interesting advantages to general “second-class” types like these. If values of second-class type can’t be returned from functions, stored in mutable fields, or captured by first-class functions, then you can statically know that they don’t escape their scope. That would be very useful for e.g. Linq delegates, which are generally “downward” (non-escaping).
