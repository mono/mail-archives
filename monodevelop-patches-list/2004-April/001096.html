+    <h1>MonoDevelop 0.3</h1>
+    
+    <p><a href="http://www.monodevelop.com">MonoDevelop</a> 0.3 has been
+      released. MonoDevelop is a GNOME IDE primarily designed for C# and
+      other .NET languages. You can always find current info 
+      <a href="http://www.monodevelop.com/">here</a>. There is also a nice 
+      tutorial <a href="http://www.monodevelop.com/tutorial.aspx">here</a>.
+      
+    </p><h2>Installing MonoDevelop</h2>
+    
+    <h3>Getting setup</h3>
+    
+    <p>You can get detailed instructions on how to build MonoDevelop 0.3 from <a href="http://www.monodevelop.com/tutorials/package_install.aspx">here</a>.</p>
+      
+    <p>You can download MonoDevelop 0.3 source from <a href="http://www.go-mono.com/archive/monodevelop-0.3.tar.gz">here</a>. The <a href="http://www.go-mono.com/download.html">go-mono</a> site may contain packages for your distro.</p>
+      
+    <h2>New Features</h2>
+
+    <h3>Gettext support</h3>
+
+    <p>MonoDevelop now contains the ability to be fully translated like any
+      other GNOME application. Currently we are shipping french and japanese
+      translations, with more to come soon.</p>
+    
+    <h3>Monodoc Integration</h3>
+    
+    <p>Monodoc documentation is now built into your code completion database
+      to provide documentation in tooltips, and in other varied locations.</p>
+    
+    <h3>Code Completion</h3>
+
+    <p>Numerous enhancements have been put into code completion for this
+      release. Some of them are graphical (XCode style type ahead), however
+      most are functional.</p>
+
+    <p>The Control+Space keybinding has been greatly improved in this release.
+      Now, local variables are included in the completion list.</p>
+    
+    <h3>Java project support</h3>
+    
+    <p>MonoDevelop's support of Java projects has greatly improved. 
+      !!Jluke, can we get some more info here?!!
+    </p>
+
+    <h3>Nemerle project support</h3>
+
+    <p>MonoDevelop now includes support for <a href="http://www.nemerle.org">Nemerle</a> projects.</p>
+    
+    <h2>Coming Soon</h2>
+    
+    <h3>Enhanced Project Support</h3>
+    
+    <p>There has been some discussion, which has resulted in
+      <a href="http://primates.ximian.com/~miguel/texts/md-build.html">this</a>
+      beta document to start describing a new build setup. Note, there is no
+      intention of breaking compatability with #D or #D projects.</p>
