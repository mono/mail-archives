What platform are you running this on?  It looks like you have =
Windows-style
drives and paths, which suggests you're on Windows.

How are you executing the program?  Via "mono program.exe" or just
"program.exe"?

I suspect you're just doing "program.exe", and you DO NOT have .NET
installed.  If this is the case, the reason is because .NET assemblies =
are
PE-formated object files, so Windows attempts to load the .exe as a =
normal
program.  One of the dependent libraries is mscorlib.dll, and if you =
don't
have .NET installed, (1) this library won't be found, and (2) the =
program
won't launch, which is what you're seeing.

Bottom line, I don't think  this is a P/Invoke issue, it's an =
installation
issue.

Fixes: install .NET, run your program as "mono program.exe", or =
contribute
support to Mono so that Mono behaves like .NET on Windows (allowing you =
to
use "program.exe" to launch your app).
