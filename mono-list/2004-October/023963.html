Can't help you with gtksourceview; I've used, and liked, ulb-gnome on
SusE 9.1 and had the same problem.  On the other, do an "echo
$PKG_CONFIG_PATH" to see if it includes the directory where
bonobo-activation.pc lives.  If not, add it and export it. However,
I've seen this error persist even when PKG_CONFIG_PATH is correct.
