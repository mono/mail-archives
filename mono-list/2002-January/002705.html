Hello,

If you want a little bit more information try:
http://securityresponse.symantec.com/avcenter/venc/data/w32.donut.html

While this is the first virus to specifically attempt
to identify and infect CLR files, it isn't really an
MSIL virus.  That is (as can be seen by the prefix
W32), it is really just a Win32 ASM virus which copies
a block of MSIL (and metadata) code into the CLR file.

The MSIL itself is not the replication routine at all.
