I think this is a bug.  If so, please tell me what category to file it under
and I'll put it in Bugzilla.  This happened on this morning's CVS of mono.

/usr/bin # mono gacutil.exe -i /tmp/SomeStrongNamedAssembly.dll
Option /tmp/SomeStrongNamedAssembly.dll takes 1 argument
/usr/bin #

Workaround:

/usr/bin # cd /tmp
/tmp # mono gacutil.exe -i SomeStrongNamedAssembly.dll
SomeStrongNamedAssembly.dll installed into the gac (/usr/lib/mono/gac)
/tmp #

It looks as if using a full path to point to the assembly to add to the GAC
breaks it.  You have to have it in the current directory?

I'd be happy to write a patch for this if this is a bug.
