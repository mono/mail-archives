Actually, unless you're not using Apple's X11 or have an esoteric 
configuration on your X server, there's no problem: the exact command 
line executed is

  clear && export DISPLAY=:0.0 && "/path/to/mono" "/path/to/exe.exe" && 
exit 1

(the exit 1 prevents Terminal from closing the window on program 
termination.) The DISPLAY is all that's needed on a factory Apple X11 
installation.

Another thing I forgot to mention: this uses Cocoa bindings to store 
and show preferences and as such MIGHT not start on 10.2 and I'm quite 
sure it won't start except in late versions of 10.2 (10.2.6-10.2.8). It 
will work fine on any 10.3 box. The day I'll stop using Apple's 
programmer aids and write real code, it'll work on older 10.2 boxes 
too.
