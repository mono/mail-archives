Again, yes. To compile blabla.cs, you do not need to
install the mcs package.

>> To get mono running, you just need this package.
>>
>> The mcs package contains the class library source
>> code, and you need it only if you want to compile
>> the class library by yourself.
> Why would someone want to do this?

The mcs package contains the source code of the C#, VB.NET,
and ILASM compilers, the resource generator, the linker, ...
and of course the class library. Bug fixes, modifications,
and additions take place here. So if you want to modify one
of the component listed above, you need to recompile the
mcs package, but most of the times, mono users do not need
to modify and recompile it.

I hope now is clearer the difference between the mcs and the
mono package.
