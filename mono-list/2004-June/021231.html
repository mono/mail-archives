I ran into a different problem when running MOD_MONO, but the same exact
problem initially when running xsp. I had installed all of mono beta 2
except for mod_mono (since it wasn't on red-carpet) via red carpet on a
fresh install of SUSE 9.1. After reading your message, I copied the
ICSharp zip lib dll into my test/bin folder, but then got the error
message that there were support assemblies it couldn't find.

After digging around for awhile, I determined that my issue is a path
issue. my MONO_PATH environment variable was empty, so I couldn't find
any assemblies from the GAC or anything else.

as root I did

export MONO_PATH /usr/lib:/usr/local:/usr/local/bin:/usr/lib/mono/1.0

The next time I started xsp (as root) it started up and I was able to
access the samples.

Does anyone know how/where I need to include these environment variables
in SUSE 9.1 so that I can use mod_mono and xsp without having to declare
them after bootup?

regards,
