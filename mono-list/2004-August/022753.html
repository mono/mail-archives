Hi Abe,

Maybe your test.dll requires a VisualC.dll or another dll which isn't =
available? Most often this happens if you've compiled a DLL by VS.NET =
and managed C++. In this case, the error message in the page is saying =
(wrongly) that the dll wouldn't be there. But in reality, the XSP writes =
on console the correct error message: that this dll would require an =
additional DLL.
