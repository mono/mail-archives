Does anyone have any passionate feelings about the usefulness of
the samples/ directory in CVS module monkeyguide? It's currently got
a number of useful examples, but the Makefiles seem to be windows-specific.
This isn't exactly a problem, but it makes it a little harder for
developers on Linux to get the samples up and running. Also, since the 
public face of the monkeyguide module is mostly the Mono Handbook as 
seen in Monodoc, I'm concerned that that code may not be reaching
the people who need to see it. 

I'm considering putting more sample code in the actual chapters of the 
handbook as part of the major edit I'm doing. If you're new to Mono, 
I'd love to hear from you about your use of sample code and what you'd 
like to see more samples of in the documentation.
