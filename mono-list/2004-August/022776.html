> Try to do 'export GDIPLUS_NOX=1' and then run the application again.
> This define prevents libgdiplus (the core of System.Drawing on Linux)
> from openning the current X display.
> 
> I hope it helps.
> 
> Jordi,

It worked perfectly! Thanks.
