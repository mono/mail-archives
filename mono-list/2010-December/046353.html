> I find MonoDevelop to be much lighter on resources than VisualStudio 2008
> but 2010 actually runs well on more constrained hardware.  I was

I also found that VisualStudio 2010 Express to be a lot more performant than 2008. Definitely worth the look if you have not yet. I did however have one issue on an old low powered laptop, which turned out to be graphics driver related. I believe it was due to WPF using Direct X, and so you need to make sure you have the latest drivers installed to minimise issues.

Only just started looking at MonoDevelop myself, but so far resource wise it seems very low on the same low powered laptop. My only concern seems to be GTK# related, in that the redraw seems to be very sluggish. However the benefit of having all the additional features in MonoDevelop is starting to win through.
