On 14.12.2010 13:07, noelkennedy wrote:
>
> I'm a bit confused about how the LGPL 2.0 license for the mono runtime works.
> I apologise if this question has already been asked somewhere else, but I
> couldn't find any unambiguous answers to my specific question.
>
> Would using Parallel Environments on client's installations violate the
> terms of the LGPL license?

If I understand correctly, your application seems to be a managed
one. In this case it's even *hard* to violate the LGPL because
you don't need to modify or statically link the Mono runtime
(the only part of Mono that's LGPL).

The only error you could commit evolves along how you're redistributing
a LGPL software, but this is pretty clearly explained by the LGPL
itself.
