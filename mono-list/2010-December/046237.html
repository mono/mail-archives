You're talking about "DbMetal" right?

It sounds like the context name is provider-specific.  I use DbLinq
for Postgres and it does use the name of the database.  You might,
instead, try outputting a .dbml file first, edit it, and then use that
as your mapping file.  Unfortunately that means when your DB schema
changes you have to a) do that process each time, b) maintain the
.dbml file manually, or c) write a script that does those steps
automatically.

Lastly, there's a chance the mapping file won't even work.  DbMetal is
pretty limited in its functionality; case in point, I was unable to
get the /entitybase switch working.

Good luck.
-Abe
