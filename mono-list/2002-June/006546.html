>I'm building a Java VM for .NET and would like to invite anyone
>who's interested to participate.

What advatages would the JVM for .NET have?
I only see, that a JVM would be very slow.

At first your JVM-JITter creates IL-code, and then the .NET-VM creates 
native code. I think, this needs a lot of time.

If you want to run .class-files fast, there existings Suns JVM or Kaffe and 
the other JVMs written in C/C++.

And if you want to use Java-Classes in .NET you can use JILC 
(http://jilc.sourceforge.net/), which converts Java-Binariews to 
.NET-Binaries.

But where is the advantage to compile Java-Binaries Just-In-Time to 
.NET-Binaries?
