In other words, if you want portable code then Don't. P/Invoke. Ever.
(Unless you know *exactly* what you are doing and know *exactly* how
to make it work portably.)

If you have not yet read this, it is an excellent guide to making sure
that your code is portable, even managed code:
<http://mono-project.com/Guidelines:Application_Portability>.  Things
like "don't hardcode \ as a path separator" seem obvious in hindsight
but can trip you up at first.  This is a fairly complete list of
common things that are obstacles to running .NET apps on Linux.
