>Hello group
>
>can we configure apache2 on windows to use mod_mono and hence avoid using
>xsp ?  i believe earlier it was not possible to do it (gathered this from
>archives ) .  can anyone point me to any relevant documentation.

No, mod_mono.c has not been ported to apache on windows, but if you make
it compile and use a TCP socket instead of a unix socket, it should work
fine. BUT it will still use xsp's cousin: mod-mono-server.exe to perform
most of the work involved in server asp.net pages.
