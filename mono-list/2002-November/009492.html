Have a look at
http://radio.weblogs.com/0109845/
The author of this program wants to create a JVM running on the .net-VM.
I like the idea, because then only the .net-VM (in our case the mono-vm) 
must been ported to the different platforms.
If you want to create something like JNBridge, then the platform needs the 
.net-VM _and_ the Java-VM. And the problem of the Java-VM is, that it is not 
OpenSource. If you port it to a new platform you are not allowed to publish 
the different steps of porting. You are only allowed to publish a Java-VM, 
which is completely ported and witch have solved the Java-Compatiblity-Kit.
