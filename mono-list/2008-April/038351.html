Hi Jose (sorry for the missing accent :P )

We did take a look at NHibernate but we did not like some issues. Or  
API is part of a bigger and larger project which we found useful for  
us, and therefore we want to released it as open source. We currently  
are far away from having a mature framework although we currently  
support 3 databases (MySQL, PostgreSQL and SQLite) and we want to work  
to provide some more. We use reflection in such a way that the SQL is  
just generated once and cache, we did this because although useful  
reflection is expensive and we wanted to avoid an over use of it.

In general the problem we try to tackle is the same one, we just had  
some discrepancies with the way NHibernate does it and decided to make  
our own framework :).

The more choices the better (some times) and we do not try to  
subtitute NHibernate since is a project that has been running from far  
longer and has more resources.
