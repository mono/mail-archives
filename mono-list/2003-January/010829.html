With some option enabled, visual studio or some other ms compiler will
create non-ECMA compliant binaries that use the '#-' heap type instead
of the standard '#~' one. The quick solution is for you to rebuild your
assembly with the correct options. The best solution would be for
someone who has the tools to do a little reverse engeneering of the
format (that should be only slightly different from the usual table
format: it should have uncompressed values and use the *Ptr tables
somehow) and document it. Or someone should get the MS people to release
the spec for the format.
So far, nobody was bothered enough by this to do anything about it.
If somebody can tell we what options to use with csc to produce such
binaries, I can have a look at implementing the support in mono to load
such non-standard heaps.
