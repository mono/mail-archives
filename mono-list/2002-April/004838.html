On Sun, 2002-04-07 at 22:41, Erik Bågfors wrote:
> > * then, we'll implement System.Data.OleDB using libgda (only libgda, not
> > libgnomedb at all) on Unix/linux, and on OleDB on windows.
> 
> What's the reason not to use gnomedb? Is there something wrong with
> gnomedb or is it just to keep the gnome-dependencies off mono???
> 
libgnomedb is a widget library, based on libgda. So, although we could
use it in the UI database stuff in Mono, it might not make too much
sense, since that UI part might be, AFAIK, not database-specific, so
adding there an "unneeded" feature (can be done without the dependency)
might not be a good idea.

Anyway, we'll see what to do when we get to implement UI stuff for
System.Data.*. But before that, we must finish all the basic data access
stuff, for which we only need libgda.
