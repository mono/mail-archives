Exists a security feature in .NET that diferentiates from where a assembly 
is being loaded, and then imposes restrictions set in a policy.

Default policy, goes somewhat like that:
Local assembly files run nearly unrestricted.
Local Network assembly files can´t run lots of system classes, for example, 
file I/O.
Internet assembly files are the most restricted ones.

It may be the cause of your problem...
