But the java design model doesn't fit to good with the C# framework design. I think we should go with the ASP.Net and IIS design model, we only make it fit perfectly to other hosts also. 
 
My goal is to give other platforms the ASP.Net support, including all page directives and likewise. I looking forward to be able to run my ASP.Net apps on Solaris or any OS that our customers want to use. 
 
The only reason why I want to implement a webserver is to support SOAP and HTTP/XML in normal apps and the best way to go is to support the same infrastructure as the high end server part like Apache. Therefore I think we should keep the webserver simple.
 
I will post my server in a few days and you will see what I mean.
