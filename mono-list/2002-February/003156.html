Hi

I checked in my internal call tool - it's in mcs/tools/ictool. Just cd
into there and run make (I should really set up an nant file for the
tools directory).

Make sure you have the *latest* class libraries compiled, unless
you're particularly interested in seeing the MS runtime die a horrible
death at the hands of a fatal execution engine error ;)

The sample config file in the ictool directory is ictool-config.xml.
I've set this up to generate the icall headers that are currently
needed by the files in mono/metadata. You can alter this if you need
different mappings, etc. When you're done, just run ictool.exe and it
will generate the three C output files.

This is pretty experimental at the moment, but I'd appreciate feedback
on its results.

Thanks,
