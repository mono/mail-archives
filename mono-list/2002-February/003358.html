On 02/19/02 Ahmad Baitalmal wrote:
> Would the runtime change if the class library changed?
> I guess what I'm asking, could someone post the runtime binaries, or do
> I need to recompile the runtime also if the class library changes?

There are some special classes in corlib that the runtime needs to
know in detail how they work: Type, Array, String, Exception, all the
Reflection ones... So if there are changes in either corlib or the
runtime related to these classes, they need to be kept in sync.
