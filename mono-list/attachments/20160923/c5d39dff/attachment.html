<a href="http://ASP.NET" target="_blank">ASP.NET</a> ServiceStack (3.9.72) application and mod-mono-server. I am running the application on Debian 8.5 with Apache 2.4.10 and mono 4.6 with Debian packages from
<a href="http://mono-project.com" target="_blank">mono-project.com</a>. Everything works as expected except for a shutdown of the mod-mono-server. The shutdown occurs either on apache restart, apache stop or mod-mono-server –terminate. My application is shutdown
 properly (Application_End is executed), but then the mono process goes into a CPU loop. In case of apache restart a new process is spawned afterwards, but the hanging process remains. The hanging process can only be killed with kill -9.<o:p></o:p></p>

<p><span style="font-family:Symbol">·</span><span style="font-size:7.0pt">

ASP.NET</a> example application <o:p></o:p></p>
<p><span style="font-family:Symbol">·</span><span style="font-size:7.0pt">        
</span>The hang occurs only after my application was loaded by mod-mono-server, i.e. at least one request to the application is needed before the shutdown<o:p></o:p></p>
<p><span style="font-family:Symbol">·</span><span style="font-size:7.0pt">        
</span>mod-mono-server logs a “mod-mono-server received a shutdown message” message before the hang<o:p></o:p></p>
