<DIV>We use the NLDAP-C# quite heavily. As an FYI, the NLDAP-C# was created by Novell (~2003)&nbsp;in a different repository around the time Ximian was acquired.</DIV>
<DIV>&nbsp;</DIV>
<DIV>The version of NLDAP-C# in the mono repo was forked and had some Java "stuff" added to it.&nbsp; Not all of the bug fixes added to the Novell&nbsp;repo made it back into the mono repo after the fork.&nbsp; The mono repo could use a refresh.</DIV>
<DIV>&nbsp;</DIV>
<DIV>An example of that is your reported bug.&nbsp; It does not exist in the Novell&nbsp;copy.</DIV>
<DIV>&nbsp;</DIV>
<DIV>Comparing the Mono&nbsp;SVN copy of the code to the Novell CVS repository...</DIV>
<DIV>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Array.Copy((System.Array) SupportClass.ToByteArray(utf8Bytes), 0, (System.Array)SupportClass.ToByteArray( octets), iOctets, utf8Bytes.Length);<BR>to</DIV>
<DIV>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Array.Copy((System.Array) (utf8Bytes), 0, (System.Array) octets, iOctets, utf8Bytes.Length);<BR></DIV>
<DIV>As for the C# naming guidelines... the entire project source is fubared.... probably best to leave alone.</DIV>
<DIV>&nbsp;</DIV>
<DIV>You can find the Novell copy here and perform a&nbsp;sync to take the bits you want.</DIV>
