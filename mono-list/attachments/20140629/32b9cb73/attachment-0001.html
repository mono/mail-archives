<p>Is SPARC not supported by Mono any more?</p>

<p>I finally got the 3.4.0 runtime to build (many thanks to Steve Hall!), but when I try to run anything with it, I get:</p>

<pre>Assertion: should not be reached at method-to-ir.c:2566</pre>

<p>Now, that is in this method:</p>

<pre>
static void
set_rgctx_arg (MonoCompile *cfg, MonoCallInst *call, int rgctx_reg, MonoInst *rgctx_arg)
{
#ifdef MONO_ARCH_RGCTX_REG
        mono_call_inst_add_outarg_reg (cfg, call, rgctx_reg, MONO_ARCH_RGCTX_REG, FALSE);
        cfg->uses_rgctx_reg = TRUE;
        call->rgctx_reg = TRUE;
#ifdef ENABLE_LLVM
        call->rgctx_arg_reg = rgctx_reg;
#endif
#else
        NOT_IMPLEMENTED;
#endif
}       
</pre>

<p>mono/mini/mini-sparc.h does not define <b>MONO_ARCH_RGCTX_REG</b>... so we can't use JIT. But it also doesn't define <b>MONO_ARCH_LLVM_SUPPORTED</b>... so we can't use LLVM either!</p>

<p>Am I missing something obvious here, or is support for SPARC dead?</p>
