I just wanted to write Miguel et. al and thank everyone for the amazing
product that is Mono!<br>
<br>
My company, Obtuse Software, recently rolled out their online store which
is 100% powered by Mono (you can see the Mono badge proudly displayed at
the bottom of every page). &nbsp;Some key points about the site and its
Mono utilization:<br>
<br>
o PostgreSQL database backend connected by Npgsql.<br>
o Webforms authentication with various Web.config &lt;location&gt;s.<br>
o Logging via log4net.<br>
o Google Checkout integration using the Google Checkout .Net API.<br>
o The purchased software is built on-the-fly by kicking off an external
NAnt build process. &nbsp;NAnt builds both a license assembly and the NSIS
Windows installer.<br>
<br>
The whole system is all Linux (CentOS) / all awesomeness. &nbsp;This is
my first production use of Mono and it has been a great joy. &nbsp;Mono's
ASP.NET has been greatly improved since my last failing attempt at a Mono
site (somewhere back around Mono 1.1.x). &nbsp;Mono sensing updates to
assemblies, .aspx files, and the Web.config is a big help.<br>
<br>
Now, of course, the entire experience wasn't w/o problems:<br>
<br>
o log4net's config doesn't work when embedded into the Web.config file.
&nbsp;I had to use a separate config file and use log4net's XmlConfigurator.Configure(FileInfo
pathToConfig)<br>
o It's difficult to predict the root directory of the webapp. &nbsp;I'm
using Mono's ASP.NET auto-configuration so I *think* what happens is Mono
configures the &quot;~&quot; path (as in, Server.MapPath(&quot;~&quot;))
relative to the first .aspx page that's loaded. &nbsp;But I probably can
just fix this by specifically configuring the webapp in the Apache config
file.<br>
o We're using https for our site and have found that Mono cannot handle
loading the same webpage using both http:// and </font><a href=https:///><font size=3 color=blue><u>https://</u></font></a><font size=3>
&nbsp;For example, if I load </font><a href=http://www.mysite.com/default.aspx><font size=3 color=blue><u>http://www.mysite.com/default.aspx</u></font></a><font size=3>,
and then later </font><a href=https://www.mysite.com/default.aspx><font size=3 color=blue><u>https://www.mysite.com/default.aspx</u></font></a><font size=3>,
it barfs. &nbsp;It looks like it stays configured to whatever was first
loaded (if http first, then only http works / if https first, then only
https works). &nbsp;Now this is also not a big deal since I just need to
stratify the webapp into separate http and https pieces and use Apache
to setup different VirtualHosts.<br>
o Mono doesn't let you change around the .aspx page on-the-fly like MS
ASP.NET does. &nbsp;For example, if you have a button with an OnClick method
like: &nbsp;&lt;asp:Button id=&quot;fooButton&quot; OnClick=&quot;FooClick&quot;
runat=&quot;server&quot; /&gt; you cannot remove the OnClick attribute
w/o recompiling on Mono, MS you can.<br>
o Had to use the deprecated ServicePointManager.CertificatePolicy method
for SSL communication with Google. &nbsp;Mono does not yet implement the
newer method using ServerCertificateValidationCallback.<br>
<br>
Other than these few small hindrances, Mono worked great. &nbsp;Thank you,
again, for such a quality product!<br>
