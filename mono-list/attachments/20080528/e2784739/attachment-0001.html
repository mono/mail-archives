="http://www.mysite.com/default.aspx">http://www.mysite.com/default.aspx</a>, and then later&nbsp;<a href="https://www.mysite.com/default.aspx">https://www.mysite.com/default.aspx</a>, it barfs. &nbsp;It looks like it stays configured to whatever was first loaded (if http first, then only http works / if https first, then only https works). &nbsp;Now this is also not a big deal since I just need to stratify the webapp into separate http and https pieces and use Apache to setup different VirtualHosts.<br>o Mono doesn't let you change around the .aspx page on-the-fly like MS ASP.NET does. &nbsp;For example, if you have a button with an OnClick method like: &nbsp;&lt;asp:Button id="fooButton" OnClick="FooClick" runat="server" /> you cannot remove the OnClick attribute w/o recompiling on Mono, MS you can.<br>o Had to use the deprecated ServicePointManager.CertificatePolicy method for SSL communication with Google. &nbsp;Mono does not yet implement the newer method using ServerCertificateValidationCallback.<br><br>Other than these few small hindrances, Mono worked great. &nbsp;Thank you, again, for such a quality product!<br><br>-Abe<br><br>PS - If you're interested in what Obtuse Software does, you can checkout the press release here: &nbsp;<a href="http://www.obtusesoft.com/pr.html">http://www.obtusesoft.com/pr.html</a>&nbsp;&nbsp;zigGIS (Obtuse Software's inaugural product) is an Open Source GIS extension to ESRI ArcMap.</body></html>
