<p style="margin: 0.0px 0.0px 0.0px 0.0px; font: 13.0px Arial">I tested the forum page containing Dave's message (<a href="http://go-mono.com/forums/#nabble-td3096414%7Ca3096611"><span style="text-decoration: underline ; color: #0166cc">http://go-mono.com/forums/#nabble-td3096414%7Ca3096611</span></a>) in Safari 5.0.3 (6533.19.4), Google Chrome 8.0.552.231, Firefox 3.6.12 and Opera 11.0 build 1156 all on my Mac running OS X 10.6.5. &nbsp;I wanted to see if the "mono" link is hot in any of them and what the cause might be. The "mono" link was hot only in Opera. The link was inaccessible in Safari, Chrome and Firefox.</p>
<p style="margin: 0.0px 0.0px 0.0px 0.0px; font: 13.0px Arial; min-height: 15.0px"><br></p>
<p style="margin: 0.0px 0.0px 0.0px 0.0px; font: 13.0px Arial">After looking at the HTML source for that page I was able to see that the real problem is slightly different. (The "mono" link actually is hot as you can see in the code snippet below.) The real problem is that the "mono" link is <i>inaccessible</i> because it is being eclipsed by the div that contains the word "Forums" shown just above the "mono" link. &nbsp;The solution that works on all 4 browsers I tested is quite simple. In the code snippet below on line 2 change&nbsp;<span style="font: 13.0px Courier">&lt;div id="content-header"&gt;</span>&nbsp;to&nbsp;<span style="font: 13.0px Courier">&lt;div id="content-header" style="z-index:-1"&gt;</span>. Adding the&nbsp;<span style="font: 13.0px Courier">style="z-index:-1"</span>&nbsp;attribute will make sure all browsers render that div behind the "mono" link thus making the "mono" link always accessible.</p>
<p style="margin: 0.0px 0.0px 0.0px 0.0px; font: 13.0px Arial; min-height: 15.0px"><br></p>
<p style="margin: 0.0px 0.0px 0.0px 0.0px; font: 13.0px Arial">Now that I've diagnosed the bug and provided a solution how can I get the solution incorporated into the forums code?</p>
