Francois, <div><br></div><div>You cannot debug managed code running in mono from within VS. You could use MonoDevelop and the <a href="http://www.mono-project.com/Mono:Runtime:Documentation:SoftDebugger">mono soft debugger</a> to debug the managed code at the same time that you are debugging the native code in VS. If using MonoDevelop, I believe you will also need some minor 'hacks' to make it possible to debug an embedded mono. See detail in the question here: <a href="http://stackoverflow.com/questions/9582365/debugging-w-embedded-mono-how-to-set-breakpoints-in-c-sharp-code">http://stackoverflow.com/questions/9582365/debugging-w-embedded-mono-how-to-set-breakpoints-in-c-sharp-code</a></div>
<div><br></div><div>Also, if you are using the soft debugger at the same time as being attached with the native VS debugger you may run into issues with SEH exceptions being raised internally by the soft debugger. See solution I mentioned <a href="https://bugzilla.xamarin.com/show_bug.cgi?id=5963">here</a> about using AddVectoredExceptionHandler rather than SetUnhandledExceptionFilter.</div>
