<div dir="ltr">Hi!<div><br></div><div style><a href="http://ASP.NET">ASP.NET</a> MVC 3 works fine on Mono, as does <a href="http://ASP.NET">ASP.NET</a> MVC 4. I haven't tried using MonoDevelop as I do development in Visual Studio, but deployment to a Linux server running Mono 3.0.2 works fine.</div>
<div style><br></div><div style>That exception generally occurs when you're using Microsoft's version of Microsoft.Web.Infrastructure.dll. Microsoft's version relies on internals of IIS, and hence Mono has its own implementation. You need to either:</div>
<div style>1) Delete the Microsoft.Web.Infrastructure.dll file from the bin directory manually when running under Mono; or</div><div style>2) Set "Copy Local" to false for the Microsoft.Web.Infrastructure reference.</div>
<div style><br></div><div style>I don't think System.Web.Helpers is bundled with Mono - Just use Microsoft's version. Set Copy Local to True on the reference, or just copy it from a Windows machine (should be at C:\Program Files\Microsoft <a href="http://ASP.NET">ASP.NET</a>\<a href="http://ASP.NET">ASP.NET</a> Web Pages\v2.0\Assemblies\System.Web.Helpers.dll) into your bin directory.</div>
