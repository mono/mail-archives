<P><FONT size=2>I think the following DRM protection would be a lot easier and much safer: (but it requires that your end user has Internet access when they start your application)</FONT></P>
<DIV dir=ltr id=idOWAReplyText88976>
<DIV dir=ltr><FONT color=#000000 size=2>The Roozz pluign has a feature where it can download the main .exe file to memory and launch the application from memory, thus the main .exe file is never saved to disk.&nbsp;This way&nbsp;you can make sure that only people with the right credentials get access to your application. Also if you have some secrete code in that main .exe file then it is a little harder for hackers to reverse engineer that file, since it is never saved to disk. (but of cause&nbsp;they can make a memory dump and dissect it)</FONT></DIV>
<DIV dir=ltr><FONT size=2></FONT>&nbsp;</DIV>
<DIV dir=ltr><FONT size=2>To avoid that crackers make illegal copies of your product and distribute it,&nbsp;you will have to pay a little extra for the RoozzDRM product, which will mutate the main .exe file continuously.&nbsp;And the main .exe file will communicate back to the server to make sure it is the this .exe file is indeed from&nbsp;you and not some cracked version.</FONT></DIV>
