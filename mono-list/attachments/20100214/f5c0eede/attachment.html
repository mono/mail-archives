<p class=MsoNormal><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'>Rather than just looking for System.Core MonoDevelop should
really look for the System.Runtime.CompilerServices.ExtensionAttribute in any
of the referenced assemblies.&nbsp; You can define this type in your own
assembly and not reference System.Core and still use extension methods.&nbsp;
We do this in IronPython so we can run on .NET 2.0SP1 and above but compile w/
the 3.5 compiler. <o:p></o:p></span></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p>I guess MonoDevelop should not provide completion for extension methods when
System.Core is not referenced. MD should also reference System.Core by default
in new 3.5+ projects. Please file bugs. Arguably gmcs should give a better
error message too, so please file a separate bug against gmcs for that.<o:p></o:p></p>

<blockquote style='margin-top:5.0pt;margin-bottom:5.0pt'>

<p class=MsoNormal>On Feb 13, 2010 11:37 AM, &quot;Abe Gillespie&quot; &lt;<a
href="mailto:abe.gillespie@gmail.com">abe.gillespie@gmail.com</a>&gt; wrote:<br>
<br>
Not sure if this is a bug or just some sugar that <a href="http://VS.NET"
target="_blank">VS.NET</a> does. &nbsp;If you<br>
have extension methods in one assembly and then use those extension<br>
methods in another, if you do not reference System.Core then the<br>
compiler can't find those methods. &nbsp;I think the big problem is it's<br>
hard to diagnose. &nbsp;MonoDevelop will happily offer code-completion for<br>
those extension methods. &nbsp;However, when you compile, it fails.<br>
Unfortunately the error info doesn't give you any sort of clue that<br>
it's a missing System.Core reference and *NOT* a missing method.<br>
<a href="http://VS.NET" target="_blank">VS.NET</a> will happily offer
code-completion *and* compile. &nbsp;Example of<br>
an error message:<br>
<br>
Messenger.cs(34,34): Error CS1061: Type<br>
`Dimebrain.TweetSharp.Fluent.IFluentTwitter' does not contain a<br>
definition for `AuthenticateAs' and no extension method<br>
`AuthenticateAs' of type `Dimebrain.TweetSharp.Fluent.IFluentTwitter'<br>
could be found (are you missing a using directive or an assembly<br>
reference?) (CS1061) (Infrastructure)<br>
<br>
The last part - &quot;are you missing a using directive [...]&quot; - is<br>
misleading because that makes me think I need to add a reference to<br>
TweetSharp. &nbsp;But what it really needs is the System.Core reference.<br>
<br>
Hope that's useful.<br>
