Adding my 2c: I'm running a medium sized (about 30 DLLs, about 220 classes, not counting the ORM engine) web application with mono 2.10.8 on Linux (Debian 6.0) and PostgreSQL 9.1. The application uses WebForms and AjaxControlToolkit. It runs flawlessly, I regularly develop with VS Express and run it under Mono (compilation is done on-site on the server with a NAnt script). The only problem is that I had to target Framework 3.5 since the 4.0-targeted version of AjaxControlToolkit had several problems under Mono, but since the application is stable and I don't need 4.0-level features in my project, I didn't care. When I have time, I'd like to try Framework 4.0 + Mono 3.0.x and see what happens.<div>

<br>
Mono 2.10.8 (current stable release) is compatible (green color on <a href="http://mono-project.com/Compatibility" target="_blank">http://mono-project.com/Compatibility</a> ) with ASP.Net MVC 1, 2, and 3.  If you're planning to run it on Linux, you'll probably use Apache and mod_mono or Mono FastCGI.<br>
