SQLite essentially has three data affinities (see <a href="http://sqlite.org">sqlite.org</a> for more information on this): TEXT, REAL and INTEGER.&nbsp; Any datatype in your CREATE table definition that is not REAL or does not begin with INT (INT and INTEGER mean the same thing) will be assigned a TEXT affinity.&nbsp; Bear in mind that type affinity works differently in SQLite than strict datatypes do in other RDBMS&#39;s.&nbsp; You can always store any value in an column in SQLite.&nbsp; The only way to get the exact syntax from your CREATE TABLE statement back is to query the sqlite_master table like this:
<br><br>SELECT * FROM sqlite_master WHERE name = &#39;my_table_name&#39;;<br><br>Alternatively you can use:<br><br>PRAGMA table_info(&#39;my_table_name&#39;);<br><br>to get each column number and name, along with the datatype that you passed to CREATE TABLE (not the affinity), nullability, default value and a flag indicating whether or not it is a primary key column.
<br><br>Otherwise your managed <a href="http://ADO.NET">ADO.NET</a> provider will always report the type affinity rather than the string that you passed in your CREATE TABLE statement.&nbsp; Hope this helps.<br><br>Dan Poage<br>
