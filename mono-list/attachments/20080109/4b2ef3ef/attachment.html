Hi everyone! <br> <p>I&#39;m happy to announce the release of Specter 0.8, a behavior-driven <br> development framework for the CLI, powered by Boo meta-programming <br> capabilities.</p><p>Wonder what does it look like ? Check out a specification example at : 
<br> <a target="_blank" rel="nofollow" href="http://specter.sf.net/examples/MiniBarSpec.boo.html">http://specter.sf.net/examples/MiniBarSpec.boo.html</a> <br></p>(both specification and implementation are in Boo in this example, however one can specify C# implementations or any other .NET language).
<br><br><br>New in this release : <br><p> - works with recent Boo releases (actually Boo 0.8.0.2742 or superior is required)<br> - a spec can be compiled as a standalone runner assembly [1] <br> - specter-console tool. <br>
 - subject macro to mark which objects are the subject of your specification <br> moreover allowing the spec to be compiled even if the object doesn&#39;t <br> actually exist yet : subject foo = MyObject() [2]. <br> - a priority can be set on a context (context &quot;context description&quot;, 100: 
<br> ) &nbsp;[3] <br> - actual vs expected reports [3] <br> - human-friendly error messages for the most common spec syntax errors. <br> - nunit dependency is loaded automatically only if needed (no more &quot;import <br> NUnit.Framework
&quot; or booc -r:nunit.framework). <br> - Specter can be installed into the GAC. <br> - build system now based on NAnt instead of #develop. <br> - new homepage and a mailing-list. <br> </p><p>Try it while it&#39;s hot : 
<a target="_blank" rel="nofollow" href="http://specter.sf.net/">http://specter.sourceforge.net/</a> <br> </p><p>[1] when a spec is compiled as an executable it will compile as a standalone <br> runner with no nunit dependency. If compiled as a library it will compile as 
<br> a nunit testsuite (for integration with existing third-party tools) ... both <br> can be run through specter-console though. <br> [2] the spec will compile and will throw a SubjectTypeNotFoundException or <br> SubjectConstructorNotFoundException at run-time if the type or specified 
<br> subject/constructor is not implemented yet. <br> [3] works when the spec is ran through standalone runner or specter-console. <br> </p><p>Note to users of previous versions: you need to rename your <br> imports/references from Specter to 
Specter.Framework <br> </p><p>Special thanks to the original author of Specter, Andrew Davey, who trusted <br> me to take care of the baby. <br> Thanks to Jeffery Olson for helping in testing. <br> </p>Keep tuned on the mailing-list at : 
<br> <a target="_blank" rel="nofollow" href="http://groups.google.com/group/specter-framework/">http://groups.google.com/group/specter-framework/</a><br><br><br>
