If you need the same numbers for a given seed, why are you using a random number generator? If you need reproducible pseudo-random numbers then you could just SHA1 has your seed, which gives you 20 bytes (4 integers). Then if you need more numbers, SHA1 hash the result from the previous operation to get a new set of 20 bytes (4 integers) and keep going until you have all the numbers you need. This is guaranteed to produce the same psuedo-random numbers on every platform every time.
