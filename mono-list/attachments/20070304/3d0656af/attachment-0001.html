Hi again José:<br>Ok, let&#39;s say you have a file named &quot;MyBitmap.bmp&quot; and you have embedded it into the assembly.<br><br>The firt you need to do is to get a reference to the assembly. If you are retrieving the image from the same assembly where the code is executing, you can use:
<br><br>Assembly as = Assembly.GetExecutingAssembly();<br><br>With the assembly, you can inspect what resources you have. The easiest way is to get a list of files embedded using:<br><br>string[] resource_names = as..GetManifestResourceNames();
<br><br>Then you can get a stream to the embedded resource you want:<br><br>Stream s = as..GetManifestResourceStream( res_name )<br><br>where &quot;res_name&quot; is a resource name that you got previously. This way you get a stream to the embedded resource. Now, you can read the stream (or do whatever you need).
<br><br>For example:<br><br>Bitmap bmp = new Bitmap( as.GetManifestResourceStream( &quot;MyBitmap.bmp&quot;) );<br><br>You need to take into account the way MonoDevelop and VisualStudio take care of embeded resources, because the names of the file of the resource does not always match the name in the resource. MonoDevelop, for example, uses the name of the resource directly (this way, the latest line of code I&#39;ve written is actually valid), but VisualStudio uses the &quot;Default Namespace Name&quot; in the project to prepent it to the resources. So you can end up with a &quot;
this.is.mi.namespace.MyBitmap.bmp&quot; instead. This is the reason to inspect resource names before you actually try to get a stream for them.<br><br>Another problem you may find, is the way to load back the resources, because you have a stream, instead a physical file. Librsvg allows you to load directly from a memory chunk, instead of a file, so try this way instead to load directly from an on-disk file.
