I solved my issue, by keeping the bufferOutput on true, but calling the Response.Flush(). I seemed, by accident, to call the Response.OutputStream.Flush() which doesn't have the same effect. Now the file is downloaded without buffering it completely in memory, and without the socket being closed (although this is still something that should not have happened, I guess).<BR>
