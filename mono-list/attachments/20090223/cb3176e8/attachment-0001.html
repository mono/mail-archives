I&#39;ll explain this by jumping to Java for a moment.&nbsp; Java&#39;s arrays (but not its generics) are broken in that they do what you want, which is broken:<br><br>Number[] numbers = new Double[1];<br>numbers[0] = 5;<br><br>
This causes an ArrayStoreException at runtime.&nbsp; C#&#39;s arrays and generics are not broken in this manner, and will give a compile error instead.<br><br>A list of strings is not a list of objects, as long as a list is mutable.&nbsp; C# 4 adds covariance and contravariance.<br>
<br>IEnumerable&lt;object&gt; objects = new List&lt;string&gt;(); will be valid.<br><br>List&lt;object&gt; more = new List&lt;string&gt;(); will not be valid.&nbsp; List will still be declared as List&lt;T&gt;, and IEnumerable will be interface IEnumerable&lt;out T&gt; { stuff }.<br>
<br>Right now I&#39;d use IEnumerable&lt;object&gt; objects = stringList.Select(x =&gt; (object)x); or if you must have List&lt;&gt;, List&lt;object&gt; objects = stringList.Select(x =&gt; (object)x).ToList();<br><br>The advantage of the IEnumerable is that no actual copying is done.<br>
