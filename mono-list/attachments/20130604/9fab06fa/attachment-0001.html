<br><br>On Friday, 16 April 2010 06:01:06 UTC-7, jonp  wrote:<blockquote class="gmail_quote" style="margin: 0;margin-left: 0.8ex;border-left: 1px #ccc solid;padding-left: 1ex;">The DbLinq[0] team is proud to announce the release of DbLinq 0.20.1,
<br>following the DbLinq 0.20 release from last week.
<br>
<br>DbLinq is a reimplementation of System.Data.Linq.dll for use with SQL
<br>servers in addition to Microsoft SQL Server.  Support is provided for:
<br>
<br>      * Firebird [1]
<br>      * Ingres [2]
<br>      * MySQL [3]
<br>      * Oracle [4]
<br>      * PostgreSQL [5]
<br>      * SQLite [6]
<br>      * SQL Server [7]
<br>
<br>Note that not all servers are supported equally: some tests will pass
<br>under some database backends while failing under others.  I maintain the
<br>SQLite and SQL Server backends, while the community handles the others.
<br>
<br>This is a minor update to address some DBML validation deficiencies
<br>found in the 0.20 release.  Fixes include [8]:
<br>
<br>      * 233: Fix IOE in Processor.<wbr>ValidateAssociations()
<br>      * 234: Fix NRE when looking up DataContext base class name.
<br>      * 235: Support <Association/>s with empty @Storage and @Member.
<br>      * 237: Empty //Table/@Name strings are not valid table names.
<br>
<br>Enjoy!
<br>
