    <div class="moz-cite-prefix">Why debugging symbols are required to
      show full stack trace? MS .NET doesn't require that.<br>
      <br>
      On 30/12/2013 20:42, David Curylo wrote:<br>
    </div>
    <blockquote cite="mid:A5AA0E8F-86F0-45AF-BD75-BC2F17FE52E4@asme.org"
      type="cite">
      <meta http-equiv="Content-Type" content="text/html;
        charset=windows-1252">
      <div>You need to tell mono to create the debugging symbols when
        building.</div>
      <div><br>
      </div>
      <div>mcs -debug ConsoleApplication2.cs</div>
      <div><br>
      </div>
      <div>You should see ConsoleApplication2.exe.mdb in the output
        directory, and now when you run with the --debug option you
        should see line numbers in the stack trace.</div>
      <div><br>
      </div>
      <div>Please keep in mind you are telling the compiler to generate
        debugging symbols, and not actually telling it to build a debug
        release.  This is the same behavior as you would see on MS .NET
        by calling the compiler as follows:</div>
      <div><br>
      </div>
      <div>csc /debug ConsoleApplication2.cs</div>
      <div><br>
      </div>
      <div>I would recommend building with this option and keeping
        debugging symbols archived with any of your releases even if you
        don’t ship them to customers by default. Then if someone
        encounters an issue and you need line numbers in the stack
        trace, you can provide the symbols for that release and get
        detailed stack traces.</div>
