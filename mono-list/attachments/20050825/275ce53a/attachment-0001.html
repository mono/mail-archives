<FONT COLOR="#000000">&gt; &gt; Is it possible to use mkbundle on a 2.6 kernel distro and compile an</FONT>
<FONT COLOR="#000000">&gt; &gt; executable which can run on a 2.4 kernel + glib combination?</FONT>
<FONT COLOR="#000000">&gt;</FONT>
<FONT COLOR="#000000">&gt; Compiling something on the glibc post-rh9 and running it pre-rh9 is</FONT>
<FONT COLOR="#000000">&gt; risky. The version numbers for some functions changed. For example, we</FONT>
<FONT COLOR="#000000">&gt; know that rh9-built mono packages will *not* run on sles8. I don't think</FONT>
<FONT COLOR="#000000">&gt; there is any way around this.</FONT>
<FONT COLOR="#000000">&gt;</FONT>
<FONT COLOR="#000000">&gt; You *can* build on 2.4 kernels and run on 2.6. (btw, I assume you meant</FONT>
<FONT COLOR="#000000">&gt; glibc not glib. glib may also have problems. Some of the newer versions</FONT>
<FONT COLOR="#000000">&gt; use macros to call functions that are only added in newer versions) The</FONT>
<FONT COLOR="#000000">&gt; safest path is to compile with the *oldest* distro you have and test on</FONT>
<FONT COLOR="#000000">&gt; *every* distro. This was the only way we felt safe building mono on only</FONT>
<FONT COLOR="#000000">&gt; one platform.</FONT>
