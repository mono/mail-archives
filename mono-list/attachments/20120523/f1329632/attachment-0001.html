I am developing server with mono using async socket model. Currently it's<br>
about 500 simultaneously working clients with 20-100 operations per seconds.<br>
The code should send messages accordingly theirs creation time, so I use<br>
such pattern:<br>
<br>
Send()<br>
{<br>
    //Dequeue message from queue. Messages placed in the queue accordingly<br>
theirs creation time<br>
    SendStateObject state=new SendStateObject();<br>
    state.buffer=Dequeue();<br>
    state.socket=client;<br>
<br>
    //some checks in the code were removed, this function is called only<br>
when no other send callbacks were ran<br>
    client.BeginSend(state.buffer, 0, state.buffer.Length, SocketFlags.None,<br>
new AsyncCallback(SendCallback), state);<br>
<br>
}<br>
<br>
private void SendCallback(IAsyncResult ar)<br>
{<br>
    SendStateObject sendState = (SendStateObject)ar.AsyncState;<br>
    Socket client = sendState.workSocket;<br>
<br>
    try<br>
    {<br>
          client.EndSend(ar);<br>
    }<br>
    catch() /*Some exceptions handling */<br>
    {}<br>
    finally<br>
    {<br>
          sendState.buffer=null;<br>
          sendState.workSocket=null;<br>
    }<br>
<br>
    if (SomeItemsInQueue()) Send();<br>
<br>
}<br>
<br>
And this code produces huge memory leaks under high load. Profiler says,<br>
that there are millions of AsyncCallback objects and<br>
System.Net.Sockets.SocketAsyncResult objects.<br>
<br>
I think, I could minimize number of AsyncCallback objects by creating it<br>
only one time in constructor and passing it to BeginSend (I'll check it<br>
later), but what to do with SocketAsyncResult (and byte buffers which it<br>
contains in)?<br>
I don't create this object, it's created inside of BeginSend and in most<br>
cases it is not claimed by garbage collector.<br>
<br>
Maybe I use wrong pattern and must remove calling Send() from the end line<br>
