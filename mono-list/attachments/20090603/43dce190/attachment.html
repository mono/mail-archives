<p class=MsoNormal>&nbsp; &nbsp;I've found that you have two choices:<br>
<br>
1. If you have source to the library you're interop'ing to, you can make the C
interface take a &quot;unsigned short*&quot; instead of &quot;wchar_t*&quot;
and it will work using the managed CharSet.Unicode for both systems. &nbsp;Of
course your C code will have cast the paramter to a wchar_t* on windows but use
a conversion function between UTF-16 and UCS-4 or UTF-8 on the non-windows
library.<br>
<br>
2. You can use C#'s custom marshal code to marshal as 2 bytes wide on windows
and 4 bytes wide elsewhere and leave your C code using wchar_t everywhere.
&nbsp;See the interfaces of System.Runtime.InteropServices.ICustomMarshaler and
the attribute &quot;[MarshalAs(UnmanagedType.CustomMarshaler,
MarshalTypeRef=typeof(YourClassDerivedFromICustomMarshaler))]&quot;. &nbsp; If
you need an example of this let me know but the MSDN documentation using these
classes should be enough for you to follow.<o:p></o:p></p>
