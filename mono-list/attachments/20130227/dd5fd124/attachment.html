<div>
<p class="MsoNormal">That said, there are a few things that matter to a company like Xamarin.   We need a brand that can be protected (for example, to avoid someone misrepresenting the products).    <o:p></o:p></p>

<p class="MsoNormal"><o:p> </o:p></p>
</div>
<div>
<p class="MsoNormal">The choice to use "Mono" as a name in the Ximian days predates a lot of the sophisticated thinking that goes into this.   I for one, liked the name, and forced it despite marketing objections and trademark objections.    <o:p></o:p></p>
</div>
<div>
<p class="MsoNormal"><o:p> </o:p></p>
</div>
<div>
<p class="MsoNormal">But "Mono" itself is a bad trademark, it is hard to defend, since it is a very common word.    <o:p></o:p></p>
</div>
<div>
<p class="MsoNormal"><o:p> </o:p></p>
</div>
<div>
<p class="MsoNormal">The second problem is that the owner for the trademark is Novell.   So while we have the rights to use the name, if some sketchy third party in the started to sell something called "MonoTouch" which was some scam, it would be hard for us
 to enforce compliance without a lot of overhead rom Novell.  <o:p></o:p></p>
</div>
<div>
<p class="MsoNormal"><o:p> </o:p></p>
</div>
<div>
<p class="MsoNormal">We do not need that overhead.<o:p></o:p></p>
</div>
<div>
<p class="MsoNormal"> <o:p></o:p></p>

 and Mono.   The results are in, and we know that growing the scope of the conference to go beyond a single implementation is much healthier, and much better for everyone.<o:p></o:p></p>

 developers and Mono developers both share an interest: more .NET software, more .NET expertise, more .NET open source, more .NET adoption.<o:p></o:p></p>
