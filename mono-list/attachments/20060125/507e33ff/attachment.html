Hi Andreas, I have posted the same question on the Axis users list and
they gave me a solution, that raise another question.<br>
The solution is to modify the WSDL file modifying the attribute
&quot;elementFormDefault&quot; of the tag &lt;element&gt; to
&quot;qualified&quot; .<br>
Actually&nbsp; in the WSDL generated by mono this attribute is called
&quot;form&quot;.<br><br>
The effect on the XML sent by Axis is the following:<br><br>
&lt;?xml version=&quot;1.0&quot; encoding=&quot;UTF-8&quot;?&gt;<br>
&lt;soapenv:Envelope
xmlns:soapenv=&quot;http://schemas.xmlsoap.org/soap/envelope/&quot;
xmlns:xsd=&quot;http://www.w3.org/2001/XMLSchema&quot;
xmlns:xsi=&quot;<a href="http://www.w3.org/2001/XMLSchema-instance" eudora="autourl">
http://www.w3.org/2001/XMLSchema-instance</a>&quot;&gt;<br>
&lt;soapenv:Body&gt;<br>
&lt;idprogetto
xmlns=&quot;<a href="http://localhost:8088/DocProviderWS/" eudora="autourl">
http://localhost:8088/DocProviderWS/</a>&quot;&gt;<br>
&lt;valore&gt;oi&lt;/valore&gt;<br>
&lt;/idprogetto&gt;&lt;/soapenv:Body&gt;<br>
&lt;/soapenv:Envelope&gt;<br><br>
As you can see there is no xmlns attribute at all in the tag valore and
this assumed by mono as to get the same xmlns of the father.<br><br>
<br><br>
The reason is that in .NET this attribute is generated as
&quot;unqualified&quot; by default&nbsp; but assumed as
&quot;qualified&quot; when a call to the web service comes.<br>
In fact scrolling the source of my web service and looking at the valore
definition I can see:<br><br>
[System.Xml.Serialization.XmlElementAttribute(Form=System.Xml.Schema.XmlSchemaForm.Unqualified)]<br>
<br>
public string valore;<br><br>
<br>
To give you the complete picture: the web service stub has been generated
from another wsdl using the &quot;wsdl.exe /server&quot; command<br><br>
