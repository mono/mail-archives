Profiling is only supported on linux.<br><br>The default profiler is not recommended on recent versions of mono (2.4 or trunk),<br>please try the new logging profiler[1], which have a work-in-progress GUI[2] for it.<br><br>

Keep in mind that under statistical profiling you need to run a long enough program<br>to get enough samples and build a meaningful result. Usually 1-15 minutes of steady<br>state execution is enough to capture enough information.<br>

<br>The new statistical profiler returns information of managed and unmanaged code<br>mixed together so if you get a result with too many frames from the mono runtime <br>I can give you a hand on figuring out the problem if I get access to the sampling file.<br>
