Hello,<BR>
<BR>
Mono 1.1.8.1 is a minor bug fix release to Mono 1.1.8, for details on 1.1.8 see its <A HREF="http://www.go-mono.com/archive/1.1.8">release notes</A>. <BR>
<BR>
<H3>
<B><FONT SIZE="4">Important Bug Fixes in 1.1.8.1</FONT></B>
</H3>
Fixed a bug in our Socket.Select implementation that was introduced in 1.1.8 which failed to work on read descriptors (Gonzalo). <BR>
<BR>
Fixed a long-standing deadlock in the runtime. This bug was only visible with heavy thread usage at startup on SMP systems but it would freeze certain applications (Beagle, iFolder and others). The bug has been finally squashed (Ben, Gonzalo). <BR>
<BR>
MacOS X and MonoPosixHelper: in Mono 1.1.8 we shipped a non-working Mono.Posix runtime for OSX users, this has been corrected (Geoff). <BR>
<BR>
Support code generated by latest IKVM (Zoltan). <BR>
<BR>
<BR>
<H3>
<B><FONT SIZE="4">Extra Comppiler Support</FONT></B>
</H3>
Support for NPTL as the default threading system: this will benefit Gentoo and Fedora Core 4 users (Ben). <BR>
<BR>
Workaround GCC 3.3 bugs (exposed on AMD64, but possibly other platforms) (Zoltan). <BR>
<BR>
Support for GCC4 has been added for SUSE 9.3+ and Fedora Core 4 users (Ben). <BR>
<BR>
Special thanks to Marcus Meissner  for his help. <BR>
<BR>

</H3>
