<DIV>I haven't tried the ASP.NET 2.0 features, so I can not comment on them.</DIV>
<DIV>&nbsp;</DIV>
<DIV>However, the ASP.NET 1.1 features should work well with Oracle.</DIV>
<DIV>&nbsp;</DIV>
<DIV>See <A href="http://www.mono-project.com/Oracle">http://www.mono-project.com/Oracle</A>&nbsp;</DIV>
<DIV>which will give you an idea how complete System.Data.OracleClient is.</DIV>
<DIV>&nbsp;</DIV>
<DIV>Even when using NET_2_0 profile in Mono, the NET_1_1 features should still work.&nbsp; Its the new NET_2_0 features which may or may not be missing.&nbsp; Stay away from using abstract classes like DbCommand&nbsp;that were added in NET_2_0.</DIV>
<DIV>&nbsp;</DIV>
<DIV>ODBC support in Mono is good with System.Data.Odbc, and there is Oracle ODBC drivers for every platform that the Oracle client softare exists.&nbsp; The Oracle Instant Client may work for you.</DIV>
<DIV>Places like <A href="http://www.openlinksw.com/">http://www.openlinksw.com/</A>&nbsp;should have some ODBC Oracle drivers you could use if the drivers from Oracle are not up to speed.</DIV>
<DIV>&nbsp;</DIV>
<DIV>If the above will not work for you, there is always an Oracle driver from datadirect.com which is fully managed and does not require any Oracle software to be installed.</DIV>
