Today I use a &quot;Document Generator&quot; written in mono that use LaTeX as pdf generator. The mono program build a LaTeX document (using special LaTex cls) and call LaTex with Process.Execute to build the pdf... It works well, but LaTeX models are very difficult to maintain... I've done several test with Mono OO bindings, and it seems possible to use them to replace LaTeX in this program.<BR>
<BR>
I use another &quot;Document Converter&quot; written in mono. It convert any document (recognized by oo.org) into a pdf... Today there's a Macro in openoffice basic, and mono call oo.org (with Process.Execute) and tell oo.org to execute the macro. I've rewritten this program yesterday with oo.org bindings and it works well!<BR>
<BR>
I've not found for instance any problem with the bindings...<BR>
