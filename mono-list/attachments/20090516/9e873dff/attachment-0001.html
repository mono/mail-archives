Hi,<br><br>I&#39;m working on a summer of code project for MonoDevelop, and
tend to write extremely verbose code, which becomes a little hard to
handle on stdout/stderr, especially for large programs that make it
easy to miss critical things on stdout. I started writing a new logging
solution, which is centered around having multiple modules, fine
grained control and flexibility, and a GUI to top things off. I wanted
to keep this confined to my soc project, but I feel that the community
might benefit from my idea(if there isn&#39;ta  better one out ther), so I
thought of metioning it here.<br>
<br>I&#39;ve written a project called &quot;PowerLogger&quot;, a C# project on
monodevelop. I have written a basic implementation and will be
uploading on github as soon as some bugs are fixed.<br>The API is written in two classes: LoggerWindow, and LoggerInterface.<br>
A single LoggerWindow object is created, and initialized for the whole
application. It has a treeview, a textview and some other options . It
runs on it&#39;s own thread<br>A
loggerInterface is created for a class, which wants to log. I have a screenshot with a very early build of the tool running in
the top right corner of the screen.<br><a href="http://img200.imageshack.us/img200/2765/logger2.png">http://img200.imageshack.us/img200/2765/logger2.png</a><br><br>I&#39;ll give an example of the current api:<br><br>LoggerInterface logger = new LoggerInterface(loggerWindow, &quot;Module Name&quot;); // the loggerwindow object must be declared<br clear="all">
logger.INFO(&quot;The
value of my object is: &quot;, content); // content can be any object. We
use the C# reflection to generate a JSON representation of the object.<br>
// we can also use DEBUG, ERROR, etc.<br><br>The
main advantage is each module is represented on the tree, and each gets
it&#39;s own log. So I can create multiple modules, and by double clicking
on a tree object, I can see that module&#39;s log - no poring through
stdout, or stderr to find what&#39;s relevant :)<br>
also, I&#39;ve added some elementary functionality to pause, change verbosity, save, redirect, etc.<br><br>If
you have any suggestions, questions, or if you know such a project
which exists like this, please let me know. I&#39;m very very new to C#.<br>
