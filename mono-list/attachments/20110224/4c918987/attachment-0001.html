Hi,<div><br></div><div>When I faced a similar issue, I came at it from a slightly different angle.  First, the ConfigurationManager (<a href="http://msdn.microsoft.com/en-us/library/system.configuration.configurationmanager.aspx" target="_blank">http://msdn.microsoft.com/en-us/library/system.configuration.configurationmanager.aspx</a>) will expose two section handlers by default - ConnectionStrings (for database connections) and AppSettings (for general settings).  </div>


<div><br></div><div>If you want more complex configuration option then you can define your own handlers by inheriting from ConfigurationSection and ConfigurationElement.  See <a href="http://msdn.microsoft.com/en-us/library/system.configuration.configurationsection.aspx" target="_blank">http://msdn.microsoft.com/en-us/library/system.configuration.configurationsection.aspx</a> and <a href="http://msdn.microsoft.com/en-us/library/system.configuration.configurationelement.aspx" target="_blank">http://msdn.microsoft.com/en-us/library/system.configuration.configurationelement.aspx</a>.  You may also need to provide a class inheriting from ConfigurationElementCollection as well, if you want to be able to configure a collection of values.</div>
