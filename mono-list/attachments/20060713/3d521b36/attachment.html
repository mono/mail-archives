Interesting idea, but not necessarily in the way you meant it.&nbsp; The things you list are applications that happen to run on Mono, but they aren't really integral to Mono as a platform or runtime environment.&nbsp; It's hard to see the overall benefit of such a loose collection of applications (unless there's some level of integration I don't know about that would support the concept of a 'suite'). 
<br><br>I find more interesting the idea of a Mono &quot;bundle&quot; that added useful things that aren't part of Mono proper to enhance/expand the capabilities of a Mono installation. &nbsp; Things like the Tao bindings, say, or other bindings to useful libraries so a developer or application distributor could have &quot;batteries included&quot; support for things like OpenGL or SDL.&nbsp; 
<br><br>All told, however, this kind of thing is probably more useful on Windows or other platforms without good package management support, since the Linux distributions I know of that support Mono already do a good job of managing Mono-dependent applications and libraries as discrete packages.&nbsp; There's really no need for a bundle, since the package manager will pull in the packages you need as dependencies of the application or library you are interested in using.
