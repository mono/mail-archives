<p class="MsoPlainText">Imre  –  you missed the point of my email, and obviously I could have made the program a lot faster than I did.  The point is that your program is generally slow for other reasons, though you seemed to be worried about using Arrays or
 Lists which is peanuts compared to other things you could do.  In particular, in both Mono and .NET, method calls are expensive.  In my experience, adding numbers and incrementing values takes no time compared to calling the method that does these things. 
 I was suggesting you should do the following things:<o:p></o:p></p>
<p class="MsoPlainText"> <o:p></o:p></p>
<p class="MsoPlainText">1 – Make sure the method is inlined, avoid the method call.<o:p></o:p></p>
<p class="MsoPlainText"> <o:p></o:p></p>
<p class="MsoPlainText">2 – Do not insert the for loop inside a useless do while loop.  You are checking conditions inside and outside the method and likely preventing important optimizations like caching the list.count property.  I don’t know what you are
 trying to do, but note that if you replace the do/while code with the code below, than you inline the Advance method, can still do everything inside the loop and it is over an order of magnitude faster on either .NET or Mono.  Not sure if you can do this with
 your code, but the point is mono v. .NET performance differences pale in comparison to the nested for-loop-method-call-inside-do-while-loop performance penalty you are paying right now.
