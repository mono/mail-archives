<p class="MsoNormal" style="margin-bottom:12.0pt">I played around a bit with this and also noticed a very substantial MS
<br>
versus MONO difference, some notes on this: <br>
<br>
*The biggest performance difference here is not the mono vs. .NET but rather <br>
how the program is written.  Right now the bulk of the code on both Mono and <br>
.NET is spent on the do-while loop, and this is a HUGE waste of repeated <br>
method calls.  Every increment of an array element involves a method call, <br>
and I could drop the runtime of the program about two orders of magnitude by <br>
writing a different advance function as shown at the end of this email. <br>
That function can't really be inlined properly by either MS or mono, and the <br>
runtime goes from about 1.9 seconds on my machine to .01 seconds with this <br>
change, though the code does the same thing.  If the code you are writing in <br>
practice does look like this, it can be improved a lot by having the called <br>
method not return until a condition is met, rather than returning, checking <br>
the condition, and calling again. <br>
<br>
 *Strangely, after the code is changed to accommodate this, mono and MS on <br>
my test machine appear to have an unbelievably large difference in speed. <br>
This appears to be "fake" differences due to how the startime variable is <br>
set and nothing to do with the execution speed of the main method.  If <br>
starttime is reset inside the method by setting it again after printing it <br>
to the console, the performance difference disappears.  There still appears <br>
to be a <2X mono performance penalty when mono is used with a List type, but <br>
with an array for the counters, I seem to get the same results on mono and <br>
ms.   <br>
<br>
If you try the program below, I would be interested to know if the speed was <br>
equivalent on your machine for mono and .net.  One problem is that it is so <br>
fast on both now that it is hard to compare. <br>

PS It would still be nice to have a way to view the emitted assembly code on <br>

<br>
//improved program <br>
using System; <br>
using System.Collections.Generic; <br>
namespace SpeedTest <br>
{ <br>
    class SpeedTest <br>
    { <br>
        <br>
        static void Main(string[] args) <br>
        { <br>
            <br>
            var totals = new int[] { 10, 10, 100, 100, 100 }; <br>
            var counters = new int[] { 0, 0, 0, 0, 0 }; ; <br>
            int counter = 0; <br>
            //Variable needs to be set again after printing, otherwise <br>
result is wrong <br>
            var startTime = DateTime.Now; <br>
            Console.WriteLine("START TIME: {0}", startTime.ToString()); <br>
            startTime = DateTime.Now; <br>
            int total = 1; <br>
            for(int i=0;i<totals.Length;i++) { <br>
                total*=totals[i]; <br>
                <br>
            } <br>
            Console.WriteLine("Total = {0}", total); <br>
            Console.WriteLine("Dif= {0}", (DateTime.Now - <br>
startTime).ToString()); <br>
            Advance(counters,totals); <br>
            //just to make sure it doesn't avoid the advance method call <br>
            Console.WriteLine(counters[2].ToString()); <br>
            var endTime = DateTime.Now; <br>
            Console.WriteLine(); <br>
            Console.WriteLine("END TIME: {0}", endTime.ToString()); <br>
            Console.WriteLine("RUN TIME: {0}", (endTime - <br>
startTime).ToString()); <br>
            Console.ReadLine(); <br>
        } <br>
        static void Advance(int[] current, int[] total) <br>
        { <br>
            <br>
            for (int index = current.Length - 1; index >= 0; --index) <br>
            { <br>
                while(current[index] != (total[index] - 1)) <br>
                { <br>
                    ++current[index]; <br>
                } <br>
            } <br>
        } <br>
        <br>
    } <br>
} <br>
