Hi,<br><br>I&#39;ve been trying to connect to an Sql Server instance inside a guest virtual machine (VirtualBox) and using Ubuntu 7.10 as host. The networking is setup using bridge utils as instructed by the VirtualBox manuals.
<br><br>I was unable to connect getting several &quot;Sql Server Not found&quot; and timeouts from the host but not from a separate windows box running .Net 2.0. I tracked down the problem and found that the method GetHostEntry was recuring an HostEntry with 1 and only 1 IP which was 
<a href="http://127.0.0.1">127.0.0.1</a>. I assume this has something to do with the bridge and the internal C mono code getting this IP but after some trials with .Net to several machines and virtual machine with similar configurations I found that .Net only returned the IP specified in the parameters even if the host had more IPs attributed.
<br><br>From my observations, and that makes sense after all if one specifies an IP there is no logic to try to get more IPs, I&#39;ve created a patch that successfully fixed my problem. <br><br>I&#39;ve created bug #351908 (
<a href="https://bugzilla.novell.com/show_bug.cgi?id=351908">https://bugzilla.novell.com/show_bug.cgi?id=351908</a>) and included the patch there.<br><br>Cheers,<br clear="all"><br>-- <br>Alexandre Gomes, Portugal
