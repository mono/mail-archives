<p class=MsoPlainText>Hi,<o:p></o:p></p>

<p class=MsoPlainText><o:p>&nbsp;</o:p></p>

<p class=MsoPlainText>MySQL Connector/Net 1.0.8 RC has been released.&nbsp;
MySQL Connector/Net is an all-managed ADO.Net provider for MySQL.&nbsp; While
this release is suitable for any version of MySQL, it is strongly encouraged
that this release not be used on any production data.&nbsp; <o:p></o:p></p>

<p class=MsoPlainText><o:p>&nbsp;</o:p></p>

<p class=MsoPlainText>** IMPORTANT **<o:p></o:p></p>

<p class=MsoPlainText>We are making this release available as an RC because
there were significant changes made to the provider beyond just simple bug
fixes.&nbsp; Due to this, we felt it prudent to ask for feedback from the
community before declaring the provider safe for production use.&nbsp; We plan
to release the next version (1.0.9) as GA within the next 2-3 weeks
incorporating any feedback we get from the community.<o:p></o:p></p>

<p class=MsoPlainText>** IMPORTANT **<o:p></o:p></p>

<p class=MsoPlainText><o:p>&nbsp;</o:p></p>

<p class=MsoPlainText>It is now available in source and binary form from the
Connector/Net download pages at <a
href="http://dev.mysql.com/downloads/connector/net/1.0.html">http://dev.mysql.com/downloads/connector/net/1.0.html</a>
and mirror sites (note that not all mirror sites may be up to date at this
point of time - if you can't find this version on some mirror, please try again
later or choose another download site.)<o:p></o:p></p>

<p class=MsoPlainText><o:p>&nbsp;</o:p></p>

<p class=MsoPlainText>This release includes many bug fixes along with the
following enhancements/changes:<o:p></o:p></p>

<p class=MsoPlainText><o:p>&nbsp;</o:p></p>

<p class=MsoPlainText>* Implemented a stored procedure cache.&nbsp; By default,
the connector caches the metadata for the last 25&nbsp;&nbsp;&nbsp; procedures that are
seen.&nbsp; You can change the number of procedures that are cached by using
the &#8216;procedure cache=&lt;size&gt;&#8217; connection string setting.&nbsp;
You can disable the procedure cache by setting a zero size.<o:p></o:p></p>

<p class=MsoPlainText><o:p>&nbsp;</o:p></p>

<p class=MsoPlainText>* Executing stored procedures now no longer requires the
user to have select privileges on the mysql.proc table.<o:p></o:p></p>

<p class=MsoPlainText><o:p>&nbsp;</o:p></p>

<p class=MsoPlainText>* Shared memory connections now work as expected.<o:p></o:p></p>

<p class=MsoPlainText><o:p>&nbsp;</o:p></p>

<p class=MsoPlainText>* Queries containing a large number of parameters now run
many times faster than before.&nbsp; As an example of the performance gains
seen, an internal testing app that needed more than 30 minutes to execute using
1.0.7, completes in less than 8 seconds using 1.0.8.<o:p></o:p></p>

<p class=MsoPlainText><o:p>&nbsp;</o:p></p>

<p class=MsoPlainText>* We are no longer providing binaries compiled with .NET
1.0.&nbsp; We are doing this for a couple of reasons.&nbsp; First, we believe
the vast majority of our users are using .NET 1.1 or 2.0.&nbsp; Also, removing
.NET 1.0 binaries reduces the number of frameworks we have to test thereby
reducing the time it takes to verify releases.&nbsp; If 1.0 is needed by a
sufficiently large number of users, we are willing to re-evaluate this
decision.<o:p></o:p></p>

<p class=MsoPlainText><o:p>&nbsp;</o:p></p>

<p class=MsoPlainText>* ICSharpCode ZipLib is no longer used by the connector
and is no longer being distributed with it.<o:p></o:p></p>

<p class=MsoPlainText><o:p>&nbsp;</o:p></p>

<p class=MsoPlainText>The installer for this release installs the assembly into
the Global Assembly Cache (GAC) without giving you an option to not do
this.&nbsp; In the GA release (1.0.9), the installer will include an option to
not register the provider in the GAC.&nbsp; <o:p></o:p></p>

<p class=MsoPlainText><o:p>&nbsp;</o:p></p>

<p class=MsoPlainText>Also, the installer attempts to install both 1.1 and 2.0
binaries into the respective GAC.&nbsp; If you do not have both frameworks
installed, the installer will give an error.&nbsp; This will be fixed in the
final product but for now you can work around this by doing a custom install
and choosing the version of the driver for the framework you want to install. <o:p></o:p></p>

<p class=MsoPlainText><o:p>&nbsp;</o:p></p>

<p class=MsoPlainText>Thank you for checking out this release of
Connector/Net.&nbsp; <o:p></o:p></p>
