<DIV><SPAN class=546511918-01102007><FONT face=Arial size=2>I'm trying to 
serialize a DateTime and ensure that it is serialized in a time zone independant 
way, so that I can deserialize it anywhere and that it still represents the same 
point in time.&nbsp; I'm using the methods DateTime.ToBinary and 
DateTime.FromBinary to do that.&nbsp; In .NET, it is working properly, but in 
mono, dates that are of the DateTimeKind.Local are not deserialized 
properly.&nbsp; </FONT></SPAN></DIV>
<DIV><SPAN class=546511918-01102007><FONT face=Arial 
size=2></FONT></SPAN>&nbsp;</DIV>
<DIV><SPAN class=546511918-01102007><FONT face=Arial size=2>When serializing a 
DateTime that is identified as local, .NET converts the ticks in UTC, so that 
when it is time to deserialize it, they can convert it back using the local 
machine settings and have a DateTime that identifies the same point in 
time.&nbsp; Mono doesn't do any conversion, and there is no way to know what is 
the TimeZone of the serialized date, so I don't think it is correct.&nbsp; This 
also makes mono's implementation incompatible with .NET.</FONT></SPAN></DIV>
