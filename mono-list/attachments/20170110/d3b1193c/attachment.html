<p style="margin-left:.5in"><span style="font-family:Corbel">Hi,<br>
<br>
We can deadlock Mono launching a few HttpWebRequests. We can consistently reproduce in our app, but only sometimes happens in the attached program.<br>
<br>
We initially detected this on a server pulling data from Azure. It deadlocks and it is unable to handle a single new request (since we rely on Socket.BeginReceive for that, and the callback is never invoked once the deadlock happens).<br>
<br>
We suspect it could still be related to this ancient issue: <a href="https://na01.safelinks.protection.outlook.com/?url=http%3A%2F%2Fwww.mono-project.com%2Farchived%2Farticlethreadpool_deadlocks%2F&data=02%7C01%7Cmiguel%40microsoft.com%7C9fa698bca9f44310f44d08d42b26b563%7C72f988bf86f141af91ab2d7cd011db47%7C1%7C0%7C636180892637291812&sdata=%2BSrNzQ4qjxdxSjyZltlqdwN%2BgfHhZORprCNzkNXdSrk%3D&reserved=0">
http://www.mono-project.com/archived/articlethreadpool_deadlocks/</a><br>
<br>
I have attached a small sample application to reproduce the issue. It occasionally deadlocks on
<b>Mono 4.6.2</b>.<b>7</b> (and older) no matter whether the machine has 1 core or 8. What is consistent is that it performs worse than expected and fails. Not sure if this is a hint.<br>
<br>
What the app does is:<br>
<br>
* It launches 40 HttpWebRequest.GetResponse() in parallel.<br>
* Most of them fail ("The authentication or decryption has failed.", "The request timed out") .<br>
* The program retries if they fail - and retries also fail.<br>
* CPU is very high (90% on my test box).<br>
* After a while sometimes the entire program hangs.<br>
* We also tried the HttpClient class using async calls and no threadpool but result wasn't any better.
<br>
<br>
Some interesting notes:<br>
* The same code works well in .NET/Windows.<br>
* And launching the 40 concurrent requests is super fast there compared to super slow using Mono.<br>
<br>
<br>
Any hints or workarounds will be greatly appreciated :-P<br>
