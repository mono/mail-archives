First of all, it&#39;s preferred that you choose using MySql Connector/NET instead of an MySql Connector/ODBC. It&#39;s simplier than using ODBC and it&#39;s far wide used. <br><br>For using MySQL Connector/NET, put your connection string like this:<br>
<br>    &lt;connectionStrings&gt;<br>        &lt;add name=&quot;MySql&quot; providerName=&quot;MySql.Data.MySqlClient&quot; connectionString=&quot;server=127.0.0.1;user=root;pwd=root;database=MyDatabase;&quot;/&gt;<br>    &lt;/connectionStrings&gt;<br>
<br>And make sure that MySql.Data.dll is registered on GAC, or it&#39;s inside the bin directory of your <a href="http://ASP.NET">ASP.NET</a> website.<br><br>If you can&#39;t avoid using ODBC:<br><br>1) Reinstall (or install, if you haven&#39;t) the MySQL Connector/ODBC from this page <a href="http://dev.mysql.com/downloads/connector/odbc/5.1.html">http://dev.mysql.com/downloads/connector/odbc/5.1.html</a>. <br>
2) Configure your unixODBC again and check for incorrect informations. If you need this URL has a good guidance <a href="http://www.unixodbc.org/odbcinst.html">http://www.unixodbc.org/odbcinst.html</a><br>3) Test your provider following the instructions on <a href="http://www.mono-project.com/ODBC">http://www.mono-project.com/ODBC</a><br>
<br>This way you will be using the libodbc.so and not MySQL Connector/NET (MySql.Data) to access the database.<br><br>It should work as expected. If this doesn&#39;t work, then I will need to see your files.<br><br>But seriously, you should use the first option, it&#39;s far better and easier to configure.<br>
