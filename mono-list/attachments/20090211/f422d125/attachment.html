<p class=MsoNormal><font size=2 color=navy face=Arial><span style='font-size:
10.0pt;font-family:Arial;color:navy'>A really good point to make regarding this
is to make sure that when you &#8220;have&#8221; to serialize across platforms,
then do it at the lowest level possible.<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 color=navy face=Arial><span style='font-size:

<p class=MsoNormal><font size=2 color=navy face=Arial><span style='font-size:
10.0pt;font-family:Arial;color:navy'>Basically, my solution was to *<b><span
style='font-weight:bold'>not</span></b>* serialize the SortedList by marking it
[NonSerialized], putting the data in a *<b><span style='font-weight:bold'>more</span></b>*
generic container and then creating a accessor to load the data from the
generic store into the sorted list when needed the first time. Still all in the
same class, so I have 1 binary still and it works across the platforms without
issue.<o:p></o:p></span></font></p>

<br>
Mono's implementation isn't incorrect. It's just incompatible<br>
regarding binary serialization.<br>
<br>
Since SortedList&lt;K, V&gt; does not implement ISerializable, there<br>
is no way to intercept the serialization process other than<br>
trying to design the class layout like MS did. It's not an<br>
easy process given that mono's developers never decompile<br>
and look at MS' implementation.<br>
<br>
Here's an exception I got while trying to deserialize<br>
a stream created with <a href="http://MS.NET" target="_blank">MS.NET</a> 2.x:<br>
<br>
Unhandled Exception: System.TypeLoadException: Could not load type<br>
'System.Collections.Generic.SortedList`2+KeyList[[System.Int32,<br>
mscorlib, Version=2.0.0.0, Culture=neutral,<br>
PublicKeyToken=b77a5c561934e089],[System.String, mscorlib,<br>
Version=2.0.0.0, Culture=neutral, PublicKeyToken=b77a5c561934e089]]'.<br>
<br>
This means that MS' implementation is using an internal class<br>
SortedList&lt;K,V&gt;.KeyList. No one knows exactly what this class is<br>
for and how it's implemented...<br>
<br>
You should replace the SortedList from your WebService API with<br>
something that can be serialized better, like an array of pairs.<br>
<br>
More on serialization: <a href="http://www.mono-project.com/FAQ:_Technical"
