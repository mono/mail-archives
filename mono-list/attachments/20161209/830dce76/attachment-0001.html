<br>
</span>You don't need mono if you're running in windows. It's not even very good for testing purposes, because nobody in the real world will run your application (whatever it is) using mono on windows. Mono is generally used to bring .NET (a microsoft native platform) to non-microsoft platforms, such as linux. The best way to learn .NET and/or mono is to install Visual Studio Community (free) on windows, and install xamarin studio (free) on a mac or linux desktop. Start writing and sharing code between the two systems. Start with Console projects, because the GUI components will definitely create an obstacle for you, that you don't need.<br>
<br>
No additional installs necessary. Even <a href="http://ASP.NET" rel="noreferrer" target="_blank">ASP.NET</a> projects simply run natively in Visual Studio in win, or Xamarin Studio on mac/linux, with no external apache installers or anything. You should be aware that you'll have to learn C# and .NET before you can move on to learning <a href="http://ASP.NET" rel="noreferrer" target="_blank">ASP.NET</a>. It's a whole new world stepping into C# / .Net for the first time, and it's again, a whole new world going from there to <a href="http://ASP.NET" rel="noreferrer" target="_blank">ASP.NET</a>. So start with C# Console Applications, to do some basic stuff like downloading something from a webpage, connecting to a database, writing a file, handling some events that get raised by things like FilesystemWatcher, that trigger asynchronous events on the ThreadPool.<br>
<br>
After you're good at .NET, when you're ready to learn <a href="http://ASP.NET" rel="noreferrer" target="_blank">ASP.NET</a>, I highly recommend "Up and running with <a href="http://ASP.NET" rel="noreferrer" target="_blank">ASP.NET</a>" on lynda. Lots of colleges and companies offer complimentary subscriptions to lynda, but even if you have to pay, it's well worth the $30 or so. For that matter, it's very likely they have some good classes to help you learn .NET too. I dunno, haven't checked.<br>
<br>
You're obviously not going to deploy your solution to the world, running inside the VS or XS debugger. You will have to go through some setup process to make it deploy on LAMP, but don't bother with that until you're comfortable running things in your debugger.<br>
<br>
When you're at the point of wanting to make an <a href="http://ASP.NET" rel="noreferrer" target="_blank">ASP.NET</a> project work on LAMP, don't mess around with XAMPP on windows, because again, nobody in the real world uses mono on windows. That's not the point of mono. Get yourself a centos/rhel or ubuntu server VM (not ubuntu desktop, because obviously, you would never deploy a real server that way), and install mono according to <a href="http://www.mono-project.com/docs/getting-started/install/linux/#centos-7-fedora-19-and-later-and-derivatives" rel="noreferrer" target="_blank">http://www.mono-project.com/<wbr>docs/getting-started/install/<wbr>linux/#centos-7-fedora-19-and-<wbr>later-and-derivatives</a> or whichever is appropriate for your platform. You should expect difficulty making an <a href="http://ASP.NET" rel="noreferrer" target="_blank">ASP.NET</a> application in visual studio compatible with linux, or vice-versa.<br>
<br>
You should expect things to work very well, if you either (a) use xamarin studio to develop on mac or linux, to develop <a href="http://ASP.NET" rel="noreferrer" target="_blank">ASP.NET</a> that gets deployed to linux mono servers, or (b) use visual studio on windows, to develop <a href="http://ASP.NET" rel="noreferrer" target="_blank">ASP.NET</a> that gets deployed to Azure or some other microsoft-based <a href="http://ASP.NET" rel="noreferrer" target="_blank">ASP.NET</a> server. You should expect some level of hassle (it's disputed if it's a tiny hassle or a big hassle) if you develop on mono and deploy to microsoft, or develop on microsoft and deploy on mono.<br>
