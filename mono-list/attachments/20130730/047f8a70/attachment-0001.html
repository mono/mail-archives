<p dir="ltr">Dllimport tells the loader the library name.</p>
<p dir="ltr">A large combination of things work as library loading and shared objects differ greatly on windows and unix.</p>
<p dir="ltr">On windows, the library could be installed in the side-by-side cache or in the same folder as your assembly etc.</p>
<p dir="ltr">If you have a file called libfoo.so then best practice would be to do DllImport("foo")</p>
<p dir="ltr">If that still doesn't work then you may actually be missing a library required by foo and may need to set LD_LIBRARY_PATH or the equivalent on your platform.</p>
<p dir="ltr">You might also need to mess with dllmap config files too</p>
