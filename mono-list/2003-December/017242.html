Unlike the Mono Class Library documentation where the stubs are already
set in stone, and the markup language is a tiny subset, doing a general
purpose HTML editor for the tutorial is not likely the solution to the
problem.

The problem with the handbook is a human problem: someone needs to read
the whole text, make changes, edits, removals, clarifications, updates
and this is a manual process.  Unlike the class library documentation
where things are fairly well isolated and the framework is set, in this
case you need someone to actively drive this process, and a tool is not
the solution to the problem, the Monkeyguide problem has nothing to do
with tools.
