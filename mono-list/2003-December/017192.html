>> All,
>>
>> Here is a .pkg installer for Mono 0.29: =20
>> http://www.morford.org/mono/mono-0.29.pkg.zip
>>
>> A couple things:
>>
>> 1. Built with GCC 3.3 on Panther 10.3.1 with DarwinPorts using glib =20=

>> 2.2.4 and dependents and pkgconfig 0.15. The BSD Subsystem is =20
>> required, which is installed by default in 10.3.
>>
>> 2. The default install path for the .pkg is /usr/local. You can =20
>> install the files anywhere you want by choosing to customize the =20
>> install path. You can also "Show Package Contents"  from Finder and =20=

>> go into the Contents folder and grab the .gz of the files.
>>
>> 3. You'll need glib2 and dependents installed, either from =20
>> DarwinPorts or Fink. The DPorts Portfile for glib2 contains a fairly =20=

>> readable listing of the glib2 dependencies:
>>
>> http://darwinports.opendarwin.org/darwinports/dports/devel/glib2/=20
>> Portfile
>>
>> 4. If anyone has hosting space for this installer, perhaps on =20
>> go-mono.com or opendarwin.org, please let me know.
>>
>> 5. The JIT Engine for PPC is definitely coming along and on that =20
>> congrats! mint works really well now and I've not encountered any =20
>> issues yet compiling and running small to medium size assemblies with =
=20
>> it. I've not had time to test on Darwin 7 on x86.
>>
>> 6. I've got a PortFile almost working for 0.29 and I had GTK# 0.14 =20=

>> working through X11 for a very very very simple test but I'll =20
>> definitely need more time to work out a PortFile for that and, with =20=

>> mint, it runs slow. Like Markus I think the PPC JIT should be closer =20=

>> to complete before committing it into DPorts but perhaps as a =20
>> development port or with an unstable marker.
>>
>> 7. No GC. :( I'd like to be able to figure this out and see if it is =20=

>> even a problem now on 10.3. A good overview of all the new =20
>> capabilities in OS X v.3, including commands and libs from FreeBSD 5, =
=20
>> GCC 3.3, pthread signaling, I/O, aio_* updates, System V poll system =20=

>> calls and more of the many updates, additions and changes can be =20
>> found at the following:
>>
>> http://www.apple.com/macosx/pdf/Panther_Unix_TB_10082003.pdf
>> http://www.apple.com/macosx/features/unix/
>>
>>
>> I would like to work out providing a "Mono.framework" for OS X. If =20=

>> anyone is interested in kicking off a Mono OS X hacking subproject to =
=20
