Hi all,

Glen Parker sent a patch to add better exception support to Npgsql.

NpgsqlException now gives aceess to error collection through the Errors 
property. This property return a collection of NpgsqlErrors objects. 
Also it gives access to error message, hint, severity and code. This 
fixes the feature request 689 in gborg.

I also removed TlsException dependency from Npgsql. Now it can be 
changed to internal.


Please, give it a try and let me know if you have any problems.
