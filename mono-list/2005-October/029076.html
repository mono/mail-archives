On Mon, 2005-10-17 at 19:24 -0700, Shankari wrote:
> one way I thought of is to pass it as a function
> parameter, so that mono can strip it off before
> pushing it to stack. So, the function that gets called
> still has the right parameters.  Then based on the
> flag I am doing some assembly code modifications in
> the unmanaged(C code).

As others have noted, this won't work too well on Win32 as it conflicts
with the STDCALL calling convention (in which the *callee* pops the
arguments from the stack, and thus needs to know how many were pushed).

Have you considered just providing two versions of the function, and/or
using an Attribute to specify this extra information, and/or using a
Thread-Local-Storage variable to hold this information?
