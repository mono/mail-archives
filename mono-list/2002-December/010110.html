These questions have been answered before.

The short answer is Yes.

Yes, you can use VB.NET.
Yes, you can run the EXE/DLL generated by VB.NET under Mono. No, you don't
need to recompile your programs for Mono.

But...  The above is only true if the libraries (assemblies) you depend upon
already exist under Mono.  For example, Mono 0.16 doesn't have
Microsoft.VisualBasic.dll, so if your app uses it, it won't run properly.
CVS has a partial implementation of the DLL in the works, but I have no idea
how complete it is.

Ditto for any other assemblies.  Several of the assemblies are in an
incomplete state, so your code may or may not work without changes.

Mono-Basic is an implementation of VB.NET for Mono.  AFAIK, it's nowhere
near complete, and I have no idea when it will be.
