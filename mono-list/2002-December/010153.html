	[ Subjecte: [Mono-list] Subversion repository ]
> I suggest getting the Mono sources into a Subversion 
> repository to make contributing to the project easier and 
> more effective.

This would seriously hinder someone like me, who does most of
her work from lots of different computers, some very slow, some
with very... interesting... systems on them, and CVS is a lot less
lightweight and a lot more portable than svn.  As for more effective,
how do you mean?

Really, things like this never get beyond bikesheds and whatnot,
and basically what you've got is what you've got.  If you want to
know "which SCM is better", I'd say Perforce, and so would Microsoft,
if you're familiar with the source management migration they've done
for the Windows source tree[s].  The real answer though is that CVS
is fairly prolific, and for a project developed like Mono is, it's as
near to perfect as one can get, both from the perspective of developers
and from the perspective of development model.
