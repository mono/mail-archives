Theoretically, Qt# will already work on MacOSX, but as always we have some 
practical considerations:

1.  Qt for the Mac exists and it looks and behaves like a native toolkit.  It 
is also proprietary and requires a developers license.  This is incompatible 
with the current license of Qt#, the GPL.  We are looking to change the 
license to allow commercial/proprietary developers to use Qt#, but we need to 
shed our dependency upon QtC first ... which we are working on ;)

2.  Qt for the Mac also exists if you use an X server on the Mac. Then you can 
use the GPL Qt/X11.  You can currently do this and Qt# should have no 
problems this way.  Of course, right now you'll have to compile QtC too.
