Stack overflow detection is not yet implemented in the segmentation
fault signal handler and the current handler dies because there is no
more room in the stack for it to run. The fix is a bit tricky (using
sigaltstack and then checking if the segmentation fault was actually
caused by a stack overflow). Any volunteers?
