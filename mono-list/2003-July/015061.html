Hi all,

I've to decide how to structure the installation of Gekkota.
Well, I think there are two options:

Option 1
--------

/usr/local/
    + gekkota
         + bin
         |  + Gekkota.Service.exe
         |  + ...
         + lib
         |  + Gekkota.Core.dll
         |  + Gekkota.Security.dll
         |  + ...
         + doc
            + en-US
            |   + gekkota.source
            |   + gekkota.tree
            |   + gekkota.zip
            + de-DE
                + ...

Option 2
--------

/usr/local/
    + bin
    |  + Gekkota.Service.exe
    |  + ...
    + lib
       + Gekkota.Core.dll
       + Gekkota.Security.dll
       + ...
       + monodoc
            + sources
                + gekkota.source
                + gekkota.tree
                + gekkota.zip

Which convention will the future Mono/Gnome applications follow?
Which of the options described above is the correct one?

How does monodoc plan to handle the installation of multi-language
documentation? Like this?

       + monodoc
            + sources
                + en-US
                |   + netdocs.source
                |   + netdocs.tree
                |   + netdocs.zip
                |   + ...
                + de-DE
                |   + ...
                + ...
