Hello,

> Any reason why VB.Net hasn't been mentioned as a supporting language
> in future?

It might, but there are no full time developers working on it at this
point.

Rafael has done some work to write a parser front-end to the C# compiler
that would accept VB code, and produce the current C# semantic tree.  

It should be noted that I do not know a lot about the VB.NET language,
but I am not convinced that this approach will ever work. I believe that
a VB.NET compiler will require a different semantic analysis back-end,
even requiring a complete re-implementation of it. 

Miguel.
