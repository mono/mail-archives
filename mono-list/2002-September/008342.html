Hi, everybody!

Although I've already heared about it, this is really stupid:
An .NET 1.0 compiled application won't run on 1.1 without distributing a =
config-file to redirect the version number.

Else, you have to install 1.0 along with 1.1.
It wouldn't be necessary, as 1.1 doesn't break backwards compatiblity.

How will mono handle this? Currently mono compiled binaries target 1.0.
Should that be updated to 1.1?

And if MS breaks compatiblity with 2.0, how should mono act?
Fork a second code base for 2.0?

Johannes
