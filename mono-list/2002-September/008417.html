Hello Mono-List people,

    I recently expressed my worries about whether or not a VB.NET program
could not run under Mono without major work to implement some specificVB.NET
Microsoft libs. (That was under the "Platforms Mono will support"
thread)

    Apparently, there is no need to worry with that.  I recently installed
Mono-0.15-stable on my Windows 2000 system and after removed all dependency
to the Microsoft.VisualBasic namespace (and an accidental dependency to
SWF), I managed to sucessfully start my windrop.NET IRC bot, coded 100% in
VB.NET.  Needless to say, the program crashes shortly after starting because
of some failures in the XML config file reader (something to do with
XmlNamespaceManager and XmlNameTable).  I did not dig much into this problem
but I'm not worried I'll find and alternative solution to make it work.

    Maybe getting the lastest libs snapshot from the CVS would get me
further in the execution.

        Good job!  Go Mono!
