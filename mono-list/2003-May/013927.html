Hi,

Here is my work on the SoapFormatter class (the
System.Runtime.Serialization.Formatters.Soap one).

It is not fully implemented yep but it should handle serialization &
deserialization in files.
Simple types (int, long, doubles, string....) and arrays of simple types
should be both readable by the Mono and .Net formatters. More
complicated ones: depending on they have the same number of fields and
if fields have the same name on the 2 plate-forms.=20
I'm also currently working on the appropriate ChannelSinks.

If everybody is happy with what I have done, maybe we could find a place
for it on the CVS ?

To build it, you'll have to untar the archive under mcs/class and buid
the library. The output file is called
System.Runtime.Serialization.Formatters.Soap1.dll because there already
is a System.Runtime.Serialization.Formatters.Soap.dll.



PS: You'll have problems if you try to serialize Arrays for the moment
because it seems that the Type.IsSerializable flag is false for Arrays.
I've submitted a bug report for it (bug 42779). Can anybody have a look
at it ? Thanks
