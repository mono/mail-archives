At this point, about as much effort seems to be going into creating
a high performance open source CLR implementation as is going into
creating high performance open source JVM implementations.

Since the CLR is a slight superset of the JVM, I think 
the other direction is actually more interesting and also simpler: a 
Java compiler that targets the CLR.  That would allow people to continue to
write and reuse Java code, while at the same time being able to
take advantage of value classes, unsafe modules, and simpler
callouts to C++ from C# modules (or even Java, if a few keywords
are added).  It would also make a large number of Java libraries available
to Mono instantly.
