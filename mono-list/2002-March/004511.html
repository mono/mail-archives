>Only asking because I've recently been implementing a PriorityQueue
>collection class for a game I have been writing and thought you might like
>to include it in Mono. I think it's general enough to be useful in many
>settings (the only obvious specialization is the use of "double" for
>priority values) and the performance is pretty good.

I´d suggest to put it in a namespace Mono.Queues, and deliver it in a 
Mono.PriorityQueue.dll (assembly).

Also, take care of licensing it correctly (MIT X11) if you want to tie it 
with Mono. You can also keep it as a separate project, like nziplib for 
instance, so that you could choose GPL/LGPL, if you prefer. In that case, 
choose another root namespace.

Welcome aboard... And thanks for your contribution, thru Mono or not.
