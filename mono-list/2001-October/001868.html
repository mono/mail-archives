>Subject: [Mono-list] Any thoughts on .Net My Services?

>I was wondering if anyone working on Mono has taken a look at .Net
>My Services (Hailstorm) and what your thoughts are on it.

I´ve read the news and the WS-Specs that MS is proposing to W3C.
My personal feel is mostly good, because it´s based on standards (both 
existing, proposed an to be proposed). If every piece is based in open 
royalty-free standards we can interoperate, at a minimum, and probably offer 
an alternative open-source implementation.

>Is the Mono project/ximian going to try and accomplish the same sort of
>project as an alternative to the MS approach.

Miguel is there at the PDC, seeing the technical details, but what I can say 
is that from the start Project Mono is bound to create the a compatible 
development framework (compilers and class library).
Other parts of the .NET initiative, are being implemented by other projects 
inside the open-source community.

>I'm not against MS, but I believe that competition/alternatives breed
>innovation and ultimately benefit everyone.

That´s why after 20+ years developing with Microsoft tools for Microsoft 
platforms, I´m contributing to Project Mono.
