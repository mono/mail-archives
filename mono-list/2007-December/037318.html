Konstantin

 

Unless you have already done so you MUST use the surface area tool on the
SQL Server host machine to allow remote connections to it.  This affects SQL
Server 2005 only.  If you have not used this utility to allow remote access
you get the behaviour you observe. That is, you can telnet to the port but
you cannot connect.  This is because the default configuration after
installation is to disable remote access.
