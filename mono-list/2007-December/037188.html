Hi All,

I tested mojoPortal just now and it seems to be working fairly well.

Only problem I found was UpdatePanel doesn't seem to work right, it
just does normal postsback like not using updatepanel and I am logging
some errors related to System.Web.Extensions. The update does happen
and the user experiences no error in the browser. This functionality
can be tested in the blog by clicking the Add Category button. It
should do ajax update with no visible postback but that is not the
actual behavior, it does normal postback.

Other than that mojoPortal seems functional. I didn't test every
feature yet but did create new user, created new page, put blog module on
page made a post, and a few other common tasks. No errors at all
experienced in the site just
logging scriptmanager errors as shown below.

I tested by using the 1.2.4 Mono virtual machine on open suse 10.2.
I added the yast source for the preview, did online update then in
software management upgraded to the preview.

I checked out mojoportal svn trunk, compiled with MonoDevelop 0.17, no
errors, just the usual warnings.

Renamed Web.config to Web.config.windows
Renamed Web.config.mono to Web.config
set connection string to pgsql db
cd into Web folder
xsp2
open browser and visit http://localhost:8080/Setup/Default.aspx
setup ran smooth
site works as described above.

Best Regards,

Joe
