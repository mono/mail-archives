Dan Smithers wrote:
> OK, I've got SOAP working. I can construct the stream using
> SoapFormatter.Serialize, but I can't get Deserialize to reconstruct my
> data structure. Is there any trick to this that I haven't spotted?

There is no trick. One difference between Binary and SOAP is
the inability of the latter to store more than one object
graph, i.e. calling Serialize twice per stream will produce
something that cannot be deserialized. This is by design.
