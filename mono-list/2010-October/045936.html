FromHwnd(0) creates a Graphics object for drawing directly to the
screen, and I can't think of a reason for a well-behaved program to do
that. So I would say PdfSharp should not do that.

In theory, libgdiplus should be able to function without an X server
if all of the work is done on bitmaps. It should be possible to create
a bitmap and get a Graphics object for it without an X server. But I
don't know if this is true in practice.
