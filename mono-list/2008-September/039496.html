Hello,

I've been troubleshooting an issue while trying to get a gSoap client to
communicate with a C# server using ServiceHost. When I first tried to get
the communication to work with a .NET server I ran across a problem where
the c# server was complaining about a mismatch in the SOAP Action field:

"The message with Action '' cannot be processed at the receiver, due to a
ContractFilter mismatch at the EndpointDispatcher. This may be because of
either a contract mismatch (mismatched Actions between sender and receiver)
or a binding/security mismatch between the sender and the receiver.  Check
that sender and receiver have the same contract and the same binding
(including security requirements, e.g. Message, Transport, None)."

I found a work around that causes the service host to do dispatch based off
of message body and not SOAP action. MSDN sample here:
http://msdn.microsoft.com/en-us/library/ms750531(VS.85).aspx

referred by this forum post (which has little more detail on how to mod a
generated *.cs file):
https://forums.microsoft.com/MSDN/ShowPost.aspx?PostID=1052544&SiteID=1

This got my .NET/C# server working. However, the mono/C# server (same exact
code) still had a problem and it looks to be because of the blank SOAP
action field.

Anyone have any ideas?

Thanks,
Joe
