Yes,
 
You can compile code-behind pages in the same way. Just use "--target
library" in mcs, to build a library and not an executable.
 
Here is an example:
 
mcs Global.asax.cs Configuration.cs ConfigurationHandler.cs
TestDB.aspx.cs --target library -o bin/web.dll -r System.Data
 
Hope this helps

Dixie
