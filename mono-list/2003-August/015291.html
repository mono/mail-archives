Hi,
 
I'd like to know if it's possible to develop a native compiler, e.g.
similar to ngen.exe; The resulting project would not require the runtime
framework to execute; or at least only a small subset of it - i.e. only
the methods of the runtime used are extracted at compile time and
included in a new static dll for deployment. Similar to msvcrt etc.
 
My concern is that to deploy a .net application you must also deploy the
runtime which carries a 20MB payload. 
 
Regards
Jan
