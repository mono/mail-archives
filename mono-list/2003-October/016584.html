Don't belief everything you read on slashdot. 
Microsoft describes  winFX as "a comprehensive set of managed APIs for
writing "Longhorn" applications"
So, basicly instead of using p/invoke to access the windows APIs, you can
just use the managed libaries. Since.net apps which p/invoke the windows API
won't work on mono , it doesn't make it useless. It would even be possible to
implement the winFX APIs on linux and make mono even more usefull.

but read it yourself at http://longhorn.msdn.microsoft.com
