> What should Mono 1.0 be?

Yes, the documentation should be comfortable and prepared with some =
examples comparable with the MSDN documentation. This is important for =
acceptance to many developers which are developing for the MS.NET =
framework currently. Especially the points which may differ between =
(paths and drives handling for example) MS.NET and Mono on Windows and =
Mono on Linux should be clear and easily understandable.

By the way: I think, we should also provide all the needed stuff for =
VB.NET in Mono 1.0. This is because more developers in the ASP.NET scene =

many developers wouldn't come back in a future release.=20
