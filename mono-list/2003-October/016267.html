Hi Guys

On the subject of a mono v1.0  - As a developer on both windows and linux
tring to make a business audiance understand that mono 1.0 is not
compatiable with .net 1.0 would be disastrous - don't do it . Mono has to be
feature and version number exact with MS if this is to work. Basically avoid
FUD

Release a "Significant Fixed Feature" release i.e mono v0.4.0  this is a
stable release of the current development tree. allow the main development
branch to continue towards the 1.0 goal but stabilise a version and give it
a clear name. reserve the 1.0 for a release which is 100% .net 1.0
compatible.
Alternativly be very clear about released parts of the platform that are
.net v1.0 compatible.

On the subject of a conference - Barcalona is a really nice place, and very
cheap to fly to from the UK and most of the rest of Euorpe.

Jay
