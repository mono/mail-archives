On Tue, 2007-01-30 at 02:05 +0300, Maxim wrote:
> It seems like the best licenses for this solution are LGPL and MIT X11. 
> It would be good to allow to build proprietary software using ORM 
> engine, and to build web-sites based on CMS with proprietary modules.
> But I've already used some GPL libraries in all parts of my CMS, for 
> example MySQL data adapter and SharpZipLib. So, I should to use GPL 
> license for whole CMS?

If you use GPL libraries, it is only necessary that (1) your license be
GPL compatible (which is true for both MIT/X11 and LGPL), and (2) you
distribute all *binaries* under the GPL.

So your code can be MIT/X11, and anyone can freely re-use *your* code,
but the combined work of Your Code + MySQL will be *distributed* under
the GPL.  Does that make sense?
