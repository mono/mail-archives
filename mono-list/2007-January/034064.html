There is no incompatibility between the 1.2.2 and 1.2.3 embedded API.
Since you mentioned you messed with your installation of mono, I
wouldn't be surprised if you broke your compilation environment:
you likely linked to two different versions of mono at the same time.

> It is certainly not an issue to switch development to using 1.2.3  
> instead of the 1.2.2 release, but something seems awry in the

The fact that you're the only one having messed up his system and having
compilation issues should be telling you that this is not a problem in
mono, but on your box. The embedded API has remained compatible since
the 1.0 mono days.
