Hi,

I'm getting ready to add yast installation source support for the suse
distros we have available.

Currently, the opencarpet and yum repositories set up these channels:

mono
gtk-sharp
gtk-sharp-2.0
mono-tools
mono-deps

For yast repositories to fit in line with our current repos, I would be
setting up a single yast repository for each of the channels.  But, I
can't imagine having to add 5 yast installation sources.

Even though this is what's done for yum, it isn't a big deal since you
only have to place one file in /etc/yum.repos.d.

Does anyone have anything against having just one mono channel for each
of these services?  I'm not aware of any advantages of having multiple
channels.

Comments, suggestions?
