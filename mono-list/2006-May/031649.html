On Wed, 2006-05-17 at 10:59 +1000, Hamish Murphy wrote:
> Hi,
>  
> I have a peculiar problem. I have a link to a pdf that doesn't exist
> on my website (Mono 1.1.7 on a Fedora Core 3 box with Apache 2.0.52).
> This is a known scenario and I would like to trap error 404 and
> display a custom error message. I tried adding the <error> tag to the
> <customErrors> section in the Web.config - but that's for aspx pages
> only. I then added an ErrorDocument directive to the Apache virtual
> server - but I get a Mono error instead (Server error in '/'
> application). So where can I configure this?

If you used 'SetHandler mono', you can add an 'Application_Error' method
to your global.asax. If you used 'AddHandler', the .pdf extension will
be handled by apache itself and you'll need what Kornél said in his
reply to your email.
