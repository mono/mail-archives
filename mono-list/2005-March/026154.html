I was just trying to build gtkmozembed-sharp from svn and it was
complaining that "gtk_moz_embed_set_profile_path()" was trying to
convert a string parameter to a IntPtr.

I checked gtkmozembed-sharp's history and it doesn't seem to have been
changed recently, so i checked out revision 41030 of gtk-sharp and did a
make, make install on that.

With 41020 of gtk-sharp installed gtkmozembed-sharp can now make and
install no probs.

It seems someone checked some code into gtk-sharp that broke
gecko-sharp.
