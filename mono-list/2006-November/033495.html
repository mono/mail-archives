Hi,

I've now built mono-1.2.1 and libgdiplus-1.2.1 for Fedora Core on both
i386 and x86_64 architectures. This version releases my previously
released version (1.2).

NOTE : THIS IS NOT AN OFFICIAL FEDORA RELEASE - IT'S MINE. ALL PROBLEMS
WITH THE RPMS SHOULD COME TO ME AND NOT THE FC BUGZILLA. PROBLEMS WITH
MONO AND LIBGDIPLUS NEED TO GO TO THE MONO BUGZILLA.

(sorry for the caps, it seems to have caused confusion when I did this
with 1.2!)

The src rpms are also available. This build does not include mono-basic
support.

If you're on FC-5, please be careful - very careful! The current version
of mono in FC-5 places mono bits in /usr/lib on both x86_64 and i386 -
this version doesn't. If you're on x86_64, mono bits will go
into /usr/lib64 - this may completely shoot anything mono related
through it's head.

I repeat - FC-5 users, be damned careful.

As and when Fedora update to 1.2.1, these archives will vanish.
