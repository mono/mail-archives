> As the article points out, it does not cover Linux.

As I understand it, Microsoft intends that WPF/E will run on Linux.
There is a certain amount of conflicting information here - the source
you quote says Microsoft won't do this, whereas the guys I spoke too
said "maybe Linux and Solaris". So make of that what you will.

However, presuming that Microsoft does *not* provide a Linux binary, the
plan (as I understand it) is to provide source code to a third party for
a commercial implementation. Would anyone want to take this on? How
would such a third party make money? Or will it in fact be sponsored by
Microsoft in some way so that it is made available for free? I don't
know - pure speculation - but I was told that Microsoft will provide the
necessary technical resources.

So I'd argue that the plan is for a cross-platform CLR.
