On Fri, 2006-03-10 at 13:45 +0100, Marcin P wrote:
> The tool I'm going to write will be a kind of agent responsible
> for watching the processes on a local machine and storing
> very simple statistics (e.g. CPU and RAM "consumed" by the process).
> Of course, System.Diagnostics and it's Process class in the response,
> but how about using it on Mac (OS X)?

Process forking is one of the places where the class libraries just
follow the windows model, and the linux/unix model is very different.
System.Diagnostics.Process does not map very cleanly to what you want to
do.  In addition, mono's implementation of System.Diagnostics.Process
will only show mono processes run by the same user, not every process on
the machine.

You will probably need to p/invoke to a native stats-gathering library.
I've no idea if libgtop has been ported to macosx.
