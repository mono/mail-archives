Hey guys,
I'm one of the lead developers of Media Browser 3, and we're looking for a
mono developer to join our team and help us port our project so that we can
go after linux users.

We are a free, open source, personal media server with streaming and
transcoding. We're built with all new technology including the .net 4.5
framework and Service Stack, another popular open source project. Two areas
where we really stand out are performance and data - we have a lot of data
about your media and the presentations are able to do some really cool
things with it.

I recently ran the mono migration analyzer tool (MOMA), and results were
very encouraging. Not too many issues. I just do not have the time to
personally take on the port given all of my other responsibilities.

We have the following clients in production:

- Media Browser Classic (wmc)
- Media Browser Theater (new htpc client, in WPF)
- Android
- iOS
- Roku
- Win Phone
- Win RT
- HTML5 Web client (built into server)
- Media Portal 1/2

The web client also serves as the server's dashboard, configuration and
metadata management area, and works well on tablets and mobile devices.
Please check us out. Thanks.

PS - If there's another place that would be better for me to recruit, please
let me know.
