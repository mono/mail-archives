Microsoft's compiler (csc) gives two similar warnings (in some later pass - 
so you'll only see them depending on not having errors in the assembly 
compilation): one if you try to use reference-type members/variables before 
initializing them, and another telling you that an accessible member is 
never initialized and will keep it's null/zero value forever.

If mcs does not issue those warnings, fill in a bugzilla case, please.
