That means the execution engine can't locate the assembly you've linked
with
(in this case, System.dll)

When it finds an assembly reference, the Microsoft runtime looks in
three
places: (1) the global assembly cache, (2) the application's current
directory,
(3) locations specified in the application's config file.

So the quickest way to make it find the Mono System.dll is to copy it to
the
same directory as your test.exe file.
