That's pretty nice. They're essentially wrapping the compile/build/run loop up
in a nice gui, and I'm sure we could have a go at doing that ourselves.

What I'd be even more interested in is a full-fledged regex debugger a bit like
the one in the ActiveState IDE (although I've never used it personally). They
use Perl's evaluation syntax ?{...} to insert callbacks after each step of the
expression, allowing the gui to single step through the match.

Mark-Jason Dominus' paper describes it in good detail:

  http://perl.plover.com/Rx/paper/

Since we have our own interpreter code to mess with, it would be simple to
insert callbacks to aid debugging. The question is, how to do it within the
bounds of the Microsoft API.
