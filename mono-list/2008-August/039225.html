Temporarily resolved. This is what I did:
1. copid the shell script 'monodevelop' into a new file 'monoxxxx'
2. edit the script to remove mentions of monodevelop, leaving the other
instructions
3. run mono from this new script such as 
CLI> monoxxxx yyyyyy.exe

and now the WebControl behaves as expected in both monodevelop and run from
the CLI.
