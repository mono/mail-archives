Hello,

Today I've committed code to CVS which should get mint running, for the
most part, on the SPARC architecture.  From preliminary tests, mint
works with the GC with shared handles disabled on simple examples.

I would like it if people with Solaris/SPARC machines to test the code
out, and let me know if/where it breaks.  There are some known bugs in
the architecture specific code, as well as some potential alignment
bugs.  Also, the delegate code isn't completed yet.

If people could test this out and let me know what breaks, I'd
appreciate it.

Also, if you're handy with SPARC assembly, I may need some help with
the JIT when the PPC people are finished cleaning up things.  Something
about processors that have real floating point implementations... ;)

Thanks,
Mark Crichton
