I've commited changes to mbas for it to understand correctly various kinds 
of literals, but not all, see the Changelog:

2003/11/10  Rafael Teixeira <rafaelteixeirabr@hotmail.com>
	* mb-tokenizer.cs : integer literal (Hex starts with &H not 0x), handle 
Octal literals (&O)
	* 	handle unicode specialized double-quotes in string literals
	* 	corrected some mishandling of conversions in numeric literals
	*	implemented support for char literals ("z"c)
	* Makefile : verbose target uses verbose argument
	* testmbas/WriteOK.vb: many tests for literals implemented
	* literal.cs: correcting lack of a ShortLiteral class
	* mb-parser.jay: removed support for unsigned int literals, added support 
for short int literals

Next I will tackle the Date Literals that have no C# counterpart.

A bit -at-a-time but always forward,
