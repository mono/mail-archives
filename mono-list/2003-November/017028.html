Hi!

Now I have read on an german side 
(http://www.pro-linux.de/news/2003/6203.html) that there exists a cost-less 
Web-Debugger for Linux, called "AskIgor":
http://www.st.cs.uni-sb.de/askigor/

I think, it can be used to debug the mono-jitter.


Ok, now I try to translate the text of pro-linux.de to english *puh*:

----------
»Igor« searched for program-errors

Research scientists of university of Saarland [Saarland is a part of 
germany] have startet a internet-service, which detect error-causales of 
computer-programs. The "AksIgor" [http://www.askigor.org/] called Service is 
cost-free and exists at the moment only for Linux-programs. A congruous 
diagnosis for Java- and Windows-programs is in development. Furthermore it 
would be completed with correction-suggestion in the future.

Who wants to test ore use AskIgor, must up-load his executable program and 
input two sentences of arguments: One with which the program runs incorrect 
and one with which the program runs connrect.
The program shoud be compiled with the -g option. Needed files can be 
up-loaded, too. If you start the process, the analysis beginns, which in a 
few minutes should creates the result.

In this time Igor compares two program runs: One where the program runs 
correct, and one where it runs incorrect. With systematically search and 
allways new testruns, Igor find the relevant differences in the program flow 
- the error cause. How it cann look like, you can see on some pictures 
[http://www.st.cs.uni-sb.de/askigor/images/] .

.... etc.

-----------

the rest of the text is unimportant I think.
