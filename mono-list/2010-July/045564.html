>> On Windows and running on .NET CLR, is there any way to have
>> Mono.Security.dll use the certificates that are stored in Windows?

> No, at least not without some additional code. You can either:
>
> (a) write some glue code to the windows certificate stores. Mono code
> was written to be modular and adaptable to alternative sources for
> certificates.
>
> (b) write a tool that read windows certificates and add them into mono's
> certificate store (e.g. a bit like mozroots does but windows specific).
>
