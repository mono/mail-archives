You're wrong.  Your third example (mixed cout/Console.WriteLine) should
compile.  It is this property which makes MC++ useful for bridging
native C++ libraries and .NET libraries (which I've done before).

Additionally, the first example (Console.WriteLine) will have native
code in it.

For fun, run ildasm on the resulting executable of the first one.  One
of the symbols (IIRC) should be _mainCRT (or something similar), which
looks like it calls into the MSVC native libraries.

Last I knew, it was *impossible* to use MC++ to create a "pure" .NET
program.

 - Jon
