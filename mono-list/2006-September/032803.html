On Thu, 2006-09-21 at 11:08 +0200, Martin Feja wrote:
> David Abrames schrieb:
> > When I run my application I get a Dll Not Found Exception on the Windows
> > DLL.  The name of my wrapper library is MyAPILib.dll and the .config file is
> > called MyAPILib.dll.config.
> >   
> I thought that libraries can not have a config-file..

In general, they can't.  .NET will not read .dll.config files.

As a special extension, Mono will read .dll.config files, but *only* for
DLL remapping purposes.  So you can't use a .dll.config file in
combination with System.Configuration.ConfigurationSettings.
