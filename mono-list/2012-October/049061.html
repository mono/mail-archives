I happen to know that they do work.

They can work on a mode that can preclude you from having to have  
Oracle client software installed on the client (OCI level).  In that  
mode, it is not as fully functional than if it uses the P/Invoke mode,  
but it does behave like a good ASP.NET 2.0 capable provider with  
Connection, Command, DataReader, etc.

My experience has been using DevArt DotConnect for Oracle on an MVC 3  
app built using VS 2010 and later deploying to Open SUSE 12.1 running  
Mono 2.10.6.

It worked flawlessly.

However, it really is about time that Oracle make a good old clean  
managed provider. :)
