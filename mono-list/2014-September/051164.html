Hi Peter,

I might be wrong but I *think* this is something to do with the current version of Mono on your machine.  If it's of a certain vintage then it can't be used build the latest C# code that's part of Mono.  I think you can either remove the current mono (or at least hide it) and let the build process sort it out, or upgrade in increments.  e.g If you have 3.2.x now, don't try to go straight to 3.8, but do 3.4, 3.6 etc first.

Again, I could be wrong but this rings a bell.

Good luck!
