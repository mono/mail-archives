The VS.NET solution for the runtime is in mono/msvc/. Beware it's not
always up-to-date and cygwin is still required for the build process.

The class libs usually do not have VS.NET solutions/projects.
Only district libs have some unsupported and not always up-to-date
project files, see Managed.Windows.Forms.

If you (still) want to contribute, please start with these docs:

http://mono-project.com/Contributing
http://www.mono-project.com/AnonSVN
http://www.mono-project.com/Compiling_Mono
