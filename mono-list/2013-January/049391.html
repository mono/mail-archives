In this regard, mono is kinda like java, perl, python, or php.  Somebody writes an application, and to run it, you "mono someapp"  
Mono is only running as long as the application is running - because mono is the application.

If you quit your application and still see mono running, you probably didn't actually quit the application.  But you think you did.  So it should probably be ok to just kill mono, in order to finish quitting your application.

You won't see it running automatically after boot or anything like that.
