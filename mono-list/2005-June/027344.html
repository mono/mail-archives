Folks,

Project approved and up and running at
http://forge.novell.com/modules/xfmod/project/?monoindigo
 Discussed earlier in this list, MonoIndigo will be a free implementation of 

Longhorn's communication stack (codenamed Indigo) on top of Mono. MonoIndigo 

will require Mono 2.0.
 I've also started developing some straightforward things of Indigo, 
like its most common attributes and classes. Indigo supports several 
types of *bidings*, which specify how communication is done. We plan 
to first implement the default *BasicProfileHttpBinding*, which conforms 
to WS-I Basic Profile 1.0 (basically HTTP-SOAP). I expect we can 
leverage very much here from Mono's System.Web.Services. Let's see! 
 Those interested in joining forces please drop me a note!
 Rod
