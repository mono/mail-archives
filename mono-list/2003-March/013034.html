Here is another idea....

mono-build-w32.sh forces the use on the stable version of automake, 
autoconf ....
when the generated configure script tests the presence of gc/gc.h, there 
is small warning about redefinition of GC_WIN32_THREADS
I think the configure script thinks the test failed then it stops.

Just comment the line 65 in mono/doc/mono-build-w32.sh about AUTO_DEVEL 
then the configure script will be generated by the developpement version 
of autoconf and the test shouldn't failed.
It works for me.
