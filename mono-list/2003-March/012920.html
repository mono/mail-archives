>Hi, I'm new to mbas and I want to contribute using it.
>Do I need to pass to mbas *.vb files?

Yes, a list of all the ones you want compiled in the same assembly (EXE or 
DLL). In VS.NET you see it as all the sources inside a project inside the 
solution.

>How can I pass all the .vb s files?
Just separated by spaces.

Example:

mbas source1.vb source2.vb source3.vb


>Do I have to specify in the command line the imports or it is automatically 
>done by mbas?

Yes, with the /r option

Example:

mbas /r System source1.vb source2.vb source3.vb

>May I pass a .sln or .vbproj file?

No you can't, but tha are tools to convert them to makefiles
