On 11/01/06, Jonathan Pryor <jonpryor at vt.edu> wrote:
> Consequently, "reliance on distributions" is a good thing -- you get the
> entire package dependency tracking & update mechanism for free.

I'd go for that if you could rely on the distributions to be
up-to-date, but for example Ubuntu Breezy is on 1.1.8 so if I want my
application to work "out-of-the-box" I have to develop with an older
version.

I'm hoping that with the forthcoming 2.0 release that this will become
much less of a moving target for the distributions, but for that to
really work the developers working with (not on) Mono need to be
convinced to use the stable release and stop tracking the development
ones.
