It is not ASP.NET specific, but I've started to use rico 
<http://openrico.org> on a project and in some initial experimentation 
with ASP.NET (albeit simple) it worked fine. It requires some specific, 
but simple, XML response from the server, and is a capable client 
library. Uses an Apache 2 license. There is a pdf under the 
documentation section that explains the relevant pieces, it was pretty 
easy to get going using a IHttpHandler on the server to send the proper 
responses.
