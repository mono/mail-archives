Microsoft's behaviour is correct (maybe ECMA states it otherwise, but I
don't know) because to fully specify an assembly you must either provide a
qualified name like this (when the assembly resides in Global Assembly
Cache):

System.Data, Version=1.0.3300.0, Culture=neutral,
PublicKeyToken=b77a5c561934e089

or just "System.Data" (when your assembly is in the same directory as *.exe)

Actually there are some more options, like "codebase" hint in application
config file. See MSDN for more information.

For now, mono just loads assemblies from directory pointed by MONO_PATH
variable.
