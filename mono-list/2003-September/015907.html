As the author of one open source C# project (Bamboo.Prevalence) I can say
that I pretty much expected nant build files to be enough for compatibility.

I think that making sure the latest nant releases run fine under mono is
probably easier and more productive than going after c# open source authors
and ask them to include makefiles. Of course, open source projects can
always count on the help of great people (and Rafael Teixeira is helping bp
to compile under mono ;-)) but that effort could be put to better use could
we count on nant files being enough.
