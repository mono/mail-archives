> We previously had problems with software written in
> OCaml using OpenGL and had to shelve the product line 
> because the binaries would not run reliably between distros.

Have you looked at the Linux Standard Base standards? Those standardise
binary interfaces, including OpenGL, between different versions of
Linux, 
and seem to be the best way to get binary software running across
different 
distros. I've used them successfully for the Parasolid commercial solid-
modelling library. 

They don't know anything about OCaml, of course, but I suspect that 
if OpenGL was your problem, rebuilding your OpenGL bindings for 
OCaml as LSB-compliant libraries might solve the problem.

www.linuxbase.org
