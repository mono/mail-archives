Hello Atsushi,

> dblinq does not really use too new features of nunit.
> It's just that test assemblies built with NUnit 2.4 (that are 
> what they ship) do not work with 2.2. TestFixtureAttribute 
> and TestAttribute from 2.4 are ignored in 2.2 (probably 
> because they have different assembly qualified names).

Hmmm... that's odd. Of course 2.4 was built for backward compatibility
with 2.2, and what you describe is more like forwared compatibility.
The general idea was that people would use new copies of NUnit with
old framework references, not the reverse, so I have never tested 
in this direction.

That said, it should work  - barring bugs - since the namespace
has never changed. If it's useful to fix this, send me an example
offline and I'll try to figure it out.
