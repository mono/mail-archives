Katmai wrote:
> Is Mono fully compatible with .net framework now?If not,when will it
> be?(or will it be?)

There are pieces of .NET that are not yet implemented in Mono, but they
may be pieces you do not need. APIs are implemented based on real world
use, so chances are the APIs you need are implemented and well tested.

If you have an application you want to migrate, try running the
migration analyzer (MoMa) on it to see if Mono supports the pieces of
.NET used by your app.
