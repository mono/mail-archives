In addition to the other responses, note that you COULD use Xwt, but force it
to use the Gtk bindings even if on Mac and/or WPF, or better yet, you could
allow user control over which interface to use.  This would allow you to
choose the "faster" and "smaller footprint", but still give your users
ability to forego this and select the interface they may be familiar with.

That said, Xwt is probably too early in development, and you may find lots
of missing functionality.

For what it is worth, I've been developing my application with a Gtk GUI,
but have started to concurrently redo the GUI in Xwt.  I can run either of
the interfaces (the only GTK only, or the new Xwt), so it has not stopped
current development.  The works especially well if you keep your backend and
GUIs properly seperated.
