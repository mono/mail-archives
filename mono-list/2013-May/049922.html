I am not talking about application security here. Both Java and .NET are
vulnerable to things like SQL injection, disassemblers, plain text
configuration files, etc. My point is that Unix-like servers are
architecturally more secure than Windows servers. For example, Unix-like
servers address the issue of privilege separation at the architectural
level:
http://www.techrepublic.com/blog/security/unix-vs-microsoft-windows-how-system-designs-reflect-security-philosophy/4627.

>Point #2 also, very arguable, not worth arguing.

Solaris, for example, has demonstrated linear scalability to 64 processors
in a system, has databases larger than 5 terabytes, and lets thousands of
users connect. Windows does not come close. Simply google the web and you
will also find that Unix-like servers provide higher availability than
Windows.
