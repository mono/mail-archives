I've released the first version of an Open Source grid, dubbed MyGrid (http://mygrid.sourceforge.net). This grid uses .NET Remoting heavily, and seems a much better design than Globus or OSGA and overpriced commercial grids like DataSynapse. The idea is to port it to Mono and Java and make run on linux. See below for features requests. Any sf.net users can pick a task and start contributing right away.
 
Features needed:
 
1. Goal: Make grid run on linux
    Task: Compile Engine and dependencies on Mono
    Output: Mono Engine, Mono Shell JobProcessor, interoperability w/win32 Cluster
    SF request url: http://sourceforge.net/tracker/index.php?func=detail&aid=881912&group_id=99733&atid=625181
 
2. Goal: Make grid run on Java
    Task: Create java version of the Engine 
    Suggestion: use IIOP.NET http://iiop-net.sourceforge.net
    Output: Java Engine, Java Shell JobProcessor, interoperability w/win32 Cluster
    SF request url: http://sourceforge.net/tracker/index.php?func=detail&aid=881913&group_id=99733&atid=625181
 
3. Goal: Add Job Deployment feature
    Task: add a feature to make job/resource deployment to the client Engines 
   SF request url: https://sourceforge.net/tracker/index.php?func=detail&aid=885618&group_id=99733&atid=625181
 
4. Goal: Add embedded web server for Cluster/Engines management and monitoring
Preferrably w/o using IIS.
Suggested: XSP server shipped with Mono
