But how this could be seen from the code?
It seems too bad if environment has such a big effect on the code.
Besides Mono has its own coding conventions.

As for csproj files for the library - it would be hard to maintain.
Library makefiles depend upon *.src files in each directory to add/exclude
files from the build and also to mark the configuration-dependent files
(windows/unix).
So this should be mapped to VS projects, and it's better to be done at least
semi-automatically.
That is, if cspproj file is not somehow derived from makefile/src, this will
lead to two sets of build files to be manually maintained in parallel.
On the other hand, I have a scipt to create NAnt build files from Mono's
build files. I'll take a look whether it's possible to generate csprojs in
the same manner.
