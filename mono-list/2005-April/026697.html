On Wed, 2005-04-20 at 10:42 -0500, Jayme wrote:
> Hello,
> 
> Do you guys have any information as to when Monodevelop 0.6 will be
> available in the linux binary installer for mono or one of the Fedora
> Core 3 RPM releases? I want to be able to use the latest Monodevelop
> when it comes out but currently this requires getting the tarball and
> compiling against a bunch of other dependencies that are not available
> as a single mono release (Gtk# 2.0, which depends on Gnome-Sharp 2.0,
> and wierd versions of others).

As mentioned in my previous emails, Gtk# 1.9.3 (the beta of 2.0) is
available from the new FC yum repo. Once you get that, you will need to
get gecko-sharp and gtkhtmlsourceview-sharp (I am working on packaging
these as well) and then install md from tarballs.

For gecko-sharp, you'll want to use SVN -- when I tried to package it,
there was a build error vs 1.9.3. We'll repackage and rerelease soon.

> The Monodevelop IDE being included with the linux installer is great I
> just would like to see you guys come up with (or help start) a way to
> make sure that folks can get the latest monodevelop milestone releases
> without a ton of hunting and pecking for the dependencies. Maybe this
> is more something to ask the Monodevelop team than this list?

I think it is unlikely the new MD would be included with the installer.
Gtk# 2 requires GTK > 2.4 (and will soon be > 2.6). That conflicts with
the goal of portability for the installer.
