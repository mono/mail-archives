On Tue, 2005-04-12 at 09:56 +1000, simon wrote:

> Here is an example of the problem using nGallery which makes use of 
> httpHandlers
> 
> in my apache conf file
> 
> Alias /nGallery "/var/mono-apps/nGallery"
> MonoApplications "/nGallery:/var/mono-apps/nGallery"
> 
> in my sites-enabled (using the ubuntu distro)
> 
>         <Location /nGallery>
>                 AddHandler mono .aspx .ascx .asax .ashx .config .cs .asmx
> #               SetHandler mono
>         </Location>
> 
> Now if you browse to the url below, you will see that the images which 
> are generated on the fly using the httpHandler do not show. 
> 
> http://144.132.106.142/nGallery/default.aspx

I think the problem is that, as apache can't find the directory to read
the configuration from, it fails. Does that /var/mono-apps/nGallery/1
directory exist? If not, create it and apache will pass the request to
mod_mono (even if the file does not exist). Otherwise, with AddHandler
only, it never gets that far.

Is there anything under that directory that you want served by apache
and not mod_mono? If not, AddHandler does not make any difference over
SetHandler.
