Hello,

While the current version of Mono on github can consume PCL libraries, Mono
does not have a full open source implementation of the Microsoft Reference
Assemblies.

Reference Assemblies are basically API maps that describe which APIs you
can consume for a particular combination of runtimes you can target.

For example, if you want to target something that includes the old Xbox,
Silverlight 2 and modern .NET you would be given a particular profile that
gives you that.    If you want to target the Windows Store and Windows
Phone 8, which have a newer version of the APIs, you would be given a
different profile.

We currently lack the manpower to reproduce the reference assemblies that
Microsoft has created for all the combinations that they support.

The choices are: (a) to always build on Windows, where the Reference
assemblies exist, or (b) mount the Windows file system and expose the
reference assemblies in a place that Mono can pick them up.

On MacOS that is:
/Library/Frameworks/Mono.framework/External/xbuild-framework

And on Linux that is: $(libdir)/mono/xbuild-framework

The layout is compatible with the Microsoft's layout, so you would need to
place the files like this:
