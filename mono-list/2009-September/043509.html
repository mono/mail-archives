If path to mono.exe contains unicode symbols (russian for example) and one
try to launch something (like  ...\mono.exe helloworld.exe) then mono can
crash with error:

The assembly mscorlib.dll was not found or could not be loaded.

The error is not stable. It can be repeated on purely installed WinXP SP2.
The problem dissapears after installation some soft. For example,
installation of .NET Framework 2.0 redist fixed the problem and
uninstallation get it back.
