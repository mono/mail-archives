There is a patch in CVS for mod_mono to work with multiple virtual hosts.  I
have not tested it out yet so I cannot give you any pointers on using it.

As far as the "application" goes, each ASP.NET page does not have to be part
of an application.  They can be served up in the same way php and perl can.
What an application allows you to do is actually build a web application in
much the same manner as you would go about writing a desktop application.
There is an optional global application file that can be used so multiple
pages in the application can share information with each other as well as
define tasks common to the application as a whole.

I hope that helps.
