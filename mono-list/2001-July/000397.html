Second mail, that one answer my questions:

<< A consequence is that if you choose to use GPL'd Perl
modules or Java classes in your program, you must
release the program in a GPL-compatible way,
regardless of the license used in the Perl or Java
interpreter that the combined Perl or Java program
will run on.  >>

It seemed to me from reading the LGPL license that the program should be
available for modification to the end user. It puts some restrictions on the
licensing of the program being linked to the mono class library, and as such
we should choose a non propagation license for the class library.

I fully agree and embrace the idea of releasing the JITs, compilers and such
as GPL.

Sorry for the confusion between the applications and the class library,
hopes this message clears everything.
