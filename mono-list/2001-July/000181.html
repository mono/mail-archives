> Microsoft have said repeatedly that the CLI *can* support multiple
> languages, but how many *does* it?

Lots.

Microsoft themselves ship production compilers for Managed C++, VB.NET,
C#, JScript.NET and MSIL. The Framework SDK also include demo source for
compilers that target massively sub-setted Lisp, a C variant called SMC,
and a Tiny C-like language.

Third parties are/have produced compilers for Perl, Python, Cobol,
Eiffel#, Scheme, Haskell, Mondrian, Mercury, Component Pascal, you name
it...

There's a list here (watch for wraps):
http://www.gotdotnet.com/resourcecenter/resource_center.aspx?classificat
ion=Language%20Vendors
