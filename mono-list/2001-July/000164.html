> java should be the only two for the time being.

If the point of this project is an alternate implementation of the CLI,
why on earth worry about Java yet? It's not in the CLI specs and there's
no .NET code written in Java yet outside of research labs :-).

IMHO the focus should be laser-sharp, on building a high-quality C#
compiler and CLI implementation (and here I *specifically* mean the VES
& 250-odd CLI classes, not the 3800+ that are in .NET and *not* in the
CLI). 

More classes and other language compilers/porting tools are a natural
once the core is in place, but let's walk before we run, for heavens
sake!
