The issue is broader, because if someone uses COM-Interop, specially as a 
consumer of COM objects, while developing in Windows, he is making his code 
non-portable, because most probably the COM object won´t have a twin brother 
in Linux and won´t run out of Windows. Unless WINE and/or Lindows really 
mature enough or the COM object runs on Windows and is accessed via DCOM 
(using EntireX or a Mono substitute).
