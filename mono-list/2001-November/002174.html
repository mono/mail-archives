From my understanding, dcop requires qt's serialization methods and are
therefore not portable. We would have to depend on QT for everything in
that case, and I dont think we want to add that dependency. But, I dont
beleave anyone is aposed to someone writing a class to bind to dcop so
that KDE developers can use it from mono. We are planing on doing
something similar for GNOME so we can access Bonobo.

We dont need to just support Bonobo or Dcop. We can support whatever
anyone wants to bind.
