You could just go into mono/runtime/Makefile
and where you see this line

monobins_DATA = mcs.exe monoresgen.exe

remove monoresgen.exe

If you already have a working mcs and mono,
and you have he mcs source, cd to mcs/monoresgen

on Linux, type: make -f makefile.gnu
on Windows with .NET Framework, type: make

From cvs or any tarballs, both mcs and mono sources should be placed in a
common
directory, such as, mono.

.         mono (common directory)
.           |
.    ---------------
.    |             |
.  mono           mcs
.    |- data       |- class
.    |- doc        |- doc
.    |- docs       |- doctools
.    |- libffi     |- errors
.   ...           ...
.    |- runtime    |- monoresgen

I find this monoresgen thing to be annoying, luckily I already had mcs
binaries on my computer.
