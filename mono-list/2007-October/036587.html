Managed D-Bus, aka. dbus-sharp, has hit 0.6.0. There is also a new 
release of the GLib integration module, 0.4.0.

ndesk-dbus 0.6.0
ndesk-dbus-glib 0.4.0

Changes for these releases can be viewed at:
   http://git.ndesk.org/?p=dbus-sharp
   http://git.ndesk.org/?p=dbus-sharp-glib

Sources can be downloaded at:
   http://www.ndesk.org/archive/dbus-sharp/

For more information:
   http://www.ndesk.org/DBusSharp

The changes here are mostly incremental, with the two most notable 
additions being a fix for Bus.Register(), which was previously accepting 
a redundant parameter and a fix for a PPC regression in the GLib module. 
API/ABI remains compatible with all previous releases.

The build system has been switched from a set of custom Makefiles to 
autotools to ease the job of packaging and the upstream tarballs have 
also been renamed as planned.

A user pointed out to me recently that Fedora appears to be providing at 
least four or five different and antiquated versions of these modules in 
their distribution. This is not a responsible way of shipping 
security-critical system libraries. The good news is that all the other 
distributions, even the silly ones, are getting this right.
