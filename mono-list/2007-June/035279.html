Found the problem...Novell is looking for the Mono.Security.dll in the
following location on 2003

C:\Windows\System32\inetsrv

A good way to find out is to download filemon and this will tell you where
Mono is trying to be accessed from. 

Would have been nice for this to be in some sort of installation
documentation from mono all the same....

Hope this helps someone
