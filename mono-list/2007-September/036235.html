Dear MonoDevelopers,

 

I don't know if this should qualify as a bug or as a new feature, but there
are some lacking formats in the array returned by
DateTime.GetDateTimeFormats()

The Microsoft version returns an array with I think about a 100 formats,
whereas the mono version returns an array of only 26 I think. 

 

I don't know if this is wanted or not. Maybe a warning in the documentation
or enlarging the array to the same number of formats then Microsoft could be
done, so that applications don't give index out of range exceptions.

Anyway, just wanted to be sure that you guys were aware of this.

Regards,

Matthias
