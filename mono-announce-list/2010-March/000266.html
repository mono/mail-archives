Hi Everyone,

I'm proud to announce Tomboy 1.2.0, the first release of the
stable 1.2.x series, which coincides with GNOME 2.30.0.

Tomboy has been branched as "gnome-2-30" for stable (1.2.x) development.
Git master is now open for new feature development (1.3.x).


What is Tomboy?
==================================================

Tomboy is a simple personal note-taking application designed to be
unobtrusive and friendly, while supporting inter-note-links similar to
a Wiki to help you organize your notes and ideas.  It can run either as
a GNOME panel applet or a notification tray icon, and is also supported
in your Windows tray or Mac OS X dock.

What's new in Tomboy 1.2.0
==================================================

Major changes since the 0.14.x series include the following:
* Automatic Background Synchronization
* Copy Rich Text To Evolution, OpenOffice.org, etc (Gabriel Burt,
requires gtk-sharp >= 2.12.10)
* Safer note renaming
* Almost Instant Startup
* Jump List support on Windows 7 (Sefan Cosma)
* Expanded DBus Interface (Celemens Buss, Matt Jones)
* See below for the many bug fixes

Changes since 1.1.4 (the last development release) include the following:
* Handle new application/x-note mimetype (#611865, Olivier Le Thanh Duong)
* Add a text/path-list target when drag-and-dropping from Search window,
  allowing basic note transfer via Empathy (#611858, Olivier Le Thanh Duong)
* Allow icon themes to customize the panel/tray icon separately (#612887)
* Fix icon embedding in Mac build (#612893)
* Fix errors when using GTK+ >= 2.19.7
* Translation updates: bn, da, el, eu, fi, gl, it, ko, nb, lt, pa, pt,
ru, sv, uk

The next six months...
==================================================

Next cycle, I intend to focus almost exclusively on stabilizing Snowy,
launching Tomboy Online, and improving the user experience of Tomboy note sync
even more.  Specifically, I'd like to:
* Launch Tomboy Online concurently with the GNOME 3.0 release
* Allow Tomboy Online users to share notes with each other directly in Tomboy
* Let automatic background sync provide feedback to user when server
is down, etc
* Streamline Tomboy UI by removing as many dialogs as possible
* Enable opt-in anonymous usage statistics collection (like Banshee)
* Continue improving startup performance and memory usage
* Continue whittling away at our bug list

I'd love to see other feature work from the community, though, so feel free to
bug me with your own innovative patches. :-)
