Dear Debian CLI package maintainers,<br>
<br>
I am happy to announce the release of cli-common 0.7 and with that an<br>
updated Debian CLI Policy version.<br>
<br>
This policy update is announced as it contains important changes for<br>
packages, which all CLI package maintainers should be aware of.<br>
<br>
     Changes from 0.5.1 to 0.7:<br>
<br>
        * Section 3.1.2, `File Locations&#39;: GAC libraries must now go in<br>
          /usr/lib/cli/assembly_name-X.Y instead of<br>
          /usr/lib/upstream_package_name-X.Y.  as one source package<br>
          might ship many assemblies with different ABI versions.  This<br>
          would produce very confusing directory names.<br>
<br>
        * Section 3.2.1, `Naming &amp; Versioning&#39;: Late-GAC install is now<br>
          mandatory.<br>
<br>
        * Section 3.1.4, `Build Dependencies&#39;: Added CLI SDKs as<br>
          alternative to the compiler.<br>
<br>
        * Section 3.2.5, `Signing&#39;: Using the mono.snk key of<br>
          cli-common-dev is now mandatory if upstream doesn&#39;t ship one.<br>
<br>
        * Section 4.3, `MONO_DISABLE_SHM&#39;: Replaced MONO_SHARED_DIR<br>
          workaround with cli.make and MONO_DISABLE_SHM.<br>
<br>
        * Section 6.2.2, `debhelper 7 Example&#39;: Added debhelper 7<br>
          example.<br>
<br>
        * Section 3.2.2, `Policy Files&#39;: Added reference to the<br>
          mono-api-check tool and made raising clilibs version<br>
          mandatory.<br>
<br>
        * Section 2.4, `&quot;.NET&quot; or long &quot;Microsoft .NET Framework&quot;&#39;:<br>
          Updated URL to Microsoft .NET Guidelines.<br>
<br>
        * Section 2.6, `Package Names&#39;: Made upstream tarball names<br>
          clearer.<br>
<br>
        * Section 3.1.3, `File Permissions&#39;: Replaced find commands<br>
          with dh<br>
          + cli.make.<br>
<br>
        * Section 3.2.1, `Naming &amp; Versioning&#39;: Removed <a href="http://ASP.NET" target="_blank">ASP.NET</a> as it&#39;s<br>
          not a programming language and added IronPython and IronRuby.<br>
<br>
As usual you can find the current Debian CLI Policy in the cli-common<br>
[0] package and also on the pkg-mono website [1].<br>
<br>
For people interested in the development of the CLI common<br>
infrastructure, the cli-common source package is now maintained in a<br>
git repository [2].<br>
<br>
If you want to read the updated policy using the debian package,<br>
you probably have to wait for the next dinstall + mirror push.<br>
<br>
In this policy update are not all known issues addressed yet. For Debian<br>
CLI Policy 0.8 we seek for discussion to address them. Details will be<br>
posted later with a separate post.<br>
