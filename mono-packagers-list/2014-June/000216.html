Hello list,

since we have had several annoying bugs in the past like eg
https://bugzilla.xamarin.com/show_bug.cgi?id=18690, I have now set up
a nightly build for Mono master and the current branches.
That will run make dist and build an RPM.
This should catch errors immediately, which has already worked: for
example see pull request https://github.com/mono/mono/pull/1098
It is much easier to fix a packaging problem early, when the problem
was just added to master...

See the log of builds here:
http://lbs.solidcharity.com/detail/tpokorra/mono/mono-opt-nightly

The script for it is stored here (setup.sh) and the sample spec file:
https://github.com/tpokorra/lbs-mono/tree/master/mono-opt-nightly

I have done the same for monodevelop as well:
http://lbs.solidcharity.com/detail/tpokorra/mono/monodevelop-nightly

I hope this will benefit in the end all packagers, since there should
be a usuable tarball on release day.

All the best,
  Timotheus
