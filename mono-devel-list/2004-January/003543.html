> Uri from string.

> 
> Steps to reproduce the problem :
> 1. Create new Uri from string that already contains escape characters

> 
> Actual result :
> "C:\\tmp\\a%2520a\\"
> 
> Expected result :
> "C:\\tmp\\a%20a\\"
> 
> Enclosed is a small test case and a patch.

> Thank you in advance,
