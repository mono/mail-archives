Hi *,

Here are the patches for System and System.XML build.

The first is related to circular System and System.XML references. This "sort" of patch is required because without it we cannot build CLS-Compliant System.dll because it depends on not CLS Compliant dll (old one) and this leads to many CLS Compliant error. Only one workaround is building the System.XML via csc. But this is probably not good solution.

Second patch is patch for previous patch for mcs. It fixes error when you use #if after CR line end (Bug #53386). And without it you get build errors on the windows and maybe on the Linux too (I didn’t test it).
