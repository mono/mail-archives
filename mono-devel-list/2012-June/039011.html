Maybe you're running an older version of my unit test, before I added code that ensured the event-handler was being called.  I've attached a fresh copy of patch #1.

Enclosed is a reworked version of patch #4, one whose behavior has been tested against .NET.  Apply the unit test patch, see that you get a unit test failure, then apply my patch, and see that the unit test no longer fails.  Also enclosed is a ComboBoxTest project containing the unit test, so you can run it under .NET easily.

Also, I noticed that bringing up a dialog box before calling Application.Run() tends to make Mono freak out.  Maybe that's the problem with your DgwTest project?  The enclosed ComboBoxTest project waits until the Form's Load event to bring up dialog boxes.

I will go and redo patch #3 so that it works under .NET, and try to come up with unit tests for patches 2 and 7.  Those require user interaction...if I can figure out how to fake a mouse click, I should be able to do this.  Looks like the accepted method is to create a subclass and then call OnMouseClick() directly.
