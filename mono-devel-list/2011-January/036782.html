Up until 2.6 series I used to rebuild from source rpms offered by novel for
suse/opensuse, but at some point it started to be a bit cumberstone and
required too much tweakening of some the spec files.

Right now I just use fedora's mono source rpms with upgraded mono versions
and some minor fixes where aplicable.. (ie. when some dependency does match
or is still not available for centos/el). For the core rpm's it's nearly
painless.. just rpmbuild -ba package.spec and go fly.. ;)

However I'm just in the same situation as yours (for each new mono version
I'm in the need of new packages) so if you feel we can colaborate or share
resources I'll be glad ;)
