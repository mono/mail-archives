As I said earlier, we have a building tool (buildroot) and we can't change
our makefile(s) every time a tarball release is broken (for any packages,
not just mono).
The official release is the tarball, right ? Git is for devel, furthermore,
our building server do not have any link to the internet (and no, we do not
want to have a local clone of any packages we need). All packages we used is
in tarball format

In the 3.0.10 tarball, there is a missing directory (this is simple text
file instead...), so, we can't build, the configure step fails miserably.

FYI, I made a workaround that delete the text file and add the missing dir,
it works well in 32 bits but the mcs build is broken too in 64 bits.
