And the server seems to exist, even if it's not the "official" name anymore.

In case a user needs to switch to a different server URL (like
anonsvn, for example), that magic can be handled by the rewriteRoot
config file option of git-svn.  In my case, on the other hand, I don't
have svn commit access anyway and the current repo seems to be updated
frequently, so I probably don't need to use git-svn at all.
