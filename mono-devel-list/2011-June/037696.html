According to the release notes, the bug that Zoltan suggested as the culprit
did not make it into Mono 2.10.2:

http://www.mono-project.com/Release_Notes_Mono_2.10.2

So, running Mono 2.10.2 does nothing to verify if this is the bug here or
not. You would need to build from source (which is pretty straight forward
on Linux once you get the dependencies for libgdiplus installed).

IMHO, building the source of the latest stable branch (2.10 right now) seems
like the best way to verify that any bug you run into has not already been
fixed. It is also a great way to not run into those bugs at all and save
yourself a bunch of time.
