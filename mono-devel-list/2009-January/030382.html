>From the site: 
"System.Data.SQLite is the original SQLite  <http://www.sqlite.org/> database engine and a complete ADO.NET 2.0 provider all rolled into a single mixed mode assembly"

Mixed mode assemblies aren´t portable, period. They contain x86 native code in Windows-specific format, so it won't run in any other OSes or any other CPU architecture.

If you really need a portable solution (across OSes) you should use Mono.Data.SqliteClient, and package both native library versions o sqlite (.dll and .so), the correct one will be used automatically by the managed provider. CPU portability is harder to tackle, and would only be reasonable to achieve by implementing a 100% managed version of the engine+provider.

Hope it helps,
