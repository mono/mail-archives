Small update:

Since I control the content of the DynamicMethod, I add some "if (firsttime)
return" code at the beginning and before I call
dynmethod.MethodHandle.GetFunctionPointer() I run the dynamic method once
with Invoke(). That allows me to get the function pointer without mono
crashing. But I still get intermediate problems later when the patched
original is jumping to my new method.

The basic question remains: where is the assembler code of a dynamic method
so one can jump to it?
