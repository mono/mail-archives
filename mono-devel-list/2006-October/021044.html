I agree with Zoltan, that we should use the same parameter names as 
Microsoft does. And as he said corcompare will ensure compatibility if we 
add support for that.

And I believe that all publically visible parts of the metadata should be 
considered being the API, and one of our aims is to provide an API that is 
compatible with MS.NET.

So the answer is that you can rely on parameter names but just like anywhere 
else in Mono there can be incompatibilities that can be considered being 
bugs and you can file bug reports and/or propose patches.
