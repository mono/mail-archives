I've noticed a few CVS commits recently with non-existent or useless log
messages.  Here are a few examples to embarrass the guilty, and
hopefully to educate everyone:

--
Module name:    mcs
Changes by:     carlos  03/10/20 17:24:28

Modified files:
        class/Mono.Security/Mono.Security.Protocol.Tls: TlsProtocol.cs 

Log message:
Updated file

--
Module name:    mcs
Changes by:     carlos  03/10/20 18:14:35

Modified files:
        class/Mono.Security/Mono.Security.Protocol.Tls: TlsSession.cs 
                                                       
TlsSessionContext.cs 
                                                        TlsSocket.cs 

Log message:

--
Module name:    mcs
Changes by:     eran    03/10/22 02:13:36

Modified files:
        class/System.Data/System.Data.Common: ChangeLog 

Log message:

--

Please, everyone, lets have concise but comprehensive commit messages. 
The usual best practise is to copy the ChangeLog message into the CVS
commit buffer.
