Hey guys,

    I usually read the ChangeLog messages and notice that bugs are
fixed, but they only have a number associated with this.  It is best if
on the ChangeLog entry you describe what the bug is and what the bug fix
does.  Remember that part of the asset in the ChangeLog is that it can
be reviewed offline.

    Also a downside is that if we ever loose the Bugzilla (think 10, 20
years from now), it is possible to understand what decisions and changes
were done without having the database around.

    Another small request for people applying patches on behalf of third
parties: when I prepare the release notes, I typically grep for a few
patterns to find all the contributors, it would be very useful if you
put in your commit message:

	"Applied patch from NAME"

    So I do not miss people when writing the credits.

Thanks in advance,
Miguel.
