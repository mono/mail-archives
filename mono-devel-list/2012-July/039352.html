You can try a small trick to see if this is the case.

On your main function, before you go to that hot loop (it must be on a
separate function), you call into the vector code.

Eg:

static void Main () {
 new MathNet.Numerics.LinearAlgebra.Double.DenseVector (1); //or whatever
type you use.
 method_that_uses_it ();
}


If this works, then it would be interesting to see how MS get away with it.
