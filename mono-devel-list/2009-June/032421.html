We (at Unity) tried running some old version of Mono (1.1.x probably)
on Win98 some 3 years ago. It was a custom (Visual Studio based)
build, built with these delay loading instructions.

In short, it mostly worked, except when it did not. I.e. from time to
time there were some mysterious crashes that looked like some
peculiarity in memory mapping or in GC or somewhere else. I did not
investigate it further and instead used it as an argument for "nah, we
don't need to implement win9x support" :)

Based on the casual-ish game player statistics we have [1], Windows
2000 accounts for 0.6% of the market share. So I'd expect Windows 9x
is pretty much dead.


[1] http://unity3d.com/webplayer/hwstats/pages/web-all-os.html -
statistics from 2008 February till now, about 10 million sample size.
