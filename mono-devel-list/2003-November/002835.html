Hello All

    Clearly for the purposes of releasing and persuading businesses to adopt
it a 0.x versions number isn't very attractive. Something that we could
consider (although it goes against fine unix tradition) is to use an
alternative system for versioning. For example "Mono Release A" followed by
Release B or "Mono I", to be followed by "Mono II". These would allow
versions that couldn't be confused with the microsoft naming schemes.
Of course this does not fully address the problem that users will try their
MS.NET apps against mono and find that they may not work 100%, there will
still need to be a big push on the "mono doesn't do everything - yet" idea.

Theo
