Well, I'm still an outsider too ;).
You're certainly right, but I think the problem is that .90 would indicate
that it isn't quit finished, and much ppl say:"Let's wait till 1.0". While
actually, the things released then are fully completed, be it that it
doesn't contain every technology contained in MS.NET 1.0. 
In that perspective 0.90 would be the same as 0.28, it tells ppl "we're not
quite done yet."

So Mono wants to tell:"Our technology is ready for broad usage", at the same
time clearly indicating that it doesn't yet contain everything MS.NET does.

Sounds like a problem for the Ximian/Novell marketing department to solve ;)

I would recommend as part of the marketing to create a "customer" site for
Mono, targeted at users and managers only(you could see the current Mono
site as a developer site). Much like Sun has done with java.com / java.net. 
This site would be part of getting the Mono message to users, telling them
what Mono is/ isn't, and what they can/can't do with Mono.
