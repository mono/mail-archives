Ok, I was able to get around, but I still think it is a bug with xsp on 
windows and will fill
a bug report about it.

In Windows directory, monobasepath was containing a line:
set MONO_CFG_DIR="C:\Program Files\Mono-0.28\etc\mono"
that I changed to:
set MONO_CFG_DIR=C:\Program Files\Mono-0.28\etc

Then xsp.exe was able to read my machine.config file and / error in 
application with
String returned by HttpRuntime.GetResourceStringFromResourceManager then
disappeared.

I think monobasepath should not put \" characters.
And I think that xsp is wrongly adding mono at the end of MONO_CFG_DIR.
