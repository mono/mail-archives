Matthew, I think you're confusing winforms with GUI applications. GTK# 
lets you build GUI apps, but it's not WinForms. WinForms refers to the 
Windows.Forms namespace in the Microsoft .NET Framework and it's 
classes. You /can/ write GTK# apps today. GTK# apps are portable between 
Windows and Linux but you need the gtk-sharp libraries on the target 
machine. Advantage of the WinForms implementation would be that you can 
do WYSIWYG development on Visual Studio .NET and run the app on 
non-Win32 platforms.
