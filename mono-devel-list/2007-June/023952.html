Hi guys,

in preparation of merging the `debugger-dublin' branch, I had to
increase the debugger version number in the runtime.

I'm keeping backwards compatibility, but in order to achieve this, I had
to bump the version number.

This means that the 0.50 "Dublin" packages of the debugger won't with
the latest mono from SVN anymore.  If you want to use the latest mono
from SVN, you also need to use the latest debugger from SVN.

The only change on the debugger's is that it now also accepts (but does
not require) the new runtime (in MonoThreadManager.cs:
MinDynamicVersion == 58, MaxDynamicVersion == 59)

We may make updated 0.50 "Dublin" packages to reflect this, but can't
promise anything yet.
