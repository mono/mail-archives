I was able to take the daily build from 7/30 and perform a clean compile on
OS/X 10.2.x (the most recent update) using the December 2002 dev tools w/o
the GCC 3.3 update. Of course, I had downloaded some packages with
Fink/FinkCommander necessary to support the build as well.

The only things that would not build were mini and profiler.

mcs and mint were functional, albeit a little slow (expected with no
jitter).

I compiled a few test apps with success. I also managed to successfully
compile a multi-threaded socket listener app. It threw an internal exception
regarding my use of AddressFamily.InterNetwork stating that it was an
unknown value (0x200??).

Downloading the mcs 0.25 tarball and attempting a build fails. More info on
that later.

Aside from that, I am able to work with mono in os/x again. Thanks to
everyone for tracking down the issues!
