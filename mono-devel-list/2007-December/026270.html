On Thu, 2007-12-20 at 14:56 +0100, Lluis Sanchez wrote:
> Fair enough. Just make sure you guys consider Mono.Addins if MonoDoc is
> going to include any kind of extensibility support (including support
> for third party libraries to extend MonoDoc with new documentation).
> Otherwise we'd be reimplementing functionality that M.A. already
> provides.

Too late.  MonoDoc has had this kind of extensibility support (i.e.
adding documentation for additional external libraries) for *years*;
it's how Gtk# and Mono.Fuse documentation is added, for example.

Now, this doesn't mean that using Mono.Addins wouldn't be an
improvement; it likely would, as you could update the documentation
(more) independently than is currently possible.  (It *is* currently
possible, it just requires having root access to overwrite the existing
files within $prefix/lib/monodoc/sources. :-)  Which is why it doesn't
happen except during complete package upgrades...
