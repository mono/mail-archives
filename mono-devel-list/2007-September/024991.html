On 09/21/07 Atsushi Eno wrote:
> Recently (I think it's been since middle of this week or so) I have
> been unsuccessful to build non-bootstrap mono on windows, it
> complains about missing "CreateString". Is it only me or is the
> build broken? Note that it won't show up as monobuild status
> as it is non-bootstrap build.)

This happens if you have built/installed a msimatched mono/mcs
combination. The way to fix is the same as described in the README:
install mono from a tarball to fix your system and then build and
install from svn with both the mono and mcs modules updated.
