I've run into a problem with the latest xsp from the trunk. 

Under mono 1.1.9.1 I can repeatedly cause xsp to crash by simply
pointing Microsoft's Web Application Stress Tool at it and having it
request any file, even a static file. When the Stress tool finishes its
run xsp crashes from what appears to be the result of a socket
exception. The included patch seems to solve the issue, but I don't
claim to have a solid understanding of the inner workings of xsp and
what else this may effect.

If I can provide more information that may help please let me know what
you need.
