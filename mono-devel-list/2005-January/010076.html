Maybe some effort should be made to make the valuetype operations
"smarter" for example in this code:

struct X {
...
}

if (foo) {
    X x = new X (blah);
    // do stuff with x
}

if (bar) {
    X y = new X (baz);
   // do stuff with y
}

We could assign x and y the same stack slot, even though ldarga is
probably executed on both of them.
