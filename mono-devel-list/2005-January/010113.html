Thanks for testing.

The idea of the modifications was exactly to handle the dependency. I
mean, to remove it. :)
I saw two threads in Mono ML about dependency problems of Npgsql on SWF
and so I tried to modify mono build to not use it.

I thougth that only ms users with vs.net would like to use Npgsql with
design time support so I thougth ms build would the only one which
needed it.

Do you think it is worth while to keep the dependency?

I thought that some server side installs of Mono would not have the SWF
installed. And as Npgsql install packages, like rpm, are provided on
their own, I tought it would be easier to not add the SWF dependency.

Maybe we should provide two versions of Npgsql, one with design time
support and other without it?
