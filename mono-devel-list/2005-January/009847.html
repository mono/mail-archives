This small patch is meant to test abcrem.
What it does is the following: if the check can be removed,
leave it there, but throw a different (hopefully unexpected)
exception when the check fails.
This way, the program behavior will be *immediately* different
and the bug will not go unnoticed (otherwise a misplaced check
removal is hard to detect).

This patch is not meant to be committed (I know I should also
override symbols for amd64...), but I'm posting it because I
have a strange problem.
The "inssel.c" file is generated with three extra '}' at the
bottom (which breaks the build).
Deleting them the build can be resumed.

Has anybody a clue on how this could happen?
(or give me hints on where to start to look at to fix this...)

Ciao,
