Let me try to summarize the philosophy of X-develop:

X-develop's main focus is on-the-fly sourcecode analysis covering the 
whole solution. This technology enables background error checking of all 
source files, a working usage search (unlike the one in Whidbey), 
overrides/overridden icons, reachability analysis, etc. All those 
features are AFAIK not available in VS.net, Monodevelop or #develop.

 > A question: is it planned a Windows version? when would it be released
 > aproximately?

X-develop is already available for Windows and Linux from 
http://www.x-develop.com. Please note that the Windows version currently 
supports the Microsoft .NET framework (v1.1 or v2.0beta) only. 
Developing against the Mono framework on Windows (and switching 
frameworks) is planned for a future build.
