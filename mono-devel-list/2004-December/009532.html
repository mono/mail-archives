Thanks for the feedback.

> This IDE looks good.  Any thoughts about visual designers, either for
> ASP.NET or for forms?

Not for version 1. For the first release we focus on code-centric features.

> What widget set are you using?

X-develop is written in Java and uses the Swing toolkit. This allowed us
to reuse a considerable amount of tried and tested code from our Java
IDE CodeGuide .

> Are you considering supporting NAnt as a build tool, as well as
> MSBUILD?

Yes, why not. However that would be an add-on as X-develop uses the
MSBUILD file format for its project files for compatibility with VS.net
2005. As such the developer would have to create NAnt files by hand or
we could autocreate them every once in a while from the *.*proj files in
the solution.

Does the NAnt package include any interoperabiliy tools
(converters/importers) for MSBUILD projects?

> I would also recommend exposing more aspects of your IDE in your
> plug-in API than just language support.  

This is planned, but we had to start somewhere...

> It'd also might be smart to
