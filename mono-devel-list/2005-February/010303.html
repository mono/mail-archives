Hi John,

True, newest Managed C++ from MS generates pure managed code, but it
depends on a new version of it's specific runtime managed library
(Microsoft.VisualC.dll which we doesn't have anyone actively
developing), and in many changes to the corlib (as you should know it
targets version 2.0 of the framework, which is still ahead of us both
for MS and Mono).

So I would be surprised if any Managed C++ compiled dll, would be able
to run in mono, just now.

Care to help us getting Microsoft.VisualC.dll fully implemented?
