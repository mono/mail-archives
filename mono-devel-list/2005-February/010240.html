An update on mono segfaulting.

Building mono 1.0.frog-knows from the mono1-0 branch of svn for x86
produces a binary implementation of bin/mono which does not segfault on
an amd64 kernel with NX support.

Building mono 1.0.frog-knows from the mono1-0 branch of svn for amd64
produces no binary implementation, as expected, but the produced
bin/mint segfaults on trivial .net assemblies.

This does not affect me, but I thought you might want to know.
