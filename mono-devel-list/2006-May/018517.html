Hi,

I don't think that this is ambiguous.

"A mailing list specifically dedicated to discussions about developing Mono 
and programming *with* Mono."

Because two different things are listed the first one has to mean Mono 
itself while the second one means third-party (i.e. your own) programs that 
use Mono.

"Programming with mono-specific assemblies"

This is a general purpose again not just "How to use mono-specific 
assemblies when improving Mono itself."

"Discussions about getting a managed application run with mono"

This one is clearly stating that mono-devel-list is about third-party 
programs as well.

If we want to make mono-devel-list a list that is only about Mono itself 
http://www.mono-project.com/Mailing_Lists should be modified.

Of course mono-devel-list is not for as trivial questions as "how to get the 
assembly version?" but we haven't got much of them.

Kornél
