Hi,
I have started a project, that I have called Fyre, that aims to provide 
a common library for functional languages targeting the Mono/.NET 
platforms. It will consist of a library include common types (lists, 
tuples), and also a set of guidelines and attributes for special 
constructs that need to be "translated" to the Objet-Oriented 
Intermediate Language that the jitter uses (such as sum types or variants).
There is a mailing list located at http://groups.google.com/group/fyre 
There discussions will be held, and preliminary releases of the 
guidelines that we should create will be posted. Everyone is encouraged 
to participate, so interoperation between this kind of languages can be 
actually achieved. People who have developed these kind of languages are 
specially welcomed.

Hoping lots of people contribute with the project,
Alejandro Serrano
