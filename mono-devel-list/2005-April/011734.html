> Have you considered that embedding Mono may require you to open

AFAIK, the Mono runtime is distributed under the LGPL license, while
the class libraries use mostly the X11 license.
Some tools (like the C# compiler) are GPL, but they are not involved
in embedding the runtime...

So, if they use proper care (dynamic linking?), they should be OK.

> That is unless you use commercially licensed version of Mono about
> which I know nawt.

Others will know better, but commercial support is available.
But as I said, just embedding the runtime is not a problem if you
can comply with the LGPL.
