On Sun, 2005-04-03 Duncan Mak wrote (in a message Miguel forwarded):
> In between FC2 and FC3, yum changed their metadata format in the headers
> directory (generated by yum-arch) to files inside the 'repodata'
> directory. To generate this new metadata format, we need a new tool call
> 'createrepo'. I installed the RPM and ran it on 1.1.6/fedora-3-i386. All
> should be well now.

Shortly after receiving the above message, I successfully used yum to
upgrade my x86_64 box from 1.1.4 to 1.1.6.  However, I just tried to use
yum to install 1.1.6 on my i386 box and it failed.  'yum install mono-
core' yields:

<snip>
Resolving Dependencies
--> Populating transaction set with selected packages. Please wait.
---> Downloading header for mono-core to pack into transaction set.
http://www.go-mono.com/archive/1.1.6/fedora-3-i386/mono-
core-1.1.6-1.novell.9.1.i386.rpm: [Errno 4] IOError: HTTP Error 404: Not
Found
Trying other mirror.
Error: failure: mono-core-1.1.6-1.novell.9.1.i386.rpm from mono-1.1.6:
[Errno 256] No more mirrors to try.


Note that it is trying to get *.novell.9.1.* packages, but the packages
on the server are *.novell.9.2.* packages.

My x86_64 system has the 9.1 packages installed and my browser indicates
that the the 1.1.6 download page was modified on 4/6 (after I
successfully upgraded the x86_64 box), so I suspect someone just changed
the package names on 4/6, but didn't update the yum metadata.  Probably
not arch-specific, but it's hard for me to tell for certain.
