This patch add the "platform code" checks. In fact much of this is being
delegated to the host (i.e. moonlight plugin in this case) to determine
if an assembly is, or not, part of its platform code definition. From
there mono itself can enforce the coreclr security model.

The email also includes the Moonlight patch to use this. However the
moonlight patch is not 100% complete since the makefiles needs to be
adjusted for the new assemblies location.
