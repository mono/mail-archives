One of the Microsoft Bloggers has an informal survey of Mono opinions in 
his weblog at 
http://snowstormlife.com/blogs/bliz/CommentView.aspx?guid=b8872625-3381-4a7e-910f-ad2836177c91 
if you're interseted in contributing an opinion or three.  The questions 
are as follows:


A) Good for developers? B) Bad for developers?  C) Somewhere in between?
Why?
2) Is Mono...
A) Good for Microsoft? B) Bad for Microsoft? C) Somewhere in between?
Why?
3) Do you think Mono infringes on Microsoft's intellectual property rights 
for .NET?
A) Not a problem. B) I wonder when Microsoft is going to lower the 
boom.  C) Somewhere in between
Why?
4) What do you think about the viability of a Mono-based solution?
A) Academic exercise only. B) Ready today (or soon) for prime time.  C) 
Somewhere in between.
Why?
5) Do you see Mono becoming a viable web application environment?
A) No. B) Getting there fast. C) Already there.
Why?
6) Do you see Mono becoming a viable desktop application environment?
A) No. B) Getting there fast. C) Already there.
Why?
7) Do you think Mono is a threat to the Microsoft Windows platform?
A) No.  B) Yes. C) Somewhere in between.
Why?
8) Will Mono provide "write once/run anywhere" ability (as in a run on 
Mono/Microsoft .Net without re-compiling)?
A) Never. B) Will be common. C) Somewhere in between.
Why?
9) Do you use Mono for production applications?
A) Yes.  B) No. C) Currently developing one.
Why?
10) Any other comments?
