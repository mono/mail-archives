> One of the interesting points that was brought up is that the 
> System.Windows.Forms library is emulated through wine under NIX because of 
> the fact that System.Windows is very much oriented towards M$ Windows, and

Support for SWF via winelib is no more. Mono will be using it's own
implementation which talks to the underlying window manager to implement
the SWF bits and pieces. Upshot - makes blow all difference the platform
mono is being used on SWF on Windows will work and look the same as SWF
under *nix and look the same as SWF under OSX etc etc etc

Check out the latest mcs from cvs (it's all in the
mcs/class/Managed.Windows.Forms directory)
