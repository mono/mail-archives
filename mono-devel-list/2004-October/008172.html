> Hi,
> 
> I've decided to delve into the mono class library source code to figure out
> why my code breaks.
> 
> In the tarball, I find some .cs files, and a lot of .custom files, that
> appear to contain regular c# code.

It is only in the gtk-sharp tarball that you can find .custom files.

> Can anyone tell me why these .custom files are used and how they are
> different?

Since the bulk of the Gtk# source code generated, the .custom files are
used as extensions to be added after the generated source code. The cs
file in the 'generated' directory always contain the final code
(generated + customizations)
