Hello,

This patch will solve the problem that Windows "installer version" of
mono cannot run xsp. That holds mono.exe in ./lib instead of ./bin.

With the patch CSharpCodeCompiler and VBCodeCompiler tries to find
./bin/mono.bat first and if not found then tries to find ./bin/mono.exe.
