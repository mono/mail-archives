Mono-list might be more appropriate, as mono-devel-list tends to focus
more on developing Mono itself and not the uses of Mono (except when
those uses reveal bugs, in which case we're still "developing Mono
itself").

Regardless, what's your question?  The source code is being modified so
that the license is in the file header, so soon (now?) you can just read
the source to know for sure.

There's also the general overview: C# class libraries (mscorlib.dll,
etc.) are MIT/X11, C runtime libraries (libmono.so, etc.) are LGPL, and
applications (mcs.exe, mono) are GPL.  There are some exceptions (not
all applications are GPL, for example), but this is the general pattern.
