On 06/09/04 Urs Muff wrote:
> There are some well known gtk# issues, just look @ miguel's blog how he got
> MonoDevelop to run...

Which are not Gtk# issues at all.

> Gtk# is such a dependency beast (about 200 very complex ones) that only few
> hackers dared to tackle them; and none succeeded to make a binary self

There aren't 200 very complex dependencies. If you use Fink you can
trivially install the needed packages in 10 minutes with a single
command. I assume the same holds for Darwinports.

> Btw: I was the first hacker that got MonoDevelop to run on the Mac and am
> very aware of all your pains, but right now solving them of all Mac users is
> out of my control, sorry.

The only current (and past issues) after I fixed the JIT are the trivial
config file edits explained in miguel's blog (only needed for
MonoDevelop since beta2). Apart from that, MacOSX users need to fight
with autotools and pkg-config more than the average because there isn't
a standard install of those tools so they end up with tools installed in
different prefixes (and none of this is Mono's fault) and they need to
setup PATH, PKG_CONFIG_PATH etc to get the build tools to cope with
the weird installations they have.
