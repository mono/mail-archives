> 
> Well, is this quite right?  After all, the GC library seems to be modified

Our changes are completely unrelated. Moreover, since it's a warning
it's not likely we'll spend time fixing it especially when we don't have
the hw/sw combination where we could test. So it's up to you to debug
and submit a patch to us or you need to ask on the libgc mailing list
where people have far better insight on the code base.

> After all, if mono is using a modified version, how (supposing the developer

> into the GC that's part of the mono tree?

Some of our changes are local changes and don't belong to the main
version. Bugfixes get sent to the upstream developers as you could
easily check in the libgc mailing list archives.
