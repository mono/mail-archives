I've attached a patch which adds a --quiet commandline to suppress the 
"Compilation succeeded" message, but only when there are no warnings or errors.

Please consider the patch for the 1.0 release as some of us find the display of 
this message useless noise when compiling large numbers of files and would 
prefer the more conventional approach of being completely quiet unless there is 
a problem.
