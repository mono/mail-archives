I really like your idea (except the part of the hybrid three bits part,
I will explain that later) and I'm going to rework the patch to
implement it.<br>
<br>
In addition, it puts simd_version on a 32 bit boundry in the struct so
there should be no padding if we changed it from 8 bit to 32 bit member.<br>
<br>
The reason for me not liking the hybrid bit idea is keeping the bit
arraignments the same here as in AccelMode. Manly to keep things simple
for other people look at it and port the simd framework to other
platforms.<br>
<br>
But, after the NEON bit there is room for one more simd version using
this idea so we don't have to worry about that at the moment. ;)<br>
<br>
Oh, the changes in mini.h / mini-x86.c / mini-amd64.c are needed
because I was moving simd_version from a bit index marker to a bit map
to correspond directly with AccelMode enum in C#.<br>
