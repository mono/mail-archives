Hello,<div><br></div><div>     Over the years, to match the various profiles in .NET we have used conditional compilation to match the profile.    In the case of CLSCompliant attribute we had written code like this:</div><div>
<br></div><div>#if NET_XXXX  /* New in NET_XXXX */</div><div>      [CLSCompliant (false)]</div><div>#endif</div><div>      void SomeMethod (uint foo);</div><div><br></div><div>    It turns out that those CLSCompliant attributes are really bug fixes that have been done over the years.   We should stop considering those a profile-specific feature, and instead just add the attribute if required, regardless of the profile. </div>
<div><br></div><div>    So do not waste your time adding the ifdefs when it comes to this attribute.</div><div><br></div><div>Miguel.</div>
