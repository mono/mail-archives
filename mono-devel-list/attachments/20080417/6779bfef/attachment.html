<H3>
<P>Pash - <A href="http://pash.sourceforge.net/"><FONT 
color=#0a8fbc>http://pash.sourceforge.net/</FONT></A></P></H3>
<P>PowerShell open source cross-platform&nbsp;reimplementation.<BR>
<H2>About the name</H2>
<P>Pash = PoSH (PowerShell) + bash (one of the Unix shells)<BR>
<H2>Goals</H2>
<P>The main goal is to provide a rich shell environment for other operating 
systems as well as to provide a hostable scripting engine for rich applications. 
The user experience should be seamless for people who are used to Windows 
version of PowerShell. The scrips, cmdlets and providers should runs AS-IS 
(<STRONG><U>if they are not using Windows-specific functionality</U></STRONG>). 
The rich applications that host PowerShell should run on any other operating 
system AS-IS. Secondary goal: the scripts should run across the machines and 
different OS's seamlesly (but following all the security guidelines).<BR>
<H2>Environment</H2>
<P>The current implementation of Pash is written using pure .Net 2.0. It 
compiles on VS 2008 as well as on Mono. So all the developers can choose the 
environment that fits their needs and preferences. The produced assemblies can 
be executed "right out of the box" on Windows, <A 
href="http://www.go-mono.com/mono-downloads/download.html"><STRONG><FONT 
title="http://www.go-mono.com/mono-downloads/download.html&#10;CTRL + Click to follow link" 
color=#0a8fbc>Linux, Mac (or others)</FONT></STRONG></A> without any additional 
recompilation. <B>Note:</B> for Windows Mobile and Windows CE the produced code 
should be recompiled against the .NET Compact Framework.<BR>
