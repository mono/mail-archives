Hi everyone,<br>in case you had any interest, after a month of work I have bundled up my efforts on JaCIL into version <a href="http://0.5.0.0">0.5.0.0</a>.&nbsp; JaCIL supports about 138 instructions--a lot of the remaining ones are overflow and unsigned variants or instructions that are not usable in verifiable code.
<br><br>Major highlights of this release are:<br><ul><li>Object creation/initialization works (<span style="font-weight: bold;">newobj</span>).<br></li><li>Exception handlers are translated for catch and finally.&nbsp; Required CIL instructions such as 
<span style="font-weight: bold;">throw</span>, <span style="font-weight: bold;">rethrow</span>, <span style="font-weight: bold;">endfinally</span>, and <span style="font-weight: bold;">leave</span>/<span style="font-weight: bold;">
leave.s</span> are supported.<br></li><li>Managed pointers are supported.&nbsp; This includes indirect load and store (<span style="font-weight: bold;">stind.*</span>/<span style="font-weight: bold;">ldind.*</span>), array element addressing (
<span style="font-weight: bold;">ldelema</span>), local variable addressing (<span style="font-weight: bold;">ldloc.*</span>), and static/instance field addressing (<span style="font-weight: bold;">ldflda</span>/<span style="font-weight: bold;">
ldsflda</span>).</li><li>Some overhaul in the type mapping for unsigned types has been done to make things more consistent.<br></li></ul>My next target is <span style="font-style: italic;">Value Types</span> and after that there are some odds and ends before I am feature complete for my Master's project.
<br><br>JaCIL is developed primarily using Mono 1.1.13 as my development platform for the compiler (I try to test on MS .NET for completeness).&nbsp; Mono Cecil is used to read assembly meta-data and IL and <a href="http://IKVM.NET">
IKVM.NET</a> with ObjectWeb ASM is used for concrete Java class file generation.<br><br>Best regards,<br>Almann<br><br>-- <br>Almann T. Goo<br><a href="mailto:almann.goo@gmail.com" target="_blank" onclick="return top.js.OpenExtLink(window,event,this)">
