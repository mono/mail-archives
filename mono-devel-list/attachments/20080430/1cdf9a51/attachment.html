Using gettext from an <a href="http://ASP.NET" target="_blank">ASP.NET</a> app would get complicated because<br>
libintl gets the language from the LANG environment variable, which<br>
would obviously cause problems trying to use multiple languages within<br>
a single process, i.e. a threaded web server. However, there&#39;s no<br>
reason in principle why you could not hack the libintl library to<br>
allow different &#39;instances&#39; to be used from different threads, or even<br>
port it to C# and use the current thread&#39;s .NET culture.<br>
<br>
Once these problems were solved, you could use expressions like &lt;%#<br>
GettextCatalog.GetString (&quot;Translate me&quot;) %&gt;. You&#39;d also need a<br>
scanner for find extracting translatable strings, but MonoDevelop&#39;s<br>
gettext scanner could do that with a couple of regexes.<br>
<br>
FWIW, the <a href="http://ASP.NET" target="_blank">ASP.NET</a> quickstart page claims that the localisation model<br>
in 2.0 can be swapped out, but there doesn&#39;t seem to be much info on<br>
this.<br>
