<BR>
(WARNING - heavy opinions follow) - .NET is way behind J2EE in the areas of enterprise development and community maturity (breadth/depth maturity not mental maturity :-).&nbsp; However, .NET has a superior technical foundation - which in the long run could make it the defacto standard (if Java doesn't <S>smarten</S>open up).&nbsp; To me some of the major advantages of .NET are:<BR>
<BR>
&nbsp;&nbsp;&nbsp; - MSIL and the CLR is just way cooler than java bytecode and the jvm.&nbsp; Not just because of the language independence thing (which is rad - iron python will kick the crap out of jython), but more importantly because of the depth of dynamic code generation capabilities built right into the design.&nbsp; Dynamic Code generation has become huge recently in Java.&nbsp; All the newer trends (including AOP) rely on some outside byte code generation library (JavaAssist, BCEL, cglib, just to name a few... ).&nbsp; These are heavily used by JBoss, Hibernate, AspectWerks, Spring, Cactus, and other popular libs.&nbsp; Not only Code generation, but MSIL also enables huge benefits to reflection and code/assembly meta data<BR>
<BR>
&nbsp;&nbsp;&nbsp; - GAC and assembly version!! YEAH.&nbsp; Classloaders in Java blow.&nbsp; Ask anyone who's worked on distributed app servers (for an example, take look at JBoss - the unified classloader sucks and it is a huge improvement over what it used to be).&nbsp; Built in version support and Strong naming is a beautiful thing.&nbsp; Ask anyone trying to manage multiple projects and third party dependencies (or better yet, talk to some Maven folks, or take a look for yourself at their approach <A HREF="http://www.ibiblio.org/maven">http://www.ibiblio.org/maven</A>).&nbsp; Why Java never enforced the manifest or try to improve it, I will never know.&nbsp; Did I mention side by side execution of multiple versions rules.<BR>
<BR>
&nbsp;&nbsp;&nbsp; - Code Access Security (although overly complicated and 99% of apps will run fully trusted), this could put an end to some of the windows plagues of today.&nbsp; (but does it make the user any smarter?)&nbsp; CAS is Java Policy on steroids.<BR>
<BR>
&nbsp;&nbsp;&nbsp; - JNI sucks.&nbsp; enough said.<BR>
<BR>
&nbsp;&nbsp;&nbsp; - Swing Sucks.&nbsp; SWT is cool(er), Win Forms are weak.&nbsp; Avalon I know nothing about (other than the name is not original <A HREF="http://avalon.apache.org">http://avalon.apache.org</A>) How does it compare to XUL? from the million mile height they look similar.&nbsp; GTK # is the best.&nbsp; Keep up the great work!&nbsp; and without a doubt, .NET client side development trumps Java<BR>
<BR>
&nbsp;&nbsp;&nbsp; - Remoting rocks.&nbsp; Although they may have made it too easy.&nbsp; Distributed application development should never be brainless.<BR>
<BR>
Don't get me wrong though.&nbsp; For the last 6+ years my heart and soul have been in enterprise development with Java.&nbsp; There are many things about .NET I don't like (but a lot of them are either just nit picks, or just simply because its still in its early stages).&nbsp; <BR>
<BR>
