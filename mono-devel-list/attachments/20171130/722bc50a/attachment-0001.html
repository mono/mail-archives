Mono soft debugger<br>
==================<br>
<br>
<a href="https://eladnava.com/debug-remote-mono-apps-via-xamarin-studio/" rel="noreferrer" target="_blank">https://eladnava.com/debug-<wbr>remote-mono-apps-via-xamarin-<wbr>studio/</a><br>
<br>
The embedded managed code can be debugged (including breakpoints and single stepping) using Xamarin Studio.<br>
<br>
Note that to be able to debug an assembly the corresponding .mdb file must be present.<br>
If this is not the case then breakpoints are not respected.<br>
<br>
Managed app acts as a client that connects to the Xamarin studio hosted debug server.<br>
In the example below the Windows 10 VM has a configured IP address of 192.168.1.72.<br>
If using Fusion ensure that the Network Adapter preference is set to Autodetect : then set the Windows client IP address as above.<br>
<br>
1. MAC : Soft debugger client<br>
2. MAC : call +setRuntimeOptions: with IP address and port of Windows VM running the Xamarin Studio debugger.<br>
3. MAC : Run app outside of Xcode. The app will pause if the remote debugger is not responding.<br>
4. PC : Ensure that MONODEVELOP_SDB_TEST env variable is set to 1 for the current user. Right mouse Computer - Advanced System Settings.<br>
5. PC : Run Xamarin Studio (preconfigure env var as above to enable the soft debug menu) on the VM and load the target solution.<br>
<br>
7. PC : on Xamarin 5 - Select Run - Run with - Custom commnad Mono soft debugger.<br>
8. PC : on Xamarin 6 - Select Run - Run with - Custom Configuration - Run Action - Debug - CustomCommand Mono SoftDebugger - Debug.<br>
9. PC : Enter IP address and port.<br>
<br>
8. PC : Click Listen. The debugger needs to know what specific IP to listen on as it may have multiple IP addresses.<br>
9. PC : Set breakpoints as normal.<br>
10. MAC : Exercise app to trigger break point after which the managed code can be stepped through.<br>
<br>
Notes on Use of the Mono Soft Debugger<br>
==============================<wbr>========<br>
<br>
See <a href="http://www.jeffongames.com/2012/03/debugging-embedded-mono/" rel="noreferrer" target="_blank">http://www.jeffongames.com/<wbr>2012/03/debugging-embedded-<wbr>mono/</a><br>
<a href="http://mono.1490590.n4.nabble.com/remote-debugging-a-hello-world-application-td4591791.html" rel="noreferrer" target="_blank">http://mono.1490590.n4.nabble.<wbr>com/remote-debugging-a-hello-<wbr>world-application-td4591791.<wbr>html</a><br>
<br>
The debugee should connect to the debugger on the configured listener IP and port.<br>
The Windows firewall will need to allow the incoming connection : it should pop up a confirmation alert.<br>
<br>
To check if the connection is up and running use telnet:<br>
<br>
// manual check if connection to debugger listener on Windows VM can be established<br>
telnet 192.168.1.72 10000<br>
<br>
// check state of established network connections<br>
netstat -n -f inet<br>
<br>
For the soft debugger signal handler to work correctly the app must be executed without lldb attached.<br>
This can be achieved in two ways:<br>
<br>
1. Run app under the Xcode Test (ie: no debugger attached) scheme.<br>
2. Run binary outside of Xcode.<br>
<br>
Other signal handlers, such as those installed by HockeyApp, must be disabled while the soft debugger is in use.<br>
Remember to use a debug build of the managed code!<br>
<br>
<a href="http://www.mono-project.com/docs/advanced/runtime/docs/soft-debugger/" rel="noreferrer" target="_blank">http://www.mono-project.com/<wbr>docs/advanced/runtime/docs/<wbr>soft-debugger/</a><br>
