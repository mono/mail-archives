<br>
The right way to access cryptography in .NET is to use the Create()<br>
(factory) methods from the base class that represent the algorithm, not<br>
the implementation.<br>
<br>
 &nbsp; &nbsp; &nbsp; &nbsp;I.e. you should not be using SHA1Managed (or<br>
 &nbsp; &nbsp; &nbsp; &nbsp;SHA1CryptoServiceProvider) but SHA1.Create()<br>
<br>
This allows the Mono (or .NET) runtime to select the right (default or<br>
user selected) implementation for any algorithms.<br>
<br>
 &nbsp; &nbsp; &nbsp; &nbsp;E.g. it would let someone change the default SHA1 to a native<br>
 &nbsp; &nbsp; &nbsp; &nbsp;(or hardware assisted) implementation without modifying any<br>
 &nbsp; &nbsp; &nbsp; &nbsp;application.<br>
<br>
Historical note: it also served (MS) as a workaround for a bug in<br>
SHA1CryptoServiceProvider (fx1.x) on multithreaded servers. They simply<br>
(for WSE 1.0) updated machine.config to use SHA1Managed instead. That<br>
fixed the problem for every application who followed the design<br>
correctly.<br>
