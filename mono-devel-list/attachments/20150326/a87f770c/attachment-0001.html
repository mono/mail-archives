    Hi Gelin,<br>
    <br>
    I think mono is doing a good job, especially for backend uses.
    LISTEQ uses it to host their BoXedVDI Cloud Desktop software
    (<a class="moz-txt-link-abbreviated" href="http://www.listeq.com">www.listeq.com</a>) - where the mono/.net application handles all the
    network I/O, video compression and encryption. I also use mono for
    much of the globaltuners.com infrastructure, including audio
    streaming, DSP and HTTPS proxying. Both systems can easily run for
    weeks, probably much longer if it weren't for software updates,
    without crashing.<br>
    <br>
    We were recently affected by a possible (regression) bug in mono
    which caused crashes while running external processes under some
    circumstances, but this seems to have improved. You may have to be
    careful with updates, but overall it works well and can run reliably
    for a long time.<br>
