Short answer: You need a per managed dll .config file.<BR>
<BR>
<A HREF="http://www.mono-project.com/Interop_with_Native_Libraries#Library_Names">http://www.mono-project.com/Interop_with_Native_Libraries#Library_Names</A> is<BR>

<BR>
Another exceprt from<BR>
<A HREF="http://www.mono-project.com/Interop_with_Native_Libraries">http://www.mono-project.com/Interop_with_Native_Libraries</A><BR>
<BR>
Note: Mono uses GLib to load libraries, and GLib has a bug on Mac OS X where<BR>
it doesn't use a .dylib extension, but instead uses the Unix .so extension.<BR>
While this should eventually be fixed, the current workaround is to write a<BR>
.config file which maps to the .dylib<BR>
file&lt;<A HREF="http://www.mono-project.com/Interop_with_Native_Libraries#Library_Names">http://www.mono-project.com/Interop_with_Native_Libraries#Library_Names</A>&gt;,<BR>
e.g.<BR>
<BR>
&nbsp;&lt;configuration&gt;<BR>
&nbsp;&nbsp; &lt;dllmap dll=&quot;mylib&quot; target=&quot;mylib.dylib&quot; /&gt;<BR>
&nbsp;&lt;/configuration&gt;<BR>
<BR>
<BR>
<BR>
