Short answer: You need a per managed dll .config file.<br>

<br><a href="http://www.mono-project.com/Interop_with_Native_Libraries#Library_Names">http://www.mono-project.com/Interop_with_Native_Libraries#Library_Names</a> is particularly helpful. <br><br>Another exceprt from <a href="http://www.mono-project.com/Interop_with_Native_Libraries">http://www.mono-project.com/Interop_with_Native_Libraries</a><br>
<br><p style="margin-left: 40px;">Note: Mono uses GLib to load libraries, and GLib has a bug on Mac OS X where it doesn&#39;t use a <tt>.dylib</tt> extension, but instead uses the Unix <tt>.so</tt> extension.  While this should eventually be fixed, the current workaround is to <a href="http://www.mono-project.com/Interop_with_Native_Libraries#Library_Names" title="">write a <tt>.config</tt> file which maps to the <tt>.dylib</tt> file</a>, e.g.
</p>
<pre style="margin-left: 40px;"> <span style="color: rgb(0, 153, 0);"><span style="font-weight: bold; color: black;">&lt;configuration<span style="font-weight: bold; color: black;">&gt;</span></span></span><br>   <span style="color: rgb(0, 153, 0);"><span style="font-weight: bold; color: black;">&lt;dllmap</span> <span style="color: rgb(0, 0, 102);">dll</span>=<span style="color: rgb(255, 0, 0);">&quot;mylib&quot;</span> <span style="color: rgb(0, 0, 102);">target</span>=<span style="color: rgb(255, 0, 0);">&quot;mylib.dylib&quot;</span> <span style="font-weight: bold; color: black;">/&gt;</span></span><br>
 <span style="color: rgb(0, 153, 0);"><span style="font-weight: bold; color: black;">&lt;/configuration<span style="font-weight: bold; color: black;">&gt;</span></span></span></pre><br><br>Hope it helps,<br><br><div class="gmail_quote">
