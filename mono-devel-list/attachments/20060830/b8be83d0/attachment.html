<FONT COLOR="#000000">Hello,</FONT>

<FONT COLOR="#000000">    Mono 1.1.17 has been released.</FONT>

<FONT COLOR="#000000">    Full release notes:</FONT>

<FONT COLOR="#000000">        <A HREF="http://www.go-mono.com/archive/1.1.17">www.go-mono.com/archive/1.1.17</A></FONT>

<FONT COLOR="#000000">   Mono was branched at version 1.1.13 to become the stable version of Mono</FONT>
<FONT COLOR="#000000">   that is distributed by Novell on its enterprise products. That series of</FONT>
<FONT COLOR="#000000">   releases are only getting bug fixes.</FONT>

<FONT COLOR="#000000">   Before each release we run all of the regression tests on Mono, so we</FONT>
<FONT COLOR="#000000">   consider this release usable for deployment, but there are still a few</FONT>
<FONT COLOR="#000000">   changes in various areas.</FONT>


<FONT COLOR="#000000">   This release is mostly a bug-fix release, there are very few new</FONT>
<FONT COLOR="#000000">   developments.</FONT>

<FONT COLOR="#000000">                           Changes since Mono 1.1.16</FONT>

<FONT COLOR="#000000">Highlights</FONT>

<FONT COLOR="#000000">   Basic world: The Mono Basic compiler and the Basic runtime have been</FONT>
<FONT COLOR="#000000">   removed from the Mono distribution. A new compiler that is compatible with</FONT>
<FONT COLOR="#000000">   Visual Basic 2005 and a matching runtime are now part of a separate</FONT>
<FONT COLOR="#000000">   distribution. On this particular release, we are offering the basic</FONT>
<FONT COLOR="#000000">   runtime, but the compiler is not able to run completely on Mono yet.</FONT>

<FONT COLOR="#000000">   Windows.Forms: Printing is now supported.</FONT>

<FONT COLOR="#000000">   This release is able to compile and build IronPython 1.0 RC2.</FONT>

<FONT COLOR="#000000">   COM: Basic COM support has been integrated.</FONT>

<FONT COLOR="#000000">  Inotify watcher</FONT>

<FONT COLOR="#000000">   The FileSystem will now use inotify directly on systems that support it</FONT>
<FONT COLOR="#000000">   without having to go through an external library like FAM or Gamin, this</FONT>
<FONT COLOR="#000000">   should make our use of inotify reliable. [Gonzalo Paniagua]</FONT>

<FONT COLOR="#000000">  Async Process Notification</FONT>

<FONT COLOR="#000000">   2.0 support for asynchronous reads and writes from the Process class is</FONT>
<FONT COLOR="#000000">   now supported [Gonzalo].</FONT>

<FONT COLOR="#000000">  Mono Loading as a Shared Library Works Again</FONT>

<FONT COLOR="#000000">   This was a problem that mostly affected the OpenOffice plugin, which</FONT>
<FONT COLOR="#000000">   loaded Mono as a separate process, this is now fixed [Zoltan Varga]</FONT>

<FONT COLOR="#000000">  Gtk# Split</FONT>

<FONT COLOR="#000000">   As part of Gtk# becoming one of the supported language bindings in the</FONT>
<FONT COLOR="#000000">   Gnome platform and Tomboy, a Gtk#-based application, becoming part of the</FONT>
<FONT COLOR="#000000">   Gnome desktop, Gtk# has been split up into multiple packages, instead of a</FONT>
<FONT COLOR="#000000">   single one.</FONT>

<FONT COLOR="#000000">   All the packages are available from our download site [Mike Kestner].</FONT>

<FONT COLOR="#000000">  Mono.Cairo</FONT>

<FONT COLOR="#000000">   Mono.Cairo bindings now supports a DirectFB surface now [Alp Toker].</FONT>

<FONT COLOR="#000000">  System.Drawing</FONT>

<FONT COLOR="#000000">   This release includes an upgraded Cairo stack (from 1.0 to 1.2) and</FONT>
<FONT COLOR="#000000">   allowed us to enable printing in System.Drawing and System.Windows.Forms.</FONT>

<FONT COLOR="#000000">   The original work was done by Jordi Mas, the Cairo upgrade by Peter Bartok</FONT>
<FONT COLOR="#000000">   and the work was completed by Chris Toshok.</FONT>

<FONT COLOR="#000000">  2.0 API updates</FONT>

<FONT COLOR="#000000">   Process now support the async io handling [Gonzalo Paniagua]</FONT>

<FONT COLOR="#000000">   String.Normalize is included [Atsushi Enomoto]</FONT>

<FONT COLOR="#000000">   ADO.NET 2.0 updates, included an implementation for</FONT>
<FONT COLOR="#000000">   SqlConnection.GetSchema (Nagappan, Nagappan).</FONT>

<FONT COLOR="#000000">  Registry</FONT>

<FONT COLOR="#000000">   Updated to the 2.0 API. [Miguel de Icaza]</FONT>

<FONT COLOR="#000000">   Gert added support for splitting the registry across user and system level</FONT>
<FONT COLOR="#000000">   settings. [Gert Driesen]</FONT>

<FONT COLOR="#000000">  mod_mono</FONT>

<FONT COLOR="#000000">   Added support for X.509 client certificates. It's now possible to use</FONT>
<FONT COLOR="#000000">   System.Web.HttpClientCertificate with Apache. Certificate validation can</FONT>
<FONT COLOR="#000000">   be done by Apache, Mono or both (default). [Hubert Fongarnand, Sebastien</FONT>
<FONT COLOR="#000000">   Pouliot]</FONT>

<FONT COLOR="#000000">  Security</FONT>

<FONT COLOR="#000000">   SN now accept password-protected PKCS#12/PFX files to strongname</FONT>
<FONT COLOR="#000000">   assemblies. This feature is enabled in both 1.x and 2.0 profiles</FONT>
<FONT COLOR="#000000">   [Sebastien Pouliot]</FONT>

<FONT COLOR="#000000">  Additions</FONT>

<FONT COLOR="#000000">   CodeDOM JScriptCodeProvider code JavaScript code is now included</FONT>
<FONT COLOR="#000000">   [Akiramei]</FONT>

<FONT COLOR="#000000">   An EventLog implementation is available on both Unix and Windows, to use</FONT>
<FONT COLOR="#000000">   set the MONO_EVENTLOG_TYPE variable like this:</FONT>
<FONT COLOR="#000000">     * local[:path] generates a log file in the given path. If the path is</FONT>
<FONT COLOR="#000000">       not given, it will store the results in &quot;/var/lib/mono/eventlog&quot; on</FONT>
<FONT COLOR="#000000">       Unix and in &quot;%APPDATA%\mono\eventlog&quot; on Windows.</FONT>
<FONT COLOR="#000000">     * win32: This uses the native Windows API to send the log messages to</FONT>
<FONT COLOR="#000000">       the system event log.</FONT>
<FONT COLOR="#000000">     * null: discards all of the events</FONT>

<FONT COLOR="#000000">   to a pathname where the events should be logged to [Atsuhi Enomoto, Gert</FONT>
<FONT COLOR="#000000">   Driesen]</FONT>

<FONT COLOR="#000000">   COM Interop: Basic support for Runtime Callable Wrappers (RCWs). This</FONT>
<FONT COLOR="#000000">   allows users to use unmanaged components from managed code. [Jon Chambers]</FONT>

<FONT COLOR="#000000">   Sqlite now exposes a Version property to detect which underlying database</FONT>
<FONT COLOR="#000000">   is available (2.x or 3.x) [Joshua Tauberer]</FONT>

<FONT COLOR="#000000">   Mono.Posix now features an abstract Unix end point in addition to Unix End</FONT>
<FONT COLOR="#000000">   Points [Alp Toker].</FONT>

<FONT COLOR="#000000">  XML Land</FONT>

<FONT COLOR="#000000">   Fixed XmlSchemaSet and XmlSchemaCollection problem across multiple</FONT>
<FONT COLOR="#000000">   namespaces [Atsushi Enomoto]</FONT>

<FONT COLOR="#000000">  Important Bug Fixes</FONT>

<FONT COLOR="#000000">   Dynamic linking of Mono is now possible in applications that were using</FONT>
<FONT COLOR="#000000">   the TLS (open office) [Zoltan Varga].</FONT>

<FONT COLOR="#000000">   Newly created AppDomains no longer inherit the list of loaded assemblies</FONT>
<FONT COLOR="#000000">   from the main domain. This has an important side-effect, to get XSP and</FONT>
<FONT COLOR="#000000">   mod_mono running, you must install the latest versions of it (released in</FONT>
<FONT COLOR="#000000">   this iteration), older versions will not work [Lluis Sanchez].</FONT>

<FONT COLOR="#000000">   A number of missing pieces of System.IO.Ports have been implemented</FONT>
<FONT COLOR="#000000">   (ReadChar, ReadLine, BytesToRead, BytesToWrite, ReadTo, return USB tty</FONT>
<FONT COLOR="#000000">   devices) [Miguel de Icaza].</FONT>

<FONT COLOR="#000000">   ASP.NET Cache will check dependencies now (79002) [Gonzalo Paniagua]</FONT>

<FONT COLOR="#000000">  Alpha Port</FONT>

<FONT COLOR="#000000">   The beginning of a port to Linux/Alpha has been contributed to Mono,</FONT>
<FONT COLOR="#000000">   [Sergey Tikhonov]</FONT>

<FONT COLOR="#000000">  Cilc</FONT>

<FONT COLOR="#000000">   Cilc, a tool to generate C wrappers for managed routines has been</FONT>
<FONT COLOR="#000000">   relicensed from the GPL to the MIT X11 [Alp Toker]</FONT>

<FONT COLOR="#000000">  Postgress</FONT>

<FONT COLOR="#000000">   Updated the Posgress data bindings to RC3 [Francisco Figueiredo].</FONT>

<FONT COLOR="#000000">                             Installing Mono 1.1.17</FONT>

<FONT COLOR="#000000">        </FONT>
<FONT COLOR="#000000">         $ ./configure --prefix=/devel</FONT>
<FONT COLOR="#000000">        </FONT>

<FONT COLOR="#000000">   You can then setup your PATH to include /devel/bin to access the Mono 1.1.</FONT>
<FONT COLOR="#000000">   Alternatively you can replace your Mono installation with 1.1.17</FONT>

<FONT COLOR="#000000">   Binary Packages and Source Code Downloads:</FONT>

<FONT COLOR="#000000">     Source code and pre-compiled packages for SUSE, SLES, Fedora Core 3, 4,</FONT>
<FONT COLOR="#000000">     RHEL, MacOS and Windows in a variety of platforms available from our web</FONT>
<FONT COLOR="#000000">     site from the download section.</FONT>

<FONT COLOR="#000000">   Quick source code installation:</FONT>

<FONT COLOR="#000000">     If we have no packages for your platform, installing from source code is</FONT>
<FONT COLOR="#000000">     very simple.</FONT>

<FONT COLOR="#000000">     mono:</FONT>

<FONT COLOR="#000000">        </FONT>
<FONT COLOR="#000000">     $ tar xzf mono-1.1.17.tar.gz</FONT>
<FONT COLOR="#000000">     $ cd mono-1.1.17</FONT>
<FONT COLOR="#000000">     $ ./configure</FONT>
<FONT COLOR="#000000">     $ make</FONT>
<FONT COLOR="#000000">     $ make install</FONT>


<FONT COLOR="#000000">     Then compile libgdiplus:</FONT>

<FONT COLOR="#000000">        </FONT>
<FONT COLOR="#000000">     $ tar xzf libgdiplus-1.1.17.tar.gz</FONT>
<FONT COLOR="#000000">     $ cd libgdiplus-1.1.17</FONT>
<FONT COLOR="#000000">     $ ./configure</FONT>
<FONT COLOR="#000000">     $ make</FONT>
<FONT COLOR="#000000">     $ make install</FONT>
