If it is on machines that you control, what you need is not<br>
necessarily mono in a parallel environment, because you only need Mono<br>
once.<br>
I guess if you have your own build packages, you won't need the<br>
Xamarin Repo. So you could just use the src rpms from Xamarin (see<br>
<a href="http://download.mono-project.com/repo/centos/m/mono-core/" rel="noreferrer" target="_blank">http://download.mono-project.<wbr>com/repo/centos/m/mono-core/</a>, look for<br>
src.rpm) at a point where you want it, and create your own repo with<br>
that version, and only install from your own repo.<br>
<br>

disable it after installation. (enabled=0 in the /etc/yum.repos.d/<br>
xamarin repo file). Only when you have verified on a test machine,<br>
that a newer version from the Xamarin repo would not break your<br>
application, you enable the repo file (enabled=1), and upgrade.<br>
