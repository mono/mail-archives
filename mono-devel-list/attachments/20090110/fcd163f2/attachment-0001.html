<p class=MsoNormal>Hi all, I&#8217;m working on getting Mono running under
Android. So far, I&#8217;ve been able to get things running with next to no
code changes (by way of configure, compiler and linker options). <o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>I did run into one issue though that required a code change.
Mono has a dependency on libc.so. On Android, Google implemented a custom
lightweight version of libc.so, which they call &#8220;Bionic&#8221;. I have
tried building Mono against Bionic, but at the moment, it seems basically
impossible because Bionic does not have Unicode support. (I will dig into other
solutions for this further, but it&#8217;s low priority at the moment&#8230; I
want to get things running <span style='font-family:Wingdings'>J</span>)<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>So, as a result, Mono on Android needs to be packaged with
the standard ARM/Linux version of libc. &nbsp;However, there are significant
differences between the Linux file system and Android file system. So when Mono&#8217;s
uses libc&#8217;s getpwuid and getpwuid_r, it fails, and causes further
problems in the runtime. Linux&#8217;s getpwuid works by examining various
files in /etc. But, Android does not have a /etc. Android has a uid/gid repository
that is very different from typical Linux distributions.<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

getpwuid (there is no getpwuid_r):<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>getpwuid is used in three locations Mono. The two troublesome
ones are in mono/metadata/security.c in the functions &#8220;GetTokenName&#8221;
and &#8220;IsMemberOf&#8221;. (The third is in pwd.c for Mono&#8217;s Posix syscall
wrapper, but that is not causing the problem. I&#8217;ll fix that later.)<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>I have attached a patch file that I have successfully built
and tested against Android.&nbsp; It is also regression safe, as all the new
code is inside a #ifdef PLATFORM_ANDROID block. The workaround is simple, and
there are comments explaining how the Android uid model works.<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>Alternatively, I can obviously compile a custom version of
libc.so for Mono, but I do not think that is a good solution. Mono is already
using #ifdef PLATFORM_WIN32 at the same points in code that I placed my
PLATFORM_ANDROID, and I&#8217;m assuming that the Mono developers will be more
open to my proposed changes.<o:p></o:p></p>
