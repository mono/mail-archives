This patch almost complete the SecureString implementation. The missing pieces are:<br>
- the new Marshal.SecureStringTo... methods; and<br>
- that, AFAIK, the runtime doesn't support CriticalFinalizerObject<br>
<br>
SecureString is implemented on top of ProtectedMemory. On Windows Data
Protection API (DPAPI) is used to implement ProtectedMemory while on
Linux (and other platforms) I added AES support into the runtime (so we
get partial ProtectedMemory support).<br>
<br>
The AES implementation used (too large to be part of the patch) can be found at:<br>
<a href="http://www.mirrors.wiretapped.net/security/cryptography/algorithms/aes/aes-c-rijmen/">http://www.mirrors.wiretapped.net/security/cryptography/algorithms/aes/aes-c-rijmen/</a><br>
<br>
P.S. Support for some of the new Marshal methods will be added later.<br>
