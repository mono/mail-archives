<p class=MsoNormal style='margin-bottom:12.0pt'>Hi all,<br>
<br>
I have a custom web control written in C# that I develop using MonoDevelop
(trunk) and release for use under both Mono (trunk) and .NET.&nbsp; Under some
circumstances it needs to be installed in the GAC so I sign it.&nbsp; It works
fine when referenced from a C# or <a href="http://VB.NET">VB.NET</a> Web Site
Project and from a C# Web Application Project, but does not work when
referenced from a <a href="http://VB.NET">VB.NET</a> Web Application Project.
The error I get when loading the page with my control on it is:<br>
<br>
<b>Compiler Error Message: </b>BC30652: Reference required to assembly
'Brettle.Web.NeatUpload, Version=1.3.2469.19122, Culture=neutral,
PublicKeyToken=c95290d92c5893c8' containing the type
'Brettle.Web.NeatUpload.MultiFile'. Add one to your project.<br>
<br>
The reference is definitely there and if I replace the reference with either a
reference to an unsigned assembly or with a signed assembly built from the same
source using VS 2008 Express SP1, the error goes away.&nbsp; To sum up, <i>all</i>
of the following need to be true for the error to occur:<br>
<br>
1. <a href="http://VB.NET">VB.NET</a> Web Application project, not c# Web
Application project or <a href="http://VB.NET">VB.NET</a> Web Site Project.<br>
2. Assembly is signed.<br>
3. Assembly is built/signed using Mono, not .NET.<br>
<br>
Also, if I use .NET's sn.exe (with the -vf option) to verify the signature on
the assembly built/signed with Mono, it says it is valid.<br>
<br>
Has anyone else run into this or doese anyone have any suggestions for ways to
diagnose/fix/workaround the problem?<br>
<br>
Thanks,<br>
--Dean<br>
