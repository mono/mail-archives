<div class="">We’ve recently added initial support for building Doxygen documentation[1]. The master docs are currently available on Jenkins[2]. We intend to set up a central location online where these docs are deployed, as a convenient way to browse the runtime code.
 My hope is that this helps new developers and people unfamiliar with different parts of the runtime.</div>
<div class=""><br class="">
</div>
<div class="">But first, many comments need to be updated to use Doxygen syntax in order to produce useful docs. It wouldn’t be very productive for me to do all of this myself, so I propose that when we change some code, we make sure that if the code is documented,
 then its documentation appears correctly in the Doxygen output.</div>
<div class=""><br class="">
</div>
<div class="">I am <i class="">not</i> proposing that we write new documentation at this time, only verify the docs we already have and get them ready for deployment.</div>
<div class=""><br class="">
</div>
<div class="">You can also build the docs locally with “make doxygen -C docs” in the Mono repository, then open “docs/doxygen-output/index.html” to view the results. (This can take several minutes.)</div>
