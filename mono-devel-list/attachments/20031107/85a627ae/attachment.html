<DIV>The macros word0() and word1() violate the ANSI C rule about accessing an object using a type other than what it was declared with. If compiled -O2 with gcc3.3, strtod returns incorrect results (Single.MaxValue for example)</DIV>
<DIV>&nbsp;</DIV>
<DIV>The solution is to either compile mono with -fno-strict-aliasing or to fix the word0() word1() macros by wrapping the double in a union.</DIV>
