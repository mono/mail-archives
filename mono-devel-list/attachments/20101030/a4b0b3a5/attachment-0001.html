Attached is another set of updates for Mac Mono Winforms. My .net<br>
applications now work quite nicely with my changes to Mono.<br>
<br>
NativeWindow has been simplified to using Hwnd to provide the<br>
Handle-Window lookup. I have found that storing handles in Hashtable<br>
sometimes results in the handle being lost with the result being<br>
zombie windows. Replacing Hashtable with a linked list solved the<br>
problem. Placing the most recently used link at the head of the linked<br>
list maintains efficiency.<br>
<br>
MS Windows places paint events at the lowest priority with all other<br>
events preempting WM_PAINT. A separate queue (linked list in my case)<br>
for paint events is required for Mono to duplicate this effect.<br>
<br>
I find the exposure of method variables to outside objects without<br>
using properties to be one of the poorer coding practices used in<br>
Mono. Apart from that, I found Mono to be quite well written and a<br>
pleasure to work on.<br>
