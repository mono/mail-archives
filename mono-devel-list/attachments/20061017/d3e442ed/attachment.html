The icall for ves_icall_System_Diagnostics_DefaultTraceListener_WriteWindowsDebugString was calling the OutputDebugString function in kernel32 via loading the module and getting the symbol. The function pointer should have been declared as STDCALL. But, I couldn't see a reason why we couldn't just call OutputDebugString directly, since it's on all windows platforms mono supports so I just made that change instead.
