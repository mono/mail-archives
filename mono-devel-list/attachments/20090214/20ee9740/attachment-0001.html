<div dir="ltr">Hi Lucas,<br><br>This depends on the shell you are using.&nbsp; If you are using /bin/bash or other <br>/bin/sh derivative the instructions are correct.<br>Call the script directly as:<br>/tmp/build_deps/env.sh<br>
to run the commands in a sub-shell, or <br>. /tmp/build_deps/env.sh<br>to run the commands in the current shell (notice the &#39;.&#39; in the beginning of the line)<br><br>If you are using /bin/tcsh or other /bin/csh derivative, the &#39;source&#39; command is used<br>
to execute commands in the current shell.&nbsp; This is the same behavior as the &#39;.&#39; command<br>in /bin/bash.<br>In this case, I believe the &#39;source&#39; command is wrong because the commands in the mentioned<br>file are /bin/bash or /bin/sh commands.<br>
