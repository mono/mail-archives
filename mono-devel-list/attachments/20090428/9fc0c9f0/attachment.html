The problem is likely your <TT>$PATH</TT> -- Mono was installed with <TT>/usr/local/</TT> as the installation prefix, thus there <I>should</I> be a <TT>/usr/local/bin/mcs</TT> script, but <TT>/usr/local/bin</TT> isn't within <TT>PATH</TT>.<BR>
<BR>
Try doing this in your shell:
<BLOCKQUOTE>
<PRE>
export PATH=/usr/local/bin:$PATH
mcs --help
</PRE>
</BLOCKQUOTE>

</BLOCKQUOTE>
Running apps from the command line is ~completely different from running ASP.NET apps.<BR>
<BR>
To run an ASP.NET app, you need to do one of two things:<BR>
<BR>
1. <TT>cd path/to/your/asp.net/app</TT>, then run <TT>xsp2</TT>.&nbsp; This will start a local web server listening to <A HREF="http://localhost:8080">http://localhost:8080.</A><BR>
<BR>
2. Configure Apache and mod_mono: <A HREF="http://mono-project.com/Mod_mono">http://mono-project.com/Mod_mono</A> (and likely lots of other Google-able sources).<BR>
<BR>
Finally, for command-line apps (<I>not</I> ASP.NET apps), you should read over: <A HREF="http://www.mono-project.com/Guidelines:Application_Deployment">http://www.mono-project.com/Guidelines:Application_Deployment</A>
