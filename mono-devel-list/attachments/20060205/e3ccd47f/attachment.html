I am merging two DataTables based on the key.&nbsp; The resulting table's schema
is a superset of the columns from each table.&nbsp; On windows I am building
using .NET 2003 and the 1.1 framework.&nbsp; On Mono I was originally using
1.1.10 and have just switched to 1.1.13 on a Linux Redhat 9
machine.&nbsp;&nbsp; This problem did not occur in Mono 1.1.10, but showed up
immediately after switching to 1.1.13.<br>
<br>
One of the two tables is a result of a SELECT using the MySql .NET connector
(latest).&nbsp; The other is constructed from data mining, but has a subset of
the columns of the SQL query.&nbsp; Under Mono 1.1.13 I suddenly get an
exception that one of the merged lines (an added line) cannot have a specific

data, so it gets set to the default (which is null).<br>
<br>

Mono 1.1.13 part of it is set.<br>
It appears that Mono is now correctly getting the flag that a column cannot be
NULL from the SQL schema, but is not correctly getting the default value.&nbsp;
Since I am building the MySQL .NET connector from source on both targets, this
is an issue under Mono, not an issue within the MySQL code I think.<br>
