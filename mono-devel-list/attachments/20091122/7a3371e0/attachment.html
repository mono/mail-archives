<p class=MsoNormal><span lang=EN-US>This patch adds better support for mapping
of ASP.NET virtual paths on the XSP FastCGI Backend (previously it primarily
assumed mapping to physical files and directories on the file system).&nbsp; As
a result, MVC now also seems to be working through FastCGI, which extensively
uses URL remapping.<o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal><span lang=EN-US>The code includes a few workarounds for an
issue I ran into with Mono&#8217;s HostingEnvironment.MapPath implementation
(comments included), but that can be addressed separately (and the workarounds reworked
later).&nbsp; For now, this patch does at least get things off the ground and it
seems to work as expected.<o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal><span lang=EN-US>There are two attachments: one for the
trunk (&#8220;trunk/xsp/src&#8221;) and one for the main branches (&#8220;branches/mono-2-4/xsp/src&#8221;
and &#8220;branches/mono-2-6/xsp/src&#8221;).&nbsp; Please commit.<o:p></o:p></span></p>
