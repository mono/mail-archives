I&#39;m trying to read packages that were created with System.IO.Packaging on a<br>
Windows machine using .NET.<br>
<br>
It seems that that there is a problem with PackageParts that contain spaces<br>
in the specified package URI.<br>
<br>
Using the following code produces different results on .NET / Mono:<br>
<br>
string fileName = &quot;Test.zip&quot;;<br>
Uri uri = PackUriHelper.CreatePartUri(new Uri(&quot;/My Folder/Test.txt&quot;,<br>
UriKind.Relative));<br>
<br>
using (Package package = Package.Open(fileName, FileMode.Create))<br>
{<br>
    PackagePart part = package.CreatePart(uri,<br>
System.Net.Mime.MediaTypeNames.Text.Plain);<br>
    using (StreamWriter sw = new StreamWriter(part.GetStream()))<br>
    {<br>
        sw.WriteLine(&quot;Hello World&quot;);<br>
    }<br>
}<br>
<br>
On Linux using Mono the ZIP file contains a folder &quot;My Folder&quot;, but with<br>
Windows and .NET the name of the folder is escaped (&quot;My%20Folder&quot;).<br>
