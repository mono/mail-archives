<br>
Here the Debian NEWS teaser of Mono 2.4:<br>
<br>
mono (2.4-1) unstable; urgency=low<br>
<br>
  * Mono 2.4 ships a lot of goodies:<br>
    + SIMD support in Mono.<br>
    + No expensive polls in the threading implementation any longer.<br>
    + Speed-up garbarge collection on multi-core systems.<br>
    + Optimized XPath (using 15% less memory).<br>
    + Faster DateTime.TryParse implementation (by not using try/catch).<br>
    + Support for precompiled <a href="http://ASP.NET" target="_blank">ASP.NET</a> pages.<br>
  * Complete Announcement of Mono 2.4 (vs Mono 2.2):<br>
    <a href="http://www.mono-project.com/Release_Notes_Mono_2.4" target="_blank">http://www.mono-project.com/Release_Notes_Mono_2.4</a><br>
<br>
  * Mono 2.2 also ships with lovely things:<br>
    + The tree-based code generation engine in the JIT was replaced<br>
    with the<br>
      Linear IL engine, which allows better optimizations.<br>
    + Generic sharing is now used in all cases (reduces memory usage).<br>
    + Generic sharing is now supported on ARM.<br>
    + Support of full Ahead of Time Compilation.<br>
    + New PerformanceCounters implementation that allows to monitor the<br>
      runtime internals. Including the GUI tool &quot;mperfmon&quot; to view them.<br>
    + Interactive shell called &quot;csharp&quot; that allows to execute C#<br>
      expressions inside a shell. There is also a GUI version available<br>
      called &quot;gsharp&quot;.<br>
    + Live Inspection. You can attach using the csharp shell to a<br>
      running process and run code inside to debug things.<br>
    + The C# compiler will now optimize empty strings (&quot;&quot;) away with<br>
      String.Empty (which reduces memory usage).<br>
    + The Regular Expressions engine has been rewritten being more<br>
      efficient.<br>
      At the same time compiled regex are also supported now.<br>
    + <a href="http://ASP.NET" target="_blank">ASP.NET</a> supports now new routing handlers needed by <a href="http://ASP.NET" target="_blank">ASP.NET</a> MVC.<br>
    + In the WinForms implementation were almost 200 bugs fixed (since<br>
    Mono 2.0)<br>
  * Complete Announcement of Mono 2.2 (vs Mono 2.0):<br>
    <a href="http://www.mono-project.com/Release_Notes_Mono_2.2" target="_blank">http://www.mono-project.com/Release_Notes_Mono_2.2</a><br>
<br>
  * Mono 2.0 shipped with:<br>
    + A console debugger (mdb) part of the mono-debugger package.<br>
    + WinForms 2.0 API is complete.<br>
    + WinForms&#39;s WebBrowser was implemented using Mozilla&#39;s Gecko HTML<br>
      rendering engine.<br>
    + WinForms now supports international keyboard input through XIM.<br>
    + Performance of locking (used by threading) was significally<br>
    improved.<br>
    + New debug parameter &quot;--debug=cast&quot; which print outs the types<br>
      in forInvalidCastException.<br>
    + The C# compiler supports now expression trees (for LINQ) which<br>
      completes the C# 3.0 support.<br>
    + The C# compiler is now dual-licensed under MIT/X11 and GPLv2<br>
    (only).<br>
    + LINQ and LINQ to XML are now complete.<br>
    + Big Arrays for 64bit architectures are now support (but have to be<br>
      compiled using the --enable-big-arrays configure switch)<br>
  * Complete Announcement of Mono 2.0:<br>
    <a href="http://www.mono-project.com/Release_Notes_Mono_2.0" target="_blank">http://www.mono-project.com/Release_Notes_Mono_2.0</a><br>
