<p class="MsoNormal"><span style="font-size:11.0pt">What this is: The Mono team has a CI (continuous integration) system which builds and runs automated tests on every commit checked in to git (specifically the master branch). We have a </span><a href="https://jenkins.mono-project.com/view/All/job/jenkins-testresult-viewer/Test_Result_View/"><span style="font-size:11.0pt">test
 log viewer</span></a><span style="font-size:11.0pt"> on Jenkins that tracks the results (currently only accessible to github project admins, sorry). Once a week I sweep through and write an email with a list of the most frequently-failing automated tests.</span><o:p></o:p></p>
<p class="MsoNormal"><span style="font-size:11.0pt"> </span><o:p></o:p></p>
<p class="MsoNormal"><span style="font-size:11.0pt">A lot of our new bugs from last week have been handled since the last weather report, I think we are making progress. However there was a lot of churn this week so I didn't drill down very hard on the less
 frequent bugs. The new bugs this week are #2, #4, and #5 (and #1, in the sense that #1 is an old bug exhibiting new behavior). They all seem to be threading-related. There has been a lot of refactoring on threads the last couple weeks so maybe this is expected.<o:p></o:p></span></p>
