<div dir="ltr">I just wanted to thank the Mono team for the awesome work that's been done over the last few years with regards to <a href="http://ASP.NET">ASP.NET</a> MVC compatibility. I tried out <a href="http://ASP.NET">ASP.NET</a> MVC 2 a few years ago (with Mono 2.8 or 2.10 I think) and there were lots of small issues with getting it running, lots of edge cases that broke things, and things that Mono did subtly different compared to Microsoft .NET. It didn't feel like something that could be used in production at all.<div>
<br></div><div>I recently decided to try it out again, rewriting my blog in C# using <a href="http://ASP.NET">ASP.NET</a> MVC 4 and running it on Mono 3. I used the latest version of all these libraries:<div><ul style="font-family:arial,sans-serif;font-size:13px">
<li style="margin-left:15px"><a href="http://asp.net/" target="_blank">ASP.NET</a> MVC 4 with Razor 2 and its built-in CSS and JS minification (WebGrease)</li><li style="margin-left:15px"><a href="https://github.com/ServiceStack/ServiceStack.OrmLite" target="_blank">ServiceStack OrmLite</a> with MySql.Data for the data access layer</li>
<li style="margin-left:15px"><a href="http://simpleinjector.codeplex.com/" target="_blank"><span class="">SimpleInjector</span></a> for IoC / Dependency injection</li><li style="margin-left:15px"><a href="http://www.dotlesscss.org/" target="_blank">DotLESS</a> for CSS preprocessing</li>
<li style="margin-left:15px"><a href="http://code.google.com/p/elmah/" target="_blank">ELMAH</a> for error logging</li></ul><div style><font face="arial, sans-serif">It turns out that all of these work fine with Mono 3.0.2, and the site is running just as well as it does on Microsoft .NET. </font><span style="font-family:arial,sans-serif">The live site is at <a href="http://dan.cx/">http://dan.cx/</a> and <a href="http://dan.cx/blog">http://dan.cx/blog</a>, and the source code is available on Github (<a href="https://github.com/Daniel15/Website">https://github.com/Daniel15/Website</a>) </span></div>
