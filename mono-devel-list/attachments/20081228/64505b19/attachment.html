Hi all,<br><br>Marek Habersack emailed me about problems putting machine level browser files in the App_Browsers folder to workaround limitations in Mono&#39;s default machine level browser file (Compat.browser) .&nbsp; It turns out that putting replacement machine level browser files in App_Browsers is not supported under .NET either, so allowing app developers to do that under Mono would only encourage creation of non-portable apps.&nbsp; Instead, the attached patch allows app developers to place replacement machine-level browser files in App_Data/Mono_Machine_Browsers/.&nbsp; If any browser files are found in that location, they are used as the machine-level browser files and the default machine-level browser files are ignored.&nbsp; I chose to use a location under App_Data so that the browser files would be hidden under both Mono and .NET.<br>
<br>The patch also fixes a compatibility bug where Mono required an &lt;identification&gt; section in each browser def, while .NET did not.<br><br>Comments?&nbsp; OK to commit to mono-2-2 and trunk?<br><br>--Dean<br><br>
