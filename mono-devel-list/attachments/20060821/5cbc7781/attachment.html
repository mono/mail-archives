<FONT COLOR="#000000">&gt; Hi all,</FONT>
<FONT COLOR="#000000">&gt; </FONT>
<FONT COLOR="#000000">&gt; I want to report on my recent experiences getting the C# port of  </FONT>
<FONT COLOR="#000000">&gt; BouncyCastle Crypto API (<A HREF="http://www.bouncycastle.org/csharp/">http://www.bouncycastle.org/csharp/</A>) running on </FONT>
<FONT COLOR="#000000">&gt; Mono. It will be a reasonably brief report as it turned out to be mostly </FONT>
<FONT COLOR="#000000">&gt; painless :).</FONT>
<FONT COLOR="#000000">&gt; </FONT>
<FONT COLOR="#000000">&gt; (BC is written entirely in C# for .NET 1.1 by the way).</FONT>

<FONT COLOR="#000000">BouncyCastle is a great cryptographic library. However it's C# port</FONT>
<FONT COLOR="#000000">doesn't use the cryptographic base classes of the FX so it somewhat</FONT>
<FONT COLOR="#000000">limits it's usefulness (e.g. yet another API to learn and difficult</FONT>
<FONT COLOR="#000000">interop with existing code).</FONT>

<FONT COLOR="#000000">&gt; Here's what I did:</FONT>
<FONT COLOR="#000000">&gt; - Installed Mono 1.1.13.6  and MonoDevelop 0.10 via the Ubuntu package </FONT>
<FONT COLOR="#000000">&gt; manager.</FONT>
<FONT COLOR="#000000">&gt; - Imported the existing VS2003 solution file in MonoDevelop, and built </FONT>
<FONT COLOR="#000000">&gt; everything, with no significant issues.</FONT>
<FONT COLOR="#000000">&gt; - The existing NAnt build file also seemed to work without modification </FONT>
<FONT COLOR="#000000">&gt; (although I am having a little trouble running the nunit tests via nant).</FONT>
<FONT COLOR="#000000">&gt; - Ran our regression tests revealing only one failure, which I tracked </FONT>
<FONT COLOR="#000000">&gt; down to a bug in the x86 runtime (Bug# 79087), fixed in svn.</FONT>
<FONT COLOR="#000000">&gt; </FONT>
<FONT COLOR="#000000">&gt; ...and that's all there was to it.</FONT>
<FONT COLOR="#000000">&gt; </FONT>
<FONT COLOR="#000000">&gt; I certainly expected it to be more painful; that it wasn't is to the </FONT>
<FONT COLOR="#000000">&gt; credit of the Mono developers.</FONT>
<FONT COLOR="#000000">&gt; </FONT>
<FONT COLOR="#000000">&gt; The regression tests take a few minutes to run, allowing a (very) </FONT>
<FONT COLOR="#000000">&gt; ballpark performance observation that Mono is about twice as slow as </FONT>
<FONT COLOR="#000000">&gt; MS.NET for running these tests. I guess anyone concerned with </FONT>
<FONT COLOR="#000000">&gt; performance of Mono might find some or all of these tests a useful </FONT>
<FONT COLOR="#000000">&gt; benchmark.</FONT>

<FONT COLOR="#000000">The results are similar to running our cryptographic code on top of the</FONT>
<FONT COLOR="#000000">MS runtime, i.e. they get twice as fast.</FONT>

<FONT COLOR="#000000">&gt; I would be interested in hearing from people who are able to test </FONT>
<FONT COLOR="#000000">&gt; BC/Mono on other architectures than x86. The crypto algorithms do a lot </FONT>
<FONT COLOR="#000000">&gt; of bit/byte twiddling, and could potentially shake out similar problems </FONT>
<FONT COLOR="#000000">&gt; in other runtimes to the bug above.</FONT>

<FONT COLOR="#000000">You can get an approximate idea of the time required by other</FONT>
<FONT COLOR="#000000">architectures when looking at monobuild, compare the unit tests</FONT>
<FONT COLOR="#000000">execution time with the runtime build time. However the results will be</FONT>
<FONT COLOR="#000000">more &quot;general&quot; (and better reflect real-life scenarios) than CPU-bound</FONT>
<FONT COLOR="#000000">cryptographic tests.</FONT>
