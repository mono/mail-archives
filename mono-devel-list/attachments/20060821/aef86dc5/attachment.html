<FONT COLOR="#000000">You're right that the framework isn't easy to use, even for the most</FONT>
<FONT COLOR="#000000">simple things. Having a wrapper class for some common scenarios, like</FONT>

<FONT COLOR="#000000">I had a very quick look at your library. It looks mostly correct except</FONT>
<FONT COLOR="#000000">for the following:</FONT>


<FONT COLOR="#000000">First, you're not using the power of the Fx design to help you. </FONT>

<FONT COLOR="#000000">E.g. you should be able to have a single class accepting a</FONT>
<FONT COLOR="#000000">SymmetricAlgorithm instance (or, with a different ctor, an</FONT>
<FONT COLOR="#000000">AsymmetricAlgorithm and/or an X509Certificate ...) and hide all</FONT>
<FONT COLOR="#000000">difference inside your code.</FONT>

<FONT COLOR="#000000">This would reduce code duplication and make any (existing or future)</FONT>
<FONT COLOR="#000000">cryptographic algorithm usable by your library (without any more</FONT>
<FONT COLOR="#000000">work/code).</FONT>


<FONT COLOR="#000000">Second, you should also take care about non-portable API.</FONT>

<FONT COLOR="#000000">E.g. des.Key = pdb.CryptDeriveKey(&quot;RC2&quot;, &quot;MD5&quot;, 128, /*Add a little salt</FONT>
<FONT COLOR="#000000">*/ new byte[] {0x49, 0x76, 0x61, 0x6e, 0x20, 0x4d, 0x65, 0x64});</FONT>

<FONT COLOR="#000000">In this case this is not even 100% portable within Windows - because the</FONT>
<FONT COLOR="#000000">code will call the default CSP, which can be different from machine to</FONT>
<FONT COLOR="#000000">machine (and implement a different key derivation algorithm).</FONT>

<FONT COLOR="#000000">Note: Of course using a static salt isn't a good idea.</FONT>


<FONT COLOR="#000000">Finally (well from my quick look ;-) you should use &quot;using&quot; for</FONT>
<FONT COLOR="#000000">IDispose-able types. This will ensure that all the critical data will be</FONT>
<FONT COLOR="#000000">zeroize as soon as possible. This is &quot;less&quot; a problem when you're</FONT>
<FONT COLOR="#000000">dealing with strings (because of the way the VM deals with them) but a</FONT>
<FONT COLOR="#000000">good idea nevertheless (e.g. if you want to deal with private keys</FONT>
<FONT COLOR="#000000">later).</FONT>
