<p class="MsoNormal">Hi All,<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">I’m seeing a rare hang in Socket.ReceiveFrom() where the receive timeout isn’t respected. What’s also odd is then on the server side once this wedge occurs the server gets the same packet that had been sent to it over and over and it keeps
 responding to it endlessly - but the client never returns from ReceiveFrom(). This is in a test sending packets over localhost between two threads.<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">Then, if I switch to use Socket.Connect(), Socket.Send(), and Socket.Receive() instead the problem doesn’t occur. Appears to only get me if I use the SendTo/ReceiveFrom variants, which I gather are more commonly used for UDP sockets.<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">I filed a bug with a test app to reproduce it, but wanted to mention it in case anyone else has run into this already or can point out if I’m doing something incorrectly:<o:p></o:p></p>
