Hi,<br><br>Not quite, a sparse file is different to a normal file in how allocations happen. For example if i wanted to write 1 megabyte at an index of 100 megabytes, a seek + write would result in a 101 megabyte file on the disk.
<br><br>With a sparse file, there&#39;d actually be 1 megabyte physically taken up on disk but any program that tries to read that file would see that 1 megabyte existing at an offset of 100 megabytes. (At least that&#39;s my understanding of it).
