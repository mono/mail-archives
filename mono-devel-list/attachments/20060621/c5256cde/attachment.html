I solved it by myself. I post the solution in order to help other people
with the same problem.<br>
I think that the problem is that mod_mono (rh 1.1.14) (doesn't use the
path you place in /etc/mono/config file but always search the file inside
default directories (in fact specifying the full path was not a solution
for me, a bug?).<br>
This implies that you have to correctly setup the LD_LIBRARY_PATH
according to your Oracle Instant Client installation.<br>
If you do that in profile configuration and you manually launch xsp
(1.1.15)(or mod-mono-server I think) always goes well.<br>
The problem is that apache (httpd for RH) doesn't use that configuration,
you have to manually add it to apache initialization files (I placed it
into /etc/init.d/httpd) a command like <br><br>
export LD_LIBRARY_PATH=&lt;oracle client path&gt;:$LD_LIBRARY_PATH<br>
export TNS_ADMIN=&lt;oracle client path&gt;<br><br>
In this way when apache load the user configuration adds your oracle
Instant Client installation to the path and mod_mono finds the
libraries.<br><br>
Maybe someone with more experience in linux, apache and mod_mono can
explain better why this happens, and why specifying full path in mono
config doesn't work but using this method you can solve your oci missing
error.<br>
Hope this helps.<br><br>
