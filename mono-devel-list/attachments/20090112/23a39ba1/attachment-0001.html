<p>Hi all, I'm working on getting Mono running under
Android. So far, I've been able to get things running with next to no
code changes (by way of configure, compiler and linker options). </p>

<p>&nbsp;</p>

<p>I did run into one issue though that required a code change.
Mono has a dependency on libc.so. On Android, Google implemented a custom
lightweight version of libc.so, which they call "Bionic". I have
tried building Mono against Bionic, but at the moment, it seems basically
impossible because Bionic does not have Unicode support. (I will dig into other
solutions for this further, but it's low priority at the moment… I
want to get things running <span style="font-family: Wingdings;">J</span>)</p>

<p>&nbsp;</p>

<p>So, as a result, Mono on Android needs to be packaged with
the standard ARM/Linux version of libc. &nbsp;However, there are significant
differences between the Linux file system and Android file system. So when Mono's
uses libc's getpwuid and getpwuid_r, it fails, and causes further
problems in the runtime. Linux's getpwuid works by examining various
files in /etc. But, Android does not have a /etc. Android has a uid/gid repository
that is very different from typical Linux distributions.</p>

<p>&nbsp;</p>

<p>For reference, here is Bionic's implementation of
getpwuid (there is no getpwuid_r):</p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;; color: blue;">struct</span><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;"> passwd* getpwuid(uid_t uid)</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">{</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;&nbsp;&nbsp; stubs_state_t*&nbsp; state =
__stubs_state();</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;&nbsp;&nbsp; <span style="color: blue;">struct</span>
passwd*&nbsp; pw;</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;&nbsp;&nbsp; <span style="color: blue;">if</span>
(state == NULL)</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; <span style="color: blue;">return</span> NULL;</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;&nbsp;&nbsp; pw = &amp;state-&gt;passwd;</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;&nbsp;&nbsp; <span style="color: blue;">if</span>
( android_id_to_passwd(pw, uid) != NULL )</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; <span style="color: blue;">return</span> pw;</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;&nbsp;&nbsp; <span style="color: blue;">if</span>
(uid &lt; AID_APP) {</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; errno =
ENOENT;</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; <span style="color: blue;">return</span> NULL;</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;&nbsp;&nbsp; }</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;&nbsp;&nbsp; snprintf(
state-&gt;app_name_buffer, <span style="color: blue;">sizeof</span>
state-&gt;app_name_buffer,</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
<span style="color: rgb(163, 21, 21);">&quot;app_%d&quot;</span>, uid - AID_APP );</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;&nbsp;&nbsp; pw-&gt;pw_name&nbsp; =
state-&gt;app_name_buffer;</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;&nbsp;&nbsp; pw-&gt;pw_dir&nbsp;&nbsp; = <span style="color: rgb(163, 21, 21);">&quot;/data&quot;</span>;</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;&nbsp;&nbsp; pw-&gt;pw_shell = <span style="color: rgb(163, 21, 21);">&quot;/system/bin/sh&quot;</span>;</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;&nbsp;&nbsp; pw-&gt;pw_uid&nbsp;&nbsp; = uid;</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;&nbsp;&nbsp; pw-&gt;pw_gid&nbsp;&nbsp; = uid;</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;</span></p>

<p style=""><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">&nbsp;&nbsp;&nbsp; <span style="color: blue;">return</span>
pw;</span></p>

<p><span style="font-size: 10pt; font-family: &quot;Courier New&quot;;">}</span></p>

<p>&nbsp;</p>

<p>getpwuid is used in three locations Mono. The two troublesome
ones are in mono/metadata/security.c in the functions "GetTokenName"
and "IsMemberOf". (The third is in pwd.c for Mono's Posix syscall
wrapper, but that is not causing the problem. I'll fix that later.)</p>

<p>&nbsp;</p>

<p>I have attached a patch file that I have successfully built
and tested against Android.&nbsp; It is also regression safe, as all the new
code is inside a #ifdef PLATFORM_ANDROID block. The workaround is simple, and
there are comments explaining how the Android uid model works.</p>

<p>&nbsp;</p>

<p>Alternatively, I can obviously compile a custom version of
libc.so for Mono, but I do not think that is a good solution. Mono is already
using #ifdef PLATFORM_WIN32 at the same points in code that I placed my
PLATFORM_ANDROID, and I'm assuming that the Mono developers will be more
open to my proposed changes.</p>
