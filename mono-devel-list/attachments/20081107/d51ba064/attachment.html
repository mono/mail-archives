Hello,<br>&nbsp;&nbsp;&nbsp;&nbsp; The reason the MONO_DEPENDENCIES_PREFIX environment variable is used is because there is no standard location where the dependencies are located. Using this environment variable lets you have one set of dependencies for multiple mono source trees, etc. You can set it on your system, but I just set it in a bat file or on the command line before launching Visual Studio.<br>
