<br>Mixed mode assemblies arenÂ´t portable, period. They contain x86 native code in Windows-specific format, so it won&#39;t run in any other OSes or any other CPU architecture.<br><br>If you really need a portable solution (across OSes) you should use Mono.Data.SqliteClient, and package both native library versions o sqlite (.dll and .so), the correct one will be used automatically by the managed provider. CPU portability is harder to tackle, and would only be reasonable to achieve by implementing a 100% managed version of the engine+provider.<br>
