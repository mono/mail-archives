    I thought I might add ZIP code DB + Region DB + Country DB to the
    things I want to provide on the support site, under some rubrique
    like 'INTERNATIONALIZATION'.<br>
    <br>
    If some people here can provide their country's zip-codes,&nbsp; I'll add
    it to the db.<br>
    Naturally, I would provide ALL of Switzerland's ZIP codes.<br>
    <br>
    Format:<br>
    <br>
    CREATE TABLE [dbo].[T_ZIP_YourCountry](<br>
    &nbsp;&nbsp;&nbsp; [ZIP_UID] [uniqueidentifier] NOT NULL,<br>
    &nbsp;&nbsp;&nbsp; [ZIP_ID] [int] NOT NULL,<br>
    &nbsp;&nbsp;&nbsp; [ZIP_Number] [int] NULL,<br>
    &nbsp;&nbsp;&nbsp; [ZIP_String] [nvarchar](4000) NULL,&nbsp;&nbsp;&nbsp; <br>
    &nbsp;&nbsp;&nbsp; [ZIP_Name_Short] [nvarchar](4000) NULL,<br>
    &nbsp;&nbsp;&nbsp; [ZIP_Name_Long] [nvarchar](4000) NULL,<br>
    &nbsp;&nbsp;&nbsp; [ZIP_State_Abbv] [nvarchar](20) NULL<br>
    &nbsp;&nbsp;&nbsp; [ZIP_Country_ISO] [nvarchar](2) NULL<br>
    ) ON [PRIMARY]<br>
    <br>
    Make sure ZIP_Name_ short+long is in the native language, not
    English. If it's only provided in English (and English is not the
    native language, add an additional nvarchar column, like name_en)).<br>
    Else it's going to be a mess.<br>
