<p>We saw several seg faults when AppDomain.Unload was called
in Mono on an ARM processor.&nbsp; The same code executed fine on x86 mono and
.NET on Windows.&nbsp; What I think was happening, but not sure, was that we
were attempting to unload the only app domain in the process.&nbsp; It seems
like mono under x86 and .NET on windows handles this by ignoring the call and
doesn't cause a seg fault.&nbsp; Also, when we were seeing this, it was
not at the line of code, but just randomly after the call.&nbsp; I believe this
had to do with garbage collection removing references that it shouldn't
have.&nbsp; We could change the timing of the seg fault by introducing some
more intensive logging around the area.</p>
