<p class="p1" style="background:white"><span class="s1"><span style="font-family:Calibri;color:black">I have written a description of my prototype implementation of adaptive ASCII/UTF-16 strings in Mono:</span></span><span style="font-family:Calibri;color:black"><o:p></o:p></span></p>
<p class="p2" style="background:white"><span style="font-family:Calibri;color:black"><o:p> </o:p></span></p>
<p class="p3" style="background:white"><span class="s2"><span style="font-family:Calibri;color:black"><a href="http://www.mono-project.com/docs/advanced/runtime/docs/ascii-strings/" id="LPlnk834666">http://www.mono-project.com/docs/advanced/runtime/docs/ascii-strings/</a></span></span><span style="font-family:Calibri;color:black"><o:p></o:p></span></p>
<p class="p2" style="background:white"><span style="font-family:Calibri;color:black"><o:p> </o:p></span></p>
<p class="p2" style="background:white"><span style="font-family:Calibri;color:black">Introduction:<o:p></o:p></span></p>
<p class="p2" style="background:white"><span style="font-family:Calibri;color:black"><o:p> </o:p></span></p>
<p class="p1" style="background:white"><span class="s1"><span style="font-family:Calibri;color:black">> For historical reasons, System.String uses the UCS-2 character encoding, that is, UTF-16 without surrogate pairs.</span></span><span style="font-family:Calibri;color:black"><o:p></o:p></span></p>
<p class="p1" style="background:white"><span style="font-family:Calibri;color:black"><o:p> </o:p></span></p>
<p class="p1" style="background:white"><span class="s1"><span style="font-family:Calibri;color:black">> However, most strings in typical .NET applications consist solely of ASCII characters, leading to wasted space: half of the bytes in a string are likely
 to be null bytes!</span></span><span style="font-family:Calibri;color:black"><o:p></o:p></span></p>
<p class="p1" style="background:white"><span style="font-family:Calibri;color:black"><o:p> </o:p></span></p>
<p class="p1" style="background:white"><span class="s1"><span style="font-family:Calibri;color:black">> Since strings are immutable, we can scan the character data when the string is constructed, then dynamically select an encoding, thereby saving 50% of string
 memory in most cases.</span></span><span style="font-family:Calibri;color:black"><o:p></o:p></span></p>
<p class="p2" style="background:white"><span style="font-family:Calibri;color:black"><o:p> </o:p></span></p>
<p class="p1" style="background:white"><span class="s1"><span style="font-family:Calibri;color:black">I would like to solicit feedback on this proposal from runtime developers and users alike. In particular:</span></span><span style="font-family:Calibri;color:black"><o:p></o:p></span></p>

<p class="p1" style="background:white"><span class="s1"><span style="font-family:Calibri;color:black">- Specific objections regarding performance characteristics, compatibility issues, &c.</span></span><span style="font-family:Calibri;color:black"><o:p></o:p></span></p>
<p class="p1" style="background:white"><span class="s1"><span style="font-family:Calibri;color:black">- Questions about unclear or underspecified parts of the proposal</span></span><span style="font-family:Calibri;color:black"><o:p></o:p></span></p>
