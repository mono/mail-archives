When I build mono from source and install on linux, I&#39;ve noticed that I end up with .so&#39;s next to some of the installed dll&#39;s and exe&#39;s. This provides a nice speedup when running compilers/tools from the mono distribution. Now that we have AOT working on osx, the same optimization makes sense there, too.<div>

<br></div><div>I decided to try this out with mono 2.8.2 on an intel mac. I manually AOT&#39;d:</div><div><br></div><div>/Library/Frameworks/Mono.Framework/Versions/Current/lib/mono/*/*.exe</div><div>/Library/Frameworks/Mono.Framework/Versions/Current/lib/mono/gac/*/*/*.dll</div>

<div><br></div><div>I have a project that is about 500kloc of C# and builds with gmcs. AOT&#39;ing the mono distribution as I&#39;ve described dropped the build time for this project from 84s to 46s on my c2d macbook pro. This is a 44% speedup!</div>

<div><br></div><div>I&#39;m unsure who owns the OSX build/packaging, but it seems like this should be a simple change to make since we already do it on linux, and it would be great to get this kind of performance out of the box.</div>
