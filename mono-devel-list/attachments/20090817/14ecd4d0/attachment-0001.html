Hi Paolo,<br><br>I think that mono_method_get_param_names should be changed to not require the caller to<br>figure out the number of parameters. In adition to that, this function has no way to return an<br>error back to the caller.<br>
<br>My suggestion is to change it to <br>MonoError mono_method_get_param_names (MonoMethod *method, char **names, int *params)<br><br>On success names points to a g_malloc&#39;d array of *params strings that come from assembly memory.<br>
On failure names and params are left untouched.<br><br><br>Cheers,<br>Rodrigo<br>
