AFAIR, Monodevelop reads csprojs which are msbuild files but with specific semantics, and deals with the needed tasks. <br>Monodevelop doesn&#39;t try to implement all msbuild functionality and specially it&#39;s extensibility features, which are used by <a href="http://VS.NET">VS.NET</a>.<br>
<br>It may be feasible to evolve Monodevelop to use xbuild but probably it would impact the support for the other buildings tasks Monodevelop and its addins currently support. The main reason is that msbuild (and therefore xbuild) doesn&#39;t use Mono.Addins for its extensibility and doesn&#39;t consume/expose the extension points in/to MD/MD-addins, but those of <a href="http://VS.NET/msbuild-addins">VS.NET/msbuild-addins</a>.<br>
