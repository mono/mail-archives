<p class="MsoNormal"><span style="font-size:11.0pt">I am using Mono version 4.8.0.524 and have noticed a memory leak each time I create/unload an AppDomain. I have managed to localise the leak down to running the code from this bug report: <br>
<a href="https://bugzilla.xamarin.com/show_bug.cgi?id=30849" target="_top">https://bugzilla.xamarin.com/show_bug.cgi?id=30849</a><br>
<br>
Basically run AppDomain Create/Unload in a tight loop with garbage collection. <br>
<br>
Looking at the heap usage I can see it grow by about 1MB on each Create/Unload. Eventually it will oom. <br>
<br>
Note: I am running mono in a docker container and am using the cAdvisor docker to watch the heap usage on my mono container. I don’t think that should have an effect but thought it was worth mentioning. <br>
<br>
I have tried using both sgen and boehms GC but am seeing the same behaviour with both. <br>
<br>
Has anyone else seen this? <br>
