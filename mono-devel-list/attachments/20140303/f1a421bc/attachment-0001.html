<p class="MsoPlainText">I had a problem, and I figured it out, thanks to this page:<o:p></o:p></p>
<p class="MsoPlainText"><a href="http://www.cc.dtu.dk/?page_id=304">http://www.cc.dtu.dk/?page_id=304</a><o:p></o:p></p>
<p class="MsoPlainText"><o:p> </o:p></p>
<p class="MsoPlainText">I have a system with mono built as follows:<o:p></o:p></p>
<p class="MsoPlainText">./configure --prefix=/usr/local/mono-3.2.8 && make && echo "" && echo "Done" && echo""<o:p></o:p></p>
<p class="MsoPlainText">followed by <o:p></o:p></p>
<p class="MsoPlainText">sudo make install && echo "" && echo "Ok Ok Ok" && echo ""<o:p></o:p></p>
<p class="MsoPlainText"><o:p> </o:p></p>
<p class="MsoPlainText">I would expect, based on the above webpage, that the ./configure && make should have built mono using LD_RUN_PATH set to /usr/local/mono-3.2.8/lib, but apparently it doesn't - Should that be considered a bug in the Makefile?<o:p></o:p></p>
<p class="MsoPlainText"><o:p> </o:p></p>
<p class="MsoPlainText">The workaround is to specify the LD_LIBRARY_PATH on each call to mono<o:p></o:p></p>
<p class="MsoPlainText">env LD_LIBRARY_PATH=/usr/local/mono-3.2.8/lib /usr/local/mono-3.2.8/bin/mono FunWithFooBar.exe<o:p></o:p></p>
