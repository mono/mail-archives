Bug Report #76429:<BR>
<BR>
Description of Problem: 
<PRE>

On an ASP.NET form with multiple buttons, and RequiredValidators, if you
set a button to CausesValidation=&quot;false&quot; validation is not run but the
button does not generate a submit anymore.  When omitted, the
CausesValidation attribute defaults to try and causes validation.  This is
a problem if you want to cancel validation and simply navigate to another
page. 

Steps to reproduce the problem:
1. Create a ASP.NET form&nbsp; include a textbox, RequiredValidator, and two
buttons.
2. Set CausesValidation attribute to &quot;true&quot; on one but and &quot;false&quot; on the
other button.
3. Create and add (+= new System.EventHandler) event handlers for both
buttons &quot;Click&quot; events in a code behind.
4. For the button with CausesValidation=&quot;false&quot; code the eventhandler to
simply Redirect.Response to another page.

Actual Results:

Pressing the button with CausesValidation=&quot;true&quot; triggers validation.  Once
validation conditions have been met a submit is performed as aspected.

Pressing the button with CausesValidation=&quot;false&quot; does nothing.


Expected Results:

Pressing the button with CausesValidation=&quot;false&quot; should result in the
browser navigating to the page specified in the Redirect.Response call in
this buttons Click callback event.


How often does this happen?  All the time.


Additional Information:
See the problem online at : <A HREF="http://65.12.160.30:8080/Dept.aspx">http://65.12.160.30:8080/Dept.aspx</A>
Press the &quot;Add Dept.&quot; button.
Now try to click the Add button without entering the Department number and
name fields, validation occurs as expected.
Click &quot;Cancel&quot;, nothing happens.
Click &quot;Back&quot;, page is redirected to <A HREF="http://65.12.160.30:8080/Dept.aspx">http://65.12.160.30:8080/Dept.aspx</A>,
this is the behavior that should have occurred when you clicked cancel.
The &quot;Cancel&quot; button at <A HREF="http://65.12.160.30:8080/DeptEdit.aspx">http://65.12.160.30:8080/DeptEdit.aspx</A> behaves as
expected and this page does not contain a R-quiredValidator.
</PRE>
