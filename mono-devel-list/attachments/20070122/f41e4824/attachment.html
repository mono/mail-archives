<p class=MsoNormal><font size=2 face="Courier New"><span style='font-size:10.0pt;
font-family:"Courier New"'>Hi, Marek and everyone.<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face="Courier New"><span style='font-size:10.0pt;
font-family:"Courier New"'><o:p>&nbsp;</o:p></span></font></p>

<p class=MsoNormal><font size=2 face="Courier New"><span style='font-size:10.0pt;
font-family:"Courier New"'>Attached is a diff to improve support for J2EE
portlets for the TARGET_J2EE configuration. In order to keep the code paths
common, most of the diff is not under &#8216;#if TARGET_j2EE&#8217;. Therefore
I&#8217;d like you to review and approve it before I can commit.<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face="Courier New"><span style='font-size:10.0pt;
font-family:"Courier New"'>The main issue stems from the fact that portlets in
J2EE render HTML fragments and not the whole page. That is one page can contain
several portlets and therefore several ASP.Net pages. This breaks implicit code
assumptions about uniqueness of global variables. The most important example is
the &#8216;theForm&#8217; variable. For Portlet support one cannot assume that
on the client side there is exactly one &#8216;theForm&#8217; defined.<o:p></o:p></span></font></p>

<p class=MsoPlainText><font size=2 face="Courier New"><span style='font-size:
10.0pt'>The changes common with Mono are:<o:p></o:p></span></font></p>

<p class=MsoPlainText style='margin-left:36.0pt;text-indent:-18.0pt;mso-list:
l0 level1 lfo2'><![if !supportLists]><font size=2 face=Symbol><span
style='font-size:10.0pt;font-family:Symbol'><span style='mso-list:Ignore'>&middot;<font
size=1 face="Times New Roman"><span style='font:7.0pt "Times New Roman"'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
</span></font></span></span></font><![endif]><span dir=LTR>An internal property
called &#8216;theForm&#8217; is added to System.Web.UI.Page. Its value for TARGET_J2EE
will be different for every portlet.<o:p></o:p></span></p>

<p class=MsoPlainText style='margin-left:36.0pt;text-indent:-18.0pt;mso-list:
l0 level1 lfo2'><![if !supportLists]><font size=2 face=Symbol><span
style='font-size:10.0pt;font-family:Symbol'><span style='mso-list:Ignore'>&middot;<font
size=1 face="Times New Roman"><span style='font:7.0pt "Times New Roman"'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
</span></font></span></span></font><![endif]><span dir=LTR>We pass the
&#8216;theForm&#8217; as a parameter to all the JavaScript functions that use
it directly.<o:p></o:p></span></p>

<p class=MsoPlainText style='margin-left:72.0pt;text-indent:-18.0pt;mso-list:
l0 level2 lfo2'><![if !supportLists]><font size=2 face="Courier New"><span
style='font-size:10.0pt'><span style='mso-list:Ignore'>a.<font size=1
face="Times New Roman"><span style='font:7.0pt "Times New Roman"'>&nbsp;&nbsp;&nbsp;
</span></font></span></span></font><![endif]><span dir=LTR>In cases of callback
functions (like functions assigned to window.onload) we create a function whose
name depends upon the portlet namespace and that passes &#8216;theForm&#8217;
to the global JavaScript function.<o:p></o:p></span></p>

<p class=MsoPlainText style='margin-left:36.0pt;text-indent:-18.0pt;mso-list:
l0 level1 lfo2'><![if !supportLists]><font size=2 face=Symbol><span
style='font-size:10.0pt;font-family:Symbol'><span style='mso-list:Ignore'>&middot;<font
size=1 face="Times New Roman"><span style='font:7.0pt "Times New Roman"'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
</span></font></span></span></font><![endif]><span dir=LTR>We save a reference
to &#8216;theForm&#8217; within the tree view data (also in details view and
grid view data). This allows the JavaScript of these controls to pass the form
to the &#8216;WebForm_DoCallback&#8217; function.<o:p></o:p></span></p>

<p class=MsoPlainText><font size=2 face="Courier New"><span style='font-size:
10.0pt'><o:p>&nbsp;</o:p></span></font></p>

<p class=MsoPlainText><font size=2 face="Courier New"><span style='font-size:
10.0pt'>It is important to note that before this change some key JavaScript
functions were compatible with MS JavaScript functions. After the change these methods
have an extra parameter indicating the form. This means that if someone used such
a function from his JavaScript code then his code will be broken. <o:p></o:p></span></font></p>

<p class=MsoPlainText><font size=2 face="Courier New"><span style='font-size:
10.0pt'>Do you think that being compatible with MS on JavaScript is an
important feature? Unlike the C# API MS does not document these functions and even
tries to abstract their existence in some of System.Web API (e.g. the ClientScriptManager
class).<o:p></o:p></span></font></p>
