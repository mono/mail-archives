<p class="MsoNormal">He future of the Mono framework is bright and long! It’s in our core business to support the different platforms that we support today (macOS, Linux, Windows, iOS, Android, PS4, Xbox, FreeBSD, Haiku, etc.), an. d we are not planning to
 drop support for any of these platforms any time soon. We are even currently adding support for WebAssembly to be able to run .NET on the Web.<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">.NET Core and Mono complement each other with Mono going to places where .NET Core doesn’t go. Also, Mono supports different capabilities than .NET Core does and so has different advantages. If you still want to be able to target both runtimes,
 you should target .NET Standard which is the common bits between and the .NET runtimes (.NET Fx, .NET Core, Mono and .NET Native).<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
