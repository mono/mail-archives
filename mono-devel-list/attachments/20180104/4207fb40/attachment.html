<p class="x_MsoNormal">Hello,</p>
<p class="x_MsoNormal"> </p>
<p class="x_MsoNormal">Mono and .NET core serve different purposes, so they will continue to be developed together.</p>
<p class="x_MsoNormal"> </p>
<p class="x_MsoNormal">Mono’s main use these days is for mobile applications (Android and iOS) as well as providing the .NET Desktop API surface, which is used by existing applications, among those MonoDevelop/Visual Studio for Mac.</p>
<p class="x_MsoNormal"> </p>
<p class="x_MsoNormal">.NET core is optimized for high performance web workloads, and server tasks, and will continue to evolve in that direction.</p>
<p class="x_MsoNormal"> </p>
<p class="x_MsoNormal">For the last couple of years we have had an effort to reduce duplication code, so improvements into one flow into the other, it has been going great and will continue to advance.</p>
