<DIV><FONT face=Arial size=2>is it possible to automate an application using 
Mono?</FONT></DIV>
<DIV><FONT face=Arial size=2></FONT>&nbsp;</DIV>
<DIV><FONT face=Arial size=2>Until last week we wanted to automate one of our 
applications using JavaScript, but Mono came to our mind to be possibly the 
better solution since it has some advantages.</FONT></DIV>
<DIV><FONT face=Arial size=2></FONT>&nbsp;</DIV>
<DIV><FONT face=Arial size=2>Automating means, that we define in our C++ host 
application a lot of classes (and object instances) that shall be accessible by 
a scripting language. </FONT><FONT face=Arial size=2>For example we provide an 
object of type "Circle" and instances named "Circle1", "Circle2", etc. (or 
whatever names).</FONT></DIV>
<DIV><FONT face=Arial size=2></FONT>&nbsp;</DIV>
<DIV><FONT face=Arial size=2>In the script language the user shall be able to 
enter code like "Circle1.color = COLOR_RED" to change properties of the object. 
It shall also be possible to read properties as well as calling methods of 
objects. </FONT><FONT face=Arial size=2>Vice versa our C++ application needs to 
be able to call methods of an objects of&nbsp;the C# environment (ok, we think 
this is already possible).</FONT></DIV>
<DIV><FONT face=Arial size=2></FONT>&nbsp;</DIV>
<DIV><FONT face=Arial size=2>With JavaScript this can easily be done by defining 
a class in a C++ structure and exporting the class to the JavaScript 
engine.</FONT></DIV>
<DIV><FONT face=Arial size=2></FONT>&nbsp;</DIV>
<DIV><FONT face=Arial size=2>A class is defined in C++ in the following way 
(simplified example), by <FONT face=Arial size=2>defining the name of the class 
as well as a set of properties and a set of methods:</FONT></FONT></DIV>
<DIV><FONT face=Arial size=2>typedef struct</FONT></DIV>
<DIV><FONT face=Arial size=2>{</FONT></DIV>
<DIV><FONT face=Arial size=2>&nbsp;&nbsp;&nbsp;const char *Name;</FONT></DIV>
<DIV><FONT face=Arial size=2>&nbsp;&nbsp; MonoPropertySpecs 
*CirclePropertySpecs;</FONT></DIV>
<DIV><FONT face=Arial size=2>&nbsp;&nbsp; MonoMethodSpecs&nbsp; 
*CircleMethodSpecs;</FONT></DIV>
<DIV><FONT face=Arial size=2>} MonoClass;</FONT></DIV>
<DIV><FONT face=Arial size=2></FONT>&nbsp;</DIV>
<DIV><FONT face=Arial size=2></FONT>&nbsp;</DIV>
<DIV><FONT face=Arial size=2>For our "Cirlce"-class definition this might look 
like this:</FONT></DIV>
<DIV><FONT face=Arial size=2>MonoClass Circle =</FONT></DIV>
<DIV><FONT face=Arial size=2>{</FONT></DIV>
<DIV><FONT face=Arial size=2>&nbsp;&nbsp; "Circle",</FONT></DIV>
<DIV><FONT face=Arial size=2>&nbsp;&nbsp; CircleProperties,</FONT></DIV>
<DIV><FONT face=Arial size=2>&nbsp;&nbsp; CircleMethods</FONT></DIV>
<DIV><FONT face=Arial size=2>};</FONT></DIV>
<DIV><FONT face=Arial size=2></FONT>&nbsp;</DIV>
<DIV><FONT face=Arial size=2>The properties are defined in a separate 
MonoPropertySpecs array of structures:</FONT></DIV>
<DIV><FONT face=Arial size=2>(For "Methods" the definition is almost the 
same.)</FONT></DIV>
<DIV><FONT face=Arial size=2>MonoPropertySpecs CircleProperties[]</FONT></DIV>
<DIV><FONT face=Arial size=2>{</FONT></DIV>
<DIV><FONT face=Arial size=2>&nbsp;&nbsp;&nbsp;"Color", CircleGetColor, 
CircleSetColor, Type,</FONT></DIV>
<DIV><FONT face=Arial size=2>&nbsp;&nbsp; "LineStyle", CircleGetLineStyle, 
CircleSetLineStyle, Type,</FONT></DIV>
<DIV><FONT face=Arial size=2>&nbsp;&nbsp; etc.</FONT></DIV>
<DIV><FONT face=Arial size=2>};</FONT></DIV>
<DIV><FONT face=Arial size=2></FONT>&nbsp;</DIV>
<DIV><FONT face=Arial size=2>
<DIV><FONT face=Arial size=2>The problem starts exactly here!!! In JavaScript 
everything is TYPELESS, but not in C#.</FONT></DIV>
<DIV><FONT face=Arial size=2>"Type" must be the type of the property (for 
example MonoInteger, or System.Drawing.Color), I think this is really a problem 
since C++ does not know anything about the classes and types defined in outside 
assemblies.</FONT></DIV></FONT></DIV>
<DIV><FONT face=Arial size=2></FONT>&nbsp;</DIV>
<DIV><FONT face=Arial size=2>For example in our true C++ implementation of our 
"Circle" class, the Color-Member is just a "long" which holds the RGB value of 
the color.</FONT></DIV>
<DIV><FONT face=Arial size=2></FONT>&nbsp;</DIV>
<DIV><FONT face=Arial size=2>Finally, CircleGetColor, CircleSetColor, etc. are 
true C++ functions:</FONT></DIV>
<DIV><FONT face=Arial size=2>void CircleGetColor(void *ObjectInstance, Type 
**ReturnValue)</FONT></DIV>
<DIV><FONT face=Arial size=2>{</FONT></DIV>
<DIV><FONT face=Arial size=2>&nbsp;&nbsp; Circle *p = (Circle 
*)ObjectInstance;</FONT></DIV>
<DIV><FONT face=Arial size=2>&nbsp;&nbsp; *ReturnValue = MonoNew 
Type(p-&gt;Color);</FONT></DIV>
<DIV><FONT face=Arial size=2>};</FONT></DIV>
