<FONT FACE="Calibri, Verdana, Helvetica, Arial"><SPAN STYLE='font-size:11pt'> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;The reason that you cannot called inherited events is because the syntax you use for OnBar is really syntactic sugar for something like this:<BR>
<BR>
protected event OnBar<BR>
{<BR>
&nbsp;&nbsp;add { _onBar = Delegate.Combine(_onBar, value); }<BR>
&nbsp;&nbsp;remove { _onBar = Delegate.Remove(_onBar, value); }<BR>
}<BR>
<BR>
private EventHandler _onBar;<BR>
<BR>
Once you expand this, it&#8217;s easy to see that you can&#8217;t call _onBar in child classes because it&#8217;s private. In most cases, this really does make sense, but if you want different behavior, you can either explicitly define OnBar in this way and make _onBar protected (or whatever), or you can provide a protected method (RaiseBar?) that raises the event if OnBar is not null.<BR>
