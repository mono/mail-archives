while porting a library of ours to be usable under mono, we just come
accross an issue which has some impacts on non-C# .NET languages.<br>
<br>
in Mono.Security there is a MiniParser class which is declared to be
non-CLS compliant:<br>
<a class="moz-txt-link-freetext" href="http://svn.myrealbox.com/viewcvs/trunk/mcs/class/Mono.Security/Mono.Xml/MiniParser.cs?rev=22828&view=markup">http://svn.myrealbox.com/viewcvs/trunk/mcs/class/Mono.Security/Mono.Xml/MiniParser.cs?rev=22828&amp;view=markup</a>
<br>
<pre><b><font color="#0000ff">namespace</font></b> Mono.Xml {

[CLSCompliant(<b><font color="#0000ff">false</font></b>)] </pre>
I did not try to find out what CLS compliancy requirements are not met
in detail, but there is one which causes this class to be less usable
with some .NET languages:<br>
<br>
It defines public/protected identifiers that are distinct by char case
only:<br>
<br>
<pre><b><font color="#0000ff">public</font></b> <b><font color="#0000ff">class</font></b> XMLError : Exception {
[...]
  <b><font color="#0000ff">protected</font></b> <b><font color="#0000ff">int</font></b> line, column;
[...]
  <b><font color="#0000ff">public</font></b> <b><font color="#0000ff">int</font></b> Line {get {<b><font
 color="#0000ff">return</font></b> line;}}
&nbsp; <b><font color="#0000ff">public</font></b> <b><font color="#0000ff">int</font></b> Column {get {<b><font
 color="#0000ff">return</font></b> column;}}

</pre>
This makes all compilers stumble which are not casesentive (such as
Pascal).<br>
Note: I'm not trying to start a discussion if casesensitivity makes
sense or not, this is just about&nbsp; "compatibility". To provide a library
that is thought to be availble for many different .NET
compilers/languages, non-cls compliant code should be avoided as much
as possible. In this "line vs. Line" case its just a road block and
should be changed.<br>
<br>
ps: the [CLSCompliant(<b><font color="#0000ff">false</font></b>)]
correctly declares&nbsp; that piece of code&nbsp; to be non-compliant, but of
course it does not make Pascal-compilers case-senstive ;-) <br>
