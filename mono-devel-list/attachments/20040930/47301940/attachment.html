<DIV>Just a quick comment on this from me (I try to follow Don Box's 
exploits<BR>in this world closely). I agree 100%. Remoting should be limited 
to<BR>interappdomain/process communication. The recommenced path (according 
to<BR>Microsoft) is to use ASMX primarily, or if higher performance is 
needed<BR>use DCOM I believe. Remoting was reserved for communication 
between<BR>processes where types can be "guaranteed" to be the same.<BR> 
<BR>Also with Indigo they are going toward the "WSE" style 
of<BR>interoperability. So instead of using a "remoting" style of 
interop,<BR>they are using the ASMX style of connected systems where the 
contracts<BR>are shared but not the types. I would recommend in Mono an 
investment be<BR>made WSE classes as they would be a precursor to Indigo 
classes<BR>in Mono. I know it will take some effort creating the channels and 
ports<BR>and so on, but I think it would be a step in the right 
direction.<BR><BR>According to some of the quotes I've read, remoting is in a 
similar<BR>state as COM.. It is not dead, but it is pretty much done. 
Most<BR>development will be in the WSE style of coding and 
connecting.<BR><BR>Anyway, that's my 2 cents on the
