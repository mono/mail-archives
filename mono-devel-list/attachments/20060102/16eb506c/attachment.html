<FONT COLOR="#000000">Hello folks,</FONT>

<FONT COLOR="#000000">    I know that this is a very short notice, I should have mentioned</FONT>
<FONT COLOR="#000000">this in mid-December, but I completely blanked out.</FONT>

<FONT COLOR="#000000">    We are going to API freeze Mono this week.  If you have any API</FONT>
<FONT COLOR="#000000">changes that you want to make, please contact me directly.</FONT>

<FONT COLOR="#000000">    What this means is that the public API of the Mono-specific</FONT>
<FONT COLOR="#000000">libraries will cease to change, although we can continue to bug fix it</FONT>
<FONT COLOR="#000000">and document it.</FONT>

<FONT COLOR="#000000">    The .NET libraries already had their APIs set in stone, so those</FONT>
<FONT COLOR="#000000">will continue to be developed (implementing NotImplementedExceptions,</FONT>
<FONT COLOR="#000000">fixing bugs and implementing NET_2_0 protected code).</FONT>

<FONT COLOR="#000000">    Changes to the .NET public API to fix differences against the</FONT>
<FONT COLOR="#000000">published .NET interface is ok (specially in the 2.x universe, as Mono</FONT>
<FONT COLOR="#000000">1.2 will not guarantee .NET 2.0 support in full anyways, so things are</FONT>
<FONT COLOR="#000000">much more lax there).</FONT>

<FONT COLOR="#000000">    So in short: you can change implementation bits, but not public</FONT>
<FONT COLOR="#000000">interfaces.</FONT>

<FONT COLOR="#000000">    If an API change is /absolutely required/ this needs to be discussed</FONT>
<FONT COLOR="#000000">before the patch makes it into SVN.</FONT>

<FONT COLOR="#000000">Miguel</FONT>
