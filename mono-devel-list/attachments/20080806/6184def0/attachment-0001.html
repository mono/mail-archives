<p class=MsoNormal>I just posted version 1.0.55.0 of the System.Data.SQLite provider
here:<o:p></o:p></p>

<p class=MsoNormal><a href="http://sqlite.phxsoftware.com">http://sqlite.phxsoftware.com</a><o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>Included in this update is an option to recompile the
managed provider portion of the code with a SQLITE_STANDARD preprocessor symbol
defined.&nbsp; If set, the provider will compile as managed only and p/invoke
against the stock &#8220;sqlite3&#8221; library with about 99% functionality of
my modified version.<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>I also added the URI connection string from Marek&#8217;s mono
modifications.&nbsp; I *<b>think</b>* that at this point all you&#8217;d have to
do is rename the namespace and re-case the class names to get my provider sync&#8217;d
up to the Mono.Data.Sqlite provider.<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>If anyone&#8217;s interested in working on that and giving
me feedback, I&#8217;d be happy to make any other changes needed in my version to
help keep the mono edition as easy to maintain for you guys as possible.<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>One major caveat:&nbsp; Dr. Hipp only just this version
(3.6.1) added precompiled binary support for several metadata API functions.&nbsp;
The Mono.Data.Sqlite provider was p/invoking these functions already, but the
default binary from sqlite.org did not export them.&nbsp; They included
sqlite3_table_column_metadata(), sqlite3_column_orig(), and several other
metadata-related functions.<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>As such, the only version of System.Data.SQLite (and in fact
any version of the Mono.Data.Sqlite provider) that will pass a complete round
of unit tests is when backed to SQLite 3.6.1.<o:p></o:p></p>
