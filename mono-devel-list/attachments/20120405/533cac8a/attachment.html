<p class="MsoNormal" style="margin-bottom:0.0001pt"><span style="font-size:10.0pt;font-family:"Verdana","sans-serif""> </span></p>

<p class="MsoNormal" style="margin-bottom:0.0001pt"><b><span style="font-size:10.0pt;font-family:"Verdana","sans-serif"">Project Architecture:</span></b></p>

<p class="MsoNormal" style="margin-bottom:0.0001pt"><b><span style="font-size:10.0pt;font-family:"Verdana","sans-serif""> </span></b></p>

<p class="MsoNormal" style="margin-bottom:0.0001pt"><span style="font-size:10.0pt;font-family:"Verdana","sans-serif"">Windows Azure does two main things: It runs
applications and it stores their data.</span></p>

<p class="MsoNormal" style="margin-bottom:0.0001pt"><span style="font-size:10.0pt;font-family:"Verdana","sans-serif"">On Windows Azure, an application typically has
multiple instances; it consists of Web role instances and Worker role
instances, with each instance running in its own virtual machine.</span></p>

<p class="MsoNormal" style="margin-bottom:0.0001pt"><span style="font-size:10.0pt;font-family:"Verdana","sans-serif""> </span></p>

<p class="MsoNormal" style="margin-bottom:0.0001pt"><span style="font-size:10.0pt;font-family:"Verdana","sans-serif"">The Azure version of The Wall Application
utilizes the web role to host <a href="http://ASP.NET">ASP.NET</a> code, and worker role is used for some
background processing tasks such as storing the user entry in table and images
into blob. The communication between the web and worker role takes place via
the Azure Queue.</span></p>

<p class="MsoNormal" style="margin-bottom:0.0001pt"><span style="font-size:10.0pt;font-family:"Verdana","sans-serif""> </span></p>

<p class="MsoNormal" style="margin-bottom:0.0001pt"><span style="font-size:10.0pt;font-family:"Verdana","sans-serif"">Each Web role instance accepts incoming HTTP (or
HTTPS) requests. A Web role can be implemented using <a href="http://ASP.NET">ASP.NET</a>, or another .NET
Framework technology that works with IIS. Windows Azure provides built-in load
balancing to spread requests across Web role instances that are part of the
same application. As shown in diagram.</span></p><p class="MsoNormal" style="margin-bottom:0.0001pt"><span style="font-size:10.0pt;font-family:"Verdana","sans-serif""><br></span></p><p class="MsoNormal" style="margin-bottom:0.0001pt">
</p><p class="MsoNormal" style="margin-bottom:0.0001pt"><span style="font-size:10.0pt;font-family:"Verdana","sans-serif"">A Worker role instance,
by contrast, cannot accept requests directly from the outside worldits not
allowed to have any incoming network connections, and IIS isnt running in its
VM. Instead, it gets its input from a Web role instance, typically via a queue
in Windows Azure storage.</span></p><p class="MsoNormal" style="margin-bottom:0.0001pt"><span style="font-size:10.0pt;font-family:"Verdana","sans-serif""><br></span></p><p></p><p class="MsoNormal" style="margin-bottom:0.0001pt">
