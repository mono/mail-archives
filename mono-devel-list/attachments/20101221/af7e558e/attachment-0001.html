<div>There is a very simple solution to this: build everything but deploy only the dlls that you need. Use mono-linker to find out which dlls you are linking against, so if you don&#39;t need System.Net.dll, System.Data.dll, etc, keep them out.<br>

</div><div><br></div><div>I am using this approach for a game. The Mono runtime fits in just 4MB 7zipped (this includes support for Linq, XML and System.Drawing without libgdiplus.so) and I am embedding the runtime into a trivial, native launcher. Everything is working perfectly so far!</div>
