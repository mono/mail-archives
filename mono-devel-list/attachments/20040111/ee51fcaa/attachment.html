Escape characters are not threated in the right way when creating new
Uri from string.<br>
<br>
Reference : RFC2396 part&nbsp; 2.4 (subsection 2.4.2).<br>
<br>
Steps to reproduce the problem : <br>
1. Create new Uri from string that already contains escape characters
(for example "C:\\tmp\\a%20a\\");<br>
2. Inspect&nbsp; AbsoluteUri property of Uri created.<br>
<br>
Actual result : <br>
"C:\\tmp\\a%2520a\\"<br>
<br>
Expected result :<br>
"C:\\tmp\\a%20a\\"<br>
