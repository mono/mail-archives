As far as I can tell, the interface exposed by the API I attached earlier would be trivially implemented in a cross platform manner.<br><br>Here are the methods/properties exposed:<br><br>IShellItem:<br>- string DisplayName: usually just the filename of the file.
<br>- bool IsFolder: returns 'true' when the item is a folder.<br>- Icon LargeIcon: the large icon for the item<br>- ShellFolder Parent: the item's parent folder<br>- Path: the item's full filesystem path. If the folder is virtual, throws an exception.
<br>- Icon SmallIcon: the small icon for the item<br><br>ShellFolder:<br>- Implements IShellItem<br>- Can be constructed from a filesystem path, or from an Environment.SpecialFolder. <br>- IsImmediateParentOf(): tests to see if a folder is an immediate parent of another folder.
<br>- IsParentOf(): tests to see if a folder is a parent of another folder at any level.<br>- HasFileSystemChildren: if the folder is a virtual folder, but it has filesystem children, returns 'true'. Would probably always return 'true' on non-Windows platforms.
<br>- IsFileSystem: if the folder is a virtual folder, returns 'false'. Would probably always return 'true' on non-Windows
platforms.<br><br>ShellItem:<br>- Implements IShellItem<br>
- Can be constructed from a filesystem path or by iterating over a ShellFolder.<br>- ToShellFolder(): Returns a ShellFolder representing this item if the item is a folder. If the item is not a folder, throws an exception.
<br><br>There are obvious missing pieces here, such as finding an item in a folder. But I can't see anything that would cause problems on other platforms.<br><br>Thoughts?<br><br>Cheers<br>Steve<br><br><div><span class="gmail_quote">
