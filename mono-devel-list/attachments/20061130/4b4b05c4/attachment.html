  <DIV></DIV>As far as I can tell, the interface exposed by the API I attached 
  earlier would be trivially implemented in a cross platform manner.<BR><BR>Here 
  are the methods/properties exposed:<BR><BR>IShellItem:<BR>- string 
  DisplayName: usually just the filename of the file. <BR>- bool IsFolder: 
  returns 'true' when the item is a folder.<BR>- Icon LargeIcon: the large icon 
  for the item<BR>- ShellFolder Parent: the item's parent folder<BR>- Path: the 
  item's full filesystem path. If the folder is virtual, throws an exception. 
  <BR>- Icon SmallIcon: the small icon for the item<BR><BR>ShellFolder:<BR>- 
  Implements IShellItem<BR>- Can be constructed from a filesystem path, or from 
  an Environment.SpecialFolder. <BR>- IsImmediateParentOf(): tests to see if a 
  folder is an immediate parent of another folder. <BR>- IsParentOf(): tests to 
  see if a folder is a parent of another folder at any level.<BR>- 
  HasFileSystemChildren: if the folder is a virtual folder, but it has 
  filesystem children, returns 'true'. Would probably always return 'true' on 
  non-Windows platforms. <BR>- IsFileSystem: if the folder is a virtual folder, 
  returns 'false'. Would probably always return 'true' on non-Windows 
  platforms.<BR><BR>ShellItem:<BR>- Implements IShellItem<BR>- Can be 
  constructed from a filesystem path or by iterating over a ShellFolder.<BR>- 
  ToShellFolder(): Returns a ShellFolder representing this item if the item is a 
  folder. If the item is not a folder, throws an exception. <BR><BR>There are 
  obvious missing pieces here, such as finding an item in a folder. But I can't 
  see anything that would cause problems on other 
  platforms.<BR><BR>Thoughts?<BR><BR>Cheers<BR>Steve<BR><BR>
