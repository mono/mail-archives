<div>We recently implemented all the missing capabilities in the HttpBrowserCapablities class.</div>
<div>During the implementation we implemented this throw to be compatible with MS.Net which </div>
<div>throws if the capability isn&#39;t defined at all.&nbsp; The code is based on the assumption that the </div>
<div>browscaps.ini file should contain the correct values for all the common browsers.</div>
<div>However, this is not the case!&nbsp; Since browscaps.ini has it&#39;s own maintainer we did not update</div>
<div>that file.</div>
<div>&nbsp;</div>
<div>In order to fix this problem add the missing definitions to the browscaps.ini file.</div>
<div>Find the browscaps.ini file next to your machine.config file.&nbsp; This is the code that</div>
<div>finds and loads it, from CapabilitiesLoader.cs:</div>
<div>#if TARGET_J2EE<br>&nbsp;&nbsp;&nbsp;&nbsp;string filepath = &quot;browscap.ini&quot;;<br>#else<br>&nbsp;&nbsp;&nbsp;&nbsp;string dir = HttpRuntime.MachineConfigurationDirectory;<br>&nbsp;&nbsp;&nbsp;&nbsp;string filepath = Path.Combine (dir, &quot;browscap.ini&quot;);<br>&nbsp;&nbsp;&nbsp;&nbsp;if (!File.Exists (filepath)) {
<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;// try removing the trailing version directory<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;dir = Path.GetDirectoryName (dir);<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;filepath = Path.Combine (dir, &quot;browscap.ini&quot;);<br>&nbsp;&nbsp;&nbsp;&nbsp;}<br>#endif<br>&nbsp;</div>
<div>If you don&#39;t know what values to put in the configuration you can find the values from looking </div>
<div>in Microsoft&#39;s definitions in the C:\Windows\Microsoft.NET\Framework\v2.0.50727\CONFIG\Browsers</div>
<div>folder.</div>
<div>&nbsp;</div>
<div>I hope this helps.</div>
