After doing some work on an internal product I encountered some issues with the WS stack in Mono.<BR>
<BR>
1) When serializing to XML it don't respect optional for value types. I have a type that get the below proxy code from wsdl tool:<BR>
<BR>
/// &lt;remarks/&gt;<BR>
[System.Xml.Serialization.SoapType(Namespace=&quot;urn:packetfront_becs&quot;)]<BR>
public class @object {<BR>
<BR>
&nbsp;&nbsp; /// &lt;remarks/&gt;<BR>
&nbsp;&nbsp; public System.UInt64 oid;<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;<BR>
&nbsp;&nbsp; /// &lt;remarks/&gt;<BR>
&nbsp;&nbsp; [System.Xml.Serialization.SoapIgnore()]<BR>
&nbsp;&nbsp; public bool oidSpecified;<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;<BR>
&nbsp;&nbsp; /// &lt;remarks/&gt;<BR>
&nbsp;&nbsp; public System.UInt64 parentoid;<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;<BR>
&nbsp;&nbsp; /// &lt;remarks/&gt;<BR>
&nbsp;&nbsp; [System.Xml.Serialization.SoapIgnore()]<BR>
&nbsp;&nbsp; public bool parentoidSpecified;<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;<BR>
&nbsp;&nbsp; /// &lt;remarks/&gt;<BR>
&nbsp;&nbsp; public string creator;<BR>
}<BR>
<BR>
The issue here is that it always sends the oid and parentoid field regardless of value of &lt;field&gt;Specified.<BR>
I got a patch (against SVN) for it, don't know if it's correct but solves the problem for me. So now it only sends oid and parentoid when &lt;field&gt;Specified is set to true. This was tested with 1.1.17.1 but the problem is in SVN as well from what I can see.<BR>
<BR>
2) When using wsdl2 from latest release of Mono I have the following issues:<BR>
&nbsp;1) When using properties for member access, and field is the same name as a keyword it don't prefix the property name with @.<BR>
&nbsp;2) It seems to generate each service binding twice. One with the normal name and another with &lt;name&gt;1. I.i<BR>
&nbsp;&nbsp;&nbsp; [System.Web.Services.WebServiceBinding(Name=&quot;packetfront_becs&quot;, Namespace=&quot;urn:packetfront_becs&quot;)]<BR>
&nbsp;&nbsp;&nbsp; [System.Diagnostics.DebuggerStepThroughAttribute()]<BR>
&nbsp;&nbsp;&nbsp; [System.ComponentModel.DesignerCategoryAttribute(&quot;code&quot;)]<BR>
&nbsp;&nbsp;&nbsp; public class packetfront_becs : System.Web.Services.Protocols.SoapHttpClientProtocol {<BR>
&nbsp;&nbsp;&nbsp;&nbsp; &lt;snip&gt;<BR>
&nbsp;&nbsp;&nbsp; }<BR>
&nbsp;&nbsp;<BR>
&nbsp;&nbsp;&nbsp; [System.Web.Services.WebServiceBinding(Name=&quot;packetfront_becs&quot;, Namespace=&quot;urn:packetfront_becs&quot;)]<BR>
&nbsp;&nbsp;&nbsp; [System.Diagnostics.DebuggerStepThroughAttribute()]<BR>
&nbsp;&nbsp;&nbsp; [System.ComponentModel.DesignerCategoryAttribute(&quot;code&quot;)]<BR>
&nbsp;&nbsp;&nbsp; public class packetfront_becs1 : System.Web.Services.Protocols.SoapHttpClientProtocol {<BR>
&nbsp;&nbsp;&nbsp;&nbsp; &lt;snip&gt;<BR>
&nbsp;&nbsp;&nbsp; }<BR>
<BR>
#2 can be worked around by using the wsdl tool instead.<BR>
<BR>
Other than this it works like a charm so kudos to you all for doing an magnificent work.<BR>
