Well Mono is the father project of Moonlight.<div><br><div>As Silverlight depends on .NET, Moonlight depends on Mono.</div><div><br></div><div>Mono is an open source implementation o .NET, for many platforms, notably Linux and MacOSX.</div>
<div>Moonlight is an open source implementation of Silverlight, which is a competitor to Adobe Flash, a browser plugin that runs visually sophisticated apps.</div><div><br></div><div>Moonlight is mainly written in C, specially the graphic routines, but embeds a shrink-down version of Mono and a not-so-small subset of its class libraries, that match what you have in Silverlight.</div>
<div><br></div><div>With Moonlight you can write your RIA app in C# + XAML (the xml language for defining UI), add managed libraries written in almost any managed (.NET) language, and pack for execution in the user&#39;s browser.</div>
<div><br></div><div>For developing Silverlight/Moonlight apps you&#39;ll need MS tools, as currently there aren&#39;t good enough tools in Linux. You can develop writing everything by hand, but that is very hard, as Moonlight is a huge beast to master, and visual aspects are very important in such  apps.</div>
<div><br></div><div>Support for developing for Moonlight with the MonoDevelop IDE, is improving but it is far from perfect currently.</div><div><br></div><div>Hope it helps,<br><div><br></div><div><br clear="all">Rafael &quot;Monoman&quot; Teixeira<br>
