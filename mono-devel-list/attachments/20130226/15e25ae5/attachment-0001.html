      <div class="moz-cite-prefix">I don't know how good it is for a
        student project, <br>
        but one of the things that have annoyed me quite a bit in the
        past <br>
        is the lack of automatic indentation in MonoDevelop.<br>
        <br>
        If you copy-paste code from one place to another <br>
        (e.g. stackoverflow post to your program), <br>
        you have a horrible mess afterwards.<br>
        And while it's trivial to clean up the mess, <br>
        it's relatively time-consuming and highly annoying.<br>
        In Visual Studio, if you put the closing brace }, <br>
        it automatically indents the code for you.<br>
        <br>
        I find this feature immensely useful, <br>
        yet profoundly lacking in MonoDevelop. ;)<br>
        <br>
        The bad thing is, that when I work on Linux and copy a lot <br>
        (for example when I wrote a C# <a moz-do-not-send="true"
href="http://stackoverflow.com/questions/10820311/c-c-c-howto-do-mount-a">mount

          wrapper</a>, <br>
        for which I copy-pasted and then converted a lot of C-code) , <br>
        I have to occasionally switch over to Visual Studio to clean up
        the mess, <br>
        because I cannot work anymore in that mess.<br>
        This means I have to copy the MonoDevelop project <br>
        from the Linux machine to the windows machine, <br>
        and then back again, over a not that fast wireless network, <br>
        not to mention the long startup time of windows.<br>
        <br>
        Now that debugging and IntelliSense in MonoDevelop work great, <br>
        the moment that automatic indentation arrives would be the
        moment <br>
        I'd dump Visual Studio more or less completely.<br>
        <br>
        One of the nicest features in Visual Studio is, <br>
        that you can choose to customize the indent formatting, <br>
        e.g. you can have it indent not only like  K & R, but also
        like ANSI.<br>
        For example I like ANSI indentation, like in the C# code, <br>
        but the default indentation for JavaScript is K & R, <br>
        but you can choose to have newlines before opening braces in <br>
        Tools-> Options ->SourceCode Editor -> JScript<br>
        which then effectively indents like ANSI.<br>
