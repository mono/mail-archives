On Sat, 2008-02-09 at 18:00 -0200, Edson - gMail wrote:
> I would like to know which is the best way to install mono and 
> monodevelop on Debian Etch 4.0. Debian Etch installs mono by debian 
> repositories, but it doesn't install monodevelop. I would like to know 
> this way, because I want to update my installation and install 
> monodevelop. Thanks by any help.

Debian/Etch ships Mono 1.2.2.1 and no MonoDevelop, thats correct.
I am maintaing backports of Mono and MonoDevelop from Debian/Testing for
Debian/Etch.

Mono 1.2.6 packages for Debian/Etch are available from:
http://www.backports.org

Mono 1.2.6 and MonoDevelop 0.18 is available from:
http://debian.meebey.net/etch-backports
(I didn't upload MonoDevelop yet to backports.org)

If you want always the latest Debian packages of Mono and MonoDevelop
you can use this APT source line:
deb http://debian.meebey.net/etch-backports /
this will give you Mono 1.2.6 + MonoDevelop 0.18 currently

Or if you want to stay with Mono 1.2.2.1 from Debian/Etch, you can also
only install MonoDevelop 0.18 with this APT source lines:
deb http://debian.meebey.net/etch-backports/monodevelop /
deb http://debian.meebey.net/etch-backports/mono-addins /
deb http://debian.meebey.net/etch-backports/cli-common /
this will give you MonoDevelop 0.18 only currently

Just add those APT repositories to your /etc/apt/sources.list and you
get the latest official packages (official as in, packages maintained by
the official debian maintainer).

If you use backports.org, make sure you read
http://www.backports.org/dokuwiki/doku.php?id=instructions
