allways behave in the same way when running in your AppServer, in a
standalone XSP server, in the Apache mod, or in any custom server
application.

However, an application server provides some extra services that can be very
useful for some type of applications. For example: hosting of remote
components. My document talks about this service because COM+ provides
component hosting (hosted components are remotely accessible through DCOM
and SOAP, but not remoting), although strictly speaking it is not part of
EnterpriseServices.

I think it is good to keep EnterpriseServices independent from any
application server, it will give people more choice (choice between
application servers, or choice to not use any application server at all).
