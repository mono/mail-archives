The CodeDom support for JScript should be in place although it hasn't 
been tested extensively. And it should be enabled somewhere in the 
ASP.NET classes.

The important missing piece is the JScript's compiler support for 
defining classes, access to the class libraries, and the other MS 
extensions to the language, without that it's useless. Currently only 
the ECMAScript subset is supported on the compiler.
