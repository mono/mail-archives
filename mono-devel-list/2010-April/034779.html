It would help if you could file a bug report with a reduced, self-contained
test case. Bug reports stay around, mailing list messages don't.

Also, it can be a PITA to recreate a full test case from an incomplete one,
and it's also error prone. It's better for the bug reporter to create a
complete stand-alone test case, rather than expect every interested person
to create their own test case from an excerpt (which may or may not
reproduce the bug in the end, in case it depends on something else not
included in the code extract, or depending on people's interpretation of
what's missing)...
