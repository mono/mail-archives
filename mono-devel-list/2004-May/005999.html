I've merged a patch from Rogerio Pereira Araujo <webmaster at cisnet.com.br>
into the the current sources. It implements a fallback for NUMBERs incorrectly
reported with Scale == 0. This happens when the number field in Oracle is 
defined without a  scale parameter.

but no other came to my mind.
 
If there are no objections against this patch I will commit it to CVS.
