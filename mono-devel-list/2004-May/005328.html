On a machine where there is no suitable system install command so the script
install-sh is used the way libgc's makefile was setup doesn't work because
INSTALL is propagated down rather than recalculated as the proper relative
path in each Makefile (this affects installing libgc-mono-private.h). This
patch just removes the propagation of the INSTALL related variables - and
I'm not 100% sure why this was done in the first place as other Makefiles
don't do it.

Would be good to get into the beta for cleaner installs assuming it's OK.
