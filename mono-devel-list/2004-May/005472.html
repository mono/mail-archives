> I haven't fully followed this discussion, so if I'm reitirating something just

> the configuration features one has as Admin in the .NET Configuration 
> Console? I think one shouldn't forget those issues in this discussion.

What security issues? We expect user compiling programs to trust:

  a) the root or anyone else who has write access in /usr/lib/*, and
     /usr/lib/mono/gac and /usr/lib/pkgconfig specifically
  b) the users who have write access in his PATH
  c) himself, or anybody else who has write access in current
     compilation directory
     
This are the usual Unix assumptions.
