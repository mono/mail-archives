Hello all,

There has been a lot of changes in public key and versions in CVS as we try
to keep the build working toward beta1. Right now we are able to delay sign
most assemblies (like Fx, IBM.Data.DB2) using their original keys and even
fully sign some of them (like Nunit and ICSharpCode.SharpZipLib) when
possible. That, and the original version numbers, should make it easier to
co-exist assemblies between Linux and Windows.

This is not final and may still change anytime if we encouter new problems
or incompatibilites, so beta1 may not be perfect with this. Please report
any differences in bugzilla so we do not loose track of them (in long email
threads ;-).

A big thanks (even if this isn't finished) to everyone that helped.
