The libc.so symlink comes from a -devel package in most distros:

[malekith at roke src]$ rpm -qf /usr/lib/libc.so
glibc-devel-2.3.3-0.20040318.1+nptl

We probably don't want users to have to install glibc-devel in order to
run mono programs.
