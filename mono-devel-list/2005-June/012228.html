Hi,

I was wondering if we can have some tool (like bug-buddy? havent used
it personally tho') which can collect background/environment/etc info
on a client machine which can be used for filing bug-reports. My point
is that as mono gets more n more popular, we'll have people who have
little or no experience with linux n all, trying to run their .net
apps on mono so it would be good if we can get useful bug reports from
such people as these kinda things will really exercise mono and our
class libraries implementation.

We have a doc on mono-project.com (http://www.mono-project.com/Bugs) "
How to make a good bug report" but that isnt much useful for noobs (to
mono), maybe having things like "use MONO_LOG_LEVEL=debug" might be
useful there.

If everyone agrees that the tool can be useful, then i can work on it,
but i dunno what ALL might be required that can be helpful in
debugging.
