Hello,

So the situation is :
1) Everyone wants these tests to be run as often as possible;
2) These tests need big files that cannot be packaged in mono-*.tar.gz;
3) The abilty to be offline when running these tests is important;
4) Massive downloads each time the tests are run are not acceptable.

What do you think about having :
1) A simple shell script whose goal is to download the necessary package
(from oasis and w3c and others if needed).
2) two targets : run-tests an run-partial-tests
a) run-tests will require the full test packages. If the packages are
not there, it will print an error message and instructions on how to
download them using the shell script described in 1.
b) run-partial-tests will not require any external package but will
print a big warning recommending loudely to use the target run-tests
before comitting any modifications.

Thanks,

--yas
