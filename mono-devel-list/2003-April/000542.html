In preparation for the next mono release scheduled for next week,
I have made mini the default jit. The new jit is now available with the
binary name 'mono', while the old jit can still be accessed with the
name 'oldmono'. The same happened to the library used for embedding:
the new jit one is called libmono and the old one is called liboldmono.
Hopefully, after the next release we'll be able to drop the old jit 
completely.
mini has not been tested (or compiled) on windows and *bsd that I know
of, so, if anyone has access to those systems, please test the new build
and the new jit instead of the old one.

Thanks!
