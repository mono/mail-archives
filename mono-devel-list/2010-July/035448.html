I suppose it would be possible to use libgdiplus on Windows but what are you
trying to accomplish?

Although I do not know, I assume that Cairo and the other libraries that
libgdiplus depends on are just calling down to GDI+ on Windows anyway. 
Adding libgdiplus would just be a useless layer.

It would seem far superior to use the actual GDI+ in Windows rather than to
try to bolt on a compatibility layer that was designed for Linux.  The whole
point of libgdiplus is to allow System.Drawing to call into the Windows API
while running on Linux.  Obviously both Windows and Wine support the Windows
API natively and do not need the extra library.
