Whatever direction Mono is going to take in the transaction space, I
sincerely hope it will not be anything complex like J2EE. I personally
would like to see a simple solution based on the Transaction Processing
standards of the Open Group standard (see
http://www.opengroup.org/bookstore/catalog/tp.htm).

FWIW, My company provides a product for integrating .NET with the
Tuxedo transaction processing platform of BEA Systems, both client and
server side (see http://www.otpsystems.com/DotTux.html). A port of
DotTux for Mono is planned but has not started yet.
