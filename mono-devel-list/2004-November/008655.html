                                           Hi,

  I'm not an expert in libgdiplus matters, but libgdiplus is a normal
unix shared
library which the runtime tries to load. You don't need gdiplus.dll or
anything like
that, instead you have to somehow convince the runtime to load libgdiplus.so
when it tries to load 'gdiplus.dll'. This can be done by adding an entry for
gdiplus.dll into the runtime config file which is named 'config' and should be
somewhere under /usr/local/etc/mono.

If that doesn't work, could you run you app with the env var MONO_LOG_LEVEL
set to 'debug' and send the output to the list ?

                                                           Zoltan
