> Microsoft has recently introduced support for Portable Class Libraries,

> recompiling the source.  There are restrictions on which APIs are available,
> and some minor API changes from their non-portable equivalents to gain
> consistency across the platforms.

> to deliver equivalent Portable Class Library system library implementations,
> to be used as library references when compiling .Net 4 / Silverlight 4
> portable code with gmcs?
>

> build profile and probably use linker or hack the compiler to strip all IL
