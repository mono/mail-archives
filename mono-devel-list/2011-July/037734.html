Microsoft has recently introduced support for Portable Class Libraries,
allowing the same DLL to be reused for both .Net 4 and Silverlight 4 without
recompiling the source.  There are restrictions on which APIs are available,
and some minor API changes from their non-portable equivalents to gain
consistency across the platforms.

http://visualstudiogallery.msdn.microsoft.com/b0e0b5e9-e138-410b-ad10-00cb3caf4981

<http://visualstudiogallery.msdn.microsoft.com/b0e0b5e9-e138-410b-ad10-00cb3caf4981>What
additional development effort would be required for the Mono project to
deliver equivalent Portable Class Library system library implementations, to
be used as library references when compiling .Net 4 / Silverlight 4 portable
code with gmcs?
