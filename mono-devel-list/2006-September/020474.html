This is a first hack at getting eglib to build on msvc 2005.  Please
note that this patch is a total mess, but it at least highlights the C
language and header file issues that are present in msvc.  Also, due to
the lack of default bash environment to run configure scripts, I create
forked copies of the generated header files for non cygwin/mingw builds.
These could conceivabally be generated using some vbscript or javascript
launched from the events in the project file as well.  This patch is
less intended to be applied as is, and more looking for feedback as to
how address the issues with building eglib as is on msvc.  I look
forward to your feedback on these issues.
