No, you have to list all the options as you would if you wanted to invoke
the MS compiler (vbc) from the commandline.
If you build using Visual Studio in the output window you'll see the entire
commandline that is used to execute vbc, just copy it and change the path of
the compiler to vbnc.exe and it should work.

Please note that vbnc.exe does not yet do the same magic as MS when dealing
with platform invokes.
