Hi,

Rather than ask questions in a piece-meal fashion about Mono APIs that 
Scala.NET could use, I guess it's better for me to mention the context for 
those questions, i.e. project ideas that might be of mutual interest (to 
both the Mono and the Scala open source communities). In no special order, 
besides the already mentioned "using ilasm as a library", here they go:

(a) in-memory ILAsm compilation
(b) C# to Scala ("Scalify") [1]
(c) a Scala compiler plugin to automate (some) API migration tasks from JDK 
to .NET [2]
(d) presenting the Scala-level API for an assembly compiled from C# (similar 
to "Metadata as Source", [3])

We even have a mailing list just for compiler internals :-) [4]

Additional resources:
[1] 
http://www.sts.tu-harburg.de/people/mi.garcia/ScalaCompilerCorner/MixedSourceScalaCSharp.pdf
[2] Project Ideas at 
http://lamp.epfl.ch/~magarcia/ScalaCompilerCornerReloaded/
[3] http://msdn.microsoft.com/en-us/library/ms236403.aspx
[4] http://www.scala-lang.org/node/199
