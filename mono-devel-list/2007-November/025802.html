I did, I have the FFmpeg decoding working already. For streaming input, there's just a few config changes to get it to do that and it should be up as well, so if your camera can do streaming (to a format FFmpeg can recognise), I'll need the protocol information to ensure the streaming FFmpeg can work with it.

There are two ways to handle that. One is via network streaming, which can support UDP/RTP, HTTP/HTTPS/FTP/MMS,or RTSP, and the other is via Video4Linux. If it's Video4Linux, you would need to use VLC media player to re-stream it, as it would not be on the same machine as the capture device. VLC can then take the Video4Linux stream and convert it ("transcode") it into something FFmpeg can view remotely.

Otherwise, if your bitmap iplImage capture can capture the image every 1/15th of a second, encode that to a jpeg (via Bitmap.SaveToStream) and send that across the network, that will be almost as good, for a LAN scenario.

Cheers,

Euan.
