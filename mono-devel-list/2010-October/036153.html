I've created a site for the new Harry Potter movie which is expected to get a high volume of traffic, and on launch day we are now crashing.

We are getting the following errors on the webservers due to traffic:
Maximum number of concurrent mod_mono requests to /tmp/mod_mono_dashboard_default_2.lock reached (20 active, 20 waiting). Request dropped.
I have the following config:
User nobody
Group nobody
include /etc/httpd/conf/mod_mono.conf
MonoSetEnv MONO_THREADS_PER_CPU=500

And have also added this:

MonoMaxWaitingRequests 0

These settings do not seem to be taking affect.  What other steps should we take to allow mono to handle the traffic?
