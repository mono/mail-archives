Hi,

If you can, using Valgrind is usually the best tool to track down
memory corruption, it will slow down everything immensely though.

See: http://www.mono-project.com/Debugging#Using_Valgrind_on_Mono

I also find it slightly strange that managed code (#14 in your stack
trace) calls monoeg_g_hash_table_lookup, so I'd figure out which frame
that is. Trying to figure out what managed code causes this will
likely make it easier to produce a smaller test case.

Rolf
