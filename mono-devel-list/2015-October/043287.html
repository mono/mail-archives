Hello.  I originally reported these issues on Mono-osx, but perhaps they
belong here, and now I have more information on them anyway.  I am using
10.11.0.

1. When 4.0.4.4 was released, the link on
http://www.mono-project.com/download/ was not updated, so it still points to
4.0.4.1 (erroneous commit here:
https://github.com/mono/website/commit/2fa39f6c35a05938a4e2745bd36905086fb5ee43#diff-36ad12fb058ca51dc440761f8c20584f).

2. I had previously reported that installation to the command-line was not
working for me, and I assumed it was due to SIP blocking /usr/bin/ because I
saw messages to that effect in Console when installing older Mono releases. 
However, the messages did not show up when attempting to install 4.0.4.1 and
.4, yet I was still left with no "mono" available on the command line.

Upon inspecting the Installer package "postinstall" scripts for 4.0.4.1 and
.4 (where are these scripts in GitHub, anyway?), I see that they were in
fact changed to use /usr/local/bin/, so that's nice, but here's the catch:
if the directory is not created yet in OS X, this script will silently fail. 
That's apparently why it hasn't been installing for me on my new laptop,
which does not have this directory.
