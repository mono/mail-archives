We are a team involved in developing an open source SOAP implementation 
using C# in the .NET framework. We are still in the preliminary stages....

We would like to contribute towards MONO by providing our SOAP 
implementation. For this, we want to know whether MONO has already developed 
a SOAP implementation (similar to the Microsoft .NET SOAP implementaion and 
Web Services Support), and if not, what are the plans regarding enabling Web 
Services Support in the new .NET Framewwork built in MONO.

Therefore anyone who is familiar with deploying/consuming Web Services in 
AXIS Java/Axis C++ will find it easy to switch over to our SOAP 
implementation in .NET.

whether these classes have been implemeted in MONO.
