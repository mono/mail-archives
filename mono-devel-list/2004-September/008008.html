One of the critical shortcomings of .NET technology web services is the 
effective use of global objects.

http://blogs.geekdojo.net/brian/archive/2004/08/26/MSApplicationBlocks.aspx

Obviously, many of us want to write and use global files, documents, and so 
on, but at best, a Web service can only use a value object which can be 
stored and retrieved from the Application object or Cache.

Now, with Application Blocks, such as the Cache block, Microsoft ( and this is 
new stuff ) makes global objects accessible to multiple application sources 
and users.

Question: Does or will mono implement this technology for Linux ?   I feel 
it's /absolutely/ critical.   Not only that, but Microsoft has open sourced 
the Block technology.
