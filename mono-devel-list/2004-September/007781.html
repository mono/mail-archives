I do not know if this is a big help, but running the unstable release of
mono on Debian I got a lot of System.NullReferenceExceptions too. I got
them when using mcs, monodevelop, and my own problems. They have stopped
after I upgraded to kernel 2.6.X. So, if you are running kernel 2.4.X
you should consider upgrading.

I do not have time to dig up the references (sorry), but looking at the
bug reports for various mono packages you will find that I was not the
only person having trouble combining mono with kernel 2.4.X.
