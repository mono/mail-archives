Is it worth investigating mono for use in small systems?

I'm attracted to it, but wrote off Java solutions because their installation
footprint is rather large.

If I don't need any GUI support, but do need basic networking and XML (and
the web server), how small an image could I get away with?

This would be om a mini-ITX running linux or *BSD from compact flash.

Is there much of a savings to be made by discarding the JIT?

I don't believe performance is too much of an issue.  Much of what I want
for the core system will be in a native library anyway, and my application
is one that could probably run on a TINI - its just (perhaps paradoxically)
somewhat cheaper to use a mini-ITX.
