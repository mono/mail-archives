Well, you have to download and install Oracle Client from Oracle to be
able to access an Oracle Database with any of the alternatives
(specific ADO.NET provider, OLEDB, ODBC).  That is why that bug case
was closed with NOTABUG.

I could not find any open source alternative that directly connects to
the Oracle Server,  all the ones I've found (like the ones for PHP,
PERL, Python) just wrap, and so depend on, the OCI libraries installed
by the Oracle Client installer.

That is a different situation from trying to connect to a MS SQL
Server, whose line protocol has been reverse engineered, and so we
have, as in Mono, completely open/free implementations of the client
stack.

Hope it clarifies things a bit,
