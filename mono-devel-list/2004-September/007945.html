If mono merges ByteFX in place of Mono.Data.MySql a few important things are 
fixed.

The code will be in a repository that people have access to or that people can 
be granted access to.  The sourceforge ByteFX project requires that one of 
the two authors give write permission to commit changes.

Mono would have an improved Mono.Data.MySql.  Why have Mono.Data.MySql if you 
are going to recommend people don't use it?  Mono wouldn't need to refer 
people to an unsupported project instead of internal libraries.
