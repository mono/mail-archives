The first suspicious thing in the output is the pixel format of -1 and the width/height of 0. This should not be the case for the sws to get it's correct pixel conversion context.
Alos, the merged C# program seems threaded as you're getting a stack trace halfway through the FFMpeg, with presumably some phidgets calls mixed in. FFmpeg is constantly creating memory buffers and dumping into that, if something from phidgets is leaking into memory, that can cause FFmpeg to trip.
The first thing to then do here is to work out which line of code is causing the problem, debug statements to narrow that down, and then look at what's happening before it.
