Hi Mauricio,

    My understanding from reading the site is that for LGPL or GPL-based applications, the "free" one is available as compiled from source. I'm not part of mono project, so this is just my interpretation from the site.

 

The source downloaded successfully for me from:

 

http://anonsvn.mono-project.com/source/branches/wii

 

However the following one listed on the website didn't work:

 

http://anonsvn.mono-project.com/source/trunk/wii

 

neither did:

 

http://anonsvn.mono-project.com/trunk/wii

 

However, as you state, this does not summarise the status of the port and what features work vs don't work (i.e. for Icarus Scene Engine and monoBotics, PInvoke support for Tao or OpenTK would be essential (OpenGL support specifically, the player is agnostic enough that if we needed a different OpenGL provider, it can be quickly replaced), as well as Reflection, Generics, Threading, System.Drawing, System.Threading, System.Data (sockets particularly), System.Xml, and Code-dom to get ISE.Flow and the MBF compiler running, as well as some parts of WinForms for inspector attributes.

 

Note that WinForms is ONLY needed in ISE.Player for the Attributes to compile Icarus, so push comes to shove, we just need to add that part of the source inside wihtout needing to actually get WinForms running.

 

Cheers,

 

Euan.
