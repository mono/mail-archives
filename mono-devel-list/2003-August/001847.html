Hello,

> Can I ask a really stupid question?
> 
> Why is there a version of mono for windows? There is already a perfectly
> good implementation of the CLR there.

Dennis has a good initial list, here are some other reasons:

	* Since the Mono community is probably 50% Unix-developers,
	  and 50% Windows developers, we need to make sure the Windows
	  contributors can tests their code.

	* Helps keeping our code portable.

	* Source code ;-)

If any of the above are not important to you, feel free to not install
Mono/Windows on your box though ;-)
