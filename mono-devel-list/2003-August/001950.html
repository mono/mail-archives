> I tried to build the debugger from CVS, but I couldn't due to an
> error (GTK include files not found).

When I last tried to build the debugger, I noticed that the configure
script had a rather strange failure mode: if you had the gtk2
development files installed, but not the libzvt ones, then the
configure script wouldn't fail like it should.  Instead, the build
would fail, complaining that the GTK libraries can't be found.

So: assuming you already have most of the GTK libs installed, make
sure you install the libzvt libraries and development headers as well.
