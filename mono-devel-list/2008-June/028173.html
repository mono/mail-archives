Hello,

Miguel asked me to document DTrace support in the mono(1) man page. As  
a draft I've therefore briefly listed the current probes, below the  
valgrind section, and added instructions how to check availability of  
the probes. Is this what you meant? It's my first man page, so let me  
know if the formatting should be done differently, I just copied  
blindly from the sections below and above.

Another thing to consider is whether to distribute the mono.d provider  
file for reference. Valgrind's mono.supp file is apparently being  
distributed, but that is more portable than DTrace I assume.

The libtool developers btw seem open to adding support for the odd  
DTrace build steps on Solaris, but it'll likely require users to  
install a new libtool version.
