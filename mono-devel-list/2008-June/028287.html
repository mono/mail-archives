>> eglib directory is missing in bz2 file[1]

> Eglib was never released and it was never included in the tar files.
> If you want to use it, you must compile Mono from SVN.

We once added it to the list of distributed dirs but had to revert  
that change because it broke the automated build. The problem was that  
the buildbot tried to run `make dist` inside eglib when eglib wasn't ./ 
autogen.sh'ed and configured, so with no Makefile present in eglib the  
whole build broke.

Unless something has changed, it's possible to include it in a tarball  
by running autogen.sh and configuring --with-glib=embedded, then  
running make dist iirc. Whether that is desired is another question.
