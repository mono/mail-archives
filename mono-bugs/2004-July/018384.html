+Summary: Permit Linking between Mono Handbook and Class Library Docs
+
+Feature Request:
+
+Permit a way to link (as in HTML anchor link) between the Mono Handbook and
+the Class Library documentation.
+
+For example, it should be possible to link to "T:System.Diagnostics.Trace"
+from the Mono Handbook.
+
+This would permit more complete Mono Handbook information, as any guides
+could directly reference the pertinant documentation.
+
+Linking should also be bi-directional: the Class Library documentation
+should be able to link to the Mono Handbook.  This could be used so that
+DllImport and Marshal could reference the Managed and Unmanaged Code
+Interop guide.
+
+At present Handbook -> Class Library docs don't work as the links are
+"munged" by prefixing all local URLs with
+"source-id:7:xhtml:html/en/wrapping/" (in the case of the Mananged and
+Unmanaged Code Interop guide).
+
+I haven't looked at Class Library -> Handbook linking, so this might
+already be possible.
