+Summary: [zmd] Exception in remoting call when trying to run updates
+
+Description of Problem:
+When trying to run updates as a non-root user using mono 1.2.5p4, ZMD dies
+saying Exception in remoting call.
+
+
+Steps to reproduce the problem:
+1. Install openSUSE 10.2 i386 with the default apps
+2. DO NOT install any updates
+3. Upgrade to mono 1.2.5p4 from here:
+   http://mono.ximian.com/monobuild/preview/download-preview/suse-102-i586/
+4. reboot the computer
+5. Login to the gui as a normal user
+6. Wait for the zen-updater icon to turn into an exclamation point
+7. Click on the Exclamation point
+8. The updater will say that there are ~123 updates, however zen-updater
+will only show 1 update to libzypp (I think this is supposed to happen)
+9. Click "Update" to install the libzypp update
+10. Add your non-root user as a privileged user of ZMD
+11. Click "Apply" on the Change Summary dialog
+12. Click "Close" on the "update was successful" dialog
+13. Notice that now the list of updates contains the other ~122 updates. 
+14. Click "Update"
+15. Notice that zen-updater just sits there on "Resolving Dependencies"
+
+
+Actual Results:
+zen-updater just sits there on resolving dependencies forever. I let it sit
+for > 3 hours and it never pop-ed up the list of dependencies (this step
+takes < 2 minutes on 1.1.18.1).
+
+In the zmd-messages log file it says:
+Exception in remoting call
+
+and then it seems that ZMD gets restarted.
+
+
+Expected Results:
+It should resolve the deps and install them
+
+How often does this happen? 
+Everytime IF above procedure is followed. It works if you run it as root.
+
+Additional Information:
+Works on 1.1.18.1 (tried it twice)
+Fails on 1.2.4 (tried it once)
+Fails on 1.2.5p4 (tried it three times)
