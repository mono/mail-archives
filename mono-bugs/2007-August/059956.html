+Summary: [Plastic] Can't exit from the "merge tool" back to the "merge branch" dialog
+
+Description of Problem:
+If you try to merge a branch, after the merge tool pops up, if you "exit
+without saving" to go back to the merge branch dialog, a blank box appears
+which makes it impossible to click on any other parts of Plastic GUI (it
+seems that this blank box has focus).
+
+This might be related to bug 82405.
+
+This is a regression in 1.2.5p3 as this does not happen on 1.2.4. I tried
+repeatedly to make this happen in 1.2.4 and couldn't make it happen.
+
+
+Steps to reproduce the problem:
+1. Download the plastic on Mono VMWare Image
+     wget http://www.vodagest.net/mono1.2.4_plastic.zip
+2. Upgrade the VMWare image to 1.2.5p3
+     http://mono.ximian.com/monobuild/preview/download-preview/  
+3. Reboot the VM
+4. cd into /home/mono/wk_mono
+5. su to root (pw mono)
+6. change the plastic start script to use the system mono:
+      Delete this from the first part of the line:
+         "/opt/PlasticSCM/config/mono_setup"
+
+7. Save the file and exit back to the mono user
+8. run "plastic"
+9. Expand the left hand treeview
+10. Select the subitem "xmemtest.gcc"
+11. Click on the Branches toolbar icon
+12. In the "Branches View" (right side) window right click on  "/main/task001"
+13. Select "Merge from this branch"
+14. Click the "Find Merge" button
+15. Select the found item
+16. Click the "Merge" button
+17. Click the "Ok" button on the Comments window
+18. The "Merge Tool" will open, close it
+19. Click the "Exit without saving" button
+20. Notice that the "Merge Branch" dialog cannot be clicked on, and that
+there is a blank box drawn on the screen.
+21. You might have to repeat these steps a few times to get it to happen. 
+
+Actual Results:
+See screen shot
+
+Expected Results:
+It should exit back to the Merge Branch dialog.
+
+How often does this happen? 
+Pretty often.
+
+Additional Information:
+This happens with the system mono 1.2.5p3 and not the 1.2.4 included with
+plastic.
