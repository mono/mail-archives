Once you start an application with the debugger, you cannot create new
breakpoints, or remove existing breakpoints.

New breakpoints will be ignored.

Deleted breakpoints will cause the application to halt execution at the
original breakpoint location, but VS will not attempt to debug at that point,
so the app just stops running.
