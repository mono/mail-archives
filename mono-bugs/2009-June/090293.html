Description of Problem:
When I try to run the default VS winforms template remotely in mono, VS just
sits there.

On the linux side, monovs-gui-server prints out:
Cannot open assembly 'bin/Debug/WindowsFormsApplication2.exe': No such file or
directory.

This worked in MonoVS 0.1, so this is a regression.

Marc reports that he is able to run webapps remotely without a problem.


Steps to reproduce the problem:
1. Create a new solution based on the C# winforms template
2. Click "Mono / Run remotely in mono"
3. Choose the VM, click "Ok"
4. Notice nothing happens.


Actual Results:
Cannot open assembly 'bin/Debug/WindowsFormsApplication2.exe': No such file or
directory.


Expected Results:
The app should run


How often does this happen? 
every time
