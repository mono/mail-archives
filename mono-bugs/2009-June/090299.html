When running monodoc with --edit, an exception is raised when I try to view
anything under the toplevel node for the edited documentation (this means
namespaces, classes, members, whatever).  This same exception happens when
viewing some other, system documentation, but only when --edit is used.

I have observed this with Monodoc 2.4, 2.4.2, both with GtkHtml and WebKit.

Reproducible: Always

Steps to Reproduce:
1. Make documentation structure with mdoc update --out=docs
2. Run monodoc --edit docs
3. Try to view the documentation.
