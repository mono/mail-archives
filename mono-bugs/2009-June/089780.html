Description of Problem:

The data in the columns (name, email) on the dbpage1 xsp 1.1 test page are
reversed.  This is a result of the sqlite3 database containing data in the
wrong fields. 

The database is generated during the make process and rusty fixed this earlier
with r121562.  The fix is still there, however, the binary sqlite3 db file that
is in mono-2.4.2 is broken and contains data in the reverse order.

So, either it's getting built incorrectly now or an old binary db is being
slipped into the build environment somehow.
