I had to add the paths "\lib\monodoc" to the presumably automatically generated
monodoc.bat file on Windows.  The batch file looks in the Mono v2.10
installation directory, not in the lib\monodoc directory, where "browser.exe"
is located.

Reproducible: Always

Steps to Reproduce:
1. Install on Windows 7 x64 (don't know about other Windows versions).
2. Run "monodoc.bat" by typing "monodoc" with Mono in your path.

Actual Results:  
An error about "browser.exe" not being found.

Expected Results:  
Browser.exe being launched.
