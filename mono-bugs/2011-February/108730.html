The MJ12 distributed www crawler crashes every time after uploading a finished
work unit back to the project's servers. This only happens with 64-bit Mono,
32-bit version works just fine. Also this didn't happen with Mono 2.4, it
started happening with 2.6 and is still happening with current git head.

There is no stacktrace, the only thing it says is: "Stacktrace: <empty line>
Native stacktrace: <empty line> Segmentation fault (core dumped)". There seems
to be some Apport generated crash dump in /var/crash, please let me know if
you'd like to take a look at it, or if there is any other way I can help
solving the problem.

This bug has been confirmed by many of the project users with many different
setups. It's always the 64-bit version of Mono that crashes.

Reproducible: Always

Steps to Reproduce:
1. Use a 64-bit Mono
2. Run the client
(http://www.majestic12.co.uk/files/mj12node/mono/mj12node_linux_v170b2.tgz) -
you need to create an account though.
3. Let it run until it finishes a work unit and starts to upload it back to
base.
4. The client crashes right after finishing the upload.
Actual Results:  
Crash.

Expected Results:  
Continue working.
