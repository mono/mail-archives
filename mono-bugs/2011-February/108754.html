The device logging doesn't automatically connect to the device when the user
starts an application in debug mode.  This results in the user having to
manually connect the logging console (and go through the device selection
again) before/after they have started debugging.

It would be nice to streamline the whole connection/debug process by :
1.  Select the target device once on first launching the debugger, and allow
the user to change the 'default' device;
2.  Restart a projects 'default' vm image when you start debugging for the
first time ;
3.  If the device is connected or mage is started, then just deploy and run on
the 'default' device.
4.  Connecting the device logging console prior to starting the to be debugged
application.
