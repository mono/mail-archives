--- Comment #3 from Jonathan Pryor <jpryor at novell.com> 2011-02-16 17:16:50 UTC ---
BTW, this is fixed by using `aapt --custom-package`, as suggested in:

http://lists.ximian.com/mailman/private/monodroid/2011-February/002920.html

As a result, the application package name will be mixed case, but the package
used by the R.java file will be in a fully lowercased package, further showing
that application package name != java package name.
