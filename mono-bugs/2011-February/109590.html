--- Comment #1 from Sebastien Pouliot <spouliot at novell.com> 2011-02-26 17:24:56 UTC ---
This sounds a bit confusing to use properly - at least as a runner option.
However you can already do something similar with gd2i.

E.g.

1) run gendarme *without* an ignore list to generate an XML with all defects

2) run gd2i (on the XML) to produce a *new* .ignore file

3) compare your existing ignore file with the one generated with gd2i (or
simply start using the new ignore file if you had 0 defects)

You could try experimenting with this and report how easy/hard it is to the
mailing-list. I feel it's pretty close but likely not (yet ;-) a perfect
solution.
