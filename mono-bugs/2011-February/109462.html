A malformed enum definition can crash the gmcs compiler.
In the example below, a dot replace the comma.






Reproducible: Always

Steps to Reproduce:
The program was created and compiled under monodevelop.

This little program reproduce the problem.


using System;

namespace TestEnum
{
    public enum RefreshType : int
    {
        Cyclic                = 0,
        Delta                = 1.  // dot at eol
        CyclicAndDelta            = 2
    }
}

The command line is :
/usr/bin/gmcs /noconfig
"/out:/home/abeguin/Projects/TestEnum/bin/Debug/TestEnum.exe"
"/r:/usr/lib/mono/2.0/System.dll" /nologo /warn:4 /debug:+ /debug:full
/optimize- /codepage:utf8 /platform:x86 "/define:DEBUG"  /t:exe
"/home/abeguin/Projects/TestEnum/Main.cs"
"/home/abeguin/Projects/TestEnum/AssemblyInfo.cs"
