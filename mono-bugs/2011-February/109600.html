In a simple windows form program a a button the callbacks linked to the click
event and the form's OnClosing event are fired in a different order with
respect to .NET and older Mono version when buttons DialogResult propery is set
to DialogResult.Cancel.



Reproducible: Always

Steps to Reproduce:
1. compile the attached sample: gmcs  TestWF.cs -r:System.Windows.Forms.dll
-r:System.Drawing.dll

2. run the sample: the program invokes automatically the button and exit
immediately writing on the console the calling sequence 

Actual Results:  
OnClosing Called
OnOkClick Called
6


Expected Results:  
OnOkClick Called
OnClosing Called
4
