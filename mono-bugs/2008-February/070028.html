Tested in SVN and in 1.2.6.

In MSFT MonthCalendar, when MaxSelectionCount is greater than one, one can
choose the selection date range by dragging the mouse (click on start date,
release on end date -- also works end->start).  This doesn't work in Mono, I'm
not sure  whether it's meant to.

The range can also be set by clicking on the start date and shift-clicking on
the end date.  So that's the workaround.

Repro app on bug 363648.  First set the range count to e.g. 5 (and hit the
button), then try dragging between two dates.
