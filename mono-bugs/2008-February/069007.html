I didn't managed to get much useful information out of gdb, but that's probably
because I have very little knowledge of it ...

However, I did managed to create a "simpler" repro for the issue.

To reproduce:
1. extract the attached gzipped tar archive.
2. run 'make keygen'.
3. run 'make xsp'.
4. open another terminal session.
5. run 'make run-test'.

Now, you probably will not be able to reproduce the assertion from the first
try. So, to run the repro again:

1. Switch to the terminal session where xsp is running.
2. Press Enter to close xsp.
3. Run 'make xsp'.
4. Switch to the "test" terminal session.
5. Run 'make run-test'.

You may need to repeat these steps a few times to get the assertion error.

Just running 'make run-test' multiple times will not allow you to reproduce the
problem. You really need to restart xsp in between runs.
