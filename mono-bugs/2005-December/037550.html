+Summary: [2.0] XmlTextWriter.WriteStartElement should allow prefix with zero-length namespace
+
+In .NET 2.0, XmlTextWriter.WriteStartElement allows a prefix to be 
+specified if the namespace is a zero-length string.
+
+In that case, the prefix will not be output though.
+
+Note: An ArgumentException should still be thrown if a prefix is 
+specified and the namespace is null.
+
+I'll add a test case named "WriteStartElement_Prefix_EmptyNamespace" 
+(marked NotWorking) to XmlTextWriterTests.cs that allows you to reproduce 
+this issue.
