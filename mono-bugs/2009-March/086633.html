This was all observed with 0.20 but may exist on older versions.

Monsoon has various exceptions when it imports a torrent file that has
read-only permissions.  This happens pretty much all the time because firefox
removes write permissions from files that it downloads to /tmp, so the normal
usage of clicking on a torrent in FF throws exceptions.

When you first grab a torrent things work fine but if you try to grab the same
one again it throws an exception because it's read-only.

If it's just the first time you've downloaded then you'll see the exception if
you try to remove the torrent.


It might be a good idea to rename the torrents when they are placed in
config/monsoon/torrents maybe to be to a hash or something so that two
torrents with the same name but different contents won't conflict.
