Description of Problem:
Publishing a precompiled website to one virtual path and trying to serve it
from a different virtual path fails on xsp2

Steps to reproduce the problem:
1. Publish BlogEngine.NET (Ensure Virtual path is still configured as default,
BlogEngine.Web)
2. Serve the web site from xsp2 with the / virtual path


Actual Results:
This is a marker file generated by the precompilation tool, and should not be
deleted!

Expected Results:
The web site content

How often does this happen? 
every time

Additional Information:
Running the same application with Visual Studio's ASP.NET Development Server
will run as expected.

Running the same application on xsp2 from the /BlogEngine.Web virtual path will
run as expcted.

This is particularly a problem with Web Application projects published with Web
Deployment projects, as neither of these projects seems to be allow the
developer to configure the virtual host the application will be precompiled to.
