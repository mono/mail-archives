--- Comment #1 from Miguel de Icaza <miguel at novell.com>  2009-03-20 09:33:47 MST ---
We should not expose in /usr/bin anything that we would not expose ourselves as
part of Mono.

All of the helper stuff that we compile should remain in our install directory
and we should probably wipe them from /usr/bin

Additionally let us think about things that should not even be part of the Mono
distribution on OSX: it seems like gtester, gtester-report, gtk-demo,
libgalde-convert, msgfmt, msgmerge should not even ship at all.   We need it at
build time for glib, but we should not install those and we likely do not need
docs, man pages, header files, static libraries and all the other junk.
