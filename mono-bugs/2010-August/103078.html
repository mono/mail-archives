I run the VS2010 MonoDroidApplication template application on my HTC hero
device.  The black activity screen is displayed for 4-5 seconds, then gets
moved to the background, although it is still running (Activity1 still appears
in the task switcher).  When I bring it back to the foreground, the same thing
happens.  The Button control is never displayed.

Reproducible: Always

Steps to Reproduce:
1.Create a new MonoDroid project in VS2010.
2.Connect HTC hero device.
3.Start project without debugging on the device.
Actual Results:  
Activity screen displayed for 4-5 seconds, then gets backgrounded.

Expected Results:  
A clickable button should be displayed on the activity window.

The device is a rooted Orange UK GSM HTC Hero, running a custom Android 2.1 ROM
(VillainROM 12.0.0, kernel version 2.6.29)
