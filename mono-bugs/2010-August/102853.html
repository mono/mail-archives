Building mono-2.6.7 from source download on a system that has an existing Mono
2.4.2.3 installation and in an environment where MONO_PATH is defined.
The 2.6.7 build is to replace 2.4.2.3, so is configured the same way:
/configure --prefix=/usr --with-large-heap=yes
A make in the top level mono-2.6.7 directory fails on testing mcs, and monolite
is also not usable.
After examination of bug reports, the make command was changed to:
EXTERNAL_MCS=false EXTERNAL_MONO=false MONO_PATH="" make
..and the make completes successfully.

Reproducible: Always

Steps to Reproduce:
1. Configure an environment as described (existing mono, MONO_PATH set)
2. Download and unpack mono-2.6.7
3. Attempt a build



Suggest, as a fix, that the mono-2.6.7 ignores any existing MONO_PATH
