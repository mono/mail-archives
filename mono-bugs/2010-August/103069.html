Ther German Edition of VS.NET does not show the MonoDroid Project template.

The workaround for me was:

1. Go to C:\Programme\Microsoft Visual Studio
10.0\Common7\IDE\ProjectTemplates\CSharp\MonoDroid and create a copy of the
"1033" directory and name it "1031" (for German).

2. Delete the C:\Programme\Microsoft Visual Studio
10.0\Common7\IDE\ProjectTemplatesCache directory and recreate it.

Please note that all other (Microsoft) VS.NET plugins also have the "1031"
directory on my setup, sometimes in addition to "1033" or other directories.
So,  the monodroid installer probably should at least detect the locale of the
computer and create an appropriate copy. There probably is a more correct way
to deal with this.
