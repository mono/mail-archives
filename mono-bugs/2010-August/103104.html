i) In the Application tab in a Monodroid project properties window, there
doesn't appear to be a scrollbar for accessing the rest of the property page if
the window size is small.

ii) In addition, if VS2010 is restarted and the properties page is loaded
automatically, small scrollbars appear, but the majority of the property page
is still inaccessible.

Reproducible: Sometimes

Steps to Reproduce:
For i):
1.Open a Monodroid project in VS2010 and resize the window to make it fairly
small.
2.Right click the project in Solution Explorer -> Properties -> Application Tab

For ii):
3.Close the IDE with the properties page still open, restart it and reload the
Monodroid project.
Actual Results:  
For i) there are no scrollbars in the properties window.
For ii) there are small, but useless, scrollbars.

Expected Results:  
Scrollbars should appear to make the entire properties page accessible.
