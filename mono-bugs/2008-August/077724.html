Atsushi, as expected MS is not going to fix this bug:

"Hello, Thank for reporting this issue. You are right that we should not allow
moving an attribute from one element to another like this. However, we have
decided not to fix this problem as they might be users that are actually
relying on this behavior. If we start throwing exception, their app would break
when they upgrade to the next version of the .Net Framework. If you want to add
an attribute to multiple elements, please clone it so each element has its own
copy. I hope you understand our reasons why we are not fixing this. Thank you,
-Helena Kotas, System.Xml Developer"

This makes sense from their point of view. Should we update our implementation
accordingly for the sake of compatibility?
