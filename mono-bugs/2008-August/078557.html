Until this bug is fixed, I put together a patch for fixing this in gbrainy.

What it does is when the locale is Invariant (the user's locale is not
supported) it calls localeconv Posix call to get the locale parameters and
changes them in the running thread. For gbrainy I only need the decimal
separator but other's many need to extend this for reading dates and so on.

Attached the patch just in case helps other people.
