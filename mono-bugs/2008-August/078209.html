--- Comment #6 from Marek Habersack <mhabersack at novell.com>  2008-08-19 04:56:12 MDT ---
I am unable to initialize the packaged version (2.2.4.6.b) on OpenSUSE 11
because of some postgresql errors during MojoPortal setup (and the rpm doesn't
contain the sql initialization script), but the issue is that the
mojoportal.mds doesn't reference the CSSFriendly.dll (found in the _libs/
subdir of the top-level mojoportal source directory) and thus it isn't copied
to the Web/bin/ directory on compilation.
The latest MojoPortal version (2.2.6.8) correctly references the assembly
mentioned above, so I suggest upgrading the package to that version. I've
tested 2.2.6.8 with both trunk and the 2.0 branch and it works correctly.
I think we can consider this bug as fixed since copying the CSSFriendly.dll to
the correct place fixes the original issue.
