+Summary: Monodoc Gtk GUI, preview steals focus when editing text
+
+Description of Problem:
+
+I use monodoc 1.1.9 through the Debian/sid mainline on IA32.  When editing
+documentation, the updated text is immediately displayed in the preview
+window below the edit window.  This steals the focus from the edit window,
+making any edit frustrating, as after two or three typed characters the
+cursor is lost and one has to re-select the edit window to regain it.
+
+Steps to reproduce the problem:
+1. In monodoc-gui (Gtk# GUI), click on any [Edit] label within the
+documentation, so the edit window appears.
+2. Try to type one sentence into the text edit window.
+
+Actual Results:
+
+Focus stolen, no further text entry is possible except by reselecting the
+text edit window.
+
+Expected Results:
+
+Focus stays in the edit box.
+
+How often does this happen? 
+
+Always.
