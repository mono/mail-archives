Description of Problem:

MonoVS VMware image w/monovs-server-0.2.2583-1.1
Windows Vista x86_64 w/monovs-0.2.2604.msi

Visual Studio will crash when you stop debugging using the Window's mono tray
icon after a missed breakpoint.

After following the steps in bug 518522, if you click "stop" in the mono
Windows tray icon, VS crashes with a dialog saying that Microsoft Visual Studio
has experienced a problem and will restart.

Steps to reproduce the problem:

   0. Use attached BlogEngine.NET zip.
   1. Open the BlogEngine solution
   2. Set a break point in Default.aspx.cs:16 (protected void Page_Load(object
sender, EventArgs e))
   3. Click "Mono / Debug Remotely in Mono"
   4. Select the host to run it on
   5. Break point does not get hit.
   6. Close browser window
   7. click "stop" in Mono's Windows tray icon



Actual Results:

VS crashes

Expected Results:
A clean shutdown of remote debugging.

How often does this happen? 
everytime.
