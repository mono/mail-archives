FYI: I prefer marking tests as NotWorking instead of marking them as ignored.
This is because I do not exclude NotWorking tests when I run our test suite on
MS (while marking a test ignored disables it on both Mono and MS). That way I
can spot regression/changes in .NET for these tests, and still not run them on
Mono (where they would fail).

Perhaps we should introduce a NotMono category for tests that do not apply to
Mono. These can still be useful as it allow us to track changes in .NET for
stuff that we do not (yet) implement.
