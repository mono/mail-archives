Description of Problem:

When adding a story in Graffit and clicking the "Publish" button, the user is
now presented with an incorrect Javascript confirmation dialog reading:

"Are you sure you want to navigate away from this page?

You will lose any non-saved text

Press OK to continue, or Cancel to stay on the current page."


This should not appear and is a new issue since mono-2.4.2.3rc1.  This started
happening in mono-2.4.2.3rc2 so this is a regression.

Steps to reproduce the problem:
1. Run 'setup_test.sh' version of graffiti located on mono testing lab nfs
share c21:/share/mtester/tests/manual/graffiti/
2. connect to http://your_testing_machine:8083
3. click on the "Write" tab
4 Add a story title and content
5. Click "Publish" at the bottom of the form.

Actual Results:

Confirmation dialog is presented to user

Expected Results:

Story should be submitted without warning of losing data

How often does this happen? 
every time
