Description of Problem:

MonoVS VMware image w/monovs-server-0.2.2583-1.1
Windows Vista x86_64 w/monovs-0.2.2604.msi

MonVS debugger fails to stop on Class Property break point.

Steps to reproduce the problem:
   0. Use attached BlogEngine.NET source code
   1. set a break point on "line 164 of page.aspx.cs "public string AdminLinks"
property.
   2. Start debugging remotely
   3. Log in to blogengine.net as admin, admin
   4. Create a new page by clicking on the "settings" link in the
Administration menu on the right side of the front page
   5. Click the "Pages" tab
   6. Create a new page: Enter title, some content then click "Save Page"


Actual Results:
 Break point is not hit. 

Additional Information:
  This break point is properly hit using VS/.NET
  This is a regression because it worked when I created the test case, which I
believe was for 0.1p4
