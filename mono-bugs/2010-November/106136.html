Testcase for the issue

User-Agent:       Mozilla/5.0 (Windows; U; Windows NT 6.1; en-US; rv:1.9.2.10)
Gecko/20100914 Firefox/3.6.10

Inheriting from ValidationSummary control and overriding Render method without
calling base.Render() results in unworkable ValidationSummary.

The same control works on MS.NET, but does not work on Mono.

Reproducible: Always

Steps to Reproduce:
1. Inherit from ValidationSummary control.
2. Override Render method and don't call base.Render()
3. Use the custom ValidationSummary control and try validating.
Actual Results:  
Custom ValidationSummary does not work at all.

Expected Results:  
Custom ValidationSummary should work same way as on MS.NET

The problem seems to be related to the difference in registering
Page_ValidationSummary array between Mono and MS.NET.
