Description of Problem:
I have attached a project where the debugger fails to start. I have removed
many files to this project in order to nail down the problem without success.
The only workaround I've found is to stop referencing the SharedCode assembly
and use link files instead.

I have tried with two Mac Mini (Intel Core 2 Duo) with the same result. I have
tried on a iPhone device and it works. So it seems to be related to the
simulator.

Components version I use:
MonoTouch v3.1.3
MonoDevelop v2.4.1
Mono v2.8.1
iOS 4.1

Steps to reproduce the problem:
1. Start the attached project in debug mode on the simulator = Exception.
2. Remove a file from SharedCode assembly (ex: BaseClass.cs or MapableMath.cs)
and the debugger is working fine!
