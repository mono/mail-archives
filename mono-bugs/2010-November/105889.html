The bad file is a binary file produced with an old Borland compiler.
Unfortunately, I cannot share it since the copyright does not belong to me.

However, you can get the BadImageFormatException very easily if you project has
managed and not managed files. 

For example, aaclient.dll (version 6.0), 'basesrv.dll' or 'cabinet.dll'
included with Windows Vista produce a BadImageFormatException. 

That's for real binaries, if you have a misnamed file (like a text file) you
will get the exception also.

I hope this helps
