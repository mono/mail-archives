On doing a ./configure on mono-0.16 in Cygwin, I got the uninformative error

  checking for C compiler default output... 
  configure: error: C compiler cannot create executables

On further investigation, I found that

  gcc -mno-cygwin mytestprogram.c

gave the error

  gcc: installation problem, cannot exec `cc1': No such file or directory

while

  gcc mytestprogram.c

was successful.

Therefore, the error was occuring because gcc-mingw was not installed in my copy of Cygwin.  I installed gcc-mingw, and the error went away.

Perhaps, configure should check for gcc-mingw?
