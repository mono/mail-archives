When starting a debug session against the emulator, Visual Studio detaches the
debugger and exits with the following: 

“The program ‘Mono’ has exited with code 255 (0xff)”

The application in the emulator continues to run.

Reproducible: Always

Steps to Reproduce:
1. Get default image from Microsoft for Visual Studio 2010 in a Virtual Machine
at:
http://www.microsoft.com/downloads/en/details.aspx?FamilyID=5e13b15a-fd74-4cd7-b53e-bdf9456855bd&displaylang=en
2. Install Java 6 Update 24, Android SDK, Mono for Android
3. Create new project using template
4. Debug the project without any changes (make sure Output window is open)
5. Click the button about 50 times or just wait. 
6. See message in Output: “The program ‘Mono’ has exited with code 255 (0xff).”
Actual Results:  
“The program ‘Mono’ has exited with code 255 (0xff)”

Expected Results:  
Debugger should not detach.

Tried different editions of Visual Studio 2010, tried both 32-bit/64-bit
operating systems, tried older version of Java, and tried new installation of
development environment. The previous beta worked fine.
