In the latest 4.0 build if you set LLVM and ARM7 for optimisations and you have
Debug or Release Simulator selected, the compiler throws a "mtouch failed with
no output" error.

Reproducible: Always

Steps to Reproduce:
1. Goto Project Options page
2. Select Project Build->Advanced
3. Tick LLVM and ARM7

Build!
Actual Results:  
Compiler throws a "mtouch failed with no output" error.

Expected Results:  
A message saying that this configuration is not allowed.
Or even better would be for the IDE to not enable the LLVM when Simulator is
selected.
