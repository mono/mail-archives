--- Comment #9 from Bart Vries <b.vries at afas.nl> 2011-07-08 08:15:48 UTC ---
After looking at the problem a bit further if looks like it is caused by some
changes in the Socket.BeginConnect function in 2.10.2. There are some changes
committed to the 2.10 branch in git after the 2.10.2 release. Some have to do
with bugs on OS X and various other bugfixes on the async socket api. 

I hope and expect the issues to be resolved after the next 2.10 based release
but I haven't tested it yet. As soon the next 2.10 based release is released I
will post my findings.

See:
https://github.com/mono/mono/commits/mono-2-10/mcs/class/System/System.Net.Sockets/Socket_2_1.cs
https://github.com/mono/mono/blob/mono-2-10/mcs/class/System/System.Net.Sockets/Socket.cs
