My first attempt to use the monodroid preview resulted in the provided
application not being able to be compiled, with an error stating that it could
not find the android sdk.  After trying to set this in vs2010 in the options
and then restarting, the sdk was still not able to be found.

However, I was able to manually update the registry to set:

HCU/Software/Novell/MonoDroid/AndroidSDKDirectory to point at the appropriate
path on my file system.  After making this update, I was able to close and open
the VS project, and it compiled just fine.

As I had used the Settings window in VS to try and set this property, the error
might be more closely related to that value not properly being updated.

Thanks,
Mike
