So, I created a new MonoDroid app project called MyDroidApp.

In my project properties, the package name is generated as
"myDroidApp.myDroidApp".  In reality, if I make a Service class, the package
for that class is "myDroidApp.Service" and not "myDroidApp.myDroidApp.Service".

So, If I reference my service in the manifest (like in a <service
android:name=".Service" />), the service fails to be found.  If I explicitly
use <service android:name="myDroidApp.Service" /> it works fine. 

If I change my package name to just "myDroidApp" and use <service
android:name=".Service" />, my main Activity fails to be found.

So it seems there's some discrepancy here between how Activity's need the
package name and how the manifest completes the path if you use the .Service
shortcut instead of specifying the full myDroidApp.Service package name...


Reproducible: Always
