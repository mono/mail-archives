Description of Problem:

The situation is like this:  1) main thread, Thread-A spawns a WorkerBackground
thread Thread-B.  Work running under the WokerBackground (Thread-B) spawns a
new thread Thread-C.  This new, Thread-C is blocked until Thread-B ends.  

Steps to reproduce the problem:

using System;
using System.ComponentModel;
namespace TestBackgroundWorker
{
      class MainClass
      {
            public static void Main (string[] args)
            {
                  BackgroundWorker thread_B = new BackgroundWorker();
                  thread_B.DoWork += HandleWorkerDoWork;
                  thread_B.RunWorkerAsync();
                  System.Threading.Thread.Sleep(new TimeSpan(0,0,5));
                  Console.WriteLine ("End of Main Thread(Thread_A)");
            }
            static void HandleWorkerDoWork (object sender, DoWorkEventArgs e)
            {
                  BackgroundWorker thread_C = new BackgroundWorker();
                  thread_C.DoWork += HandleWorkerDoWork1;
                  thread_C.RunWorkerAsync();
                  System.Threading.Thread.Sleep(new TimeSpan(0,0,3));
                  Console.WriteLine("end of Thread B");
            }
            static void HandleWorkerDoWork1 (object sender, DoWorkEventArgs e)
            {
                  Console.WriteLine("Begin of Thread C");
            }     
      }

}

Actual Results using Mono 2.8.1 runtime:

  end of Thread B
  Begin of Thread C
  End of Main Thread(Thread_A)
  Press any key to continue . . .


Expected Results Microsoft .NET runtime:

  Begin of Thread C
  end of Thread B
  End of Main Thread(Thread_A)
  Press any key to continue 

How often does this happen? 
  Always
