Description of Problem:

using mono-2.2p1 on opensuse 11.0 x86_64 (and other platforms), 'mdtool
-f:<sln_file>' fails to compile.  This is not only a bug, but is preventing
testing of certain RPMs which are supposed to be compiled using the latest
release of mono before they are tested.  mojoportal is one such RPM.

The same RPM compiles successfully on opensuse 11.0 x86_64 with the mono that
shipped with that distro.

I am including the outpuf from a successful compile on opensuse 11.0 and a
failed compile using the same distro but mono-2.2p1.
