On Mono, the FilePath of a Configuration - as returned by
ConfigurationManager.OpenExeConfiguration
(ConfigurationUserLevel.PerUserRoamingAndLocal) - does not match the path of to
the actual configuration file (as used by CustomizableFileSettingsProvider).

I noticed this for the PerUserRoamingAndLocal, but I think it also applies to
PerUserRoaming.

To reproduce:
1. extract the attached gzipped tar.
2. run ´make´.
