Testcase (sources + binaries)

I don't think this is a regression. I verified this bug is reproducible in Mono
1.2.6, 1.9.1, 2.0 and 2.0.1, on any OS. 

Run the attached testcase.
First, use the menu on the right to trigger the bug (File / Mono TreeView bug).
This process isn't automated but has to be done manually, else the bug isn't
triggered.

Then, a TreeView will be displayed. With Mono, the first item of this TreeView
will initially not be displayed.

At this point, if you resize the window and then select TreeView items, the
TreeView is repainted properly, making the proper TreeView items appear one by
one as you click on them.
