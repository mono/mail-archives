Running any mono winforms application application under mono 2.6.4 on linux
with kdrive 1.5.3 xserver where mono process is the only X client at startup
results in no keyboard input being seen by mono.

If another X application (such as xterm or xev) is started _before_ starting
mono all works fine.



Reproducible: Always

Steps to Reproduce:
1. start xdrive xserver
2. start mono winforms app
3. observe keyboard input not working
