I have 3 issues with using build-mingw32.sh when cross compiling mono on linux
for windows. The attached patches fill fix these.

1. build-mingw32.sh includes a copy of glib*.dll, while this is not needed as
mono builds with eglib.

2. I had an CROSS_PKG_CONFIG_DIR without any *.pc files. This causes a build
failure.

3. The build-mingw32.sh script did only work with svn checkouts, not with dist
tarballs.
