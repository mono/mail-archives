The right way to fix this bug requires quite some work:

-Add a MONO_ARCH_MAX_COPY_WORDS with a possible default value.
-Rework all callers of mini_emit_memset and mini_emit_memcopy to not do a size
check and emit a memcpy/memset intrinsic.
-Add a mono_arch_emit_memcpy and mono_arch_emit_memset functions to optionally
perform a backend-optimized copy.
-Change mini_emit_memset and mini_emit_memcopy to check against
MONO_ARCH_MAX_COPY_WORDS and call the backend.


Those changes will take quite some work, but I'll do it anyway. As a bonus,
I'll add support for using SSE for those ops.
