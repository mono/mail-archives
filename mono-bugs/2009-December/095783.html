When I build mono-2.6-RC1 on SLES10-SP1 using the
/configure --prefix=/opt/mono --with-moonlight=no 2>&1 | tee configure32.txt

then run make, make check the check fails with:

378 test(s) passed. 1 test(s) did not pass.

Failed tests:

marshalbool.exe

/tests/marshalbool.exe.stdout contains:

test_0_VariantBool_In_Native failed: got 517, expected 0
Regression tests: 30 ran, 1 failed in marshalbool

/tests/marshalbool.exe.stderr is empty.

I have been told that mono should generate a marshalling stub for this but
looking at the generated code I don't see any.



Reproducible: Always

Steps to Reproduce:
1. tar -jxf mono-2.6.tar.bz2
2. ./configure --prefix=/opt/mono --with-moonlight=no 2>&1 | tee
configure32.txt
3. make
4. make check
Actual Results:  
378 test(s) passed. 1 test(s) did not pass.

Failed tests:

marshalbool.exe

Expected Results:  
complete make check without error.

This has been around for a while, I have not had time to debug it due to other
work taking priority.
