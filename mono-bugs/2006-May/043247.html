+Summary: using generic with wrong number of types should produce CS0305
+
+Here is some sample code in which a generic class is referred to with the
+wrong number of types:
+
+public class MyList<T> {}
+public class MyClass<T> {}
+
+class MyClass
+{
+   public static void Main()
+   {
+      MyList<MyClass, MyClass> list1 = new MyList<MyClass>();   // CS0305
+      MyList<MyClass> list2 = new MyList<MyClass>();   // OK
+   }
+}
+
+Running this on MS.NET 2.0 produces:
+
+C:\Documents and Settings\Scott Ellington\Desktop>csc cs0305.cs
+Microsoft (R) Visual C# 2005 Compiler version 8.00.50727.42
+for Microsoft (R) Windows (R) 2005 Framework version 2.0.50727
+Copyright (C) Microsoft Corporation 2001-2005. All rights reserved.
+
+cs0305.cs(9,7): error CS0305: Using the generic type 'MyList<T>' requires '1'
+        type arguments
+cs0305.cs(2,14): (Location of symbol related to previous error)
+
+Running on Mono ( 1.1.13.6 ) produces:
+
+$ gmcs cs0305.cs
+cs0305.cs(9,7): error CS0246: The type or namespace name `MyList`2' could
+not be found. Are you missing a using directive or an assembly reference?
+Compilation failed: 1 error(s), 0 warnings
+
+I think Mono should give an CS0305 error.
