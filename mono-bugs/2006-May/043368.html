+Summary: ListView: bring back file selection when it got focus.
+
+This is related to item 4 of bug #78446.
+
+When ListView lost focus, it makes item selection invisible (i.e. it looks
+as if nothing were selected) correctly. However, when ListView got focus
+again, it does not bring selection back.
+
+Steps to reproduce the problem:
+1. Run FileOpenDialog.
+2. Select a file from the file list view.
+3. Push tab key.
+4. Push shift+tab key.
+5. Push cursor so that the file selection should move.
+
+Actual Results:
+
+The file selection is moved, reflecting the selection at the bottom
+"filename" textbox, but it is not visible in the file ListView.
+
+Expected Results:
+
+The ListView should also bring back the file selection and thus the
+selected file is highlighted.
