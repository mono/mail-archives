+Description of Problem:
+
+When using ReadKey () to read a single keypress, it works, but a 
+subsequent call to ReadKey () reads the same keypress.  ie: each key 
+pressed causes 2 ReadKey calls to return with that key.  (Purely 
+speculation but perhaps both the KeyDown and KeyUp or KeyUp and KeyPressed 
+are both putting the key pressed into the buffer?)
+
+Steps to reproduce the problem:
+1. Put the following code into contest.cs:
+----------------------------------
+using System;
+
+namespace A
+{
+	public class B 
+	{
+		public static void Main ()
+		{
+			Console.WriteLine ("line1");
+			Console.WriteLine (Console.ReadKey ().Key);
+			Console.WriteLine ("line2");
+			Console.WriteLine (Console.ReadKey ().Key);
+			Console.WriteLine ("line3");
+			Console.WriteLine (Console.ReadKey ().Key);
+			Console.WriteLine ("line4");
+		}
+	}
+}
+----------------------------------
+2. Compile: gmcs contest.cs
+3. Run: mono contest.exe
+
+Actual Results:
+- Console displays "line1" and waits for input.
+- After any key is pressed (lets say "A"), console 
+displays "A", "line2", "A", "line3" and again waits for input
+- After any key is pressed (lets say "B"), console displays "B", "Line4"
+
+Expected Results:
+- Console displays "line1" and waits for input.
+- After any key is pressed (lets say "A"), console displays "A", "line2" 
+then waits for input
+- repeat previous line 2 more times
+
+How often does this happen? 
+
+Every time.
