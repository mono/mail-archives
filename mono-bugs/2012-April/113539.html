Since the 4.4.2.3-28.el5_8 security update to rpm, it is impossible to check
signatures on mono-addon-core and other packages.  This causes createrepo to
fail to include mono packages in my custom distribution.

Reproducible: Always

Steps to Reproduce:
1. Download
http://ftp.novell.com/pub/mono/download-stable/RHEL_5/x86_64/mono-addon-core-2.6.7-6.1.x86_64.rpm
2. Update to latest security updates to rpm
3. Run rpm --checksig mono-addon-core-2.6.7-6.1.i386.rpm
Actual Results:  
[mbehrens at cb-builder rpms]$ rpm --checksig mono-addon-core-2.6.7-6.1.i386.rpm
error: mono-addon-core-2.6.7-6.1.i386.rpm: rpmReadSignature failed: sigh load:
BAD
