Marc, there's something you can do here (apart from adding the 
/etc/xsp/2.0/applications-available/blogengine.webapp file to the report):
compile and install mod_mono with the '--enable-debug' option for configure.

That will sent a log of debugging info to the apache error_log that might help
tracking down the restart problem. Also, let us know if the PIDs of the
mod-mono-server processes change after you run restart (they should).
