--- Comment #8 from Andrew Jorgensen <ajorgensen at novell.com> 2010-09-29 16:22:28 UTC ---
I've just reproduced this bug, but we're going to have to change the title. 
What we know is that without IOMAP BE doesn't show the links in the admin panel
that it should (Categories, etc.).  What we're seeing here is that sometimes
even with IOMAP set (at least in the environment) that bug is still showing.

I verified that mod_mono (or rather apache) is in fact passing MONO_IOMAP=all. 
This can be verified by finding the mono process for your app and cat'ing
/proc/<PID>/environ.

So the problem is not in mod_mono.  It's possible that mono is not activating
IOMAP as it should (is there a way to verify that?) or it's possible that some
other (maybe additional) bug is showing here in BE.
