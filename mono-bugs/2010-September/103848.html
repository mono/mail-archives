Error and warning messages are missing from the Visual Studio tasks window;
that is if the C# compilation succeeds but MonoDroid does not.

Reproducible: Always

Steps to Reproduce:
1. Implement an abstract class.
2. Don't override abstract members.
3. Build
Actual Results:  
Errors are visible in the output, but not errors window.

Expected Results:  
Errors are visible in the output and errors window.

This is easy to implement using the format described by an MSFT employee:
http://blogs.msdn.com/b/msbuild/archive/2006/11/03/msbuild-visual-studio-aware-error-messages-and-message-formats.aspx
