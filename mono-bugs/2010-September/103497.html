R.cs should be clearly marked as generated by a tool.  At a minimum, this
should be in the header text at the top of the file (with a clear warning that
users should not edit the file and changes will be overwritten.)

It would also be nice if the file would treated in a similar fashion to other
generated files in Visual Studio.  For instance, could the file be renamed to
R.designer.cs or something similar?  Does the file need to exist in the project
root?  It seems like it would make more sense to have it in the Resources
folder.
