It varies a lot. Sometimes it's just a few minutes, sometimes it's a few hours.
Some other users have reported it to happen very often though.

You can register via a web GUI: start the client, press S to start the web gui
if it's not already started, then point your browser to http://localhost:1088.
Please note that (depending on settings) the client will use lots of bandwidth.
My settings are max bandwidth ~50 000 kbits up/downstream, 250 workers, 150 max
open URL buckets. This results in 8 000 - 18 000 kbits downstream bandwidth
usage, it depends a lot on the quality of the currently used URLs. Also please
note that if you use 64-bit Mono you'll face a crash every time a completed
bucket is uploaded back to the project servers, there's a different bug report
about it open.

Since it seems that the point where the crash happens is known, would it be
possible to add some debug code around it? I could patch my Mono build to add
such stuff if it helps.
