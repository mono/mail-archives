Description of Problem:
After Graffiti CMS is run with XSP, the runtime says it "terminated in an
unusual way."

This however doesn't seem to affect graffiti at all (thus the severity of the
bug is just normal).

This was found in Mono 2.2p3.


Steps to reproduce the problem:
1. Unzip the graffiti zip file
2. Run xsp inside the Graffiti\Web directory
3. Access the graffiti web page (you don't have to login, just access the main
page)
4. Hit <enter> in the xsp command prompt to shut down xsp
5. notice the error on the xsp command prompt


Actual Results:
**

ERROR:mini.c:10951:mono_get_lmf_addr: code should not be reached



This application has requested the Runtime to terminate it in an unusual way.

Please contact the application's support team for more information.


Expected Results:
no error.


How often does this happen? 
every time.
