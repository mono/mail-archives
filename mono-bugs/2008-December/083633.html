Following the icon creation guidelines for Windows XP
(http://msdn.microsoft.com/en-us/library/ms997636.aspx), I made an icon file
that includes 9 versions of the icon, 3 for each resolution (16x16, 32x32 and
48x48), 3 times for each color definition (16 colors, 256 colors, and 32-bit
colors).

On the main program window, Mono will display the 16-color icon, while it
should show the 32-bit ("true-color") one (which is what the MS .NET framework
does).

I've noticed this bug on Windows XP with Mono 2.0.x and 2.2.x. However, on
Ubuntu with Mono 1.9.1, the icon is as expected.
