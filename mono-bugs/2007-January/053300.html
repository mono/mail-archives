+Hi,
+
+ I am trying to package db4o in Debian (it is taking me ages from some
+reason or other). I am with version 6.0.200, which uses Cecil and
+Cecil.FlowAnalysis. As the latest cecil tarball available (0.4.3) does not
+contain it, Debian package doesn't either.
+
+ IMHO, cecil tarball should be updated, including FlowAnalysis, which is
+being used in some projects. Including a .dll precompiled is something that
+can have some interest in MS Windows world, but in Linux that actually
+imposes a barrier, as projects include .dll from other projects including
+functionality that it is not released yet.
+
+  I won't make here a manifest about security of included precompiled
+binaries, but this approach is something that is hurting Mono in Linux a
+lot, as packagers find lot of troubles when trying to include some Mono
+packages in their distribution. For example, not including db4o in Debian
+is also holding tangerine, which uses it.
+
+  Thanks
