When an ImageSurface using an array to store the image data is created in
Windows,  the GC eventually moves the array, resulting in an access violation.
Same code works fine on Linux.

Reproducible: Always

Steps to Reproduce:
1. Create an image surface with something like:
    imageData = new byte[w* h* 4];
    surface = new ImageSurface( ref imageData, Format.ARGB32, w, h, 4 * w);
2. Perform some rendering operations on a context created like:
    using (Context cr = new Context(surface))
    {
        // do something with cr
    }
Actual Results:  
The rendering will work fine for a couple seconds, but then throw an access
violation on some call to the context.

Expected Results:  
Continued to render without throwing the exception.

This bug can be fixed by pinning the image data with something like:

gch = GCHandle.Alloc(imageData, GCHandleType.Pinned);

before it is sent to the ImageSurface. While this is an okay workaround, it
should be done inside of Mono.Cairo.
