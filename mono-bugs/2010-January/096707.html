When debugging the VS Designer by attaching to the VS process, the attached
message box is displayed.  However, it does not prevent debugging the VS
Designer.

Reproducible: Always

Steps to Reproduce:
1. Start two VS instances (with Mono Tools for VS installed, Version 1.x)
2. Create a default Windows Application in one
3. Create a new code file and place some kind of component manipulation code
inside
4. Save the project
5. Open the new code file created in the second instance of VS
6. Select Tools > Attach To Process
7. Select the VS instance listed and continue
8. View error message
Actual Results:  
Error message dialog.

Expected Results:  
No error message dialog.
