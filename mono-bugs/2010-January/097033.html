When double clicking an existing .pkgxml file in the VS 2008 interface, then
clicking the Create Package button, something crashes VS (the target remote
machine, in this case, is running openSUSE 11.2 KDE with MonoVS 1.0 and Mono
framework 2.4.2.3 installed [not a VM]).

Reproducible: Always

Steps to Reproduce:
1. Open some VS 2008 project
2. Select Mono > Create Linux Package for Project
3. Fill in the details (see attached pkgxml file)
4. Click "Create Package" button
5. Select non-VM instance of openSUSE 11.2 running MonoVS 1.0
6. Observer VS crash
Actual Results:  
VS 2008 crashed

Expected Results:  
Built a Linux package

Why does Mono Tools for VS require a remote machine running MonoVS to build a
package?
