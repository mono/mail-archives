'mdoc export-html' won't regenerate files if the source files haven't changed. 
This is by design (performance increase).

However, if some of the files HAVE been changed, then it should ALSO update the
corresponding index.html files that use the changed files (e.g. namespace
summary descriptions).  Otherwise you can wind up with <summary/> in a Type.xml
file, but the namespace/index.html isn't regenerated and thus shows 'To be
added' as the Type's summary.
