As far as I can tell the two bitmaps are identical. This match the fact that
there was very few changes in libgdiplus between both releases. In any case the
behavior you described (font substitution) is not implemented in libgdiplus -
any such occurrence would come from outside.

As you get different results I suspect they are due to system changes (e.g.
different versions of fontconfig, cairo....) or configuration changes (e.g.
installed fonts).

Good luck in finding the culprit!
