+------- Additional Comments From frederik.carlier at ugent.be  2007-02-06 10:16 -------
+One last remark. The ECMA specs say the same, that "/" and "\" are
+equivalent. Hence, I think in some way you should see the Linux paths
+containing "\" incompatible with the ECMA specs, and not the other way
+round.
+
+Therefore, in my opinion, Mono's default behaviour (if it is to adhere
+to the specs) should be to support "\" as a directory separator. That
+means that linux file names containing "\" are incompatible with Mono. 
+Then, if somebody would like to use "\" as part of a file name, it's
+up to them to invoke a compatibility layer.
+
+Hardcoding "\" as a directory separator, as referred to in the
+MONO_IOMAP documentation, is not bad design, but 100% compatible with
+the specs.
+
+I was looking for background information on MONO_IOMAP (as to see what
+arguments were raised for the current implementation), but couldn't
+find it. Hence, if I'm repeating arguments, I'm sorry.
