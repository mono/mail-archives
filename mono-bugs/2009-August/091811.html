Description of Problem:
If you close the terminal a winforms is running in on Linux, then you get this
eror:
"Could not create session. Process creation timed out."

and you can no longer run anything remotely in mono. 

Also, this can lead to VS crashing.

This was found in MonoVS 0.3p4.2.


Steps to reproduce the problem:
1. Create a new solution based on the C# Winforms template project
2. On Linux, make sure "run in terminal" is turned on
3. On Windows click "Mono / Run Remotely in Mono"
4. Choose your host and click "Ok"
5. Notice that a terminal will pop up as will the winforms app
6. Leave the winforms app alone
7. Close the terminal windows that popped up
8. Notice that the winforms app closed as well
9. Notice that on windows, the "remote mono" tray icon is still running
10. On Windows, click "Mono / Run Remotely in Mono" again
11. Notice you get the error: "Could not create session. Process creation timed
out."
12. If you don't get this error, try to run remotely again. Sometimes the 2nd
run will actually succeed.


Actual Results:
Error "Could not create session. Process creation timed out."


Expected Results:
No error


How often does this happen? 
often, but somewhat sporadic
