We now allow the user to set a range of ports XSP should use on Linux, so they
can open appropriate firewall ports.  (The default is 8816-8872.)

However, we need to define what should happen if all these ports are in use
when they start a new session.

Right now my guess is we silently fail, so we need a better solution than that.
