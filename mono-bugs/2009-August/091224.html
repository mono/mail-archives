Description of Problem:
Sometimes the VMware image boots into text mode instead of gui mode. This, of
course, means that the monovs-gui-server process never gets started.

This happens very sporadically and is hard to reproduce. However, I was able to
reproduce it and allow ajorg to ssh into the vmware image to see what he could
find.

According to ajorg, it's caused by dbus not starting up properly.


Steps to reproduce the problem:
1. Reboot the VM
2. If the VM boots to a GUI, go to step 1


Actual Results:
The VM boots to text mode instead of GUI.


Expected Results:
The VM should always boot to the GUI.


How often does this happen? 
Extremely sporadically, but it seems to be fairly rare.
