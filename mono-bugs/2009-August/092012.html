Description of Problem:

Manual dependancy field is incorrectly parsed, breaking up RPM package names.
In this case, the name 'apache2-mod_mono' was added to the dependancy
information however, in the created RPM the string 'mod_mono' was the only
thing to show up so we are throwing away portions of the package name when
manually entered.  I'm not sure if this is limited to parts of the string
seperated by a '-' or if other symbols will cause the same thing to happen.
