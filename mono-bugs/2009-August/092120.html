Description of Problem:

We are supposed to be able to toggle the writability of a directory in the
MonoVS file list.

In blogengine, I attempted to change this for the App_Data directory by right
clicking on the directory and selecting "Toggle Directory Writable".  The icon
for the directory changed to a pencil over the directory indicating that the
attribute had been flipped however, when creating the RPM and installing it on
Linux, the directory was not writable.

This bug is related to bug 535787.  A simple world-writable setting is not
acceptable.  In addition to this, we need the option to make such a toggle be
recursive and apply to files as well as directories.  in the blog engine case,
there are several files inside that App_Data directory which need to be
writable as well as the top level directory.
