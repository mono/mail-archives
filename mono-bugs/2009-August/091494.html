--- Comment #2 from Thomas Wiest <twiest at novell.com>  2009-08-12 15:58:32 MDT ---
ajorg said that this was caused by the repo and creds files being owned by an
rpm.

I verified that the files are not owned by RPMs anymore in the MonoVS 0.3p3 VM.


linux:/mnt/qa/smoketests # rpm -qf /etc/zypp/repos.d/MonoVS_SLE-11.repo 
file /etc/zypp/repos.d/MonoVS_SLE-11.repo is not owned by any package

linux:/mnt/qa/smoketests # rpm -qf /etc/zypp/credentials.d/MonoVS 
file /etc/zypp/credentials.d/MonoVS is not owned by any package
