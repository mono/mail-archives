There are a number of areas of functionality in CoreAudio that would be useful
to application developers (and especially game developers) from CoreAudio.

Currently the only options for audio is SystemSound and OpenTk.Audio.OpenAL. 
Although OpenAL is very useful, there are a number of things that it doesn't do
as well as CoreAudio.  There are also numerous reports on the internet of a
buggy OpenAL stack on the iPhone (causing pops/cracks) though I have not
verified this either way.  The SystemSound is good for basic audio but isn't a
good solution for low latency playback or more advanced uses.

In any case I'm sure you plan on binding CoreAudio at some point, so this is
just a little reminder.
