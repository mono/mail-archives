Hi,

This script is designed around the way suse does packaging: that is, packages
on 64bit are named the same as a package on a 32bit install.  But a 32bit
package on 64 bit is appended with -32bit.

You'll have to use a custom script that doesn't query for the package, but just
depends on the actual library.

I'm not sure if it's possible to reliably detect distribution within the
script.

I'll post a patch that may work for you.
