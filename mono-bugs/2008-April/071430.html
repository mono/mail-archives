When an image is used on a ToolStripStatusLabel, the label's text contents are
drawn on the left end of the StatusStrip, rather than in the correct position. 
In the test case provided, the top StatusStrip contains image-free labels that
are drawn correctly, while the bottom StatusStrip contains labels with images
whose text contents are drawn incorrectly.  I tested this in Windows XP, and
the code worked correctly; it only malfunctioned in the Linux environment.
