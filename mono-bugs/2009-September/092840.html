The new writable permissions enhancement has caused a regression in the latest
monovs-0.4p1 (4016)

This bug has been exposed now that we are listing individual files in the file
list when adding writability to the directory or file.

In the case where there are file names or directories with spaces, the full
path should be quoted using double quotes in the file list.  If not, the list
looks to RPM as though there are multiple files listed on each line that
contains spaces.

This causes packaging of a monovs project to fail if there are spaces in any
names, which is quite common on Windows and .NET.
