I think it would handle a majority of cases.  However, I think it would be best
if we could look into setting ownership permissions and users/groups more in
the future.  Changing to wwwrun only addresses web apps.  But no other apps
which need to either be accessible only by certain users or groups or run as a
different user/group.

You would need to be able to set ownership and permissions on a set of files,
not necessarily limited in scope to the one's you've flagged as writable. So,
doing this in the advanced tab probably would not be flexible enough.

I think what we have now is sufficient for this release. I suggest we look into
it further for future release.
