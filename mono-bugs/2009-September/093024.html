[From IRC discussion..]

We do not build RPM's from source.  We build the binaries in Visual Studio,
ship the binaries to Linux, and build an RPM from them.

Thus the only thing useful in the source RPM would be the .spec file.  If the
user really wants this, they can disable temp directory deletion and fish it
out of there.

I don't think it's worth confusing the user by introducing new concepts like
source RPMs and spec files when we've done everything we can to shield them
from these.
