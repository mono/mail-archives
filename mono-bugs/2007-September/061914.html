In r86058.

In the window 'half width' case the behaviour now is that only the label is
ever visible, the progress bar is never seen.  This is an improvement as the
final text is now shown correctly.

The missing feature is that in MSFT during the 'build' process the progress bar
is drawn right-aligned, over the text (and disappears on completion).


In the 'narrowest' case one does not expect to see the progress bar so all is
good there.

(Note that the final "Rebuild All succeeded" label is not visible in Mono but
is with MSFT.  This is NOT a bug -- it is just a slight measurement difference.
 In MSFT this behaviour can be reproduced -- see repro for 325390 where the
window can be made arbitrarily narrow and the label disappears on MSFT too).
