+Summary: in Solaris there is already something named mcs and it lives in /usr/ccs/bin
+
+Description of Problem:
+
+mcs is also the name of a utility in /usr/ccs/bin,
+something to "manipulate the comment section of an object file".
+
+So if the build process tries to use a previously built mono C# compiler
+in order to bootstrap a new C# compiler
+and if it then tries to use /usr/ccs/bin/mcs,
+it will fail.
+
+It's pretty hard to do w/o /usr/ccs/bin in your PATH on Solaris,
+because that's where "ld" and so on are in.
