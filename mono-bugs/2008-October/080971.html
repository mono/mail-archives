The application working under .NET and wine doesn't matter as your problem is
related to mono.

Your problem can be in two places:

-In mono, in which case we have to understand it and fix it.

-In wine, in which case you should report with them.

Since you haven't provide any clues on what application you are running I can't
do anything but ask you to test under windows to assert if it's indeed mono's
fault.

There is, of course, the easy route for you on this case. Report the bugs
related to running your program under mono on linux without wine.

Wine is a fast moving target with forks which makes it a less than ideal target
to develop and test against.

No one on the mono team does any kind of testing under wine, but we can help to
troubleshoot any bugs that wine might have.
