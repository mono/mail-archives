Hello,

according to http://www.go-mono.com/issues.html, each mono application maps
regular files for IPC. This means that whenever a mono application is running,
~/.wapi/shared_* files are updated every 40s (40s on my debian/sid). This is
not nice with power savings and disk usage as this occurs even when
applications are totally idle.

Would it be possible for mono to use another IPC mechanism that doesn't need to
trash the disk every 40s ? Thanks.

(bug against tomboy was http://bugzilla.gnome.org/show_bug.cgi?id=514434)
