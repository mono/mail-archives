Adam, I'm sure that patch will not fix the issue you reported. I suspect the
issue may be caused by CharSet changes in System.Data.Odbc/libodbc.cs.

I've only ever tested Mono's ODBC support with MySQL (Linux) and SQL Server
(Windows), and both continue to work fine for me.

I could prepare a patch that will revert part of our changes, if you can build
Mono from source. The CharSet changes were necessary to fix our unicode support
on Windows, but perhaps they broke our ODBC support on Linux (for some ODBC
drivers).
