Due to the wording of class/Makefile.am, the cs-errors files are always
deleted. This prevents the source from being built twice in a row (i.e. "make;
make clean; make" fails)

Removing reference to cs-errors.zip and cs-errors.source from CLEANFILES fixes
this.

See
http://svn.debian.org/wsvn/pkg-mono/monodoc/trunk/debian/patches/02_keep_cs-errors.dpatch?op=file&rev=0&sc=0
for a downstream packager patch used to fix this bug.
