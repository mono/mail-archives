I don't know of anyone or anything using that lib either - but my concern at a
downstream level is that a dependency IS there (whether it's dumb or not), and
that I need to make that dependency visible to the packaging system (to avoid
random exceptions when users try running their apps)

Within Ubuntu, there are two main types of package - Main are core apps in the
default desktop (and with commercial support available), and Universe is for
community-supported random stuff. libgda2 is pretty much abandoned upstream &
has been moved to Universe - but as Mono is a core framework installed by
default, any dependency of any kind on Universe packages is a no-no. Hence why
I'd like to eliminate that dependency if I can.

http://www.unixodbc.org/ would be where you find an ODBC API (which is in Main,
as it happens), if a transition is seriously considered. If not... is it even
worth bundling this assembly if that few people use it?
