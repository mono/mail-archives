Description of Problem:

Currently we do not set the timezone and region when a user creates a new
appliance from MonoVS.  The defaults are useless to 99% of the users.  We
should look at the current users settings in windows and set those defaults in
the appliance.  Most of the time, the user will be building an appliance that
will be run in their own region and timezone.
