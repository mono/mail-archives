Closing MonoVS, then reopening it while a Visual Studio instance has open
connections to it renders Visual Studio unable to create processes, it does
seem to get reconnected, but pops up the process creation timeout dialog.

Reproducible: Always

Steps to Reproduce:
1. Open Visual Studio
2. Run application remotely in Mono
3. Close MonoVS
4. Close application, if it is still open
5. Load MonoVS
6. Attempt to run remotely in Mono again
Actual Results:  
Process creation timeout dialog shown.

Expected Results:  
Process would be created.
