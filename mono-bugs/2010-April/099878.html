In MeeGo, we need to have a fairly compact Mono, just enough to ship Banshee by
default, but not crippled such that other apps cannot be installed.

There is one critical problem with the mono-data package for openSUSE/SLE (2.6)
- it pulls in mono-wcf, which in turn pulls in winforms:

  mono-data -> mono-wcf -> {mono-extras,mono-winforms} -> libgdiplus0 ->
glib2-devel

This issue was reported by Intel as "libgdiplus0 requires glib2-devel," which
is its own issue (pulling in dozens of other devel packages).

If mono-data did not depend on mono-wcf, the Mono installation out of the box
on MeeGo would be /much/ more compact.

We have done the instrumentation in MeeGo to ensure that openSUSE/SLE Mono
packages run on MeeGo as produced from the openSUSE build service.
