Test case for Uri path bug

User-Agent:       Mozilla/5.0 (Macintosh; U; Intel Mac OS X 10_6_3; en-us)
AppleWebKit/533.8+ (KHTML, like Gecko) Version/4.0.5 Safari/531.22.7

The value returned by Uri.AbsolutePath when an url starts with ".." is
incorrect, and differs from that of MS.NET.

var baseUri = new Uri("http://www.site.com/");
var newUri = new Uri(baseUri, "../js/test.js");

In MS.NET AbsolutePath is /js/test.js
in Mono it is /../js/test.js

Note that the AbsoluteUri property is correct, however.

Reproducible: Always

Steps to Reproduce:
1. Run above code or supplied solution
2.
3.
Actual Results:  
AbsolutePath=="/../js/test.js"

Expected Results:  
AbsolutePath=="/js/test.js"
