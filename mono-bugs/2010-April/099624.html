Beagle on opensuse 11.1 stops indexing with mono-2.6.4rc3.  This is a
regression because it works with the same beagle RPMs with the shipping mono
(mono-core-2.4.2.3-2.3.x86_64)

Two identical machines (snapshots) one which is the shipping mono and the other
one has the latest mono-2.6.4rc3 RPMs installed.

1) stop beagle daemon
2) remove .config/beagle
3) start beagle daemon
4) copy some OOo documents to the desktop
5) allow some time for indexing
6) search for words in documents
7) no indexing is being done on the newer mono
8) The shipping mono VM indexes the documents correctly

There is a stack trace that contains errors relating to
"Mono.Data.Sqlite.SqliteBusyException could not be loaded".  I'll attach it.
