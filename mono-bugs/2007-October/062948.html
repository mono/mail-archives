Version: libgdiplus 1.2.5 src tar package

libgdiplus is needed by System.Drawing and thus needed by ASP.NET applications.
ASP.NET application does not need any X cairo backend. For embedded system it
would be great to have the option to compile libgdiplus (and the embedded
cairo) without X dependencies.

Cairo already support this and it's possible to hack libgdiplus to compile
without X (there are only a few calls to X which can be trivially removed), but
some constants from X headers still cause problems...
