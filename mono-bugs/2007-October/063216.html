--- Comment #1 from Joseph Hill <jhill at arcfocus.com>  2007-10-16 17:49:03 MST ---
Also, when using MONO_IOMAP=all or MONO_IOMAP=case with web applications, the
bin directory must have init-cap casing (i.e., Bin), or else Mono will not be
able to find the assemblies.

This problem can be reproduced by renaming the Bin directory to bin in any
ASP.NET application that works with mono and then running xsp or xsp2 with one
of the aforementioned MONO_IOMAP settings.

It's probably worth noting that although VS 2005 defaults Web Sites to use a
Bin directory, VS 2003 and VS 2005 Web Application Projects will default to use
the all lower-case bin directory.
