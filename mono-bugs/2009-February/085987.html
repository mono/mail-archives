Description of Problem:
If you play audio in the vmware image, it's played too fast (to the point that
you quickly hear a sound, and then it shuts off, even for longer sounds).

This is critical because without this, banshee and moonlight sounds will not
work.

This was found in the Mono 2.4p2 vmware image.

Oddly enough, this command does work (ajorg says it's because it uses alsa
instead of puls):
speaker-test -Dplug:front -c2 -l5 -twav


Steps to reproduce the problem:
1. Run the vmware image
2. try playing a sound in banshee (it won't work).
3. go into the gnome control panel / sound applet
4. play the login sound
5. notice that it goes way to fast (like a 5 second sound takes only 100ms)


Actual Results:
sound is too fast


Expected Results:
sound should work

How often does this happen? 
every time
