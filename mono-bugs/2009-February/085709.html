Service detection works as expected. As soon as "resolving" is switched on for
the mzclient demo or is hooked and triggered in the source, the browser stops
working.

1.) mzclient -p "_workstation._tcp 123 Foo [ Something='true' ]
2.) mzclient -r
3.) Kill mzclient from step 1; browser from step 2 doesn't see it
4.) Start mzclient from step 1 again; browser from step 2 doesn't see it

I have seen this in Windows Server 2003 and Windows Vista x86 (business), both
with Bonjour 1.0.6
