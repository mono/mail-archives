--- Comment #10 from Dominique Leuenberger <dominique.leuenberger at tmf-group.com>  2009-02-12 04:44:05 MST ---
A new roundup of RPMs has been published in Virtualization:VMware
(2009.01.21-11.1)

- The desktop file has been changed to no longer start vmware-user or
vmware-user-suid-wrapper, but a newly introduced vmware-user-autostart-wrapper
- /usr/bin/vmware-user-suid-wrapper is packaged with mode 0755
- /etc/permissions.d/open-vm-tools contains information that
vmware-user-suid-wrapper needs to be set to 4755. This is done by SuSEconfig
(permissions)
- A new script vmware-user-autostart-wrapper has been added.
  It checks for the existence of /proc/fs/vmblock/dev, which is created when
the vmblock driver initializes (this happens in the init script). If the device
node is not available, the script sleeps for some time (time is doubling at per
iteration). 8 passes (so last one waits for a bit over a minute already).

> 4. Add .desktop file to autostart on login
This should not be needed to change, as the vmware-user.desktop is copied to
/etc/xdg/autostart and should be used by all DEs. 

Basically this new RPMs should solve all of your issues reported (except the
vmmouse assignment, but this is handled by sax2. vmmouse is actually part of
Xorg, and not of the open-vm-tools).
