Additional testing shows:
  Original report isn't correct (sorry).  The bit depth in the BMP header IS
correct (offset 0x1C reads 18).  However, offsets 0x26 through 0x2B (pixel per
metre horizontal and vertical) appear to be different, which was causing
another system to incorrectly interpret this as a bit depth issue.
  I believe it IS still an issue, however not exactly as reported.
