+Summary: Bugzilla cleanup / how to use the Priority field

+I've done a little bugzilla cleanup this afternoon.
+There bug priority is now set as follows:
+
+blocker and critical - major build problems which need to be
+                       addressed immediately ("Everything is broken")
+
+major - mcs rejects/miscompiles valid code and this affects a
+        lot of people's work
+
+normal - mcs rejects/miscompiles valid code
+
+minor - mcs fails to report an error or a warning
+        mcs rejects/miscompiles valid code, but this is so exotic that
+        nobody will ever use it.
+
+cosmetic - mcs uses an incorrect/misleading error message
+           everything which is not wishlist, this includes
+           "wishlist items" which have a realistic chance of being
+           implemented anytime soon.
+
+wishlist - a bug which will most likely never be fixed or at least not
+           within any reasonable time; I'm currently using this to keep
+           the "no idea what to do" bugs open
