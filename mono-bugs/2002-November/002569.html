+Summary: Problem with internalmethods 
+
+There is a problem while compiling code that references an assembly that
+has internalmethods declared that causes mcs to generate a lot of debugging
+messages:
+
+Take the following file:
+
+using System;
+using System.Runtime.CompilerServices;
+
+class Gnumeric {
+	[MethodImplAttribute(MethodImplOptions.InternalCall)]
+	public extern static void SetCell (int col, int row, string value);
+}
+
+Compile it to an assembly:
+
+mcs /target:library gnumeric.cs
+
+Then compile a program that mentions it:
+
+class X {
+    static void Main () { Gnumeric x = null; } 
+}
+
+If you compile it, you get a bunch of errors:
+
+mcs b.cs -r gnumeric.dll
