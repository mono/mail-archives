Present in 1.2.6, 1.9 etc.

In MonthCalendar, a selection on the last row remains when the selection is
moved earlier in the month.  See screenshot.  If a repaint of the last line is
forced then the selection disappears.

Repro MonthCalendarInit.exe from bug 363648 comment 2:
1. Start with (max count 7 and) dates 2008-Jan-30 and 2008-Feb-05, do
Set->SelectionRange (or set start, set end).
See the selection over penultimate and last lines.
2. Change start date textbox to 2008-Jan-3 (3rd) and hit Set Start.  See the
two visible selection ranges.
