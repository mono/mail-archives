Description of Problem:
If you go through the steps of test case 842134, it will say that you are up to
date on your Mono definitions, even though you've deleted everything in the
Definitions directory.

This was found in MonoVS 0.1p1.


Steps to reproduce the problem:
1. Go to the Mono Tools install directory.  (Generally Program
Files\MonoVS\Definitions).
2. Delete the Definitions folder.
3. Create a new winforms project.
   1. File
   2. New Project
   3. Visual C#
   4. Windows
   5. Windows Forms Application
4. Mono->Scan with MoMA
5. Go to Tools->Options->Mono->MoMA Settings.
6. Choose "Check for new definitions".
7. Notice a dialog pops up that says that you have the most recent definitions,
even though you just deleted everything out of the Definitions directory.


Actual Results:
Doesn't download the 2.4 definitions again.


Expected Results:
It should download the 2.4 definitions.


How often does this happen? 
every time.
