Right now the Virtual Path setting is auto populated.

I don't think we should auto populate this because:
1) it's not clear to the user that it's happening. So when they go to test
drive their appliance in suse studio, it will give them a "403: access
forbidden" and they won't know what went wrong (miguel ran into this).

2) since this and the suse studio feature are mostly for people who want to
build an appliance, it's very likely that they'll want their app to be on the
apache root anyway.

3) there's no way we can possible pick the proper name that they want to have.
So it's very likely they'll have to edit it anyway.
