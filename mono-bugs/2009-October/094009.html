Description of Problem:

MonoVS 0.7p2 (4221)

Whitespace is not ignored in the credential dialog in MonoVS.

If a user types in a string for the user name or API key with leading or
trailing whitespace, authentication fails.

To reproduce add either leading or trailing whitespace to the user name or API
key in the Suse Studio authentication dialog and press Next.

Leading and trailing white space should be ignored when submitting credentials.
