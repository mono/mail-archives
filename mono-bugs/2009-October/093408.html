Description of Problem:
If the MonoVS VM is set to use VMware's NAT network interface, then MonoVS
becomes extremely slow (3 minutes to run the default winforms app remotely).

This was reported by our users on MonoVS. Daryl Cooper has helped to diagnose
this.

Jhill was able to reproduce this using MonoVS 0.5.

I am currently setting up an environment to try to reproduce this.


Steps to reproduce the problem:
1. Run Windows with VS / MonoVS as the Host machine.
2. Run the MonoVS VM inside VMware as the Guest machine.
3. In VMware, change the MonoVS' network card to use NAT
4. Create a new solution based on the default C# winforms project
5. Click Mono / Run Remotely in Mono
6. Notice the slowness


Actual Results:
very slow


Expected Results:
Should work at normal speed


How often does this happen? 
every time in this configuration
