While in a "Remotely Debug in Mono" debugging session the debuggee sometimes
freezes (eg. in the circumstances described in bug 546743, but also in other
cases). Visual Studio shows the debuggee as "running" at that stage. If I then
select Debug, Break All (or click the "pause" toolbar button) Visual Studio
completely freezes - it no longer responds to keyboard or mouse input.
Attaching a debugger to it shows stack traces in the attached file. The only
way to recover from this is to stop the debuggee process in the Linux VM or
kill the Visual Studio process.



Reproducible: Always

Steps to Reproduce:
1. Debug Remotely in Mono.
2. Wait until the debuggee to freezes.
3. Select Debug, Break All.
Actual Results:  
Visual Studio stops responding to any user input.

Expected Results:  
Visual Studio remains responsive, whether or not breaking into the debuggee
succeeds.

Host: Windows XP Pro x64 SP2, Visual Studio 2008 Pro SP1
Guest: VMware image 0.3.3771, MonoVS 0.5.4128
