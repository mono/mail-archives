Right now, when MonoVS is "running remotely" or "debugging remotely" on
BlogEngine, in the VS status bar, it says:

"Setting up connection to server..."

This stays there for a long time because what's really happening is that
BlogEngine is being copied over to the remote host, and it's fairly large.

So, the user is left with the impression that there might be some connection
issue since it's taking so long.

It would be great if MonoVS changed the status bar in VS to say:

"Copying files to server..."

Or something to that affect so that the user knows what's going on.

What would be even better would be to have percent of progress messages in the
status bar.
