This issue was resolved by modifying variables for the user that launched the
mono app(s).  In that user's .bashrc, the following was added:

export MONO_DISABLE_SHM=1
MONO_DISABLE_SHM=1


Since there was no code fix and no one from the MONO side owns this bug, I
think it can be closed.
