You can't be sure that it is mono problem. It is not always mono bug in such a
situation. For example when a binary (dll/exe) was compiled with a compiler
that was broken and created a broken binary, or incorrectly allowed wrong
sources to be built, then it is not mono that needs fixes.

Anyways my guess is no one is likely to take care of that bug unless you
provide how to build and run it (It is described at terms 4,5,6 and 7 at "How
to make a good bug report" section on http://www.mono-project.com/Bugs ).

Changing component to corlib. I don't know banshee sources but Linq stuff is
not supported and hence it must not have been used, so it is very likely corlib
bug (if it is really a blocker).
