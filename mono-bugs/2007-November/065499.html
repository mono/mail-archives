This is not always reproducible and can be worked around however, I can make it
happen most of the time.

Notepad can start in a state that does not allow text to be input.

To reproduce this, start Notepad, while it's window has focus but before
clicking on it, type some chars i.e. 'asdf'.  Then click on the title bar, do
the same - type some chars.  Then click in the text area and try to type.  At
this point, it's often not possible to input text.  This did not happen in
1.2.5.

The work around I found was to change the size of the text then return to the
document and start typing.  I doubt it has anything to with changing text size
but rather, changing the state of Notepad in some way.

I don't think many people will run into this however, it would be nice to see
why it's happening.
