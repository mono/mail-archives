In MS ASP.NET, the default compiler language is VB while we default to C#.

Should we default to VB, even though we know that it is not available on the
1.0 profile.

If we leave the default to C#, then code running fine on Mono will not work on
MS (since they assume it's VB code).

Using VB as default will just force users that use C# code to explictly
specifiy C# as language, which is a good (and common) practice anyway.
