With the PropertyGrid sample attached -- which logs even property 'get' and
'set'.  (On load see lots of 'gets'). Click in one field -> more gets.  Click
in another field, and along with more 'gets' see a 'SET' of the previous
selected property!  MSFT also does lots of gets but never does a set unless
there was intentional direct user action to make a change.

same value that it read from the same property -- and that good properties will
thus ignore, or 'normal' as it might cause problems with those properties that
react to any value change.
