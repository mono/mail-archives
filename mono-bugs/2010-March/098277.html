The attached code produces different output on .net than on mono.
It seems that xml attributes with namespaces cause the
XmlWriterSettings.NewLineOnAttributes Property to be ignored under .net but not
under mono.

Reproducible: Always

Steps to Reproduce:
Execute the attached code under mono and windows and compare the output.
Actual Results:  
net output:

<?xml version="1.0" encoding="utf-8"?>
<xml xmlns="http://www.lalala.com">
  <Element
    Attribute1="attribute"
    Attribute2="No problem" />
  <Element
    Attribute1="attribute" xmlns:Attribute2="I break things" />
</xml>

mono output:
<?xml version="1.0" encoding="utf-8"?>
<xml xmlns="http://www.lalala.com">
  <Element
    Attribute1="attribute"
    Attribute2="No problem" />
  <Element
    Attribute1="attribute"
    xmlns:Attribute2="I break things" />
</xml> 

Expected Results:  
The same line breaking on both platforms.
