+Summary: Assembly.LoadWithPartialName doesn't work in version 1.0.4 on FC2
+
+Description of Problem:
+On Fedora Core 2 Assembly.LoadWithPartialName doesn't work anymore (in an 
+older 1.0 version it worked)
+
+Steps to reproduce the problem:
+run this code:
+
+using System.Reflection;
+class C{
+  public static void Main(){
+    System.Console.WriteLine(typeof(string).Assembly);
+    Assembly a;
+    a=Assembly.Load("mscorlib, Version=1.0.5000.0, Culture=neutral, 
+PublicKeyToken=b77a5c561934e089"); //works
+    System.Console.WriteLine(a);
+    a=Assembly.LoadWithPartialName("mscorlib"); //doesn't work
+    System.Console.WriteLine(a);
+    a=Assembly.LoadWithPartialName("mscorlib, Version=1.0.5000.0, 
+Culture=neutral, PublicKeyToken=b77a5c561934e089");  //doesn't work
+    System.Console.WriteLine(a);
+  }
+}
+
+Actual Results:
+mscorlib, Version=1.0.5000.0, Culture=neutral, 
+PublicKeyToken=b77a5c561934e089
+mscorlib, Version=1.0.5000.0, Culture=neutral, 
+PublicKeyToken=b77a5c561934e089
+
+
+
+Expected Results:
+mscorlib, Version=1.0.5000.0, Culture=neutral, 
+PublicKeyToken=b77a5c561934e089
+mscorlib, Version=1.0.5000.0, Culture=neutral, 
+PublicKeyToken=b77a5c561934e089
+mscorlib, Version=1.0.5000.0, Culture=neutral, 
+PublicKeyToken=b77a5c561934e089
+mscorlib, Version=1.0.5000.0, Culture=neutral, 
+PublicKeyToken=b77a5c561934e089
+
+Additional Information:
+On Windows XP Assembly.LoadWithPartialName works with version 1.0.4
