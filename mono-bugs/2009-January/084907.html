Description of Problem:

UsingWebBrowser does not accept pasted content from the clipboard.  The pasted
text appears as boxes and is not accepted when clicking the "View" button. 
This is a regression and worked in mono-2.2.


Steps to reproduce the problem:
1. Open UsingWebBrowser
2. Highlight a valid URL that has been typed in an xterm window or one from a
Firefox window so that it is copied to the clipboard.
3. Using the middle mouse button, paste the clipboard text into the text box on
either of UsingWebBrowser's text input boxes (Simple Browser or HTML Doc
Viewer).
4. Click 'Go' or 'View' depending on the form you filled out


Actual Results:
An error dialog is shown which says "Invalid URL" on the Simple Browser or "The
Browser could not navigate to ...." on the HTML Doc viewer.

Expected Results:
The page is displayed correctly

How often does this happen? 
Always.
