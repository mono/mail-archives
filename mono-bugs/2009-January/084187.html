Description of Problem:

mojoportal-2.2.7.9-1.5 fails to access newly created pages on mod-mono/apache2
with mono-2.2rc1 on opensuse 11.0 x86_64.  However, the exact same RPM used on
mono-2.0 works correctly and the same mojoportal used directly with xsp2 works
correctly. 

Steps to reproduce the problem:
1. log in as a user with administrator privileges.
2. click the "add new page" icon (a small page with a '+' sign)
3. create a page with the default values so that it's tab appears at the top of
the main mojoportal site.
4. try to access the new page by clicking on it's tab

A error will occur.   However, if apache is shut down and the same, unmodified
mojoportal installation is run using xsp, then everything works and the new
pages can be accessed fine.

Actual Results:

An error is displayed to the screen:

"We're sorry but we could not find the page you are looking for. Please try the
Site Map.
Home   Site Map"

Expected Results:

The new page should be displayed.
