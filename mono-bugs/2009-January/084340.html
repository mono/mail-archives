When fixing this, make sure not to change Mono's behavior when clicking on a
node with the default color (Color.Empty). In this case, the back color
*should* change when clicked.
Let me know if you'd like me to upload an updated testcase to check this
behavior.

PS: I noticed another little difference between Mono and .NET. With Mono, the
dotted focus rectangle around the clicked node only appears after you release
the mouse button. With .NET, the dotted focus rectangle first appears right
when clicking, before you release the button (but is also still there
afterward).
