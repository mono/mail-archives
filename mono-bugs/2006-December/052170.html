+Summary: [PATCH] Form.Closing event cannot be canceld for non modal forms
+
+Description of Problem:
+
+The behaviour of the Form.Closing event for not modal forms is incorrect. 
+The cancel event cannot be canceled for non modal forms. Under .NET CLR 
+the cancel event can be canceled for non modal forms.
+
+That attached patch fixes the problem
+
+
+Steps to reproduce the problem:
+1. Create a form and register a CancelEventHandler to the Form.Closing 
+event
+2. in the event handler cancel the CancelEventArg e: e.Cancel = true
+3. Open the form and try to close it with a click on the X in the 
+titlebar. -> The form is still closed, the event is not canceled.
+
+Actual Results:
+The event is not canceled, the form is closed.
+
+Expected Results:
+The event should be canceled, the form should stay open.
+
+
+How often does this happen? 
+Alyways.
+
+Additional Information:
+
+The bug is fixed by the attached patch against the latest SVN trunk. The 
+patch has to be applied to the file Form.cs in the directory 
+mcs/class/Managed.Windows.Forms/System.Windows.Forms .
