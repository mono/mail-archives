+Summary: Text Priority in Master Pages ASP.NET 2.0
+
+Description of Problem:
+When using master pages, it is possible to add a default text in a 
+contentplaceholder between beginning and closing tag. If the content pages 
+does not define a content referencing the contentplaceholder, the default 
+is displayed, it is normal behaviour.
+But if the content page defines a content referencing the 
+contentplaceholder, then the default text should not be displayed, but in 
+current implementation, it is displayed.
+
+Steps to reproduce the problem:
+1. Create a master page with a contentplaceholder containing literal 
+content between its beginnig and closing tags.
+2. Create a content page with a content control referencing the 
+contentplaceholder and containing literal content too.
+3. View the content page with xsp2, the 2 text are displayed.
+
+Actual Results:
+The two text are displayed
+
+Expected Results:
+Only the content text should be displayed.
+
+How often does this happen? 
+Every time
