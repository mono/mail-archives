Mono checks the environment variable MONO_TRACE_LISTENER for the default
location to send System.Diagnostics.Debug output.  When debugging in
MonoDevelop, there should at least be an option to set this to display output
in the output console.  In fact, if the variable is unset, perhaps it should be
the *default* behavior to set it to an appropriate value.  It was very hard to
figure out why I couldn't get output when using the System.Diagnostics.Debug
class.
