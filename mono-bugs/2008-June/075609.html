--- Comment #2 from Dan Winship <danw at gnome.org>  2008-06-17 16:00:07 MDT ---
There is a test case in the original report:

> One specific problem this causes is that if you
> set the panel task list to "Always group windows", it will group the
> windows from all MWF apps together into a single group (with no name),
> rather than having a separate group for each app.

ie, right click on the handle of the task list, choose "Preferences", check
"Always group windows", then start two different Windows Forms apps. If the
main windows of the two apps are grouped together in the task list, then the
bug still exists. Looking at
http://anonsvn.mono-project.com/viewcvs/trunk/mcs/class/Managed.Windows.Forms/System.Windows.Forms.X11Internal/X11Hwnd.cs?rev=84264&view=auto
it looks like the bug is still there.

I don't have a test case other than that; I found the bug by looking at the
source code when I was comparing how different X11 widget toolkits handled
window groups and session-leader windows.
