Geoff and I did some digging and discovered that with Firefox2 (does 3 work?),
NPP_StreamAsFile() is not being called for that particular image (but is
correctly being called for all of the others) even though we are getting all of
the proper NPP_Write() calls as well as a NPP_UrlNotify() callback and the end
with the reason arg set to DONE.

Since we rely on NPP_StreamAsFile() being called for our Completed event, the
image never gets loaded.

This seems to be a bug in Firefox, but we may need to work around it if it
becomes a real problem.

We should also check to see if this has been resolved in Firefox3, possibly we
can backport the fix to Firefox2 for SUSE packages.
