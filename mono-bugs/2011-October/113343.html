I'm running an ASP.NET 4 (Debian Mono 2.10.5-1) application.
Globalization configuration is ignored and mono is allways converting output to
UTF8.


Reproducible: Always

Steps to Reproduce:
1) Insert in web.config:

        <system.web>
                <globalization
                    fileEncoding="iso-8859-1"
                    requestEncoding="iso-8859-1"
                    responseEncoding="iso-8859-1"
                    culture="es-CL"
                    uiCulture="es-CL"
                    />
                ...

2) Try test.aspx with this content:

===
<meta http-equiv="content-type" content="text/html; charset=ISO-8859-1" />
áéíóúñ (ISO-8859-1 characters)
===

Actual Results:  
test.aspx allways returns content converted to UTF-8.
If you change the encoding in the browser to use UTF-8, the content is
displayed correctly (but this is not the expected result, since the content
should be ISO-8859-1).

Expected Results:  
Characters should be displayed correctly using ISO-8859-1 on the browser.
