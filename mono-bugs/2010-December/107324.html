--- Comment #12 from Marc Christensen <mchristensen at novell.com> 2010-12-17 21:40:48 UTC ---
An important note.

The bug appears to be not with winforms but with file IO of some kind.  If
Reflector is successfully run in a mono-2.4.3 environment one time, the
preferences file ~/.local/share/Red Gate/.NET Reflector 6/Reflector.cfg gets
written with the mono runtime choice saved there and the opening dialog does
not have to open and instead the main windows opens directly.

Now, that the pref file exists, Reflector successfully runs on all versions of
mono. However, if that file is deleted, then the original bug behavior is seen
where it hangs right after choosing a runtime environment.

So, it appears that it has something to do with opening and saving this
preference file the first time.
