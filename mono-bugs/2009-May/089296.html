> > For cross development we are using the eldk 4.2 toolchain.?

> What is this? Is it based on GLIBC/EGLIBC? Does it support pthreads? Is thread
> local storage supported/enabled in your toolchain?

Eldk stands for embedded linux development kit and is a collection of prebuilt
linux/gnu packages for embedded arm/ppc/mips systems. Eldk is based on glibc.
Glibc version in eldk 4.2 is 2.6. It also supports pthreads. An overview of
eldk 4.2 packages and versions is given as part of the online eldk
documentation http://www.denx.de/wiki/view/DULG/ELDKPackages#Section_3.9.2.
