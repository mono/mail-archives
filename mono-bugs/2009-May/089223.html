Printing doesn't work at all. Print dialog do not list available printers
(tested on openSUSE 11.1 and on Ubuntu 9.04). On Windows printers are listed,
however, options button used to adjust printer settings is missing. If OK is
pressed, application crashes. On Linux cups-pdf printer was installed. On MS
NET everything works fine. Attached are binaries compiled with MS VS 2008 for
NET 2.0.

Reproducible: Always

Steps to Reproduce:
1. Start attached binaries
2. Click on File -> Print menu option
3. Print Dialog appears with described problems.
4. Print on OK ---> application crashes
Actual Results:  
Printing does not work. Application crash

Expected Results:  
Printing is an essential part of most GUI applications. Printing should be
fixed.
