Description of Problem:
If while you're selecting a remote host to run / debug your app on, you check
the "Always use this host", it will continue to use the selected host until you
restart VS.

Once VS is restarted, you'll get prompted again.

If this is the desired functionality, we might want to change it to say
something like "Use this host for this entire session" or something like that.

"Always use this host" to me seems like it should use this host even after I
restart VS.


This was found in MonoVS 0.1p4.


Steps to reproduce the problem:
1. Create a new solution based on the winforms template project
2. select "mono / run remotely in mono"
3. Check the check box "Always use this host"
4. Click "Ok"
5. close VS
6. open VS again, and re-open the project
7. select "mono / run remotely in mono" again
8. Notice that you get prompted to choose a host again.


Actual Results:
"Always use this host" doesn't work after VS is restarted.


Expected Results:
Either change the functionality so that the selected host is always used. Or
change the working.


How often does this happen? 
every time.
