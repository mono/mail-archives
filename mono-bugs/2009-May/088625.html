Description of Problem:
When "Run in terminal" is checked in monovs-gui-server, you can no longer
remotely run apps.

This was found in MonoVS 0.1p4.


Steps to reproduce the problem:
1. Create a new winforms app based on the default winforms project template
2. Run monovs-gui-server from a terminal
3. Right click on the icon and check the "Run in terminal" check box
4. In VS, click "Run remotely in mono"
5. Notice that VS just sits there saying "Setting up connection to server..."
(without "Run in terminal" checked, it pops right up)


Actual Results:
The app never runs remotely.


Expected Results:
It should run remotely, _and_ it should be run in a terminal.


How often does this happen? 
I ran through these instructions twice, and it happened both time.
