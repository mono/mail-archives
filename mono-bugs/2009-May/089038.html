Description of Problem:

BlogEngine.NET fails to run correctly on mono-2.4.1 (the mono we are shipping
with monovs).  Administration menu items fail to be found (404 errors) while
the same code works find on mono-2.4.

Steps to reproduce the problem:
1. use OpenSuse 11.1 x86_64 with monovs-server-0.1.1866-1.1 (0.1p4) installed. 
Make sure that you have installed using the internal repository with zypper so
that you get the required mono RPMs as well.  A better way would probably be to
grab the monovs VMware image.

>From inside the firewall, you should be able to get the VMware image here:

http://c100/software/mono-project.org/monovs-previews/0.1-p4/

1. Untar the attached archive of BlogEngine.NET
2. CD into the directory and run 'xsp2'
3. connect to http://<monovs_server>:8080
4. log in using username/password = admin/admin
5. Select "Settings" from the right side Administration menu

Actual Results:

The following error is displayed:
"Ooops! I can't find the page you're looking for
This one's down to me! Please accept my apologies for this - I'll see to it
that the developer responsible for this broken link is given 20 lashes (but
only after he or she has fixed this problem)."

Expected Results:

An administration settings panel should appear.

How often does this happen? 
Always

Additional Information:
If you run the exact same tar file content on a mono-2.4 machine, it works
fine.
