+Summary: Sometimes fails to pass basic auth credentials to web service
+
+I have a web service, that is protected using basic authentication. My test
+program is occasionally aborting with:
+
+System.Net.WebException: The request failed with HTTP status 401: Unauthorized
+in <0x004c3>
+System.Web.Services.Protocols.SoapHttpClientProtocol:ReceiveResponse
+(System.Net.WebResponse response,
+System.Web.Services.Protocols.SoapClientMessage message,
+System.Web.Services.Protocols.SoapExtension[] extensions)
+
+An ethereal dump shows that no Basic auth credentials are being passed on
+the HTTP request, and the web server is responding with a 401 response. At
+that point I would expect mono to re-send the request with my credentials,
+but instead it just raises an exception.
+
+My code to set the credentials is:
+         Pool p = new Pool();
+         p.Url = "http://hostname/url/to/service";
+         p.Credentials = new NetworkCredential("admin", "password");
+
+(Pool is the service class generated using wsdl.exe)
+
+The odd thing is that (with mono 1.1.7) the first few SOAP requests are
+correctly retried with the credentials after the Web server has responded
+401, but it seems the 3rd or 4th fails with this error (although sometimes
+it all works fine).
+
+Also with mono 1.0.6, only the first request works, subsequent ones
+_always_ failed.
