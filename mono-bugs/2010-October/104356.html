Description of Problem:
Graffiti takes about 5 minutes to login, which causes all the automatic tests
to fail on the page. This is only when running Graffiti from Windows.
Navigating pages after you login takes a long time as well.

Steps to reproduce the problem:
1. Start graffiti: 
cd "C:\Program Files (x86)\Mono*\bin"
xsp2 --root
"c:\Users\rupert\Desktop\mtester\tests\manual\graffiti\Graffiti\Web" --port
8083
2. Go to http://151.155.155.117:8083/ (with the IP address of your vm)
3. Click 'Login' on the lower right hand side of the page.
4. Login with the username 'admin' and password 'mono'.


Actual Results:
The page will eventually login, but it takes a long time.

Expected Results:
The page should login and load immediately.

How often does this happen? 
Always.
