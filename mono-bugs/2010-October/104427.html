namespace Test
{
    using System;

    /// <summary>Documentation Text</summary>
    public delegate void FirstTestDelegate<T>(T obj) where T: Exception;

    /// <summary>test</summary>
    public interface TestInterface { }
}


Reproducible: Always

Steps to Reproduce:
1. build the source snippet with gmcs with doc generation enabled

Actual Results:  
$ gmcs /doc:test.xml /target:library DelegateTest.cs
DelegateTest.cs(8,7): warning CS1587: XML comment is not placed on a valid
language element
DelegateTest.cs(9,22): warning CS1591: Missing XML comment for publicly visible
type or member `Test.TestInterface'
Compilation succeeded - 2 warning(s)

The generated test.xml does not contain the documentation for TestInterface.


Expected Results:  
I would expect no warnings. when compiling and that all documentation is
available in the output xml file.

The error goes away when the "where T: Exception" constraint on the delegate is
removed.
