--- Comment #3 from Jonathan Pryor <jpryor at novell.com> 2010-10-21 16:30:27 UTC ---
As for DocComments, I'm not sure what we can do at this time. 
Resources.designer.cs contents are generated by parsing the output of 'aapt
dump resources', which doesn't provide any additional context (such as the
actual string values), and most resources will be files and the resource name
will be based on the filename (e.g. Resources\raw\foo.txt is Resource.raw.foo).

I can see how doc comments would be nice for strings, but there's no easy way
to grab this short of doing a second parse pass over
Resources\values\strings.xml, which won't be happening prior to 1.0 (if ever).
