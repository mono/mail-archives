Hi Alexandre,

The attached patch removes the requirement of mbas to use the
``Microsoft.VisualBasic'' namespace from the current SVN checkout and
allows using the copy of this DLL that is already installed on the
system (it will only use the system version, if, of course, the build
system is configured to do so).

This patch allows me to do a checkout of 'mcs', edit build/config.make
and then only run make in the mbas/ directory without recompiling the
full class library.
