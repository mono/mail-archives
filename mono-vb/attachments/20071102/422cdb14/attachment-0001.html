<p><b><span style='font-family:"Courier New"'>Program details:</span></b> <o:p></o:p></p>

<p><b><span style='font-family:"Courier New"'>This is based on something I
remember from an OS/2 LUG that implemented this in REXX.&nbsp; The primary form
has a text box to fill in your name, and an &quot;OK&quot; button to mark
yourself as &quot;in attendance&quot;.&nbsp; There is a [readonly]
checked-listbox control with everyone's name, and pressing &quot;OK&quot;
searches the list and checks your name if found.&nbsp; If not found, exposes
(makes visible) a frame with an &quot;e-mail&quot; textbox/prompt and a second
&quot;OK&quot; button -- pressing this button adds you to the list and marks it
as checked.&nbsp; There is also an area on the form devoted to information
about the current meeting [who the speaker is, what the topic is, etc.]&nbsp; A
second form, accessible by right-clicking the member list, allows editing of
user information [deleting duplicate names, changing e-mail address]</span></b><o:p></o:p></p>

<p><b><span style='font-family:"Courier New"'>Behind the scenes is a
(disconnected?) &quot;dataset&quot; with tables for attendees (LUG members),
&quot;meeting information&quot;, and a cross-ref table of meetings and
attendees [who attended what meeting]&nbsp; These are stored (for the moment)
as XML files in the user's home directory [loaded at the start, saved when the
program closes, lost if the program crashes...]&nbsp; Ultimately this will be
replaced by direct access to a mysql or similar database, but for now getting
the program going is the primary concern.</span></b><o:p></o:p></p>
