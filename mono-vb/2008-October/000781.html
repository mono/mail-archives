Hi,

Are you sure you're using mono 2.0? This specific error was a problem some
time ago (with mono 1.2.3 or 1.2.4 I think), I'm unable to reproduce it with
latest version. I know some users have had problems with apache using older
versions of mono installed on the system.

You can try putting that test code in a file, start xsp2 in that directory
and load the file in a browser, and then execute 'xsp2 --version', this will
tell you if you're hitting a wrong mono version.
