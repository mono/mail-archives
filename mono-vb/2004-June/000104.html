NOT ALWAYS - some of them come in pairs with a missing method what shows 
that probably we have a method signature mismatch. I think Jochen already 
said that, but to clarify it a bit further follow the example below...

In the class status page for MS.VB.dll in the FileSystem module we currently 
have

x FileGet(System.Int32, System.Array&, System.Int64, System.Boolean, 
System.Boolean)
+ FileGet(System.Int32, System.Array&, System.Int64&, System.Boolean&, 
System.Boolean&)

this means that we have code like:

FileGet(int x, ref System.Array z, ref long q, ref bool b, ref bool bb)

that must become:

FileGet(int x, ref System.Array z, long q, bool b, bool bb)
