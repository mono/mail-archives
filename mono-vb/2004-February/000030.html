>What exactly is ICU?

These are Open Source libraries donated by IBM, that deal with Unicode and 
Internationalization. We currently link with them because that is a HUUUGE 
piece of code to rewrite and debug, but some new problems are arising that 
indicate we will have to revert to our original C# implementation and 
complete it, but I don't know the timeframe...

>Also did we ever come to conclude on how we'll deal with the C sharp'isms 
>in the mbas code?

We are dealing with them as we go:
-- case-sensitiveness was the first,
-- dumbing down the language so that option explicit/strict off work is what 
really is ahead of us. Fortunately that is just a lot of 'if's and 
additional late-binding code, and most of it lives in 
Microsoft.VisualBasic.dll actually.
