Hello,

     Another point about documentation: one thing that we could really
use is documentation for the method that we define that are not
automatically bound, as there wont be any tool that will automatically
guess what the documentation is.

     Basically, we should provide summary/remarks content for any
class, method and field that is defined manually in
monomac/src/*/*.cs, those are a higher priority than documenting the
methods that are generated from the API definition files
(monomac/src/*.cs)
