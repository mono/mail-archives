Hmmm. This may or may not be interesting to anyone else, but I noticed this
line in Application Output window, in between some of the "Loaded assembly:
...etc" lines. 

Resolved pending breakpoint at
'/private/tmp/source/bockbuild/profiles/mono-2-10/build-root/mono-2.10.12/_build/mono-2.10.12.git/mcs/class/corlib/System/String.cs:431'
to String System.String:TrimEnd (Char[]) [0x0001f].

Since I didn't set this breakpoint (I have never at any stage compiled Mono
from the sources, I just downloaded the binaries) I am wondering where this
breakpoint came from or how it got there.

However the good news is that when I "Clear All Breakpoints" from the run
menu, my problem with execution breaking on this line goes away.

Pretty stupid of me, I suppose, not to try doing this before, but since I
hadn't set any breakpoints, it didn't occur to me that I would need to clear
any... particularly not breakpoints within MonoFramework which I only have
installed as binaries.

Anyway. 
Hopefully I haven't wasted anyone else's time on this.
Go figure!  :-)
