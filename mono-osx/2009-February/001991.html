Duane Wandless wrote:
> Delphi Prism sounds interesting and warrants some investigation though.

The "innovative" part of Delphi Prism, for the OS X perspective, is the 
code templates and IDE support of OS X development it provides. Nothing 
more really - the rest is smoke and mirrors. The bitter pill is the 
reliance on Parallels or VMWare fusion, the requirement to own a license 
for Windows XP (or greater) and the fact that integration with Interface 
Builder is still more voodoo magic than a real solution. IMO, if you 
have one part of the puzzle missing, the deck of cards falls and it is 
no more useful than using MonoDevelop or a C# code project plugin for XCode.
