Hello Wade et al.,

Still being rather new to the Mac it is my understanding that to  
uninstall a piece of software I just drag the bundle to trash.  
However given the nature of Mono and its /bin symlinks we need to run  
an uninstaller script which is (for some reason) located inside the  
original installation package. Having gone through this process  
multiple times it turns out to be rather unhandy...

Today I uninstalled the software Flic4Mac, and it had the nice  
feature of having installed an "uninstaller" package - a normal OS X  
installer package saying install etc. in the GUI but running it had  
the effect of removing the software from the system.

Wouldn't this be an option for Mono, too? My guess would be that the  
existing uninstall script could simply be repackaged into such an  
additional package, to be installed by the existing installation  
package. Such a feature would definately simplify testing and  
developing against different versions of Mono!

Best regards,

Andreas
