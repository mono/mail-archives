If you want X11 backend just add MONO_MWF_MAC_FORCE_X11 environment<br>
variable, it should work.<br>
The compiled gdiplus backend in the package has no carbon support, so if you<br>
want it you&#39;ll have to recompile a few libs and add a dllmap to this new<br>
libgdiplus in your mono config :<br>
<br>
        &lt;dllmap dll=&quot;gdiplus.dll&quot;&gt;<br>
          &amp;lt;dllentry dll=&amp;quot;&amp;lt;pathtoyourlib&amp;gt;libgdiplus.dylib&quot;<br>
name=&quot;GdipCreateFromContext_macosx&quot; target=&quot;GdipCreateFromContext_macosx&quot;<br>
wordsize=&quot;32&quot; os=&quot;osx&quot;/&gt;<br>
        &lt;/dllmap&gt;<br>
<br>
<br>
Libs to be recompiled :<br>
- libgdiplus<br>
- cairo<br>
- fontconfig<br>
<br>
<br>
Don&#39;t forget to force 32 bits compilation as the provided mono package is in<br>
pure 32 bits (and carbon 64 bits is unsupported). You&#39;ll have to patch<br>
libgdiplus for the required libpng support if you want to keep system libpng<br>
1.5, otherwise you should compile and link with your own libpng 1.2. Don&#39;t<br>
