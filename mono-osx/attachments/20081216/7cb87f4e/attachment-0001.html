The answer is not that simple. &nbsp;To create an OSX application that is &#39;native&#39; from an end user&#39;s&nbsp;perspective&nbsp;you want to use Interface Builder to define the GUI. &nbsp;What happens underneath the GUI is where mono comes in. &nbsp;Using libraries such as monobjc or mobjc/mcocoa you can use mono and C# to interact with the NIB files created by Interface Builder. &nbsp;You can have C# code that is shared between a Windows app and an OSX app. &nbsp;But the code that directly interacts with the GUI needs to be specific to Windows or OSX.<div>
<br></div><div>All that said you can write a C# application that will run on Mac and Windows. &nbsp;It will not look native on the Mac though. &nbsp;And you could put this window&#39;s EXE into a folder bundle on the Mac so that it at least invokes natively.</div>
