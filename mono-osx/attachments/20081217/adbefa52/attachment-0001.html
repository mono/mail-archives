We are proud to announce that a new release of Monobjc bridge is<br>
available (<a href="http://www.monobjc.net/" target="_blank">http://www.monobjc.net/</a>). The Monobjc bridge provides the<br>
necessary tools to develop and run .NET applications (written in C#,<br>
<a href="http://vb.net/" target="_blank">VB.NET</a>, etc.) that interact with Objective-C frameworks and libraries<br>
under Mac OS X. Existing Objective-C classes can be used in .NET code<br>
in an almost transparent manner.<br>
<br>
This release brings:<br>- the support of <a href="http://sparkle.andymatuschak.org/">Sparkle</a> update engine<br>- the ability to embed private frameworks when packaging<br>- the abiilty to skip assemblies when searching for exposed types<br>


- new symbols for NSSplitterView and NSWindow<br>
- numerous fixes for wrong constants<br>- a tutorial for Sparkle integration<br>- two sample applications on CoreAnimation and Sparkle<br><br>
Downloads: <a href="http://www.monobjc.net/index.php?page=downloads" target="_blank">http://www.monobjc.net/index.php?page=downloads</a><br>
