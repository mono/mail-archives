<p class=MsoPlainText><span style='font-family:"Arial","sans-serif"'>This
e-mail is to announce the first alpha release of NSDuctTape.<o:p></o:p></span></p>

<p class=MsoPlainText><span style='font-family:"Arial","sans-serif"'><o:p>&nbsp;</o:p></span></p>

<p class=MsoPlainText><span style='font-family:"Arial","sans-serif"'>NSDuctTape
is a new library that serves as a bridge between Apple&#8217;s Objective C
runtime and the CLR. It was designed from scratch to make it as easy as
possible to create Models and Controllers to interact with Views created with
Interface Builder.<o:p></o:p></span></p>

<p class=MsoPlainText><span style='font-family:"Arial","sans-serif"'><o:p>&nbsp;</o:p></span></p>

<p class=MsoPlainText><span style='font-family:"Arial","sans-serif"'>The
library is far from complete, but enough has been done that it is usable for
certain kinds of simple applications. You can read more here: <a
href="http://fallingcanbedeadly.com/blog/archive/2008/02/18/about-nsducttape.aspx">http://fallingcanbedeadly.com/blog/archive/2008/02/18/about-nsducttape.aspx</a>,
or you can download source and binaries here: <a
href="http://code.google.com/p/nsducttape/downloads/list">http://code.google.com/p/nsducttape/downloads/list</a>.<o:p></o:p></span></p>

<p class=MsoPlainText><span style='font-family:"Arial","sans-serif"'><o:p>&nbsp;</o:p></span></p>

<p class=MsoPlainText><span style='font-family:"Arial","sans-serif"'>I welcome
feedback (and patches!), and I hope that others find it useful.<o:p></o:p></span></p>

<p class=MsoPlainText><span style='font-family:"Arial","sans-serif"'><o:p>&nbsp;</o:p></span></p>

<p class=MsoPlainText><span style='font-family:"Arial","sans-serif"'>NSDuctTape
is released under the MIT license.<o:p></o:p></span></p>

<p class=MsoPlainText><span style='font-family:"Arial","sans-serif"'><o:p>&nbsp;</o:p></span></p>

<p class=MsoPlainText><span style='font-family:"Arial","sans-serif"'>--Dave<o:p></o:p></span></p>

<p class=MsoPlainText><span style='font-family:"Arial","sans-serif";color:black'><o:p>&nbsp;</o:p></span></p>

<p class=MsoPlainText><span style='font-family:"Arial","sans-serif";color:black'>P.S.&#8212;I
understand that there are already other bridges in existence, such as Cocoa#
and Dumbarton. I considered using these, but my primary goal is to be able to
reuse models made for other systems (particularly WPF) with a minimum of extra,
Cocoa-specific code. Currently, implementation of a single interface from my
library is the only thing that is required for Cocoa&#8217;s binding system to
work with .NET types, and a very similar interface was defined in Microsoft&#8217;s
.NET 3.5, so I plan to switch to having that as a requirement as soon as it is
included in a relese of Mono (it&#8217;s possible that it&#8217;s already
there; my current install of Mono is a bit out of date&#8230;).<o:p></o:p></span></p>
