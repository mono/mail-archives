I am the lead developer of SDL.NET. For OSX, SDL.NET does not require the
X11 backend.
The upcoming SDL.NET 4.0 does have a SurfaceControl that displays an SDL
surface in a System.Winform.
The X11 backend is required for that case.

The only problem I have encountered with SDL.NET on OSX is mouse/keyboard
input problems.
The demos do not respond to keyboard and mouse events unlike the same code
on Windows and Linux.
Anyone have any ideas?
