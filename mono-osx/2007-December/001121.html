On 29-Dec-07, at 10:07 PM, Miguel de Icaza wrote:

> This version is lighter because there is no need to run the X11  
> server,
> and talks directly to the native windowing system.
>
> This means:
>
> 	* It is lighter, runtime-wise.
>
> 	* It would be easier to integrate with other Carbon APIs if
> 	  the developer requires it.
>
> 	* Opens the door for someone to use some kind of theme API
> 	  (I think its been discussed in the past, but back then there
> 	  were no options really).
>

In fact I cooked up a demo of this at the summit that I can provide  
for anyone so interested in working on this.

> 	* Avoids doing everything through X, so deeper integration will
> 	  be possible.
>

In addition:

Apples X11 implementation has poor clipboard / drag n drop support in  
specific, where we will integrate directly with the apple supported  
apis.  As well as integration with the rest of the OS in terms of dock  
icons, apple+tab; expose, etc etc etc

I'm sure there's more that I'm not thinking of right now, but miguel  
covered most of it.
