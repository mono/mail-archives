Someone should fix that link in the installer's ReadMe file, I guess. The
correct URL of the Cocoa# Wiki is

http://www.cocoasharp.org/cocoaSharp/

For API info, look to Apple's documentation for Cocoa. Their documentation is
for Objective-C and Java, but it's easy to translate to C# once you get the
hang of it.

http://developer.apple.com/cocoa/

> is cocoaSharp a dead project?

> regardless if someone even wants to email me zip's of example code,

There is a dearth of example code, unfortunately. When someone (myself
included) first learns of Cocoa#, the first question they post to the
cocoasharp mailing list (a better venue for your questions than mono-osx, by
the way) is "Where's the example code?" Currently, the only accessible
example code is in the Cocoa# SVN repository. If you check out Cocoa# from
SVN, you'll find a Cocoa# monodoc and other examples. More example code is
needed (and preferably not in SVN, IMHO). There's a page for example code on
the Cocoa# Wiki, but many of the links are currently broken. Tim Davis
recently added some nice tutorials there, so take a look.

Hope this helps,
Ed
