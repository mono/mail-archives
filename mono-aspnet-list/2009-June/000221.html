Robert, thanks for your reply.

My difficulty was that whether or not I installed the 2.0 profile it would
use mod-mono-server2.exe regardless of my setting in mod-mono.conf.  If I
install 2.0 it finds mod-mono-server2.exe but threw up lots of errors and
didn't load properly (I think this is because it was trying to use
mod-mono-server2.exe with the 1.0 libraries).  If I enabled ASP.NET 2.0 it
would work correctly in 2.0 mode.  

This was replicated by reinstalling Ubuntu 9.04 server (AMD64) from scratch,
and doing nothing but installing apache and the relevant mono packages.  The
same problems were immediately present.

The only way I could fix it (it is now fixed) was by uninstalling ubuntu's
mono packages and compiling libgdiplus, mono2.4, xsp and mod_mono from
source.  I think this confirms that it is a problem most likely related to
Ubuntu 9.04 mono packages.
