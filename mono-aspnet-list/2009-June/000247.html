This nice error message typically means that your application is referencing 
to ms entity framework assembly.
This assembly reference is added by default in VS.
You can remove this reference from project.
Also you can apply patch from bugzilla whichs shows the assembly name caused 
exception.

It would be *extremely* useful if this error message will be impoved in mono 
as entity framework assembly is referenced automatically when new project is 
created is VS.
