Is anyone working on a policy that will allow mod_mono to work properly on a
system with SELinux enabled?

The mod_mono page ( http://www.mono-project.com/Mod_mono ) has a link to a
set of instructions for setting up mod_mono on Fedora.  It points here:

http://www.inprose.com/articles/10-enable-aspnet-support-in-fedora-linux.html

The instructions say to disable SELinux.  This isn't going to fly where I
work, so I need to come up with a policy that works.  I'd like to see such a
policy worked into the mod_mono distribution.  Searching the web, I've found
several others stumped by this problem.

I've gone through several iterations tweaking settings, adding local
policies etc., in hopes of finding the best policy/solution.  My notes are
here:

http://devinvenable.blogspot.com/2010/02/modmono-centos-5-64-bit-and-selinux.html

I'd like to get with whomever might be working on this issue and find a
solution that can be added to the mod_mono packages for Fedora and Centos.
