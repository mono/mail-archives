<DIV>Something like this (fixing any typos I've made): </DIV>
<DIV><FONT face="courier new, monospace">cd /usr/local/src</FONT></DIV>
<DIV><FONT face="courier new, monospace">wget <A 
href="http://download.mono-project.com/sources/mono/mono-3.2.1.tar.bz2">http://download.mono-project.com/sources/mono/mono-3.2.1.tar.bz2</A></FONT></DIV>
<DIV><FONT face="courier new, monospace">tar jxf mono-3.2.1.tar.bz2</FONT></DIV>
<DIV><FONT face="courier new, monospace">cd mono-3.2.1<BR>./autogen.sh 
--prefix=/opt/mono-3.2</FONT></DIV>
<DIV><FONT face="courier new, monospace">make</FONT></DIV>
<DIV><FONT face="courier new, monospace">make install</FONT></DIV>
<DIV> </DIV>
<DIV>Then you can use <FONT 
face="courier new, monospace">/opt/mono-3.2.1/bin/mono</FONT> for any apps that 
you want to use Mono 3.2.1. Apps currently using Mono 2.10 will continue to do 
so unless you explicitly change them.</DIV>
<DIV> </DIV>I use <A href="http://www.gnu.org/software/stow/">GNU Stow</A> 
to manage my Mono versions. They're all stored at /usr/local/stow/ in 
directories per version (so /usr/local/stow/mono-3.2.1 for example, similar to 
the /opt paths above). Stow handles creating symlinks in /usr/local/bin to the 
version you want to use by default.</DIV>
