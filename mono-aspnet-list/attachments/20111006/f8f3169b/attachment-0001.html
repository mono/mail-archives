I think you may be confusing the Visual Studio tool with the behavior of a typical web server. Visual Studio allows you to set the startup page for debugging and testing. On the flip side, a web server has configured default documents (typically Default.aspx, index.html, etc). <br>
<br>Alberto has the idea by either checking authentication in the default page and then redirecting you to a login page.  Another thing you can look at is using the built-in Asp.Net authentication/authorization functionality, where you can lock down all your pages except for the login. You can then provide a default redirect for unauthorized users that will bounce to login.aspx regardless of what page they try to visit. <div>
<br></div><div><a href="http://msdn.microsoft.com/en-us/library/ff647070.aspx">http://msdn.microsoft.com/en-us/library/ff647070.aspx</a> will give you a basic idea of how this works in 2.0 Asp.NET.  4.0 is nearly identical minus the few unnecessary modules. </div>
<div><br></div><div>You can set specific permissions on urls using the &lt;location path=&quot;login.aspx&quot;&gt; configuration. You would drop a system.web and authorization to allow all users. </div><div><br></div><div>
For the default authorization, you would simply set deny users=* under &lt;authorization&gt;. </div><div><br></div><div>Hope that helps! </div><div>-Nathan</div><div><br><div class="gmail_quote">2011/10/5 Alberto León <span dir="ltr">&lt;<a href="mailto:leontiscar@gmail.com">leontiscar@gmail.com</a>&gt;</span><br>
