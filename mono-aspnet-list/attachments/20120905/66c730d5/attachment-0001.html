what are the crucial differences of using Visual Studio asp.net on
Windows versus using Mono asp.net on Linux?</pre>
    </blockquote>
    From my experience and from <a
      href="http://www.mono-project.com/ASP.NET">http://www.mono-project.com/ASP.NET</a><br>
    <br>
    Our team did some evangelism for mono year ago and at that point it
    was possible to<br>
    start WebApplication projects and WebSite projects out-of-the-box on
    linux.<br>
    I can remember:<br>
    <ul>
      <li>web site</li>
      <li>web app</li>
      <li>MVC2</li>
    </ul>
    <p>Today I read that some people are running MVC3 and even MVC4. But
      not sure.<br>
    </p>
    <p>From app-types:<br>
    </p>
    <ul>
      <li>Web Forms (no WebParts)</li>
      <li>WebServices (SOAP)<br>
      </li>
      <li>WCF incomplete!</li>
    </ul>
    <p>You'll need to get familiar with Apache, FastCGI and nginx http
      servers, but this is not a biggy.<br>
    </p>
    <p>And now potentially problematic part:<br>
      A lot ASP.net applications are not bare APS.net they use different
      frameworks, libs etc.<br>
      A lot are managed .net and can be run w/o hassle, but (there;s
      always but) there are some<br>
      that are not mono compliant. To see if they are mono-compliant run
      MoMA against them.<br>
    </p>
    <p>But this is moving target, because paople are porting libraries
      and even better MS is putting<br>
      libs in FOSS domain - see Entity Framework which is already on
      mono's github, but not yet<br>
      integrated and tested. I believe EF will be part of mono in
      version 2.12+<br>
    </p>
    <p>For more info see <a
        href="http://www.mono-project.com/FAQ:_ASP.NET">http://www.mono-project.com/FAQ:_ASP.NET</a><br>
    </p>
    <p>So, to make story short: there are no crucial diffs, but "the
      devil is in the details"<br>
    </p>
