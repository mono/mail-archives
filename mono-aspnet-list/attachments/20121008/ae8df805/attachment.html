An update on where we are with this issue!<br>
<br>
The libraries that we are now using are as follows:<br>
mono 2.10.2<br>
fluent nhibernate 1.2 (nhibernate 3.1)<br>
MVC3<br>
<br>
We've spent the past couple of months trying to figure out why our<br>
applications are leaking memory and have found a few worrying issues. I<br>
won't go into details about some of the problems we've seen but will<br>
concentrate on the leaks since they cause the web applications to gradually<br>
grind to a halt.<br>
<br>
I developed a test application (MVC 2 & 3) which contained some simple<br>
javascript to poll the server very quickly. Monitoring this with the mono<br>
profiler we found that it never seemed to be garbage collecting the<br>
CacheItems. However it didn't leak at the same rate as our full web<br>
applications so something else was going on too.<br>
<br>
Finally with a lot of experimentation we have found that if we switch our<br>
web applications to sessionless (sessionState mode="Off" in Web.config) they<br>
no longer seemed to leak CacheItems and we found that running nhibernate in<br>
"thread_static" mode did not seem to leak but "web" did. Since we couldn't<br>
leave it as thread_static for the web applications we switched to our own<br>
simplified CurrentSessionContext implementation.<br>
<br>
With those changes our web applications appeared to no longer leak, but in<br>
fact the applications stopped leaking when running in XSP and as soon as we<br>
deploy to mod_mono they happily start chewing up memory again.<br>
<br>
We are also developing a daemon to run alongside the web applications that<br>
uses the same fluent based data provider as the web application (although<br>
running in thread_static mode) and the memory usage for that appears to be<br>
solid, so we know it is not a problem with our data provider implementation.<br>
All the evidence seems to point towards the System.Web related libraries<br>
(with hints of CacheItem and HashMap thrown in), but we are not familiar<br>
enough with how it all works to even know where to begin looking, and<br>
frankly we've wasted too much time on this already.<br>
<br>
We have been trying to solve these issues for so long now that we're having<br>
to seriously consider deploying to .NET, despite the fact that we've<br>
invested a lot of time in building a linux-based server module for the<br>
daemon and web applications.<br>
<br>
If we are no longer seeing leaks using XSP, is there a possibility that we<br>
could deploy our web application using that instead of mod_mono (by setting<br>
up apache with mod_proxy)?<br>
