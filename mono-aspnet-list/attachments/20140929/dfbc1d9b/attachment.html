<br>
I could never get MVC5 to work on Mono but if you don't use any of the new<br>
features in MVC5 you can just rebuild your app on MonoDevelop referencing the<br>
System.Web.Mvc.dll version 4 from Microsoft and it should work, here's what I<br>
did to get it working:<br>
<br>
1. Download and install MVC4 if you haven't already.\<br>
<br>
2. Open your VS2013 project in MonoDevelop on Linux and change the<br>
System.Web.Mvc reference to the one that ships with MS MVC4. Make sure the<br>
copy local checkbox it ticked.<br>
<br>
3. Change the System.Web.WebPages.* references to the version 3.0.0 that comes<br>
with mono. I copied them from the GAC to a directory on my solution tree and<br>
added the reference from there checking copy local. On the latest version of<br>
MonoDevelop if you add it from the GAC or the framework directory and tick<br>
copy local it will copy a lot of stuff that will break your app.<br>
<br>
4. Change both Web.config files so that all references to System.Web.Mvc use<br>
version 4.0.0.0 and all references to System.Web.WebPages use version 3.0.0.0.<br>
<br>
5. Build and run the solutiion on MonoDevelop. It should work.<br>
<br>
When you scaffold an edit view with VS2013 it will add the css classes to the<br>
Html.EditorFor, MVC4 ignores it so the form controls are not formatted for<br>
bootstrap, my solution was to use jQuery to set the css class. Also VS2013 mvc<br>
template references site.css but the file is called Site.css, you need to fix<br>
this so it works right on XSP.<br>
<br>
The Web.config files that VS created for me are called Web, not web so you'll<br>
probably have to rename yours.<br>
