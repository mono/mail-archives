I mis-stated something in the original post:

The 400 Bad Request response under Mono/Apache 2.2 should read:
> GET /app_name/http://[CAS provider FQDN]/cas/login?service=[referring
> restricted page] 400 Bad Request 

(note the prepended application virtual path)

So you see, it's bungling the URL in Mono in a way it does not in IIS7.
