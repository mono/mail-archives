Am 11.12.2014 um 21:36 schrieb Niels Heinemann:
> what's yout point against mod_mono?

As I understand it, mod_mono works like all other Apache-integrated 
modules, by running all applications in the www-user context. My server 
is a shared hosting server that requires different users to be 
restricted in their own system account. This includes accessibility to 
the other users' home directories.

FastCGI provides a process separation so that the extension can be 
executed in the user's account, together with suexec.
