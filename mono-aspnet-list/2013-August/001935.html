   - Don't copy Microsoft.Web.Infrastructure.dll, Mono has its own
   implementation. If you're using Visual Studio, you can set "Copy Local" to
   False so that it doesn't get copied over when you publish the site. (my
   suggested method of deployment is to do a "Publish" in Visual Studio into a
   temporary directory, and then copy this
   - Are you using Entity Framework? The current released version does not
   work with Mono. Remove references to EntityFramework.dll and
   System.Entity.dll or switch to the latest beta version
   - Last time I checked, I found that The OAuth implementation
   (Microsoft.Web.WebPages.OAuth.dll + DotNetOpenAuth) seemed to break random
   things, and removing it made everything work. Are you using OAuth? If not,
   remove it.

I'd say rather than start from a bloated template with so much stuff you
may not even use, start from the "Empty" template and add the things you do
use (they're all NuGet packages), testing on Mono every time you add a new
dependency. My blog uses MVC 4 and it was working fine with
System.Web.Optimization (which is what is throwing the 404s for you) on
Mono 3.0.4, so it can definitely be used on Mono. I switched to Combres
though.
