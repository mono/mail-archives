It would seem that it is trying to open ole32.dll which is unavailable
on Linux. If it's just SOAP calls, perhaps you can find or create the
schema/XSD for the objects, and send them manually. Mono has good
support for SOAP1.1 and 1.2. Other code for SOAP at
http://servicestack.net/ might also be helpful.
