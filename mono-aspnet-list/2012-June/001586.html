Do you have <clear/> as the first item below the <httpModules> entry in
your web.config?  If not, you may be trying to add a module that the base
configuration already has.  There's a chain of inheritance for those
configurations, starting with a machine-wide one, so a lot of times  you'll
see <clear/> as the first item, then the config will add only the parts it
needs.  (This applies to more than just the httpModules entry.)

If you do have clear, and it's still happening, that very well may be a
bug; you might check the changelog for versions since 2.10.5 to see if it's
identified as being fixed in any of those.
