> > Thanks!  Is there any way to adjust this globally for MonoDevelop instead
> of
> > per-project?
>
> You probably want to set it on the solution instead of the project,
> since it cascades to all child projects unless explicitly overridden.
>
> Edit->Default Policies can be used to edit the settings that are
> applied to new solutions and solutions that don't have these settings.
> When you save one of these solutions, the default settings will be
> saved into the solution, so after that you'll have to edit them on the
> solution properties.
>
> This is because it doesn't make sense for any of these settings to be
> per-user.
