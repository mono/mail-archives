> *.pidb are code completion data files. I don't know if it would be
> worthwhile putting these in version control, maybe if you have a very
> large project. Personally I leave them out.

Clarification: pidb files should never be committed to the VCS.
They're cache files, entirely derived from the source files. Putting
them in the VCS would cause a lot of pain and very little gain (if
any).
