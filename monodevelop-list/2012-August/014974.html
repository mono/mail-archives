You need .NET 4.0 for the latest version of MonoDevelop, but you don't need
MonoDevelop to develop a Mono-compatible application. And you don't have to
make an app that requires .NET 4.0, if you'd rather target 2.0 or 3.5.

Actually, to make a WinForms app, it's preferable to use another IDE,
because MonoDevelop does not have a WinForms designer. I suggest you just
create your application with Visual Studio or SharpDevelop, then test it
with Mono.
