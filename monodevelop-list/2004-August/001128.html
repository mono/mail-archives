> >> I've installed mono 1.0 and monodevelop 0.5 and the related packages
> >> from the Mono red carpet channel on a RH 9 system with Ximian Desktop
> >> 2 also from red carpet, and I'm having a problem: when I double-click
> >> on a .cs file in a project, the file is opened in the Nautilus file

>

> I had the same problem if charater '#' was in the path (e.g.
> \home\user\prj\app\ui\gtk#\filename.cs).

Confirmed, thanks. Removing the # from the directory name cured the
problem.

I submitted it as a bug to ximian, but it's been marked as a bug in
Gnome-VFS rather than a problem in monodevelop itself. No word on when
it'll be fixed.

So, warning to all: don't use # characters in names of directories
containing monodevelop projects...
