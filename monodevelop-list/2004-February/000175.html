I like the idea of replacing the pure C# code with a
wrapper for a native editor (basically because I
wouldn't have to maintain the current one ;-) but, to
be fair, the truth is that the performance of the port
sucks because I did a quick port to try to have
something to start porting #D. I had no clue about
pango in day one and the rendering was complete in day
five. The port uses full layouts for each word instead
of using properties.

The point is that performance is bad not because it is
pure managed code, but because it is a dirty hack
coming from an ignorant programmer (me). So I'd remove
performance from this debate.
