I'm sure MonoDevelop can build SL projects the were initially created
in VS, but that's about as far as you can get.  MonoDevelop has almost
no tooling specifically for working with Silverlight applications.
There have been some attempts at XAML designers and such but you'd
have to figure out a pretty manual work flow to do everything w/a pure
Mac OS X / Mono environment.

Probably the best way to develop Silverlight apps on Mac OS X is run
Windows in a VM and use their free Web Developer Express product.

BTW, the plugin that MS has developed for viewing and using SL apps on
Mac OS X is fully update-to-date with SL 4.  But the plug-in has
nothing to do with developing said apps.
