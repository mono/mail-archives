> Hi All,
> 
>  I guess I am at a point now that I should send the
> code I have been working on for feedback. Please be
> gentle :)
> 
>  Background: This is the initial version of what would
> become generic support for version control in MD. In
> short:
> 
>  * VC is the core generic version control code in MD;
>  * Every specialised (cvs, svn) backend registers with 
>    VC and becomes available to a project.
>  * Depending on which VC you choose for a project, your
>    context menus, project options, etc. are different.
> 
>  The tar file contains a patch and the files I have 
> developed, so it should untar in the MD root directory.
> 
>  Warning: if you want *working* support for version control,
> this is NOT for you - this code is still incomplete. However,
> it contains a skeleton of a CVS backend, which:
> 
>  1) Shows how the VC backends and the VC core will communicate;
>  2) Shows how each VC backend options will be shown in the
>     project dialog;
>  3) Shows the context menu in the project browser pad and
>     how it will communicate with the backends.
> 
>  Note:
> 
>  *) The VC core code is implemented as an Addin, and is quite
>     independent (so far) from the Core code. I did this just
>     to simplify things for me - it could be easily moved into
>     the core code if desired.
> 
>  Things I am planning to start immediately:
> 
>  1) Make each VC backend conditional in the XML files
>     (by the way, thanks Todd for replying to my previous message)
>  2) Add support for saving/restoring the properties set in
>     the options dialog (I accept implementation suggestions on
>     this one :)
>  3) Add more and complete the existing CVS context menus and
>     hook them up to sharpcvslib to see if that works;
>  4) Implement some sort of icon/textual state indications in the
>     project browser tree view, so that the user will know if
>     files need to be updated, checked in, etc.
>  5) Adding a VC browser pad?
> 
>  So if you have comments, suggestions, feedback, etc. This is
> the time.
> 
> Thanks
> Rubens
>
