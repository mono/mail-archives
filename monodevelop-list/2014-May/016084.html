Hello,

First I want to thank everyone involved for releasing Monodevelop 5.0!

Two problems I can see:
The release notes are still missing.

My other concern is that we got another tarball that cannot be used to
build a release.
see for details: https://bugzilla.xamarin.com/show_bug.cgi?id=18883

To avoid this, I am going to set up a server that builds a tarball
every night on master and the release branches, and attempts to build
a package.
That way, we can react quicker to breaking changes in the code, and
make sure that on release day we have a working tarball.

Perhaps the release process should also include a release candidate,
to give the community a chance to spot and fix obvious problems like
these?

Thanks for your work,
  Timotheus
