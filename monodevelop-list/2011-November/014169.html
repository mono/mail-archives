It does appear that the Mono and MonoDevelop landscape has changed a
bit. However, rather than see this as a problem where everyone needs
to go build it themselves, why don't we self-organize some groups to
try to make the situation better. In fact, it could be even better
than it was before.

Mono and MonoDevelop has never had very good distro support, other
than OpenSUSE. Now would be a great opportunity to step in to fill the
void.

What I think is needed is a repository containing the build scripts
for building deploying  distro packages of the latest versions of
Mono/MonoDevelop. I've been working on bringing the IronPython packing
up to latest Debian, but this is a non-trivial task involving many
different layers of stuff.

Count me in!
