Hi,

We have made changes in the last release or two to better auto-detect the
encoding of a file. However it is worth pointing out that not all encodings
are available on all platforms so it will always be important that you make
sure you choose a 'good' encoding. If a file contains text an unrecognised
encoding the only thing monodevelop can do is corrupt your file by
attempting to convert it to UTF-8 (or whatever) from the unknown encoding.
The best solution would be for the user to open the problematic file in the
application which created it (as it can understand the encoding) and then
re-save it under a usable encoding.

Hope this helps,
Alan
