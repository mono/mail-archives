</div>Yes, it is, but you need all the supporting libraries that MD needs<br>
installed into your parallel Mono. Assuming you are using a properly<br>
separated parallel Mono, you can set MONO_GAC_PREFIX in the parallel<br>
environment to make it fall back to another Mono&#39;s GAC:<br>
<a href="http://www.mono-project.com/Parallel_Mono_Environments#Tip:_MONO_GAC_PREFIX" target="_blank">http://www.mono-project.com/Parallel_Mono_Environments#Tip:_MONO_GAC_PREFIX</a><br>
<br>
The reason you can build is that pkg-config (which is used to find<br>
libraries at compile-time) has fallbacks hardcoded:<br>
<a href="http://www.mono-project.com/Parallel_Mono_Environments#BEWARE:_pkgconfig_fallback_behaviour" target="_blank">http://www.mono-project.com/Parallel_Mono_Environments#BEWARE:_pkgconfig_fallback_behaviour</a><br>
<br>
Note also that MD 2.2 can target parallel Monos without having to run<br>
on them. Add your parallel Mono using Edit-&gt;Preferences&gt;.NET runtimes,<br>
then switch the target runtime using Project-&gt;Target runtime.<br>
