    <br>
    You probably mean like in Xamarin Studio exactly Xamarin.Android
    (AKA Mono for Android - M4A<br>
    or even further in history Monodroid)<br>
    <br>
    I don't think it is possible like in M4A, at least not with a lot
    tweaks/secrets etc.<br>
    M4A is commercial product and this binding of java jars to c# world
    is this simple automagic process<br>
    is not implemented elsewhere (with few steps more it can be done
    with ikvm)<br>
    The core of this process is generator[.exe] which produces
    assemblies for M4A and those cannot be<br>
    used in ASP.net projects (some cmdline switches are Android specific
    like SDK and API version).<br>
    <br>
    With ikvm you can <br>
    <ul>
      <li>run java code inside mono CLR, or</li>
      <li>compile (jit) java jars to native x86 (what c, c++ would
        output) and use PInvoke<br>
      </li>
    </ul>
