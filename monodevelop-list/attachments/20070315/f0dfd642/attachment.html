      <font face="Dialog" size="2">Thanx for offering to help out. I am willing to assist&#44; but please bear in mind that I am still learning C&#35;. In Delphi/Lazarus you have a hierarchy of objects that provide you with the ability to create database applications without writing a single line of code. </font>    </p>
    <p style="margin-bottom: 0; margin-top: 0">
      <font face="Dialog" size="2">The visual component &#40;widgets in MD&#41; are derived from the standard TGrid&#44; Textbox&#44; checkbox&#44; listbox etc components and they have additional properties called DataSource and DataField. The DataField needs to be set to the field name of one of the fields in the Datasource. In the Delphi/Lazarus IDE&#39;s these properties show up as listboxes that you just select the field name or datasource. The Datasource is an abstraction layer between the visual data-bound components and the actual database components. It is not strictly necessary&#44; but makes it very easy to switch database backends. The only property I ever set on that is the DataSet. The Dataset is the workhorse that is unique to each type of database backend you connect to. It provides properties that enable you to connect to the backend &#40;username&#44; password&#44; ip address&#44; protocol level etc&#41; and then returns the rows in the table or query.</font>    </p>
