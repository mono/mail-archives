</div>Did you build the mdb debugger addin?<br>
<br>
You&#39;ll need MD 2.0 alpha 2 or later (right now SVN trunk is superior<br>
to alpha 2), and Mono 2.0 or 2.2 with the Mono debugger.<br>
<br>
If you&#39;re using MD packages, such as the ones in the openSUSE buidl<br>
service, you just need to install the monodevelop-debugger-mdb<br>
package.There are some one-click installs for openSUSE:<br>
<a href="http://software.opensuse.org/search?baseproject=openSUSE%3A11.1&amp;p=1&amp;q=monodevelop-debugger-mdb" target="_blank">http://software.opensuse.org/search?baseproject=openSUSE%3A11.1&amp;p=1&amp;q=monodevelop-debugger-mdb</a><br>

<br>
If you build MD from svn, use .&quot;/configure --select&quot; and you&#39;ll be<br>
able to select which &quot;extras&quot; you want to include in the build. Make<br>
sure you select MonoDevelop.Debugger.Mdb. Note that building from SVN<br>
is convenient because you can &quot;make run&quot; to run MD without installing<br>
it, so you won&#39;t interfere with installed MD packages.<br>
<br>
If you build MD from tarballs, you&#39;ll need to configure and build the<br>
monodevelop-debugger-mdb tarball against a matching install of MD.<br>
<br>
<br>
For the easiest and most reliable way to get it working, I suggest<br>

the link above), as I regularly push new build when bugs are fixed.<br>
