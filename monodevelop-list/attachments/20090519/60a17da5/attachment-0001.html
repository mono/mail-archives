</div></div>Although your answers are 100% correct for addins in main, my original<br>
advice was for setting up an addin as a separate solution in extras.<br>
<br>
In the case of having the addin in extras/YourAddinName, it is necessary to:<br>
<br>
1) configure your projects to output assemblies to extras/YourAddinName/build<br>
2) set PKG_CONFIG_PATH to point to the local-config directory so MD<br>
will allow you to reference the local MD&#39;s assemblies as packages, and<br>
hence build and get code completion against the local MD<br>
-then, either-<br>
3a) add a makefile and configure script to the extras/YourAddinName so<br>
that it can be built with the top-level configure script and<br>
makefiles. This makefile must use the PKG_CONFIG_PATH passed to it by<br>
the configure script. MonoDevelop.AspNet.Mvc is an example of a such a<br>
makefile that invokes mdtool. Also add your addin to prpfiles/all,<br>
then run configure --select, and make sure the addin is selected as<br>
part of the build.<br>
-or-<br>
3b) if not using a makefile, add a YourAddinName.addins file to<br>
local-config, pointing at the extras/YourAddinName/build directory, to<br>
make the local MD load the addin<br>
<br>
We should really have a page on the wiki for this. It would also be<br>
convenient if we could eliminate step 2 by having the workspace handle<br>
pkgconfig resolution in some way.<br>
