<div>Hi,</div><div><br></div>Look at Initializer.cs in OnFrameChanged(), you'll see that it calls IdeApp.Workbench.OpenDocument() with the file & line and OpenDocumentOptions.Debugger.<div><br></div><div>OpenDocumentOptions.Debugger is actually a set of bit flags: BringToFront | CenterCaretLine | TryToReuseViewer</div>
<div><br></div><div>You could redefine Debugger to not include the CenterCaretLine bit flag as a simple workaround, but a more correct solution is to find a way to make the editor not scroll if the line is already in view.</div>
<div><br></div><div>I made an attempt at this 9 months to a year ago and committed it, but I think we eventually reverted it because it broke something else. I think the problem was that sometimes you really do want the caret line to be centered, but I could easily be wrong on that. The correct solution is probably to create a new bit field that means "Don't scroll unless the line isn't in view" and include that bit in the Debugger flags in place of the CenterCaretLine bit flag.</div>
<div><br></div><div>Hope that helps,</div><div><br></div><div>Jeff</div>
