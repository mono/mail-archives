    I had similar problems with ASP.net vNext restore which uses nuget.
    Some users report that certificates should be installed into machine
    store not user store:<br>
    <br>
    This will install certificates into user store:<br>
    <br>
    <meta charset="utf-8">
    mozroots --import --sync<br>
    certmgr -ssl <a class="moz-txt-link-freetext" href="https://go.microsoft.com">https://go.microsoft.com</a><br>
    certmgr -ssl <a class="moz-txt-link-freetext" href="https://nugetgallery.blob.core.windows.net">https://nugetgallery.blob.core.windows.net</a><br>
    certmgr -ssl <a class="moz-txt-link-freetext" href="https://nuget.org">https://nuget.org</a><br>
    <br>
    and this will install into machine store<br>
    <br>
    <meta charset="utf-8">
    sudo mozroots --import --machine --sync<br>
    sudo certmgr -ssl -m <a class="moz-txt-link-freetext" href="https://go.microsoft.com">https://go.microsoft.com</a><br>
    sudo certmgr -ssl -m <a class="moz-txt-link-freetext" href="https://nugetgallery.blob.core.windows.net">https://nugetgallery.blob.core.windows.net</a><br>
    sudo certmgr -ssl -m <a class="moz-txt-link-freetext" href="https://nuget.org">https://nuget.org</a><br>
