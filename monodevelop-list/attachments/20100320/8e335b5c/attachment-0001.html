After MD 2.2.2 was built and installed, the problem ceased to exist.
The only change made to the system was the compilation and installation
of this version of MD.<br>
<br>
If the default browser setting had been the problem, then the test code
wouldn't have successfully called the browser when run from the
console. If it had been one of the other libraries, then just building
2.2.2, and nothing else, wouldn't have fixed the problem.<br>
<br>
Yesterday, I built and installed mono 2.6.3 and re-built MD 2.2.2, and,
again, the problem did not appear.<br>
<br>
It is, of course, possible that I made some error with one of the
libraries when I built 2.2, an error I corrected without knowing it...
but I don't believe I did.<br>
