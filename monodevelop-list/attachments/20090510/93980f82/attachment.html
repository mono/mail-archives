</HEAD>
<BODY>
<FONT FACE="Calibri, Verdana, Helvetica, Arial"><SPAN STYLE='font-size:11pt'>Hi all, first time posting here. Let me know if there&#8217;s a better/preferable way to report bugs. Anyways, there are twp bigs:<BR>
<BR>
If a C# project (.csproj) does NOT contain a ProjectTypeGuids element, it will properly load. However, if the guid is defined in the project file with the proper explicit value, it fails:<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&lt;ProjectTypeGuids&gt;{FAE04EC0-301F-11D3-BF4B-00C04F79EFBC}&lt;/ProjectTypeGuids&gt;<BR>
Briefly looking into the code related to this, it seems that the parsing/handling of project subtypes is being done properly... Just a bug in there somewhere.<BR>
<BR>
In addition, I noticed that the CSharpBinding.addin.xml file only contains the guid for .NET Projects, and not .NET Compact framework projects (which is merely a subset):<BR>
<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&lt;Extension path = &quot;/MonoDevelop/ProjectModel/MSBuildItemTypes&quot;&gt;<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&lt;DotNetProject <BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;guid=&quot;{FAE04EC0-301F-11D3-BF4B-00C04F79EFBC}&quot;<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;/&gt;<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&lt;/Extension&gt;<BR>
