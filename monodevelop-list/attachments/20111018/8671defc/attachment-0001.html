    Notes:<br>
    <br>
    Problems after upgrading from Ubuntu 11.04 to 11.10<br>
    <br>
    14th -17th October 2011<br>
    <br>
    1. Performance &#8211; very very very slow &#8211; everything was a dog<br>
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &gt;&gt; installed Gnome shell and changed themes to Adawaita
    &#8211; better<br>
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &gt;&gt; installed Chrome as Firefox was repeatedly hanging<br>
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &gt;&gt; using the Gnome desktop (you need to RIGHT click on
    the cog in the Login box after starting the PC to select Gnome) and
    Chrome has made life bearable, but it was 'hell' for 3 days whilst I
    figured this out &#8211; not much help on the net/forums etc<br>
    <br>
    2. Still broken &#8211; myRSS (own C# news feed reader) not working -
    running from Terminal shows numerous error messages 'missing method
    System.Type' and in MonoDevelop it seems that all the target
    assemblies are now wrong!!<br>
    <br>
    3. Still broken &#8211; MonoDevelop &#8211; attempt to build fails with 'no C#
    compiler for .NET/3.5' !! this is the pits...<br>
    <br>
    4. Still broken &#8211; network printing &#8211; other (Windows) PCs on the home
    network cannot print<br>
    <br>
    5. Fixed &#8211; LibreOffice Calc &#8211; this would just crash on start-up,
    after the start-up splash &#8211; no error message &#8211; Writer worked, as did
    Draw (didn't try the others) &#8211; running 'libreoffice --calc %U' from
    Terminal at least showed an error message &#8211; tried without the %U
    parameter (which I assume is user, but could not confirm) also
    failed &#8211; finally one error message started to make sense, something
    about 'another instance of LibreOffice is accessing your personal
    settings which are locked' and that 'it's not safe for simultaneous
    access to your settings &#8211; do you want to continue, Yes or No' &#8211;
    neither Yes or No would work &#8211; around this time also found that
    Nautilus (from Places) would disappear on start-up, in the same way
    as Calc &#8211; reboot sorted this problem, Nautilus worked &#8211; checking
    Syslog saw that LibreOffice and Nautilus were SegFaulting &#8211;
    rummaging around the file system (now that Nautilus worked) I came
    across a '.lock' file in /home/.libreoffice/3 &#8211; this file showed the
    user as 'root' (not my normal user) &#8211; renamed this file to
    '.lockOLD' and, miracle of miracles, Calc finally worked (and
    created another '.lock' file with my usual user name).<br>
