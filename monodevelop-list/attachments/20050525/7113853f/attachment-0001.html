<div><span class="341322914-24052005"><font face="Arial" size="2">I have worked on 
several open source .NET based projects before Mono was out (specific to 
Microsoft's runtime) and lately have been spending more free time using Mono on 
both Windows XP and FC3 to start working&nbsp;on new projects and ideas. Monodevelop 
is so important to the Mono community (we need a stable free IDE bad!), and Mono 
is equally important to getting people using linux for .NET development 
&quot;switched over&quot; from Windows. The current way&nbsp;Monodevelop is being distributed 
is a bummer though, as it assumes everyone who wants to contribute code to it is 
a linux developer or has no problem running makefiles on windows and&nbsp;getting 
dependency builds from SVN for all kinds of other stuff Monodevelop relies on. 
Realistically this isn't the case.</font></span></div>

<div><span class="341322914-24052005"></span>&nbsp;</div>

<div><span class="341322914-24052005"><font face="Arial" size="2">I would like to 
throw&nbsp;a simple idea out there for a&nbsp;build solution&nbsp;for making it easier for 
people to get SVN builds of&nbsp;Monodevelop on thier box and in turn getting to a 
production capable release quicker. We all know the power of open source 
development is leveraging the community and the reality of the current situation 
is that&nbsp;alot of the people who could really help get Monodevelop done that 
aren't already working on the project are&nbsp;using&nbsp;Windows.</font></span></div>

<div><span class="341322914-24052005"></span>&nbsp;</div>

<div><span class="341322914-24052005"><font face="Arial" size="2">I would recommend 
switching the build system for Monodevelop to NAnt (<a title="http://nant.sourceforge.net" href="http://nant.sourceforge.net/">http://nant.sourceforge.net</a>).</font></span></div>

<div><span class="341322914-24052005"></span>&nbsp;</div>

<div><span class="341322914-24052005"><font face="Arial" size="2">- Create a build 
script that someone with NAnt installed can download to their computer and&nbsp;run 
on their local windows / linux / OSX box that will use the subversion tasks for 
NAnt to get the code from SVN and any other dependencies on wierd versions of 
GTK# etc. This seems to be happening each time you guys release and expecting 
people who are just making the switch to linux to understand all the issues with 
dependency management when you could do this with a NAnt build script seems to 
not be the most efficient method for getting people going with a development 
build of Monodevelop they can use to contribute patches back 
with.</font></span></div>
