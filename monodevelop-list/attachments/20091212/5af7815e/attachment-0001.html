<table cellspacing="0" cellpadding="0" border="0"><tbody><tr><td valign="top" style="font:inherit">I&#39;ve written a Mono application using Gtk# and am using it on the Mac. It&#39;s icon at the bottom of the screen is a generic one (black/grey window of some kind). Can I use a custom image instead? If so, how do I go about it.<div>
<br></div><div>I&#39;m using Monodevelop to build the application, if that helps. </div></td></tr></tbody></table></blockquote></div><div><br></div>Packaging an &quot;app bundle&quot; for Mac requires a little work, and there isn&#39;t any automated packager for Mono apps yet that I&#39;m aware of.<div>
<br></div><div>You should create a directory called YourAppName.app, which is the &quot;app bundle&quot;.</div><div><br></div><div>Create a file YourAppName.app/Contents/Info.plist. This is the app manifest, and specifies various information about the application - its name, icon, version, file associations, etc. It&#39;s quite well documented on Apple&#39;s site.</div>
<div><br></div><div>The icon referred to in the Info.plist should be an &quot;icns&quot; file which you can compose from pngs using Apple&#39;s Icon Composer tool, and should be located in YourAppName.app/Contents/Resources.</div>
<div><br></div><div>The Info.plist also specifies the executable that should be run when the app bundle is run - this should be a shell script in YourAppName.app/Contents/MacOS invokes Mono to run your exe. Your program&#39;s &quot;real&quot; executable files and supporting files should be in YourAppName.app/Contents/MacOS or some subdirectory of it.</div>
<div><br></div><div>For your shell script I suggest taking some code from MonoDevelop&#39;s script (<a href="http://anonsvn.mono-project.com/viewvc/trunk/monodevelop/main/build/MacOSX/monodevelop?view=markup">http://anonsvn.mono-project.com/viewvc/trunk/monodevelop/main/build/MacOSX/monodevelop?view=markup</a>), specifically the exports of DYLD_FALLBACK_LIBRARY_PATH  and PATH, which will ensure your app works even if MacPorts interferes with the Mono environment, and the section &quot;#mono version check&quot;, which will prompt your users to install Mono if it is not installed, instead of just dying silently.</div>
