<BR>
I suggest a parallel environment<BR>
(<A HREF="http://www.mono-project.com/Parallel_Mono_Environments">http://www.mono-project.com/Parallel_Mono_Environments</A>) using the 2.0<BR>
branch (branches/mono-2-0/). Build then install libgdiplus, then mono,<BR>
then the dubugger. After this, build mono-basic, mono-tools and xsp if<BR>
you wish. All of these should be from the 2.0 branch. Finally build<BR>
MonoDevelop from trunk.<BR>
<BR>
Tips:<BR>
1) make sure each of these in built and installed in your parallel<BR>
install before proceeding. If any of the configure scripts picks up<BR>
the working Mono (i.e. not yuoru apallle one), things can break in<BR>
strange ways.<BR>
2) You can use the GTK# etc libraries from your distro using<BR>
MONO_GAC_PREFIX (the Parallel Environment page explains this). If you<BR>
want to install updated version into your parallel install, user the<BR>
latest stable release un;ess you ahve good reason to do otherwise.<BR>
3) You can use the same trick to set up a parallel HEAD environment.<BR>
however, the debugger will work with 2.0, and HEAD is usually less<BR>
stable. I have separate 2.0 and HEAD environments, but use 1.9.1<BR>
day-to-day.<BR>
