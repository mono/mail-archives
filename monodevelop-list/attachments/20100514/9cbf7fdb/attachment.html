<p>You do not need to run MonoDevelop on Mono to debug Mono. In fact, I strongly recommend against running MD on Mono on Windows, as it&#39;s only tested on .NET on that platform.</p>
<p>The solution is much simpler. MD can *target* multiple runtimes, including debugging, regardless of the runtime it&#39;s running on.</p>
<p>Simply install Mono 2.6+ on Windows, and it will show up as Run and Debug options in the &#39;Run-&gt;Run with&#39; menu in MD. You can also change the target runtime between Mono and .NET on a per-project basis using the &#39;Project-&gt;Target Runtime&#39; menu, or change the default in Preferences.</p>
