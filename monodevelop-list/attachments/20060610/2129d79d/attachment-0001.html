I've been contemplating both the GUI and the parser parts of code
completion (i.e. MonoDevelop.Projects.Gui.Completion and
MonoDevelop.Projects.Parser).<br>
<br>
Firstly:<br>
<blockquote>It seems to me that fixing the 'Parser' part will not be a
big problem. The following code should be added:<br>
</blockquote>
<blockquote>in IClass:<br>
  <tt>StringCollection GenericParameters {</tt><br>
  <tt>    get;</tt><br>
  <tt>}</tt><br>
  <br>
in IMethod:<br>
  <tt>StringCollection GenericParameters {</tt><br>
  <tt>    get;</tt><br>
  <tt>}</tt><br>
  <br>
in IReturnType:<br>
  <tt>StringCollection GenericArguments {</tt><br>
  <tt>    get;</tt><br>
  <tt>}</tt><br>
  <br>
in IParameter:<br>
  <tt>StringCollection GenericArguments {</tt><br>
  <tt>    get;</tt><br>
  <tt>}</tt><br>
  <br>
The <tt>GenericParameters</tt> property<tt></tt> provides virtual
names of the generic types, where on the other hand <tt>GenericArguments</tt>
property provides actual types for them. A class/method indicates with
non-null and non-empty generic parameter list that it is generic.<br>
  <br>
In code completion an instance of a generic type should have its
virtual generic parameters replaced with its actual ones.<br>
  <br>
Should I proceed with adding above code or is there still something I
have missed?<br>
  <br>
Should I also modify language bindings to accommodate upper changes?<br>
</blockquote>
<br>
