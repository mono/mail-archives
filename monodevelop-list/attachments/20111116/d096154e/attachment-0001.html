    There are few comments about this post:<br>
    "7zip does not extract files deeper than
    “…\monotouch-eval-3.1.1\monotouch.pkg”."<br>
    The fact is that unzipping must be done twice in order to get what
    You need!<br>
    Remember with all this modern stuff (xap, apk, pkg) if You're stuck
    try renaming it<br>
    to zip and unzipping it - it may be hidden/renamed zip archive. It
    usually works for us.<br>
    ---------------------------------------------------------snip
    ---------------------------------------------------------<br>
    ::echo off<br>
    <br>
    set MONOTOUCH_PKG=monotouch-eval-4.2.2.pkg<br>
    set MONOTOUCH_UNPACKED=%MONOTOUCH_PKG:.pkg=%<br>
    set PATH_DESTINATION=C:\bin\iOS\MonoTouch\<br>
    <br>
    set PACKER_7ZIP="%ProgramFiles%\7-zip\7z.exe"<br>
    ::set PACKER_7ZIP=c\bin\7za.exe"<br>
    ::set PACKER_7ZIP=c:\bin\bzip2-105-x86-win32.exe<br>
    :: c:\bin\bzip2-105-x86-win32.exe --decompress %MONOTOUCH_PKG%.zip<br>
    <br>
    echo unpacking %MONOTOUCH_PKG% <br>
    echo        to %MONOTOUCH_UNPACKED%<br>
    <br>
    %PACKER_7ZIP% <br>
    <br>
    <br>
    echo f | xcopy /q /y /i %MONOTOUCH_PKG% %MONOTOUCH_PKG%.zip<br>
    %PACKER_7ZIP% x %MONOTOUCH_PKG%.zip -o%MONOTOUCH_UNPACKED%<br>
    <br>
    <br>
    cd %MONOTOUCH_UNPACKED%\monotouch.pkg<br>
    dir Payload<br>
    echo f | xcopy /q /y /i Payload Payload.zip<br>
    %PACKER_7ZIP% x Payload.zip -oPayload.unpacked<br>
    <br>
    cd Payload.unpacked<br>
    move Payload Payload.zip<br>
    %PACKER_7ZIP% x Payload.zip -oPayload.unpacked<br>
    <br>
    <br>
    pause<br>
    ---------------------------------------------------------snip
    ---------------------------------------------------------<br>
