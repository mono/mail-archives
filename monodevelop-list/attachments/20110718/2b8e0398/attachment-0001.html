<br>
</div>The short answer is yes. MonoDevelop is an Integrated Development<br>
Environment (IDE) that supports writing C# applications that run on Windows,<br>
Linux, OS X (Mac), and other platforms. You can think of MonoDevelop as a<br>
Visual Studio clone that you can use instead of Visual Studio in your<br>
Mono/.NET workflow.<br>
<br>
I should point out though that .NET (and Mono) programs written using Visual<br>
Studio on Windows are also completely cross-platform. If written properly,<br>
they will run unmodified on any platform Mono supports. Simply move the<br>
compiled .EXE and .DLL files to a Linux or Mac box and run them with Mono.<br>
It is a common misunderstanding that you must use MonoDevelop to create Mono<br>
apps.<br>
<br>
MonoDevelop uses the Visual Studio solution and project formats. So, you can<br>
start a project in Visual Studio and then edit it in MonoDevelop or<br>
vice-versa.<br>
<br>
The real benefit of MonoDevelop is that it itself runs on Windows, Linux,<br>
and Mac. That means that you can develop your programs on the platform of<br>
your choice instead of being forced to use Windows to use .NET.<br>
<br>
There is no portability benefit of using MonoDevelop per se. One possible<br>
benefit is that if you write your code first on Linux or Mac you are certain<br>
that it will run on Mono. If you write it entirely on Windows first you may<br>
use some functionality that Mono does not support or make coding errors that<br>
make your program unsuitable for Linux and Mac (like hard-coding Windows<br>
path information, using P/Invoke to access non .NET libraries, or assuming<br>
case-insensitive filesystems).<br>
<br>
Another reason to prefer MonoDevelop is if you are developing desktop apps<br>
that use the GTK# toolkit. MonoDevelop has a built in designer for this<br>
purpose and just does a nicer job overall of supporting this kind of app. Of<br>
course, Visual Studio and SharpDevelop both do a nicer job of supporting the<br>
creation of Windows Forms GUIs.<br>
