Hey Timotheus,<br/>
<br/>
The best way for the MD team to review your patches is to propose them <br/>
in the form of pull requests in github.<br/>
<br/>
As a recommendation, open one pull request per issue you find, so you <br/>
can explain each change independently (in the commit message), and each <br/>
of the changes can be merged separately (otherwise, if you add all <br/>
changes in the same pull request, it may get longer to review).<br/>
<br/>
