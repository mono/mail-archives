escriure:<br>
&gt; Hello,<br>
&gt;<br>
&gt;       What is the role of this element?<br>
<br>
This element registers a new MSBuild project type.<br>
<br>
&gt;<br>
&gt;       How do I generate a guid for a currently unsupported language?<br>
<br>
If you want to be able to open F# projects created with Visual Studio,<br>
you should use the same guid VS is using, which is<br>
{F2A71F9B-5D33-465A-A702-920D77279786}.<br>
<br>
To get this guid, I created a new F# project with VS 2010. In the .sln<br>
file, there is one line like this:<br>
<br>
Project(&quot;{F2A71F9B-5D33-465A-A702-920D77279786}&quot;) = &quot;ConsoleApplication2&quot;, &quot;ConsoleApplication2\ConsoleApplication2.fsproj&quot;, &quot;{38CFFCBD-67CE-4023-8C35-15DDD8C6A3D4}&quot;<br>
<br>
The first guid is the id of the project type (F#). That&#39;s the one to be<br>
registered for F# project types in MD.<br>
<br>
&gt;<br>
&gt;       What is the role of &quot;import&quot; sub element?<br>
<br>
That &#39;import&#39; attribute specifies the MSBuild &#39;targets&#39; file that has to<br>
be imported in the .fsproj project file. A &#39;targets&#39; file contains<br>
commands and rules for building a project, so they are<br>
language-specific.<br>
<br>
To be compatible with Visual Studio, it should be<br>
&quot;$(MSBuildExtensionsPath32)\..\Microsoft F#\v4.0<br>
\Microsoft.FSharp.Targets&quot;.<br>
