                  I had similar problems with ASP.net vNext restore
                  which uses nuget. Some users report that certificates
                  should be installed into machine store not user store:<br>
                  <br>
                  This will install certificates into user store:<br>
                  <br>
                  mozroots --import --sync<br>
                  certmgr -ssl <a moz-do-not-send="true"
                    href="https://go.microsoft.com" target="_blank">https://go.microsoft.com</a><br>
                  certmgr -ssl <a moz-do-not-send="true"
                    href="https://nugetgallery.blob.core.windows.net"
                    target="_blank">https://nugetgallery.blob.core.windows.net</a><br>
                  certmgr -ssl <a moz-do-not-send="true"
                    href="https://nuget.org" target="_blank">https://nuget.org</a><br>
                  <br>
                  and this will install into machine store<br>
                  <br>
                  sudo mozroots --import --machine --sync<br>
                  sudo certmgr -ssl -m <a moz-do-not-send="true"
                    href="https://go.microsoft.com" target="_blank">https://go.microsoft.com</a><br>
                  sudo certmgr -ssl -m <a moz-do-not-send="true"
                    href="https://nugetgallery.blob.core.windows.net"
                    target="_blank">https://nugetgallery.blob.core.windows.net</a><br>
                  sudo certmgr -ssl -m <a moz-do-not-send="true"
