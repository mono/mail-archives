I'm surprised this has not been talked about more.  The potential here 
is huge.  It will be like Visual Basic V1 all over again. Place your 
widgets on a form, put some logic behind it, and compile.  Only now it 
will be placing the widgets on a canvas, and deploy to the web.  
Monodevelop could become the de-facto way to do this, if only more devs 
realised what a game changer this could be.

QT already supports NaCl.  Maybe the mono.cxxi project may help here.
