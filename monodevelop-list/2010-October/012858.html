On Sat, Oct 23, 2010 at 10:31 PM, Chuck McCrobie <mccrobie2000 at gmail.com> wrote:
> Hello MonoDevelop Users:
>
> I would like to gauge the interest in an OpenVMS EDT-keypad mode for
> MonoDevelop.  I found the "Vi" mode and I am quite impressed with the
> CaretMoveActions and friends - really well engineered with great
> flexibility.
>
> Please see: http://en.wikipedia.org/wiki/EDT_(text_editor)  for a
> description about the EDT Keypad.  Google can help also.
>
> The idea is that the editor is always in "insert" mode and cursor
> control, cut/paste,  find, etc. can be performed with the keypad.
>
> I have started to support some basic cursor movements, but need to
> support selection and search.  Before I start on that, I was wondering
> if _anyone_ would be interested - or even know what I'm blathering about.

This is the first I've heard of it, but I certainly think it's cool to
have support for other input modes :)

> (Yes, I'm showing my age, but if there's one thing I miss about OpenVMS,
> its the EDT Keypad mode.  Vi is great and all, but the yanking still
> leads me to counting lines on the screen to know how many to yank - with
> Keypad mode, I can start the select, move the cursor and have the screen
> show me the selection, then cut / change case, etc.  _all_ without
> moving my hands.   No more 22y or 8dd for me.  Lessens the "hunt for the
> mouse" also.  And no more "what line am I on so I know to do :345,.y" )

What about vi's visual and visual line modes? We do have support for
those in MD - e.g. 8dd -> Vjjjjjjjd.
