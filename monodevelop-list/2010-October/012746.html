Ahmed and David:

MonoDevelop supports Mono 2.6.* which is equivalent to the 3.5 .NET
Framework.  Mono 2.8, which hasn't been released yet, is equivalent to the
4.0 .NET Framework.  Although, using Mono 2.8 inside MonoDevelop for your
projects works.  You should be aware, it's not fully supported yet.  There
are known bugs and things should get ironed out in future versions of
MonoDevelop after Mono 2.8 releases.

Ahmed, to retrieve Mono 2.8, you can download one of the preview
releases[1] or you can wait for the final release which should be soon. 
Another option is cloning master on github[2].  Make sure you read setting
up a parallel environment[3].

[1]: http://mono.ximian.com/monobuild/preview/download-preview/
[2]: http://github.com/mono/mono
[3]: http://www.mono-project.com/Parallel_Mono_Environments
