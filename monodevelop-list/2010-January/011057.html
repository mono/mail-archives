After more researches, I finally solved my problem, but still don't know what
caused it.

With an older version (2.0) of Monodevelop, the error message (An element
with the same key already exists in the dictionary) contained the name of an
inner project of the solution that could not be loaded. I removed a
duplicated line corresponding to this project in my solution file (.sln),
and the solution could then be loaded.

It looks like Monodevelop allowed me to import a project with an existing
name, but refused to load the solution after this.
