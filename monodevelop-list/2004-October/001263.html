In fact Ihave to add that this is not a problemonly with automake 1.4.
MD have many dependencies with many components that block its use with more 
recent versions of those components.
In one way Linux lacks of a Model like the COM in Windows. So when we build 
warpers, example:gtk-sharp, there isn't any way to rebuild the warper when a 
new version is released. This fact is making the mono adoption very slow, 
taking my enterprise and some friends as example. This is why I had some 
trouble with monodevelop and its dependencie in gtk-sharp and gtkhtml after 
updating to Gnome 2.8 in my FC2 box.

I think this is something we should work, build a "auto-warper" that works 
in linux more or less like the COM interops in Windoze.
