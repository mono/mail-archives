I don't have a strong opinion on this. I believe in general closing the
dialog is ok, although I remember in some cases being annoyed for having
to reopen the dialog when doing several searchs. We could leave the
current behavior for a while and see if other people are missing the old
behavior.

In any case, I think the Find button in the Replace in Files dialog
should not close the dialog. It is very common to do a find before a
replace to see what's going to be replaced. Having to reopen the replace
dialog is very annoying.
