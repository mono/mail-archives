I think Bugzilla is only for assigning bugs to the team, not third parties.

What I did when I fixed bugs and added features (in the Novell days) was
fork on git hub, make a pull request referencing the ticket when I was done
and possibly put a comment I'm the ticket to say I was looking at it when I
started.

At the end of the day, the team can only rely on their own staff not to
vanish, and one of the guys skool have to merge the change back anyway.

Hope that helps.
