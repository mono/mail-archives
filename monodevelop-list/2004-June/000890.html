I would like to list a set of features in VS.NET that are super powerful 
and any good IDE will need to support in the .NET Space.

No. 1 - Typed DataSets!  In VS.NET you can use the XML Schema designer 
to literally generate your systems base code from your XSD.  This code 
includes the typted datasets tables, columns, fields etc.  No more 
coding class libraries when you build a system if you have this feature. 
  Cuts way down on development time.

No. 2 - Ability to import web references and have all the code generated 
for the proxy.

No. 3 - Generation of source code documentation from C# XML comments

No. 4 - Ability to include a database server reference.  Currently 
VS.NET only includes  support for SQL SErver obviously.  What would be 
fantasmic would be to have the ability to reference SQlite, MySQL, 
PostergreSQL.  This would include the ability manipulate database 
objects, and manage the data for quick turnaround debugging and testing.

No. 5 - A good extensibility architecture so that CVS or sub version can 
be built into the solution management part of your IDE

Although I have so much work I can't take part in a project like this. 
Because I use VS.nET extensively I feel like I have a handle on what 
helps us the most on projects.   If you guys are interested in more 
information and I have some let me know.

Thanks,
Allan
