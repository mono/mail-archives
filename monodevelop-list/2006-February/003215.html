I think the problem might be caused by such that, the "mono(xxx-sharp) =
2.4.0.0" needed by package monodevelop is actually the packages with
name "xxx-sharp2" instead of "xxx-sharp".

For example, the current version of gtk-sharp appeared in my yum/rpm is
"1.0.10-3.fedora4.novell", but there does exist a package "gtk-sharp2"
with version "2.4.0-2.fedora4.novell". And what yum is looking for, is
in fact "gtk-sharp-2.4.0-2.fedora4.novell".

Correct me if I am wrong. Again, my arch is Fedora Core 4 for x86_64.
