Run "./autogen.sh --prefix=/usr" again as it is thinking that
gecko-sharp in another prefix:

-r:/usr/local/lib/mono/gecko-sharp/gecko-sharp.dll   \

Another possibility is that the pkg-config .pc files for gecko-sharp
are pointing for /usr/local/lib, in that case you need to run the
standard-recipe in gecko-sharp directory:

./autogen.sh --prefix=/usr
make
(as root) make install

HIH,
