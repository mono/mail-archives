I upgraded to MonoDevelop 3.0.3.5 (beta).  Identical procedure as before (ie, it was an automatic update).  I compiled my app with no new changes and saw this message:
	Corlib not in sync with this runtime: expected corlib version 632, found 96

I did a clean all, rebuilt the app, same error as before.

To isolate anything in my code, I closed out of MonoDevelop relaunched it and created a new template project.  
Project steps: (C#- Monotouch- iPad Storyboard - Tabbed Application)
	SAME ERROR as before.

I restarted my Mac.  Opened the newly created template project, did a clean all and built the app.  Same error as before.

I reinstalled MonoDevelop.  Same results.

It was only AFTER contact with Xamarin that I was able to downgrade to 3.0.3.4.

Once I downgraded, I loaded the template project, did a clean and build and VIOLA!  works perfectly.

I loaded my app, and VIOLA!  works perfectly. 

Something happened between 3.0.3.4 and 3.0.3.5 that seriously affected the performance of a tabbed application.

I just consider this a TEMPORARY workaround, as I would like to be on the latest version, but clearly, I cannot at this time until this issue is resolved.
