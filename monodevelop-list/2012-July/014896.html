Hey,

The simplest way to work around the problem with existing releases is
to ensure that your test assembly is compiled using the
nunit.framework.dll supplied by MonoDevelop (i.e. select it from the
project references and do not use a local binary included in your
source) and then also ensure that Local Copy is disabled for that
reference. That should allow everything to work as it used to.

Alan
