I agree that this is very annoying as a feature - without regard to
any performance issue.

Consider this situation...

I have a number of files open.
I want to examine several files in a specific folder, so...
  I expand the folder in the solution tree
  I open a file and examine it
  I close that file, intending to move to another file
  BUT... the IDE changes focus to another open file in a different
place, so I have to
  navigate again to the folder with files that currently interest me.

Personally, I'd much rather not have this behavior at all and instead
have a context
menu entry for the tab that would track to the proper part of the solution tree.
