VB.NET support is very old, and is going to be reworked. The VB.NET
compiler is being also being reengineered, so even though it is
possible to use current VB.NET support in MD, it may not be able to
compile a not so simple application.

Also currently the stetic gtk# ui-designer has still some rough edges
when dealing with languages other than C#, but you can use Glade for
designing UI for GTK# apps for mostly any language MD supports.

C# (in the core), Boo, Nemerle and Java (with their plugins) are the
best supported languages in MD, currently.
