Hi,

I am using the latest version of monodevelop from SVN and have noticed that
adding a new WebForm to an ASP .NET web application and then saving and
reopening the project 'breaks' the project. 

When a new WebForm is added to a Web Application and then saved the
<ProjectTypeGuids> are removed from the webproject.csproj file. When the
project is subsequently reloaded monodevelop no longer sees the application
as a web application and it is therefore not possible to 'Run' the app,
debug it or add any more web forms.

It is possible to 'workaround' the bug by manually adjusting the .csproj
file so that it contains the correct project type guids - but this is a bit
of a pain in the neck.

Is there any chance someone would have a look at this, please?
