Hi Christ-Jan,

There is currently no work on implementing a QT GUI for MonoDevelop and I
fear it would be a monumental effort to do so.

If you (or anyone else) are interested in working on a QT GUI, my
suggestion would be to look at https://github.com/mono/xwt (a
cross-platform abstraction over various toolkit backends) and start working
on a QT backend.

The MonoDevelop team is slowly working on migrating to Xwt, so if you have
a QT backend, it might one day be possible for you to use QT instead of Gtk.

Hope that helps,

Jeff
