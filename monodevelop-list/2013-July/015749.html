Chris Root wrote
> I want to use a plugin for monodevelop called dmono that supports the D
> programming language. It requires more up to date versions of mono (2.10.9
> or higher) then is available through the packages (2.10.8) for the newest
> versions of either Fedora or Ubuntu provide....

Alexander Bothe, the author of the D add-in has made available some
precompiled tarballs for MonoDevelop.  I haven't tried them (built from
source after some trial and error) but you can check them out here:

http://simendsjo.me/files/abothe/

Here's the relevant posts from his blog:

http://mono-d.alexanderbothe.com/?p=958

http://mono-d.alexanderbothe.com/?p=1014

Or, it might just be that you need to purge all Mono packages from your
system so that it will then use the version that you compiled yourself. 
This is what I had to do in my case when I installed Mono to /usr/local.

Good luck!
