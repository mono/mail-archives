> I've created a new Addin for MD which integrated Gendarme framework.
> 
> First release can be downloaded at:
> http://gendarmeaddin.googlecode.com/files/gendarmeaddin-0.1.tar.gz
> 
> I am open to suggestions/bug reports/comments.
> 
> Right now it does the following:
> 
>       * A dockable pad for running the tests, which shows the problem,
>         details and it's solution

I would suggest to put the "Run test" on some other place. For example
on the Solution/Project right click menu. The button wastes a lot of
space IMHO. You could make the entry dynamic so you can choose what test
to pass (I don't know if it's feasible). Also, I don't get the point of
having the violations tree discoverable.

>       * A configuration panel for specifying the rules file and the
>         rule set
>       * Ability to run tests on the whole solution or on the current
>         project

Having the "Run gendarme tests" on the menu makes the run on all items
in the solution option useless, that is IMO a bit confusing.

>         Ability to jump directly to problem source, if a rule
>         violation was found
> 
> Hope someone will find it useful!
> 
> Regards, 

> Eli Yukelzon

After all, great work!
