You will need to install cygwin and make sure svn
installed.

http://www.cygwin.com/

Also, would try to build mono and gtksharp from svn on
cygwin before attempting to build monmodeveloper from
svn.

http://www.mono-project.com/Compiling_Mono#Windows_Compilation

Once you have successfully built mono, mcs, and
gtksharp form svn, and you are able to run gtk#
applications on cygwin, then you can consider building
monodevelop on windows.

Here is some info about building monodevelop on
windows.  It is outdated because monodevelop no longer
runs on windows.  Well, it was very unstable on
windows when it did.

http://www.monodevelop.org/FAQ#Does_MonoDevelop_work_on_Windows.3F

I suggest getting a new gtk+ runtime at least 2.10+
and see if that helps as well.

The problems could be anywhere in the stack, glib2,
gtk+2, libgnome, gtk#, mono, or monodevelop.  Some
problems could be assumptions with linux files, paths,
file URLs, or loading libraries...
