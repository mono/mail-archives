Sure.

To tell MD that those libraries are actually used by your software,
from the Solution pane, right click on the references and then select
"Edit References" on the popup. Simply select or deselect libraries as
you please.

If you don't do these, your program won't even compile. It is not
necessary to write "using Tao;" at the top of the file, it is even a
cause of an error if you do not tell MD you'll be using the library,
and it is not even required. For example, I myself never use those
lines, I use MD autocompletion feature instead (that way I can write
"Gt.Tre" instead of "TreeView" plus I get a view on what I have
available).

Think about it, mono's and MS's compilers work the same way.
Referencing a library is equivalent to using the "-r" command line
switch. How would you like MD to automatically add references to every
librarary in the GAC?
