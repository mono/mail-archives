Il giorno mar, 31/10/2006 alle 11.09 +0100, Matej Urbas ha scritto:
> I haven't been able to figure out much yet (will have to do much more
> research before jumping into the code), but I think I've figured that
> there are two basically distinct approaches to underlying text buffers:
> 
>  - One where all the text is in a simple array

You can discover which editor uses this method by loading a file with
pretty long lines (say 1000 chars): it usually slows down to turtle
crawl during any insert/delete operation. Ok, nobody actually write such
long lines but sometimes you open a file with wrong end-line marker and
having to kill the editor because it has become unusable doesn't make me
happy.

So I am all in favour of line-segmented or even
arbitrary-chunk-segmented editors.
