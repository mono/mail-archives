a) it's an advanced feature that would just get in the way of most
users, it's only useful for custom remote and embedded targets (and
productized versions of remote/embedded targets should be implemented
as project types)
c) the UI isn't polished or actively maintained, since it was intended
for testing and because of (a) it's not used much

If someone can think up a good UX that solves the first problem then
we could enable it by default.
