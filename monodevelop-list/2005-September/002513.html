I found how to fix the problem - someone on the #monodevelop irc channel
helped me. It's simply that monodevelop 0.7 has its "example solutions"
set up with some references to gtk-sharp 2.0.0.0, pango-sharp 2.0.0.0
and so on, whereas I had installed an environment with gtk-sharp 2.4.0.0
etc. It was an easy fix to delete the references to the 2.0.0.0
libraries and replace them with references to my installed 2.4.0.0
libraries, upon which the example gtk# solution built and ran fine.
