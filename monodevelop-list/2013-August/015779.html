The documentation at https://github.com/mono/gtk-sharp says that "Install the
gtk-3 development headers first". Apparently this will build without
gtk-sharp but ultimately is see a couple of compile issues on monodevelop
which are from a changed function prototype. I wrote a bug on this but I'm
not clear if it is a bug. monodevelop can be built with gtk+2 and will run
but it requires just the right combination of libraries. I have built it
with gtk+2 and have seen another build that uses gtk-2. If it requires gtk-3
then this is all wrong and the build with a backlevel gtk is unstable. I'm
working on Centos 6.4. Gtk-3 requires a lot of dependency changes all of
which have to be built from source because there are no Centos packages. I
have no evidence that the build is unstable with gtk-2. This document
https://github.com/mono/gtk-sharp gives no indication what level of GTK+ it
needs. The documentation lhttps://github.com/mono/monodevelop lists  the
dependencies. It does not mention gnome-sharp which I know it needs.
Everyone on this list builds monodevelop. How is monodevelop supposed to be
built? What level of GTK is necessary? If it is true that gtk+3 is necessary
for Centos 6.4 what is the best way to resolve this dependency?
