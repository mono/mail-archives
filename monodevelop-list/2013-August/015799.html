For a while now, maybe since February, the debugger will steadfastly refuse
to open objects for inspection. I filed a bug on this but, even offered a
bounty, but so far the bug persists. I have recently noticed some details
and thought I'd share in the hope that it might click for someone.

When first starting MD and invoking the debugger, everything is fine,
objects are inspectable. But after a time (maybe after the 10th inspection,
or perhaps after stopping it and starting it, this detail is hard to pin
down) the debugger no longer permits object inspection. The hover brings up
the object but the triangle does not open it. In the Locals panel, the
local objects are there but not inspectable. Value types are inspectable
(doubles and strings and such), so it appears related to the opening
aspect.

If I restart MD everything works, same program, same breakpoints that
previously failed now work. But after a short time it all stops. My current
work flow is to restart MD to get the debugger working, which is
inefficient to say the least.

Any help on this would be appreciated.
