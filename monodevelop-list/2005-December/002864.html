There is simply no built in support for localization in C# doc comments
and IF we include those translations in the sources as well, the lengthy
documentation will mess you. And if you split translation documents
out from the source, it won't reflect on-the-fly documentations for you.

When it comes to monodoc, we can at least theoretically use non-"en"
documentations as its basis (and it would be pretty simple to localize).

since monodevelop has its own parser, so it would make much sense
to have its own doc comment support.
