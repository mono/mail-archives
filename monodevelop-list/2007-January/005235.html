Yeah, the current situation is a bit confusing. Here is a proposal that
would simplify it from the user point of view: remove the "Import VS2003
project" option from the File menu, and instead add a new file VS2003
format in the file open dialog. 

When the user tries to open a VS2003 project a warning message box is
shown which says that the project file format is old and needs to be
saved in a new file format (which could be either the native MD format
or VS2005 format). I haven't tried, but I guess that VS2005 would do the
same if you try to open a VS2003 project. 

Maybe it would also be a good idea to split the Open command in two:
Open File and Open Project.

Does it make sense?
