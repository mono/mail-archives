Monodevelop currently has a really small set of addins, in addition to
the base distribution I mean. Also updates for Monodevelop/Mono are
not carried out by the Monodevelop addin manager, you need to use your
distro update manager, as they (the distro packagers) are the ones
that package it with all the correct dependencies and put in the
distro repositories.

Yes, it is different from the centralized style used by Netbeans and
Eclipse, but Mono and Monodevelop are a lot more connected to the
native platform than the Java IDEs are, and so have to be specifically
packaged for each distro/OS.
