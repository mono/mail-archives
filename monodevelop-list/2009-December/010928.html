As far as I'm aware, the F# binding that you linked is unmaintained,
and uses old MD API that's since been changed. When we change MD API,
we update all the projects in MD SVN, but we can't update
externally-stored projects. We have not been able to import the F#
binding into MD SVN because it contains GPL-derived code, which it
against our current policies because it conflicts with licenses like
MS-PL and Apache v1.

A few people have expressed interest in taking over maintainership or
writing a new (non-GPL) version, but no-one has actually stepped up to
do it.
