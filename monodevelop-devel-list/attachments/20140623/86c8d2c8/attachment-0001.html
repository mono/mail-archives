    I chose to stay with GetCharAtÂ  because most people preferred it
    (only Jeff liked the indexer).<br>
    <br>
    I don't think it's 'hard' to discover. The indexer makes sense IMO.
    It shouldn't be looked at the editor/document level - just at the
    ITextSource. That's very close to a char[] ... the document &
    editor is just extending that. I had to 'discover' that in the
    roslyn API ... took me about < 5 seconds. In the Roslyn version
    it's needed to use the Roslyn text API from time to time - then we
    would need to mix GetCharAt and Indexers anyways. This would be more
    consistent.<br>
    <br>
    The GetCharAt OTOH is easier to read/in line with GetTextAt - and -
    I can't guarantee that it will stay O(1). One strong argument for
    staying with GetCharAt is that we need fewer code changes and we're
    all used to that.<br>
    <br>
    btw. the API can still be changed. I would like to hear opinions - I
    just wanted to bring up one topic to start the discussion about
    that. Working with text is an important feature of the IDE. <br>
