<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'>There seem to be a lot of newcomers to <st1:place w:st="on">Cocoa</st1:place>#
right now, and a fair bit of confusion about the best way to submit patches,
work one&#8217;s way into leadership, etc.<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'><o:p>&nbsp;</o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'>At the moment, I&#8217;m not actually actively working on
Cocoa# (there are some things lower down in the Mono stack that require my
attention before I can play with fancy graphical things&#8230;), but I&#8217;ve
found a system that works pretty well with Mono, and I would assume that this
process would also work as you move up the stack.<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'><o:p>&nbsp;</o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'>So in the interest of getting things to move forward,
here&#8217;s the way I do things. If you&#8217;ve been contributing longer than
I have, or if you think you have a better process, feel free to correct me. I
haven&#8217;t had a lot of guidance in this either, but I hope that by throwing
something out there, we can pull something together.<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'><o:p>&nbsp;</o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'>Obviously, the first thing to do is to sign up for as many
of the Mono mailing lists as you can keep up with, but you know that already
since you&#8217;re reading this mailing list&#8230;<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'><o:p>&nbsp;</o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'>After that, if you&#8217;re interested in contributing <i><span
style='font-style:italic'>anything</span></i> to any Mono project, you really
need to get an account on Ximian&#8217;s Bugzilla server at <a
href="http://bugzilla.ximian.com/" title="blocked::http://bugzilla.ximian.com/">http://bugzilla.ximian.com</a>.
No one actually has to do anything for you to get into this, so it&#8217;s a
really easy step. Bugzilla also seems to be the best vehicle for submitting
patches.<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'><o:p>&nbsp;</o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'>If you know of any existing bugs, file them right away (if
they aren&#8217;t already in the database). I&#8217;ve actually had a few
instances in which a bug that I filed was fixed by someone else, but that kind
of thing won&#8217;t happen if no one else knows what to fix.<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'><o:p>&nbsp;</o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'>Now that people know about your bugs, it&#8217;s time to fix
them. For best results, always make sure that you&#8217;re hacking on the
latest, most bleeding edge version of the source code. Mono uses subversion,
and most of the details about it can be found at <a
href="http://www.mono-project.com/SVN"
title="blocked::http://www.mono-project.com/SVN">http://www.mono-project.com/SVN</a>
(it also has a few details about code submission etiquette. I&#8217;m assuming
here that you&#8217;re not an absolute newcomer to subversion, but if you are,
this is where to start: http://subversion.tigris.org/. The one thing that I
will cover may be of use to people who use subversion at work, but who are not
used to the workflow of creating patches. If you&#8217;ve made changes to a
file or folder in the Mono tree named &#8216;blah&#8217;, type this command:<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'><o:p>&nbsp;</o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'>svn diff blah &gt; somefile.patch<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'><o:p>&nbsp;</o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'>This will create a patch file that you can then either
attach to your bug on Bugzilla (which is what works best for me) or send
directly to a mailing list.<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'><o:p>&nbsp;</o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'>Mono&#8217;s page on SVN says that once you&#8217;ve
submitted several patches to Mono, you can ask Miguel for write access to SVN
(this doesn&#8217;t mean that you have a free rein in the code, but it does
make things easier). I haven&#8217;t done this step (although I&#8217;ve made
enough contributions that I ought to&#8230;the terribly curious or nosy can
look for Bugzilla submissions from <a href="mailto:dmitchell@logos.com"
title="blocked::mailto:dmitchell@logos.com">dmitchell@logos.com</a>), but this
is mainly because I have no idea how I would acquire or what I would do with a
public OpenSSH key (yes, I know that it&#8217;s for encryption, but beyond
that?). For that matter, can anyone help me out here? Please?<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'><o:p>&nbsp;</o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'>So there you have it: my hard-won knowledge on how to break
into the Mono project. Some patches will move quite quickly into production
(less than a day), while others may take a bit longer (I&#8217;ve had to wait
over a month, in some cases), but once you start, it gets easier to build
momentum.<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'><o:p>&nbsp;</o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'>If you&#8217;ve been wanting to start contributing, but
don&#8217;t know how, give this method a shot, and if you have been
contributing for a while, and you don&#8217;t think that this is the best way
to go, then by all means, correct me.<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'><o:p>&nbsp;</o:p></span></font></p>
