In mono there is a etc\mono\config file that does the dll mapping, I 
can't remember what it would be in microsoft windows .net, but I would 
assume it will look in the current path order, including system32 and 
current dir.

There is also a private assembly resolve path in the current appdomain, 
however I do not know if that is used for DllImport mapping, but MSDN 
would say so.

I would definitly be interested to see your progress, keep us posted 
here.  (I'm a windows hacker too, since was doing most in VS, hence the 
vs proj files in the repository.
