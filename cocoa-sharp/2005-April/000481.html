>>>> I too had this problem. I'm guessing that you've installed some =20
>>>> stuff using darwinports, it seems that darwinports also uses the =20=

>>>> pkg-config program to help configure the software that you're =20
>>>> building and installing.
>>>>
>>>> Whilst I don't know much about pkg-config, It seems like it looks =20=

>>>> through a list of *.pc files to determine which packages have =20
>>>> already been installed. The problem arises because the .pc file for =
=20
>>>> cocoa-sharp isn't in the location where the pkg-config executable =20=

>>>> (from darwinports) expects to find it.
>>>>
>>>> I solved (worked around) this problem, by creating a symbolic link =20=

>>>> to the actual cocoa-sharp.pc file in the location where the =20
>>>> darwinports pkg-config was looking for it:
>>>>
>>>> /opt/local/lib/pkgconfig/cocoa-sharp.pc -> =20
>>>> /Library/Frameworks/Mono.framework/Versions/Current/lib/pkgconfig/=20=

>>>> cocoa-sharp.pc
