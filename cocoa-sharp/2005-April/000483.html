Hi All,
This is my first Interop project.
We have a Windows .NET client server application. We now want to PORT the
Windows .NET client to Mac OS X with very minimal efforts.

I have picked the latest Mac OS X binaries [Mono framework-1.1.5.dmg] from
the mono website & installed it on Mac OS X 10.3.8 

The read me of Mono framework installer clearly mentions that all the
dependency of Mono will be taken care and hence I did not check what else is
required.

NOTE:
As we are porting the code from Windows to Mac, we do NOT want to use
Cocoa#. We have extensively used System.Windows.Forms in our Windows project
and want to retain the same.

Have a simple hello.cs file as follows:

using System;
using System.Windows.Forms;

class MainApp
{
	public static void Main()
	{
		Console.WriteLine("First .NET Pgm on Panther 10.3.8\n");
		MessageBox.Show("Hello World");
	}
}

This is compiled and invoked as 
mcs -r:System.Windows.Forms hello.cs
mono hello.exe
Only the Console.WriteLine executes then throws exception as shown a bit
below.

Now I get to read somewhere that I need to use 'macpack' to invoke the
application created when using System.Windows.Forms.

Should I use macpack even if I don't want to use Cocoa#?

How should I compile and invoke the above simple application on Mac OS X
using macpack but WITHOUT Cocoa#.

Kindly reply a bit elaborately . Thanks
