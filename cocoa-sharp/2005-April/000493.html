 Hi All,

Thanks to each and every one of you who replied to my e-mail query.

I am at last successfully in running the Windows Forms on Mac.

Have used, Mac OS X 10.3.8. My main problem was that I did not have X11
installed which is very much required for the Windows Forms to work.
All other dependencies is taken care by the Monoframework-1.1.5 installer.
It was a cake walk once I installed X11.

I was preparing a step-by-step guide to get things working on Mac but "Oscar
Papel" beat me to it.
http://oepapel.blogspot.com/2005/04/windowsforms-and-mac-osx.html
He has given a very nice compact tutorial for any one looking at making
Windows Forms work on Mac.

I have used "Monoframework-1.1.5.dmg" and faced no problem from the same.
Even WRT the libgdiplus.

Now I am trying to explore other controls. Controls like TextBox,ComboBox
etc are not rendered properly on Mac. A drop down combo has the dropdown
part away from the Form (on the desktop sometimes). A click on the contents
of the list box or down arrow key selection of items makes the scrollbars to
disappear. Double clicking the text in textbox does not select the entire
text. I was not able to use Tab Control at all. I just added 2 tab pages
with names. This too didn't work.

Are there any one else who is also facing such problems with the Win forms
on Mac.?

Please comment.

Thank U all once again for all the responses.

Warm Regards,
Sindhu Gururaj,

Hello all,

There is a problem with the 1.1.6 release of Mono for Mac OS/X I believe
libgdiplus is missing from the binary installation package.

In version 1.1.5, it is present in Mono.framework/Current/lib it is not
there in the 1.1.6 version.

I can switch back and forth between the two versions and fix or break
existing S.W.F apps.

Was it moved to a different location?  I have programs that run fine on
1.1.5 and throw DLLNotFoundExceptions for libgdiplus on 1.1.6.

I initially thought I had the same problem as Andreas but Adhamh's fix clued
me into the fact that the mapping is pointing to a nonexistant .dylib file.

Cheers,

Oscar
