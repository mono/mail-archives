Using DarwinPorts or Fink in conjunction with Cocoa# will create  
problems like this.  Having separate PATH variables for Cocoa# and  
DarwinPorts/Fink might fix this, but you are better off just not  
installing Cocoa# or Mono from DarwinPorts/Fink.

On Apr 3, 2005, at 4:27 PM, Jake Macmullin wrote:

> I too had this problem. I'm guessing that you've installed some  
> stuff using darwinports, it seems that darwinports also uses the  
> pkg-config program to help configure the software that you're  
> building and installing.
>
> Whilst I don't know much about pkg-config, It seems like it looks  
> through a list of *.pc files to determine which packages have  
> already been installed. The problem arises because the .pc file for  
> cocoa-sharp isn't in the location where the pkg-config executable  
> (from darwinports) expects to find it.
>
> I solved (worked around) this problem, by creating a symbolic link  
> to the actual cocoa-sharp.pc file in the location where the  
> darwinports pkg-config was looking for it:
>
> /opt/local/lib/pkgconfig/cocoa-sharp.pc -> /Library/Frameworks/ 
> Mono.framework/Versions/Current/lib/pkgconfig/cocoa-sharp.pc
>
> Regards,
