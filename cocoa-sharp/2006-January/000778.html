TableView is working great!

Found a little quirk though, not sure if this is normal behavior or not.

If you have a tableview set up with a datasource inside a nib file,  
and then use this code on appwillfinishlaunching, or anywhere else,  
then the problem presents itself...

Ok, you have the nib set up.
Anywhere, if you use the following code, the code works, an item is  
in the table, but if you click on it, then to an empty cell, then  
back on it, it crashes...

myTable.DataSource = new DataSource()
((DataSource)myTable.DataSource).Add("test");    //Still does it  
wether you add item like this, or create datasource and add item then  
assign it
myTable.ReloadData();

The strange part is that when I added console.writeline statements it  
would show the datasource returning a count of 1 item in the list,  
then return that item, then 1 in the list and return that item, then  
0 in the list then crash.  That and if you have more than one item  
added the problem goes away.

Now if you have the stuff set up in a nib file, and DON'T set the  
DataSource property then all is well....

Is this normal?
