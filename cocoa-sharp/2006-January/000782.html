On 16/01/2006, at 12:18 PM, Aaron Flynt wrote:

> You find instructions for contributing documentation in the "How To  
> Contribute" section of the Cocoa-Sharp Wikki, and here:
>
> http://www.mono-project.com/Generating_Documentation
>
> If you're not contributing code changes you won't have to use the  
> 'make update' target as described in the Wikki.
>
> Basically, you just need to edit the xml files in the doc/en folder  
> of the svn repository, and then send your documentation changes as  
> a patch to this list. Eventually we'd like to get CocoaDoc.app set  
> up with an edit mode like the Gtk# monodoc Browser.exe has to make  
> this process easier.
>
> When your done editing, from the doc/en directory, just type:
>
> svn diff > /path/to/where/you/want/the/patch/docs.diff
>
> Then send the resulting file to the list as an email attachment.
>
> -=Aaron=-
