Sorry, can't comment it inline...
- I believe the AlertStyle remarks is the wrong place to document 
default values used somewhere else. This should be in the value element 
of Alert.AlertStyle. This would also apply to other properties, such as 
Panel.FloatingPanel.
- Independent of what mechanism we use for the sheet callbacks, I would 
think we should use Cocoa.Object instead of IntPtr for contextInfo.
- Regarding AlertWithMessage I was actually getting crashes because it 
didn't support null (only String.Empty) as second and third string 
argument - can you add this feature, or do you want me to?
- Regarding Application.ApplicationIconImage documentation: The 
incorrect default icon is a bug in Cocoa#/Mono and therefore should not 
be documented as is. With a different launcher the Info.plist icon is 
working just fine.
- I noticed that you too needed to import [DllImport ("libobjc.dylib")] 
internal static extern IntPtr sel_getUid (string name); I would propose 
to move this functionality into a method public static IntPtr 
Native.ToSelector(string) so that we do not have to import it wherever 
we need it.
