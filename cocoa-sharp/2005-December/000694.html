On 17-Dec-05, at 5:25 PM, Andreas Färber wrote:

> Hello,
>
> Attached is a patch proposal to make doc generation work on Cocoa#  
> from SVN with Monodoc from SVN. Includes a ChangeLog entry (or  
> template ;-)).
>
> I added a path parameter for autoconf's AC_PATH_PROG pointing to  
> the installation location of monodoc's make install target, that  
> is /Library/Frameworks/Mono.framework/Versions/Current/bin. This  
> way it gets detected for me, some documentation generation takes  
> place and CocoaDoc is compiled.
> This idea I got from the official autoconf documentation, but that  
> parameter wasn't explained, so I don't know whether this searches  
> only the path in the path parameter or whether this is an  
> additional location it searches. However, as this is probably the  
> standard location where monodoc gets installed I would propose to  
> commit this change anyway to enable more users to easily contribute  
> to the new Cocoa#.
>
> Best regards,
>
> Andreas
