True. It was not intended to go in yet, but I didn't think of that and 
just did an svn diff. :-$

>> - Independent of what mechanism we use for the sheet callbacks, I  
>> would think we should use Cocoa.Object instead of IntPtr for  
>> contextInfo.
>
> That was my initial feeling but as Kangaroo pointed out, contextInfo  
> could be ANYTHING, so IntPtr was used.

In that case what about object? That way we would free the user from 
calling that himself, which was my intent.

>> - Regarding Application.ApplicationIconImage documentation: The  
>> incorrect default icon is a bug in Cocoa#/Mono and therefore should  
>> not be documented as is. With a different launcher the Info.plist  
>> icon is working just fine.
>
> The icon specified in the Info.plist shows up in the dock when you  
> first launch the appliation, but calling the applicationIconImage:  
> selector returns the default one.

Finder/Dock access the bundle you launch. However when using the 
macpack-generated launcher it is not using the correct bundle - probably 
looking at the location of mono instead, or doesn't it look for any 
because mono is a C++/Carbon app?

> What launcher are you referring to?  Not the one generated by macpack, 
> right?

Right, I'm currently using Gareth Baker's launcher written in 
Objective-C, posted on this list some months ago for the previous 
version of Cocoa#. This avoids these problems but as binary code makes 
the bundle platform-dependent.

> Since this is the current behavior of the library I think the  
> documentation should warn people about this and give them a  
> workaround until it's fixed.

I'm not saying we can't use this as a way of telling people a 
workaround; I just objected to the description of the "default value". 
The default value is in fact the specified bundle icon or the default 
icon if no bundle or custom bundle icon is found.
But I don't see any problem writing something like "By setting this 
property you can fix the icon" into the remarks section, that's what 
it's for after all. :-)
