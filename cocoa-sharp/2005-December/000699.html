> I have installed Mono 1.11 on Mac OS X 10.4, like Pamalite. Then I  
> added a plugin to XCode to create Cocoa# Projects, but when I look  
> linked Framework, XCode can't found Cocoa# Framework.
>
> Is this installation version like Mono 10.1 (without Cocoa#) ?

No, but 1.1.11 includes a new version of Cocoa# that is incompatible 
with old code. The Cocoa Foundation and AppKit classes are no longer in 
the Apple.AppKit namespace but instead in Cocoa. You will need to change 
all references to e.g. NSWindow to Window and e.g. .enabled to .Enabled.
When you're done with that migration you might notice that some features 
you had been using are not yet available - in that case you will need to 
use Cocoa# from SVN (as outlined here: 
http://www.cocoasharp.org/cocoaSharp/default.aspx/CocoaSharp/HowToContribute.html) 
and contribute the classes or methods/attributes you need.

Don't know about XCode.
