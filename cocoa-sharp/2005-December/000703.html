CocoaDoc.app runs fine for me without crashes, but I do get these  
errors reported in Console:

2005-12-18 09:41:06.113 CocoaDoc[28394] _NXCreateWindow: error  
setting window property (1002)
2005-12-18 09:41:06.116 CocoaDoc[28394] _NSSetWindowTag, error  
clearing window tags (1002)
2005-12-18 09:41:06.116 CocoaDoc[28394] _NSSetWindowTag, error  
setting window tags (1002)
Error: did not find one of the files in sources//Library/Frameworks/ 
Mono.framework/Versions/1.1.11/lib/monodoc/sources/cocoa-sharp-docs

The only PITA that I haven't found a workaround for yet is that it  
doesn't look in the sources directory in the bundle for the docs, so  
to preview documentation changes I have to copy the generated cocoa- 
sharp docs to /L/F/M/V/C/L/MD/S, and the monodoc.xml file to /L/F/M/V/ 
C/L/MD.

Also, the basefile attribute in doc/cocoa-sharp.source needs to be  
changed from "cocoa-sharp-docs" to "cocoa-sharp" to match the prefix  
of the generated doc files (this is the reason for the last error  
above).

Another option you have for viewing your documentation changes is to  
use the monodocs2html command as follows:

monodocs2html --source=/path/to/doc/en --dest=/where/you/want/docs
