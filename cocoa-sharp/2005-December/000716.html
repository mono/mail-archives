Hi kangaroo,

> Problem #1; you've changed (correctly) the signature for the Window  
> with stylemask/backingstore but havn't changed CocoaDoc.cs in the  
> samples folder; so it will not compile.

Now, that is not correct: CocoaDoc is using a nib and _not_ my modified 
constructor. Have recompiled it, fixing some changes required for my 
Control changes, and verified it's still not running.
But thanks for making me aware of the samples!
Might it be useful to include a dependency on cocoa-sharp.dll for the 
samples? That way they would get rebuilt whenever cocoa-sharp.dll is 
compiled.

> Problem #2; coding style.  We use the mono coding style exclusively  
> and you've got that in about 95% of the case but not all.  I'll point  
> out a few.

No, you don't ;-) - but I've changed it where I've noticed it (and am 
keeping my opinion to myself).

I'm attaching a new patch which includes the previous changes, plus the 
requested changes, plus my Control changes. Control now has two events, 
Action and DoubleAction, of (standard) type EventHandler. When an event 
handler is attached to one of them a new class instance is created and 
set as the target, and exported functions dispatch the event to the 
registered delegates. CocoaDoc was adapted to the changes but could not 
be tested; the Action event however is working fine for Button in my 
application. Documentation was updated in a couple of places.

Andreas
