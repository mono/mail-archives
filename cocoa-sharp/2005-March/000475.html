Double-clicking on an app created with macpack will fail to start the app if
the path to the app has a space in it. Ditto for the name of the app. This
affects both Cocoa# and MWF apps on Mac OS X.

The last two lines of the script created by macpack to launch apps needs to
be changed from

cd $APP_PATH/Contents/Resources
mono $APP_NAME

to

cd "$APP_PATH/Contents/Resources"
mono "$APP_NAME"

I've opened a bug on this and assigned it to kangaroo at his request:
http://bugzilla.ximian.com/show_bug.cgi?id=74221
