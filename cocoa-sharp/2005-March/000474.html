I'm seeing this same problem with the CurrencyConverter example, and 
also with a simpler test file generated using NibWrap.app.

Here's what I've discovered so far.

It looks like the resource path for the application's MainBundle is, for 
whatever reason, set to /usr/bin.  And thus LoadNamed_owner is looking 
there for the nib.  If I place the simple test app's nib into /usr/bin, 
the app correctly loads it.

Thoughts? Comments?

-Andy
