There are a couple issues here;

First; Cocoa# in general:
	We have a generic Mach-O Binary/Header parser which will create stubs 
(in theory if its bug free) for any .framework.

Second: Tiger
	The Tiger seeds that are out today choke on our generator (for various 
reasons; they've introduced some new binary types).  We're reluctant 
(for lack of a better word) to fix these issues now; as the format 
changes (as they relate to us) are still a bit of a moving target.  
When tiger is finalized we will patch the generator to work with AppKit 
and Cocoa; if you need to generate CoreData for any reason and the 
generator chokes; feel free to file bugs (AT THAT TIME).

If you have a valid argument as to why it should be included in the 
default distro (keeping in mind that would be exposing API to panther 
users who can't use it) and a plan to prevent Panther users; feel free 
to submit it (patches preferred :))
