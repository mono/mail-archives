MonoDevelop instructs like the Gtk ones;

The reason for 2 packages from CVS?  The stable generator in gtk# v1.0  
generates uncompilable code; I'm not 100% certain this is a OSX only  
problem; but fairly confident in that assesment (it might be a gtk  
versioning problem)


_WARNING_:  These are not for the feint of heart; you'll have to hack 1  
configure file and fix some dll mappings (thats about it)


1. Follow "GTKSharp.OSX.Build.txt" and make sure Gtk# is working.

2. Install the following fink packages:
fink install gtksourceview gtksourceview-dev gtksourceview-shlibs  
mozilla-dev

3. Install monodoc-1.0
PKG_CONFIG_PATH=/sw/lib/pkgconfig:/usr/X11R6/lib/pkgconfig \
PATH=/usr/bin:$PATH \
./configure --prefix=/Library/Frameworks/Mono.framework/Versions/1.0
make
sudo make install

4. Install gecko-sharp
PKG_CONFIG_PATH=/sw/lib/pkgconfig:/usr/X11R6/lib/pkgconfig \
PATH=/usr/bin:$PATH \
./configure --prefix=/Library/Frameworks/Mono.framework/Versions/1.0
make
sudo make install

5. Install gtksourceview-sharp (FROM CVS)
PKG_CONFIG_PATH=/Library/Frameworks/Mono.framework/Versions/1.0/lib/ 
pkgconfig:/sw/lib/pkgconfig:/usr/X11R6/lib/pkgconfig  
PATH=/usr/bin:$PATH:/Library/Frameworks/Mono.framework/Versions/1.0/bin  
./autogen.sh --prefix=/Library/Frameworks/Mono.framework/Versions/1.0

Manually remove the broken lines in ./configure
Run:
PKG_CONFIG_PATH=/Library/Frameworks/Mono.framework/Versions/1.0/lib/ 
pkgconfig:/sw/lib/pkgconfig:/usr/X11R6/lib/pkgconfig  
PATH=/usr/bin:$PATH:/Library/Frameworks/Mono.framework/Versions/1.0/bin  
./configure --prefix=/Library/Frameworks/Mono.framework/Versions/1.0
make
sudo make install

6. Install svn
7. Install monodevelop
svn co svn+ssh://mono-cvs.ximian.com/svn/monodevelop/trunk/MonoDevelop
export PATH=/Library/Frameworks/Mono.framework/Versions/1.0/bin/:$PATH
export  
CFLAGS="-I/Library/Frameworks/Mono.framework/Versions/1.0/include/"
ACLOCAL_FLAGS="-I  
/Library/Frameworks/Mono.framework/Versions/1.0/share/aclocal -I  
/sw/share/aclocal"  
PKG_CONFIG_PATH=/sw/lib/pkgconfig:/usr/X11R6/lib/pkgconfig ./autogen.sh  
--prefix=/Library/Frameworks/Mono.framework/Versions/1.0
make
sudo make install

8. Add the following mappings to $PREFIX/etc/mono/config
         <dllmap dll="libgtk-x11-2.0.so.0"  
target="libgtk-x11-2.0.0.dylib"/>

Correct the mappings in $PREFIX/lib/monodevelop/bin (change .so.0 ->  
.dylib):
         MonoDevelop.Gui.Utils.dll.config
         MonoDevelop.SourceEditor.dll.config
         gdl-sharp.dll.config
                 (Add:   <dllmap dll="gdldock"  
target="libgdldock.dylib"/> to this one)



9. Run monodevelop witht he following command:
export  
DYLD_LIBRARY_PATH=/sw/lib:/Library/Frameworks/Mono.framework/Versions/ 
1.0/lib/:/Library/Frameworks/Mono.framework/Versions/1.0/lib/ 
monodevelop/bin/
MOZILLA_FIVE_HOME=/sw/lib/mozilla/ ./monodevelop
