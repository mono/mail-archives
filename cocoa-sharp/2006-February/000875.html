A Breif Explaniation (From Glancing at the Src weeks
ago)
Basiclly what it does is it uses the Mono Embedding
functionality to the Mono Environment... Its held
within a neat ObjC Wrapper... see:
http://www.mono-project.com/Embedding_Mono (C
Implementation)

Personally i think its a great idea for using Mono
resources within a current ObjC/ObjC++ Application...
Might even be usefull to create a Mono OSAX
AppleScript Extension...

As for creating apps using it... Im afraid i see only
1 point... and that is, for a Cocoa developer knowing
only C/ObjC(++)/C++ to embed Mono functionality and
access to mono code in to their current base.

Ive been meaning to play with both Dumbarton and Mono
Rmbedding however i think that all has to wait till i
can replace the HD in my iBook.
