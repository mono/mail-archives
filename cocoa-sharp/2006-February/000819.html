- == instance call
+ == static/class call

Tim Davis wrote:
> I'm still working on NSCell conversion, and I'm almost done but I just 
> noticed something...
>
> Some of the entries in the apple docs have a - or a + beside 
> them....what does that signify?
>
> Example..
> – mouseDownFlags 
> <http://developer.apple.com/documentation/Cocoa/Reference/ApplicationKit/ObjC_classic/Classes/NSCell.html#//apple_ref/occ/instm/NSCell/mouseDownFlags>
>
> Returns the modifier flags for the last (left) mouse-down event, or 0 
> if tracking hasn’t occurred yet for the cell or no modifier keys 
> accompanied the mouse-down event.
>
> + prefersTrackingUntilMouseUp 
> <http://developer.apple.com/documentation/Cocoa/Reference/ApplicationKit/ObjC_classic/Classes/NSCell.html#//apple_ref/occ/clm/NSCell/prefersTrackingUntilMouseUp>
>
> The default implementation returns NO 
> <http://developer.apple.com/documentation/Cocoa/Reference/ObjCRuntimeRef/Reference/reference.html#//apple_ref/doc/c_ref/NO>, 
> so tracking stops when the cursor leaves the NSCell; subclasses may 
> override.
