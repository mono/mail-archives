Hey folks I checked in the latest genstub.pl modified version that cj sent
earlier.

I also checked in the generated stubs, I had to manually modify about 3
files to make them compile (NSObject, NSArray, ...): 

1.	oneway prefix has to be ignored --> 'NSObject release' for instance
2.	A header file can contain methods for multiple interfaces and right
now the header file used to determine the interface name rather then the
surrounding 'interface' tag, causing double defined methods and methods to
be defined in the wrong class/interface.
3.	We don't deal with @interface NSString(Extra) construct at all, this
is some funny objective-c construct allowing someone else to extend the
NSString interface with the 'Extra' extension.  I don't know too much about
extensions and how they would map into .net but we can look @ the java
binding for some ideas about that.
4.	I fixed the issue that sometimes the parameter type is missing: then
'id' is implied
5.	We don't generate .cs files @ all :(

More to come this is just on top of my head.

- Urs
