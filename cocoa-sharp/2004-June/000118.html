Did a couple hours of hacking tonight and came up with this;

You specify the frameworks and search paths in a configuration file 
(generator.xml by default; overridden with -xml:foo.xml).  The code 
will process each framework; and properly detect:
interfaces
protocols
categories

Each of these is created as a class instance of a 
Protocol/Category/Implementation (look at the code to see) which all 
keep an ArrayList of methods.  Any method found contained in them gets 
added ot the methodlist of the current place we're at.

The 3 types are stored on a per-framework basis as well; but are 
exported public so that other frameworks can search in them for the 
data.  They might all work as globals; I'm not sure if we'd have name 
clash in that case (its a little late; I'm not htinking 100% straight).

The big holes in this right now; it doesn't jump over comments; we'll 
need an appropriate regex to hop over them; and Method.cs needs to 
properly read the incoming method and parse it to its glue and C# 
counterparts.

Before I carry on with those bigger parts; or someone else does; take a 
look at the code and lets agree this is the type of direction we want 
to go in.  This would easily allow us to have a NSObject.custom which 
would put custom code in a #region block in the generated file; and 
also to override methods (NSObject.overrides; think of it as an xml 
with a key and a value; where key is the hashtable key; we deserialize 
it; see if the key is there before adding the one from the IDictionary; 
if so we override).
