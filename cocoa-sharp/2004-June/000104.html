Hello,

> My concern regarding the naming is that Object, Array, String, Number and a
> few others have counterparts in System, and this becomes a namespace
> conflict if a user does the following:
> 
> using System;
> Using System.Collections;
> Using Apple.NS.Foundation;
> 
> Namespace myNameSpace
> {
>     Class foo 
>     {
>         Foo()
>         {
>             String someString = new String();
>         }
>     }   
> }
> 
> Is someString a System.String or is it an Apple.NS.Foundation.String?

To begin with, I think that NSString should be hidden, there is no point
in the user knowing anything about NSStrings, NSObjects and NSArrays. 
They are part of a different universe.

Lets go back to my example with Gtk+: Gtk has a GString type, and a
GArray type, and a GList type.  

And we worked very hard to remove every one of those from the public
API, as they overlap with the first-class citizens of the .NET world.  I
think the benefit is big.

My job is just to pass this information to you guys, maybe its useful to
you, maybe its not.  

If you find this discussion useful, we can continue it, but at this
point with the differing opinions I do not think I can be of any further
help. 

Miguel
