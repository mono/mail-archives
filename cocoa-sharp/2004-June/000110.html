Some issues that have to be addressed with the stubs generator:

-                      A header files is not equal to a class/interface -->
we can't output a .net class per header file, we have to collect all the
methods of a class/interface and the output the class/interface after
parsing all the headers

-                      In the type mapping we have two different mappings

o        One mapping is from C/ObjC to .Net for the static PInvoke methods 
(for instance NSString* --> IntPtr, SEL --> IntPtr, BOOL --> bool, NSView*
--> IntPtr, id --> IntPtr)

o        The other mapping is for public class and instance methods
(for instance NSString* --> string, SEL --> string, BOOL --> bool, NSView*
--> NSView, id --> object)

o        If glue and public function does match the type (as shown) use
Net2OC for out and OC2Net for inbound objects (as shown in NSButton.cs)

-                      Method name: that .net allows method overloading so
we can drop the additional message parts and just use the first part as
public name

-                      We have to treat property like methods differently

o        A pattern of [-/+] (TYPE) propertyName; and [-/+] (void)
setPropertyName: (TYPE) argName; can be translated into
public TYPE PropertyName { 
   get { return $convOut($glueGetMethod(Raw)); }
   set { $glueSetMethod(Raw,$convIn(value)); }
} (as shown in NSButton.cs)

-                      No hard coding for Foundation/AppKit

-                      NS*.h should not be a filter for header parsing

-                      We have to parse enum types as well, so that we can
add them to classes (as shown in NSButton.cs)

-                      We should also parse struct types like NSRect,
NSPoint, NSSize, to convert into .Net structs as shown in Apple.AppKit

-                      Create standard constructors, and convert all init*
into constructors too (as shown in NSButton.cs and NSControl.cs), note that
init* function of the parent type have to be repeated, otherwise the default
constructor is not called, which allocates the right type of native object
