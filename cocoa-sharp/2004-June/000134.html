All I have to say to that is the following:

The whole and only reason for having sym-link in /usr/bin is that the =
mono
compiler and tools would be available without the user having to do =
anything
other then installing the Mono.framework.
Moving the anywhere else defeats the purpose of having sym-links and =
the
user could just add /Library/Frameworks/Mono.framework/Commands to this =
path
to achieve the same.

Applications can safely assume /Library/Frameworks/Mono.framework for
binding to the runtime, and tool integrations like Xcode should do the =
same.

If that is not acceptable by Apple, then Apple has to come up with a
comparable solution to the end user (just installing the =
Mono.framework) to
have the runtime, the compiler and it's tools available to the user =
within
the terminal.  (But that is just my opinion).

Just my 2 cents.
