On Jun 19, 2004, at 12:23 AM, C.J.Collier wrote:

> Would you mind enlightening me a bit?
>
> What is ObjC#?  How does it relate to Cocoa#?  Does Cocoa# supersede 
> it?  Is one a .NET wrapper around Cocoa while the other is a Cocoa 
> wrapper around .NET?
>

ObjC#:  ObjcC -> Mono  (Really its Foundation -> Mono)
Cocoa#:   Mono -> Cocoa   (Really its Mono -> Foundation/AppKit)

> If this is true, shouldn't one appease one audience while the other 
> does the other?

Yes.  But I don't think the "demand" is going to be for ObjC#.  If you 
are going to write code for OS X then you are going to use Cocoa 
straight up.  I think the real use for this is for Mono developers to 
be able to get their apps to run on OS X.
