Great.  In that case, we need to make the API as easy-to-learn by people
who use .NET on Unix platforms as possible.

What should that API look like, if it is not going to look like ObjC and
Cocoa?  I vote that we make it look like glib#.  The closer we can get
the API to look, the more comfortable those who develop for glib and
glib# will be with Cocoa#.

Please discuss whether you agree or not.  We should specify the API at
some point, and we might as well discuss it so that we get it right.

Cheers,

C.J.

PS, Miguel, if you don't want to be involved in this discussion, please
let me know and I won't cc you.  I assume that you are not subscribed to
this list.
