Thanks Oscar,

Finnaly, I solved my problem by changing the way of starting my app !

I used to to this :

static int Main(string[] args)
{
     Application.Init();
     Application.LoadNib("MyNib.nib");
     Application.Run();
     return 1;
}

But with this way all my "ressource path" problems disappeared...I  
must admit I really don't know why !!!

static int Main(string[] args)
{
     if(!NSBundleAppKitExtras.LoadNibNamed_owner(" MyNib.nib",  
NSApplication.SharedApplication))
         return -1;

     //This is the one way to start the Cocoa run-loop and its  
probably the simpliest.  More advanced
     //ways to interact with the user are available and documented at  
http://developer.apple.com.
         //At this point the nib has loaded and the application is  
ready to interact with the user.
     NSApplication.SharedApplication.run();
     return 1;
}
