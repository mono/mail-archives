> I've noticed from various samples that I've found along with looking
> at the source to the Cocoa# monodoc that you guys put various debug/ 
> console output statements in your code.  How can you run a Cocoa# app  and 
> have a console window visible to see these outputs?  That would  be real 
> handy right about now :)

Run a standard OS/X program called Console.app (not to be confused with 
Terminal.app).  It receives the stdout messages from programs that do not 
start from a command line or from within XCode (which has it's own console).

And yes, it is very handy.
