> (on my system I did successfully build Hello.exe but Mono crashed

This can happen when you run "mono Hello.exe" and it doesn't find 
libCocoaSharpGlue.dylib. Did you use the macpack utility as outlined in 
the CocoaSharp.org Wiki?

> I did look at the source tree via the web interface but did not find  
> a large collection of C# classes mapping NS classes.

Don't know about the repository, but there are lots of NS* CLR classes 
in the Apple.Foundation.dll and Apple.AppKit.dll assemblies installed as 
part of Mono.
As the CIL bytecode is independent of the C# programming language, the 
classes can just as well be generated using System.Reflection.Emit or 
any tool that outputs CIL. Overhearing some of the more technical 
discussions here it appears the so-called "generator" does this.
