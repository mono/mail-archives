We decided to integrate because you can't use Cocoa# without Mono.  
Cocoa# is always installed in the current mono framework.

This is pretty typical with dependent frameworks.

> Hi Pat,
>
>> Hi, I was just wondering, are we now putting everything in 
>> Mono.framework instead of having a seperate CocoaSharp.framework? 
>> When I built from the nightyl tarball it put things in 
>> Mono.framework. Not that it is a big deal but I need to know in order 
>> for my Xcode template to work.
>
> I have no authority but I do have an opinion if we are putting it to a 
> vote.
>
> I would like to see them separate.
> Since Cocoa Sharp is on a separate release schedule to Mono, it really 
> doesn't make sense to treat them as if they are one and the same.
> This would also allow for use of Cocoa Sharp from runtimes other than 
> mono.
>
> Also, which Mono should it be installed in? Current? 1.02? 1.04? 1.1.x?
>
> What if I install 1.02, install CocoaSharp, then upgrade to 1.04? Do I 
> lose CocoaSharp since the Current symlink has changed?
>
> It's much easier to reference CocoaSharp.framework.  That way, the app 
> decides.
