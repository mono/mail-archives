Hi,

I have a couple of suggestions after trying the new template. It also 
relates to the "what libraries to copy in" debate.

Firstly I think that the comments in the script should be "#" rather 
than "//"

Now - LIBRARIES. I can actually get the test program (main.cs) to run 
WITHOUT any libraries copied in... If you double click on the 
executable within XCode you get a dialog with a number of tabs: The 
"General" tab is where you should set the working directory for the 
executable; the second tab "Arguments" is where you can set an 
environment variable which is passed to the executable, here you can 
enter DYLD_LIBRARY_PATH to point to 
/Library/Frameworks/Mono.framework/Versions/Current/lib and this seems 
to negate the need to copy all the libraries into the application.

Well it works for me - YMMV :-)
