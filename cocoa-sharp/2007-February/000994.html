Todd, I don't think that's a fair assessment. While I do tend to agree that
the project could use more leaders and more activity in general, I think you
must have missed or forgotten Geoff Norton's e-mail to
<cocoa-sharp at lists.ximian.com> on 12 Feb 2007. I'll include a copy here for
easy reference (more discussion from me below):

------- Start of message excerpt -------
Date: Mon, 12 Feb 2007 23:02:17 -0500
From: Geoff Norton <grompf at sublimeintervention.com>
To: David_Hudson at capgroup.com
References: <OFF6B43262.75114855-ON88257280.00803BAF-88257280.00816943 at capgroup.com>
Cc: cocoa-sharp at lists.ximian.com
Subject: Re: Anybody mind if I do some cleanups?

My personal opinion is; I'm still the de factor maintainer; so I'm still 
approving all patches that go into the repository.

Please send all patches to the list as per CONTRIBUTING (iirc) in the 
repos; and I'll approve and/or commit accordingly.

-g

------- End of message excerpt -------

Note that the above e-mail clearly states that Geoff would prefer any
Cocoa# patches to be mailed to the mailing list for review. By "as per
CONTRIBUTING in the repos," Geoff is referring to the following document:

http://svn.myrealbox.com/viewcvs/trunk/cocoa-sharp/CONTRIBUTING?rev=53792&view=markup

For Mono/OSX stuff unrelated to Cocoa#, bugs and patches should be filed in
Ximian Bugzilla. Refer to the following web pages for more info:

http://www.mono-project.com/Contributing
http://www.mono-project.com/Bugs

Hope this helps,
Ed
