Am 23.02.2007 um 18:40 schrieb Andrew Satori:

> Core2Duo is an INTEL chip. ( http://www.intel.com/core2duo/index.htm?
> iid=homepage+c2d )
>
> In most Mac configurations, it is the Intel 64 bit architecture
> implementation.  [...]  Only the 2nd generation Mac Intel hardware  
> with the
> Core2Duo's sport the 64bit implementation, but even then, the
> operating system isn't truly 64 bit, and only supports a subset of 64
> bit operation through Intel libraries.  Not until Leopard will 64bit
> be really relevant on the Mac x86 platform.
>
> The only place that 64bit Mac operation is valid is on the G5, where
> PPC64 *is* an option, however at this point in time, it would be a
> waste of time and effort to bother with it since the PPC is
> effectively a dead end on the Mac.

To my knowledge Paolo is right in referring to amd64. The Intel 64- 
bit architecture (IA64) is the Itanium, which as I understand has  
nothing to do with Core (2) Duo. Itanium was a flop (mostly) so Intel  
is now using AMD's 64-bit architecture.

I am actually running on ppc64. Tiger does allow 64-bit console  
applications so if supported a 64-bit Mono would be usable today.  
Leopard will be fully 64-bit *and* will continue to support the ppc64  
architecture. So it is not dead for some time to come.
