Sorry... In May I mentioned a Perl script called "tinyglot" which I 
found for merging .strings files for localization. Doing real menu 
translations now I've recently noticed some serious problems with it, 
breaking my incremental translations. It appears there's a UTF-16 
encoding problem, the author has been notified of the bug.

As an immediate solution for my project I've written a managed command 
line utility that reads nibtool's plist format output directly from 
stdout and generates or merges it and an XLIFF 1.1 file, from which it 
can generate the localized .strings file as input for nibtool. It makes 
some assumptions about nibtool's output (including its one-line 
comments) but so far it works fine for me (on 10.3.9). So if anyone is 
facing similar problems and is interested in my utility I'd be happy to 
share it.
