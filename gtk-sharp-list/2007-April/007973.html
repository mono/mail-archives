Nothing can go into gtk-sharp that isn't in the platform library set.
Gtk# is a blessed binding now and is restricted in what it can bind.  

At this point, we're only considering Gnome platform library bindings
for inclusion in Gtk# (and now Gnome#) even though we currently have a
few grandfathered non-platform libs.  We've experienced some pain trying
to bind libraries with no API stability guarantees.

So I guess the way to get this into Gtk# is to get it into gtk+.
