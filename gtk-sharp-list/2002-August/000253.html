Was there a particular bit of gnome-vfs you were needing?

I ask this because I think application authors would (or should) rather
use the built-in .NET classes for this. If there are URI
incompatibilities between Gnome widgets (file selector) and the .NET URI
facilities, we should provide some sort of compatibility mapping. We
might want to provide some sort of .NET-ish way of accessing the
"custom" directories in gnome-vfs such as the vfolder ones. But I can't
imagine ever writing a straight binding for gnome-vfs, custom solutions
using the .NET interfaces is really the way to go here.

-- Rachel
