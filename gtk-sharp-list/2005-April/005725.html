Announcing release 1.9.3 of Gtk#.  This is an unstable development
release for the GNOME 2.6 bindings.  It is parallel installable with the
latest stable release.

Tarball and binaries are available for download from:

http://www.mono-project.com/Downloads


About Gtk#: 

Gtk# is a .Net language binding for the Gtk+ Graphical User Interface
toolkit and several GNOME platform libraries. The GNOME 2.6 platform is
targeted.

Changes since 1.9.2:

* Improved Container Child API. (Dan Winship)
* Improved null handling in the generator. (MK)
* Manual string marshaling for win32 utf8 encoding support. (MK)
* CDecl native to managed callback mapping for win32 assemblies. (MK)
* Improved memory management of callback parameters. (MK)
* Profiling and leak-plugging (Ben Maurer, Joe Shaw, Jon Trowbridge, MK)
* Improved (version-friendly) GType to GLib.Object mapping (MK, Paolo
Molaro)
* Substantial scrub of GtkDemo sample (DanW)
* Container enumerator support and new sample PolarFixed. (DanW)
* configuration support for debug info with --enable-debug (DanW)
* Additional API customizations and Corrections: (Alexandre Gomes, DanW,
Jeroen Zwartepoorte, MK)
* Bugfixes (Raja Harinath, John Luke, Anthony Taranto, Alp Toker, DanW,
Jeroen Zwartepoorte, MK)
* documentation additions (DanW, MK)


Discussion of Gtk# occurs on the mailing list at
gtk-sharp-list@lists.ximian.com.  Bugs can be reported to
http://bugzilla.ximian.com, module gtk#. Check the Gtk# homepage for
additional info at http://gtk-sharp.sf.net.
