On Sun, 2003-02-23 at 13:00, Sam Clegg wrote:
> Is anyone else interested in wrappers for monkey-media? 
> 
> I hacked some together the other week before noticing the API
> generator used in gtk-sharp.  Should I attempt to add monkey-media to
> the list of wrapped libraries?  Or is it still too volatile an API?  

There are plans to merge monkeymedia's functionality into GStreamer's
metadata architecture. If you can't wait for this to happen, I'd advise
maintaining the monkeymedia bindings separately as is indeed not a
stable API.

Hopefully when things settle down a bit it'll become more clear where
bindings for 3rd party libraries should be stored and developed.
