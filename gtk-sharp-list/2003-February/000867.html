> There is a stable API?  Can I use gtk-sharp with gtk2.2?

The API is not stable, but it is also not changing quickly.  The
majority of API changes lately have been to add overloaded convenience
methods, or to fix bugs in the generated API.

I will probably wait to update Gtk# to the Gtk2.2 api when Gnome 2.2 is
released so that I can do it once for all the wrapped libs.  It's not
just laziness on my part.  There really aren't many distributions
tracking Gtk2.2 yet, so binaries aren't widely available.
