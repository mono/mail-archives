I was making a new project on Novell Forge. They are asking me some
questions that I think I better check on you before replying:

 1. Project full name:
> Gtk.Controls

 2. Project purpose and summarization (for Novell Forge, not the public):
> We intend to make a set of controls that eventually would end up being adopted by Gtk#.
> These controls would be written on C# and written using the Gtk# toolkit. We plan on
> using the SVN and mailing list while the prorject is still small enough to be managed, and
> Bugzilla or others as we grow or the need arises.
>
> Our controls are not necessarily our own creation, we will take interesting controls locked
> away inside the sources of open source projects and merge them together underneath
> the Gtk.Controls heirarchy, so that people can have easy access to them from any
> project. That is way we hope to be adopted into Gtk#.
>
> We will start making a Datagrid control, that will be able to fetch its data from the
> program running it or from ADO.NET. This control will also work on ASP.NET. This
> control should provide read/write access to a data set, have support for foreign key
> fields, support for different data types (numeric, string, date, time, boolean, generic
> objects, perhaps large objects too in the long run).
>
> We are thinking of other controls too, we have started to discuss using Lilly's Dock
> Toolbars on MonoDevelop. We are willing to accept more controls later.

 3. License:
> LGPL (according to Daniel Morgan's statement on Gtk#'s license)

 4. Short public description:
> I have thought of using the project purpose and summarization for this purpose.

 5. Short name:
> Gtk.Controls
