Hello!
I've recently tried gtk-sharp and discovered that
Gdk.Drawable.DrawPolygon is buggy (it requires one point
where an array is needed). I wondered how this can be
corrected and it seems that the only requirement is to
place array="true" in xml api description (where a pointer
means an array). I also found that some other drawable
members are incorrect. The diff to gdk-api.xml that
fixes some of them is in attachment.
Regards,
Artemis.
