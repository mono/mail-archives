> On Tue, 2003-03-18 at 00:14, Charles Iliya Krempeaux wrote:

[...]

> Why all the $(OUTPUT) stuff?  Seems gratuitous to me, and makes this a
> much larger and harder to review patch than it needs to be.  Unless
> there's a good reason to do all that, please remove it, resubmit a patch
> for the uninstall stuff, and I'll review it then.

The $(OUTPUT) stuff was added to make it easy to name the .dll files
to something else if we ever choose to.  (Or if the user wanted to.)

(It makes it so you can change the name of the .dll file by editing
the Makefile.in file in only one place.)

Like if someone wanted to use the "Mono" convention for naming .dll
files.  (I.e., using the namespace name.  And have Gnome.dll instead
of gnome-sharp.dll.  Have Gnome.VFS.dll instead of gnomevfs-sharp.dll.
Etc.)  Then they could (more) easily do this.

I was thinking of (eventually) making a patch for this (for 
"configure.in") to make it so the user could use various switches to
name the .dll files.  (And this was a first step to doing it.)
