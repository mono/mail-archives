Gtk# 2.x has reached stable released status.  Packages and source
tarballs are now available for gtk-sharp-2.4.0 at:

http://www.mono-project.com/Downloads

This release is based on the efforts of dozens of contributors over the
last 15 months.  Our sincere appreciation goes out to all the
contributors.

We've assembled a page on the Wiki to highlight some of the new
features:

http://www.mono-project.com/GtkSharpNewInVersion2x

Version 2.4.0 provides bindings for gtk+ 2.4 and several GNOME 2.6
libraries.  Note that we have also released a source tarball for version
2.6.0 which binds gtk+ 2.6 and several GNOME 2.10 libraries.  Both
releases are stable.  There are no current plans from the mono project
to provide packages for the 2.6.x releases, but the tarball is available
for download at:

http://go-mono.com/sources/gtk-sharp-2.0/gtk-sharp-2.6.0.tar.gz

We have also released an unstable version of gtk+ 2.8 bindings for
people who want to start experimenting with the cairo API.  This release
is not for the faint of heart, and comes with no API stability guarantee
for the new 2.8 API.  Again, this is a source only release and the
tarball is available for download at:

http://go-mono.com/sources/gtk-sharp-2.0/gtk-sharp-2.7.1.tar.gz

Since 2.8 is a relatively small API change over 2.6, we hope to quickly
move to stability.  If you find any issues, with the new API, please
raise them promptly to ensure they are fixed.  As always, please file
bug reports for any issues at http://bugzilla.ximian.com under product
gtk#.
