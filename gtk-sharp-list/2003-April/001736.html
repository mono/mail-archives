Hi,

I think that we need to redesign Gtk#. I think that we need two versions
of Gtk#: one identical to Gtk+ and other with very changes in the
design.

The first thing I would note is that the most classes in Glib can be
obtained from CTS, then, I think that Gtk# must use CTS ones. There are
a lot of out structs as parameters that must be objects (and we save the
out value using references). Another thing is the GtkSharp namespace: it
must be needed only Gtk namespace (and Gdk, Glade, etc... for specific
things.

Because change all of these things makes Gtk# very different of Gtk+, I
said that there should be two versions (automatically generaged, ok, but
two).
