I have prepared a tarball for people wanting a recent version of Gtk# to
go with the new mono release.

Source is available at:
http://primates.ximian.com/~tvgm/gtk-sharp-0.6-pre1.tar.gz

About this pre-release:

* This is a pre-release. It has not been tested on Windows, or any
other platform than Debian GNU/Linux 3.0 x86. Mike was not consulted
regarding this pre-release, nor was he available at the time. This
is not an official release; system packagers should not release
binary packages for it.

* I have confirmed that Gtk# works with Mono 0.17, but as that version
shipped with finalizers disabled, large Gtk# applications running under
Mono 0.17 will leak memory like a sieve.=20

* The samples appear to work. I cannot confirm that CVS mphoto,
or any other Gtk# application, works with this release, because
mphoto crashes for me right now.


Preliminary release notes
-------------------------

About Gtk#:=20
Gtk# is a C# language binding for the Gtk+ Graphical User Interface
toolkit. The binding is fairly complete, but it is still
largely unproven and not recommended for production work.=20

Changes since 0.5:
* Namespace fixes: Mike
* GStreamer support: Alp
* Configuration summary: Gonzalo
* Compilation fixes: Duncan (scribble), Gonzalo (DbClient)
* New GNOME.NET tutorial: Johannes Roith, Martin Willemoes Hansen
* New tutorial sample code: Alejandro Snchez Acosta
* Clipboard fixes: Vlad
* New .NET-style struct null handlers: Vlad
* Refcounting fixes: Vlad
* UTF8 conversion of source files: Gonzalo
* Add pango constants: Daniel Morgan
* Introspection and GValue object store fixes: Radek
* New dialog constructor: Mike
* Metadata fixes and updates: Mike
* TextBuffer convenience "Text" property: Duncan
* EventArgs dynamic construction fixes: Gonzalo
