This looks like a native segfault. Some of the native libraries you
are using are getting into invalid state and crashing.

You should track down those GLib-GObject-WARNING etc messages to try
to find where the invalid operations are being performed. You can do
this by installing GLib log handlers -- take a look at
http://anonsvn.mono-project.com/viewcvs/trunk/monodevelop/main/src/core/MonoDevelop.Core.Gui/MonoDevelop.Core.Gui/GLibLogging.cs?view=markup
for an example.

That said, these errors might be a result of that odd
"CGBitmapContextGetBitsPerPixel: invalid context" error, which appears
to be GTK-Mac-specific.
