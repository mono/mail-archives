On Sun, 2003-12-07 at 12:59, jeroen wrote:

> It's normal for applications to only show 1 GnomeAbout dialog at a time.
> So selecting Help->About again, show only raise an existing dialog.

It is not normal for users to be launching an About dialog multiple
times in a session.  About dialogs are copyright holders and vanity
mechanisms for developers, not a feature that users demand frequently. 
The only thing I ever use them for is to check a version number if I
think I might need to upgrade.

Gnome.About is "show-once" by implementation.  Trying to make it
something else is just not something worth spending time on, IMO.

> What's the proper way of doing this in C#/Gtk#?

Don't hold refs to Gnome.About.  Create a new one each time it's
requested. Make it modal if you really think users would be tempted to
leave one laying around and you don't want duplicates.
