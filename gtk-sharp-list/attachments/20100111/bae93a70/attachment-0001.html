I did a small app a while ago using Gtk# and Mono that had the functionality you&#39;re looking for (I think, I can&#39;t see your mockup behind the proxy at work). Basically, I had everything in an EventBox* as the high level &quot;Canvas&quot;. Then I add a Fixed layout to the EventBox and an Entry to that. So now there&#39;s an Entry that I can move to anywhere inside the EventBox using Fixed.Move() and show and hide at will. There&#39;s a link to the file in question below.<div>
<br></div><div>This solution seemed to work well. I basically made a method on my Canvas that told it to show or hide the line entry at a certain position. So any object in the scene can decide to get text from the user when they want to (as long as there&#39;s only one object doing it at a time). I suppose you could create an Entry for each object in your scene that needs to be able to get text from the user, but that seems unnecessary. </div>
