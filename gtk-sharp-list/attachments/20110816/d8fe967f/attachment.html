        Here is how I did it, for Gtk and KDE programming it is
        considered bad <br>
        form to alter the users theme but the Windows theme is often not
        too <br>
        good looking. It is worth noting that if you compile the app
        with VS or <br>
        use the Windows .NET library for compiling it will take on the
        windows <br>
        theme instead of the default Gtk theme.<br>
        <br>
        <br>
        // this is a Windows only option that specifies the theme<br>
        // the default Gtk themes for Windows is is just not something I
        feel can<br>
        // be relied on at the moment<br>
         if (!string.IsNullOrEmpty(strPathToThemeFile))<br>
         {<br>
            Gtk.Rc.Parse (strPathToThemeFile);<br>
         }<br>
        MainWindow win = new MainWindow ();<br>
         win.Show ();<br>
        Application.Run ();<br>
