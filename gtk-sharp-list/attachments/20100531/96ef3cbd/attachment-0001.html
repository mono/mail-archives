<p class=MsoNormal><font size=2 color=navy face=Arial><span lang=EN-GB
style='font-size:10.0pt;font-family:Arial;color:navy'>The configure event is
fired when the widget&#8217;s window (but not the widget itself) is resized or
moved. That&#8217;s why you don&#8217;t see it fired.<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 color=navy face=Arial><span lang=EN-GB

<p class=MsoNormal><font size=2 color=navy face=Arial><span lang=EN-GB
style='font-size:10.0pt;font-family:Arial;color:navy'>To detect a widget
resizing, you should try the SizeAllocate event, or better: override the
default handler OnSizeAllocated in your custom widget.<o:p></o:p></span></font></p>

<p><font size=2 face="Times New Roman"><span style='font-size:10.0pt'>I'm
making a custom widget which contains an Image widget but allows the<br>
image's size to grow and shrink as the widget is resized. The problem is<br>
that I can't find an appropriate event to do this on.<br>
<br>
I tried to use the Expose event, but as changing the Image widget's pixbuf<br>
forces it to be redrawn, the Expose event runs again, producing a sort of<br>
infinite loop.<br>
<br>
I tried the Configure event, which supposedly is fired whenever the widget<br>
is resized or moved but it doesn't get fired at all (even if I use<br>
[Glib.ConnectBefore]).<br>
<br>
I tried the SizeRequest event, but it seems it only runs one in the<br>
beginning and then never again even if the widget is resized.<br>
<br>
How do you think I should do this? Is there another event which is more<br>
appropriate, or a way to solve the problem of one of these three events?<br>
