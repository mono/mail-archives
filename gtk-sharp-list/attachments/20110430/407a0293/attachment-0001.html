        Hey,
<br><br>&gt; <a href="http://www.mono-project.com/GtkSharp_TreeView_Tutorial" rel="nofollow" link="external" target="_blank">http://www.mono-project.com/GtkSharp_TreeView_Tutorial</a>. First I create the
<br>&gt; model (ListStore) by initializing it with the string type for the data of
<br>&gt; every column. Then I append the columns of the data table to the tree
<br>&gt; view.
<br>&gt; Then I engage the sqlite reader, and for every entry I add its data to the
<br>&gt; model. Then I add a cell for each column pointing to the data. Finally, I
<br>&gt; give the tree view its model.
<br><br>The clean way would be to create your own TreeModel for the kind of data you are trying to read.
<br><br>I have written a model for Sytem.Data.DataViews and generalized to it to take any type of list some time ago, and these are sufficient probably for most of all queries.
<br><br>Using your own TreeModel causes a significant performance boost compared to copying the data in the beginning and is a more clean approach to the problem.
<br><br>Simply use treeModel = new Gtk.TreeModelAdapter(new IListStore(mydataview)));
<br>to make it work.
<br><br>Maybe some day we can get this integrated into Gtk# trunk. I still think that IList support would be a valueable addition, but unfortunately I did not have the time to pursue this further.
<br><br>You might also want to look at gtk-databind at <a href="http://gtk-databind.arsis.net/" rel="nofollow" link="external" target="_blank">http://gtk-databind.arsis.net/</a>, but the project seems to be dead.
<br><br>Unfortunately, I cannot help you with the specific error you are seeing with your code.
