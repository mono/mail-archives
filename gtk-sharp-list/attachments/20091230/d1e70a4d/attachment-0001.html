OK, so the likely problem here is that the underlying native code of<br>
the imagesurface expects to keep a reference to the  byte[] that&#39;s<br>
passed to it. The delay implies that it&#39;s marshalled as a pointer<br>
directly into the managed byte[], and .NET&#39;s compacting GC is moving<br>
the byte[] afterwards. Things that you pass into p/invokes are pinned<br>
for the duration of the call, but not afterwards.<br>
<br>
You should probably use a GCHandle to pin the byte[]:<br>
var gch = new GCHandle (imageData, GCHandleType.Pinned);<br>
and make sure you gch.Free() when you dispose the surface.<br>
