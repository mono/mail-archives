</div></div>System.Timers.Timer does use a thread internally, and the &quot;Elapsed&quot;<br>
callback is invoked on this other thread.<br>
<br>
See <a href="http://msdn.microsoft.com/en-us/library/system.timers.timer.aspx#remarksToggle" target="_blank">http://msdn.microsoft.com/en-us/library/system.timers.timer.aspx#remarksToggle</a><br>
which says:<br>
<br>
&quot; If you use the Timer with a user interface element, such as a form<br>
or control, assign the form or control that contains the Timer to the<br>
SynchronizingObject property, so that the event is marshaled to the<br>
user interface thread. &quot;<br>
<br>
You have been lucky that it works on winforms... it&#39;s still quite<br>
possible that your code will hang, unless you add synchronisation.<br>
<br>
<br>
You can still use timers with GTK#, but you will have to use<br>
Gtk.Application.Invoke to run delegates on the GUI thread:<br>
timer.Elapsed += delegate {<br>
 &nbsp; &nbsp;Application.Invoke (delegate {<br>
 &nbsp; &nbsp; &nbsp; &nbsp;AutoSave()();<br>
 &nbsp; &nbsp;})<br>
});<br>
<br>
GLib.Timeouts are a much nicer (thread-less) model.<br>
