It's not a big deal. Just an idea :-)

Was thinking it would be nice just for helping expose more people to it 
and make it more of a viable library for using when programming because 
developers know it's most likely included. Being in one suite, the 
libraries would be more likely to get all the libraries packaged by the 
various distros and then in-turn get preinstalled on more machines. 
It's a silly argumment to make a big deal out of, I know, as it really 
shouldn't matter. If a distro wants to package any app that uses them, 
they need to bring the libraries with it so thats one to get it in 
there. Though, possibly including it in the Windows and Linux binary 
installers would be something to consider maybe.

It's not a big deal though. I'm also just thinking back to my own 
painful experience, where I have found a few distros that include Gtk# 
packages but then didn't include any other ones like gecko#, dbus#, 
gtksourceview# or other fairly common gtk# related libraries.
