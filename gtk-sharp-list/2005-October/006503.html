On Wed, 2005-10-19 at 20:53 +0200, Pavel Bansky wrote:
> this is good news about new installer. And how about 2.0 2.4 2.x 
> versions of GTK# will be there some back compatibility? Application 
> sensitively depending on some library version is really crapy....

2.0.0.0 was only ever shipped by unstable development versions and no
backward compat will be provided for it.  In 2.6.x, we are shipping
policy.2.4.* assemblies to provide backward compat for applications
compiled against the 2.4.0.0 assemblies from 2.4.x.

On trunk, 2.7.x is already set up to provide policy.2.6 and policy.2.4
as well.
