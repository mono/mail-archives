Unfortunately, I was not able to release a new Gtk# installer that would 
use GTK+ 2.2 and have the corrected Glade 2.01.  However, I have created 
a new revision of the GTK 2.4 based installer and the glade file 
template that  is included for the creation of Glade# projects has now 
been created using the Glade-2 2.6.  At least now, if you create a new 
project and add widgets that are supported by Gtk#/Glade# 1.0.98 it 
should not generate the Null exception you might have been getting. 

I will revisit the creation of a Gtk# installer with GTK+ 2.2.4 but it 
will be an involved process (spend over 7 hours this weekend trying to 
make it work).  However, the GtkDemo app in the samples directory is 
working very nice with the release of the 
gtksharp-1.0.89-win32-0.5c.exe  installer.

I will keep everyone updated.

--Paco
