> On Wed, 2008-10-08 at 09:46 -0500, Mike Kestner wrote:
>> On Wed, 2008-10-08 at 06:40 -0400, Adam Tauno Williams wrote:
>> > I added the Medsphere Gtk# widgets DLL
>> > <http://medsphere.org/projects/widgets/> to my Gtk# project in
>> > Monodevelop.  Unfortunately the widgets provided by Medsphere do not
>> > appear in the Stetic palette.  Does anyone have a hack or patch to make
>> > the Medsphere widgets appear in the palette?
>> > The widgets from the Holly widgets
>> > <http://medsphere.org/projects/widgets/> do appear in Stetic when the
>> > DLL is added to a project.
>> Just add a [System.ComponentModel.ToolboxItem(true)] attr to the class
>> declaration and they'll be added to a "General" group.  You can add
>> [Category("Foo")] to put them in a specific group.
>
> So I put that in front of the class, like...
>
> namespace Medsphere.Widgets
> {
> ...
>        [System.ComponentModel.ToolboxItem(true)]
>        public class GridView : Container
>        {
> ...
>
> And rebuilt.  They still don't appear, and MD still says that the DLL
> contains no components (when I try to add it manually).

The ToolboxItem attribute only works with MD trunk. If using MD 1.0,
you'll need an objects.xml manifest embedded (stetic autogenerates
these).
