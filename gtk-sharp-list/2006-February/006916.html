Hi list,

i discovered a strange behaviour of GLib.Timeout.
On Windows (XP), if you start a new thread and use GLib.Timeout to invoke a
function on the gtk thread, this function won't be called until the gtk
window receives any input from e.g. mouse or keyboard. 
The attached example demonstrates this behaviour (it has been tested on
linux too and was working flawlessly there).
