On Wed, 2003-10-08 at 03:42, Martin Willemoes Hansen wrote:

> Btw, I saw you have place gtkhtml-api.xml in gtk-sharp/gtkhtml/
> instead of api/gtkhtml  .. why is that? 

Because it is an input to the gtkhtml-sharp build.  I will be moving all
the api.xml files into their respective directories and eliminating the
api dir.  

Having one big generation step is not a good long-term plan.  For
example, when I change a .custom file for gtk-sharp.dll, I don't want to
have to regenerate all the source for all the assemblies and rebuild all
of them.
