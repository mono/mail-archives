I already have a little WebKit# experience both on Linux, and on Windows. 
Theoretically, you only have to download and install the WebKit library 
itself, and then webkit-sharp.dll. The latter one is "only" a .NET binding for 
the WebKit framework, so you need to install WebKit first. A.f.a.i.k., 
compiling from source is not necessary on Windows. Don't forget to install 
webkit-sharp into the GAC.

Please note that there have been some issues with WebKit#. Sometimes it was 
almost impossible to use it on Windows, but I strongly hope that a new version 
(mine is still 0.2-9.42) has come out, which does not contain any bugs.
