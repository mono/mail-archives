Hi all,
I'm a little bit confused by different GTK+ versions on Windows that are
bundled with gtk-sharp and Mono.

Namely gtk-sharp-2.12.8-1.win32.msi package (stand-alone download) contains
gtk-sharp 2.12 (which is fine) and GTK+ 2.12 (which is quite old):
- gtk-sharp.dll: 2.12.0.0
- libgtk-win32-2.0-0.dll: 2.12.10.0


In contrast, Mono 2.6.4 on Windows (full package -
mono-2.6.4-gtksharp-2.12.10-win32-3.exe) comes with GTK# 2.12 and more
recent GTK+ 2.18:
- gtk-sharp.dll: 2.12.0.0
- libgtk-win32-2.0-0.dll: 2.18.7.0

I also checked GTK+ binaries from gtk.org, and GTK+ 2.20 is available for
Windows:
- libgtk-win32-2.0-0.dll: 2.20.0.0

Does anyone know why the stand-alone GTK# package on Windows is bundled with
rather old GTK+ version (2.12 - from 2007 or 2008)? (I don't think this is
because compatibility issues, as Mono package is bundled with GTK+ 2.18).
And after all, is there any relation between GTK# and GTK+ version numbers?
