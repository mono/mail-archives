> Isn't the wiki the best place for this?

The wiki is not the optimal repository for the examples.  If we had a
docs maintainer willing to step up and manage the integration of example
code posted to the wiki, this might be an option to make contribution
easier, though.  

The examples will be harder for people to find if they are in the wiki. 
The idea of putting them directly in the docs is to make it easier for
the people learning the API to find example code.  The more you fragment
the documentation, the harder it is to use.
