Hey guys,

Attached is a patch that makes GLib.Value a struct. This gives us tons
of global performance benefits. Basically, we are able to short circit
the GC when using a GLib.Value. Since this is a pretty common construct
in Gtk+ (for example, it is used every time you get or set a value from
a treestore), it gives us a global perf benefit.

One issue I am sure will be brought up is `Why did you write methods
like public void GetProperty (string name, out bool val)'. The issue for
that method is that I would have needed to make GetPropertyBool,
GetPropertyInt32, etc. That would have been nasty to support in the
parser, and is pretty ugly.

Thanks,
Ben
