Rasmus Bonnedal wrote:

> My problem: The following small program gives exception when returning 
> from the callback generated by dSpaceCollide, even if the callback 
> function is empty.
>
> This code works fine in c++ with ODE. Have I misunderstood something 
> with C# / Tao.Ode (quite possible) or is this a real problem?

By default, Tao.Ode is compiled for the single precision version of Ode, 
though in theory it should work with double precision Ode if 
recompiled.  I just checked your code and it runs fine for me, so I 
suspect this is the root of your problem.

Try switching to the single precision version of Ode and see if that 
works for you.  Alternatively, change the dReal alias in Ode.cs from 
"using dReal=System.Single;" to "using dReal=System.Double;" and recompile.
