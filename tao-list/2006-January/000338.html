Hello all,

I am playing around with a little api to build 3d environments along  
with physics sim.

I have a VertexBase class that handles vertex meshes of different  
styles (Points, Lines, Triangles....), mainly using Triangles. To  
draw I use OpenGL vertex arrays. It all works fine when I have only  
one such object around, or if the objects have only few vertices, but  
once I spawn a generated landscape with a few hundred vertices it  
seems to kill some of the smaller objects. It seems like I have  
written over those objects' vertices when I created the lanscape. But  
this is not possible since they are managed arrays (which I pin in  
memory before handing to OpenGL by using GCHandle.Alloc 
(array,GCHandleType.Pinned)) which would throw exceptions. Also when  
I generate the landscape first it also gets completely blown apart,  
so it looks more like the data is moving around. Also I protocol any  
changes to the array dimensions and reset the pointers in the gl  
before drawing in such a case.

Does anyone have a working example of Tao.OpenGl using vertexarrays?
