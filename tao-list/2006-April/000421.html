Hey Jordan,

You're going to have to make sure you're filling all dependencies.

1) .NET Framework 2.0 - You probably have that one down.
2) Make sure you're running a Release version of the executable along 
with the Release version of all depending libraries.
3) Are you including all win32 native dependent libraries?  You could 
test it out by uninstalling Tao and getting the ZIP package instead.  
After that, go through DLL by DLL to see which files you're missing.  
These are usually found in Tao's bin/win32 directory (last time I checked).
4) It might be a good idea to try to catch the exception and display a 
message.  The File.IO exception usually pops up when it can't find a 
P/Invoked DLL.  Encompass your initialization code in a Try Catch and 
you should have a better chance to know what file it can't find.

Good luck, I know this kind of thing can be a pain in the ass, but it's 
definitely worth it!
