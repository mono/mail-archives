On 08/04/06, Vladimir Vukicevic <vladimirv at gmail.com> wrote:
> The simplest way is to use
> System.Runtime.InteropServices.Marshal.PtrToStringAnsi(), as in:
>
> using System.Runtime.InteropServices;
> string s = Marshal.PtrToStringAnsi(Gl.glGetString(x));
>
> The reason why glGetString returns an IntPtr, by the way, is that the
> returned C string is owned by the GL implementation -- calling free on
> it would be illegal.  If it was marshalled directly as part of the
> interop (i.e. if glGetString returned string), then the runtime would
> free it and bad things would happen.
>
>      - Vlad

thanks, that looks a lot cleaner :)

scenes and making the lib more c# friendly?
