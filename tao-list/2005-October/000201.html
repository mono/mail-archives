I finally got around to fixing a bunch of the string marshalling
issues -- strings and arrays of strings were not getting passed to
extension functions correctly, so things like the GLSL methods really
weren't working right (they were getting pointers to unicode strings
instead of ascii/ansi strings).  A lot of the issues people have had
with those should be fixed when I check in to svn tomorrow; I need to
do some testing with win32 first.

Dave, you just used the nsi config stuff in tao/other/win32installer
to create the win32 installer you sent me, right?  If so I'll create
an updated one tomorrow with that and put the binaries up.. sorry for
having you create the installer last week and then not actually using
it =/

One of the things I plan to do after I get this version out finally
(1.2.1?) is to modify the Gl generator to accept overrides for methods
-- there's a lot of nonsensical overloads being autogenerated where a
method really needs just 1 or 2 varieties.  I also really like the
partial class idea, for 2.0 assemblies; I'll probably split things up
by extension.
