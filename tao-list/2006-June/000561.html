Ah, that makes a difference.  Tao (as a whole, and thus Tao.Ode) doesn't
currently support .NET 2.0 - .NET 1.1 is the target platform.  So I haven't
built or tested Tao.Ode against the 2.0 runtime.  It should work in theory,
but it's possible something like this could pop up.  Do you see the same
error using .NET 1.1?  Then it's worth looking into and fixing it.
Otherwise I'd rather finish the update to Ode 0.6, then make a new version
of Tao.Ode for 2.0.   I think it's high time Tao added support for
.NET 2.0- it's been long enough since the last discussion of the topic
when it was
decided to stay with 1.1 a bit longer.  The big question is do we phase out
1.1 support and only target 2.0, or try to support both?  There are enough
little annoyances with 1.1 (such as the need to hack the IL to fix callback
delegates) that just moving to 2.0 makes sense.
