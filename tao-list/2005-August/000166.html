On 8/3/05, Dan Moorehead <dan05a at gmail.com> wrote:
> However wouldnt it be much easier/more intuitive that instead of
> prepending Sdl to everything that you just make the namespaces the
> equivalent of static classes and have SDL.xxx instead of SDL_xxx (as
> opposed to Sdl.SDL_xxx) so that you are just using '.' instead of '_'.
>  That would mirror the design of this C libraries far more closely.

The problem is, a number of these identifiers aren't valid without
their prefix -- GL_2D, GL_3D, GL_2_BYTES, GL_1PASS_EXT etc. all become
invalid once you strip the GL_ prefix.  That was the main motivation
for keeping the GL_/gl and other prefixes there, because it removes
the need for the developer to have to think about what might have
changed in the binding.
