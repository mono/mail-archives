It isn't just Commercial Software.

Mono's 2.0 profile is still 'preview quality' according to the read me's 
from the Linux and Windows installers -- call me crazy, but I think that 
many open source projects will also not be very eager to make the move 
"right away."

My suggestion would be to use #ifdef around 2.0 specific calls, and to 
farther put #ifdef around any call that Mono can't handle, and then to 
use a #Develop 2 solution (#Develop 2 uses VS 2005 project file formats, 
by the way) with a 1.1 profile target, a 2.0 profile target and a Mono 
target.  Once Mono goes gold with their 2.0 profile and says it is ready 
for production use, then reexamine dropping support altogether.

The big issue I would see here is mono and its level of 2.0 support, 
which improves almost daily but has been improving almost daily since 
Microsoft published the C# specs and BCL changes to ECMA.
