I spoke for a bit with Randy last night.. now that .NET 2.0 is
officially released, what do you guys think of not supporting earlier
versions of .NET with future Tao releases?  This has a number of very
significant advantages, most in the OpenGL bindings:

- we can split extensions up into partial classes, as suggested before
on the list
- we can remove the need for the OpenGL postprocessor by using
delegates for extensions
- we can go to one-assembly-for-all-platforms for OpenGL by using
delegates again; we may be able to do the same for SDL and avoid the
calling convention post processor entirely as well.
