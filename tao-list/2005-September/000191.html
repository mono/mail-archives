Hi,

I still can't figure out why my shader programs don't work in my app. Or better why I can't set my uniform variables from code. 
I've used probably the most basic shader thats possible with OpenGl.
All I do is setting the color to the vertices It works well if I set the color in the fragment shader itself like this:

void main(void)
{
   gl_FragColor = vec4( 0.4, 0.0, 0.9, 1.0 );
}

Then the shader works. But as soon as I introduce a uniform variable like this:

uniform vec4 BaseColor;

void main(void)
{
    gl_FragColor = BaseColor;
}

The geometry which I've applied the shader to  becomes black. But when I try to set the value from my c# code like this:

Gl.glUniform4f(Gl.glGetUniformLocationARB(mHandleProg, "BaseColor"), 1.0f, 1.0f, 1.0f, 1.0f);
Console.WriteLine("uniloc: " + Gl.glGetUniformLocationARB(mHandleProg, "BaseColor"));

Nothing the color does not change at all. I mean black is probably the default value for a vec4 in GLSL and that does indeed get applied. Gl.glGetUniformLocationARB returns 0 which is the location of the variable. It returns -1 when something went wrong. 
Could this possibly be a problem with Tao? The shader itself does work in ATI's RenderMonkey 1.6 and I can set the BaseColor from there.
