Creating a wrapper around the C++ library would not only be
restrictive but would also be impossible to do without using either
C++.NET (in which case you loose the cross-platform appeal that is the
reason that OPAL.NET will replace the existing ODE usage) or using an
intermediary C wrapper (complex, inefficient, non-OO).   There are a
lot of advantages to developing with C# and rewriting it with this
language will allow the productivity, portability, and simplicity of
.NET to be leverages by this project.  However, good question, it
helps to clarify the intentions =)

Btw, much of the development discussions for OPAL.NET will likely take
place on the RealmForge google group mailing list
(realmforge at googlegroup.com) or else a new one will be created for it.
