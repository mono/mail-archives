> To loop my openGl drawing I call the viewport Invalidate() function in
> the main form onPaint event function and I recall it at the end of the
> viewport onPaint event function. That don't work. What is your ticks for
> looping the drawing ?

Invalidate simply calls OnPaint method on every child object of a
control (well, if called with true parameter! :P), anyway, you may
think about Invalidate function as a "force repaint" command... so,
the best thing to do, if you are using a main loop, is doing something
like

while(!m_Done) 
{
	Application.DoEvents();

	Gdi.SwapBuffers(m_hDC);
}

If you are using DuobleBuffering, otherwise, if you are using
something like the SimpleOpenGlControl, you could call the draw method
of it (it's simply an Invalidate call, if you look at the
sources...)...

If your application has not a main loop and you are using the
SimpleOpenGlControl (The editor I'm drawing redraws the scene only
when needed), you should simply call an invalidate (or draw) on your
control...

Calling Invalidate in the OnPaint method makes no difference, since
the invalidate function forces a repaint (so the OnPaint is called)...
if I was not clear, plz post again (maybe giving much more infos about
your rendering mode... looping or "on demand" :P), or ask me for a
sample code (I've got both loop example and control based example)
