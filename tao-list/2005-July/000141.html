At the moment, the only access to anything Tao-related is via SVN, so it 
seems reasonable to make library dependencies available via the same 
access mechanism for the convenience of interested developers.  Of 
course a "Tao dependencies" archive on the site and a README with 
instructions in SVN isn't terribly inconvenient, so maybe you're right. 

If the libs were put in SVN, it seems sensible to localize them to a 
separate directory tree parallel to the Tao trunk so downloading them 
can be optional.  If someone just wants the source, the extra binary 
baggage may be unwelcome.
