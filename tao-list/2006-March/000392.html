Am Samstag, den 11.03.2006, 21:58 +0200 schrieb Στέφανος Αποστολόπουλος:
> Hello everyone, this is my first message on this list! :)
> 
> I have been writing an low level OpenGL binding for C#, wrapper and  
> Toolkit in my spare time (which I will probably discontinue since Tao  
> already does a good job), and I would like to ask how the OpenGL bindings  
> are generated for Tao.
> 
> I am asking for a very specific purpose: I would like to create a local  
> source tree with a different naming convention than the one used in Tao.  
> The problem is rather minor, but a significant one nonetheless: The OpenGL  
> bindings in Tao use the Tao.Gl.Gl.glXyz() form, whereas in my wrapper they  
> are called OpenTK.OpenGL.Gl.Xyz() (note that I've dropped the gl  
> preceeding the Xyz() function, since it conveys no meaningful information  
> in the era of namespaces). This difference might not seem as anything to  
> worry about at first glance, but it makes the porting of my projects to  
> Tao impossible!
> 
> I would be very glad if someone could describe how the bindings are  
> generated for Tao. I presume some automated tool is used (the warning on  
> top of 'Gl.cs' for example indicates this - and yes I was crazy enough to  
> have written the bindings by hand for my wrapper!) Is this program  
> available in the svn repository? Or somewhere else?
The Tao bindings are generated from a bunch of python scripts that you
can find the OpenGl directory of tao. The generation is done from the
spec files that come with the official opengl SDL I think.

On the subject, I also thought that GL.xyz() is nicer than GL.glXyz().
That's why I have written such a generator for myself as well :) It's
experimental and I currently don't plan to make an official release of
this stuff but if you're interested:

	http://www.stud.uni-karlsruhe.de/~uxsm/header2csharp.tar.bz2

contains the infrastracture to parse a subset of C that is enough for
most header files. It successfully produces bindings if you feed it with
headers from OpenGL, OpenAL, SDL, FreeType along and maybe others (but
that's the one that I used myself). You can specify in a config file for
specialitites or C# specific code that should get generated. The package
also contains a OpenGl.cs with Xyz() naming scheme in the out directory.
Maybe this stuff is interesting to you.
