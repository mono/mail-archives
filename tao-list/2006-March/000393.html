Hello,

I would have to disagree that Tao.OpenGl.Gl.Xyz() is easier than 
Tao.OpenGl.Gl.glXyz().  Although it would seem easier (two less 
letters), it actually brings up some troubles when porting existing C++ 
OpenGL applications to C#.  With Tao's convention, all you have to do is 
set the using derivative and then put a "Gl." before any OpenGL call and 
there you have your C++ application running with C#.  Since these are 
low level bindings, I personally think they should stay as low level as 
possible.

But of course, these are my own personal thoughts on the matter and 
don't really reflect what actually goes on.

What you could do is make up some object oriented bindings for 
Tao.OpenGl.  A number of people have proposed such a project in the 
past, but nothing has really been made of it.  These bindings would of 
course wrap texture management lists, primitive drawing, etc, all in an 
abstract drawing layer.  One issue thing brings up, however, is would it 
really be used?  Considering the amount of tutorials and articles on 
rendering OpenGL, it's hard to believe someone would be willing to 
switch over to an object oriented interface of OpenGL.

Whatever you decide, I'd really be interested in what you accomplish!

Thanks a lot,
