Open AL is real sucky.  The version that was bound (there's a new one 
out that needs to be reviewed) defined constants where the constants 
differed across platforms...  How sucky is that?  The new version needs 
to be reviewed to see if they've done anything about that, otherwise we 
may be able to work around it at runtime.  It's been a long time since 
I've looked at it, I may have a chance to do so next week(end).

>I notice that, at least in the instance of Tao.OpenAl, there are
>OS-specific #defines. My current development/deployment style is to
>just dump all my DLLs in a single bin directory, and to include the
>windows OpenAL DLL as well. OpenAL comes by default with Tigger, and
>is fairly easy to install on most unixen, so this deployment style is
>easier in that I can just pull my repository onto whatever system I
>want to build installers for, move the files around or run a script

>the filesize.
>

>that could either let me ship a single Tao.OpenAl.dll that works on
>both Unix/windows, or some mechanism for dynamically loading something
>like Tao.OpenAl.win32.dll under windows?
