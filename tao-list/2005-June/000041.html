if you download the samples that ship with the Tao package, you will get lots of examples to inspect.
These should show you the basics about how Opengl is initialized.
If this doesn't help you then find some vb code that does OpenGL without TAO (win32 style or glut) and port this code
to TAO which is straightforward since all relevant functions exist in the package (takes a little more time ..).

If you don't want to go this way there is some SimpleGLControl or something alike. I have never tried this but it should be easier to set things up. If you don't mind to look to come c# code there are nehe.gamedev.net lessons ported to c# and tao.

On my site I have some working code too www.bitterman-page.de.vu.
