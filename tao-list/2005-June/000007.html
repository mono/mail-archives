I have uploaded DotNetPrebuild files (dnpb) for all of the Tao projects.
You can now create VS2003 project files. It's still a work-in-progress
especially since dnpb does not like build actions. Therefore,
theTao.PostProcess and Tao.GlPostProcess are not used correctly. Also,
there is a weird bug in dnpb when the solution and several projects have
similar names (i.e. Tao.openGl, Tao.openGl.Glu,
Tao.OpenGl.ExtensionLoader). As a result, the best solution file to use for
OpenGl is the Tao.FreeGlut. It contains all of the OpenGl projects.

I started to clean up the doc warnings by putting ///summary in front of
the offending methods. There are a lot of bad <cref> though. What should I
do with those?

I have started working on converting the old NAnt files to the new
directory structure. I'll check in those when I am done.
