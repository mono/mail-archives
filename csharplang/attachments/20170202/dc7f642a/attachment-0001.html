<h2 style="mso-margin-top-alt:0in;margin-right:0in;margin-bottom:12.0pt;margin-left:0in;border:none;padding:0in;box-sizing:border-box">
<span style="font-family:-apple-system;color:#333333">Problem:<o:p></o:p></span></h2>
</div>
<p style="mso-margin-top-alt:0in;margin-right:0in;margin-bottom:12.0pt;margin-left:0in;box-sizing:border-box">
<span style="font-size:10.5pt;font-family:-apple-system;color:#333333">Verbatim string literals are often used to copy in chunks of text such as SQL or XML. It's not uncommon for this text to contain embedded double-quotes, which then must be doubled-up in
 C# in order to be escaped. This makes it obnoxious if you have to copy and paste the text back and forth since it would have to be fixed each time.<o:p></o:p></span></p>

<div style="mso-element:para-border-div;border:none;border-bottom:solid #EEEEEE 1.0pt;padding:0in 0in 4.0pt 0in">
<h2 style="mso-margin-top-alt:.25in;margin-right:0in;margin-bottom:12.0pt;margin-left:0in;border:none;padding:0in;box-sizing:border-box">
<span style="font-family:-apple-system;color:#333333">Solution:<o:p></o:p></span></h2>
</div>
<p style="mso-margin-top-alt:0in;margin-right:0in;margin-bottom:12.0pt;margin-left:0in;box-sizing:border-box">
<span style="font-size:10.5pt;font-family:-apple-system;color:#333333">By supporting a custom notation for starting and ending a string literal it enables evaluating the double quotes as part of the string that do not have to be escaped. The programmer would
 use a pattern at the beginning of the string which then must be matched to denote the end of the string.<o:p></o:p></span></p>
<p style="mso-margin-top-alt:0in;margin-right:0in;margin-bottom:12.0pt;margin-left:0in;box-sizing:border-box">
<span style="font-size:10.5pt;font-family:-apple-system;color:#333333">Proposed syntax would be based on verbatim string syntax:<o:p></o:p></span></p>
<p style="mso-margin-top-alt:0in;margin-right:0in;margin-bottom:12.0pt;margin-left:0in;box-sizing:border-box">
<em><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">raw-string-literal</span></em><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">:<br>
@(" <em><span style="font-family:-apple-system">raw-string-literal-characters</span></em> </span><span style="font-size:8.0pt;font-family:-apple-system;color:#333333">opt</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"> " )<br>
@( <em><span style="font-family:-apple-system">identifier</span></em> " <em><span style="font-family:-apple-system">raw-string-literal-characters</span></em> </span><span style="font-size:8.0pt;font-family:-apple-system;color:#333333">opt</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"> " <em><span style="font-family:-apple-system">identifier</span></em> )<o:p></o:p></span></p>
<p style="mso-margin-top-alt:0in;margin-right:0in;margin-bottom:12.0pt;margin-left:0in;box-sizing:border-box">
<em><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">raw-string-literal-characters</span></em><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">:<br>
Any characters except " <em><span style="font-family:-apple-system">identifer</span></em> </span><span style="font-size:8.0pt;font-family:-apple-system;color:#333333">opt</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"> )<o:p></o:p></span></p>
<p style="margin:0in;margin-bottom:.0001pt;box-sizing:border-box"><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">The </span><code><span style="font-size:9.0pt;font-family:Consolas;color:#333333">@</span></code><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"> would
 be immediately followed by an open-parenthesis (</span><code><span style="font-size:9.0pt;font-family:Consolas;color:#333333">(</span></code><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">). Following that is an optional delimiter which
 is any legal C# identifier and then a double-quote. Next would be the raw string contents which would be interpreted exactly as they appear in the file. The terminator of the string literal would be a double quote immediately followed by the optional delimiter
 and a close-parenthesis (</span><code><span style="font-size:9.0pt;font-family:Consolas;color:#333333">)</span></code><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">).<o:p></o:p></span></p>
<p style="margin:0in;margin-bottom:.0001pt;box-sizing:border-box"><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">The following example is a raw string without a custom delimiter. It is terminated simply by the sequence </span><code><span style="font-size:9.0pt;font-family:Consolas;color:#333333">")</span></code><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">.
 Double quotes that appear within the string are interpreted as a part of the string as long as they are not followed by a close-parenthesis character (</span><code><span style="font-size:9.0pt;font-family:Consolas;color:#333333">)</span></code><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">).<o:p></o:p></span></p>

<p style="margin:0in;margin-bottom:.0001pt;box-sizing:border-box"><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">The following example is a raw string with a custom delimiter. As the custom delimiter is specified as "foo" that sets the
 terminating sequence to be </span><code><span style="font-size:9.0pt;font-family:Consolas;color:#333333">"foo)</span></code><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">. Any other characters are interpreted as a part of the string
 literal, including any portion of the terminating sequence. Note that the term "foo" appears within the string after a double-quote, but since it is not also followed by a close-parenthesis (</span><code><span style="font-size:9.0pt;font-family:Consolas;color:#333333">)</span></code><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">)
 it is not considered the terminating sequence of the literal.<o:p></o:p></span></p>
<p style="mso-margin-top-alt:0in;margin-right:0in;margin-bottom:12.0pt;margin-left:0in;box-sizing:border-box">
<span style="font-size:10.5pt;font-family:-apple-system;color:#333333">In this example the custom delimiter is "foo". Note that the attribute value is not considered as the end of the string. However, if it was followed by a parenthesis it would be and a different
 custom delimiter might have to be chosen.<o:p></o:p></span></p>
