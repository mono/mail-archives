I would like to clarify on your understanding:</p>
<ul type="disc" style="margin-top: 0in; margin-bottom: 0in;">
<li style="margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">
The operators for IntPtr ship as part of the library</li></ul>
<p style="margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">
The runtime treats IntPtr and UIntPtr as primitive types (`native int` and `unsigned native int`, respectively). As such, there are no implementations for these operators in the library (in the same way that Int32 and Double don’t actually define any operators).
 I would guess this is because it was expected that the operators on these primitive types would be supported by the various languages (as is common practice).</p>
<p style="margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">
 </p>
<ul type="disc" style="margin-top: 0in; margin-bottom: 0in;">
<li style="margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">
They are implemented directly in IL with a one-instruction body</li></ul>
<p style="margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">
They can be implemented in IL with a single instruction, the same as all other primitive operators. Adding two IntPtrs involves calling `ldarg` with each of their values and then calling `add`. There is currently no way to get the code implemented in the single
 instruction outside of creating a custom library written in pure IL to expose the functionality.</p>
<p style="margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">
 </p>
<ul type="disc" style="margin-top: 0in; margin-bottom: 0in;">
<li style="margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">
Per today’s functionality, the compiler translates application of the operator to a static method call</li></ul>
<p style="margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">
Would a library be implemented (or would these be exposed in CoreFX) as described above (written in pure IL), then the compiler would emit the call the same as any other method call (presumably)</p>
<p style="margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">
 </p>
<ul type="disc" style="margin-top: 0in; margin-bottom: 0in;">
<li style="margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">
Per today’s functionality (presumably), the JIT inlines the operator</li></ul>
<p style="margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">
Would the library be implemented, the JIT would presumably inline the operator (since it is small enough). The JIT could even be given special handling so-as to handle the calls as an instrinsic, so that the inlining/optimization is even better.</p>
<p style="margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">
 </p>
<p style="margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">

<p style="margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">
I actually have a very similar proposal against CoreFX (<a href="https://na01.safelinks.protection.outlook.com/?url=https%3A%2F%2Fgithub.com%2Fdotnet%2Fcorefx%2Fissues%2F10457&data=02%7C01%7Cdustin.campbell%40microsoft.com%7Cf91ba74b03d44f83170608d44c641711%7C72f988bf86f141af91ab2d7cd011db47%7C1%7C0%7C636217440175810582&sdata=vAP1YKlbpph5Ewq0KtOai8tnabsmA1SmtRZNlVOJJDQ%3D&reserved=0" style="color: blue; text-decoration: underline;">https://github.com/dotnet/corefx/issues/10457</a>)

<p style="margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">
It’s not that the runtime has a special opcode, it is that the runtime declares a set of primitive types, which C# follows for everything except `System.IntPtr` and `System.UIntPtr`.</p>
<p style="margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">
 </p>
<p style="margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">
Each of the primitive types defines a set of operators that are applicable to it. These all have intrinsic support in the compiler to directly emit the appropriate IL instructions, except for `System.IntPtr` and `System.UIntPtr`.</p>
<p style="margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">
 </p>
<p style="margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">
These types, regardless of how often they are mentioned in the C# spec, are used quite frequently: in Roslyn, in CoreFX, in CoreCLR, in 3<sup>rd</sup> party libraries, in Interop Code, etc…</p>
<p style="margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">
 </p>
<p style="margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">
Anytime the user needs to perform one of the primitive operations around one of these types (operations that are defined by the runtime) they currently need to either:</p>
<ul type="disc" style="margin-top: 0in; margin-bottom: 0in;">
<li style="margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">
Write unsafe code (generally involves calling IntPtr.ToPointer(), performing straight pointer arithmetic and then casting back) -or-</li><li style="margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">
To avoid writing unsafe code, they need to cast to an integer the same size as IntPtr (so Int64 on x64 and Int32 on x86), perform the arithmetic and then cast back</li></ul>
<p style="margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">
 </p>
<p style="margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">
This is very suboptimal and involves emitting a method call, which may or may not be optimized.</p>
<p style="margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">

The CLR supports a set of operators for the </span><span style="font-size:9.0pt; font-family:Consolas; color:#333333">System.IntPtr</span><span style="font-size:10.5pt; color:#333333"> and </span><span style="font-size:9.0pt; font-family:Consolas; color:#333333">System.UIntPtr</span><span style="font-size:10.5pt; color:#333333"> types
 (</span><span style="font-size:9.0pt; font-family:Consolas; color:#333333">native int</span><span style="font-size:10.5pt; color:#333333">). These operators can be seen in </span><span style="font-size:9.0pt; font-family:Consolas; color:#333333">III.1.5</span><span style="font-size:10.5pt; color:#333333"> of
 the Command Language Infrastructure specification. However, these operators are not supported by the C#.</span></p>
<p style="background: white; margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">
<b><span style="font-size:10.5pt; color:#333333">Workaround:</span></b><span style="font-size:10.5pt; color:#333333"><br>
Either directly emit IL or write non-optimal code that does not directly translate to the appropriate IL instructions.</span></p>
<p style="background: white; margin: 0in 0in 0.0001pt; font-size: 11pt; font-family: Calibri, sans-serif;">
<b><span style="font-size:10.5pt; color:#333333">Request:</span></b><span style="font-size:10.5pt; color:#333333"><br>
Language support should be provided for the full set of operators supported by </span><span style="font-size:9.0pt; font-family:Consolas; color:#333333">System.IntPtr</span><span style="font-size:10.5pt; color:#333333"> and </span><span style="font-size:9.0pt; font-family:Consolas; color:#333333">System.UIntPtr</span><span style="font-size:10.5pt; color:#333333">.
 These operators are: </span><span style="font-size:9.0pt; font-family:Consolas; color:#333333">Add</span><span style="font-size:10.5pt; color:#333333">, </span><span style="font-size:9.0pt; font-family:Consolas; color:#333333">Divide</span><span style="font-size:10.5pt; color:#333333">, </span><span style="font-size:9.0pt; font-family:Consolas; color:#333333">Multiply</span><span style="font-size:10.5pt; color:#333333">, </span><span style="font-size:9.0pt; font-family:Consolas; color:#333333">Remainder</span><span style="font-size:10.5pt; color:#333333">, </span><span style="font-size:9.0pt; font-family:Consolas; color:#333333">Subtract</span><span style="font-size:10.5pt; color:#333333">, </span><span style="font-size:9.0pt; font-family:Consolas; color:#333333">Negate</span><span style="font-size:10.5pt; color:#333333">, </span><span style="font-size:9.0pt; font-family:Consolas; color:#333333">Equals</span><span style="font-size:10.5pt; color:#333333">, </span><span style="font-size:9.0pt; font-family:Consolas; color:#333333">Compare</span><span style="font-size:10.5pt; color:#333333">, </span><span style="font-size:9.0pt; font-family:Consolas; color:#333333">And</span><span style="font-size:10.5pt; color:#333333">, </span><span style="font-size:9.0pt; font-family:Consolas; color:#333333">Not</span><span style="font-size:10.5pt; color:#333333">, </span><span style="font-size:9.0pt; font-family:Consolas; color:#333333">Or</span><span style="font-size:10.5pt; color:#333333">, </span><span style="font-size:9.0pt; font-family:Consolas; color:#333333">XOr</span><span style="font-size:10.5pt; color:#333333">, </span><span style="font-size:9.0pt; font-family:Consolas; color:#333333">ShiftLeft</span><span style="font-size:10.5pt; color:#333333">, </span><span style="font-size:9.0pt; font-family:Consolas; color:#333333">ShiftRight</span><span style="font-size:10.5pt; color:#333333">.</span></p>
