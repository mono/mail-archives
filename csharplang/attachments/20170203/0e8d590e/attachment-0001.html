<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">It’s the same as if someone came to me and said “I have this great idea called ‘linq’.  It requires lambdas, and you’ll only be able to use those lambdas on methods on collections”.   I would see the value in the proposal, but I would want
 to come up with a more *<b>general</b>* solution so that broader groups of people could benefit.<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">That’s the point I’m making here. <o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>

<p class="MsoNormal"> <o:p></o:p></p>
<p class="MsoNormal">It’s not that the runtime has a special opcode, it is that the runtime declares a set of primitive types, which C# follows for everything except `System.IntPtr` and `System.UIntPtr`.<o:p></o:p></p>
<p class="MsoNormal"> <o:p></o:p></p>
<p class="MsoNormal">Each of the primitive types defines a set of operators that are applicable to it. These all have intrinsic support in the compiler to directly emit the appropriate IL instructions, except for `System.IntPtr` and `System.UIntPtr`.<o:p></o:p></p>
<p class="MsoNormal"> <o:p></o:p></p>
<p class="MsoNormal">These types, regardless of how often they are mentioned in the C# spec, are used quite frequently: in Roslyn, in CoreFX, in CoreCLR, in 3<sup>rd</sup> party libraries, in Interop Code, etc…<o:p></o:p></p>
<p class="MsoNormal"> <o:p></o:p></p>
<p class="MsoNormal">Anytime the user needs to perform one of the primitive operations around one of these types (operations that are defined by the runtime) they currently need to either:<o:p></o:p></p>
<ul style="margin-top:0in" type="disc">
<li class="MsoNormal" style="margin-left:0in;mso-list:l0 level1 lfo6">Write unsafe code (generally involves calling IntPtr.ToPointer(), performing straight pointer arithmetic and then casting back) -or-<o:p></o:p></li><li class="MsoNormal" style="margin-left:0in;mso-list:l0 level1 lfo6">To avoid writing unsafe code, they need to cast to an integer the same size as IntPtr (so Int64 on x64 and Int32 on x86), perform the arithmetic and then cast back<o:p></o:p></li></ul>
<p class="MsoNormal"> <o:p></o:p></p>
<p class="MsoNormal">This is very suboptimal and involves emitting a method call, which may or may not be optimized.<o:p></o:p></p>
<p class="MsoNormal"> <o:p></o:p></p>
<p class="MsoNormal">However, if the compiler has built in support for the operators then it gets emitted as a single IL instruction (rather than a call) for each operation (same as with every other primitive type) and it requires no downstream work by the
 runtime or framework for the benefit.<o:p></o:p></p>

<p class="MsoNormal" style="background:white"><b><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">Issue:</span></b><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"><br>
The CLR supports a set of operators for the </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">System.IntPtr</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"> and </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">System.UIntPtr</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"> types
 (</span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">native int</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">). These operators can be seen in </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">III.1.5</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"> of
 the Command Language Infrastructure specification. However, these operators are not supported by the C#.</span><o:p></o:p></p>
<p class="MsoNormal" style="margin-bottom:12.0pt;background:white"><b><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">Workaround:</span></b><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"><br>
Either directly emit IL or write non-optimal code that does not directly translate to the appropriate IL instructions.</span><o:p></o:p></p>
<p class="MsoNormal" style="background:white"><b><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">Request:</span></b><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"><br>
Language support should be provided for the full set of operators supported by </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">System.IntPtr</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"> and </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">System.UIntPtr</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">.
 These operators are: </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">Add</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">, </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">Divide</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">, </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">Multiply</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">, </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">Remainder</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">, </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">Subtract</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">, </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">Negate</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">, </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">Equals</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">, </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">Compare</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">, </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">And</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">, </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">Not</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">, </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">Or</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">, </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">XOr</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">, </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">ShiftLeft</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">, </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">ShiftRight</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">.</span><o:p></o:p></p>
<p class="MsoNormal" style="background:white"><b><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">Other Thoughts:</span></b><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"><br>
C# seems to treat both </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">IntPtr</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"> and </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">UIntPtr</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"> as
 a type only used for interop and for representing pointers in a 'safe' manner, when it is really just:</span><o:p></o:p></p>
<p class="MsoNormal" style="background:white"><span style="font-size:10.5pt;font-family:-apple-system;color:#777777">an integer whose size is platform-specific.</span><o:p></o:p></p>
<p class="MsoNormal" style="background:white"><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">It should be possible for a user to treat the </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">IntPtr</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"> and </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">UIntPtr</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"> types
 as an equivalent to the native </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">size_t</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"> type. However, in order to do that, one must may have to deal with one
 or more of the following:</span><o:p></o:p></p>
<ul style="margin-top:0in" type="disc">
<li class="MsoNormal" style="color:#333333;margin-left:0in;mso-list:l2 level1 lfo9;background:white">
<span style="font-size:10.5pt;font-family:-apple-system">Perform multiple levels of casting</span><o:p></o:p></li><li class="MsoNormal" style="color:#333333;margin-top:3.0pt;margin-left:0in;mso-list:l2 level1 lfo9;background:white">
<span style="font-size:10.5pt;font-family:-apple-system">Use integers whose size is larger than their platform's native size</span><o:p></o:p></li><li class="MsoNormal" style="color:#333333;margin-top:3.0pt;margin-left:0in;mso-list:l2 level1 lfo9;background:white">
<span style="font-size:10.5pt;font-family:-apple-system">Have branching to use integers that are platform-specific</span><o:p></o:p></li><li class="MsoNormal" style="color:#333333;margin-top:3.0pt;margin-left:0in;mso-list:l2 level1 lfo9;background:white">
<span style="font-size:10.5pt;font-family:-apple-system">Work with unsafe code and do pointer arithmetic</span><o:p></o:p></li></ul>
