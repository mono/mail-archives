<div dir="ltr">Hello, <div><br></div><div><br></div><div><b><font size="4">What?</font></b></div><div><b><br></b></div><div>Syntax sugar which allows classes to declare interfaces with all their public methods.</div><div><br></div><div>Simplest example:</div><div><br></div><blockquote style="margin:0px 0px 0px 40px;border:none;padding:0px"><div><font face="monospace, monospace">class Configuration: <font color="#0000ff">declares IConfiguration</font></font></div><div><font face="monospace, monospace">{</font></div><div><font face="monospace, monospace">    public int MaxInstances { get; }</font></div><div><font face="monospace, monospace">}</font></div><div><br></div></blockquote><div>will declare an interface <font face="monospace, monospace">IConfiguration</font>, which otherwise would look like this:</div><div><br></div><blockquote style="margin:0px 0px 0px 40px;border:none;padding:0px"><div><div><font face="monospace, monospace" color="#666666">interface IConfiguration</font></div></div><div><div><font face="monospace, monospace" color="#666666">{</font></div></div><div><div><font face="monospace, monospace" color="#666666">    int MaxInstances { get; }</font></div></div><div><div><font face="monospace, monospace" color="#666666">}</font></div></div></blockquote><div><br></div><div>It also works really straight forward with the more complex cases:</div><div><br></div><blockquote style="margin:0px 0px 0px 40px;border:none;padding:0px"><div><font face="monospace, monospace" style="color:rgb(0,0,0)">class My : IDisposable </font></div><div><span style="color:rgb(0,0,255);font-family:monospace,monospace">    declares IMy</span></div><div><font face="monospace, monospace" color="#000000">{</font></div></blockquote><blockquote style="margin:0px 0px 0px 40px;border:none;padding:0px"><div><font face="monospace, monospace" color="#000000">    public event Action Something;</font></div></blockquote><blockquote style="margin:0px 0px 0px 40px;border:none;padding:0px"><div><font face="monospace, monospace" color="#000000"><br></font></div><div><font face="monospace, monospace" color="#000000">    public void Foo() { }</font></div><div><font face="monospace, monospace" color="#000000"><br></font></div><div><span style="color:rgb(0,0,0);font-family:monospace,monospace">    public void Dispose() { }</span><font face="monospace, monospace" color="#000000"><br></font></div><div><font face="monospace, monospace" color="#000000"><br></font></div><div><font face="monospace, monospace" color="#000000">    protected void Bar() { }</font><span style="color:rgb(0,0,0);font-family:monospace,monospace">    </span></div><div><font face="monospace, monospace" color="#000000">}</font></div></blockquote><div><br></div><div>will declare interface like this:</div><div><br></div><blockquote style="margin:0px 0px 0px 40px;border:none;padding:0px"><div><font face="monospace, monospace" color="#666666">interface IMy : IDisposable</font></div><div><font face="monospace, monospace" color="#666666">{</font></div><div><font color="#666666"><font face="monospace, monospace">    </font><span style="font-family:monospace,monospace">event Action Something;</span></font></div><div><span style="font-family:monospace,monospace"><font color="#666666"><br></font></span></div><blockquote style="margin:0px 0px 0px 40px;border:none;padding:0px"></blockquote><div><font face="monospace, monospace" color="#666666">    void Foo();</font></div><div><font face="monospace, monospace" color="#666666">}</font></div></blockquote><div><br></div><div>As you can see it includes all public members and inherits all interfaces too, so My and IMy sharing the whole</div><div><br></div><div><br></div><div><b><font size="4">Why it is needed?</font></b></div><div><b><br></b></div><div>It will simplify all cases when you have one class implementing one interface in the same assembly.</div><div>You don't have to:</div><div><ul><li>keep a separate interface </li><li>maintaining an extra file </li><
