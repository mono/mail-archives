<p class="MsoNormal">> Arguably, this request is just having C# catch up<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">Features that are just about catching up (especially in corner cases like this) just do not seem to provide enough weight for me.  There’s limited time we can spend on language changes, and I’d much rather them go to non-narrow features
 that many more customers could benefit from.<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">> IntPtr is not just any type, though. It's a type that happens to be a valuetype mapped to a primitive IL type<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">That *<b>definitely</b>* sounds like a concern of the runtime then
<span style="font-family:Wingdings">J</span>.  From the language perspective it looks just like any other type.  I mean the language specification doesn’t even talk about IntPtr (except in reference to volatile fields).  Introducing a whole section to explain
 the operators for this sort of type seems extremely niche and not worth the effort.<o:p></o:p></p>
<p class="MsoNormal"><br>
If, however, we had a general proposal (i.e. how to provide operators when types have not provided them), then I could definitely get behind that.  Then we’d have something that would warrant the effort to make the language change and could be something that
 could be added to IntPtr (without having to ship a new Fx) through things like a nuget package.
<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">--<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">TLDR: one-off, catch-up features are super low down on my list of things that I think should be done.  I’d much rather have broadly applicable features, or super nice improvements in common areas first.<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>

<p class="MsoNormal"><o:p> </o:p></p>
<div>
<p class="MsoNormal">IntPtr is not just any type, though. It's a type that happens to be a valuetype mapped to a primitive IL type, and the wrapper in question doesn't provide overloaded operators because IL has intrinsic operators capable of handling it -
 so the assumption of people who wrote System.IntPtr was likely that any language that would want to expose those operators in the first place, would expose them directly. This is in contrast to something like System.Decimal, where the overloaded operators
 are the one and only way to do anything to it.<o:p></o:p></p>
<div>
<p class="MsoNormal"><o:p> </o:p></p>
</div>
<div>
<p class="MsoNormal">You could argue that IntPtr should have had the overloads (but then why not Int32 and Double?). But it doesn't, and C# should have accounted for that. Arguably, this request is just having C# catch up. It doesn't set any meaningful precedent
 for any other type, unless such a type is also 1) part of the standard library, 2) has operators defined for it in IL, and 3) doesn't overload those operators on the corresponding valuetype. Correct me if I'm wrong, but IntPtr/UIntPtr are literally the only
 two remaining holdouts in this category.<o:p></o:p></p>
</div>

<p class="MsoNormal" style="mso-margin-top-alt:auto;mso-margin-bottom-alt:auto;background:white">
<b><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">Issue:</span></b><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"><br>
The CLR supports a set of operators for the </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">System.IntPtr</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"> and </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">System.UIntPtr</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"> types
 (</span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">native int</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">). These operators can be seen in </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">III.1.5</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"> of
 the Command Language Infrastructure specification. However, these operators are not supported by the C#.</span><o:p></o:p></p>
<p class="MsoNormal" style="mso-margin-top-alt:auto;margin-bottom:12.0pt;background:white">
<b><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">Workaround:</span></b><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"><br>
Either directly emit IL or write non-optimal code that does not directly translate to the appropriate IL instructions.</span><o:p></o:p></p>
<p class="MsoNormal" style="mso-margin-top-alt:auto;mso-margin-bottom-alt:auto;background:white">
<b><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">Request:</span></b><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"><br>
Language support should be provided for the full set of operators supported by </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">System.IntPtr</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"> and </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">System.UIntPtr</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">.
 These operators are: </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">Add</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">, </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">Divide</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">, </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">Multiply</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">, </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">Remainder</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">, </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">Subtract</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">, </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">Negate</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">, </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">Equals</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">, </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">Compare</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">, </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">And</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">, </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">Not</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">, </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">Or</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">, </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">XOr</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">, </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">ShiftLeft</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">, </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">ShiftRight</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">.</span><o:p></o:p></p>
<p class="MsoNormal" style="mso-margin-top-alt:auto;mso-margin-bottom-alt:auto;background:white">
<b><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">Other Thoughts:</span></b><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"><br>
C# seems to treat both </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">IntPtr</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"> and </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">UIntPtr</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"> as
 a type only used for interop and for representing pointers in a 'safe' manner, when it is really just:</span><o:p></o:p></p>
<p class="MsoNormal" style="mso-margin-top-alt:auto;mso-margin-bottom-alt:auto;background:white">
<span style="font-size:10.5pt;font-family:-apple-system;color:#777777">an integer whose size is platform-specific.</span><o:p></o:p></p>
<p class="MsoNormal" style="mso-margin-top-alt:auto;mso-margin-bottom-alt:auto;background:white">
<span style="font-size:10.5pt;font-family:-apple-system;color:#333333">It should be possible for a user to treat the </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">IntPtr</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"> and </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">UIntPtr</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"> types
 as an equivalent to the native </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">size_t</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"> type. However, in order to do that, one must may have to deal with one
 or more of the following:</span><o:p></o:p></p>
<ul type="disc">
<li class="MsoNormal" style="color:#333333;mso-margin-top-alt:auto;mso-margin-bottom-alt:auto;margin-left:0in;mso-list:l0 level1 lfo1;background:white">
<span style="font-size:10.5pt;font-family:-apple-system">Perform multiple levels of casting</span><o:p></o:p></li><li class="MsoNormal" style="color:#333333;margin-top:3.0pt;mso-margin-bottom-alt:auto;margin-left:0in;mso-list:l0 level1 lfo1;background:white">
<span style="font-size:10.5pt;font-family:-apple-system">Use integers whose size is larger than their platform's native size</span><o:p></o:p></li><li class="MsoNormal" style="color:#333333;margin-top:3.0pt;mso-margin-bottom-alt:auto;margin-left:0in;mso-list:l0 level1 lfo1;background:white">
<span style="font-size:10.5pt;font-family:-apple-system">Have branching to use integers that are platform-specific</span><o:p></o:p></li><li class="MsoNormal" style="color:#333333;margin-top:3.0pt;mso-margin-bottom-alt:auto;margin-left:0in;mso-list:l0 level1 lfo1;background:white">
<span style="font-size:10.5pt;font-family:-apple-system">Work with unsafe code and do pointer arithmetic</span><o:p></o:p></li></ul>
