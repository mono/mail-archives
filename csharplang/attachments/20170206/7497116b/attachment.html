The C# language should be extended to support function pointers in an unsafe context. Such as:<o:p></o:p></span></p>
<p class="MsoNormal" style="background:#F7F7F7"><span style="font-size:9.0pt;font-family:Consolas;color:#333333">unsafe

 missing.<o:p></o:p></span></p>
<p class="MsoNormal" style="background:white"><b><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">Advantages</span></b><span style="font-size:10.5pt;font-family:-apple-system;color:#333333"><br>
The compiler could take advantage of the fact this is an unsafe function pointer and emit the </span><span style="font-size:9.0pt;font-family:Consolas;color:#333333">calli</span><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">instruction
 when invoking.<o:p></o:p></span></p>
<p class="MsoNormal" style="margin-bottom:12.0pt;background:white"><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">The compiler could provide some level of type safety when invoking the function pointer (ensuring the arguments passed
 in match the types required).<o:p></o:p></span></p>
<p class="MsoNormal" style="margin-bottom:12.0pt;background:white"><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">This would allow better interop with unmanaged languages and types that use function pointers.<o:p></o:p></span></p>
<p class="MsoNormal" style="mso-margin-bottom-alt:auto;background:white"><span style="font-size:10.5pt;font-family:-apple-system;color:#333333">This would allow a user to more readily implement VTBLs for unmanaged interop when required.<o:p></o:p></span></p>
