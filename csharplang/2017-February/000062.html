compiler generates code behind the scenes, namely iterators and async
functions?

is generated and to the async State machine.

I don't have any specific use cases for this.

> I'm proposing allowing field-targeted attributes on auto-properties. The
> compiler would automatically attach that attribute to the generated backing
> field.
>
> [field: NonSerialized]
>
> public string Name { get; set; }
>
> equivalent to:
>
> [NonSerialized]
>
> private string name;
>
>
>
> public string Name {
>
>     get { return this.name; }
>
>     set { this.name = value; }
>
> }
>
> This would bring auto-properties into parity with events which have
> supported this syntax since C# 1.0:
>
> [field: NonSerialized]
>
> public event EventHandler Foo;
