I had posted a suggestion like this on UserVoice some time ago. I figure it'll fly probably about as well as a lead balloon but it might be interesting just for the sake of discussion. Tuples would likely mostly suffice in the scenarios in which this would be useful.

Currently in C# all of the members of an anonymous type are readonly. In some situations where an anonymous type is used as a working space it would be convenient to be able to mutate specific members. I propose that within the anonymous type declaration that the keyword ref could be used to declare a member as mutable:

var id = "123";

var anon = new { id, ref count = 0 };

anon.count = 2;

anon.count += 1;



anon.id<https://na01.safelinks.protection.outlook.com/?url=http%3A%2F%2Fanon.id&data=02%7C01%7Cmads.torgersen%40microsoft.com%7C7e12ec18f294439513db08d44bcfbf31%7C72f988bf86f141af91ab2d7cd011db47%7C1%7C0%7C636216803039283790&sdata=uK093sSiW9HsZ5hImnp9KplesQjaKMbHnRsW0RSsyFo%3D&reserved=0> = "456"; // compiler error

As the members marked as ref are mutable the generated anonymous type would not include them in the compiler-generated GetHashCode() or Equals() overrides.

Notes:

  1.  The ref keyword was what I had proposed on UserVoice. This was before Roslyn or any notion of ref locals/returns, so I don't know if the use of the same keyword in this context would be confusing or ambiguous.
  2.  This functionality already exists in VB.NET<https://na01.safelinks.protection.outlook.com/?url=http%3A%2F%2FVB.NET&data=02%7C01%7Cmads.torgersen%40microsoft.com%7C7e12ec18f294439513db08d44bcfbf31%7C72f988bf86f141af91ab2d7cd011db47%7C1%7C0%7C636216803039283790&sdata=WIZOWWgAQxBBugpNKQF%2FSjiX%2FfjUPT5qZb9cIuTdveM%3D&reserved=0> although reversed, where immutable members are explicitly marked with the contextual keyword Key:

Dim id = "123"

Dim anon = New With {Key .id = id, .count = 0}

anon.count = 2

anon.count += 1



anon.id = "456" ' compiler error
