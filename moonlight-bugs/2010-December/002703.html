Update on the bug:

I was able to create a working demo. Though the crash is not immediately
reproducible, it is quite easy to induce it to go down.

The bug doesn't show when using the managed Xaml parser.

The steps to reproduce are:
1. Run the TestPage.html in the newly attached working demo.
2. Press the button (it says "Show wait")
3. Three things can happen:

 - You get a tinted screen and a popup window saying "This is a waiting
window..." => In this case, just press the "Close" button and try again, by
pressing the "Show wait button". Eventually you will reproduce the bug.

 - You get a crash => Success in reproducing the bug.

 - You get a tinted screen without the popup window. This is the other way the
bug shows itself. You to have a reproduction of the bug => If you really want
to get the crash, you have to refresh the browser and press the button again.
Eventually you will get the crash.
