My application is having problems in ML with the rendering of menu items of the
Telerik RadMenu. When you hover a menu item for the first time, its children
menu items should be dropped down and displayed. However, this will only happen
when you start moving the mouse ever where they should be. Then each item will
begin to appear one by one as you move the mouse over to their position. Even
then, they are not rendered properly, even though they do become
distinguishable.

After closing the menu, the second time you pull down the menu by hovering the
mouse, it will display properly. Every other attempt will be consistent with
the second. This happens with a very simple default configuration. For
instance, if you change ClickToOpen to true (non-default value), then it will
work fine.

I am attaching a working demo so you can reproduce this bug. Unlike bug 656289
(https://bugzilla.novell.com/show_bug.cgi?id=656289), which I also reported,
this is *not* another case of gallium missing, as now I am compiling with
gallium support and this bug still shows.

I have pinpointed this problem down to commit 5e3b44033. Telerik doesn't like
the if introduced in that commit. :-P

Reproducible: Always

Steps to Reproduce:
1. Run the TestPage.html in the attached working demo.
2. Pull the menu down just by hovering it. Don't click the top menu it,
otherwise it will work as supposed, not reproducing the bug. Notice how it is
being rendered.
3. Pull down the menu a second time by hovering. See that this time it is doing
fine.
Actual Results:  
Rendering problems.

Expected Results:  
No rendering problems.
