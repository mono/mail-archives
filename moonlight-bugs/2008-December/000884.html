If the media source in use is a broadcast publishing point from a Windows Media
Server, playback does not start when the application loads. Observed this
behavior on Ubuntu Desktop 8.04 using the 1.0b1 Moonlight plugin with Firefox
3.0.4.

Repro:

1. In Firefox 3.04, obtain Moonlight plugin from
http://www.go-mono.com/moonlight and restart browser.
2. Browse to streaming video application and accept prompts to install
Microsoft Media Pack.
3. Refresh page.

Alternate repro:

1. On a machine with the plugin and codecs already installed, launch Firefox
and browse to application.

Result:

A single frame from the video stream is displayed, but stream does not play.

Correct behavior was verified in other platforms using the Silverlight 2.0
plugin (WinXP: IE and Firefox, Mac OSX: Firefox)
