Description of Problem:
It looks like Moonlight is not firing off the correct, MediaElement related,
events when playing a stream that comes from a Windows Media server.


Steps to reproduce the problem:
1. Create a XAML file containing a MediaElement
2. Set the source to a live or on demand stream from a Windows Media server
2. Hook up event handlers for BufferingProgressChanged, CurrentStateChanged,
MediaOpened, MediaEnded
3. Run it in Moonlight


Actual Results:
Video will not play and give incorrect status events, or not give these status
events at all. MediaElement appears to buffer the stream, but does not give
feedback about it untill after buffering, and then stops. MediaElement then has
to be told to start playing several times before it actually does.

Expected Results:
Live or on demand video from a Windows Media Server should just work.

How often does this happen? 
Every time.

Additional Information:
Because i'm not actively developing for SL 1.0 and only run Linux in VMs, I
have not been as complete in providing information as I could be. If you need i
could provide a test case and media links.
