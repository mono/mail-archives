--- Comment #2 from Marcel Brouns <marcel at brounsmedia.com>  2008-12-22 16:59:05 MST ---
I was about to report this as well.

The above described problem occurs in 100% of cases where a Server Sided
Playlist is used on the Windows Media Server. The OP mentions the "Broadcast
Publishing Point", which is a basically a virtual directory that hosts a Server
Sided Playlist and streams that out live or on demand. 
These Playlists can contain a mix of live and archived sources, and can be
manipulated programmatically while the Playlist is running.
Server Sided Playlists are typically used for ad prerolls/inserts, switching
between different live or on-demand feeds, repeating a live webcast, automated
failure recovery during live webcasts etc. For people using Windows Media
Server professionally, this is an important feature to have.

This sample runs in Silverlight 2.0.
mms://wmv.companywebcast.nl/playlist
This playlist is supposed to loop a short clip. 
Moonlight seems to load the stream ok initially, as it shows a frame, but never
plays the stream. 
Note that if you time your request to the stream with the moment the server
rolls to the next item in the playlist, that one entry from the Server Sided
Playlist will play in Moonlight, it will not keep playing when the server
changes to the next item in a playlist.

Microsoft themselves actually did not implement support for this untill
Silverlight 2 beta 2.

It would be cool tho, and help many people like me, if Moonlight already
supported it in 1.0.
If you look at the forums on silverlight.net, you'll see a bunch of people
asking about support for this, and finally someone from MS asking for specific
info about Server Sided Playlists from actual users, and then they implemented
in SL2 beta 2.
