--- Comment #3 from Jackson Harper <jaharper at novell.com> 2011-01-31 22:06:13 UTC ---
There are two possible ways we can fix this.

1) We could set a flag on ignored elements and check it when adding children.

2) We could use a stack to track the ignored element depth and return true from
IsIgnoredElement whenever we are inside of an ignored element.

I think #2 is probably the correct solution but its possible that we might miss
errors in the ignored XML.  So we should test to see if Silverlight catches
errors in there too.  They would have to be errors like missing properties
though, not XML formatting errors.
