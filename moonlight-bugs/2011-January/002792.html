Test Case

User-Agent:       Mozilla/5.0 (X11; U; Linux x86_64; en-US) AppleWebKit/534.10
(KHTML, like Gecko) Chrome/8.0.552.224 Safari/534.10

Hi, my client's app has something similar to the following:

<Button IsEnabled="{Binding IsEnabled, Converter={StaticResource NonExistant}}"
Content="This is a Button" />

Naturally, in their application, the resource's key isn't NonExistant, but
something that from what I've found has a meaning in other xaml files (seems to
be a case of copy-paste).

However, it just so happens that in both Silverlight and in Moonlight with the
unmanaged parser, this ends working. It even works as intended, and if the
datacontext has a boolean IsEnabled property set to false, the button will be
disabled indeed.

Moonlight's managed parser, however, is much less forgiving as it throws an
exception and fails to render the usercontrol.

I am attaching a test-case.

Reproducible: Always

Steps to Reproduce:
1. Run the TestPage.html of the test case I am attaching, using the managed
parser.
2. Notice that the page is not rendered and that you have an exception in the
terminal.
3. You can try this in Silverlight and with Moonlight using the unmanaged
parser and notice that you will get a disabled button and no exception in the
terminal.
Actual Results:  
Exception and failure to load the usercontrol.

Expected Results:  
Disabled button being displayed.
