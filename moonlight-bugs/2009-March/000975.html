To be 100% "throw exceptions!" compatible with Silverlight, doing:

PasswordBox.GetValue
(TextBox.[Text,MaxLength,Selection[Background,Foreground]]Property)

should throw an exception instead of returning the values for
PasswordBox.PasswordProperty, PasswordBox.MaxLengthProperty,
PasswordBox.SelectionBackgroundProperty and
PasswordBox.SelectionForegroundProperty, respectively.

Unfortunately, this will require some painful code to synchronize the values or
something, which I'm not excited about doing.

Marking this as a cosmetic problem because I find it unlikely that anyone will
actually write Silverlight apps that depend on the above throwing an exception.
