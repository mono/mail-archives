--- Comment #7 from Alan McGovern <amcgovern at novell.com> 2010-10-28 12:29:01 UTC ---
Just for reference, the bug in this case was caused by us incorrectly
calculating how much size a * row/column would consume when there was a grid
segment using max or min constraints. We offered a size that was too small
during the Measure pass but offered the correct size during the arrange pass.
In the case of ScrollViewer this meant that the Viewport changed which caused
an InvalidateMeasure during the Arrange pass and so resulted in an infinite
loop.

This has been fixed in git master.
