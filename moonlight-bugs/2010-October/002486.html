        <Grid.RowDefinitions>

            <RowDefinition MaxHeight="50"/>

.. // rest omitted for clarity

The problem is in the definition of the second row. If I replace MaxHeight="50"
by something different like Height="Auto", the prototype will not hang. In the
sample I attached, there is no actual element taking its place in the second
row. However, in the application I am debugging, there is and the behaviour is
the same. I've worked-around this bug on that application precisely by removing
the MaxHeight="50", putting a Height="Auto" and on the element itself, adding a
Height=50.
