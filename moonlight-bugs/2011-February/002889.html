Hi, very very occasionally my application is having an exception when loading.
This is happening very rarely, so I am having a very hard time reproducing it.

I am attaching the exception I am catching. It is the wrong exception, however,
since the method it comes from is the following:

            try
            {
                updatingRegionsListeners.Raise(null, EventArgs.Empty);
            }
            catch (TargetInvocationException ex)
            {
                Exception rootException = ex.GetRootException();

                throw new
UpdateRegionsException(string.Format(CultureInfo.CurrentCulture,
                    Resources.UpdateRegionException, rootException),
ex.InnerException);
            }

So, what we are seeing is a new exception in the catch block and not the
original one. I've tried to out-comment the try/catch and leave just the
content of the try, but still didn't succeed in reproducing again the bug.

However, digging a little deeper into the code, the following method is being
called:


        public void Raise(params object[] args)
        {
            this.listeners.RemoveAll(listener => listener.Target == null);
            // I later added this here: GC.Collect();
            foreach (Delegate handler in
this.listeners.ToList().Select(listener => listener.Target).Where(listener =>
listener != null))
            {
                handler.DynamicInvoke(args);
            }
        }

Suspecting the indeterminism of the exception is related to when GC kicks in, I
tried to force the GC, where you can find the comment // I later added this
here: GC.Collect()

After trying a few more times, I had not an exception, but in fact a crash, the
output of which I am also attaching to this bug.
