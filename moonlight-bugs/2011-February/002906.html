I have a binding that uses a boolean to visibility converter and has a
FallbackValue of Collapsed. The fallback value is apparently ignored in this
case.

Tested using revision 83e8b4f.

Reproducible: Always

Steps to Reproduce:
1. Create a Visibility binding that uses a converter and a fallback value as
described above
2. Observe the element before the binding has a value
Actual Results:  
Even though the FallbackValue is Collapsed, the element is shown (Visible).

Expected Results:  
The element should be Collapsed when the binding has no value.
