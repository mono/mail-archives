Description of Problem:
Starting from version 2.99.0.9 moonlight plugin 32 or 64 bit into Firefox or
Google-Chrome browser does not synchronize audio and video working on some
websites. For example RAI-TV official website in Italy (streaming). For a lot
of people in Italy it's a problem and the Forum talking about it.


Steps to reproduce the problem:
1. Go to Italian RAI-TV
website:http://www.rai.tv/dl/RaiTV/diretta.html?cid=PublishingBlock-57351f9c-f319-4526-b0a5-da6e19f98d74&amp%3Bchannel=Rai%2520Due 
2. You can notice the problem 


Actual Results:
Downgrade to 2.99.0.8 version: it's work!

Expected Results:


How often does this happen? 
Always, many Linux distro (32 or 64 bit) but the problem persist!


Additional Information:
To help I noticed that the problem disappears launching a script with
"xautomation" installed to move quickly the mouse. The content of the  script
is: 
while ((1))
 do 
 xte 'mousermove 1 1'
 xte 'mousermove -1 -1'
 done

placing the mouse over the plugin the problem disappears! Very strange.
