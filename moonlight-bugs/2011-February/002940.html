A simple test case to reproduce the issue

User-Agent:       Mozilla/5.0 (Windows; U; Windows NT 6.1; en-US)
AppleWebKit/534.13 (KHTML, like Gecko) Chrome/9.0.597.98 Safari/534.13

I found that a RadioButton in my project wasn't showing its content and I
narrowed it down to one attribute in the style: the
ToolTipService.PlacementTarget. When this attached property is set to a
TemplatedParent relative binding, the content on the radio button doesn't show
up.

I don't see any errors in the log and it seems that other types of bindings
don't cause this issue. I don't know if the issue has anything in particular to
do with the ToolTipService, or if it's just relative bindings, etc.

Also I found that the RadioButton's Content must be bound for this issue to
occur. A local value didn't seem to reproduce it.

Reproducible: Always

Steps to Reproduce:
1. Style a RadioButton and include this attribute somewhere in the style:
ToolTipService.PlacementTarget="{Binding RelativeSource={RelativeSource
TemplatedParent}}"
2. Bind the RadioButton's Content property to something in the data context.
Actual Results:  
The content of the RadioButton does not show up, i.e. it behaves as collapsed.
I verified from code-behind that the Content property is in fact set correctly.

Expected Results:  
The content of the button should be both visible and affect layout.
