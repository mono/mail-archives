Martin Bennedik's Silverlight chess game viewer is used extensively on popular
TWIC web site (The Week In Chess). I notice that the memory usage by firefox
increases dramatically while using this plugin. The leak speed seems to be
particularly bad when playing games from large play lists, e.g. here:

http://www.chess.co.uk/twic/chessnews/events/moscow-open-2011

On this page I played through only two games and while doing so the FF memory
consumption increased in the order of a Gigabyte. If exact numbers are desired
I can try to measure a bit more precisely. Playing several games is a sure way
to crash the browser (when memory usage exceeds available RAM).

Martin Bennedik makes his chess board available here:
http://www.bennedik.de/Silverboard.html

I informed him of the problem, but he says that this problem does not occur on
Windows with native silverlight, so it must be a problem of the moonlight
plugin.

Reproducible: Always

Steps to Reproduce:
1. go with firefox to
http://www.chess.co.uk/twic/chessnews/events/moscow-open-2011
2. select a game in the game viewer, click on the "|<" button, that should
close the game selection menu (which should have closed by itself, but that is
another, very minor bug); you should now see the begin position of the chess
game
3. play through the game by clicking the ">" button, maybe go back and forth a
few times
4. repeat 2&3 for another few games

Actual Results:  
memory consumption rapidly increases with every move played, eventually
crashing the browser.

Expected Results:  
The information content per chess game is very low, in the order of kilobytes
or less, so even with hundreds of games in a play list there should not be
large memory consumption.
