Small project to reproduce the issue

User-Agent:       Mozilla/5.0 (Windows; U; Windows NT 6.1; en-US)
AppleWebKit/532.5 (KHTML, like Gecko) Chrome/4.1.249.1064 Safari/532.5

If I dynamically invoke a lambda expression tree that contains a reference to a
local variable, a runtime exception is thrown in Moonlight, whereas in
Silverlight the code executes without a problem.

Reproducible: Always

Steps to Reproduce:
1. Create a method that accepts an Expression<Action> parameter
2. Dynamically invoke the expression in that method
3. Call the method, passing in a lambda that references a local variable
Actual Results:  
A runtime error is thrown with the message:

Message: Error while loading the 'InvokeLambdaTest.dll' assembly : Transparent
method System.Linq.Expressions.EmitContext:CreateStrongBox (object,System.Type)
cannot invoke private/internal method
System.Runtime.CompilerServices.StrongBox`1<InvokeLambdaTest.MainPage/<>c__DisplayClass0>:.ctor
(InvokeLambdaTest.MainPage/<>c__DisplayClass0).

Expected Results:  
The lambda expression should evaluate normally, as is the behavior in
Silverlight.

Tested both in Moonlight 3 preview 6 and on a recent revision (I think 157700)
