--- Comment #12 from Atsushi Enomoto <aenomoto at novell.com> 2010-02-25 02:31:47 UTC ---
The reasons why it cannot be really done in details are:

1) ChannelFactory<T>.CreateChannel() returns IClientChannel, but there is no
publicly exposed type that implements this interface. An exception is
ClientBase<T>.ChannelBase<T> but it is limited to Silverlight and since this
bug is explained to apply also to full CLR, it is not a solution.
2) CreateChannel() returns an instance of type T which is unknown. Either
something that dynamically generates such a proxy type for the type in the
application assembly, or some existing implementation is required. While
Silverlight client proxies are with ChannelBase<T> implementation, Silverlight
does not *require* such a derived class.

The workaround suggested above is related to 1). Not sure if another solution
for 2) is required.
