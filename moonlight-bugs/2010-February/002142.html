Hi..

I think where the problem lies.

This XAP uses a custom build Theming class.

What it does is, on startup the application class loads a linked (as content)
resource xaml file,

parses it using a XML reader, and manually create objects using the XamlReader
class, that it adds as resources to the application.

Maybe this is where Moonlight slightly differs from Silverlight.

I created a very simple version of my site, with the same theme.

Preview here: http://search-geenstijl.thruhere.net/search/testpage.html

Sourcecode here:
http://search-geenstijl.thruhere.net/search/simplethemeexample.zip

I have to say, the code is not pretty, and I may do things a little unorthodox,
especially knowing that Silverlight 3 offers theming, but I guess you guys are
after why it differs from Silverlight and in that search, by coding style used
here is not relevant.

Also, My Visual Studio has already been upgraded to Silverlight 3 and although
I changed the generated TestPage.html to the 2.0 runtime, there is a change you
need to downgrade the solution you find in the ZIP archive, not sure about
that.
