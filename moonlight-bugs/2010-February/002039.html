NOTE: I don't necessarily think this is an engine bug, but no other component
seemed to fit.

The TextBlock control supports setting the Content with <LineBreak/> tags to
display text with blank lines.  If you then query the Text property, on
Windows, it strips out those newlines (\r, \n and possibly other special
characters).  Moonlight does not do this.
