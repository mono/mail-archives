--- Comment #2 from Alan McGovern <amcgovern at novell.com> 2010-02-10 16:50:33 UTC ---
Ok, here's what I've found out so far:

1) Taking the testcase and changing the ComboBox to a ListBox results in
moonlight returning the string ",,, 3". Looks like we've got a bad
implementation of GetName (). The implementation might depend on whether or not
the item is its own container.

2) Taking the above testcase and clicking the button *without* expanding the
combobox works in SL but null refs in moonlight. I think moonlight is trying to
get the containers for each item whereas Silverlight probably uses the item
itself.

3) If I alter the testcase to add the items to the 'Items' property as opposed
to using the 'ItemsSource' property, the behaviour in SL doesn't change and ML
breaks in the same way.

This could be an SL2 -> SL3 change. I only ran these tests on SL3. As it
stands, I don't think this is an ItemsControl/ComboBox bug.
