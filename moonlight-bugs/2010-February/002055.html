The "Path" XAML element doesn't support any line cap types, even though it's
supported with the "Line" element.  In the test XAML segment below, Moonlight
renders the Line end caps correctly, but not the Path (it shows the default
Flat end caps).  I'm assuming this is a simple omission in the code, since I
would imagine the cairo stroke operation is used to render both elements, and
cairo obviously supports the round cap type.

Test XAML segment:

<Canvas>

<Path Stroke="Black" StrokeThickness="30" StrokeStartLineCap="Round"
StrokeEndLineCap="Round" Data="M 10,100 C 10,300 300,-200 300,100" />

<Line Stroke="Black" StrokeThickness="30" StrokeStartLineCap="Round"
StrokeEndLineCap="Round" X1="300" Y1="300" X2="500" Y2="500" />

</Canvas>
