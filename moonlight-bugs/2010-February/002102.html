--- Comment #6 from Tamas Hubai <htamas at bugmail.ces.hu> 2010-02-19 19:11:09 UTC ---
I'm not the original bug reporter, but here are the steps I used to reproduce
it:

1. Start Firefox 3.6 (create a new profile if you want)
2. Visit http://www.go-mono.com/moonlight/ and install Moonlight 2.0
3. Restart Firefox
4. Visit http://www.sipsorcery.com/

Actual results:
The sipsorcery site loads, including the silverlight plugin. A loading
animation is displayed. After a few seconds, a login form is shown with a red
circle and the error message "There is no more available binding element."
above it. If you click the "Login" button, nothing happens. (Depending on your
configuration, a backtrace might be shown under Tools / Error Console. I've
attached such a backtrace in my previous comment.)

Expected results:
After the loading animation, a teal circle should be displayed instead of the
red one, accompanied by the message "Initialising...". And within a few seconds
it should be replaced by a green dot and the message "Provisioning service
ok.". A "Create New Account" link should also appear. Clicking the "Login"
button should display an "Attempting login..." message and continue with the
login process.

Further comments:
The silverlight application is currently fetched from
http://sipsorcery.blob.core.windows.net/sipsorcerymvc/sipsorcery.xap . The
corresponding source code is available at http://sipsorcery.codeplex.com/ . I
don't know much about its internals, but Aaron, the author and maintainer, is
quite helpful, so we can invite him to the discussion if you'd like to see some
more details.
