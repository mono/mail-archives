The verifier does its job - i.e. not allowing code X to access the internals of
assembly Y unless [InternalVisibleTo] are present.

CoreCLR does its job - i.e. it does not allow application (non-platform) code
to access platform code internal code (with or without IVT).

However the proxy code Mono's WCF is using is not made from valid code
(unverifiable wrt ECMA) because it violate the verifier accessibility rules
(nothing to do with CoreCLR). 

Not sure how deep the problem reside but it definitely starts at
System.ServiceModel.dll which instantiate the proxy.
