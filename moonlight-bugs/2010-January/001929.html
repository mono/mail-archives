This fails the CoreCLR check in mono_security_core_clr_can_access_internals
because an untrusted (SRE) assembly tries to gets the internal of platform
code. Not sure how MS handles this but it could be related to the comment in
the source code:

            // in .NET and SL2, it seems that the proxy is RealProxy.
            // But since there is no remoting in SL2 (and we have
            // no special magic), we have to use different approach
            // that should work either.

However even if the CoreCLR check did not happen, this would still fail later
since there's no [InternalsVisibleTo] to allow this between the assemblies.
