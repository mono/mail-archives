Description of Problem:

After building and installing Mono 2.6.1 and the Tagged version of Moonlight
2.0 running the example simpleclock desklet app via mopen generates an error.
Normal console applications and WinForms based applications work without any
problems.

This is running on a ARM based SheevaPlug computer with a Mimo 740 USB touch
screen. The Plug is running Debian Lenny.


Steps to reproduce the problem:
1. Build and Install Moonlight 2.0 (see Additional notes)
2. navigate to /examples/desklets/simpleclock 
3. run mopen simpleclock.xap

Additional Information:

In order to get moonlight to compile under arm the follow lines needed to be
added to moon/m4/firefox-xpi around line 43

  arm*)
      TARGET_PLATFORM=arm-gcc3
      INSTALL_ARCH=arm
  ;;

Moonlight also needs to be compiled with --disable-plugin
