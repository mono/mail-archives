Unfortunately the issue is originating as an X error and X errors are
asynchronous which means the stacktrace you provided may not actually be
correct. We think that if you pass the "--sync" argument to firefox this should
cause it to use X synchronously which means if it crashes the stacktrace will
be correct. This will also probably cause firefox to be slower.

So to test, just start firefox from a term with "--sync" as a parameter and
when it crashes give us the new stacktrace, exactly like you did with the old
one.
