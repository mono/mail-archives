The second one (NullReferenceException) however is more interesting, it looks
like we end up evaluating binding expressions before objects are properly
created:

Msn.Video.Widgets.Player.Inline.EndSlate.get_SeasonLabelText accesses a
property which is only set after this:

  at Msn.Video.Widgets.Player.Inline.PlayerView..ctor () <IL 0x00043, 0x000b3>

has completed (later in the method
Msn.Video.Widgets.Player.Inline.App.OnPlayerInitialized just below).

It's possible to reproduce this locally by 
* downloading the xap from the above log
* create a dummy html file using the above xap
* run xsp2
* modify System.Windows/System.Net.Browser/PolicyBasedHttpWebRequest.cs
commenting out this chunk at line 170 approx:
if ((policy == null) || !policy.IsAllowed (wreq)) { ... }
* load the dummy file in the browser
