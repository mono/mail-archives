Reproducible: Always

Steps to Reproduce:

1. Unzip the attached archive and find the SilverlightApplication1TestPage.html
file inside the SilverlightApplication1.Web project. Open the page in Firefox
(3.6.10). A text box will show up showing the text "Loading...".

2. Place the cursor behind the L and o of "Loading", then hit the Ctrl-Enter
key combination. A dialog will come up showing that the Keyboard.Modifiers
correclty reported that the control key was held.

3. Dismiss the dialog. Notice how the text changes from a single line
"Loading..." to two lines: "L" (line 1) and "oading..." (line 2). This is
unexpected. The Enter key should have been ignored.

4. Now, hit the Enter key by itself. The application KeyDown event handler
raises the MessageBox dialog again because the Keyboard.Modifiers is still
reporting that the control key is pressed. The Keyboard.Modifiers should be set
to None.
