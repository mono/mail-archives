--- Comment #2 from Rolf Bjarne Kvinge <rkvinge at novell.com>  2009-10-07 04:08:01 MDT ---
This crashes because the site (or we) change a property and in the property's
changed event we change it again, which will delete the first value of the
property, but we may end up trying to use it anyway.

it basically goes like this:

DependencyObject::SetValueWithErrorImpl is called to set the property
-DependencyObject::ProviderValueChanged is called with the new value
--DependencyObject::SetValueWithErrorImpl is called again to set the value to
something else
--DependencyObject::SetValueWithErrorImpl deletes the old value
-DependencyObject::ProviderValueChanged uses the previous new value (which was
just deleted) and we crash
