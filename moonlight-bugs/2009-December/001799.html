On firefox-3.5.5 and 3.6b4, Gecko Media Player plugins are removed/blocked
after installing the moonlight plugin. If the moonlight plugin is disabled or
removed from firefox the pluginreg.dat file is not refreshed. Deleting
pluginreg.dat file from the firefox folder restores the gecko media plugins (
as long as the moonlight plug is removed or deleted ) This problem has been
present since moonlight 2 beta 8.

Mozilla/5.0 (X11; U; Linux x86_64; en-US; rv:1.9.2b4) Gecko/20091127
Fedora/3.6.1-0.5.b4.fc13 Namoroka/3.6b4 GTB6

http://code.google.com/p/gecko-mediaplayer/issues/detail?id=31

Reproducible: Always

Steps to Reproduce:
1.Install moonlight.
2.restart firefox
3.Check about:plugins
4.Delete or disable the moonlight plugin
5.about:plugins
Actual Results:  
The moonlight plug shows up on the plugin list and the gecko media player plugs
are disabled. If the moonlight plugin is disabled or removed from firefox the
pluginreg.dat file is not refreshed.

Expected Results:  
The moonlight plugin should not interfere with the gecko media player plugin or
any other. When the moonlight is disabled or removed, it should restore the
previous plugs without having to delete pluginreg.dat
