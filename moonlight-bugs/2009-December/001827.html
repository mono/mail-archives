The Moonlight Codecs Installer presents the user with an EULA to accept.
However, the colours used for the licence text make the text illegible if the
user uses a light-on-dark colour scheme on his desktop.

Reproducible: Always

Steps to Reproduce:
1. Use a light-on-dark colourscheme for the desktop (affects GTK+. QT, etc.)
2. Get the Moonlight Codecs Installer to show a licence agreement for
something.


Expected Results:  
Legible text. "select all" makes it legible. In the case of the Microsoft
codecs that got presented to me, that didn't help much though.. I saw letters,
but it was written in incomprehensible English legalese and French.. (!)

To prevent these kind of issues, fixed and user-defined colours should not be
mixed for fore-/background usage.

Not sure about the component this bug falls under. There are no component
descriptions!
