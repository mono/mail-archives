--- Comment #6 from Sebastien Pouliot <spouliot at novell.com> 2009-12-21 18:50:51 UTC ---
So the problem is that:

* Silverlight returns a [Internal]MemoryStream while ML returns a (read-only)
FileStream.

* StreamResourceInfo (which is not IDisposable) can be used quicker than the GC
can dispose its [File]Stream instances.

I suspect a "normal" MemoryStream is not being used so that multiple streams
can share the same (unmanaged?) memory - cutting down the application memory
requirements.
