As soon as I have installed the moonlight 2.0 plugin, Gecko-mediaplayer crashes
when playing links from eg. uitzeninggemist.nl (for example:
http://player.omroep.nl/?aflID=10439597). From that site, I am not using the
silverlight stream but the mediaplayer stream, and gecko-mediaplayer to
transfer that stream to mplayer.
My system is Debian testing, kernel 2.6.31, running kde 4.3.2 and iceweasel
3.5.5:
Mozilla/5.0 (X11; U; Linux x86_64; nl; rv:1.9.1.5) Gecko/20091123
Iceweasel/3.5.5 (like Firefox/3.5.5; Debian-3.5.5-1)

Reproducible: Always

Steps to Reproduce:
1. Install gecko-mediaplayer (done via disto-package)
2. Install moonlight 2.0 from go-mono website
3. Open a webpage with an embedded mediaplayer movie
Actual Results:  
The system crashed, after which the gecko-mediaplayer plugins were no linger
visible on the about:plugins page. Only a fresh profile, or a copy of a
pluginreg.dat from a fresh profile, makes the gecko-plugins visible again, yet
everything happens again when the moonlioght 2.0 plugin is installed

Expected Results:  
I expect a movie to be played by an embedded mediaplayer, and my plugins not to
dissappear
