It seems to me that the error is do to no image being present, PixelWidth and
PixelHeight gets the hight/width of the bitmap in pixels, not the amount of
pixels you've allocated to it when showing it.

http://msdn.microsoft.com/en-us/library/system.windows.media.imaging.bitmapsource.pixelheight.aspx

So if no image is present it should (and correctly does, on my system at least)
show 0. If you put an image of 20x20 it should show 20 and not 512 even though
you allocate 512. And so forth.

Does you have a resource with the image, and if not could you please test with
an image present?
