The latest nvidia driver improves rendering performance drastically. A version
check against the driver should be added to allow rendering to go through xlib
instead of forcing client side.

With client side rendering, a 720p video scaled to fullscreen using fast
filtering yield 100% CPU usage at 15fps.

With xlib rendering and the new driver, the same video scaled to fullscreen
using /bilinear/ filtering yields about 15% cpu usage at 52fps.

Two things:

a) allow xlib backend against new driver/hardware
b) do adaptive filtering, starting with bilinear and falling back to fast if
the target framerate is not met (not nvidia specific)

These two changes allow Moonlight to handle HD content that is fast and looks
good! EXCITING!
