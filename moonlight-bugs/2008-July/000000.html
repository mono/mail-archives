After performing a search on Tafiti, you are able to click on a "Tree View"
link at the top of the page. Once you do that, you get this spinning tree with
leaves. Each leaf is a search result.

If you mouse-over a leaf node summary string, it displays a tooltip with more
details about the result. The problem is that the URL is outside of the tooltip
background rectangle.

Looking into the javascript, the reason becomes clear. Instead of using the
tooltip.link.actualHeight to add to the tooltip.bg.height, they use
tooltip.date.actualHeight. Since tooltip.date seems to always be null/empty,
Moonlight is returning a value of 0.0 for the tooltip.date.actualHeight value.

Presumably since this works fine in Silverlight (which is probably why they
have not noticed their js logic bug), the actualHeight of a null/empty string
must be default to the height of the font.
