--- Comment #10 from Sebastien Pouliot <spouliot at novell.com>  2008-07-09 12:51:19 MDT ---
So this is "by design" because we compute bounds into rectangles (which make
sense) but without the transformation (which we apply later).

In the case of an ellipse (like a perfect circle) a rotation should *not*
change the bounds (it's still a circle). However because we use a rectangle
(and later apply the transformation to it) we grow the bounds needlessly
(because a rotated rectangle changes bounds).

In this specific case I'm not sure it change performance much since most text
boxes are already intersecting the ellipses. However it could be a problem wrt
stretching since it has proven to requires very exact bounds.
