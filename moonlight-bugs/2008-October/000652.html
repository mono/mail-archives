Repro:

Open http://the-hub.tv/silverlight
Pause the video
Show exposed regions

The text area below the video and the one in the lower right corner will now
flicker. 

The javascript is hiding a lot of areas, and then making visible only those
which should be shown every 25 ms. Issue is that those areas which should be
shown rarely changes (once every time the video source changes), causing a lot
of unnecessary redrawing.

Commenting out the javascript doing this hiding+showing, makes the site go from
4.5 fps to 11.5 fps on my machine.

I guess one idea would be to track if the visibility changed between renders /
tick calls, I have no idea if this is feasable though.

Note: SL is invalidating those areas, so it might not be possible to optimize
away the invalidation.
