A corrupted media file (attached) can cause a deadlock in
pthread_cond_wait (&queue_condition, &queue_mutex);

A simple timed wait did only fix half of the issue (it can be closed correctly)
but the mopen (using glib timeout) should have shut itself down.

timespec ts;
clock_gettime (CLOCK_REALTIME, &ts);
ts.tv_sec += 5;
if (pthread_cond_timedwait (&queue_condition, &queue_mutex, &ts) == ETIMEDOUT)
{
        g_warning ("ETIMEDOUT");
        stopping = true;
}
