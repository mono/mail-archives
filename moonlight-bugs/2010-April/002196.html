I'm running your openSUSE 11.2 VMware image with Mono 2.6.3 on VMware
Workstation 7.0.1, updated to use Moonlight 3 Preview 6. I build the
Silverlight version of Virtu 0.8.1 (http://bit.ly/96cI0r) with MS Visual Studio
2008 SP 1, then drop the binaries onto the above VM and run it. Note I'm the
Virtu project owner.

Virtu starts up but soon hangs due to the audio thread not generating events -
the machine thread syncs with the audio thread. I've added logging to the main
page that should log 'OnMediaSourceUpdate' every second, and this highlights
that this callback is stopping within a few seconds. Basically Virtu is using a
derived MediaStreamSource and generating samples in GetSampleAsync callbacks.

This works fine on my Windows PC, on a Windows PC in a VM, and on the Macintosh
running Snow Leopard, but not in the above scenario which is how I'm testing on
Linux.
