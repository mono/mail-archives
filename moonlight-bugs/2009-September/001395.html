I realize that the mouse wheel is a SL3 feature and that Moonlight is currently
targeting SL2, however, any app that uses the mouse wheel appears to not load
in Moonlight at all.

I'm not asking for full mouse wheel support.  It would just be nice if the
types were stubbed out so the apps that use them load - it's fine if the
MouseWheel event never fires.

Description of Problem:  Subscribing to the MouseWheel event causes the app to
be completely unusable in Moonlight (tested with Moonlight Beta 3)

Steps to reproduce the problem:
1.  Create a new Silverlight 3 app (Visual Studio was used when testing)
2.  Define an object that subscribes to the MouseWheel event, such as 
<ScrollViewer MouseWheel="ScrollViewer_MouseWheel" />
3.  Load the app in Moonlight
4.  Observe error

-OR-

1.  Load the app running at http://warlight.net/SinglePlayer.aspx
2.  Observe error

Actual Results:

Could not load type 'System.Windows.Input.MouseWheelEventArgs' from assembly
'System.Windows, Version=2.0.5.0, Culture=neutral,
PublicKeyToken=7cec85d7bea7798e'.

Expected Results:

A working app, even if the MouseWheel event never fires.

How often does this happen? 

Every time.
