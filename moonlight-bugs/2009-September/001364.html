--- Comment #2 from Alan McGovern <amcgovern at novell.com>  2009-09-02 11:32:13 MDT ---
This behaviour is correct for Silverlight 2.0. However, Silverlight 3.0 has
fixed the bug.

When we start releasing 3.0 compatible moonlight this will magically be fixed
so I'm marking this as not a bug.

If you care about the details, keep reading ;) The problem is that Silverlight
2.0 is missing a check in ClearContainerForItemOverride so it always sets the
'Content' property to null if the container is a ComboBoxItem. It should only
be doing this if the ComboBoxItem was autogenerated. The fix is to wrap the
call to "item.Content = null" with "if (container != item) item.Content =
null", but we can't do that until Silverlight 3.0.
