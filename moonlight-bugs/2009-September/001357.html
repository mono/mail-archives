--- Comment #7 from Rodrigo Kumpera <rkumpera at novell.com>  2009-09-01 07:36:49 MDT ---
The first issue is that our runtime doesn't handle null resolution scopes for
typerefs as ECMA 335 suggests - mono crashes trying to decode it.

Specially on this case, it's harmless because nothing uses this token, and I
believe it was improperly encoded anyway. This feature is required to support
the whole type forwarding machinery of dotnet 4.0.

The other issue looks to be the well know sane control flow restriction that
both JIT and IL verifier require. The MS runtime verifier/JIT allow dead basic
blocks with unverifiable code.

The whole thing boils down onto how much obfuscation madness are we going to
support. Both of the problems here are due to tools spewing a lot of crap in a
way we can't handle.
