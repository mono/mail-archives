Description of Problem:
Moonlight seems incompatible with Web Services client code generated in Visual
Studio. This breaks SL2 apps that are consuming Soap services (like the Google
Adwords API) through a Service Reference.

I put a simple testcase online.

Steps to reproduce the problem:
1. Open
http://services.companywebcast.com/MoonlightServiceReference/ServiceReferenceTestcaseTestPage.html
2. View this error in the FireFox error console:
Error: Sys.InvalidOperationException: Runtime error 2105 in control 'Xaml1',
method : Error while loading the 'ServiceReferenceTestcase.dll' assembly :
Exception has been thrown by the target of an invocation.
Line: 461


Actual Results:
Screen is blank.

Expected Results:
A TextBox should be shown, showing the status of a single Soap request.

How often does this happen? 
Always.

Additional Information:
Source for this sample can be downloaded here:
http://services.companywebcast.com/MoonlightServiceReference/ServiceReferenceTestcase.zip
