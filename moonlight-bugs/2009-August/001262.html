It looks like the chess board is (re)created on the resize event.

Something* (the ML bug) is causing a (extra) resize event to be generated when
creating the board. This result in an infinite recursive loop that fails once
all memory is exhausted.

* after some time you'll see NRE coming from AddSquare - but this is not what's
provoking the extra resize event. AddSquare works correctly until it's unable
to to retrieve a stream from the Merifont.ttf (likely out of resources).
