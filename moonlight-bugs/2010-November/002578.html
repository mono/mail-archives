Simple test case to reproduce the issue

User-Agent:       Mozilla/5.0 (Windows; U; Windows NT 6.1; en-US)
AppleWebKit/534.7 (KHTML, like Gecko) Chrome/7.0.517.44 Safari/534.7

Any time I set d:DesignerProperties.DataContext in xaml, I get a runtime
exception logged in MOON_TRACE:
System.Windows.Markup.XamlParseException: Unable to set property Content to
value [data context value].

This error prevents the view from loading properly.

Reproducible: Always

Steps to Reproduce:
1. Create a view and set any value in d:DesignerProperties.DataContext
2. Run the application in Moonlight
Actual Results:  
A XamlParseException is thrown

Expected Results:  
The design-time data context should be ignored at runtime and the view should
load properly.
