--- Comment #3 from Alan McGovern <amcgovern at novell.com> 2010-11-01 11:11:12 UTC ---
For every UIElement you create in managed code, there's a counterpart created
in native code. The problem is that the managed counterpart is being garbage
collected early so when the Loaded event is being invoked from native code, it
blows up because the event handler that's being invoked has been garbage
collected.

You might be able to make a more reliable repro by adding in GC.Collect calls
somewhere, but it's a tricky one to reliably reproduce. This is something we're
working on but if you can find a reasonably easy way to reproduce the issue
that would definitely help us.
