I've written a Silverlight demo that a user can load a WavPack audio file from
their computer and it will play the file. So its a full implementation of an
WavPack audio codec decoder, and should play the audio.
The demo works fine on XP (with normal Silverlight), however on Ubuntu using
Firefox 3.6.10 with Moonlight 2.99.0.9 I get an unusual issue.
There is silence for a couple of minutes after the user selects the file, then
the audio plays correctly, however after a short while (say 30 seconds) the
audio playback is corrupted and you get a metallic sound instead of the
expected music.

Reproducible: Always

Steps to Reproduce:
1. Access the demo web page
http://members.upc.ie/p.mc.quillan/silverlight/demo.html

2. Load a WavPack file from your local machine (by selecting the 'Open'
button). Note, if you do not have any WavPack audio files, you can get a sample
file from
http://samples.mplayerhq.hu/A-codecs/lossless/

The file to use is luckynight.wv
Actual Results:  
You get silence for a couple of minutes, then the file should play correctly
for a period of time, it will then stop playing normally, you will get a
metallic/screeching sound.

Expected Results:  
Playback of the sound should happen within a couple of seconds of selecting the
file and it should playback music with no issues.
The demo (with the sample file above) plays as expected on XP/Silverlight.

When I run firefox from the command line, I'm not seeing any log information
that would indicate what the issue is (I may not be running at a high enough
verbosity level).
The source code for the demo (if required) can be found at:
http://www.wavpack.com/downloads.html
