I've investigated a bit better now.

With 3.0 preview 9 the problem I originally reported is gone.

The problem I had in comment 3 was a different one.  On the host I first used
for testing Moonlight obviously is denied the SELinux access type execmem
(making memory executable).  This was why it crashed.  If I set the SELinux
boolean allow_execmem to true, it works.

I don't want to have this allowed generally.  But if I just figure out how to
make Moonlight a system plugin rather than one in my home directory, I can wrap
it using nspluginwrapper.  If I do that, I expect everything to work just fine,
since nspluginwrapper is allowed execmem by default.

But again, the problem I originally reported is indeed gone.
