We have an application that uses tabs to emulate the concept of tabbed-windows.
If we load a few tabs that are heavily loaded with controls, the browser hangs
for periods when selecting tabs. Memory consumption also grows.

I am aware this is most probably a different manifestation of the bug
https://bugzilla.novell.com/show_bug.cgi?id=652177. Nevertheless because it
might not be, I am submitting it as a separate bug.

I am attaching a working demo of this problem.

Reproducible: Always

Steps to Reproduce:
1. Read the README file in the attached working demo (you can skip this, but it
might spare you some trouble understanding the demo if you read it).
2. Run the TestPage.html, also in the working demo.
3. Press the "All In One Button". Alternatively, you can press the "Add Batch"
so it adds a batch of tabs, and select each one manually.
4. See the browser hang for periods while memory grows. The browser will not
crash unless you reach an OOM.
Actual Results:  
Browser hanging for periods while it is thinking about life.

Expected Results:  
Snappy application.
