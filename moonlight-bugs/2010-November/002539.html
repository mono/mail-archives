It seems like ML is ignoring the MinWidth of a column definition. In the
following Xaml, the text "Something" should show slightly indented to the
right, which it is not in ML:

<Grid x:Name="LayoutRoot" Background="White" HorizontalAlignment="Left"
VerticalAlignment="Top">
        <Grid.ColumnDefinitions>
            <ColumnDefinition Width="Auto" MinWidth="24" />
            <ColumnDefinition Width="*" />
        </Grid.ColumnDefinitions>
        <TextBlock Text="Something" Grid.Column="1" HorizontalAlignment="Left"
VerticalAlignment="Center" />
    </Grid>

Reproducible: Always

Steps to Reproduce:
1. Run the TestPage.html of the attachment I am sending.
2. See the text "Something" fully to the left, when it should be slightly
indented to the right.
3.
Actual Results:  
MinWidth being ignored

Expected Results:  
MinWidth should not be ignored.
