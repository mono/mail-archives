I have a ToggleButton with a custom template. That custom template defines the
following visual state group for the common states. It defines a generated
duration of 0.1 seconds for the MouseOver and Pressed visual transitions.

Somehow, these are conflicting and creating an exception in ML. I have attached
a working demo. In this demo, when you check the button, it should show a
msgbox saying "I was checked". In ML, it doesn't raise that even and shows an
exception in the console that says  "Multiple animations in the same containing
Storyboard cannot target the same property on a single element."

Reproducible: Always

Steps to Reproduce:
1. Run the TestPage.html you will find in the attachment.
2. Press the "Tester" button (it doesn't really look like a button, but like a
textblock with a gray label, but it is a toggle button).
3. See the exception in the console.
Actual Results:  
An exception and no messagebox being displayed telling the button was
successfully toggled to checked. Notice however that the visual state of the
button can be deceiving and it will look like it did change to checked.

Expected Results:  
A messagebox saying the button was toggled to checked.
