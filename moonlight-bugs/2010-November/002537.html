I am populating an instance of a Telerik RadGridView with a few elements. When
loading the elements, it will consume a lot of memory (in the magnitude of
hundreds of Mb for a load of around a 100 records). When scrolling up or down
through the grid, memory consumption goes up continuously. Scrolling in
sequence, memory consumption will only go up without ever stabilizing much less
going down. With enough effort, I was able to crash the browser with a failure
to allocate memory.

Reproducible: Always

Steps to Reproduce:
1. Grab the attachment I am sending (SLProto4.tar.gz) and run the test page.
2. Launch any tool of your preference to monitor memory consumption.
3. Press the "Load" button to load the data into the grid (it is immediately
visible how much memory is being consumed).
4. Scroll up and down and see how memory consumption only goes up, never down.

Actual Results:  
Memory going up when loading the grid and when scrolling. Application unusable
because the browser freezes while the memory is growing. This can take plenty
of seconds. Because I stripped down the prototype, this freeze is not as
noticeable as in the application we are working on.

Expected Results:  
Memory not going up as fast. When I tried the exact same scenario in SL, memory
also tends to go up, but on a very much slower rate. Occasionally it drops,
which I believe happens when the GC kicks in. It doesn't necessarily always
drop to previous values, but never to a degree where it becomes a problem.
