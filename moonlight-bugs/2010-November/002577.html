Hey,

This is a known issue with our VisualStateManager implementation. The cause of
this is due to us incorrectly detecting when we should generate an automatic
transition and when we should not.

Basically the problem is that we don't detect that a storyboard animating
"Width" and a storyboard animating "(FrameworkElement.Width)" are the same. You
can work around the issue for now by ensuring that you're consistent with how
you reference DP's in storyboards.

I am looking into a proper fix for it now, but it's not as simple as I had
originally hoped as the code which generates dynamic transitions exists only in
C# and the code which parses the storyboard property paths exists only in C
code. Hopefully I'll have something before the end of next week.
