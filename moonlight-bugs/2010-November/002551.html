Description of Problem:

I have a SL4 app developed on Windows that uses WCF data services (i.e. oData)
and out of curiosity, I tried it in ML3 Preview 9 and to my surprise, nearly
everything worked.  The only thing that didn't work was the data service calls.
 My code does a typical setup for getting data from the server, namely:

                DataServiceCollection<FttpONT> onts = new
DataServiceCollection<FttpONT>(); BusyIndicator.IsBusy = true;
                onts.LoadCompleted += new
EventHandler<LoadCompletedEventArgs>(FttpONTs_LoadCompleted);
                onts.LoadAsync(from o in mContext.FttpONTs select o);

and then handling that in:

        void FttpONTs_LoadCompleted(object sender, LoadCompletedEventArgs e)
        {
            var oc = sender as DataServiceCollection<FttpONT>;

            if (e.Error == null)
            {
                if (oc.Continuation != null)
                {
                    oc.LoadNextPartialSetAsync();
                }
                else
                {
                    foreach (var o in oc)
                    {
                        o.FttpONTDPs.Add(dummyDP);
                        o.FttpONTVPs.Add(dummyVP);
                    }

                    BusyIndicator.IsBusy = false;

                    ONTsGrid.ItemsSource = oc;

                    oc.LoadCompleted -= new
EventHandler<LoadCompletedEventArgs>(FttpONTs_LoadCompleted);
                }
            }
        }

>From looking at the traffic in the browser log, it appears to call the service
and get the first chunk of results correctly but then never go back and get the
other chunks which leads me to believe something about the continuation stuff
or LoadNextPartialSetAsync() isn't working correctly.  Unfortunately, I don't
have time at the moment to build the source and run it in the Debugger but I
thought I'd let you know.
