In the attachment I am sending I have a listbox that uses an ItemTemplate. One
of the controls in the template is an image, whose source binds to one of the
string properties of the business object list to be bound to the list. In the
code-behind, you can see how I supply a single item to the list-box and that
the path to the image I am giving is a relative one.

Here lies the problem. ML should be looking at the path where the control is as
the reference for the relative path, but instead is looking at the root of the
app. If you run this application in ML, you will not see the image, whereas in
SL, you will. As a control to the experiment, I also added an Image directly to
the usercontrol and set the source referring exactly to the same image, using a
relative path once again. It shows in both ML and SL.

Reproducible: Always

Steps to Reproduce:
1. Run the TestPage.html
2.
3.
Actual Results:  
You will see a listbox with no image and a text saying "A Node" and an image
beneath the listbox.

Expected Results:  
You should see a listbox with the image and a text saying "A Node" and a copy
of the exact same image beneath the listbox.
