A simple test case to reproduce the issue

User-Agent:       Mozilla/5.0 (Windows NT 6.1; WOW64) AppleWebKit/534.24
(KHTML, like Gecko) Chrome/11.0.696.65 Safari/534.24

I upgraded to the latest release of the Reactive Extensions for Silverlight 4
(version 1.0.10425.0), and Moonlight now fails to load my application due to an
error:

[0xb76f46e0:] EXCEPTION handling: System.Security.VerificationException: Error
verifying
System.Reactive.AnonymousObservable`1/<>c__DisplayClass1:<Subscribe>b__0 ():
Incompatible parameter with function signature: Calling method with signature
(System.IObserver`1<T>) but for argument 0 there is a
(System.Reactive.AnonymousObservable`1/AutoDetachObserver<T> (Complex)) on
stack at 0x0017

Reproducible: Always

Steps to Reproduce:
1. Using Rx (Reactive Extensions), subscribe to an event:
Observable.FromEventPattern(source, event).Subscribe(...)
2. Cause the event to occur
Actual Results:  
The System.Security.VerificationException pasted above gets thrown

Expected Results:  
The subscribed callback should be called without an error, as is the case in
Silverlight
