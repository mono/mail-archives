To follow up on my question, I just discovered Glade FAQ 4.4. It turns
out that GtkExtra as a gtkextra-config script. I followed the example
given in 4.4, substituting gtkextra for xml, but still no luck.

My configure.in file now looks like the following, but gtkextra header
and library files still cannot be found. Any suggestions? Thanks

===== My configure.in file ====================

dnl Process this file with autoconf to produce a configure script.

AC_INIT(configure.in)
AM_INIT_AUTOMAKE(ultcol, 0.1)
AM_MAINTAINER_MODE
AM_CONFIG_HEADER(config.h)

AC_ISC_POSIX
AC_PROG_CC
AM_PROG_CC_STDC
AC_HEADER_STDC

AC_PATH_PROG(gtkextra_config,gtkextra-config)
if test "g$gtkextra_config" = "g"; then
   AC_MSG_ERROR([*** gtkextra-config not found. ])
fi

GTKEXTRA_CFLAGS=`$gtkextra_config --cflags 2>/dev/null`
GTKEXTRA_LIBS=`$gtkextra_config --libs 2>/dev/null`
CPPFLAGS="$CPPFLAGS $GTKEXTRA_CFLAGS"
LIBS="$LIBS $GTKEXTRA_LIBS"

pkg_modules="libgnomeui-2.0"
PKG_CHECK_MODULES(PACKAGE, [$pkg_modules])

AC_SUBST(PACKAGE_CFLAGS)
AC_SUBST(PACKAGE_LIBS)

AC_OUTPUT([
Makefile
src/Makefile
])
