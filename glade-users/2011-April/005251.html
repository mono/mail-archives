> Is there the intention to give M$Windows executable for this version??
> 3.8 and 3.10?

Actually we need someone to make binaries for win32, it would
also be nice if someone could take charge of making osx binaries,
however I do have a macbook at my disposition and might find
time to wrap some osx binaries (I'll wait for some minor point
releases to go by first in case there are bugs).

And there have been developments on making binaries:
   https://bugzilla.gnome.org/show_bug.cgi?id=634978

I have not yet added this script to Glade but I plan
on creating a build/ directory with win32/osx subdirs
with any scripts and information needed to build win32
or osx binaries.

Anyone interested in making win32 binaries for Glade
is welcome of course and should refer to the scripts
in the bug mentioned above (until I add those scripts
to Glade, I'm not sure the scripts will work properly
for 3.10 since it uses a different python).

Cheers,

>>> What is Glade?
>>> ============
>>> Glade is a RAD tool to enable quick&  easy development of user
>>> interfaces for the GTK+ toolkit and the GNOME desktop environment.
>>> The user interfaces designed in Glade are saved as XML, and by using
>>> the GtkBuilder GTK+ object these can be loaded by applications
>>> dynamically as needed.
>>>
>>> By using GtkBuilder, Glade XML files can be used in numerous
>>> programming languages including C, C++, C#, Vala, Java, Perl,
>>> Python, and others.
>>>
