I've seen the same happen on Windows when the hicolor-icon-theme
package is missing. The only thing that package provides is an empty
share/icons/hicolor/16x16, 22x22, etc directory structure and a
share/icons/hicolor/index.theme. It is that last file that is important.

You could probably even take the index.theme file from the package
below to test with as it's nothing more than an ini style text file:
http://ftp.gnome.org/pub/GNOME/binaries/win32/dependencies/hicolor-icon-theme_0.10-1_win32.zip
