Never my question bellow. I figured out what's going on and it seems to be a&nbsp;bug in Glade.<BR>I am using Glade-2 which I believe is not supported anymore, but just in case the same happening in Glade-3.<BR>When I create&nbsp;a GtkComboBoxEntry in Glade-2, 'items' translatable property&nbsp;is&nbsp;checked&nbsp;automatically and when I save the project, I would expect .glade file to have the following line:&nbsp; <BR>
&lt;property name="items" transaltable="yes"&gt;&lt;/property&gt;<BR>
However,&nbsp;<U>if you haven't touched the&nbsp;'items' property </U>in Glade-2, the above line would not get generated and the widget would not be treated as GtkComboBoxEntry (it would complain that it can't add an item to the comboBox entry and other errors). The way around it&nbsp;is to&nbsp;uncheck, then check&nbsp;again the&nbsp;items translatable property in Glade. That way, the above line would&nbsp;be added&nbsp;to .glade file.<BR>
Again, this is in Glade-2. I am not using Glade-3, so don't know if the problem propagated there as well. If somebody&nbsp;could check&nbsp;how it&nbsp;works in Glade-3 and file a bug report if there is a problem,&nbsp;it would be a favor to other people&nbsp;who may encounter this in the future. I spent hours trying to figure out what's going on, when I was trying to add items to the comboBox inside my python code.<BR>
