</div></div>Yes, you can define classes in python and use them in Glade and GtkBuilder.<br>
<br>
For PyGTK &amp; GTK+2 you must use Glade 3.6 or 3.8, if you migrate to<br>
PyObject &amp; GTK+3<br>
then you must use Glade 3.10.<br>
<br>
There is an example of how to setup Glade so that your custom python widgets<br>
show up in Glade&#39;s palette, in Glade&#39;s API reference here:<br>
    <a href="http://library.gnome.org/devel/gladeui/3.6/pythonsupport.html" target="_blank">http://library.gnome.org/devel/gladeui/3.6/pythonsupport.html</a><br>
<br>
Basically you just have to define a short xml catalog and have it in<br>
GLADE_CATALOG_PATH<br>
when firing up Glade, and have a copy of the python source in the same<br>
directory.<br>
