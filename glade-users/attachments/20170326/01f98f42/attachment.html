Has anybody else had Glade 3.18.3 use one core of the CPU and slowly<br>
fill up RAM until the computer freezes? This only happens when Glade is<br>
displaying a Signals window. Say the Signals tab under Widget Properties<br>
or when editing a cellrenderer for a treeview.<br>
<br>
As soon as Glade is shut down the RAM usage goes normal again. At this<br>
point I refrain from having Signals windows open for any extended amount<br>
of time. I see this behavior on three different computers so it is not<br>
the computer.<br>
