I've modified your glade dialog so that the "Cancel" button resides in
a dialog_action_area. You could then run the dialog using:<br>
<pre wrap=""><tt>void on_button_pressed () 
{</tt></pre>
<tt>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; GladeXML *gxml = glade_xml_new ("mydir/dialog.glade",
"my_dialog", NULL);<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; GtkWidget *dialog = glade_xml_get_widget(gxml, "my_dialog");
<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; GtkResponseType result;
<br>
<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; result = gtk_dialog_run (GTK_DIALOG(dialog));<br>
<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; if (result == GTK_RESPONSE_CANCEL)
<br>
&nbsp; &nbsp; &nbsp; &nbsp; {
<br>
&nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; /* This will always be the result because "cancel" is
the only button */
<br>
&nbsp; &nbsp; &nbsp; &nbsp; }
<br>
&nbsp; &nbsp; &nbsp; &nbsp; <br>
&nbsp; &nbsp; &nbsp; &nbsp; /* destroy widget */
<br>
&nbsp; &nbsp; &nbsp; &nbsp; gtk_widget_destroy (dialog);<br>
}</tt><br>
<br>
That's just one way. You could do it the way you're trying to an have
code to handle the cancel button's "clicked" signal, but this might be
an easier solution.<br>
