> One more issue seems to have come up which I've seen discussed before

> toolbox! They all have little missing icon x's in their place.

> Is this just something I'm going to have to live with or is it fixable?

You're probably missing the hicolor-icon-theme package.

What version of glade are you trying to install? Are you constructing
it's environment manually? If you're only interested in getting Glade
working, then you might be interested in the bundle I provide here:

http://www.optionexplicit.be/projects/gnome-windows/20110405/glade3-bundle_3.8.0-1-20110405_win32.zip

Simply extract the zip file, go into the bin directory and run
glade-3.exe, as discussed in this bug report:
https://bugzilla.gnome.org/show_bug.cgi?id=634978#c22

If you're manually constructing an environment for glade to run in,
you need the following packages (these are from the binary packages
on ftp.gnome.org, adapt as needed if you use binaries from some
other location):
atk_1.32.0-1_win32
cairo_1.10.2-1_win32
expat_2.0.1-1_win32
fontconfig_2.8.0-2_win32
freetype_2.4.4-1_win32
gdk-pixbuf_2.22.1-1_win32
gettext-runtime_0.18.1.1-2_win32
glib_2.28.1-1_win32
gtk+_2.24.0-1_win32
hicolor-icon-theme_0.10-1_win32
libpng_1.4.3-1_win32
libxml2_2.7.7-1_win32
pango_1.28.3-1_win32
pkg-config_0.25-1_win32
zlib_1.2.5-2_win32
