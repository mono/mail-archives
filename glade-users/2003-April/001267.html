I've managed to dupe this bug several times now.

Here's what I've done to get it to trigger every time:

Have glade2 open, with a project loaded.  Have the window with the 
menubar in question open.

open the preferences->themes dialog.

click on the details... button.

Then start changing between controls, pretty rapidly, until you seem to 
be lagging from changing them.  So basically the windows in the open 
workspace only have a few seconds to complete changing to the new style 
of controls before I click on another control theme.  It doesn't seem 
to matter which themes.

Go to Glade2, try and edit menus in a menu bar, and it hangs.

I'm on a stock RH 9.0 system, using glade2 1.1.3 which came with it.

Cheers,
