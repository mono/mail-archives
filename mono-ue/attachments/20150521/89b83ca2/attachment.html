<p class="MsoNormal">Has there been any thought to sell the C# bindings Xamarin has as a Unity plugin? Calling native APIs from C# is a huge paint point in Unity. It’s so bad we built
<a href="https://github.com/Hitcents/iOS4Unity">this</a> (blog <a href="http://jonathanpeppers.com/Blog/xamarin-ios-under-the-hood-calling-objective-c-from-csharp">

<p class="MsoPlainText">As you guys have noticed, the experience to setup Mono with Unreal Engine<o:p></o:p></p>
<p class="MsoPlainText">is not very pleasant.   It involves compiling code from source, it involves<o:p></o:p></p>
<p class="MsoPlainText">a specific version of Unreal and has a very long and tedious installation process.<o:p></o:p></p>
<p class="MsoPlainText"><o:p> </o:p></p>
<p class="MsoPlainText">The Unreal EULA is hostile to third party vendors and prevents us from creating the kind of integrated experience that we would like to have.<o:p></o:p></p>
<p class="MsoPlainText"><o:p> </o:p></p>
<p class="MsoPlainText">Based on the above, we do not anticipate that we will continue to develop the bindings of Mono for Unreal Engine at this point.<o:p></o:p></p>
