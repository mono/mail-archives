Bonjour,

Yesterday Xamarin.iOS 10.4 (cycle 9) was released to our stable channel: https://releases.xamarin.com/stable-release-cycle-9/

We also rebased the `xcode8.3` branch on this new, stable branch. This means all the enhancements and bug fixes from 10.4 are now included in the packages built from `xcode8.3`. Also, the stable version of XS should work out of the box (no need to have a custom version to deploy to devices).

You might notice a version “bump”, from 10.3.99 to 10.5.99, to avoid confusion with 10.4. Release notes are still available on the wiki, the new URL is https://github.com/xamarin/xamarin-macios/wiki/xamarin.ios_10.5.99

For the curious almost all bindings, including this week beta 3, are now available and we’re working on the tooling to support watchOS SiriKit extensions.
https://github.com/xamarin/xamarin-macios/wiki/xcode8.3-bindings-status

Please continue to file any issue you fin on Bugzilla. Thanks ☺
Sebastien

Bonjour,

Work is under way to support the new API exposed with Xcode 8.3 for iOS 10.3, tvOS 10.2 and watchOS 3.2.

This is all happening in our `xcode8.3` branch [1], the binding status is available from the wiki [2] and a draft of the release notes is also on the wiki [3].

The final date for Xcode 8.3 is unknown but the changes in beta 2 are few, so this might be a smaller/quicker release than most. To allow a “same day release” all our work is based on our current stable branch (`cycle8-xi`) so, other than the new APIs, everything should be identical to our existing XI 10.3.1 release. Note that we are close to release 10.4 (RC5 out today) so this work will likely be rebased on `cycle9` soon.

If/when you’re curious to try those new API you can download [4] our most recent(s) continuous builds that comes out of our bot farm. Please file bug reports [5] if you find any issue with those builds. Note that a newer/unreleased version of XS is currently needed to deploy from device. The link to download it is inside the release notes.

Thanks!
Sebastien
