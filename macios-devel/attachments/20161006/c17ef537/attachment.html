<p class="MsoNormal"><span style="font-size:11.0pt">Bonjour,<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">If you follow the activity from xamarin-macios git repository [1] you likely have noticed a new `xcode8.1` branch [2] was added recently. This is where all activity (e.g. tooling/bindings) to support Xcode
 8.1 and the updated SDK (iOS 10.1 and watchOS 3.1) is being done.<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">We do not know when Apple will release this new version of Xcode, so all work is based on our current `xcode8` release branch. This will allow us to ship the new SDK quickly (same day) after Apple’s announcement
 and make sure we include all the fixes that made it to stable (but without any unrelated code changes).<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">If you’d like to try our Xcode8.1 support you can either build it yourselves or, quicker for a peek, download one of packages produced by our bots. Note that those continuous builds are automated and do not
 go thru any formal QA. <o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">Download links to packages (including `xcode8.1) are found in our wiki [3]. This is also where you’ll see the status of the bindings, i.e. what’s done and what remains to be bound.<o:p></o:p></span></p>
