<p class="MsoNormal"><span style="font-size:11.0pt">Hi!<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">Recently we've realized that third-party libraries that do crash reporting (such as Insights or HockeyApp) are not notified when crashes occur (SIGSEGV).<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">These third-party libraries have typically adopted code similar to this:
<a href="http://stackoverflow.com/a/14499336/183422">http://stackoverflow.com/a/14499336/183422</a> to cope with the fact that the Mono runtime
<i>must</i> handle SIGSEGV and SIGBUS before anybody else (SIGSEGVs occur for NullReferenceExceptions, and the GC uses SIGBUS frequently for its own purposes).<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">The current sequence of events is this:<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">a) The app launches, and Xamarin.iOS initializes the Mono runtime. At that time the Mono runtime checks if there are any existing signal handlers, and store those handlers (so that they can be invoked if the
 Mono runtime determines a SIGSEGV did not occur in managed code).<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">b) Managed code starts running, and the actual app initializes any third-party libraries. At this point third-party libraries typically do something like this:<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">    * Fetches current signal handlers (which would be Mono's signal handlers) and stores them.<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">   * Initializes third-party crash-reporting libraries, which installs signal handlers.<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">    * Restores the stored signal handlers.<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">    Sample HockeyApp code: <a href="https://github.com/bitstadium/HockeySDK-Xamarin/blob/master/source/HockeySDK.iOSBindings/Additions.cs#L30-L52">
https://github.com/bitstadium/HockeySDK-Xamarin/blob/master/source/HockeySDK.iOSBindings/Additions.cs - L30-L52</a><o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">c) Something goes wrong (in native code), and the app receives a SIGSEGV.<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">    * Mono's signal handler is invoked and determines it's not a NullReferenceException.<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">    * Mono calls the signal handler it stored during initialization. As you can see this is
<i>not</i> the third-party library's signal handler, it's the OS' default signal handler, which causes the process to exit (without notifying the third-party crash-reporting library) and the OS generates a crash report (which the app developer will not see,
 because he believes the third-party crash-reporting library reports those).<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">So how do we fix this?<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">A couple of ideas:<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">1) Add an API to select/set the signal handler Mono chains to:<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal" style="margin-left:36.0pt"><span style="font-size:11.0pt;font-family:"Courier New"">namespace ObjCRuntime {<o:p></o:p></span></p>
<p class="MsoNormal" style="margin-left:36.0pt"><span style="font-size:11.0pt;font-family:"Courier New"">      public class Runtime {<o:p></o:p></span></p>
<p class="MsoNormal" style="margin-left:36.0pt"><span style="font-size:11.0pt;font-family:"Courier New"">            public struct SigAction<o:p></o:p></span></p>
<p class="MsoNormal" style="margin-left:36.0pt"><span style="font-size:11.0pt;font-family:"Courier New"">            {<o:p></o:p></span></p>
<p class="MsoNormal" style="margin-left:36.0pt"><span style="font-size:11.0pt;font-family:"Courier New"">                  public IntPtr Handler; //<o:p></o:p></span></p>
<p class="MsoNormal" style="margin-left:36.0pt"><span style="font-size:11.0pt;font-family:"Courier New"">                  public uint Mask; // sa_mask => sigset_t => __darwin_sigset_t => uint32_t<o:p></o:p></span></p>
<p class="MsoNormal" style="margin-left:36.0pt"><span style="font-size:11.0pt;font-family:"Courier New"">                  public int Flags; // sa_flags => int<o:p></o:p></span></p>
<p class="MsoNormal" style="margin-left:36.0pt"><span style="font-size:11.0pt;font-family:"Courier New"">            }<o:p></o:p></span></p>
<p class="MsoNormal" style="margin-left:36.0pt"><span style="font-size:11.0pt;font-family:"Courier New""><o:p> </o:p></span></p>
<p class="MsoNormal" style="margin-left:36.0pt"><span style="font-size:11.0pt;font-family:"Courier New"">            public static bool InstallSignalHandler (int signal, SigAction handler, out SigAction previous_handler);<o:p></o:p></span></p>
<p class="MsoNormal" style="margin-left:36.0pt"><span style="font-size:11.0pt;font-family:"Courier New"">      }<o:p></o:p></span></p>
<p class="MsoNormal" style="margin-left:36.0pt"><span style="font-size:11.0pt;font-family:"Courier New"">}<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">    The idea being to do something like this (modified code from HockeyApp's link from above):<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      // Store Mono SIGSEGV and SIGBUS handlers<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      sigaction(Signal.SIGBUS, IntPtr.Zero, sigbus);<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      sigaction(Signal.SIGSEGV, IntPtr.Zero, sigsegv);<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New""><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      // Enable crash reporting libraries<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      DoStartManager();<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New""><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      +     Runtime.SigAction handler, previous_handler;<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      +     IntPtr hockey_signal = Marshal.AllocHGlobal (512);<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      +     foreach (var signal in new Signal [] { Signal.SIGBUS, Signal.SIGSEGV }) {<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      +           sigaction (signal, IntPtr.Zero, hockey_signal);<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      +           handler = (Runtime.SigAction) Marshal.PtrToStructure (hockey_signal, typeof (Runtime.SigAction));<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      +           bool rv = Runtime.InstallSignalHandler ((int) signal, handler, out previous_handler);<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      +     }<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      +     Marshal.FreeHGlobal (hockey_signal);<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New""><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      // Restore Mono SIGSEGV and SIGBUS handlers<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      sigaction(Signal.SIGBUS, sigbus, IntPtr.Zero);<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      sigaction(Signal.SIGSEGV, sigsegv, IntPtr.Zero);<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">    IMHO this is ugly, and also it can't be implemented in current third-party bindings without losing support for older versions of Xamarin.iOS.<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">    OTOH it does not put any restrictions on the signal handler to chain to (more flexibility).<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">2) Add an attribute that gives the name of the signal handler, and then Xamarin.iOS generates the required code to make sure these signal handlers are called for signals Mono doesn't handle:<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      [assembly: SignalHandler (Signal.SIGSEGV, "third_party_signal_handler", EnableAtStartup: false)]<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      [assembly: SignalHandler (Signal.SIGBUS, "third_party_signal_handler", EnableAtStartup: false)]<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">    The problem here is that the signal handler might be called before the third-party library is initialized (Xamarin.iOS would enable this signal handler
<i>before</i> mono is initialized, which means that the app has not had time to initialize the third-party library).<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">    This could be solved adding a simple function to enable the attribute's signal handler:<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">                </span><span style="font-size:11.0pt;font-family:"Courier New"">ObjCRuntime.Runtime.EnableCustomSignalHandlers ();
<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">    The big advantage here is that it can catch crashes during app initialization (including Mono's initialization).<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">3) A mix of both of the above:<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      ObjCRuntime.Runtime.EnableCustomSignalHandler (Signals.SIGSEGV, "third_party_signal_handler");
<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">   The problem here is that we'd have to use dlsym to lookup the function pointer for third_party_signal_handler, which can be problematic sometimes (#1 just gets the function pointer from the signal handler
 the native third-party library installed, and for #2 generate native code that references the function instead).<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">4) A mix of all of the above:<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      // A function that fetches the current signal handler and makes mono chain to this signal handler<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      ObjCRuntime.Runtime.EnableCurrentSignalHandler (Signals.SIGSEGV);<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">    Which would be used like this:<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      // Store Mono SIGSEGV and SIGBUS handlers<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      sigaction(Signal.SIGBUS, IntPtr.Zero, sigbus);<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      sigaction(Signal.SIGSEGV, IntPtr.Zero, sigsegv);<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New""><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      // Enable crash reporting libraries<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      DoStartManager();<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New""><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      +     ObjCRuntime.Runtime.EnableCurrentSignalHandler (Signals.SIGSEGV);<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      +     ObjCRuntime.Runtime.EnableCurrentSignalHandler (Signals.SIGBUS);<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New""><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      // Restore Mono SIGSEGV and SIGBUS handlers<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      sigaction(Signal.SIGBUS, sigbus, IntPtr.Zero);<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:"Courier New"">      sigaction(Signal.SIGSEGV, sigsegv, IntPtr.Zero);</span><span style="font-size:11.0pt"><o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt">Any other ideas? Suggestions for better naming for the API?<o:p></o:p></span></p>
