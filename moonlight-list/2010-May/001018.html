Hi Gianluca 

I am currently working on similar applications. So far I have found desklets
quite comprehensive and useable. Rather than using mopen to display .xap
files I have made modifications to the Moonlight GTK host to allow the host
application to load Xaml assemblies directly from disk which is more like
the way WPf does things. I posted a patch for this on a previous topic, you
will need to compile moonlight with this patch manually though as it wasn't
included in any release.

It is possible to use the same xaml files between WPF and Moonlight
providing you stick to Silverlight 2 (soon SL 3) as your baseline (i.e dont
use Event Triggers etc use VisualStates instead from the WPFToolkit). 

I think desklets are a good way forward for rich UI apps, but it's still a
bit clunky to build :)
