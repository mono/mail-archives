Hey folks,

   I discussed with Chris an approach that is a little bit different
from what we had said in the past regarding the System.IO primitives in
Silverlight.    The issue is that the System.IO primitives allow for
calling a number of those routines and they "root" the path into
references to files from the XAP file.

    Since there is a well known and finite list of those methods that
take these paths and map them into XAP-relative files, instead of adding
a collection of hacks to System.IO as it exists today, I think that we
should just copy the System.IO.* files into System.IO.Moon*.cs files and
modify those copies accordingly.

    Maintenance-wise this is not as bad as it sounds, this namespace
does not really see a lot of bug fixing it has an interface and an
implementation that has been very solid for years and in the Silverlight
implementation we would actually be able to remove a lot of cruft that
is not necessary.

Miguel.
