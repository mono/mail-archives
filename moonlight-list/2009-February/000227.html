On Sat, 2009-02-07 at 14:23 +0900, Atsushi Eno wrote:
> Hello,
> 
> Sorry if it has already been discussed, but is there any reason we do not
> install moon 2.1 assemblies in the GAC, unlike core assemblies (such as
> mscorlib and System) ? It usually does not matter, but it does when we
> open Moonlight project in MonoDevelop (System.Windows and
> System.Windows.Browser are missing).

Having a separate location is useful to split platform and user code.
This is how MS does it (in part, the signature is also checked) and we
need to do the same distinction (between the platform and user code).

Not that the current choice cannot be changed, but using the GAC is
probably more "end user" error prone than other locations.
