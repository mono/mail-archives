bunch of things around to better support the deployment/application
model in SL2 and those changes havn't been reflected on the managed
bindigns yet.  I unfortunately don't have an ETA on when we're going to

wanted to undertake it.  If you look at deployment.cpp and plugin.cpp
you can see where most of the deployment spin-up happens in the plugin,
some of that will simply need to be cut out for the managed side.  I'm
sure there are a few other issues with the managed bindings, and there's
bound to be more as we clean up the Surface/Deployment code more, so
unfortunately it will likely remain a moving target for the next while.
