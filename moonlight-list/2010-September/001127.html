Hi All,
My application uses a MultiScaleImage collection in a way that works on 
Silverlight but fails on Moonlight.  Moonlight doesn't seem to follow the same 
communication pattern I've seen in silverlight:
0) Make an http request for "myimages" (to the server uri I've set as the Source 
for the MultiScaleImage) 
1) Receive XML for the myimages collection, formatted according to the Deep Zoom 
Schema* 
2) Start making http requests for the appropriate tile images defined by that 
xml, e.g. myimage_files/5/0_0.jpg  etc.

When I load the same app in moonlight, it never gets to step 2.  The browser 
window shows "Transferring data from MY_SERVER", but the server never sees any 
follow-up requests (but step 0 and step 1 appear fine).  One corollary is that 
it never calls the code-behind referenced by the ImageOpenSucceeded property 
(because, of course, it did not succeed).

So I'd guess that somehow it's choking when trying to parse the collection xml 
it received.  But the same xml is fine on Silverlight -- any ideas where the 
difference would lie?

Thanks,
BSP
