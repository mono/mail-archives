We understand that building svn trunk is not straight-forward, but
that's to be expected because it is undergoing a lot of changes and
until we get it into the final state, it's not really worth spending
time updating the build documentation because it's likely to change again.

For now, if you really want to build from svn trunk, follow these loose
steps:

* checkout mono, mcs and moon from svn trunk

* build mono with `./autogen.sh --enable-moonlight=yes && make && make
install` and DO NOT `make clean` afterward (the moon build depends on
object files in the mono and mcs directories)

* build moon with `./autogen.sh --enable-managed=yes && make && make
install && make test-plugin`
