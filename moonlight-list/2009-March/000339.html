Well, if you install mono from SVN into a 'standard' location like /usr,
then you won't need to any environment variables. Unfortunately this will
break any existing mono installation as you'll be overwriting your packaged
version. This is why you were linked to
http://www.mono-project.com/Parallel_Mono_Environments which gives
instructions on how to set up Mono from svn properly (as is required in the
instructions you quoted). This tells you how to set up all the environment
variables in a very simple manner.

When the alpha is released, it will probably be a little easier to compile,
which is why we were encouraging you to wait. If you want to run moonlight
right now, then you need to follow the instructions in
http://www.mono-project.com/Parallel_Mono_Environments (as linked
previously) to set up your svn mono environment without breaking your
package install and then follow the instructions you quoted to build
moonlight. If you follow those parallel environment instructions, you can
ignore every mention of environment variables in this thread. The only thing
you have to remember to do is to enter the parallel environment in a term
and launch firefox from that environment. Then the plugins will be correctly
picked up.
