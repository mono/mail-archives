Hello everyone,

Now that all the needed* coreclr patches are inside Mono all that's missing
to use CoreCLR inside Moonlight is:

1. adding the platform code checks (see attached patch);
2. update the bots to, at least, r130389 (mono/mcs);
3. export MOON_SECURITY (until it become the default);

Doing so makes it possible to run all moon's unit tests (including the
security tests currently ignored) and most working SL2 apps. If something
does not work it's likely one of two things:

a) Verification issues (exceptions or crashes) and should be reported to
Rodrigo; or
b) CoreCLR issues (e.g. [Field|Method]AccessException) which should be
reported to me.

Anyone has a problem with the patch in (1) ? the most likely problem is that
it does change the destination of the assemblies (no more GAC is being used)
so it could break some scripts (or some Makefile targets). Right now I'm
pretty curious about how (once enabled) the bots will react for, non-unit,
tests.

Thanks
Sebastien

* the last mono patch I sent on m-d-l is not required to make it work (only
extra checks)
