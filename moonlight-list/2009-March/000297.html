Sporadically over the past two weeks, I've been working with David
Bolter from Mozilla's Accessibility team, and the way we've agreed on to
allow Moonlight to inject itself into the Firefox a11y hierarchy is via
a new enum value on NPN_GetValue: NPNVNativeAccessible.

When the page changes, Firefox walks the DOM, generating AtkObjects for
every node, and when it hits a plugin, it usually inserts a dummy
placeholder.  With my WIP patch (bmo #480317), it calls NPN_GetValue to
see if the plugin has a native accessible, and will insert that instead.
All we have to do is handle this new value in our NPN_GetValue
implementation, so there is no additional API needed, allowing us to
(hopefully) remain compatible with other NPAPI implementations.
