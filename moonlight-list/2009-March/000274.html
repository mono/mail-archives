We recently made moon have a hard dependency on mcs, to ease the
process of injecting security attributes needed for CoreCLR, the
Moonlight security model. So moon needs to know where the mcs module
is, to get assemblies in it.

Those assemblies are built during the net_1_1 profile of Mono, which
is always built.

We usually check out the modules mcs, mono, and moon (from svn
/source/trunk) in the same folder, and moon by default will look into
../mcs. If you prefer to have a different layout, you can configure
moon using the --with-mcspath switch to specify where the mcs module
is.
