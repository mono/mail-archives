On Wed, Nov 5, 2008 at 11:49 AM, Chris Toshok <toshok at gmail.com> wrote:

> In the interest of not destabilizing the build as we move toward beta1 (and
> 1.0 in general), we're going to institute code reviews for all patches
> (regardless of size.)
>
> So the steps to get a patch in are:
>
> 1) make sure moonbuilder is green.  If moonbuilder is red (or not running
> due to the box hanging, or something), nothing goes in until the breakage is
> fixed.  No more using "the tests were failing before my commit" excuse.
>
> 2) run the test suite (yes, for any patch you want to commit - everyone,
> including me, has been guilty of skipping this step :)
>
> 3) find someone who knows the code (there's easily 2 or 3 people who are
> familiar with any of the various subsystems) and have them review it.  can
> be done via email, monoport, irc, whatever, just have another set of eyes
> look at the code.
>
> 4) include the reviewer in the ChangeLog entry (something like "r:toshok"
> at the end of the entry).
>
> sound good?
>
> Chris
