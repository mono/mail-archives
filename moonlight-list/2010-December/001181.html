Hi there,

I have a problem with the Telerik RadComboBox in ML that has to do with the
following. Let's assume this code:

private void ContentControl_KeyDown(object sender, KeyEventArgs e)
{
   int key = e.*PlatformKeyCode*;
   char ch = Convert.ToChar(key);
   System.Windows.MessageBox.Show(string.Format("You pressed: {0}",
ch.ToString()));
}

According to MS, the PlatformKeyCode is non-portable. (This value is the
nonportable key code, which is operating system–specific -
http://msdn.microsoft.com/en-us/library/system.windows.input.keyeventargs.platformkeycode(VS.96).aspx
).

However, in Silverlight, the returned value is coherent with the
Convert.ToChar(key) function. This means that if I press the 'B' key, it's
PlatformKeyCode value, when passed to Convert.ToChar(key), will return a
'B', for instance.

So, how does this relate to the Telerik's RadComboBox? Let's say you have a
list of countries. Telerik's combobox tries to be "clever" and if you press
'p' it will select 'Poland'. If you press 'o', then 'r', it will select
'Portugal'. Well, they are actually grabbing the PlatformKeyCode and doing
Convert.ToChar(...) of its value. In SL, this works, because there is a
value coherence.

If I run firefox with MOONLIGHT_OVERRIDES="emulatekeycodes=yes", then
everything will work just fine. My suggestion is that it would really be
nice if this parameter were set by default on the default compilation
configuration.

Thanks!

Rui
