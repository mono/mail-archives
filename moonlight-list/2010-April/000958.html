>From revision 154982, the mono build for moonlight has changed. Moon
is now linking with a convenience library called libmono-moon which is
built when the --with-moonlight option is passed to mono (trunk
r154976). With this, it's no longer necessary to have a separate mono
checkout just for moon, and it's not necessary to pass special CFLAGS
to mono.

Because of these changes, you'll need to rerun autogen.sh on moon if
you're building mono automatically from moon, or run autogen.sh on
mono if you're building it separately. You can use your regular mono
checkout with moon, as long as you configure it with --with-moonlight.
Note: you can also use a mono checkout located anywhere if you pass
--with-mcs-path to moon (pointing to mcs, with mono alongside it).

If you're building mono from moon, check the README (or autogen.sh) if
you want to know what flags are getting passed to mono. If you want to
drive the mono build separately, you can use --with-manual-mono when
running autogen.sh on moon. If you do this, don't forget to build mono
everytime the revision requirements are updated (check the README for
the required svn version)
