Hi,

 

I just committed support for running moon-unit in parts: only automation
tests or everything else.

 

In test/2.0/moon-unit you can now run "make run-automation" to run
automation tests, or "make run-moonlight" to run all the other tests. Just
"make run" continues to execute everything. The split is also done on the
bots, there's a new "make-automation-tests" step that runs the automation
tests, and the already existing "make-tests-2.0" step won't run automation
tests anymore.

 

The filtering is done on namespace: automation tests are those in
"MoonTest.System.Windows.Automation" and
"MoonTest.System.Windows.Automation.Peers".
