I think that the TextLayout changes need not be too complicated in order
to make something usable. I think we could do the following:

1. Extend TextLayoutAttributes to have a UIElement property (and maybe a
flag property to specify whether it is a UIElement or set of text
attributes)

2. Refactor TextLayoutRun, splitting the methods into a new base class
TextLayoutInline (or whatever) that TextLayoutRun would subclass as
would a new class for rendering UIElements (TextLayoutElement?).

3. InlineUIContainers could be represented by 0-length TextLayoutAttributes

4. Modify TextLayout's methods to handle these new types of attributes



Further thoughts:

* Add an internal RenderSize property to Block so that Block containers
can smartly figure out which of their blocks to render when provided
with a region other than "everything".

* Add an internal Render method to Block.

* Add a TextLayout to Paragraph which would represent the Inlines it
contains.
