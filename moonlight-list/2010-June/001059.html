Just to clarify chris' point:

Moonlight 3 will expose the same API as Silverlight 4. This means that
Silverlight 4 applications will be able to load up in moonlight and
won't have any missing class or missing method errors. However the
code in our implementation might be a simple "throw new
NotImplementedException ()" which obviously enough will prevent the
application from doing anything useful.

Sometimes it's easy to implement SL4 features when working on SL3
features,  in those cases we'll implement them. For example most of
the SL4 databinding additions are already supported. However SL3
features are our priority so we'll implement them first.
