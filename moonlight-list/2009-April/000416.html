What is happening is that 3 different configure scripts are running. One
for Pixman, one for Cairo, and one for Moonlight itself.

> If I then run make and make test-plugin, I need to copy libmoonplugin.so
> to (opt/mono/lib/moonlight/plugin/ after creating the plugin-directory.
>   

it sounds like you may have forgotten to run "make install" which
installs the moonlight libraries.

"make test-plugin" only installs the plugin, but it doesn't install the
libraries needed by the plugin.

> Starting Firefox and opening a Silverlight 2.0 page gives various errors,
> starting with:

> /opt/mono/lib/moonlight/plugin/libmoonplugin-ff3bridge.so: cannot open
> shared object file: No such file or directory

"make install" should fix this.
