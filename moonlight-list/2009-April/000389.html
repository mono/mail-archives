Sometimes calling SetValue() on a string property with a null argument
sets String.Empty and sometimes it sets null.

I tested this same behavior in javascript but it's impossible to tell
what is actually being set because GetValue() will always get ""
(Moonlight already implements this behavior to mimic Silverlight).

Here's a breakdown of which string properties have which behavior:

SetValue (${property}, null) results in String.Empty
====================================================

AssemblyPart.SourceProperty
FrameworkElement.NameProperty
Glyphs.IndicesProperty
Glyphs.UnicodeStringProperty
TextBlock.TextProperty
TextBox.TextProperty
VideoBrush.SourceNameProperty


SetValue (${property}, null) results in null
====================================================

HyperlinkButton.TargetNameProperty
ItemsControl.DisplayMemberPathProperty
RadioButton.GroupNameProperty
TimelineMarker.TextProperty
TimelineMarker.TypeProperty
