Tomas Klacko wrote:
> Hi,
>
> I have a few questions regarding Moonlight and hardware acceleration.
>
> (cite: http://www.mono-project.com/MoonlightNotes)
>
> Currently, Moonlight uses ffmpeg codecs in order to decode audio and video.
> - Does this also imply that Moonlight uses hardware acceleration
> for the audio and video tasks, when such hardware is provided?
> (My current, shallow, understanding of the ffmpeg software is that
> it is capable of making use of some hardware acceleration)
>   

Moonlight only uses ffmpeg's decoders, so if the decoders are hardware
accelerated, then yes. If not, then no.

> In the future, Moonlight will be using Microsoft-provided codecs.
>   

Moonlight already uses Microsoft's codecs, I guess that page needs to be
updated.

> When that happens:
> - Will these (presumably closed-source) codecs be able to make
> use of any hardware acceleration?
>   

They use MMX/SSE* instructions but do not currently take advantage of
the GPU (which is what I assume you are asking). We'd like to add
support for that though. Geoff Norton would know more about this.

> - Will the current source code that utilizes ffmpeg be also maintained?
>   

yes, it is being maintained.

> - Will I be allowed to modify and release Moonlight source code
> together with codecs capable of utilizing hardware acceleration?
>   

Of course! Moonlight is Free Software! :-)

Jeff
