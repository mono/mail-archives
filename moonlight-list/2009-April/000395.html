wrt enums there is:

* no need to set the first one to 0, that's the default behavior

* no (real) need to unit test them, since this is covered/spotted by
gui-compare (no harm to add them since you already have the tests)


source guidelines:

* mono convention is to place the bracket on the same line as the
namespace and class (but not methods) so

        namespace MoonTest.System.Windows.Automation 
        {
        	[TestClass]
        	public class RowOrColumnMajorTest
        	{
        
should be

        namespace MoonTest.System.Windows.Automation {
        
        	[TestClass]
        	public class RowOrColumnMajorTest {
        
(not a very big deal)


Otherwise, and assuming moon-unit is green with the tests, then I see
any problem with committing the code.
