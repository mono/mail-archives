Yea, this is what I got from Miguel as well.

Next up is what the encoding of the filenames are in Zip files. I
noticed that there's no guarantee that they are UTF-8 filenames, so
Miguel and I poked around with that a bit.

What we found out:

- Linux's zip utilities simply use the byte array from the file system
(typically UTF-8 on most Linux systems these days)
- Visual Studio doesn't allow anything outside the ASCII range (woot!)

Because of the VS limitation, I think we can simply g_ascii_tolower()
the filenames and not worry about unicode characters in paths.
