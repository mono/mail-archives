On Thu, 2009-04-09 at 13:30 -0400, Jeffrey Stedfast wrote:
> Sebastien Pouliot wrote:
> > On Wed, 2009-04-08 at 17:23 -0400, Jeffrey Stedfast wrote:
> >   
> >> I'm probably going to be coming back to this very soon now, so I've been
> >> wondering:
> >>
> >> How would I handle lowercasing of unicode filenames?
> >>     
> >
> > IIRC it's better to use uppercase with unicode since more than one
> > (uppercase) character can be mapped to the same (lowercase) character -
> > leading to different file names representing the same "normalized" file.
> >
> > As for the other points let's talk about them on IRC next week. There's
> > a bunch of stuff that needs to keep being operational (non-rooted) like
> > some Silverlight features[1], the current tooling (like smcs) and
> > probably quite a few "internals" as well.
> >
> > [1] http://www.mono-project.com/Moonlight/SecurityStatus#Local_I.2FO
> >   
> I'm wondering if maybe a better place to put this would be inside mono's
> io-layer instead? Do the rooted checks and filename canonicalization
> there instead of in the managed code?

That's an option but I don't see how this would help to keep existing,
unrooted cases working. 

Anyway I think this needs to start with a list of API that needs to be
rooted/non-rooted*, then see where the extra check can be put without
breaking the required use cases (and without open an extra door).

* that could be easier to track with a local patch to the io-layer (but
won't be complete without looking at all the runtime code, icalls in
particular)
