Using the GtkSilver control to playback a video file (the first minute
of elephants dream), I am noticing that closing the window that
contains the control does not cause the audio to stop playing back.  I
added in some logging to the MediaPlayer::Close method to see if it is
called, and it doesn't seem to be occuring.  Is there some sort of
teardown that should be happening here?  When I use the same
xaml/video file in Firefox using Moonlight 1.0, I don't have this
problem.

The version of GtkSilver is built from svn, with other modifications,
but I don't think they are causing the problem.

The program I'm using to test GtkSilver is very simple (notice that I
don't attach to the close event for the window to call
Application.Quit() ):
	string inputFile = args[0];
			
	//init gtk
	Application.Init ();
			
	//init moonlight
	GtkSilver.Init ();
				
	//build our window
	Window window = new Window ("test");
	GtkSilver silver = new GtkSilver (640, 480);
	window.Add (silver);
	window.ShowAll ();
			
	//load our input content
	if(inputFile.EndsWith(".xap")){
		System.Windows.Application x;
		silver.LoadXap(inputFile,out x);			
		}
	else{
		silver.LoadFile(inputFile);
		}
			
	Application.Run();

Anybody have any ideas?
