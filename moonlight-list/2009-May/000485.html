If you type <Button>someText</Button> to vs 2008 IDE you got the error hint
"Button" don't support "sometext as content". Unfortunately you can compile
the App in VS2008 but it chrash in IE8.
As i know this syntax is valid in WPF but not in Silverlight.

http://msdn.microsoft.com/en-us/library/cc917841(VS.95).aspx

With the exceptions of TextBlock and Run, object elements in Silverlight may
not contain XML text nodes as an implicit way to set their text-type content
properties. For instance, <Button> hello world </Button> is not allowed in
Silverlight XAML. Again, note that there are some type converter syntaxes
(such as for Color) that might resemble object element syntax but are
technically a verbose type converter syntax.

I like your implementation more because it is compatible with WPF but
unfortunately you have to be compatible with Silverlight.
