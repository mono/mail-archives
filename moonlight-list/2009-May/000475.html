> still run on sl 3? I don't use anything not currently present in
> moonlight.) I actually managed to compile the assembly successfully,
> but Visual Studio 2008 won't let me add it as a reference to my
> silverlight project (attempt was made to load an invalid program) ,

The C# compiler might have produced an invalid binary, try running
peverify.exe on it (you'll need to copy all the dlls in
%PROGRAMFILES%/Microsoft Silverlight/v2.0.../ to the directory where your
binary is first, otherwise you'll get a bunch of errors)
