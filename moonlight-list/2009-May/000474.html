Hi Moonlight Folks,

I'm attempting to compile an assembly that targets Silverlight 3 using
Mono. (I know moonlight isn't there yet, but a sl 2 assembly should
still run on sl 3? I don't use anything not currently present in
moonlight.) I actually managed to compile the assembly successfully,
but Visual Studio 2008 won't let me add it as a reference to my
silverlight project (attempt was made to load an invalid program) ,
and my Silverlight 3 application fails when trying to load it
dynamically (AssemblyPart.Load.)

I'm hoping this is possible, maybe I'm just doing things wrong. It
does seem to be one of the main goals of moonlight.

Here's what I did on my Ubuntu Jaunty box:

1) remove mono that came with ubuntu
2) install mono 2.4 from source
3) download moonlight sdk obtained from here:
http://mjhutchinson.com/journal/2009/05/08/moonlight_development_linux_monodevelop
4) build an assembly from two .cs files and a .resources file, using
smcs, and referencing mscorlib, System, and System.Windows from
the sdk in step 3.

For the curious (open source developers always seem to be curious) the
reason I don't just build this assembly on windows is I want to
generate it automatically on an Ubuntu server from information
supplied by users.

Thanks,
-Dan
