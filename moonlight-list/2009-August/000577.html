Hi,

As the tuning step of the moon build process may cut API that our team
is using for A11y support, I'm proposing this patch here to prevent that
to happen (we will be updating it while the development continues,
thanks to a script in our tree that will warn us if the descriptors file
changes).

The patch is big so I have compressed it. However, if you want to take a
quick look, the relevant change is this one:

Index: class/tuning/Makefile.am
===================================================================
--- class/tuning/Makefile.am	(revision 139224)
+++ class/tuning/Makefile.am	(working copy)
@@ -9,7 +9,8 @@
 	$(srcdir)/Descriptors/System.xml	\
 	$(srcdir)/Descriptors/System.Net.xml	\
 	$(srcdir)/Descriptors/System.Xml.xml	\
-	$(srcdir)/Descriptors/System.ServiceModel.xml
+	$(srcdir)/Descriptors/System.ServiceModel.xml	\
+	$(srcdir)/Descriptors/a11y.xml

 SECURITY_ATTRIBUTES_INFO = \
 	SecurityAttributes/mscorlib.secattr \


The rest of the patch is the descriptors.xml file, which starts like:

Index: class/tuning/Descriptors/a11y.xml
===================================================================
--- class/tuning/Descriptors/a11y.xml	(revision 0)
+++ class/tuning/Descriptors/a11y.xml	(revision 0)
@@ -0,0 +1,23782 @@
+<linker>
+  <assembly fullname="System">
+    <type fullname="System.Collections.Generic.Queue`1">
+      <method signature="System.Void
System.Collections.ICollection.CopyTo(System.ArraySystem.Array,System.Int32)"
/>
+      <method signature="System.Boolean
System.Collections.ICollection.get_IsSynchronized()" />
+      <method signature="System.Object
System.Collections.ICollection.get_SyncRoot()" />


...


So, ok to commit?
