So here's what I've been running with on my pal branch.

The IMContext and Clipboard apis are pretty much from gtk.  we'll need 
to see what other platforms support/require.

I have gtk working with it enough to run samples in mopen, all events 
working, etc.  The few remaining gtk specific pieces are some keysym 
usage (we need to have our own keysym list and a mapping from platform 
keyvals to those), and the painttodrawable code, which should be 
relatively easy to remove.

Another issue to resolve is how we're going to deal with the 
message-pump type stuff, but that should be relatively easy.  We just 
pick the windows way of doing it and make the other platforms deal with 
it :)

Oh, also window.h still contains quite a bit of cruft, as i basically 
copied the window-gtk.h file over and massaged it.  it'll be a little 
cleaner in the future.

Chris
