I have been working for the last few days on getting Second Life
running with the CoreCLR enabled in Mono. We are using Mono 2.6.1 for
our experiments, but are some time away from being able to upgrade in
production.

Initially I attempted to enable the CoreCLR while initializing the
Mono JIT with framework version "1.0", and then later with version
"2.0" when that didn't work. In both cases the initialization attempts
to import System.Security.SecuritySafeCriticalAttribute, which fails
and hits an assert at line 45 of security-core-clr.c in
security_safe_critical_attribute().

After more experiments I was only able to get the Mono JIT to
initialize with the magic version string "moonlight", which select
framework version "2.1". We have concluded from this that with the
current Mono codebase we can only use framework version "2.1" if we
want the CoreCLR working. Some response I got on the #moonlight irc
channel yesterday from kangaroo and lupos seem to back this up to some
extent. Many thanks for the replies there, sorry I didn't have a
chance to respond much.

Can anyone confirm whether this is as expected, and also let me know
if there is any possibility of using the CoreCLR with other framework
versions in future releases of mono?

Al
