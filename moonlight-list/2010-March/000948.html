> What do you think is it good to merge those two into want?

> Lunar Eclipse should be easier but I'm not sure.

One idea that we should probably pursue is to have Lunar Eclipse run
entirely with Silverlight controls to bring Lunar Eclipse not only to
MonoDevelop users, but also as a XAML editor that Mac users can use.

Integrating Lunar Eclipse into MonoDevelop sounds like a good idea, but
you will be spending a lot of cycles into the integration itself, when
there are more fundamental limitations on it.

My feeling is that we would be better off improving the LunarEclipse
codebase to be a better editor, and also to spend cycles into making it
run either inside the browser or with Silverlight's own Out-of-browser
support so we can gain more contributors (by reaching out to our MacOS
users as well).
