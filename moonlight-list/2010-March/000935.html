Hi,

 

We have now moved to using mono/mcs/mono-basic from trunk (r152911+) instead
of the 2.6 branch.

 

You also now need mono/mcs/mono-basic checked out next to moon (the
configure option -with-mcspath is removed, these directories need to stay
next to moon now) - and moon's autogen.sh will automatically autogen mono
too - you should not run any configure/make commands in mono or mcs (in
particular not make install, moonlight configures mono to its own needs so
you'll most likely end up with a corrupted/unusable mono install).
