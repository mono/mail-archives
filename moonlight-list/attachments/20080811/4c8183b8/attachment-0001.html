The main idea behind this tool is to make it easy for us to spot<br>
performance regressions and have a general understanding of how the code<br>
changes affect performance -- we currently have some performance-related<br>
bugs in the Bugzilla which have &quot;I think it used to be fine in the<br>
past...&quot; kind of comments. Not good.<br>
<br>
The general idea of benchmarking the perf is to measure the time it<br>
takes to render a given sample &quot;as fast as possible&quot; using the manual<br>
time source.<br>
<br>
In this example report above there are only 3 tests, but obviously we<br>
need much more of them.<br>
<br>
Some info about the tool:<br>
<br>
* Written in C# (plus small C tool to run the tests)<br>
* Uses sqlite simple relational database for storage<br>
* Separate runner/report generator tools<br>
* Uses drtlist (similiar to the test harness) for input and test<br>
configuration<br>
<br>

revisions and spawing the benchmarking process.<br>
