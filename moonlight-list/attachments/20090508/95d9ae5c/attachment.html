Hey,<br><br> So I was checking out this issue in a little more detail today and I have a good idea what&#39;s supposed to be happening (I think ;) ).<br><br>public static DependencyProperty Whatever = DependencyProperty.Register (&quot;Name&quot;, typeof (IComparable)..... );<br>
<br>If i assign the double value &#39;5.0&#39; to this property and apply a storyboard with &quot;new DoubleAnimation { By = 10 }&quot; to it, I end up with the value of 15. If i assign another class/struct which is IComparable but not a double, we end up with a value of &#39;10&#39; for the same animation (test in svn for this).<br>
<br>So what this tells me is that we shouldn&#39;t box value types when pushing them to native. Native needs to recognise a double as a double, no matter what. The current way of type checking by using &#39;kind&#39; won&#39;t work for this case. What native needs to use instead is the clr method Type.IsAssignableFrom or some equivalent. This would then allow us to dump the current &#39;box_value_type&#39; code which hacks around another facet of this issue.<br>
