<h2>Hi All,</h2>

<div>
I'm 5th year Computer Science student from Gdansk, Poland. I want to introduce myself and describe my ideas about Silverlight/Web-based video editing software (form http://mono-project.com/StudentProjects).
I would like to participate Google Summer of Code this year and create some usable video montage tool. About 3 years ago I have focused on .NET platform. I'm interesting
in video montage and movie making. I have worked with montage in Adobe Premiere, Adobe Audition and Sony Vegas. Also I create sometimes animations
for my movies in Adobe After Effects.
</div>

<h2>ABSTRACT</h2>
<div>
I will develop simple and intuitive web-based video montage environment. During this project I would like to implement the most
important features typically required by users who want to montage movies using Internet. Beside I would like to include in this
project my idea of distributing media resources in distracted montage environment (such as Internet). In short my idea is:
<ol>
<li>Use automatic compression mechanism to make preview files (considerable smaller than original ones) for all media files on server (for example builds by FFMpeg libraries).</li>
<li>Add some metadata to media files (about characters, actors, requisites, weather in record, time of day in record) in XML files.</li>
<li>Distribute media resources in two versions of file: compressed (preview) one (for searching media resources by prepared metadata) and original (source) file (for rendering).</li>
<li>Streaming media from server
</ol>
</div>

<div>
This is schema of this idea. User could streaming compressed media from server or download it from server or another user. 
</div>

Of course it isn't the basic functionality of project, but I hope it could optimize sharing media resources between users. I'm aware it's problem to manage files
on local machine from Silverlight web application. I have worked out some apps in .NET connected with media and will do some research on this case before staring project.

<h2>FUNCTIONAL SUMMARY</h2>

<div>
After think over this case I decided to expose some parts of video editor. I would like to realize this parts of application as
separated controls. Below I describe basic use cases for my project idea. The User is a person who want to montage video by Internet using
my Silverlight UI.
</div>

<br/>
<div>
<img alt="use_cases" src="http://dl.dropbox.com/u/681495/USE_CASES.jpg" width="90%", height="90%" align="center"/>
</div>
<br/>

<h4>Legend:</h4>
<ul>

<h3>Project treeview</h3>
<li><b>load media files into project</b> - thirls media resource(s) to project and put it into project treeview</li>
<li><b>manipulate project items</b> - move media item in treeview, rename and delete items</li>

<br/>
<h3>Built-in player</h3>
<li><b>play selected media</b> - play media resource selected on project treeview</li>
<li><b>play project</b> - play media which are montage in timeline editor</li>
<li><b>pause playing</b> - stops playing media</li>
<li><b>stop playing</b> - interrupt playing media and go to it's begin</li>

<br/>
<h3>Trimmer</h3> 
<li><b>select region</b> - choose part of source clip</li>
<li><b>preview play media</b> - show in player media file or it's selected region form trimmer</li>
<li><b>put region into timeline editor</b> - drag&drop selected region from trimmer and put it into particular channel</li>

<br/>
<h3>Timeline editor</h3>
<li><b>manipulate media items</b> - move clips forward and backward in timeline, cut, delete, split and marge them</li>
<li><b>manipulate channels</b> - changing channels sequence order, set name and description to channel</li>
<li><b>manipulate channel properties</b> - set volume (for audio) and opacity (for video)</li>
<li><b>put media into timeline editor</b> - drag&drop media item from project treeview to timeline editor</li>
<li><b>set marker</b> - put marker in timeline and name it</li>

<br/>
<h3>Project manager</h3>
<li><b>create new project</b> - create empty project with particular properties</li>
<li><b>open existing project</b> - load project from file on hard drive or download it form server</li>
<li><b>set project properties</b> - define default setting of project - such as video format, author, workspace for media resources</li>
<li><b>save project template</b> - save project template into file or upload it to server</li>
<li><b>save project</b> - save project into file or upload it to server</li>

<br/>
<h3>Media manager</h3>
<li><b>show media details</b> - show parameters of media file as bitrate, total time, type of container, type of codec</li>
<li><b>download source media</b> - downloading original media from server to local workspace</li>
<li><b>download preview media</b> - downloading compressed copy of original file form server to local workspace</li>
<li><b>upload media</b> - uploading compressed version of local media to server</li>
<li><b>set autocompression properties</b> - automatic compressor settings</li>
<li><b>add tag to media</b> - match tag(s) with single media</li>
<li><b>search by tags</b> - return list of media files on server which are described required tags</li>

<br/>
<h3>Rendering tool</h3>
<li><b>set rendering properties</b> - define settings for rendering the project into single file</li>
<li><b>save rendering template</b> - save rendering settings into file or upload it to server</li>
<li><b>render project</b> - converts whole project form timeline editor into single file</li>
</ul>

<div>
Here is the moment I want show a preview of user interface I would like to create.
</div>

<br/>
<div>

<h3>Project manager</h3>
<div>
Part of application to manage multiple projects and project templates.
</div>
<ul>
<li>wizard for create new projects
<ul>
<li>specify video template (e.g. PAL, NTSC, HDV), fps, video frame size and pixel aspect ratio for all video channels</li>
<li>define sample rate and default volume for all audio channels</li>
</ul>
</li>
<li>describe project: title, author, tags</li>
<li>save project setting as template for future projects</li>
<li>save/load project (on hard drive or on server, to XML file)</li>
</ul>

<h3>Project treeview</h3>
<div>
The control which visualize project resources and deliver some manage them.
</div>
<ul>
<li>show project media resources</li>
<li>mark already used resources (which had been on timeline editor)</li>
</ul>

<h3>Media manager</h3>
<div>
Module providing exchange media between Silverlight video editor and media server.
</div>
<ul>
<li>define workspace for project media resources</li>
<li>add some metadata describing files as tags (save in XML files)</li>
<li>show details about media file as</li>
<li>replace on the fly used preview file (compressed) by its original version (for example download it from server)</li>
</ul>

<h3>Built-in player</h3> 
<div>
Simple media player control.
</div>
<ul>
<li>play from start of file</li>
<li>play from selected point</li>
<li>play from nearest marker</li>
<li>stop</li>
<li>pause</li>
<li>time scroll</li>
<li>change volume</li>
</ul>

<h3>Trimmer</h3>
<div>
Tool for preview and treatment single media resource before putting into montage.
</div>
<ul>
<li>load media file</li>
<li>play media</li>
<li>select region and place it in timeline editor</li>
<li>choose start point and play media form it</li>
<li>shows total time of media and current time</li>
</ul>

<h3>Timeline editor</h3>
<div>
Main element of application, contains channels (alternatively dedicated for viedo or audio media). On channels we can arrange our media and build art work:).
</div>
<ul>
<li>multiple channels (few types: video, audio, subtitles (optional))</li>
<li>add, delete channel</li>
<li>name channels</li>
<li>master settings for each channel (volume, opacity)</li>
<li>frame representation of video clip</li>
<li>select items, move, split, delete</li>
<li>select multiple items</li>
<li>set markers</li>
<li>set fade in or fade out to items</li>
</ul>

<h3>Rendering tool</h3>
<div>
Module for compression project into single media file.
</div>
<ul>
<li>choose container format (avi, mpeg2, mp4, wav, ogg)</li>
<li>choose compression method (uncompressed, dv, XviD, mp3)</li>
<li>save rendering settings as template</li>
<li>specify rendered file output directory</li>
</ul>
