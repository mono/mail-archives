<div>* ActiveX runs your code with the same user rights as everything else you are running.   Code downloaded from ActiveX has full control of your system, the only measure of protection you have is &quot;Well, at least it was signed by FooBar, and I trust FooBar&quot;.   ActiveX sole security system is the dialog box that says &quot;About to run code Foo written by Bar, do you want to allow it?&quot;.</div>
<div><br></div><div>* NativeClient provides a sandbox.   So you can connect to *any* website that embeds some native client component, and you know that it does not matter how nasty the code tries to be, at best, they will flash some ugly HTML to you, but they wont get access to your system, wont get access to your files, wont be opening any sockets, and basically can not be used as a vehicle for trojans or viruses.</div>
<div><br></div><div>* ActiveX depends on the Win32 API, so it limited to work only on Windows hosts.</div><div><br></div><div>* NativeClient provides a POSIX-ish API that runs on all supported systems.</div><div><br></div>
