In the interest of not destabilizing the build as we move toward beta1 (and 1.0 in general), we&#39;re going to institute code reviews for all patches (regardless of size.)<br><br>So the steps to get a patch in are:<br><br>
1) make sure moonbuilder is green.&nbsp; If moonbuilder is red (or not running due to the box hanging, or something), nothing goes in until the breakage is fixed.&nbsp; No more using &quot;the tests were failing before my commit&quot; excuse.<br>
<br>2) run the test suite (yes, for any patch you want to commit - everyone, including me, has been guilty of skipping this step :)<br><br>3) find someone who knows the code (there&#39;s easily 2 or 3 people who are familiar with any of the various subsystems) and have them review it.&nbsp; can be done via email, monoport, irc, whatever, just have another set of eyes look at the code.<br>
<br>4) include the reviewer in the ChangeLog entry (something like &quot;r:toshok&quot; at the end of the entry).<br><br>sound good?<br><br>Chris<br>
