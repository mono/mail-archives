<br>I&#39;ve been thinking about the ReadLocalValue implementation we have now.&nbsp; Right now in FrameworkElement::SetValueImplWithError, we check to see if the value we&#39;re setting is BindingExpression/Style, and use that value as the current_value for the Property, just so we get ReadLocalValue semantics.&nbsp; I really don&#39;t like the overloading of the current_value hashtable to contain things other than actual values, since it complicates both GetValue and SetValue, and it seems to me that we don&#39;t need it -- ReadLocalValue is just an indication of where the current value comes from.&nbsp; So the current_value hash should could contain within it the actual Value* associated with the property, not the BindingExpression or Style object.&nbsp; We&#39;d have a separate hash keyed on DP -- or maybe even just reuse the DependencyObject::bindings hash and add one for the style as well -- to determine where the value comes from.&nbsp; For a &quot;local&quot; value, i.e. one that was just set as a property, that hash would have NULL (or in the alternate case it wouldn&#39;t exist in either bindings or style hashes.)&nbsp; For BindingExpressions or Style Setters, it would have the appropriate object.<br>
