<p class=MsoNormal><span lang=EN-US>I just added support in the harness for
running unit tests.<o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal><span lang=EN-US>&#8220;cd moon/test &amp;&amp; make
run-unit-tests-harness&#8221; will now run the unit tests with the harness and
print a summary when done. This will also create an xml file on disk with the
results (in the test-run-data directory).<o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US>&#8220;cd moon/test &amp;&amp; make run-unit-tests&#8221;
does the same thing as &#8220;cd moon/test/2.0/moon-unit &amp;&amp; make test&#8221;:
pops up firefox and runs the unit tests (&#8220;cd moon/test/2.0/moon-unit
&amp;&amp; make test&#8221; still work though).<o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal><span lang=EN-US>The unit test framework actually has
support for the concept of known failures, but they&#8217;ve named it a bit
different: test methods which are known failures are marked with [BugAttribute].
The harness will always execute these tests, and mark them as failed when they
succeed (i.e. revert the result). I enhanced this a bit by adding
SilverlightBugAttribute and MoonlightBugAttribute (they both inherit from
BugAttribute) &#8211; a test marked with [MoonlightBug] will be treated as a
normal test on Silverlight and a known failure on Moonlight. The old KnownFailureAttribute
does no longer exist. <o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US><o:p>&nbsp;</o:p></span></p>
