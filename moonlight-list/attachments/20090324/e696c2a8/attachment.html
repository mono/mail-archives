<p>Currently support for Silverlight 2.0 is in pre-Alpha stage, but if you want to test it, or contribute to the code, you will need to: </p>
<ul>
<li>Obtain Mono from <a title="SVN" href="http://www.mono-project.com/SVN">SVN</a> or <a title="AnonSVN" href="http://www.mono-project.com/AnonSVN">AnonSVN</a> (version 2.2 is not enough). </li>
<li>Run the configuration for Mono like this: 
<ul>
<li>./autogen.sh </li></ul></li>
<li>Install Mono 
<ul>
<li>make install </li></ul></li></ul>
<p>This will produce a new compiler <b>smcs</b> which defaults C# to the 3.0 version and will include the Silverlight assemblies by default in a compilation. You will use this compiler to generate assemblies and libraries that are intended to be ran on Microsoft&#39;s Silverlight or Mono&#39;s Moonlight. </p>

<p>Then build Moonlight (now it is with the managed code support by default): </p>
<ul>
<li>Obtain Moonlight source code from <a title="SVN" href="http://www.mono-project.com/SVN">SVN</a> or <a title="AnonSVN" href="http://www.mono-project.com/AnonSVN">AnonSVN</a> </li>
<li>Run the configuration for Moonlight like this: 
<ul>
<li>./autogen.sh </li>
<li>make install </li></ul></li></ul></div>
