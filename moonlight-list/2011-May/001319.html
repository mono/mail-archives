I've come late to this party I'm afraid. The problem of using the moonlight
engine in a desktop environment is one I've been playing with for about a
year or so. Inface my company uses a custom version of the moonlight engine
to do just that.

I've blogged the info here http://walking-in-moonlight.blogspot.com/ (if
might not all work, so any feedback would be great :) )

to get round the issues I modified the moonlighthost class to set the
xap_dir to the local directory and the host application builds the
appmanifest manually on startup , including any dll's in the application
directory. I know its not the best option but it does work. you can modify
the host app to search a plugins directory and include those dlls if your
app supported plugin screens for example. 

the blog includes a sample project which can be compiled either in VS2010
and produce a WPF app or via xbuild to get a moonlgiht one. The blog also
points to a ubuntu repository which contains the runtime and development
environments for the moonlight desktop stuff I've done.

the sample app does not load any external dll's yet. But it should be easy
to support.
