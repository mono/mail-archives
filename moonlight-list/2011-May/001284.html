Hi,
I am trying to evaluate the feasibility of implementing an out-of-browser
Silverlight 4 application, which would require the ability to save files to
a users documents directory. The application would need to function on
Moonlight.

I require out-of-browser support, as well as elevated trust (a Silverlight 4
feature). I see from the release notes for Moonlight 4 Preview 1 that the
support for Elevated Trust is 'Incomplete' and my own testing confirms this
(and is still the case with the current nightly build). When attempting to
use Environment.GetFolderPath(Environment.SpecialFolder.MyDocuments)
Moonlight throws an exception complaining that elevated privileges are
required.

I am also encountering an issue with the out-of-browser support which was
previously posted both on this forum/list and as a Stack Overflow question
(http://stackoverflow.com/questions/5119415/moonlight-4-out-of-browser). A
comment on the Stack Overflow post suggests that this should be fixed in the
nightly build, but I am still seeing the issue on both Preview 1 and the
current nightly.

Can anyone shed any light on the current status of the above issues and
provide any indication of the planned release schedule for Moonlight 4? I'm
keen to use it for this application but I need confidence that
out-of-browser and Elevated Trust will both work and be available in a
reasonable timescale.
