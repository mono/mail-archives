It's in svn, r148138 on mono-2-6, and I just dcommitted it to trunk as 
r148184.

The problem has nothing to do with serializing.  the problem is with 
*de*serializing.  The KeyValuePair members both have getters and 
therefore serialization is trivial.  deserializing is harder given that 
both Key and Value members have no public setters.  The special case in 
our code effectively adds internal setters and then uses those setters 
(but only in KVP's case).  MS special case code, however, uses either a 
dynamically generated or statically compiled method (specific to not 
only the generic type but to the actual concrete instantiated type) to 
pass the arguments to KVP's constructor, bypassing the allocation + set 
step used for just about every other other type.

toshok
