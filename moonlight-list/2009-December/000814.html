Moonlight will be 99.9% compatible with Silverlight (there is a very small
list of things that might behave differently, such as using PlatformKeyCode
[1], assembly obfuscation, or the newly announced com support in Silverlight
4.).

Note that Moonlight do lag a bit behind Silverlight, so if you decide to
target Silverlight 4 in your application, you'll have to wait a bit until
Moonlight catches up (see our roadmap here:
http://www.mono-project.com/MoonlightRoadmap). OTOH if you target
Silverlight 2, you'll be safe. You can use some Silverlight 3 features in
your application too  (here is some more information:
http://tirania.org/blog/archive/2009/Dec-17.html), and it'll probably work.

So my advice is pretty simple: create your Silverlight application using
Silverlight 2, and don't worry about Moonlight compatibility. The only thing
you might want to do is to occasionally test using Moonlight 2, and if you
find a problem, file a bug in our bug tracker (http://bugzilla.novell.com),
and we'll fix it (compatibility with Silverlight is really our problem, not
yours. It should just work without changes).

I hope this helps.
