>Subject: [Moonlight-list] DependencyProperty setter validation in C#
>
>Hey,
>
>There is at least one place in Silverlight where there is extra validation
performed in the wrapper property for a DependencyProperty.
>
>TextBox.Text = null; // Throws an ArgumentNullException
>TextBox.SetValue (TextProperty, null); // Works fine
>
>I was just thinking about how best to solve this and I think if the
generator emitted a partial method based on the current property, it'd be
trivial to add this extra validation where it's needed and have no penalties
where it is not needed. i.e.
>
>public partial class TextBox {
>    public string Text {
>        get { return GetValue (TextProperty); }
>        set {
>            ValidateText (value);
>            SetValue (TextProperty, value);
>        }
>        partial void ValidateText (string value);
>}
>
>If no body is defined for the partial method, it'll just vanish. 


IMHO this is a good idea, but I had the same idea to fix a similar issue
some time ago (a partial Initialize method called in the ctor) and I was
voted down :)
