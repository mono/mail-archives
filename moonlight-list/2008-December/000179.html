I just added support in the harness for running unit tests.

 

"cd moon/test && make run-unit-tests-harness" will now run the unit tests
with the harness and print a summary when done. This will also create an xml
file on disk with the results (in the test-run-data directory).

"cd moon/test && make run-unit-tests" does the same thing as "cd
moon/test/2.0/moon-unit && make test": pops up firefox and runs the unit
tests ("cd moon/test/2.0/moon-unit && make test" still work though).

 

The unit test framework actually has support for the concept of known
failures, but they've named it a bit different: test methods which are known
failures are marked with [BugAttribute]. The harness will always execute
these tests, and mark them as failed when they succeed (i.e. revert the
result). I enhanced this a bit by adding SilverlightBugAttribute and
MoonlightBugAttribute (they both inherit from BugAttribute) - a test marked
with [MoonlightBug] will be treated as a normal test on Silverlight and a
known failure on Moonlight. The old KnownFailureAttribute does no longer
exist.
