This was workaround:

 1. download monodevelop 3.0.3.5 (latest stable)
 2. open Application folder
 3. right click on MonoDevelop.app, show package content
 4. Contents -> MacOS -> and there you should see MonoDevelop script
 5. cmd + i (get file info), scroll to Name & Extension, and change form
    upper case letters to lower case,
 6. it shoud look like this monodevelop
 7. download latest monotouch, and start installation, if everything is
    ok you should
    see only monotouch from requirements for installation,
 8. if you also see monodevelop that means that installer cant find
    monodevelop that you installed.
    Recheck that everything is lower case letter, and also check
    permissions, try adding
    read/write for admin as well
 9. if you succesfully installed monotouch, dont press start
    monodevelop, but go to the step 3.
10. and restore original name (monodevelop -> MonoDevelop)

I'm opening a bug ASAP.
