Missing constructor crashes that show up only with sgen enabled doesn't mean
it's an sgen bug per-se, but only that with the default GC it won't be
triggered.

Sgen is usually much more aggressive in releasing memory that the default
GC.

If you have a sample that reliably crashes with sgen (no experimental new
refcount)
in other ways, please do fill a bug report/support request as those are
very hard to
come by and we plan on switching to sgen only in the future.
