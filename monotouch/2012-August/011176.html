Well, in my case that's not doing the job...

Patern was fairly clear to get the issue on which if I terminate the
application using the PLIST value not to run in background and then using
the screen button to exit application, if I let the Iphone goes into sleep
after that, 100% of the time it would show a blackscreen on comeback and
without timeout as usual if the FinishedLaunching is not done. Did move
everything out to be able to debug and it seems that it is coming from the
use of System.Thread.Timer when the first timer is enabled. Did switch all
my timer to NSTimer instead, but still no luck there.

But as soon as I removed the ARM6+ARM7 compilation method from the
equation, everything was working!

Im about to do my 100 or so retry and got no blackscreen so far.

As soon as Im putting back the ARM6+ARM7 then it comes back intantaniously.

So I guess that was it. Might have something related to thread numbers or
maybe it is something with the Garbadge collector of the MonoTouch DLL
which isnt working.

As I already said in prior mail, what is fairly bizarre is that when in
that blackscreen reality, Process is not showed within Instrument. Just
like if it was not running.

I decided to just forget about IOS 4.0 and device which don't have the ARM7
processor for now. Will finished the application and then see if there's
any possibilities.

Thanks for your suggestions guys.

Cheers
