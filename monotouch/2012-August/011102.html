On Fri, Aug 3, 2012 at 3:29 PM, Nscavezze <nate.scavezze at gmail.com> wrote:
> I have given up on binding the ScanditSDk and I have go the ZBar bindings
> working perfectly in Debug builds.
>
> The issue I am having is that in the Release build I get a null reference
> exception when i try and get the following bound constant.
>
> extern NSString* const ZBarReaderControllerResults;
>
> The binding
>
> [Static]
> [Preserve (AllMembers = true)]
> interface Zbar {
>     [Field ("ZBarReaderControllerResults", "__Internal")]
>     NSString BarcodeResultsKey { get; }
> }
>
> I just added the Perserve part and that didn't help

[Preserve] will only help if you're not using the code (at compile
time) but still need it (at runtime) and it should not be differ in
Debug/Release builds (unless you have conditionals, like #if DEBUG,
inside your source).

> an ideas?

If you added any "Additional mtouch arguments" in your project for
your Debug configuration then you'll need to copy-paste them in your
Release configuration (as each one is unique).
