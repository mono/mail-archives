Mike,

Thank you for the suggestion.  It looks like it may have actually had something to do with stale breakpoints/breakpoints out of sync with the lines in the code or something related.  I feel like it may be that the breakpoint line numbers aren't getting synchronized when the code is changed outside of MonoDevelop.  We are actually doing most of our editing in Visual Studio - MonoDevelop opens the solution file on my Windows share.  We use the Macs primarily for UI design and deploying/debugging.  Clearing all of my breakpoints seemed to solve the problem - since I did that, I have not gotten any more of those errors.

If this is an accurate assessment of the problem, it would be a very helpful feature to have MonoDevelop track changes to the files (maybe using a modified date) that have breakpoints set in them and update the breakpoint line numbers if the file has changed since MonoDevelop last performed a save on it.
