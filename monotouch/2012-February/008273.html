Android and IOS does not use MVVM pattern like WP7 does. They use MVC
pattern instead, which does not support bindings in the same way. So
in your controller you will have to listen for changes to the the
Collections and update the Views manually. There is an attempt to get
MVVM bindings to Android in this project, but I don't know whether
there is a Mono for Android version.

More info here:
http://www.codeproject.com/Articles/145203/Android-Binding-Introduction
http://stackoverflow.com/questions/4972927/android-mvvm-design-pattern-examples
