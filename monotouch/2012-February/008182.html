On my ViewControllers I almost always put a navigation bar on interface
builder. I think that it appear "automatic" just when there is a father
Navigation Controller but I don't use it so much. I don't know if it's that
correct but I lost the fear to put some objects on interface builder to make
my own and avoid the always tap and swipe iphone navigation model on ipad,
but I'm crossing things on ipad and windows mobile.

Simulated things on interface builder are just to see how it will be if
something put or changes something by code, like view format (page, form,
fullscreen etc). So if you will do a ViewController that will be inside an
ipad "dialog" so you can draw it simulating this format to design it near
the real view.
