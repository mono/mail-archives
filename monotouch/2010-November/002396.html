FWIW, both of those approaches (no animations for hiding; timers)
worked, but I ended up doing something a little different --
overriding ViewDidDisappear() in the modal view controller to fire an
event, and using an event listener to show the next modal view. As far
as I can tell, ViewDidDisappear() isn't called till after the
animation finishes, so this works fairly well, though the bookkeeping
gets a bit hackish if you need several possible after-disappear
behaviors.
