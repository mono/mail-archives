On Wed, 2010-11-17 at 15:20 -0800, David Moles wrote:
> Does MonoTouch / the iPhone have a way to register a global exception handler?
> 
> Now that I'm at the point of testing on the device, it would be nice
> if I could get some information about crashes / uncaught exceptions
> without having to use the debugger, esp. since I've got a lot of
> assertions of the "halt and catch fire" variety.
> 

I don't know if this works on MonoTouch, but you could try this:

	static void Main ()
	{
		AppDomain.CurrentDomain.UnhandledException += OnUnhandledException;
		...
	}

	static void OnUnhandledException (object sender, UnhandledExceptionEventArgs args)
	{
		// args.ExceptionObject -> exception that triggered this event
		// args.IsTerminating -> the runtime is terminating :)
	}

This will also get unhandled exceptions thrown in threads other than the
main one.
