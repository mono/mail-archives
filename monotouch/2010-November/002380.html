Thanks, that got me going in the right direction.  This is what I ended it
up with.

            ABAddressBook ab = new ABAddressBook();
            ABPerson p = new ABPerson();
            p.FirstName = "Jason";
            p.LastName = "Jones";

            ABMutableMultiValue<string> phones = new
ABMutableStringMultiValue();
            phones.Add("9999999999", ABPersonPhoneLabel.Mobile);

            p.SetPhones(phones);

            ABMutableDictionaryMultiValue addresses = new
ABMutableDictionaryMultiValue();
            NSMutableDictionary a = new NSMutableDictionary();

            a.Add(new NSString(ABPersonAddressKey.City), new
NSString("Spring"));
            a.Add(new NSString(ABPersonAddressKey.State), new
NSString("TX"));
            a.Add(new NSString(ABPersonAddressKey.Zip), new
NSString("77379"));
            a.Add(new NSString(ABPersonAddressKey.Street), new NSString("100
Main St"));

            addresses.Add(a, new NSString("Home"));
            p.SetAddresses(addresses);

            ab.Add(p);
            ab.Save();

On Fri, Nov 12, 2010 at 11:07 AM, Jonathan Pryor <jonpryor at vt.edu> wrote:

> On Fri, 2010-11-12 at 10:07 -0600, Jason Awbrey wrote:
> > Now I need to add names and addresses, but the ABMultiValue object
> > required by SetPhones() and SetAddresses() is throwing me.
>
> The default ABMultiValue<T> value as returned by e.g.
> ABPerson.GetPhones() is immutable.  You need to instead use a modifiable
> version.  There are two ways to do this:
>
>  1. Via ABMultiValue<T>.ToMutableMultiValue() [0]
>  2. By explicitly creating a mutable subclass [1, 2, 3], and adding
>    values to the ABMutableMultiValue<T> [4].
>
> Once you have an ABMutableMultiValue<T>, you can add new values, then
> use e.g ABPerson.SetPhones(), passing the ABMutableMultiValue<T>
> instance.
>
