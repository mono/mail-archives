On Fri, 2010-11-12 at 10:07 -0600, Jason Awbrey wrote:
> Now I need to add names and addresses, but the ABMultiValue object 
> required by SetPhones() and SetAddresses() is throwing me.

The default ABMultiValue<T> value as returned by e.g.
ABPerson.GetPhones() is immutable.  You need to instead use a modifiable
version.  There are two ways to do this:

 1. Via ABMultiValue<T>.ToMutableMultiValue() [0]
 2. By explicitly creating a mutable subclass [1, 2, 3], and adding
    values to the ABMutableMultiValue<T> [4].

Once you have an ABMutableMultiValue<T>, you can add new values, then
use e.g ABPerson.SetPhones(), passing the ABMutableMultiValue<T>
instance.
