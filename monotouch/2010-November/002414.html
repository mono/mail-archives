Hi,

I'm trying to create btouch input for AdMob.

The .h file contains (a.o.) following constructor:

+ (AdMobView *)requestAdWithDelegate:(id<AdMobDelegate>)delegate;

The instructions at
http://monotouch.net/Documentation/Binding_New_Objective-C_Types
state:

For constructors that you define, they need to be declared using the
following signature inside the Interface definition: they must return
an IntPtr value and the name of the method should be Constructor.
For example to bind the initWithFrame: constructor, this is what you
would use:

   1. [Export ("initWithFrame:")]
   2. IntPtr Constructor (RectangleF frame);

So I have this in the my AdMob.cs file:

      [Export ("requestAdWithDelegate:")]
      IntPtr Constructor (AdMobDelegate Delegate);

But when I process this with btouch, it complains:

error CS0246: The type or namespace name `IntPtr' could not be found.
Are you missing a using directive or an assembly reference?

My AdMob.cs starts with 'using MonoTouch.Foundation;'

Do I need any other using statements? The instructions at the above
link do not mention any, but this looks like something is missing
here.

TIA, guivho.
