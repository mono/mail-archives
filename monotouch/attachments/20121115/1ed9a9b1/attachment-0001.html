<span style="font-family:"Arial";color:rgb(0,0,0);background-color:transparent;font-weight:700;font-style:normal;font-variant:normal;text-decoration:none;vertical-align:baseline" id="internal-source-marker_0.4930639675995814"><font size="4">Call for beta tests</font></span><br>
<span style="font-size:15px;font-family:Arial;color:#000000;background-color:transparent;font-weight:normal;font-style:normal;font-variant:normal;text-decoration:none;vertical-align:baseline"><br class="kix-line-break"><font>We are looking for developers to test their applications with a different set of semantics for Dispose() on NSObjects and provide us with feedback and let us know if this breaks their existing applications.</font></span><font><br>
</font><font><br></font><span style="font-family:"Arial";color:rgb(0,0,0);background-color:transparent;font-weight:400;font-style:normal;font-variant:normal;text-decoration:none;vertical-align:baseline"><font>The semantics for MonoTouch’s Dispose on NSObject and their subclasses as of MonoTouch 6.0.6</font></span><span style="font-family:"Arial";color:rgb(0,0,0);background-color:transparent;font-weight:400;font-style:normal;font-variant:normal;text-decoration:none;vertical-align:baseline"><font> (the current stable release) are as follows:</font></span><font><br>
</font><font><br></font><ul style="margin-top:0pt;margin-bottom:0pt"><li style="list-style-type:disc;font-family:"Arial";color:rgb(0,0,0);background-color:transparent;font-weight:400;font-style:normal;font-variant:normal;text-decoration:none;vertical-align:baseline">
<span style="font-family:"Arial";color:rgb(0,0,0);background-color:transparent;font-weight:400;font-style:normal;font-variant:normal;text-decoration:none;vertical-align:baseline"><font>Release the native reference</font></span></li>
<li style="list-style-type:disc;font-family:"Arial";color:rgb(0,0,0);background-color:transparent;font-weight:400;font-style:normal;font-variant:normal;text-decoration:none;vertical-align:baseline"><span style="font-family:"Arial";color:rgb(0,0,0);background-color:transparent;font-weight:400;font-style:normal;font-variant:normal;text-decoration:none;vertical-align:baseline"><font>Clear the handle of the object</font></span></li>
<li style="list-style-type:disc;font-family:"Arial";color:rgb(0,0,0);background-color:transparent;font-weight:400;font-style:normal;font-variant:normal;text-decoration:none;vertical-align:baseline"><span style="font-family:"Arial";color:rgb(0,0,0);background-color:transparent;font-weight:400;font-style:normal;font-variant:normal;text-decoration:none;vertical-align:baseline"><font>Clear any managed fields that cache or point to other managed objects</font></span></li>
<li style="list-style-type:disc;font-family:"Arial";color:rgb(0,0,0);background-color:transparent;font-weight:400;font-style:normal;font-variant:normal;text-decoration:none;vertical-align:baseline"><span style="font-family:"Arial";color:rgb(0,0,0);background-color:transparent;font-weight:400;font-style:normal;font-variant:normal;text-decoration:none;vertical-align:baseline"><font>Break the Unmanaged->Managed link</font></span></li>
</ul><font><br></font><span style="font-family:"Arial";color:rgb(0,0,0);background-color:transparent;font-weight:400;font-style:normal;font-variant:normal;text-decoration:none;vertical-align:baseline"><font>This means that when you call Dispose, you have effectively registered your lack of interest in the object. The above is problematic in some conditions, for instance, when the unmanaged world keeps a reference to the managed object that has been Disposed and a method is invoked on it. Typically users will see an error message like this:</font></span><font><br>
</font><font><br></font><p dir="ltr" style="margin-left:36pt;margin-top:0pt;margin-bottom:0pt"><span style="font-family:"Consolas";color:rgb(0,0,0);background-color:transparent;font-weight:400;font-style:normal;font-variant:normal;text-decoration:none;vertical-align:baseline"><font>Selector invoked from objective-c on a managed object of type MyApp.MyType (0x10D27C50) that has been GC'ed</font></span></p>
<font><br></font><div><span style="font-family:"Arial";color:rgb(0,0,0);background-color:transparent;font-weight:400;font-style:normal;font-variant:normal;text-decoration:none;vertical-align:baseline"><font>However the above is not always true. The managed object could still be alive, but the linkage between the two worlds has been broken and when MonoTouch tries to find the managed object for a particular native object it won't find it and throws this exception.</font></span><font><br>
</font></div><div><span style="font-family:"Arial";color:rgb(0,0,0);background-color:transparent;font-weight:400;font-style:normal;font-variant:normal;text-decoration:none;vertical-align:baseline"><font><br></font></span></div>
<span style="font-family:"Arial";color:rgb(0,0,0);background-color:transparent;font-weight:400;font-style:normal;font-variant:normal;text-decoration:none;vertical-align:baseline"><font>Sadly, calling Dispose() actively is a common pattern, so we are experimenting with some new semantics for Dispose that would eliminate the above problem for the conditions outlined before.</font></span><font><br>
</font><font><br></font><span style="font-family:"Arial";color:rgb(0,0,0);background-color:transparent;font-weight:400;font-style:normal;font-variant:normal;text-decoration:none;vertical-align:baseline"><font>We have implemented a new approach for Dispose in MonoTouch 6.0.7 (currently available in the Beta channel).</font></span><font><br>
</font><font><br></font><span style="font-family:"Arial";color:rgb(0,0,0);background-color:transparent;font-weight:400;font-style:normal;font-variant:normal;text-decoration:none;vertical-align:baseline"><font>In this beta, Dispose() instead performs these steps:</font></span><font><br>
</font><ul style="margin-top:0pt;margin-bottom:0pt"><li style="list-style-type:disc;font-family:"Arial";color:rgb(0,0,0);background-color:transparent;font-weight:400;font-style:normal;font-variant:normal;text-decoration:none;vertical-align:baseline">
<span style="font-family:"Arial";color:rgb(0,0,0);background-color:transparent;font-weight:400;font-style:normal;font-variant:normal;text-decoration:none;vertical-align:baseline"><font>Release the native reference</font></span></li>
<li style="list-style-type:disc;font-family:"Arial";color:rgb(0,0,0);background-color:transparent;font-weight:400;font-style:normal;font-variant:normal;text-decoration:none;vertical-align:baseline"><span style="font-family:"Arial";color:rgb(0,0,0);background-color:transparent;font-weight:400;font-style:normal;font-variant:normal;text-decoration:none;vertical-align:baseline"><font>Handle and backing fields are preserved unless this was the last reference to the object.</font></span></li>
</ul><font><br></font><span style="font-family:"Arial";color:rgb(0,0,0);background-color:transparent;font-weight:400;font-style:normal;font-variant:normal;text-decoration:none;vertical-align:baseline"><font>This means that a managed NSObject subclass that is being referenced by unmanaged code is kept intact and fully operational after a call to Dispose. The managed object will be kept alive for as long as the unmanaged code references the object.</font></span><font><br>
</font><font><br></font><span style="font-family:"Arial";color:rgb(0,0,0);background-color:transparent;font-weight:400;font-style:normal;font-variant:normal;text-decoration:none;vertical-align:baseline"><font>This has the potential downside that some objects that would have previously been eligible for collection will not be, until a later point in time.</font></span><font><br>
</font><font><br></font><span style="font-family:"Arial";color:rgb(0,0,0);background-color:transparent;font-weight:400;font-style:normal;font-variant:normal;text-decoration:none;vertical-align:baseline"><font>But the advantage is that the managed hierarchy more closely mimics the behavior of the unmanaged one.</font></span><div>
