<div dir="ltr">We have this setup running, combined with testflight for testing distribution.  Here's the commands we have for teamcity to build our iOS dev builds.  Change the <Your Solution> to whatever is appropriate for you.<div>
<br></div><div><div>/Applications/Xamarin\ Studio.app/Contents/MacOS/mdtool build "-t:Clean" "--configuration:AdHoc|iPhone" "%teamcity.build.checkoutDir%"/<Your Solution>.sln</div><div>
/Applications/Xamarin\ Studio.app/Contents/MacOS/mdtool build "--configuration:AdHoc|iPhone" "%teamcity.build.checkoutDir%"/<Your Solution>.sln</div></div><div><br></div><div>Also,we found we needed to run the teamcity agent as a dedicated user, and that user must be logged.  This was for keychain access to sign your assemblies.  We've got it running on a headless mac mini and just remote screen share after boot to log into the teamcity dedicated user and start the build agent.  </div>
