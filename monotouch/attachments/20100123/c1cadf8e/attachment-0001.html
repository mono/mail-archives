<div>Thanks to Geoff Norton and other contributors, the sqlite-net project has been updated with a variety of bug fixes. It is available from: <a href="http://code.google.com/p/sqlite-net/">http://code.google.com/p/sqlite-net/</a></div>
<div><br></div><div>For those unfamiliar, this was one of the early drivers for MonoTouch back before we had the <a href="http://ADO.NET">ADO.NET</a> driver. Its design goal is to make DB access simple.</div><div><br></div>
<div>I had let the project slide as I didn&#39;t think there was much point after the <a href="http://ADO.NET">ADO.NET</a> driver was released. On reflection however, it&#39;s not 2002 anymore and <a href="http://ADO.NET">ADO.NET</a> is feeling a bit crufty. So I have decided be a better maintainer of this project. To that end:</div>
<div><br></div><div><ul><li>A variety of outstanding bugs have been fixed (most importantly the memory corruption that resulted in AccessViolations and other exceptions)</li><li>A unit test suite is being incrementally built to test the engine</li>
<li>Features are being added to make working with the DB as painless as possible</li></ul></div><div>If you have any suggestions or comments, please post to this thread or to the project Issue system.</div><div><br></div>
