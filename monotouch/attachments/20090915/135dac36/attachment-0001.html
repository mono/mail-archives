href="mailto:demis.bellot@gmail.com">demis.bellot@gmail.com</a>&gt; wrote:<o:p></o:p></p>

<p class=MsoNormal>Don't use SOAP/WS-* protocols - they're an un-necessary
technology<br>
designed to give MS/IBM/HP/etc an excuse to charge huge consulting<br>
fees.<br>
It performs slower, is less interoperable and generally requires the<br>
overhead of a SOAP framework + generated code etc.<br>
<br>
I would recommend either REST+POX (Plain old XML) or REST+JSON for web
services.<br>
Anyone considering whether or not to use REST should read this<br>
article, it explains it quite well:<br>
<a href="http://tomayko.com/writings/rest-to-my-wife" target="_blank">http://tomayko.com/writings/rest-to-my-wife</a><br>
<br>
XML is the most interoperable language, and works best (and performs<br>
better) between 2 .NET endpoints (like MonoTouch =&gt; Mono Web Service).<br>
<br>
JSON is slower to deserialize, in .NET but is smaller in size and<br>
performs better inside a website, so I would recommend it over XML for<br>
Ajax sites.<br>
<br>
Essentially you just need to send DataContract's over the wire to a<br>
HttpHandler endpoint and use either DataContractSerializer (for XML)<br>
or JsonDataContractSerializer (for JSON) to deserialize the request<br>
<br>
I've done a lot of work in this area with my open source webservices<br>
stack which currently works in mono and takes most of the effort out<br>
of building web services and automatically provides REST+JSON and<br>
REST+XML endpoints:<br>
<a href="http://www.servicestack.net" target="_blank">http://www.servicestack.net</a><br>
source code is available here: <a href="http://code.google.com/p/servicestack"
target="_blank">http://code.google.com/p/servicestack</a><br>
<br>
I haven't yet tested it in MonoTouch yet as I haven't dual-booted into<br>
OSX for a few weeks so I think I've missed the BETA.<br>
Will probably have to wait till they release the evaluation version<br>
before I can verify that it works.<br>
<br>
- Demis<o:p></o:p></p>
