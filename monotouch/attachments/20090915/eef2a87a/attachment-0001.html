<br>
&gt;&gt;<span style='font-size:7.5pt'>Is there a better way to have the
keyboard go away then to trap<br>
the necessary event and then call .ResignFirstResponder()</span><br>
Seems to me from reading objc doco that this is the correct approach? I also
saw an approach where you create a big 'non-visible' button behind your other
controls which calls ResignFirstResponder() on *all* your inputs as a
&quot;short cut&quot; which does feel failure natural. <br>
<br>
Of course if you have multiple inputs, you might not want to 'resign' but 'move
focus'. In MapKit101<br>
<a href="http://conceptdevelopment.net/iPhone/MapKit01/Main.cs.htm">http://conceptdevelopment.net/iPhone/MapKit01/Main.cs.htm</a><br>
you'll notice that the Ended() event on Latitude sets the input to Longitude (<span
style='font-size:7.5pt;font-family:"Courier New"'>textfieldLongitude.BecomeFirstResponder();</span>).
This makes more sense to the user if you set the &quot;keyboard&quot; to show
&quot;Next&quot; instead of &quot;Done&quot; in IB.<br>
