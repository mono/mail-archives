&gt;&gt; It performs slower, is less interoperable and generally requires the<br>
&gt;&gt; overhead of a SOAP framework + generated code etc.<br>
&gt;&gt;<br>
&gt;&gt; I would recommend either REST+POX (Plain old XML) or REST+JSON for web<br>
&gt;&gt; services.<br>

&gt;&gt; article, it explains it quite well:<br>

&gt;&gt;<br>
&gt;&gt; XML is the most interoperable language, and works best (and performs<br>
&gt;&gt; better) between 2 .NET endpoints (like MonoTouch =&gt; Mono Web Service).<br>
&gt;&gt;<br>
&gt;&gt; JSON is slower to deserialize, in .NET but is smaller in size and<br>
&gt;&gt; performs better inside a website, so I would recommend it over XML for<br>
&gt;&gt; Ajax sites.<br>
&gt;&gt;<br>
&gt;&gt; Essentially you just need to send DataContract&#39;s over the wire to a<br>
&gt;&gt; HttpHandler endpoint and use either DataContractSerializer (for XML)<br>
&gt;&gt; or JsonDataContractSerializer (for JSON) to deserialize the request<br>
&gt;&gt;<br>
&gt;&gt; I&#39;ve done a lot of work in this area with my open source webservices<br>
&gt;&gt; stack which currently works in mono and takes most of the effort out<br>
&gt;&gt; of building web services and automatically provides REST+JSON and<br>
&gt;&gt; REST+XML endpoints:<br>
&gt;&gt; <a href="http://www.servicestack.net" target="_blank">http://www.servicestack.net</a><br>
