<br>>> Hi Nordis
<br>>>
<br>>> If I had to guess, I'd say this is the problem
<br>>>
<br>>> protected override void Dispose (bool disposing)
<br>>> {
<br>>> // Hopefully fixes SIGSEGV error
<br>>> //
<br>>> <a href="http://stackoverflow.com/questions/6708705/monotouch-sigsegv-crash-using-navigationcontroller-and-searchdisplaycontroller?rq=1" rel="nofollow" link="external" target="_blank">http://stackoverflow.com/questions/6708705/monotouch-sigsegv-crash-using-navigationcontroller-and-searchdisplaycontroller?rq=1</a><br>
>> BeginInvokeOnMainThread (() => {
<div class='shrinkable-quote'><br>>> searchController.Delegate = null;
<br>>> searchController.SearchResultsDelegate = null;
<br>>> searchController.SearchResultsSource = null;
<br>>> });
<br>>> base.Dispose (disposing);
<br>>> }
<br>>>
<br>>>
<br>>> BeginInvokeOnMainThread will call it and return immediately - so your
<br>>> searchController may have been disposed of by the time the background
<br>>> thread runs.
<br>>>
<br>>> try changing it to either not use the call at all, or use
<br>>> InvokeOnMainThread, which waits for the code to execute before
<br>>> continuing.
<br>>>
<br>>>
<br>>> "Unlike NSObject.BeginInvokeOnMainThread(NSAction) this method waits
<br>>> for the main thread to execute the method, and does not return until
<br>>> the code pointed by action has completed running."
