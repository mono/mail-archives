    <ul>
      <li>define smallest subset for platforms to be targeted (for me it
        is still WP, for You it'll be MTch)</li>
      <li>use source code sharing with linking (ProjectLinker tool)</li>
      <li>create projects (DLL assembly pro platform)</li>
      <li>define project links (all code will be in superset project
        present as link)<br>
      </li>
      <li>copy all code to smallest-common-subset (WP) <br>
        this creates links in superset projects (let say desktop, sl,
        asp.net etc...)<br>
      </li>
      <li>use partial classes - put/extract platform dependent code in
        separate file in class</li>
      <li>move code from step above to appropriate platform projects
        (they are something like deltas)<br>
        Step ends up with (let say in a desktop DLL project):</li>
      <ul>
        <li>common Class.cs file linked from smalles-subset project (WP
          or MT if You decide)</li>
        <li>Class.Delta.cs file with platform specific code in platofrm
          DLLs<br>
          (we use Class.WP7.cs, Class.MTch.cs and/or Class.M4A.cs)<br>
        </li>
      </ul>
    </ul>
