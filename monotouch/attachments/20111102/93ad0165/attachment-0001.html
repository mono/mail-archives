<br></div><div>That&#39;s the problem. You confused C# runtime and iOS SDK. WeakReference and hard references are C# objects. iOS knows nothing if it is a hard reference or a weak reference - it lives its own life. WeakReference tells GC it can collect this object if there are no other hard references to it. Since there is one (delegate), this will never happen.</div>
<div><br></div><div>Also, if GC decides to collect the VC, it will remove it completely. Both view and controller which is not what you want.</div><div><br></div><div>Instead, you should call the DidReceiveMemoryWarning method when you want to release the view but not the controller.</div>
