</div></div>MonoDevelop doesn&#39;t create *.xib.designer.cs files when you add a xib file.<br>
<br>
The thing MD does are:<br>
<br>
* If you have a .xib.designer.cs file for the .xib file, MD generates<br>
codebehind for the xib file in the designer file. This is so that you<br>
can disable the code generation on a per-file basis by removing or<br>
renaming the .xib.designer.cs file.<br>
<br>
* The xib file templates in MD include .xib.designer.cs files, so that<br>
when you create xib files using MD you will have code generation.<br>
<br>
* When you add an existing file that ends with .xib, .xib.cs, or<br>
.xib.designer.cs, MD looks for the other files in the &quot;group&quot; and<br>
automatically adds them. This is a convenience for adding files that<br>
have been removed or copied from other projects.<br>
<br>
Hence you can solve your problem by simply creating a blank<br>
.xib.designer.cs file and adding it. There is a slight problem in that<br>
you cannot create this file from within MD, since the file name for<br>
any C# file in MD must be a valid C# identifier name. This is a known<br>
bug.<br>
<br>
You can create empty .xib.designer.cs files for all of your xibs in<br>
the current director and subdirectories with the following bash<br>
command: for F in `find . -name *.xib`; do touch &quot;$F.designer.cs&quot;;<br>
done<br>
<br>
A possible solution to your problem would be for me to add a &quot;create<br>
designer file&quot; context menu command for xib files.<br>
