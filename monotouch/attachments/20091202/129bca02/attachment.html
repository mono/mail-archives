One point of note - <a href="http://www.bluebytesoftware.com/blog/2005/04/08/DGUpdateDisposeFinalizationAndResourceManagement.aspx">it is recommended</a> in .NET to wrap unmanaged handles inside a SafeHandle derived object. I think Mono has yet to bring SafeHandle <a href="http://www.mono-project.com/SafeHandles">to full parity</a> with .NET&#39;s implementation (e.g. ReliabilityContracts), but it still offers certain guarantees and usually eliminates the need to add finalizers to your types (a worthwhile goal in itself.)<br>
<br>This <a href="http://msdn.microsoft.com/en-us/library/microsoft.win32.safehandles.safehandlezeroorminusoneisinvalid%28VS.80%29.aspx">MSDN sample</a> implements a SafeUnmanagedMemoryHandle class that would likely suit your needs.<br>
<br>Second point, if you&#39;re storing the data in managed memory via the _data field <i>and</i> storing the same data in unmanaged memory (the _dataPtr handle) then you&#39;re probably not playing nice with your heap. I&#39;d suggest freeing one or the other after the marshaled copy unless you <i>really</i> need both and you can&#39;t afford the hit of marshaling the data every time you access it (maybe it&#39;s being used frequently.)<br>
