Chris, <div><br></div><div>the things in square brackets [] are attributes which must be 'applied' to a language construct (class, field, property, etc).</div><div><br></div><div>in your example you have a lot of attributes but nothing to 'apply' them to. in this example (from <a href="http://docs.xamarin.com/ios/tutorials/MonoTouch.Dialog">mt.d docs</a>) notice the Name, Details and IsApproved elements. although it seems like the [Section] attributes are living on their own, they are actually being associated with the subsequent field.</div>
<div><pre class="brush: csharp; collapse: false; first-line: 1; gutter: true; ruler: false; toolbar: true; wrap-lines: true; ">public class Expense
{
       [Section("Expense Entry")]

       [Entry("Enter expense name")]
       <span style="background-color:rgb(255,255,204)">public string Name</span>;
      
       [Section("Expense Details")]

       [Caption("Description")]
       [Entry]
       <span style="background-color:rgb(255,255,204)">public string Details</span>;
      
       [Checkbox]
       <span style="background-color:rgb(255,255,204)">public bool IsApproved = true</span>;
}</pre></div><div>you are getting the 'unexpected }' error because you _at least one_ (but probably more, for your example to work) field or property in your class.</div><div><br></div><div>hth</div><div>craig</div>
