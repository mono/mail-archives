<div>Storyboards provide a nice design-time experience for UI layout, navigation design and can be faster to load in your app than XIBs. Regarding l10n however, this quote is from Apple's docs:</div><div><p><strong>Note:</strong> Before Xcode 4.5, developers and localizers 
had to modify storyboards and nib files for each language an app 
supported. The task required not only that they translate the visible or
 audible text of a storyboard or nib, but that they adjust the sizes and
 positions of views after translation. </p></div><div>-------- </div><div><br></div>In iOS5 an earlier, it isn't possible to 'localize' a Storyboard file... there were two options available:<div><br></div><div>
1) create a separate *.storyboard _for each language_ (ie. copy and paste the storyboard into each *.lproj folder, and type the localized strings into the storyboard directly)</div><div><br></div><div>2) create your storyboard _not_ in an *.lproj folder, and ensure all the localizable elements have an outlet created so you can access from C#. Then in your MonoTouch code, set the display properties directly eg</div>
<div><pre><span class="n">title.Text</span> <span class="p">=</span> <span class="n">MonoTouch</span><span class="p">.</span><span class="n">Foundation</span><span class="p">.</span><span class="n">NSBundle</span><span class="p">.</span><span class="n">MainBundle</span><span class="p">.</span><span class="n">LocalizedString</span> <span class="p">(</span><span class="s">"Task Details"</span><span class="p">,</span> <span class="s">"Task Details"</span><span class="p">);</span></pre>
</div><div>Both these approaches are very manual.</div><div><br></div><div>In iOS6 Apple introduced a new concept called Base Localization - but this ONLY works on iOS6 so if you want backwards compatibility with iOS5 and earlier, you can't use it. There are details in this link</div>
<div><a href="https://developer.apple.com/library/ios/#referencelibrary/GettingStarted/RoadMapiOS/chapters/InternationalizeYourApp/InternationalizeYourApp/InternationalizeYourApp.html">https://developer.apple.com/library/ios/#referencelibrary/GettingStarted/RoadMapiOS/chapters/InternationalizeYourApp/InternationalizeYourApp/InternationalizeYourApp.html</a></div>
<div>This method introduces a new folder - Base.lproj - where you put your *.storyboard files. Then in each *.lproj folder you create a .strings file with the same name as the storyboard.</div><div><br></div><div>For now, MonoDevelop does not offer the IDE helpers that Xcode does (as described in that link) but you *can* do it manually - i've just pushed an example </div>
<div><a href="https://github.com/conceptdev/xamarin-samples/tree/master/TaskyL10nStoryboard">https://github.com/conceptdev/xamarin-samples/tree/master/TaskyL10nStoryboard</a></div><div>note that it does get tricky - you need to match up the object-ids in Xcode to create the localization keys, like this:</div>
