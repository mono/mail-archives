<p style="color: rgb(51, 51, 51); font: normal normal normal 15px/normal &#39;Helvetica Neue&#39;, Helvetica, Arial, sans-serif; line-height: 24px; margin-bottom: 1.625em; font-size: 15px; ">With MonoTouch and Mono for Android, you can already share a huge portion of your code across iOS, Android and Windows Phone. There is, however, this middle ground of APIs that are present in all three platforms, but you can&#39;t share your code that utilizes them. You currently have to write separate sets of code to use geolocation, the accelerometer, contacts, etc on these various platforms and we want to fix that. The Xamarin Mobile API Preview contains a new set of APIs designed and implemented to support these features evenly across iOS, Android and Windows Phone.</p>
<p style="color: rgb(51, 51, 51); font: normal normal normal 15px/normal &#39;Helvetica Neue&#39;, Helvetica, Arial, sans-serif; line-height: 24px; margin-bottom: 1.625em; font-size: 15px; ">We want your feedback to help guide us in this endeavor, so today we&#39;re releasing a preview. The preview contains a unified geolocation API with implementations on iOS and Android. We&#39;ll be releasing a Windows Phone implementation at some point in the future. Try it out and give us your feedback, we want to know what you think! Let us know of any pain points in the API, if something doesn&#39;t work, etc. We also want to know what other areas you care most about us unifying next, whether it be contacts, notifications, etc.</p>
<p style="color: rgb(51, 51, 51); font: normal normal normal 15px/normal &#39;Helvetica Neue&#39;, Helvetica, Arial, sans-serif; line-height: 24px; margin-bottom: 1.625em; font-size: 15px; ">As this is a preview, please keep in mind that APIs may change, things may be broken and not all of the docs are filled out. For Android, currently a minimum API level of 7 is required. Additionally on Android, heading is disabled and Geolocator&#39;s constructor takes an argument of Context.</p>
