<TITLE>Catnap: ORM Targeted To MonoTouch</TITLE>
</HEAD>
<BODY>
<FONT FACE="Calibri, Verdana, Helvetica, Arial"><SPAN STYLE='font-size:11pt'>I have created a rudimentary objection relational mapper (ORM) using sqlite3 with a specific goal of improving the MonoTouch development experience. &nbsp;It&#8217;s called Catnap. &nbsp;Find it here:<BR>
<BR>
<a href="http://code.google.com/p/catnap-orm/">http://code.google.com/p/catnap-orm/</a><BR>
<BR>
A few comments. &nbsp;First, it&#8217;s very basic. &nbsp;The immediate goal is to escape the mire of writing data access code and lots of dynamic SQL. &nbsp;Second, it is definitely alpha. &nbsp;I am only dipping my toes into MonoTouch development; indeed I have only yet deployed to the simulator. &nbsp;Hence, Catnap has only been really tested in very basic scenarios. &nbsp;I would love for folks to try it in their applications and give feedback or, better yet, contribute to its development. &nbsp;It you are willing to roll up your sleeves, I feel like together we can make it &nbsp;a very useful tool for MonoTouch development. &nbsp;<BR>
<BR>
Here are some of the features:<BR>
<BR>
</SPAN></FONT><UL><LI><FONT FACE="Calibri, Verdana, Helvetica, Arial"><SPAN STYLE='font-size:11pt'>Mapping is done via a fluent interface 
</SPAN></FONT><LI><FONT FACE="Calibri, Verdana, Helvetica, Arial"><SPAN STYLE='font-size:11pt'>Mapped entities inherit from IEntity interface 
</SPAN></FONT><LI><FONT FACE="Calibri, Verdana, Helvetica, Arial"><SPAN STYLE='font-size:11pt'>Mapped collections :<BR>
</SPAN></FONT><UL><LI><FONT FACE="Calibri, Verdana, Helvetica, Arial"><SPAN STYLE='font-size:11pt'>automatic loading - lazy or eager, specified per collection 
</SPAN></FONT><LI><FONT FACE="Calibri, Verdana, Helvetica, Arial"><SPAN STYLE='font-size:11pt'>automatic cascading - saves/updates and/or deletes, specified per collection 
</SPAN></FONT><LI><FONT FACE="Calibri, Verdana, Helvetica, Arial"><SPAN STYLE='font-size:11pt'>filtering - criteria specified per collection <BR>
</SPAN></FONT></UL><LI><FONT FACE="Calibri, Verdana, Helvetica, Arial"><SPAN STYLE='font-size:11pt'>Control connection and transaction boundaries via a basic UnitOfWork context 
</SPAN></FONT><LI><FONT SIZE="2"><FONT FACE="Consolas, Courier New, Courier"><SPAN STYLE='font-size:10pt'>Repository&lt;T&gt;</SPAN></FONT></FONT><FONT FACE="Calibri, Verdana, Helvetica, Arial"><SPAN STYLE='font-size:11pt'> base class provides basic operations 
</SPAN></FONT><LI><FONT FACE="Calibri, Verdana, Helvetica, Arial"><SPAN STYLE='font-size:11pt'>IFindSpec&lt;T&gt; provides means to abstract find queries 
</SPAN></FONT><LI><FONT FACE="Calibri, Verdana, Helvetica, Arial"><SPAN STYLE='font-size:11pt'>DbCommandSpec provides a way to create and execute any parameterized query (similar to IDbCommand) 
</SPAN></FONT><LI><FONT FACE="Calibri, Verdana, Helvetica, Arial"><SPAN STYLE='font-size:11pt'>Support for Linq predicates for finding and filtering collections (very limited at present) 
</SPAN></FONT><LI><FONT FACE="Calibri, Verdana, Helvetica, Arial"><SPAN STYLE='font-size:11pt'>Database migration utility. Provides automatic deployment of database changes based on migrations (forward only) that you write 
</SPAN></FONT><LI><FONT FACE="Calibri, Verdana, Helvetica, Arial"><SPAN STYLE='font-size:11pt'>No dependencies except sqlite3.dll <BR>
</SPAN></FONT></UL><FONT FACE="Calibri, Verdana, Helvetica, Arial"><SPAN STYLE='font-size:11pt'><BR>
There is a link to a Google group on the code site. &nbsp;Please chime in if you are trying it out. &nbsp;Let us know how it&#8217;s going, and if you want to contribute.<BR>
