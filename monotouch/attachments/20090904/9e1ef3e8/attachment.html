The short answer is: I think you&#39;re right; it has to do with text encodings.<br><br>The long answer is that I wrote this method intentionally to allow the caller to control as much of the graphics state as possible. It assumes that the caller has set the colors, the text drawing mode, and selected the font. That&#39;s why it saves the graphics state before writing the invisible text and then restoring the state.<br>

<br>When selecting the font, you also set a text encoding. The Apple CGContext class reference says that kCGEncodingFontSpecific is &quot;the built-in encoding of the font&quot; and that kCGEncodingMacRoman is &quot;the MacRoman encoding. MacRoman is an ASCII variant originally created for use in the Mac OS, in which characters 127 and lower are ASCII, and characters 128 and higher are non-English characters and symbols.&quot;<br>

<br>I tried both FontSpecific and MacRoman with a traditional Mac font such as Helvetica. FontSpecific yielded strange characters and MacRoman looked good. Unfortunately,  I have no better answer than this utilitarian one until I&#39;ve spent more time to understand fonts, text encodings, unicode, et. al.<br>

<span style="font-family: courier new,monospace;">context.SelectFont(&quot;Helvetica-Bold&quot;, 14, CGTextEncoding.MacRoman);</span><br style="font-family: courier new,monospace;"><span style="font-family: courier new,monospace;">context.ShowTextAtCenterPoint(100, 100, &quot;Helvetica-Bold 14&quot;);</span><br style="font-family: courier new,monospace;" clear="all">

<span style="font-family: &quot;Courier New&quot;;">// Inspired by technique described in

// &quot;Measuring Text Before Drawing&quot; of<br>

// Note: only works for CGAffineTransform.MakeRotation(0)<br>
public static void ShowTextAtCenterPoint (<br>
    this CGContext context, float centerX, float y, string text)<br>
{<br>
    context.SaveState();<br>
    context.SetTextDrawingMode (CGTextDrawingMode.Invisible);<br>
    context.ShowTextAtPoint (centerX, y, text, text.Length);<br>
    context.RestoreState();<br>
    context.ShowTextAtPoint (<br>
        centerX - (context.TextPosition.X -
centerX)/2,<br>
        y,<br>
        text,<br>
        text.Length);<br>
}<br>
