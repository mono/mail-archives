<a href="http://monotouch.2284126.n4.nabble.com/Question-about-the-Multi-Screened-Application-tutorial-td4404379.html" target="_blank">http://monotouch.2284126.n4.nabble.com/Question-about-the-Multi-Screened-Application-tutorial-td4404379.html</a><br>
Question about the Multi-Screened Application tutorial , where the .h file<br>
for a ViewController disappears, causing the project to generate a host of<br>
errors when you build it.<br>
<br>
I've been able to pretty reliably reproduce the problem with these steps:<br>
<br>
1) Create a new solution in MonoDevelop (number of views doesn't matter).<br>
<br>
2)  Double-click a ViewController xib to edit it in Interface Builder.<br>
<br>
3) In IB, add one or more buttons or other screen elements, open the<br>
Assistant Editor to access the ViewController.h file, ctrl+drag one or more<br>
of them over and create outlets like normal.<br>
<br>
4) Quit IB and go back to MonoDevelop. Make significant changes to the code<br>
in the ViewController file in question. In particular, cutting a big chunk<br>
of of the code and pasting in replacement code seems to do the trick.<br>
<br>
(I haven't been able to confirm if quitting and relaunching MonoDevelop<br>
between steps 4 and 5 is part of the problem or not.)<br>
<br>
5) Double-click on that ViewController's xib again, relaunching IB. Click on<br>
the Assistant Editor button. If the problem occurs, you won't have a .h file<br>
anymore and the Assistant Editor screen will say "No Assistant Results".<br>
<br>
Has anyone else experienced this, and more importantly, is there any way to<br>
to recover or recreate the missing .h file (or get MonoTouch to generate a<br>
new one) without rebuilding the project from scratch?<br>
