</blockquote><div><br></div><div>What happens when you add the "missing" ctor is that MonoTouch creates a new instance of the managed class when one is needed (and doesn't already exist). This means that any state you had in the GC'd instance, is lost. For instance:</div>
<div><br></div><div>class CustomUITableViewController : UITableViewController {</div><div>    string someValue;</div><div>    public CustomUITablewViewController (IntPtr ptr) : base (ptr) {}</div><div>}</div><div><br></div>
<div>What you will experience is that the someValue field will randomly get nulled out (when one instance of the class is GC'ed and another one created in its place). "Randomly" here means between any function calls (such as between ViewWillAppear and ViewDidAppear).</div>
<div><br></div><div>If you do not have class-level fields in your classes (or you do not care if those fields are randomly cleared out), it's safe to add the ctor.</div><div><br></div><div>Rolf</div></div>
