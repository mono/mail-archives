    It's not an uncommon problem... There is a <b>[Preserve]</b>
    attribute which you can add to any member, instructing the linker
    that it should not be stripped from output.<br>
    <br>
    From the following page: <a class="moz-txt-link-freetext" href="http://docs.xamarin.com/ios/troubleshooting">http://docs.xamarin.com/ios/troubleshooting</a><br>
    <h3 class="editable">System.MissingMethodException (anything else)</h3>
    <p>The member was likely removed by the linker, and thus doesn't
      exist in the assembly at runtime.  There are several solutions to
      this:</p>
    <ul>
      <li>Add the <a class="external"
title="http://www.go-mono.com/docs/index.aspx?link=T:MonoTouch.Foundation.PreserveAttribute"
          rel="external nofollow"
href="http://www.go-mono.com/docs/index.aspx?link=T:MonoTouch.Foundation.PreserveAttribute"
          target="_blank">[Preserve]</a> attribute to the member.  This
        will prevent the linker from removing it.</li>
      <li>When invoking <a class="external"
          title="http://www.go-mono.com/docs/index.aspx?link=man:mtouch(1)"
          rel="external nofollow"
          href="http://www.go-mono.com/docs/index.aspx?link=man:mtouch%281%29"
          target="_blank">mtouch</a>, use the <strong>-nolink</strong> or <strong>-linksdkonly</strong> options.
        <ul>
          <li><strong>-nolink</strong> disables all linking.</li>
          <li><strong>-linksdkonly</strong> will only link
            MonoTouch-provided assemblies, such as <em>monotouch.dll</em>.</li>
        </ul>
      </li>
    </ul>
    <p>Note that assemblies are linked so that the resulting executable
      is smaller; thus, disabling linking may result in a larger
      executable than is desirable.<br>
    </p>
    <p>I see you already solved you immediate issue in the time it took

      dinner... but this still might be of use to you or someone else.<br>
    </p>
