<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal style='margin-bottom:12.0pt'>We are pleased to announce the
release of MonoTouch v1.1.0 as well as an updated MonoDevelop Preview.
&nbsp;The latest version of MonoDevelop contains a new updater feature which
will notify you in the future when updates are available for MonoDevelop or
MonoTouch.<o:p></o:p></p>

<div>

<p class=MsoNormal>If you have purchased MonoTouch, you can download the
updated version from the download link provided when you purchased the
product.&nbsp;<o:p></o:p></p>

<div>

<p class=MsoNormal>If you are using the evaluation version you can download the
updated version of MonoTouch from <a href="http://monotouch.net/DownloadTrial">http://monotouch.net/DownloadTrial</a>&nbsp;to
receive the update.<br>
You can download the updated MonoDevelop at:&nbsp;<a
href="http://monodevelop.com/Download/Mac_MonoTouch">http://monodevelop.com/Download/Mac_MonoTouch</a><br>
<br>
Highlights<br>
<img border=0 width=20 height=16 id="_x0000_i1025"
src="cid:image001.png@01CA4344.DBB5C3C0"><br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; All new iPhone 3.1 APIs are now
available.<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; Native code generation is now
30% smaller for core assemblies.<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; Reduced startup time by up-to
50%.<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; Web Services support.<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; MonoDevelop now has an updater
feature, so developers can more easily get the latest Mono, MonoDevelop and
MonoTouch.<br>
Fixes:<br>
<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; Fixed invalid CFRelease in
CFRunLoop (#541524)<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; Fixed IList&lt;T&gt; in
full-aot (#540355)<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; Fixed Delegate.BeginInvoke in
full-aot<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; Fixed XElement[].Count in full-aot
(#539085)<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; Fixed MKAnnotation in Simulator<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; Fixed linker to not always pull
in Mono.Security<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; Fixed System.Timer linked-away
exception<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; Decreased startup time on
device by up to 50%<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; The System.JSon assembly is now
available for inclussion.<br>
Additions:<br>
<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; Added bindings to new 3.1 APIs<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; Added generic trampoline for
marshal-runtime-invoke wrappers, providing significant code savings (34% on
mscorlib size, equivalent savings for other assemblies).<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; Added 2 new ctors to
NSFoundation.NSMutableUrlRequest<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; Added additional support for
alternate types to NSData<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; Added new StringSize overloads
to UIView<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; Added CALayer.Sublayers
property<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; Added
CoreAnimation.CATransaction class<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; Added new SystemSound
constructors<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; Added SystemSound.FromFile
(string)<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; Added System.Web.Services
client support.<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; Added WCF client support
(EXPERIMENTAL)<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; Added support to emit DWARF
debug symbols in full-aot mode<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; Added support to be
auto-updated from MD.<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; Updated keybidings to
MonoDevelop for Emacs.<br>
Changes:<br>
<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; SystemSound.FromFile no longer
throws exceptions on error, and instead will return null.<br>
<span class=apple-tab-span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&#8226; iPhoneOSGameView no longer
calls GL.Oes.BindFrameBuffer () during the second and subsequent
CreateFrameBuffer () calls. &nbsp;Code overriding iPhoneOSGameView.CreateFromBuffer
() does not need to call GL.Oes.BindFramebuffer (All.FramebufferOes,
base.FrameBuffer) to add a depth buffer to the frame buffer.<br>
