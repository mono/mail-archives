    Our team is porting a WinCE app to MT. This app is quite complex

    much code as they can.<br>
    <br>
    So in first run/spurt we created all WinForms CE Forms as xibs and
    did some proof-of-concept<br>
    bindings to business logic. <br>
    <br>
    In the xib for Image manipulation (MainForm) we have created
    UIImageView and filled it with <br>
    some test jpegs through UIImage.<br>
    <br>
            [Outlet]<br>
            UIImageView CustomViewControl { get; set; }<br>
    <br>
    So in order to reuse his code our client has changed .designer.cs
    file into:<br>
            [Outlet]<br>
            CustomViewControlType  CustomViewControl { get; set; }<br>
    <br>
    and changed that CustomViewControlType  inherits form UIImageView.<br>
    <br>
    OK. Let us not discuss why .designer.cs is changed though there is
    comment at the beginning.<br>
    It is done and let us work from here.<br>
    <br>
    BTW @Xamarin MT team - good job with Xcode and MT integration.
    Adding new Outlets to<br>
    Xib did not alter change above - new Outlets were added w/o
    overwritting existing code!<br>
    <br>
    This step broke Xcode loading of xibs, so [Register] attribute was
    added to CustomViewControlType <br>

    CustomViewControl is created,<br>
