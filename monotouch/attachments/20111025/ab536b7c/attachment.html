I had this exact same issue and through a process of elimination, finally<br>
figured out the cause.<br>
<br>
My views that were created prior to 2.8 had an outlet for the View called<br>
view. When I opened the header file, I discovered that XCode was complaining<br>
that declaration for the view was a duplicate name.<br>
<br>
Since this seemed like it could be the source of the problem, I took the<br>
following steps:<br>
<br>
1) Deleted the view outlet in the file owner&#39;s list of outlets<br>
<br>
2) Manually removed the two references to view in the header file<br>
<br>
3) Linked the View outlet in the file owner&#39;s list of outlets directly to<br>
the view (I did not drag the connection to the header file, I dragged it to<br>
the view that is on left-hand side of the designer area). If you forget to<br>
do this step, you will receive a runtime error about the nib being loaded<br>
but the view outlet not being sent.<br>
<br>
Hopefully this will help resolve someone else&#39;s problems as well.<br>
