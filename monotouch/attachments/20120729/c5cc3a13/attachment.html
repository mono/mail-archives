<div><br></div><div>Find the pid of the app - it is printed to the application output in MonoDevelop at startup.</div><div><br></div><div>Then execute from a terminal:</div><div><br></div><div>> gdb program <PID></div>
<div>(gdb will attach, some output until a gdb prompt appears:)</div><div>$(gdb) thread apply all backtrace</div><div>(gdb will print stack traces for all threads).</div><div><br></div><div>Get all the output from the terminal, save it to a file and send it to the lists, and we'll easily be able to figure out if it's a deadlock in your code or if it's in the debugger.</div>
