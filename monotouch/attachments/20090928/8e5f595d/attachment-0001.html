Should be possible but will require some experimenting.  Here are some links and concepts that might be useful:<br>
<br>
<a href="http://monotouch.net/Documentation/Objective-C_Selectors" target="_blank">http://monotouch.net/Documentation/Objective-C_Selectors</a><br>
You can use selectors to invoke methods on classes that are loaded at runtime.<br>
<br>
<a href="http://monotouch.net/Documentation/XCode" target="_blank">http://monotouch.net/Documentation/XCode</a><br>
How to link monotouch into an xcode project.<br>
<br>
<a href="http://wiki.monotouch.net/User:Trustme" target="_blank">http://wiki.monotouch.net/User:Trustme</a><br>
On this page I do the opposite I embed an objective-c library and invoke a class on that library at runtime.<br>
<br>
Objective-C uses late binding.  So you can do this:<br>
<br>
classPtr = findClass(&quot;NameOfClass&quot;);<br>
classPtr.InvokeSelector(&quot;runMethodA&quot;, args);<br>
<br>
So you may have to wrap access to your C# library in a class that is registered to the objective-c environment.<br>
<br>
You can also use NSNotificationCenter.<br>
<br>
But in some way your C# library has to run the initialization code to register the classes with objective-c at runtime.  I did exactly what you are attempting with an objective-c desktop app.  At startup the objective-c code initialized the C# dll so the classes within registered themselves so that I could access them from objective-c.<br>

<br>
