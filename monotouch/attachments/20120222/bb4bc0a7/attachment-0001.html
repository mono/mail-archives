Firing 100 requests at once will probably cause trashing and excessive memory usage.<div><br></div><div>If you can live with doing a request at a time you could chain one request after the other</div><div>from the finished callback.</div>
<div><br></div><div>Limiting to a given degree of parallelism is tricky, but possible using callbacks as well.</div><div>You definitely needs to experiment how each approach works in practice as they all have</div><div>tradeoffs.</div>
