Hi Pratheek --<div><br></div><div>It looks like you&#39;ve got something like a mix of C# and Objective-C code there. In C# it would look more like:</div><div><br></div><div><span class="Apple-tab-span" style="white-space: pre; ">        </span>this.webMain.LoadRequest(new NSUrlRequest(NSUrl.FromFilename(...));</div>
<div><br></div><div>As far as the file path, (the ... above), I haven&#39;t tried using the NS resource stuff at all; I just mark my html files with the &quot;content&quot; build action and use paths relative to the project root.</div>
<div><br></div><div>But, if the resource stuff works, the method you&#39;re trying to call would look like:</div><div><br></div><div><div><span class="Apple-tab-span" style="white-space:pre">        </span>NSBundle.MainBundle.PathForResource(&quot;<span class="Apple-style-span" style="font-family: arial, sans-serif; font-size: 13px; border-collapse: collapse; ">cliqueitsample</span>&quot;, &quot;<font class="Apple-style-span" face="arial, sans-serif"><span class="Apple-style-span" style="border-collapse: collapse;">html</span></font>&quot;);</div>
<div><br></div><div>It takes some time to get used to translating Objective-C examples into C# for Monotouch, and the mapping isn&#39;t always intuitive. You might want to have a look at Miguel de Icaza&#39;s &quot;<a href="http://tirania.org/tmp/rosetta.html">Monotouch Rosetta Stone</a>&quot; to get a feel for it.</div>
