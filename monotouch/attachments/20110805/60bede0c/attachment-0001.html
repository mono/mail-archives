<p class="MsoNormal">If it's happening on device, the XCode organizer records all crashes.<o:p></o:p></p>
<div>
<p class="MsoNormal">Otherwise, go get traces on simulator, you can try start it from the command line doing the follow:<o:p></o:p></p>
</div>
<div>
<p class="MsoNormal"><o:p>&nbsp;</o:p></p>
</div>
<div>
<p class="MsoNormal">From one console fire:<o:p></o:p></p>
</div>
<div>
<p class="MsoNormal">/Developer/MonoTouch/usr/bin/mtouch --debugsim=$path_to_your_app<o:p></o:p></p>
</div>
<div>
<p class="MsoNormal">From another console use<o:p></o:p></p>
</div>
<div>
<p class="MsoNormal">gdb -p $pid_shown_in_the_previous_step<o:p></o:p></p>
</div>
<div>
<p class="MsoNormal"><o:p>&nbsp;</o:p></p>
</div>
<div>
<p class="MsoNormal">Then let it run - use the 'continue' &nbsp;command from the gdb shell.<o:p></o:p></p>
</div>
<div>
<p class="MsoNormal">Once it crash, you can use the command &quot;t a a bt&quot; to get a complete thread dump of your app.<o:p></o:p></p>
</div>
<div>
<p class="MsoNormal">We can start troubleshooting from there.<o:p></o:p></p>
</div>
<div>
<p class="MsoNormal"><o:p>&nbsp;</o:p></p>
</div>
<div>
<p class="MsoNormal">$path_to_your_path -&gt; this is the path where MonoDevelop puts a simulator build, for MonoTouch.Dialogs Sample on my machine it's:&nbsp;<o:p></o:p></p>
</div>
<div>
<p class="MsoNormal">/Users/kumpera/src/MonoTouch.Dialog/Sample/bin/iPhoneSimulator/Debug/Sample.app.<o:p></o:p></p>
</div>
<div>
<p class="MsoNormal">You can use relative paths with mtouch, e.g. cd ~/src/MonoTouch.Dialog/Sample/bin/iPhoneSimulator/Debug ; mtouch --debugsim=Sample.app<o:p></o:p></p>
</div>
<div>
<p class="MsoNormal"><o:p>&nbsp;</o:p></p>
</div>
<div>
<p class="MsoNormal">$pid_shown_in_the_previous_step -&gt; this is the pid of the simulated app. I'm not sure if 4.0.4.1 shows the pid (trunk certainly does), but<o:p></o:p></p>
</div>
<div>
<p class="MsoNormal">if it doesn't, you can find it on Activity Monitor or using 'ps' &nbsp;from the command line.<o:p></o:p></p>
</div>
<div>
<p class="MsoNormal"><o:p>&nbsp;</o:p></p>
</div>
