I was able to solve it.  In order to get a valid SOAP client using<br>
Xamarin/Monotouch running on iOS, you must use the Silverlight soap proxy<br>
generator:  slsvcutil.exe.  this is because iOS does not allow dynamic code<br>
execution and the standard soap proxy generator:  svcutil.exe  creates a<br>
client that depends on dynamic code execution.  The Silverlight generator<br>
produces a simpler client without dynamic code execution.  However, it is<br>
too simple in that it does NOT support soap headers.<br>
<br>
My solution to this problem is to still use the Silverlight soap proxy<br>
generator, but then implement 2 classes to manually add in the soap headers<br>
before the soap requests go out during runtime.  Here are the two classes I<br>
implemented.  You will see that the second class contains the<br>
BeforeSendRequest method which I override to manually add in my soap<br>
headers.  Of course, your soap headers will be different than mine.  Then to<br>
use these two classes, you add an endpoint behavior to your client when you<br>
construct it:  client.Endpoint.Behaviors.Add (new<br>
EditorialSoapHeaderBehavior (appName,sessionId));<br>
<br>
                private class EditorialSoapHeaderBehavior : IEndpointBehavior<br>
                {<br>
                        private string _appName = null;<br>
                        private string _sessionId = null;<br>
<br>
                        public EditorialSoapHeaderBehavior(string appName,string sessionId)<br>
                        {<br>
                                _appName = appName;<br>
                                _sessionId = sessionId;<br>
                        }<br>
<br>
                        public void AddBindingParameters (ServiceEndpoint endpoint,<br>
BindingParameterCollection parameters)<br>
                        {<br>
                        }<br>
                        public void ApplyDispatchBehavior (ServiceEndpoint endpoint,<br>
EndpointDispatcher dispatcher)<br>
                        {<br>
                        }<br>
                        public void ApplyClientBehavior (ServiceEndpoint endpoint, ClientRuntime<br>
runtime)<br>
                        {<br>
                                runtime.MessageInspectors.Add (new EditorialSoapHeaderInspector<br>
(_appName,_sessionId));<br>
                        }<br>
                        public void Validate (ServiceEndpoint endpoint)<br>
                        {<br>
                        }<br>
                }<br>
<br>
                private class EditorialSoapHeaderInspector : IClientMessageInspector<br>
                {<br>
                        private string _appName = null;<br>
                        private string _sessionId = null;<br>
<br>
                        public EditorialSoapHeaderInspector(string appName,string sessionId)<br>
                        {<br>
                                _appName = appName;<br>
                                _sessionId = sessionId;<br>
                        }<br>
<br>
                        public void AfterReceiveReply (ref Message message, object<br>
correlationState)<br>
                        {<br>
                        }<br>
                        public object BeforeSendRequest (ref Message message, IClientChannel<br>
channel)<br>
                        {<br>
                                MessageHeader mh = MessageHeader.CreateHeader ("ApplicationName",<br>
"IHS.Core.Security.WCF", _appName);<br>
                                message.Headers.Add (mh);<br>
                                mh = MessageHeader.CreateHeader ("SessionId", "IHS.Core.Security.WCF",<br>
_sessionId);<br>
                                message.Headers.Add (mh);<br>
                                return(null);<br>
                        }<br>
                }<br>
