An imageview (UIImageView) is something like the Image control for Windows.<BR>
So this guy holds ONE image and can be placed (sized, hidden, shown...) on you UIView.<BR>
&nbsp;<BR>
First approach (not the best I guess) - use multiple UIImageViews and place them individually.<BR>
Since we have a Z-Order you can place them even on top of each other.<BR>
&nbsp;<BR>
The other approach - create an image&nbsp;by drawing various elements on&nbsp;if (other images for an example)<BR>
and place that resulting image in an UIImageView.<BR>
It's a bit like drawing in windows (forms) applications. You create a device context, and use functions of<BR>
this context to draw on it.<BR>
The class for such a "drawable image" is CGBitmapContext<BR>
The function to draw an image on it is DrawImage<BR>
