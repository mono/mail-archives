<p class="MsoNormal">On Tue, Aug 30, 2011 at 11:08 AM, derik &lt;<a href="/user/SendEmail.jtp?type=node&amp;node=3779065&amp;i=0" target="_top" rel="nofollow" link="external">[hidden email]</a>&gt; wrote:<o:p></o:p></p>
<p class="MsoNormal">I recently experienced an issue where an app would build just fine, but when<br>
I attempted to run the app in the emulator it would not install.<br>
<br>
I was boggled by this and tried all manner of project settings and resetting<br>
the emulator, everything I could think of to get the app to deploy to the<br>
emulator.<br>
<br>
After a couple hours I starting thinking backward on what I had done to the<br>
project itself since it was created. I slowly started working backward<br>
through my changes and undoing them one at a time and testing at each point<br>
along the way to find the breaking change.<br>
<br>
Eventually I changed the project name in MonoDevelop back to the original<br>
name (which matched the folder containing the project file) and it worked. I<br>
started reintroducing my changes one at a time, leaving the project with its<br>
original name. I was able to use all of my changes except the project name<br>
change.<br>
<br>
If you encounter a failure to launch on the emulator and there are no error,<br>
warning, or informational logs and it just doesn’t make sense; I suggest<br>
first renaming the project in MonoDevelop to match the folder name that<br>
contains the project (which is how the project was created to begin with).<br>
<br>
Perhaps this could be logged as a minor bug and addressed in a future<br>
release. I personally don’t think it’s a big deal; there isn’t any obvious<br>
tie between the project name and the display name once you’ve taken a moment<br>
to actually set the display name.<br>
<br>
Happy Coding!<br>
