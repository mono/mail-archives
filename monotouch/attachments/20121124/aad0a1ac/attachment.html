<div><br>
</div>
<div>I use Parallels, VS 2012 and share a bunch of source (about 40k lines of .cs code) between our WinForms app and the iOS app.</div>
<div><br>
</div>
<div>My eoTouch folder (containing the iOS app) contains about 20k lines of .cs code)</div>
<div><br>
</div>
<div>In iOS, I use:</div>
<div><br>
</div>
<div>1. tf get -r ~/eostar</div>
<div>2. tf checkout -r ~/eostar/eoTouch</div>
<div>3. tf add zzz.png</div>
<div>4. tf delete mmm.cs</div>
<div>5. tf checkin -comment:'eoTouch - made it even better' (this only checks in the changes, deletions and additions - unchanged files are just reverted)</div>
<div><br>
</div>
<div>It's not brilliant, and it's not integrated into the MonoDevelop IDE - but, it's working.</div>
<div><br>
</div>
<div>The other developers are using VS 2012, and we have integrated issue tracking … I use Windows when working with the web service that's used by the app.</div>
<div><br>
</div>
<div>I used git with a private repository when I was working initially in Xcode - just fyi.</div>
<div><br>

Hi Kris,<br>
<br>
What you are describing is the VSS way of doing things.<br>
<br>
Checking files in and out is something which VSS uses to ensure<br>
concurrency, whereas other source control systems allow developers to work<br>
at the same time and merge their changes.<br>
<br>
I think you have 2 choices:<br>
<br>
1. Learn how to use a DVCS such as Git, or Mercurial.<br>
<br>
2. Use VSS which you know how to use.<br>
<br>
If the command line is a problem, there are tools which allow you to use<br>
Git in a GUI:<br>
<br>
- the GitHub GUI<br>
- SourceTree by Atlassian (I prefer this one)<br>
- TortoiseGit<br>
<br>
I use Mercurial at work, Git at home.<br>
<br>
Regards,<br>
<br>
James<br>
