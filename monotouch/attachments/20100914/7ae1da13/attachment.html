<TITLE>Image In Cell Can Leak Memory</TITLE>
</HEAD>
<BODY>
<FONT FACE="Calibri, Verdana, Helvetica, Arial"><SPAN STYLE='font-size:13pt'>I just learned something that I thought I would share. &nbsp;Did you know that this will leak memory?<BR>
<BR>
public class MyCell : UITableViewCell<BR>
{<BR>
&nbsp;&nbsp;&nbsp;&nbsp;UImageView myImageView;<BR>
<BR>
&nbsp;&nbsp;&nbsp;&nbsp;public MyCell () : base(UITableViewCellStyle.Default, &#8220;myCell&#8221;)<BR>
&nbsp;&nbsp;&nbsp;&nbsp;{<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;...<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;myImageView = new UIImageView(someRectF);<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;myImageView.Image = UiImage.FromFile(&#8220;Images/myimage.png&#8221;);<BR>
&nbsp;&nbsp;&nbsp;&nbsp;}<BR>
&nbsp;&nbsp;&nbsp;&nbsp;...<BR>
}<BR>
<BR>
It makes sense that this image is duplicated in memory for each cell. &nbsp;That&#8217;s bad enough. &nbsp;The bigger problem, it seems, &nbsp;is that these images do not get disposed when the table controller is popped. &nbsp;If you push and pop this controller enough times with enough cells, you will run out of memory.<BR>
<BR>
I solved it like this:<BR>
<BR>
public static class Images<BR>
{<BR>
&nbsp;&nbsp;&nbsp;&nbsp;static Images ()<BR>
&nbsp;&nbsp;&nbsp;&nbsp;{<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;MyImage = UIImage.FromFile (&quot;Images/ myimage.png&quot;);<BR>
&nbsp;&nbsp;&nbsp;&nbsp;}<BR>
<BR>
&nbsp;&nbsp;&nbsp;&nbsp;public static UIImage MyImage { get; private set; }<BR>
}<BR>
<BR>
Then in the ctor of the cell:<BR>
<BR>
myImageView.Image = Images .MyImage;<BR>
