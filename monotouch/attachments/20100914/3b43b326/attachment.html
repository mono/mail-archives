</SPAN></FONT><BLOCKQUOTE><FONT FACE="Calibri, Verdana, Helvetica, Arial"><SPAN STYLE='font-size:11pt'>I have to chime in here. Full disclosure: as of now, I'm not a<BR>
MonoTouch user (hopefully will change very *very* soon).<BR>
<BR>
To counter act all the hate on MonoDevelop - I find it pretty damn<BR>
stable.  I've done nearly the entire development on a very substantial<BR>
project within MD.  The project consists of an ASP.Net MVC front end<BR>
calling into a thin client business layer that interacts with a system<BR>
service (a Windows service surfaced in Linux using the mono-service<BR>
wrapper).  It all works very, very well - considering - and was about<BR>
95% developed purely on Mac using Mono / MD.<BR>
<BR>
The biggest MD issue that's yet to be resolved, IMO, is the debugging<BR>
experience.  This is probably the most unstable bit of MD.  Now<BR>
debugging isn't broken, per se, but it's fairly slow, completely stops<BR>
intermittently (no crash, it just quits out of debugging), and doesn't<BR>
handle multiple threads the best.  But it *does* work.  One bit of<BR>
advice for debugging is if the IDE seems to halt, do not kill the MD<BR>
process, but kill the run-away debug process (it'll be titled &quot;mono&quot;<BR>
and pegging your CPU).  This will allow you to simply restart<BR>
debugging w/o having to restart the IDE every time.<BR>
<BR>
I'm about 350 hours into this project so I'm pretty satisfied with my<BR>
MD experience.  If you're curious about the project there's a teaser<BR>
page here:  <a href="http://tendr.me">http://tendr.me</a>  We're aiming to be in Beta w/i 3 months.<BR>
The plan is to eventually develop a native iPhone app on top of the<BR>
MonoTouch framework.<BR>
