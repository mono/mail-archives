<font face="Calibri, Verdana, Helvetica, Arial"><span style="font-size:13pt">I just learned something that I thought I would share.  Did you know that this will leak memory?<br>
<br>
public class MyCell : UITableViewCell<br>
{<br>
    UImageView myImageView;<br>
<br>
    public MyCell () : base(UITableViewCellStyle.Default, “myCell”)<br>
    {<br>
        ...<br>
        myImageView = new UIImageView(someRectF);<br>
        myImageView.Image = UiImage.FromFile(“Images/myimage.png”);<br>
    }<br>
    ...<br>
}<br>
<br>
It makes sense that this image is duplicated in memory for each cell.  That’s bad enough.  The bigger problem, it seems,  is that these images do not get disposed when the table controller is popped.  If you push and pop this controller enough times with enough cells, you will run out of memory.<br>

<br>
I solved it like this:<br>
<br>
public static class Images<br>
{<br>
    static Images ()<br>
    {<br>
        MyImage = UIImage.FromFile (&quot;Images/ myimage.png&quot;);<br>
    }<br>
<br>
    public static UIImage MyImage { get; private set; }<br>
}<br>
<br>
Then in the ctor of the cell:<br>
<br>
myImageView.Image = Images .MyImage;<br>
