Hey,<div><br></div><div>I don't know any pros/cons OTTOMH but there are some parts of the API (exposed by MonoTouch) that still require NSStrings (even NSArrays), so you will occasionally see them, even in examples and such. Therefore it is worthwhile being aware of their existence and how to use them.</div>
<div><br></div><div>However, from a code maintenance perspective, I'd argue that sticking to one type system is probably better in the long term (regardless of performance/stability issues; though I doubt there are any such issues) -- just like any other 'coding standard' that is intended to make code sharing/debugging/maintenance easier. So stick to .NET types always, unless you *need* the NS equivalents...</div>
