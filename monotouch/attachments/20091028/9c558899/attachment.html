</SPAN></FONT><BLOCKQUOTE><FONT FACE="Calibri, Verdana, Helvetica, Arial"><SPAN STYLE='font-size:11pt'>MonoTouch 1.2 Beta 1<BR>
The beta for MonoTouch 1.2 is now available for developers interested <BR>
in trying out the upcoming features in MonoTouch.<BR>
<BR>
You can find the download links at the full release notes page: <a href="http://monotouch.net/Releases/MonoTouch_1.2_Beta_1">http://monotouch.net/Releases/MonoTouch_1.2_Beta_1</a><BR>
<BR>
The MonoTouch beta is experimental, and as such we recommend you back <BR>
up your projects.<BR>
New Features<BR>
<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Debugger, works on Simulator and Device (using Wifi)<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; C# binding generator for Objective-C Libraries.<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Console output is now redirected from the device to MonoDevelop.<BR>
<BR>
MonoTouch API additions:<BR>
<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; UITableView now has a new property &quot;Source&quot; that allows a new <BR>
UITableViewSource type to be assigned to it, it is a Model that <BR>
implements both the UITableViewDelegate and the UITableViewDataSource, <BR>
this should reduce clutter in existing code.<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; NSTimer.Dispose will now call Invalidate first, and then dispose <BR>
the timer.<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; UIActionSheet now can be initialized like this:<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; new UIActionSheet (&quot;Title&quot;) { &quot;Help&quot;, &quot;Open in Browser&quot;, <BR>
&quot;Bookmark&quot; }<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Added NSKeyedArchiver, NSKeyedUnarchiver.<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Added Encoder virtual method on NSObject<BR>
<BR>
Mono Runtime API additions:<BR>
<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Added System.Timers.Timer<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Added System.Threading.Semaphore<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Added System.Threading types to System.dll<BR>
<BR>
API fixes<BR>
Updated OpenTK to r2350.<BR>
<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; This fixes some function overloads so that they're usable<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Fixed AVAudioPlayer factory methods to be static.<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Fixed typo in UITableView.DidEndEditing<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Fixed typo in AVAudioRecorder.Pecord -&gt; Record<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Fixed BadgeValue to allow null<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; The helper class NSNotificationHandler is no longer public.<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Flagged a NSString.DataUsingEncoding as Obsolete, introduced <BR>
instead Encode methods.<BR>
<BR>
System.Data<BR>
A preview of the System.Data and SQLite for MonoTouch is now available.<BR>
<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Mono.Data.Sqlite.dll: unchanged. &nbsp;Note that this binding is against <BR>
SQLite 3.6, while iPhone has SQLite 3.0, so use of some functionality <BR>
will result in EntryPointNotFoundExceptions. &nbsp;Such missing <BR>
functionality appears to include DataSet support.<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Mono.Data.Tds.dll: unchanged.<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; System.Data.dll: Removes code-generation capabilities <BR>
(System.Data.TypedDataSetGenerator), .config handling support, <BR>
System.Data.Common.DbProviderFactories, <BR>
System.Data.OleDb,System.Data.Odbc.System.Transactions.dll: unchanged.<BR>
<BR>
Fixes<BR>
<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Fixed crash using async socket operations<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Fixed i18n in mtouch (Please provide documentation!)<BR>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; iPhoneOSGameView.Run() will no longer truncate the timeout to have <BR>
1ms resolution. &nbsp;It will now have 100ns resolution.<BR>
<BR>
If you have any questions or concerns please see <a href="http://monotouch.net/Support">http://monotouch.net/Support</a><BR>
<BR>
Thank You<BR>
