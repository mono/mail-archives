<p class=MsoNormal style='margin-bottom:12.0pt'><span style='font-size:11.0pt;
font-family:"Calibri","sans-serif"'>MonoTouch 1.2 Beta 1<br>
The beta for MonoTouch 1.2 is now available for developers interested <br>
in trying out the upcoming features in MonoTouch.<br>
<br>
You can find the download links at the full release notes page: <a
href="http://monotouch.net/Releases/MonoTouch_1.2_Beta_1">http://monotouch.net/Releases/MonoTouch_1.2_Beta_1</a><br>
<br>
The MonoTouch beta is experimental, and as such we recommend you back <br>
up your projects.<br>
New Features<br>
<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Debugger, works on Simulator
and Device (using Wifi)<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; C# binding generator for
Objective-C Libraries.<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Console output is now
redirected from the device to MonoDevelop.<br>
<br>
MonoTouch API additions:<br>
<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; UITableView now has a new
property &quot;Source&quot; that allows a new <br>
UITableViewSource type to be assigned to it, it is a Model that <br>
implements both the UITableViewDelegate and the UITableViewDataSource, <br>
this should reduce clutter in existing code.<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; NSTimer.Dispose will now call
Invalidate first, and then dispose <br>
the timer.<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; UIActionSheet now can be
initialized like this:<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226;
new UIActionSheet (&quot;Title&quot;) { &quot;Help&quot;, &quot;Open in
Browser&quot;, <br>
&quot;Bookmark&quot; }<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Added NSKeyedArchiver,
NSKeyedUnarchiver.<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Added Encoder virtual method
on NSObject<br>
<br>
Mono Runtime API additions:<br>
<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Added System.Timers.Timer<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Added
System.Threading.Semaphore<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Added System.Threading types
to System.dll<br>
<br>
API fixes<br>
Updated OpenTK to r2350.<br>
<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; This fixes some function
overloads so that they're usable<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Fixed AVAudioPlayer factory
methods to be static.<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Fixed typo in
UITableView.DidEndEditing<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Fixed typo in
AVAudioRecorder.Pecord -&gt; Record<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Fixed BadgeValue to allow
null<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; The helper class
NSNotificationHandler is no longer public.<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Flagged a
NSString.DataUsingEncoding as Obsolete, introduced <br>
instead Encode methods.<br>
<br>
System.Data<br>
A preview of the System.Data and SQLite for MonoTouch is now available.<br>
<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Mono.Data.Sqlite.dll:
unchanged. &nbsp;Note that this binding is against <br>
SQLite 3.6, while iPhone has SQLite 3.0, so use of some functionality <br>
will result in EntryPointNotFoundExceptions. &nbsp;Such missing <br>
functionality appears to include DataSet support.<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Mono.Data.Tds.dll: unchanged.<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; System.Data.dll: Removes
code-generation capabilities <br>
(System.Data.TypedDataSetGenerator), .config handling support, <br>
System.Data.Common.DbProviderFactories, <br>
System.Data.OleDb,System.Data.Odbc.System.Transactions.dll: unchanged.<br>
<br>
Fixes<br>
<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Fixed crash using async
socket operations<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; Fixed i18n in mtouch (Please
provide documentation!)<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&#8226; iPhoneOSGameView.Run() will
no longer truncate the timeout to have <br>
1ms resolution. &nbsp;It will now have 100ns resolution.<br>
<br>
If you have any questions or concerns please see <a
href="http://monotouch.net/Support">http://monotouch.net/Support</a><br>
<br>
