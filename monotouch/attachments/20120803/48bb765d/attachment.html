    This was workaround:<br>
    <ol>
      <li>download monodevelop 3.0.3.5 (latest stable)</li>
      <li>open Application folder</li>
      <li>right click on MonoDevelop.app, show package content</li>
      <li>Contents -> MacOS -> and there you should see
        MonoDevelop script</li>
      <li>cmd + i (get file info), scroll to Name & Extension, and
        change form upper case letters to lower case, </li>
      <li>it shoud look like this monodevelop</li>
      <li>download latest monotouch, and start installation, if
        everything is ok you should <br>
        see only monotouch from requirements for installation,</li>
      <li>if you also see monodevelop that means that installer cant
        find monodevelop that you installed.<br>
        Recheck that everything is lower case letter, and also check
        permissions, try adding <br>
        read/write for admin as well</li>
      <li>if you succesfully installed monotouch, dont press start
        monodevelop, but go to the step 3.</li>
      <li>and restore original name (monodevelop -> MonoDevelop)<br>
      </li>
    </ol>
