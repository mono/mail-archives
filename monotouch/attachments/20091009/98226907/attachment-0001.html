<font face="Calibri, Verdana, Helvetica, Arial"><span style="font-size: 11pt;">I have created a rudimentary objection relational mapper (ORM) using sqlite3 with a specific goal of improving the MonoTouch development experience.  It’s called Catnap.  Find it here:<br>

<br>
<a href="http://code.google.com/p/catnap-orm/" target="_blank">http://code.google.com/p/catnap-orm/</a><br>
<br>
A few comments.  First, it’s very basic.  The immediate goal is to escape the mire of writing data access code and lots of dynamic SQL.  Second, it is definitely alpha.  I am only dipping my toes into MonoTouch development; indeed I have only yet deployed to the simulator.  Hence, Catnap has only been really tested in very basic scenarios.  I would love for folks to try it in their applications and give feedback or, better yet, contribute to its development.  It you are willing to roll up your sleeves, I feel like together we can make it  a very useful tool for MonoTouch development.  <br>

<br>
Here are some of the features:<br>
<br>
</span></font><ul><li><font face="Calibri, Verdana, Helvetica, Arial"><span style="font-size: 11pt;">Mapping is done via a fluent interface 
</span></font></li><li><font face="Calibri, Verdana, Helvetica, Arial"><span style="font-size: 11pt;">Mapped entities inherit from IEntity interface 
</span></font></li><li><font face="Calibri, Verdana, Helvetica, Arial"><span style="font-size: 11pt;">Mapped collections :<br>
</span></font><ul><li><font face="Calibri, Verdana, Helvetica, Arial"><span style="font-size: 11pt;">automatic loading - lazy or eager, specified per collection 
</span></font></li><li><font face="Calibri, Verdana, Helvetica, Arial"><span style="font-size: 11pt;">automatic cascading - saves/updates and/or deletes, specified per collection 
</span></font></li><li><font face="Calibri, Verdana, Helvetica, Arial"><span style="font-size: 11pt;">filtering - criteria specified per collection <br>
</span></font></li></ul></li><li><font face="Calibri, Verdana, Helvetica, Arial"><span style="font-size: 11pt;">Control connection and transaction boundaries via a basic UnitOfWork context 
</span></font></li><li><font size="2"><font face="Consolas, Courier New, Courier"><span style="font-size: 10pt;">Repository&lt;T&gt;</span></font></font><font face="Calibri, Verdana, Helvetica, Arial"><span style="font-size: 11pt;"> base class provides basic operations 
</span></font></li><li><font face="Calibri, Verdana, Helvetica, Arial"><span style="font-size: 11pt;">IFindSpec&lt;T&gt; provides means to abstract find queries 
</span></font></li><li><font face="Calibri, Verdana, Helvetica, Arial"><span style="font-size: 11pt;">DbCommandSpec provides a way to create and execute any parameterized query (similar to IDbCommand) 
</span></font></li><li><font face="Calibri, Verdana, Helvetica, Arial"><span style="font-size: 11pt;">Support for Linq predicates for finding and filtering collections (very limited at present) 
</span></font></li><li><font face="Calibri, Verdana, Helvetica, Arial"><span style="font-size: 11pt;">Database migration utility. Provides automatic deployment of database changes based on migrations (forward only) that you write 
</span></font></li><li><font face="Calibri, Verdana, Helvetica, Arial"><span style="font-size: 11pt;">No dependencies except sqlite3.dll <br>
</span></font></li></ul><font face="Calibri, Verdana, Helvetica, Arial"><span style="font-size: 11pt;"><br>
There is a link to a Google group on the code site.  Please chime in if you are trying it out.  Let us know how it’s going, and if you want to contribute.<br>
