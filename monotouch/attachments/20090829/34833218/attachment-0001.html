I have an objective-c iphone project that is statically linked to a
vanilla c++ shared library. I want to port it all to monotouch. Is
there anyway I can use the c++ library from monotouch c#? Is there any
sort of concept of p/invoke? Can I wrap this library the same way you
guys expose native libraries as &quot;managed&quot; libs (e.g. how you expose the
NS* foundation classes as managed classes?)<br>
<br>
</blockquote>
<br>
Yes pInvoke works, but beware that C++ is name-mangled, so you&#39;ll
likely want to put a C api aroudn your C++ classes.  What you do is:<br>
<br>
1&gt; Edit your project settings in MD to have extra mtouch flags and
add &quot;-lib /path/to/your/static/lib.a&quot; (you need to do this per
configuration and have a x86.a and arm.a)<br>
2&gt; Setup your dll imports as follows:<br>
<br>
[DllImport (&quot;__Internal&quot;)]<br>
static extern void foo ();<br>
<br>
Then everything should work<br>
