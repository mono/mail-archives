<div>In iOS4, the locationServicesEnabled method was changed from an instance to a class method (and the instance method was <a href="http://developer.apple.com/iphone/library/documentation/CoreLocation/Reference/CLLocationManager_Class/DeprecationAppendix/AppendixADeprecatedAPI.html">deprecated</a>).</div>
<div><br></div><div>AFAICT, this means in iOS4 and above we are encouraged to use the class method, but the instance method still exists for backward compatibility. This seems to be true since my older apps (compiled against 3.2 and MT 2.x) still work on iPad and iOS4.</div>
<div><br></div><div>However, MT3.x has totally <a href="http://monotouch.net/Releases/API_Changes/2.0_to_3.0">removed the instance method</a>. What this seems to mean is:</div><div><br></div><div>- an app compiled with MT2 using an (<i>instance of CLLocationManager</i>).LocationServicesEnabled will work on iOS3 and iOS4 (because even though it has been deprecated, it still exists)</div>
<div>- if i update to MT3, the project will <u>not compile</u> unless i change the instance method to a class method call CLLocationManager.LocationServicesEnabled (which is bound to the obj-c static method, which doesn&#39;t exist in OS3)</div>
<div>- once the code is updated for MT3, it will run on iOS4 BUT on iOS3 (older phones+iPad) this error occurs:</div><div><font class="Apple-style-span" face="&#39;courier new&#39;, monospace">Objective-C exception thrown.  Name: NSInvalidArgumentException Reason: *** +[CLLocationManager locationServicesEnabled]: unrecognized selector sent to class 0xe67e2ca</font></div>
