&gt; BTW, if you are thinking &quot;thats very different to .NET!&quot; - you are right.<br>
&gt;<br>
&gt; Obj-C is not a method-call model. It&#39;s a message passing model. Think of it like this:<br>
&gt;<br>
&gt; .NET:<br>
&gt;<br>
&gt; foo.DoSomething();<br>
&gt;<br>
&gt; // COMPILER (or linker, or runtime) error if the version of the DLL which has foo in it, doesn&#39;t have DoSomething()<br>
&gt;<br>
&gt; in Obj-C, you are not calling a method, you are sending a message, so<br>
&gt;<br>
&gt; [foo doSomething];<br>
&gt;<br>
&gt; sends a message to foo, tells it to run doSomething. If it doesn&#39;t have a doSomething, you get an exception (kinda) or it calls the &quot;missing method&quot; routine. I think. But the linker doesn&#39;t care, nor does the compiler (tho it checks). It&#39;s all dynamic at runtime.<br>
