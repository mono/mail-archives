I agree pretty much with what Nic has said.

I've been using ASP.NET MVC 4 Web API (since beta, but now in RC) to provide some REST services that I call from my Monotouch app.  I've used POCOs within the Monotouch app (could have shared a library, but wasn't much point for me) and the excellent RestSharp to call the services.

I then used OAuth 2 to secure my services, I've pretty much baked my own implementation of this following various guides (such as <a href="http://codebetter.com/howarddierking/2011/10/11/oauth-2-0-in-web-api/" target="_top" rel="nofollow" link="external">http://codebetter.com/howarddierking/2011/10/11/oauth-2-0-in-web-api/</a>).  However, if I was starting from scratch now I'd seriously look at this: <a href="http://identityserver.codeplex.com/" target="_top" rel="nofollow" link="external">http://identityserver.codeplex.com/</a>

RestSharp has built a built in deserializer to populate your POCOs and use within your app.  Only fun and game I've had is with DateTime formats.  But you can set the default Json deserializer in your web api to suit and make sure you stick with a consistent format.
