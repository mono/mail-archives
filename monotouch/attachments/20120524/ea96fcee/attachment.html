<div>Hi,<br></div><div><br></div><div>The underlying ObjC objects are refcounted, so they're unreffed when the managed GC kicks in and frees the managed peer.</div><div><br></div><div>In particular you can not assume the ObjC object will be freed, it will only be unreffed/released.</div>
<div><br></div><div>In general you do not have to worry about destructors when writing bindings, at least not for the generated code. For your own custom code in the bindings (if you have any) the same rules applies as for everyone else (disposing objects when done with them is good, etc.)</div>
