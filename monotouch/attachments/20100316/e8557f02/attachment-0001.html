Don't embed as resource unless you're actually using the image as a<BR>
.NET embedded assembly resource, or you'll simply bloat the app size.<BR>
<BR>
* The &quot;Content&quot; build action means that the image file gets included<BR>
in the app bundle.<BR>
* The &quot;Embed as resource&quot; build action means that the image gets<BR>
embedded in the assembly (dll).<BR>
* The &quot;Copy to output&quot; and &quot;include in deploy&quot; options are irrelevant<BR>
to iPhone apps (though copy to output will waste time and space<BR>
copying files to the build directory rather than the app bundle).<BR>
<BR>
If including images in your app bundle causes hangs, please file a bug<BR>
report with a test case.<BR>
