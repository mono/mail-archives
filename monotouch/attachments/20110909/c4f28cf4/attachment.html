<p class="MsoNormal"><span style="font-size:11.0pt;font-family:&quot;Calibri&quot;,&quot;sans-serif&quot;;color:#1F497D">Thank you to everyone for the responses and help. Pete’s suggestion triggered a thought while rereading Rolf suggestion. I hadn’t specified a custom class for
 the custom cell defined in the XIB. I defined a custom class for the cell, gave it the outlet for the text field and linked it, changed the outlet type on the controller to match the custom cell type, then implemented the partial class with constructors for
 the custom cell type. That and using a full UITextFieldDelegate implementation and all is right again. The debugger no longer blows up when first responder changes to or from the UITextField in the UITableViewCell.<o:p></o:p></span></p>
