    My two cents ...
   </span>
  </p>
  <p style="margin: 0;">&#160;</p>
  <p style="margin: 0;">
   <span>Would love to see close integration between Objective-C and Monotouch. Would like to pick up a Objective-C library project and use it from Monotouch and the other way round without all the b-touch stuff. Maybe Mono Develop could automate all that. I think MonoDroid does something like that with java classes. &#160;</span>
  </p>
  <p style="margin: 0;">&#160;</p>
  <p style="margin: 0;">One of the missing areas is third party controls - It would be awesome if folks like Telerik, Infragistics, etc. would write controls that Enterprise developers could buy that are supported. Telerik&#39;s feedback on this was their controls were XAML based and Winforms based and iOS API was very different. If we could have a subset of XAML support in XTouch and XDroid, this would greatly help vendors and others to port their controls. Yeah, I know this is not native UI but IT managers just want working cross platform apps :-(</p>
  <p style="margin: 0;">&#160;</p>
  <p style="margin: 0;">Get Google to add XDroid runtime to Android 4 distribution. This would reduce the size of final app. &#160;&#160;</p>
  <p style="margin: 0;">&#160;</p>
  <p style="margin: 0;">WCF APIs - Parts of WCF and Windows Identity APIs are not implemented. It would make sense to invest effort here to make this more enterprise friendly. I would expect the Enterprise/IT .NET developers is where you can charge more and make more money. And we want you guys to make more money :-)&#160; &#160;</p>
  <p style="margin: 0;">&#160;</p>
  <p style="margin: 0;">Pricing -&#160; Telerik&#39;s pricing model is a good model. Charge more for extra support and package products together. It&#39;s easier to get one approval to buy $999 product, rather than 2-3 approvals.&#160;</p>
  <p style="margin: 0;">&#160;</p>
  <p style="margin: 0;">Activation - I understand the use of activation but it&#39;s bad choice for the user. None of the .NET component vendors do this.&#160; &#160; &#160;</p>
  <p style="margin: 0;">&#160;</p>
  <p style="margin: 0;">Miguel/Nat - It would make sense to capture this feedback into something like http://uservoice.com and let people vote on it.</p>
  <p style="margin: 0;">&#160;</p>
