<FONT FACE="Calibri, Verdana, Helvetica, Arial"><SPAN STYLE='font-size:13pt'>My app starts by pushing a table view controller onto a nav controller. &nbsp;As the user navigates forward and back the app pushes and pops instances of the same type of table view controller, each time passing in a parameter that changes what data is displayed. &nbsp;Pretty basic stuff, and it works fine.<BR>
<BR>
I want let the user jump forward and back in the stack. &nbsp;So I put a &#8220;JumpTo&#8221; method in the nav controller. &nbsp;Jumping backward works fine. &nbsp;I simply find the target controller and pop to it. &nbsp;The problem is jumping forward. &nbsp;Here&#8217;s the relevant code snip:<BR>
<BR>
while (currentTimeFrame != timeFrameToJumpTo) <BR>
{<BR>
&nbsp;&nbsp;&nbsp;&nbsp;var nextTimeframe = currentTimeFrame.Next ();<BR>
&nbsp;&nbsp;&nbsp;&nbsp;var controller = new ToDoListViewController (nextTimeframe);<BR>
&nbsp;&nbsp;&nbsp;&nbsp;PushViewController (controller, true);<BR>
&nbsp;&nbsp;&nbsp;&nbsp;currentTimeFrame = nextTimeframe;<BR>
}<BR>
<BR>
The logic is straightforward and seems to work -- it loops though, pushing new controllers until it reaches the target. &nbsp;At that point, the view displays the correct title, telling me it reached the correct target. &nbsp;I&#8217;ve also verified that all intermediate controllers are on the stack. &nbsp;&nbsp;However, the table data is not correct. &nbsp;It&#8217;s always the data from the first controller pushed. &nbsp;So let&#8217;s say I push forward 3. &nbsp;Here&#8217;s what happens:<BR>
<BR>
Controller 0 is the top<BR>
Push controller 1<BR>
Push controller 2<BR>
Push controller 3<BR>
Apply the controller 1&#8217;s DataSource to controller 3&#8217;s TableView.<BR>
<BR>
When each controller is created, I can see the DataSource for its TableView is set correctly. &nbsp;But only one data source is ever invoked &#8212; the one for the &nbsp;first controller pushed which is applied to the last controller pushed.<BR>
<BR>
I&#8217;m sure there is a stupid simple fix for this. &nbsp;Any ideas?<BR>
