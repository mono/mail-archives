> I have an existing graphics library written in C# that also utilizes
> DllImport gdi32.dll. Is it possible to port this to MonoTouch and Mono 
for
> Android since it relies on gdi32.dll?

You can't use gdi32.dll. Whether you can port it by rewriting the 
implementation to use iOS or Android graphics libraries depends on how 
your library's API was written. Does it abstract away potential 
differences between the underlying APIs or is it just a thin wrapper on 
GDI? A thin wrapper is going to have to be rewritten. An abstraction may 
be entirely reusable by just changing the underlying implementation.
