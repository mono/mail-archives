> A couple things to note. First, I ended up using the Alpstein fork of the
> Route-Me project for my work, which is the fork that MapBox is based on.
> The performance head to head for what I was doing was better on the
> Alpstein fork. The other thing is that the bindings don't cover 100% of the
> API but there's a good chunk there and because Route-Me is the basis for
> MapBox it shares a core portion of the API and should basically work for
> MapBox too (and other Route-Me based libraries). The thing you'll have to
> do if you want to use mapbox is make sure to build the MapBox dll into
> libMapView.a and replace the one that's there.
