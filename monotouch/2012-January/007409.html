I have a Silverlight /WCF service that uses DataContractJsonSerializer to
move classes back and forth. Some of the classes have an
ObservableCollection of items. I use ObservableCollection to get databinding
goodness in Silverlight. Now I have added REST capability to that WCF
service so that it can return json data in simply REST calls. 

In my Monotouch app I do a HttpWebRequest to get the data and use
DataContractJsonSerializer to deserialise it into the same business objects
I use in my Silverlight application. All look very cool. Except (had to be
one or I wouldn't be writing a post!), if the class contains an
ObservableCollection. In that case I get a 'Deserialization has failed'
error. Digging in a bit more it tells me 'failed to convert parameters'.
Silverlight can deserialize into classes containing ObservableCollections
using DataContractJsonSerializer. If I change the MonoTouch version of the
class to use List instead of ObservableCollection it works ok. I don't
really want to have special MonoTouch versions of the class though.
Shouldn't ObservableCollection work?
