Hi Alec,

I've just fixed this in git master, so the next release of MonoDevelop
should resolve this issue. What I did was to make it name the files as:

This_Is_Another_TestViewController_iPad.xib****

This_Is_Another_TestViewController.iPhone.xib


Jeff

On Tue, Jan 3, 2012 at 5:14 PM, Jeff Stedfast <jeff at xamarin.com> wrote:

> Hi Alec,
>
> This probably continues to work because the xib files using _'s are likely
> left behind in the installed app bundle (for simulator, we just copy over
> new files but don't remove stale files, if I remember correctly).
>
> As far as the rest, I'm a little fuzzy on the details, but I thought I
> remember hearing a discussion about how .'s were illegal in app names or
> the executable name or some such.
>
> However, what you encountered in here is very clearly a bug in
> MonoDevelop. At the very least, the project templates should be using the
> same sanitized string for the xib filenames as it does for the strings it
> uses in the code itself.
>
> I'll try to fix this in a minute. In the meantime, I guess avoid using .'s
> in your project names.
>
> Jeff
