Hi Rick,

 

first of all - this list will die you should use the forums http://forums.monotouch.net/

An imageview (UIImageView) is something like the Image control for Windows.

So this guy holds ONE image and can be placed (sized, hidden, shown...) on you UIView.

 

First approach (not the best I guess) - use multiple UIImageViews and place them individually.

Since we have a Z-Order you can place them even on top of each other.

 

The other approach - create an image by drawing various elements on if (other images for an example)

and place that resulting image in an UIImageView.

It's a bit like drawing in windows (forms) applications. You create a device context, and use functions of

this context to draw on it.

The class for such a "drawable image" is CGBitmapContext

The function to draw an image on it is DrawImage
