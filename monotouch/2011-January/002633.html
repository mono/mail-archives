MonoDevelop does implement Unicode text rendering and support for
input methods, but these do not work correctly on Mac due to
Mac-specific limitations in the cross-platform native libraries we use
for the GUI (GTK+) and text rendering (Pango).

Firstly, Pango does not support font fallbacks on Mac, so if the
current font does not include needed glyphs, it will not fallback to
other fonts for those glyphs
(https://bugzilla.gnome.org/show_bug.cgi?id=608929). I believe you can
find guides on the web that explain how to use configuration files to
force Pango to use different fonts to work around this issue (e.g.
http://kki-zakki.blogspot.com/2008/12/monodevelop-20-alphamac-os-x.html).
Unfortunately it's not a general solution that we could ship for all
our users.

The second problem is that GTK for the Mac does not have an
implementation of an input module to map the Mac's input method system
into its own (https://bugzilla.gnome.org/show_bug.cgi?id=504602), so
even with the aforementioned workarounds you cannot easily input
non-Western chars except via pasting.

Unfortunately we don't have any ETA to fix these problems since
they're difficult to solve, and are in the upstream native libraries
that we consume, but it is something we're tracking.
