OK. So you have:

your.dll (which is your app)
monotouch.dll

and you want to obfuscate both, and roll it out to the iOS app store
like normal. Correct?

You dont need to.

Whats in the .app file is a binary (machine code) of all the dll's
(which can't be decompiled, mostly), inculding all the .NET framework,
monotouch, your stuff, and any .a (obj-c) stuff you put in there too.

There is also a few .dll's in the .app, but those are only meta data,
no actual code is in there. No one could get your code from this. It's
for debugging only.

Does that help?
