Problem:

1) To consume WCF services, one must use SLSvcUtil.exe.EXE that comes with
the Silverlight SDK, because iPhone/Monotouch does not support dynamic code
generation that the standard .Net SDK SvcUtil.EXE uses.

2) Silverlight does not support System.Data.Dataset, therefore the code
generated by SLSvcUtil.EXE does not support datasets, even though
Mono/Monotouch DOES support System.Data.Dataset.

I guess the solution might be to use OData, which is supposedly The New Way
of Doing Things (tm).  I haven't looked in detail into that yet, and am not
sure if that is possible in Monotouch (please let me know if you are
familiar).

Ideally I would want to carry on using System.Data.Dataset, as converting to
something else would require refactoring a lot of code, and I don't have the
time budgeted to do that.

Help!  Any advice is appreciated.
