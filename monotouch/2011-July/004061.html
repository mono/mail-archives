This was a bug with the Mono for Android addin from the addin
repository (installed via the addin manager) for MonoDevelop 2.6b2.
Unfortunately, if you installed it for MD 2.6 b2, it overrides the
newer/later version that was shipped with MD 2.6 b3. There's now a
newer version of the Mono for Android addin that you can install from
the addin repository that does not have this issue.

- Michael
