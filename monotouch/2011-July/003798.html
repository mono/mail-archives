The Entry attribute has a Keyboard property, so you should be able to do

[Entry(Keyboard=UIKeyboardType.DecimalType)]

for a number pad. The autocorrect and autocaps isn't exposed - you
would need to do some changes in Reflect.cs (see what
AlignmentAttribute does, and is used in BindingContext, you wanna do
the same thing, but for the other properties)

I DO have a very old version of MT.D as my base, so it may be in a
more recent version on github (I made some changes and never
bothered/managed to merge into what Miguel has updated it to, tho to
ne honest, I've not had any bugs either!)
