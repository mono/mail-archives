> If you're running an older version of MD, you should upgrade. If you're
> running the same version I am but are still having this problem, and if you
> really want to get it working right now, you can.
> 
> The quick, dirty way to paste that particular bit of text into MD without it
> crashing is to, in TextEdit, delete the quotes and type new ones in (they
> should be plain old ASCII code 34 quotes rather than those fancy unicode
> quotes - basically, just replace the old quotes with quotes as you'd normally
> type them). However, that's a pain. You could instead do a global search and
> replace (for each quote - the opening and closing UTF-8 quotes are different),
> but if there are other problematic characters in the text you're pasting,
> you're still going to have problems.
> 
> Someone else probably has a much better understanding of how to solve this
> problem than I do - but my guess is you need an easy way to replace the
> characters in the code you're copying with their ASCII equivalents. I think. I
> might be way off. That seems to handle the problem of the quotes, and it might
> be that the quotes are the only characters that are going to give you trouble,
> so maybe doing the global search and replace is an acceptable, if suboptimal,
> way to avoid having to retype all the sample code.
> 
> Sorry I can't be more help. But it's a start... I'd be interested in a tool
> that did this if anybody knows of one. I feel like I've dealt with something
> similar before, but it's been a while...
