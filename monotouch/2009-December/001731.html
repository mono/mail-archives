 I didn't make any changes to the project template.

AND - I use the Elements PointF SizeF a lot of time in code.

So I guess the linker wouldn't remove these structs.

 

Last not least - I use this "Preserve" setting just to ensure everything is there.

I guess it would make no difference if I set this or not.

 

Anyhow - I have some simple steps to reproduce the issue.

Create a new iPhone project.

In MainWindow add a view and therein a TextViewer.

Create an outlet tvCheck for the TextViewer.

 

Then add the following code to Main.cs

 

public override bool FinishedLaunching (UIApplication app, NSDictionary options) {

// If you have defined a view, add it here:

// window.AddSubview (navigationController.View);

tvCheck.Text = SerializeIt ();

window.MakeKeyAndVisible ();



return true;

}

private string SerializeIt () {

StringWriter sW = new StringWriter ();

XmlSerializer xS = new XmlSerializer (typeof(TestData));

TestData tD = new TestData {

Name = "One",

Number = 22,

Size = new SizeF (1, 2)

};

xS.Serialize (sW, tD);

return (sW.ToString ());

}

public class TestData {

public string Name { get; set; }

public SizeF Size { get; set; }

public int Number { get; set; }

public TestData () {

}

}

 

When I run this on the simulator (no matter if debug or release) the member Size is correctly serialized.

If I run it on the iPhone only an empty node is serialized like this:

<Size />

 

I didn't change any setting at the new projcet - just used it as it is.

> Could it be that you're using the linker differently between the
> simulator and the device build?
> 
> For instance, you could be building for the simulator with
> linksdkonly, which wouldn't try to reduce the size of your
> application, but only the size of the class libraries. And for the
> device, you would be fully linking everything, hence the difference.
> 
> You can use a [Preserve (AllMembers = true)] attribute on the type you
> want to serialize, or switch to linksdkonly for the device as well.
