I agree virtual machine as compile environment is useful. But I think
graphical environment should be to run test.

Recently, I keep to use the MODEL-VIEW-VIEWMODEL design pattern to
make UI thin wrapper.  Using that design pattern, I can run almost all
test cases (except for think ui wrapper) on windows (using nunit). I
only use monodevelop to build and ui test.

So, I propose to make a solution to have a ui project and a logic
layer project, and make a (unit) test environment for the latter one
run on windows.

Thin ui wrapper may be useful to release an application for
multiplattform (windows or android (it does not support c# now...))
