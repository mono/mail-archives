Hello Shawn,

> One point of note - it is recommended in .NET to wrap unmanaged  
> handles inside a SafeHandle derived object. I think Mono has yet to  
> bring SafeHandle to full parity with .NET's implementation (e.g.  
> ReliabilityContracts), but it still offers certain guarantees and  
> usually eliminates the need to add finalizers to your types (a  
> worthwhile goal in itself.)

Support for ReliabilityContracts does not really matter on the  
iPhone;   It is designed for cases where Mono is embedded into a  
critical application, like SQL server and that scenario is just not  
important in this case (and Mono is not able to portably across all  
systems with the situations where it would matter).

So as far as SafeHandle could be the right API to support, it could be  
used in this scenario.

Miguel.
