Hi,
 
I want to fix (not beeing customizable) some items in a UITabBarController.
For this I found the property CustomizableViewControllers.
 
But this array holds UINavigationController items while NavigationControlers hold UIViewController items.
 
I guess this is a type missmatch because:
Since I have UIViewControllers (and not UINavigationControllers) I decided to remove (just to test)
the first Item of CustomizableViewControllers by placing all (but the first) items in a new array and
then assign this array to CustomizableViewControllers.
 
Looks like this:
 
UINavigationController [] uC = new UINavigationController[5];
int nCnt=tbController.CustomizableViewControllers.Length;
for(int nX=1;nX<nCnt;nX++) {
uC[nX-1]=tbController.CustomizableViewControllers[nX];   //HERE IT FAILS
}
tbController.CustomizableViewControllers = uC;
 
When I run this code I get a cast exception. So I guess the array holds (as expected) UIViewController items.
 
Even
object o = tbController.CustomizableViewControllers[0];
fails with a cast exception at get_CustomizableViewControllers.
 
Either I'm doing something totally wrong or the declaration of CustomizableViewControllers is not correct.
I would consider this as a bug - and at the moment I see no way to "fix" tabbaritems.
