Hello,

Today I have added WCF "event-based" client proxy support in our svcutil,
which is primarily for moonlight support improvements.

This somehow raised problem on MonoTouch sync client generation. Here
I copied my code comment in a svcutil source:

// FIXME: since ServiceContractGenerator at
// least for now generates only either sync or
// async stuff, and since Moonlight-compatible
// profile now emits event-based operations,
// it became impossible to generate sync methods
// in MonoTouch profile. It might be able to get
// fixed, but might be not.

As said above, I'm not sure if it's fixable. Sorry to say, MT may lose
sync operation support in the future versions. It is based on Moonlight
profile anyways.
