A coding pattern two MonoTouch.Info samples caused me some problems, which I have now solved.
Here is the problematic code pattern:
public class Abc : UITableViewController{	UITableView _tableView;
	public override void ViewDidLoad ()	{		base.ViewDidLoad ();
		_tableView = new UITableView()		{			Source = new TableSourceInnerClass(),			AutoresizingMask = UIViewAutoresizing.FlexibleHeight | UIViewAutoresizing.FlexibleWidth,			BackgroundColor = UIColor.White		};		_tableView.SizeToFit();		_tableView.Frame = new RectangleF ( 0, 0, this.View.Frame.Width, this.View.Frame.Height );		this.View.AddSubview( _tableView );
The result is I think, two full screen tables views overlaid which then causes the following problems:
(1) If the table view has enough data to scroll a duplicate set of horizontal cell separator lines will be seen, one set is fixed and the other scrolls with the table view.
(2) Initializing the UITableViewController base() constructor with Grouped table style won't have any affect because the style is applied to the 1st table that has no data.
(3) Overriding UITableViewController.ViewDidAppear() to preselect a table row will cause a runtime exception because the 2nd table view won't yet be populated with data.
The following simplified code avoids these problems, I simply use the UITableView created by the UITableViewController and point this view at my TableSource.
public override void ViewDidLoad (){	base.ViewDidLoad ();	this.TableView.Source = new TableSourceInnerClass( _wf );}
Is there a use-case for the more complex code pattern in the sample Apps?
