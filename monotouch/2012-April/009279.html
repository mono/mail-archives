Hello René,

I've seen this once. The issue (not an easy one to diagnose) was that the
application ran out of memory inside the Mono runtime [1]. It could be
different, since many (uncommon but possible) errors will lead to the same
exception, but that's the most likely condition I can think of.

If this the case then using the thread pool would have the advantage of
reusing the same threads (and memory) so this can help (as much as anything
that can lower the memory requirements).

Sebastien

[1] https://github.com/mono/mono/blob/master/mono/metadata/threads.c#L729
