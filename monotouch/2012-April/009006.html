Hello Dean,

By default the simulator use "Don't link" while device builds use "Link SDK" and both should work for your own serializable types.

However if you changed this to "Link all" then it's possible that the linker will remove unused members of the Response class. You can mark them with a [Preserve] attribute (and continue linking everything), use --linkskip on the assembly that defines this type, or go back to use "Link SDK".

Sebastien
