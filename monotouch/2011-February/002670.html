The export line for onSuccess is slightly wrong.  Do not forget the trailing ":". 
[Export("onSucesss:result:")]

Try this extra arguments line:
-v -v -v -gcc_flags "-L${ProjectDir} -lODSCertification -lOpenAPI
-framework CFNetwork -framework MobileCoreServices -framework CoreTelephony
-force_load ${ProjectDir}/OpenAPI.a"

The "-v -v -v" will help isolate which frameworks you need to add.  Check the build output.

For the libz.1.2.3.dylib library you will most likely need to manually call dlopen.

Good luck.

  I have to use an obj-c library in my MonoTouch project. In the obj-c project I can use the library as:

  - (void) viewDidLoad {
      ...
      OpenAPI oa = [[OpenAPI alloc] init];
      [oa.startWithDelegate:self];
      ...
  }

  // @protocol OpenAPIDelegate
  // @required
  -(void)onSuccess:(NSInteger)optType result:(NSDictionary*)paraDic {
      ...
  }

  So in my MonoTouch project I defined the bindings for the OpenAPI class and the protocol like below (anything wrong with that?):

  [BaseType(typeof(NSObject))]
  interface OpenApiDelegate {
      [Export("onSuccess:result")]
      [Abstract]
      void OnSuccess(int result, NSDictionary paraDic);
  }

  [BaseType(typeof(NSObject))]
  [Model]
  interface OpenAPI {
      [Export("startWithDelegate:")]
      void Start(OpenApiDelegate d);
  }

  In the assembly generated by btouch, I find an OpenAPI class with an Start method which throws You_Should_Not_Call_base_In_This_Method exception. And when I try to use the assembly in my project is does throw. I think it’s the problem that I didn’t follow the instruction at http://wiki.monotouch.net/HowTo/Interop/Consuming_3rd_Party_ObjC_Libraries, which tells me I should use the following arguments for compiling:

  -gcc_flags "-L${ProjectDir} -lFoo -framework CFNetwork -force_load ${ProjectDir}/libFoo.a"

  But I did not quite catch that – can someone wrote the argument for me in my scenarios? I have two “.a” files (libODSCertification.a and OpenAPI.a) put in my “${ProjectDir}/lib” folder, and the document tell me I need the following frameworks:

  - libz.1.2.3.dylib

  - CFNetwork.framework
  - CoreTelephony.framework
  - UIKit.framework
  - MessageUI.framework
  - Foundation.framework
