David,

I haven't gotten a chance to dig into MonoTouch yet (hopefully soon)
but you could always go the composition route; if it's not too much
work.  You'd create your wrapper class and provide all the public
methods that would normally remain with inheritance.  You could then
implement IDisposable normally and inside your Dispose() method call
the Dispose() method on the object you're wrapping.  An example would
be something like:

// This is the signature of the built-in class:
public class FrameworkClass
{
  public void SomeMethod();
  public void Dispose();
}

// This is your composition class:
public class MyClass : IDisposable
{
  FrameworkClass _me;
  public MyClass(FrameworkClass frameworkClass)
  {
    _me = frameworkClass;
  }

  public void SomeMethod()
  {
    _me.SomeMethod();
  }

  public void Dispose()
  {
    _me.Dispose();
  }
}

Mirror every public method and property in the wrapper class.

-Abe
