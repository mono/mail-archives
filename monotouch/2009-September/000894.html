I don't think anyone is downplaying how far MD has come especially on  
OS X. VS just has benefitted greatly from 3rd Party plugins for me  
most prominently coderush / refactor pro and resharper. But to your  
questions:

> What would be needed to elevate MD's code navigation capabilities from
> "most basic"?
>

For me, I've gotten used to navigating through the code with  
coderush ... the ability to jump to any symbol with the press of a  
button ... to cycle through all occurrences of a variable, or to leave  
bread crumbs that hold the current position and can be collected at  
the press of a button. It really makes navigation a dream ..

>> no refactoring capabilities,
>
> Not true. MD does have refactoring capabilities.
>

Al true ... but maybe not at the level that tools like resharper or  
refactor pro do ...

Again I'm not tying to argue that MD should have those capabilities.  
I'm perfectly happy with using those tools in VS, commit changes to  
source control and then check out things on OS X and make it work.

To get to the question of pricing ... I'd be interested on releasing  
free applications to the app store ... for that I'd be willing to pay  
$100 - $200 for a personal license ... but with $400 I'm out-priced as  
well ... maybe you could add a category for the hobbyist ... that just  
include the command line tools ... for a lesser price ...
