>> Mike,

>> Just to give you a bit of insight on what we are prototyping:
>>
>> - MVP based architecture, to follow "separation of concerns" as  
>> much as possible. So that the "Presenter" does not need to know any  
>> Cocoa, and can drive any UI interface.
>>
>> - Our model we use System.Xml and XML Serialization to save data.  
>> This is supported *today* on MonoTouch. Great job by the team  
>> identifying the need and implementing System.Xml from the .NET 2.0  
>> class lib. We also use things such as XmlReader.
>>
>> - We do HTTP requests. You can do whatever you wish. WebClient,  
>> WebRequest, WebResponse are available etc.
>>
>> Of course there may be parts where you have to be a bit platform  
>> specific. For example you may want a path to be different for  
>> saving than on a PC or Windows Mobile device. For this we are  
>> adding a mechanism for detecting what platform we are on. We try to  
>> minimize this as much as possible.
>>
>> Obviously you don't have to make the same choices, however just  
>> wanted to give you an idea what is possible.
>>
>> Thanks and take care,
>> Simon
