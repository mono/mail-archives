Thanks Joel for your help...that did it!

One thing though, the Stanford pdf notes state that for custom drawing
you should override drawRect (without calling it directly).

>From http://www.stanford.edu/class/cs193p/downloads/05-Views.pdf:

Override - drawRect: to draw a custom view
■rect argument is area to draw

Here's my (now working) class as it stands now:

	public partial class PolygonView : MonoTouch.UIKit.UIView
	{
		public PolygonView(IntPtr handle) : base(handle)
		{
		}
		
		public override void Draw (RectangleF rect)
		{
			MonoTouch.CoreGraphics.CGContext context = UIGraphics.GetCurrentContext();

			context.SetLineWidth(2.0f);
			context.SetStrokeColorWithColor(new CGColor(0f, 0f, 0f));
			context.MoveTo(10,10);
			
			context.AddLines(new PointF[] { new PointF(60, 60), new PointF(100, 100) });
			
			context.StrokePath();
			
			SetNeedsDisplay();
		}
	}

Maybe I should be drawing elsewhere?  Yet, this seems to be what is suggested.

Thanks for your help.

On Thu, Sep 3, 2009 at 1:23 AM, Joel Dentici<joeldentici at cox.net> wrote:
> had the same problem. first, you need to call setNeedsDisplay() to update
> your UIView, not Draw(). Calling Draw directly will cause invalid contexts
> to be sent to your method. Apple designed it so the system would choose when
> to redraw the view, not the code (similar to calling form.Invalidate() in
> winforms to cause it to redraw). Second, make sure in your constructor that
> you put this:
> public UIViewSubClass(IntPtr handle) : base(handle)
> {
> }
