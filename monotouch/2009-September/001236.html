Hello folks,

      I have uploaded the System.Web.Services.dll assembly that you  
can use with your MonoTouch here:

	http://tirania.org/tmp/System.Web.Services.dll

      To use it, reference this assembly (right click on References,  
click "Edit References" and in .NET assemblies add this reference).

      Then you need to create a Web Reference, in your project, right  
click and select "Add Web Reference", in the entry line, enter the URL  
to a WSDL document for your web service and you should end up with  
your web service defined.

      For example, I used an internal web service from XSP2 like this  
on OSX:

	cd /cvs/xsp/test/1/1.webservice
	xsp2

     To visit the web service, you can make Safari point to:

	http://localhost:8080/TestService.asmx

     Now, you need to get the link to the WSDL description, I used the  
url there and I added this reference in MonoDevelop:

	http://localhost:8080/TestService.asmx?wsdl=0

      Then you write:

	var service = new TestService ();
	Console.WriteLine (service.Add (10, 20));

     And off you go.

     Remember: you should not do any sync calls at startup that might  
delay the main Application Delegate method, or the iPhone watchdog  
will kill your process if you do not respond in under 10 seconds.

     The source code against Mono 2.6 is here:

	http://tirania.org/tmp/webservices.patch

Miguel.
