Hi,

Like Lee says http://www.servicestack.net is an open-source
web-services stack that provides different ways to call your
web-service (i.e. SOAP, REST+XML, REST+JSON, etc).

A live preview of it running on Linux/Mono can be seen here:
http://www.servicestack.net/ServiceStack.Examples.Clients/

Note: The 'show' links will show the code it takes to build a web
service once your setup.

The source code of the example project is available here:
http://code.google.com/p/servicestack/source/browse/#svn/trunk/ServiceStack.Examples

An example of hosting the webservice as a 'simple web application'
that you can use under mod_mono or fastcgi (or windows/IIS if you roll
that way ;) is here:
http://code.google.com/p/servicestack/source/browse/trunk/ServiceStack.Examples/#ServiceStack.Examples/ServiceStack.Examples.Host.Web

You can also host your web-services as a stand-alone Console
application which uses HttpListener (and does not require a web
server), an example of which is available here:
http://code.google.com/p/servicestack/source/browse/trunk/ServiceStack.Examples/#ServiceStack.Examples/ServiceStack.Examples.Host.Console


Once you have your web service up and running, you can use
ready-to-use 'service clients' on your client (in this case MonoTouch)
that re-use the 'request' and 'response' dto's that you used to define
your service like this:

using (var serviceClient = new
XmlServiceClient("http://www.servicestack.net/ServiceStack.Examples.Host.Web/"))
{
    var request = new GetFactorial { ForNumber = 3 };
    var response = serviceClient.Send<GetFactorialResponse>(request);
}

http://code.google.com/p/servicestack/source/browse/trunk/ExampleProjects/ServiceStack.Sakila/ServiceStack.UsageExamples/UsingServiceClients.cs


If you don't control the web service then the service client will send
the clean XML version of your request DataContract (i.e.
'GetFactorial' in the above example)  as a HTTP POST.
So as long as your service is pure REST+XML or REST+JSON it should work.

The examples below shows an example of the 'raw xml' that gets sent on the wire:
http://code.google.com/p/servicestack/source/browse/trunk/ExampleProjects/ServiceStack.Sakila/ServiceStack.UsageExamples/UsingRawHttpClient.cs

If there is enough interest in this I may write up a little tutorial
on this if I can get some free time this weekend.

Cheers,
Demis
