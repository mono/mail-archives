Safari renders web-based <select> drop-down lists as UIPickerView, so  
I think that's the best match.

Unlike this sample (which shows the wheels permanently) I guess you  
want an input field that shows the UIPickerView in place of the  
keyboard.

http://conceptdev.blogspot.com/2009/08/monotouch-restaurant-bill-splitter.html

I haven't experimented with show/hide on input 'focus' yet but I'm  
guessing (?) that's how it's done ? You would probably want to go all  
the way with input-navigating Next/Prev buttons attached to both the  
keyboard and above the UIPickerView (again, see how web form filling  
works in Safari)
