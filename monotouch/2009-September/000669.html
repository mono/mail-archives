If it helps, I am currently actively maintaining a mature open-source
web-service stack that currently works in mono:

- http://code.google.com/p/servicestack/ (code project home)
- http://www.servicestack.net/ServiceStack.Examples.Clients/ (Live
example of it running on Mono)
- http://www.servicestack.net/mythz_blog/?p=42 (my blog post providing
a brief summary of it)

it's API is clean, You use plain old DTO's to define your operations
(i.e. web services) and types and the framework takes care of exposing
them on SOAP 1.x, Xml (+REST) and Json (+REST) endpoints.

It only requires WCF for the SOAP 1.1 and SOAP 1.2 endpoints.

For the other XML and JSON endpoints it just relies on the .NET/Mono
providers XML and JSON DataContractSerializer's.
JSON works flawlessly on mono, Xml support is as good as Mono's
DataContractSerializer (which hasn't worked well for complex types in
the past, which the author hopes will be improved due to renewed
interest in Silverlight/Moonlight development)

It can be hosted on a website (using HttpHandlers), or in a windows
service/Console application (using HttpListener)

We currently use it as our web services framework to define 100+
webservices with it at work and we have developed clients in all of
the following technologies:
- flash(air)
- silverlight/wpf/winforms
- ajax
- QT
All clients use the same XML+REST or JSON+REST endpoints.

- Demis
