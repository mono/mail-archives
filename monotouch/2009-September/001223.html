> When interacting with the cocoa touch mono classes am I able to use 
> regular .net string or do I have to use NSString? 
> Will i be able to use the regular net libraries for flat file access, 
> sockets, HTTP, XML parsing, dates?

You will use "string", the runtime takes care of mapping strings to
NSStrings for pretty much every API that we expose, but in cases where
you need to manually invoke Objective-C code, we also provide an
NSString class that you can use (and there are also implicit conversions
back and forth).

As for the rest:

	* Yes, you can use the File APIs, but remember that you are
	  sandboxed, so you cant really access data files that belong
	  to other applications.

	* Yes, you can use System.Net.Sockets, System.Net.WebRequest 
	  for HTTP and HTTPS access, System.Xml and System.Xml.Linq for
	  XML processing (keep in mind that the LINQ syntax is not 
	  available though) and yes, you can use DateTime and so on.

For details, you can see our list of assemblies supported here:

	http://monotouch.net/Documentation/Assemblies
