> There is one bit of ugliness, and that has to do with structure  
> return values;   The calling conventions for ARM and X86 are  
> different so sometimes you need to call a method that contains the  
> suffix "_stret" (for struct return) and pass as the first parameter  
> an "out" value that will hold the result.
>
> And since the rules are different, you end up with code like this if  
> the device/simulator struct return rules are different:
>
> if (Runtime.Arch == Arch.DEVICE){
> ﻿   // obj_msgSend call goes here
> } else {
> ﻿   // obj_msgSend call goes here
> }
>

It should be noted the different with regard to structure return types  
only happens when the native sizeof the return type <=  
sizeof(register_t)*2 on x86.  The darwin ABI for intel specifies that  
structures under 8 bytes can be returned in EAX:EDX, anything larger  
than 8 bytes follows the same rules on device and simulator.
