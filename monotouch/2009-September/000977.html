The price is for licensing Mono's runtime for static linking and the  
MonoTouch library for static linking, both required on the iPhone.

Usually the price for Mono's license for static linking is much  
higher, as we have historically only licensed that to large users.    
This is the first time that we make Mono's runtime available under non- 
LGPL terms to the public.

Although I sympathize with the idea that MonoTouch should have a  
number of extra features, and I understand that MonoDevelop's  
experience is not the same as Resharper or Visual Studio, we do not  
feel that it is in anyone's benefits to keep MonoTouch away from the  
public for years until MonoDevelop catches up to them.

This is a calculation that everyone needs to do: for some folks not  
having an IDE that is just like Resharper plus MonoTouch is not worth  
the price at this point.

An open source OpenTouch project would at this point require a Mono  
commercial license (as a distributor, you would have to comply with  
the LGPL, in particular section 6) or the creation of a new non- 
LGPL .NET runtime.
