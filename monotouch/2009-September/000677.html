If the log file contains an error involving "CRITICAL **:
_wapi_shm_file_open", you have the wapi error seems to have cropped up
recently, possibly from installing a new Mono framework package or
upgrading to Snow Leopard. I'm not sure of the cause but fortunately
it's easy to fix, and seems to be a one-time thing.

It can be solved by ending all Mono processes then removing the
~/.wapi directory.

Run "killall -9 mono monodevelop mdtool" to be absolutely sure they're
all dead (in case something's hung in the background). If you have
other Mono apps running that may have renamed themselves, make sure
those are closed too.

Then "rm -r ~/.wapi" to remove the wapi directory and its contents.

Things should work fine after that.
