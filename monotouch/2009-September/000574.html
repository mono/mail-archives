Subject: [MonoTouch] WARNING: read this before viewing XnaTouch code

A recent email to the MonoTouch list indicated that the
recently-announced XnaTouch project was created by using Reflector to
decompile Microsoft's XNA assemblies. This is in violation of Mono's
policies on clean reverse engineering, and anyone viewing it would be
tainted and unable to contribute to a clean implementation.

This is unfortunate, as we are very excited by the possibility of
bringing the XNA APIs to Mono. We therefore strongly recommend that
people interested in contributing to XnaTouch do NOT view the code
until this issue can be resolved to our satisfaction.
