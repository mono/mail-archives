Hi all,

If you're like me & you spend your working life in Visual Studio, and  
therefore are having trouble adjusting to the key bindings & editor  
behaviour of MonoDevelop, I've got a working solution using VS:

1. Install Parallels Desktop (or VMWare Fusion), running Windows + VS  
(in my case, the VM is mapped to my Boot Camp windows partition)
2. Creates shares so you can access the appropriate Mac files from  
inside the VM - for instance, I've got shares set up pointing to my  
Mac home directory (where all my code lives), and the /Developer/ 
MonoTouch directory (so I can add a reference to monotouch.dll in VS)
3. Set up your solution in VS, add files & DLL references from the Mac  
share(s).
4. Back on the Mac side, either:
     a) create a script that builds the MyApp.app folder via mtouch &  
deploys to the sim or device; or
     b) have MonoDevelop running as well as VS, to do the final build/ 
deployment step.

This may sound pretty awkward, but in practice (once you get set up),  
it works quite well.
