Hey there,

When making a new ViewController, Monotouch creates in designer.cs file:

--

[MonoTouch.Foundation.Connect("view")]
*protected* MonoTouch.UIKit.UIView view
{
get { return ((MonoTouch.UIKit.UIView)(this.GetNativeField("view"))); }
set { this.SetNativeField("view", value); }
}

--

But when making changes in the Interface builder it changes the "view" from
*protected* to *private*:

--

[MonoTouch.Foundation.Connect("view")]
*private* MonoTouch.UIKit.UIView view
{
get { return ((MonoTouch.UIKit.UIView)(this.GetNativeField("view"))); }
set { this.SetNativeField("view", value); }
}

--

That gives this warning when compiling:
The private property 'SomeProject.SomeViewController.view' is never used
(CS0169)

You can manually set the private to protected again, but when making changes
in interface builder and saving it changes it back to private again.

NOTICE: You might need to remove the view connection and make it again in
order for this to occur.

Regards
Tom
