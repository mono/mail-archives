view?  (This seems important since it's a "hidden behaviour" if I need to

window (by putting it on the XIB and calling

"myViewController.PresentModalViewController(myView, true)" instead of
"window.AddSubView(myView.View)" and it started working.

 

HOWEVER...

 

If I add any direct reference to the View property before I call

So I contend that any programmatic access to a UIViewController's "View"
property before MonoTouch initializes the view causes the "ViewDidLoad"

Mike,

 

  You have a few different bugs in your code:

 

#1: MyView will be GC'd since you dont keep a reference to it.

#2: The MyView default ctor doesn't call into the base chain, so its never
registered with the runtime, you want public MyView () : base ("MyView",
null) {}

#3: You're adding the view to the mainwindow directly, short circuiting the
view controller, which is why the method is not raised on the view
controller.

 

-g
