On Thu, Sep 3, 2009 at 1:03 AM, Lars Magnusson <lavima at gmail.com> wrote:

> Hi
>
> I don't know much (if anything) regarding the licensing issues of using
> code from the original Xna dll, or regarding getting it past the app review.
> That being said, it seems natural that XnaTouch best would benefit from
> working with the Mono.Xna code.


Indeed.

The main difficulty is that Mono.Xna is using Tao.OpenGL directly to
implement graphics-related functionality like Texture2D etc etc, while
XnaTouch needs to implement these classes on OpenGL|ES. Fixing this won't be
easy, especially if you take the available profiles into account (OpenGL|ES
1.1 & 2.0 and OpenGL 2.1 & 3.x - right now Mono.XNA only supports OpenGL
2.1).

We'd either need a full-blown API abstraction layer or some kind of tricky
compile-time polymorphism (and I can't see how that could work right now).
