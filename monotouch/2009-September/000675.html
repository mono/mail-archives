A new drop of MonoDevelop is now available:
http://go-mono.com/archive/MonoDevelop-MonoTouch-Preview-20090904-0.dmg

MonoDevelop now requires a newer version of GTK+ that resolves a
number of known issues, including redraw bugs, control-click
functionality, and Mac keybindings for standard GTK+ widgets. This is
included in a refresh of the Mono 2.4.2.3 Framework Installer for Mac
that can be downloaded from
http://www.go-mono.com/mono-downloads/download.html

The following issues in the MonoTouch addin have been resolved in this build:

* The "Debug in Xcode" command is now run in verbose mode and output
is shown in a pad
* Variables such as ${ProjectDir} can be used in mtouch arguments,
fixing "Bug 535455 - Add ability to specify $ProjectLocation in
additional mtouch arguments".
* There is a menu for picking common variables for mtouch arguments
* The "Upload to device" command builds the project before uploading
unless the "build before running" preference is disabled
* Debug in Xcode/Upload to device are disabled while building or if
the project is not enabled as part of the current solution
configuration
* Fixed "Bug 534228 - Minor formatting issue in autogenerated code
from Interface Builder"
* The "Debug in Xcode" command is enabled when simulator platform
configurations are active
* Nibs are copied into the app root even if they're in subdirs, unless
in an lproj, fixing "Bug 536151 - XIB/Controllers in Subdirs."

There are also a number of fixes to the MonoDevelop core, including
performance improvements when scrolling and selecting text in the
source editor.

Please file bug reports at for any MonoDevelop bugs you encounter so
that we can fix them for 2.2:
http://monodevelop.com/Developers#Reporting_Bugs. MonoDevelop is now
feature-frozen for the 2.2 release, which is planned for the end of
October.
