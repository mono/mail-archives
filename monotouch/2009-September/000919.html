Hi Ali,

As far as I know the appropriate class is UIAlertView (at least that's what
I've been using).

This can have 1-n buttons, where it defaults to one button (referred to as
the "cancelButton" although you can call it whatever you like - you could
give it a value of "OK" as I have in my second UIAlertView below).

The event triggered when you click a button provides the index of the button
that was clicked.

In order to use any buttons in addition to the default one you have to
supply a UIAlertDelegate.

Code to create and invoke a UIAlertView is as follows:

using(UIAlertView alert = new UIAlertView("Alerty Alertiness","Are you
sure?",new AlertHandler(this),"Cancel","OK"))

{

          alert.Show();

}

The first parameter is the title, followed by a message, followed by the
UIAlertViewDelegate, then the "cancelButtonTitle", then the titles of any
additional buttons or null.

An example of a UIAlertViewDelegate is as follows:

 

public class AlertHandler : UIAlertViewDelegate

{

          private AppDelegate _appDelegate;

          public AlertHandler (AppDelegate appDelegate)

          {

                   this._appDelegate = appDelegate;

          }

          public override void Clicked (UIAlertView alertview, int
buttonIndex)

          {

                   string message = string.Format ("You chose  \"{0}\"",
buttonIndex == 0 ? "Cancel" : "OK");

                   using (UIAlertView alert = new UIAlertView ("Results",
message, null, "OK", null))

                   {

                             alert.Show ();

                   }

 

          }

}

Note that in this case once the UIAlertView has been displayed and a button
clicked, it will display *another* UIAlertView with a single "OK" button,
showing a message of either "You chose "Cancel"" or "You chose "OK"".
