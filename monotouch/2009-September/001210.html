>> Should be possible but will require some experimenting.  Here are some
>> links and concepts that might be useful:
>>
>> http://monotouch.net/Documentation/Objective-C_Selectors
>> You can use selectors to invoke methods on classes that are loaded at
>> runtime.
>>
>> http://monotouch.net/Documentation/XCode
>> How to link monotouch into an xcode project.
>>
>> http://wiki.monotouch.net/User:Trustme
>> On this page I do the opposite I embed an objective-c library and invoke a
>> class on that library at runtime.
>>
>> Objective-C uses late binding.  So you can do this:
>>
>> classPtr = findClass("NameOfClass");
>> classPtr.InvokeSelector("runMethodA", args);
>>
>> So you may have to wrap access to your C# library in a class that is
>> registered to the objective-c environment.
>>
>> You can also use NSNotificationCenter.
>>
>> But in some way your C# library has to run the initialization code to
>> register the classes with objective-c at runtime.  I did exactly what you
>> are attempting with an objective-c desktop app.  At startup the objective-c
>> code initialized the C# dll so the classes within registered themselves so
>> that I could access them from objective-c.
>>
>> If you do complete the integration please do post your results.
