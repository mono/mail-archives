> > The action should be "content" and the operation should be "copy if
> > newer".
>
> To include a file in the app bundle, you only need to give it the
> "Content" build action.
>
> The "Copy to output" setting doesn't affect the app bundle at all;
> it's simply a generic setting that copies the files to the build
> output directory, and isn't very useful in the context of MonoTouch.
> Setting it will just slow builds and waste disk space.
>
>
> The reason the images from the project are not showing up
> automatically in Interface Builder is that IB is quite tightly bound
> to Xcode, and not to MonoDevelop. Obviously IB does not understand C#
> or the MonoDevelop project format, so can't load the resources and
> types from the MonoDevelop projects. We looking into whether we can
> provide a resource list to IB in a fake Xcode project (and maybe type
> definitions via mapping C# to fake Obj-C headers).
>
> You should still be able to load the image in IB for now, it just
> won't shown up automatically.
