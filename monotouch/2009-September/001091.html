On Fri, Sep 18, 2009 at 2:44 PM, Frank Krueger <fak at kruegersystems.com> wrote:
> I have an idea for a plugin to MonoDevelop to assist me in my iPhone
> development, and am curious where I should start looking to make such a
> thing a reality. Please allow me to give a brief overview, and maybe some
> one can point me in the right direction.
> I tend to write real-time apps for the iPhone - games, simulations, whatnot.
> The Xcode performance tools are therefore important to me. Particularly, the
> CPU monitor - I want to know what code is dictating my performance.
> I have an idea for a plugin that will display performance stats for whatever
> function the mouse is currently over. That is, if I have something like:
> void DoComplicatedCalculation() { // 50 % total cpu, 400 bytes alloc on avg
>   DoA(); // 10 % total cpu, 200 bytes alloc on avg
>   DoB(); // 38 % total cpu, 16 bytes alloc on avg
>   DoC(); // 2 % total cpu, 184 bytes alloc on avg
> }
> then MonoDevelop would show tool tips (shown as comments) - or a side
> window, whatever - whenever I mouse over or press a key.
> So I have the problem of adding a plugin to MonoDevelop and creating a tool
> to record performance metrics and somehow integrating that tool with
> monotouch.
> Is this remotely possible? Can you point to where I should start looking for
> technical background on how to build such tools for mono?
> (I have written profilers for Microsoft's CLR, but haven't been exposed to
> any hooks mono may provide.)
> Thank you very much for your time.

Developing addins is certainly possible. Check out
http://monodevelop.com/Developers for info, in particular
http://monodevelop.com/Developers/Articles/Creating_a_Simple_Add-in
http://monodevelop.com/Developers/Articles/Architecture_Overview
http://monodevelop.com/Developers/Articles/API_Overview

Regarding profiling, there's a MD profiling addin in SVN though was
never quite finished and has bitrotted a bit, but it might be good
starting point, as it was designed to allow different profilers to be
plugged in.

Please ask on the monodevelop-devel mailing list or in the
#monodevelop IRC channel  if you have any questions about writing MD
addins.

I'm not sure how well the Mono profiler works on Mac, but I'm pretty
sure it should be possible to get it working for the simulator at some
point. I don't know about the device - integrating to the Xcode perf
tools may be necessary for that.
