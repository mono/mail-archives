On Mon, 2009-09-21 at 11:50 -0700, Ed Anuff wrote:
> I'd like to integrate some native C code into my app but I'm not sure
> the best way to accomplish this.  Do I build a dylb in xcode (there
> are apparently some ways to get xcode to produce an iphone dylib) and
> then somehow have mtouch link it in statically?  Does the DllImport
> mechanism in Mono work with MonoTouch?

As far as I know, .dylib files are always shared libraries, and thus
can't be used (against the Apple license agreement).  You should be able
to compile your native C code into a .a file and statically link that,
though.

As for invoking the C code, you can use DllImport, but the library name
will be "__Internal" instead of an actual library name.  (__Internal
tells Mono's P/Invoke to look at the hosting binary for the requested
method.)
