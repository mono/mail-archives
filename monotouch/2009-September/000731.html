Yes!

When first learning iPhone dev, I tried both with and without IB in
Xcode/Objective-C. I concluded that both were too painful: IB for obscurity
and without IB for too much code.

So far, I'm very happy using MonoTouch without IB, programatically building
all the UI code.

Oddly, with VS and C#, I much prefer using the designer. I suspect this is
because designer generates code that we can learn from and modify (very
carefully).

Or it could just be because I'm too old and can't learn new tricks!
