From

>From a API design perspective, would it add too much clutter to also add typesafe properties to the StandardUserDefaults class for the well known values?

So then we could have:
NSUserDefaults.StandardUserDefaults.UserName = "user";
NSUserDefaults.StandardUserDefaults.StartupTab = 1;

Instead of
NSUserDefaults.StandardUserDefaults["UserName"] = new NSString("user");  // no compiler type checking and no intellisense
NSUserDefaults.StandardUserDefaults["StartupTab"] = new NS(???);  // no compiler type checking and no intellisense

I know it's low priority, just cringed a little at having to explicitly create an object to use a string.
