Hello,

> > For the single developer guys with little/zero budget.. Can we clarify 
> > a couple of things.
> >
> > 1) Will it be available under LGPL ?
> >     (ie. from what I understand LGPL for Static linking means you must 
> > allow the code to be relinked by the customer ?)
> >     A developer could create a STUB class with a run command, make the 
> > source for it available, and only provide .o (object) files for 
> > linking and not violate the LGPL ?

MonoTouch is not available under the LGPL.   

To clarify, MonoTouch is made up of a few pieces:
	* Mono's runtime (dual licensed LGPL or commercial)
	* MonoTouch binding
	* MonoTouch linker
	* MonoTouch tools
	* MonoDevelop Plugin

All the pieces in the list above that start with MonoTouch are
proprietary, they are not open source.   

The Mono runtime is the only piece that is dual licensed
commercial/LGPL.   That is the only part where you could figure out
something that would allow you to comply with the LGPL.

> > 2) Will SVN access be available to non commercial users ? (assuming 
> > LGPL status)

It is available for Mono and MonoDevelop, but not for the MonoTouch
pieces listed above.
