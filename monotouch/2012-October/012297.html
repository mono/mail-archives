In MonoTouch 6.0.4, there are wrong implementations of OpenMaps and OpenInMaps methods:

public void OpenInMaps(MKLaunchOptions launchOptions = null)
{
    this._OpenInMaps((launchOptions != null) ? null : launchOptions.ToDictionary());
}

public static bool OpenMaps(MKMapItem[] mapItems = null, MKLaunchOptions launchOptions = null)
{
    return MKMapItem._OpenMaps(mapItems, (launchOptions != null) ? null : launchOptions.ToDictionary());
}

Why it is returning null when parameter launchOptions is not null? Some sort of dark magic? :)

It's much more mundane than dark magic. It's simply a bug - and already fixed for the next release, 6.0.5 :)
