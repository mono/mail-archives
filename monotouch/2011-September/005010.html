Thank you to everyone for the responses and help. Pete’s suggestion triggered a thought while rereading Rolf suggestion. I hadn’t specified a custom class for the custom cell defined in the XIB. I defined a custom class for the cell, gave it the outlet for the text field and linked it, changed the outlet type on the controller to match the custom cell type, then implemented the partial class with constructors for the custom cell type. That and using a full UITextFieldDelegate implementation and all is right again. The debugger no longer blows up when first responder changes to or from the UITextField in the UITableViewCell.

Thanks again to everyone for all the help finding the better way to do this.
