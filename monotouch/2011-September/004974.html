On Thu, Sep 8, 2011 at 1:35 PM, vinkaga <vinkaga at gmail.com> wrote:

> Thanks Jeff, it does clarify. Am I correct in assuming that since FB lib is
> native, the reference passed to it (e.g. sessionDelegate) is not counted
> for
> GC use count and therefore are GCed?
>

Correct. Mono's GC is not able to detect that native code is holding a
reference to managed c# objects and so you have to cheat the system a bit by
holding a reference yourself (e.g. by having a class member variable point
to the object).
