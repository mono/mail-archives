Because using a RavenDB server over the network from device would be
trivial. It is just a REST service.

But *running* RavenDB on device will be pretty much impossible without
rewriting large chunks of RavenDB.

RavenDB is NOT 100% managed code. It is built upon the unmanaged ESENT,
aka Jet Blue, non relational database that ships with Windows (the same
DB that powers Active Directory).

RavenDB does not even run on Mono on Linux. I looked at I feel it would
be a significant port to move it to a different storage engine. AFAICT
the open source world doesn't have a non-relational but still table
based ISAM data store. ESENT doesn't really match up to sqlite, mysql,
or BDB.

So I think you might what to have a chat with Ayende again :)

problem.
