I find the latest combination of Monodevelop very unstable. Almost impossible
to work with. Monodevelop crashes about once every hour for no reason, and
usually on exit as well. If you start xcode for editing a xib, and you
return to monodevelop, it says xcode project is synchronized, but if you
click on something, monodevelop freezes up and xcode is started AGAIN,
synchs AGAIN, and then it stops and you can continue. If it doesn't crash.

Also, a very annoying 'feature' is if you use the right scrollbar in the
editor to scroll through your code, the mouse tends to 'stick' to the
scrollbar if you leave the scrollbar... so your code is still scrolling up
and down even if the mouse is not on the scrollbar...

Also since the last versions of monodevelop and xcode, my mac mini, which
was a nice, silent little thing, starts sounding like a vacuum cleaner if
working on it with md/xcode more then 1 hour... if i then stop monodevelop
and xcode, it gets silent again...

All this together makes a day of developing very inefficient... something is
very wrong... it looks like every 'update' of monodevelop makes it a bit
worse... btw i don't use beta releases, just the official ones.
