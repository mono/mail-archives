You don't. That's not the way to do it. You should know when you are about to show a new view. Before doing that, check if it is okay. Maybe add a method that takes the view to show (or whatever) and a delegate that represents the check to perform before showing the new view. If the delegate returns false, don't show the view. That way you separate your code better.
