On Fri, Mar 12, 2010 at 11:01 AM, John McDonagh
<johnmanchester at gmail.com> wrote:
> I tried setting as content and although the app compiled, iPhone simulator
> always froze before the application came up.
> The solution I've found seems to be to keep the action as "embed as
> resource" and then for one build to set that image as the application icon,
> after which the image can be rendered anywhere else.
> I'd like to figure out why this is happening as it obviously is not ideal
> and does not fill me with confidence that I will be able to create stable
> iPhone apps with confidence and consistency.

Don't embed as resource unless you're actually using the image as a
.NET embedded assembly resource, or you'll simply bloat the app size.

* The "Content" build action means that the image file gets included
in the app bundle.
* The "Embed as resource" build action means that the image gets
embedded in the assembly (dll).
* The "Copy to output" and "include in deploy" options are irrelevant
to iPhone apps (though copy to output will waste time and space
copying files to the build directory rather than the app bundle).

If including images in your app bundle causes hangs, please file a bug
report with a test case.
