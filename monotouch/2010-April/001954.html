Memory management is a pain. Which is why I much prefer c#. I didn't go the
route of objc/xcode first but had a good understanding of the MVC pattern
already. Xcode does better integration with interface builder and can spoil
you a bit when designing the UI - especially for a UI that relies a lot on
images for buttons and such. 

 

The video series at www.pragprog.com about obj-c has a good one on memory
management. Using instruments is really nice.

 

I'm learning obj-c/xcode now - in fact porting a monotouch app to obj-c just
in case. Learning the api and the how to build the app will be a great
benefit and you'll be able to build apps even quicker when monotouch is
allowed for certain and you can use the memory management provide natively.
