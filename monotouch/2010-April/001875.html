> In the final .app what’s the difference between code compiled from c#
> and code compiled from obj-c?
> 
> I know there will be naming difference and “helper libraries” with the
> c# from monotouch, but is there any other difference?

Well, it depends on how deep you want to get this question answered.

For example, the following loop:

	for (int i = a; i < b; i++)
		array [i] = 0;

Will be compiled differently by C# and C++.   C# will do arrays bounds
checks to ensure that you do not access array below 0 and above the
upper limit, if you do, you would get an IndexOutOfRangeException.
With C, you would corrupt your heap.

This is the principle on which buffer overflow exploits work, this whole
class of security problems does not exist with languages like C#.
