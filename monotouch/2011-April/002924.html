Hi,

You should simply specify the icons in the "Icons" tab of the "iPhone
Application" panel in Project Options, and MonoDevelop puts them in
the root of the app bundle with the Apple-recommended naming,
regardless of the source file's icon and name. You should not mark
them as Content, or you will have duplicate files and/or unnecessary
file copying.

However, when investigating your problem I found an issue that affects
the 57x57 icon. If it's not called "Icon.png" and isn't in the root of
the project, the CFBundleIconFile value in the generated Info.plist
will be incorrect, because the name transformation is not applied to
this value. You should be able to work around this for now by manually
specifying the CFBundleIconFile value in the Info.plist, as that will
override the value used by MD. The CFBundleIconFiles values should all
be correct, so there's no need to specify or override those.
