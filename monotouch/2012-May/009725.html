make sure that the row that is selected is part of your model or some other class variable. And, make sure to set the accessory to either checked or none all the time. do not assume it is unchecked since cell renderers are reused so it might have reused a cell that was checked. 

basically, my rule of thumb is that anything I set in my cell renderer, I set for all cells for all conditions so that it is explicit.

On May 21, 2012, at 10:58 AM, "Chris House" <chrishouse at proag.com> wrote:

> I have a UITableView that contains ~ 150 items.  When an item is selected, I want to indicate selection via setting the selected cell accessory to UITableViewAccessory.Checkmark.  I’m accomplishing this by using the following UITableViewDelegate:
>  
>                 public class TableViewDelegate : UITableViewDelegate
>                 {
>                                 private readonly Action<int> selectionDelegate;
>                                 private UITableViewCell previousSelection;
>                                
>                                 public TableViewDelegate(Action<int> selectionDelegate)
>                                 {
>                                                 this.selectionDelegate = selectionDelegate;
>                                 }
>                                
>                                 public override void RowSelected (UITableView tableView, NSIndexPath indexPath)
>                                 {
>                                                 this.selectionDelegate(indexPath.Row);
>                                                
>                                                 if (previousSelection != null)
>                                                 {
>                                                                 previousSelection.Accessory = UITableViewCellAccessory.None;
>                                                 }
>                                                
>                                                 UITableViewCell currentCell = tableView.CellAt(indexPath);
>                                                 currentCell.Accessory = UITableViewCellAccessory.Checkmark;
>                                                 previousSelection = currentCell;
>                                 }
>                 }
>  
> This code seems to work fine when the UITableView displays fewer items than can fit on the screen without scrolling.  However for larger lists, I’m seeing that multiple items show the checkmark accessory.  It seems like every 20th item before or after the selection also shows a checkmark.
>  
> Any suggestions as to what I’m missing to ensure only the selected item in the table view shows the checkmark would be appreciated.
>  
> Thanks!
