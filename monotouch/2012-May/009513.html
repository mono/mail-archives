Mike,

I've done exactly this. Mine returns a serializable class with a flag indicating there are more rows (if I request the first 20, I select "top 21" from the db - then I know there are more) and only the 20 items you requested. The next call sends in a start value (20) and gets the next 20, including the flag indicating if there are more, and they are just appended to the table. Seemed the most efficient in terms of data transfer.
