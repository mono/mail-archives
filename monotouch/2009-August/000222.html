Exactly. WCF is dynamic code generation world, and actually it will
almost always fall back to dynamic code generation as long as you
use ClientBase<T> or ChannelFactory<T> (which I'm sure 99.9% of
WCF apps use).

Note that those WCF assemblies are include just because MonoTouch
is based on Moonlight build profile. I'm nothing more than just a
beta tester, so I haven't added any special treatment for MonoTouch
in WCF code.
