Hmm, I might have been somewhat pessimistic on it. I had a bit in-depth
look at this code generation part, and found that the problem is only
in ChannelFactory<T>.CreateChannel() and ClientBase<T>.CreateChannel().

Since ClientBase<T>.CreateChannel() can be overriden, and when Visual
Studio (probably only after 2008 SP1) creates a client proxy it is
actually overriden to just return a derived class of
ClientBase<T>.ChannelBase, it could actually avoid dynamic proxy
generator.

I'm still not sure on other parts of WCF, but there seems some light.
