No, AFAICT it's been managed for about 6 years. I don't know if we had
one at all before then - IMO an unmanaged XSLT implementation would be
bad idea, as marshalling the XmlDocument would be horrible (most
likely via a giant string copy & reparsing).

That wouldn't really save you much anyway. In order to use the
System.Xml.Xsl API, you'd also need XmlDocument and XPathNavigator
etc.
