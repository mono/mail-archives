On Fri, 2009-08-07 at 02:06 -0400, Randy Edmonds wrote:
> There appears to be a lot of documentation (in xml files)
> at /Developer/MonoTouch/usr/share/doc/MonoTouch    Are these the files
> that MonoDevelop would be using if it could? Is there anyway other way
> to view this documentation?

Not exactly.

The files in /Developer/MonoTouch/usr/share/doc/MonoTouch before install
comprise the documentation available at http://www.go-mono.com/docs.
During installation, a script is run which *merges* the Apple-provided
documentation with the MonoTouch XML documentation to provide more
complete documentation.  (This is done at install time so that we don't
distribute Apple's documentation, it's all done on the client machine
and never distributed.)

These XML files are then 'mdoc assemble'd into Monodoc documentation,
and installed into
/Library/Frameworks/Mono.framework/Versions/Current/lib/monodoc/sources,
making them available to monodoc itself.

There's still the "minor" issue of actually *reading* the documentation.
The hope was that MonoDevelop would at least be able to display the
<summary/> documentation for types & members, which should be helpful.
Actually reading the documentation can be done in one of three ways:

 1. Port monodoc GUI to OS X.  Problem: we don't have an HTML renderer 
    usable from GTK+ yet.
 2. Use the command-line 'mod' program.  Problem: It's command-line, 
    and it requires 'links', which isn't installed by default, so not
    very useful.
 3. Use 'mdoc export-html:

	cd ~/Sites
	mkdir monotouch
	mdoc export-html -o monotouch /Developer/MonoTouch/usr/share/doc/MonoTouch/en

    This takes ~2 minutes on my machine.  Documentation will then be 
    available as normal HTML files within the ~/Sites/monotouch
    directory, or http://localhost/~USER/monotouch if you have the 
    personal web server running.

    DO NOT publish these files.
