On 24-Aug-09, at 5:54 PM, Oisín Grehan wrote:

 Hey guys,
>
> I have an objective-c iphone project that is statically linked to a vanilla
> c++ shared library. I want to port it all to monotouch. Is there anyway I
> can use the c++ library from monotouch c#? Is there any sort of concept of
> p/invoke? Can I wrap this library the same way you guys expose native
> libraries as "managed" libs (e.g. how you expose the NS* foundation classes
> as managed classes?)
>
>
Yes pInvoke works, but beware that C++ is name-mangled, so you'll likely
want to put a C api aroudn your C++ classes.  What you do is:

1> Edit your project settings in MD to have extra mtouch flags and add "-lib
/path/to/your/static/lib.a" (you need to do this per configuration and have
a x86.a and arm.a)
2> Setup your dll imports as follows:

[DllImport ("__Internal")]
static extern void foo ();

Then everything should work
