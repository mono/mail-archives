Also:

1)

Renaming an action on my view's "file owner" (mainview controller), I got a
duplicate entry for:

    [MonoTouch.Foundation.Register("MainView")]
    public partial class MainView {

        [MonoTouch.Foundation.Export("showInfo")]partial void showInfo
(MonoTouch.UIKit.UIButton sender);
        [MonoTouch.Foundation.Export("showInfo:")]partial void showInfo
(MonoTouch.UIKit.UIButton sender);

I did this because IB suggested a colon, but other examples didn't use this,
so I renamed it.

2)

My view property in the partial class for the same view is now being marked
private (previous builds seem to choose protected). This results in a
semi-annoying "property 'view' is not being used" after compilation.

    [MonoTouch.Foundation.Register("MainView")]
    public partial class MainView {

        [MonoTouch.Foundation.Export("showInfo")]partial void showInfo
(MonoTouch.UIKit.UIButton sender);

        [MonoTouch.Foundation.Connect("view")]
      ***  private *** MonoTouch.UIKit.UIView view {
            get {
                return
((MonoTouch.UIKit.UIView)(this.GetNativeField("view")));
            }
            set {
                this.SetNativeField("view", value);
            }
        }
    }
