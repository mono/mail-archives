A new drop of MonoDevelop is now available:
http://go-mono.com/archive/MonoDevelop-MonoTouch-Preview-20090821-0.dmg

The following issues have been resolved in this build:

 * A race that caused the output tailing threads not to collect final
stdout/stderr output on app termination has been fixed.
 * An ordering issue in the CodeBehind grouping files that caused some
files to be incorrectly marked red/nonexistent has been fixed.
 * The "View with controller" template now uses the "File's owner" as
the controller proxy object instead of adding a controller to the xib.
 * The "View with controller" template now includes an empty constructor.
 * The err.log/out.log files were swapped. This has been fixed.
 * If the saved signing key is not present, MonoDevelop now warns and
uses the default key.
 * The app bundle version can now be set in the application settings GUI.

This also includes several fixes to the MonoDevelop core, including a
solution configuration mapping issue that prevented building/running
nUnit projects as part of an iPhone solution configuration.
