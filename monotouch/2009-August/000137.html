I have found that there are a variety of bugs exposed when linking a
solution that contains:
1. A normal Executable project (with MainWindow.xib, etc.)
2. A Library project (created as empty MonoTouch project, then set the
Compile Target to Library)

(with the Executable project referencing the Library project)

The Library project produces linker errors under the following
circumstances:

1. The Library project uses one of the System delegates (Predicate, or Func)
2. The Library project uses extension methods.
3. The Library project references the OpenTK assembly.

I ran into enough of these errors that I collapsed the solution into a
single project.

Are the developers interested in hearing about this, or are Library projects
not supported yet?

-Frank
