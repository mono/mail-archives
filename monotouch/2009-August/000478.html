On 25-Aug-09, at 4:44 AM, Peter Sakhno wrote:

> Further investigations.
>
> First...
> I put MainViewController.xib into XCode ObjC project and it works.  
> Instance of class attached to that NIB responses to all events.
>
> So... The NIB file is ok. It is correctly loaded from corresponding  
> MonoTouch class constructor and later loses connection with instance  
> of that class for some reasons.
>
> Second...
> I thought alittle and decided to replace manual creation of view  
> controller with the automatic in the main window NIB.
> I made following changes to the MainWindow.xib using IB:
> - added new outlet "navigationController" to the AppDelegate;
> - drag-n-dropped Library pallet "View Controller" and assigned its  
> Class Identity to the name of my view controller class  
> "MainViewController";
> - connected outlet AppDelegate:: navigationController to  
> MainViewController.
>
> This changes added following code to the MainWindow.xib.designer.cs  
> file:
>
> 	// Base type probably should be MonoTouch.UIKit.UIViewController or  
> subclass
> 	[MonoTouch.Foundation.Register("MainViewController")]
> 	public partial class MainViewController {
> 	}
> It produced error during compilation since class MainViewController  
> was already registered in MainViewController.xib.designer.cs file. I  
> just commented conflicting "Register" string in  
> MainWindow.xib.designer.cs since it is rearly changed compared to  
> MainViewController's one.
>
> And...
> Ta-da-a! It helped! Now this app works as desired - view controller  
> is properly created and loaded and responses to all events.
>
> In attach - zip with updated MD project and new XCode sharing the  
> same MainViewController.xib file.
