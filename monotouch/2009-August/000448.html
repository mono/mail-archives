> > I'm attempting to create a TabViewController-based application using IB
> and
> > MonoDevelop.
> >
> > I'm creating the TabViewController in my MainWindow.xib, with one NIB for
> > each tab.   In order to have the TabController create the correct
> > ViewController, I set the type for each Tab ViewController to the class
> for
> > the associated NIB.   This causes the above error because the partial
> class
> > is declared with the TabViewController and in the ViewController designer
> > file.
>
> We've considered the issue of referring to the same class from
> multiple xib files before and decided to defer the solution until we
> had real use cases. Solving the general case is complicated because it
> would involve things like finding properties that are defined in
> multiple parts of the same class and finding a common base type, etc.
> It also breaks the codebehind-like organizational model that we're
> trying to build, so could be quite confusing to users.
>
> I've played around with this in interface builder a bit, and I think
> the problem would be solved by simply having the xib generator partial
> classes when there are no actions or outlets defined. Would that solve
> the problem for you?
>
> If anyone else has use cases that would involve defining the same
> partial class in multiple xib files I'd very much like to hear them.
>
> > Is there a workaround for this?
>
> While MD doe not have the project loaded, manually rename the
> xib.designer.cs file on disk and in the csproj file. MD only updates
> designer files that match the xib name.
