On Tue, Aug 25, 2009 at 1:33 PM, Tom Renneke<tom1234 at gmail.com> wrote:
>
> I'm stuck on a problem with setting a new image on a UITabBarItem.
> I've made a connection from the App Delegate to the specific item and I'm
> using the following code:
> tabItem1.Title = "test";
> tabItem1.Image = UIImage.FromFile(@"images/book.png");
> Setting the Title works so I got the connection to the item, but when adding
> the second line to set the image the App crashes in simulator. I've attached
> the PNG image so you can test it out.
> I've also tried this:
>
> string path = NSBundle.MainBundle.PathForResource(@"images/book", @"png");
> tabItem1.Image = UIImage.FromFile(path);
> But I end up with same result. Is it a bug or am I missing something?

You need to set the build action of the image to "Content" so that
MonoDevelop knows to copy it into your app bundle.
