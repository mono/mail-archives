On Sat, Aug 29, 2009 at 12:27 PM, Geoff Norton <gnorton at novell.com> wrote:

> Its worth noting this is going to be changing int he next release.
> -lib is removed and deprecated, and is replaced by -gcc_flags so that you
> can specify many flags at once.  The reason for this is a few fold:
>
> #1: Your static lib will be stripped by the linker if you dont instantiate
> it directly in code, adding "-ObjC" will preserve all Objective-C code,
> adding "-all_load" will preserve all other code.
> #2: Your static lib may rely on other libraries (eg sqlite3.0) that we
> cannot detect, adding them to this flags would work as well.
>
> Example usage:  -gcc_flags "-L/path/to/your/staticlib -lyourLibName
> -lsqlite3.0 -ObjC"
>
> -g
>

> On 24-Aug-09, at 5:54 PM, Oisín Grehan wrote:

>  Hey guys,
>>
>> I have an objective-c iphone project that is statically linked to a
>> vanilla c++ shared library. I want to port it all to monotouch. Is there
>> anyway I can use the c++ library from monotouch c#? Is there any sort of
>> concept of p/invoke? Can I wrap this library the same way you guys expose
>> native libraries as "managed" libs (e.g. how you expose the NS* foundation
>> classes as managed classes?)
>>
>>
> Yes pInvoke works, but beware that C++ is name-mangled, so you'll likely
> want to put a C api aroudn your C++ classes.  What you do is:
>
> 1> Edit your project settings in MD to have extra mtouch flags and add
> "-lib /path/to/your/static/lib.a" (you need to do this per configuration and
> have a x86.a and arm.a)
> 2> Setup your dll imports as follows:
>
> [DllImport ("__Internal")]
> static extern void foo ();
>
> Then everything should work
>
