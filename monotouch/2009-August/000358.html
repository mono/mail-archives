On Wed, Aug 19, 2009 at 9:09 AM, Joakim Sundén<joakim at varia.nu> wrote:
> When I add an NUnit Library Project to my MonoTouch solution and do
> nothing more than run a simple test (Assert.That(false)) I get the
> following exception:
>
> "Could not get a valid path to the assembly. There may be a conflict
> in the project configurations."
...
> Can someone please tell me what is wrong here? Has anyone else
> successfully included an NUnit Library Project in their solution? I
> know of at least one more person that gets the exact same error.

I've investigated, reproduced, and found it to be a manifestation of
an existing bug in sln->project configuration mappings:
https://bugzilla.novell.com/show_bug.cgi?id=473704

You can fix your sln by editing the configuration mappings in the sln
options. Map the "Debug|iPhone" sln config to the "Debug"
configuration of the nunit project, and so on.
