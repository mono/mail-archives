On Sat, 2009-08-08 at 03:19 -0700, Christian Beaumont wrote:
> It seems to me they are trying to make it more like XNA or something.
> 

Indeed, we are trying to bring the math API (and only the math API)
closer to XNA.  We feel that this will make porting from XNA to OpenTK
easier. Besides, there are several libraries that only use XNA for math
(e.g. physics libraries) - a similar math API will allow them to work
with OpenTK with trivial changes.

That said, the changes introduced by this effort are not breaking - we
are simply introducing new methods (offering new functionality) or
deprecating (but not removing) existing methods.

> 
> Also the fact that the OpenGL constants are in the All enum now where
> they were in separate enums when I was using OpenTK on Windows.

Specific enumerations for desktop OpenGL are the result of years of
accumulated effort. This effort will have to be redone for OpenGL|ES
from scratch: it involves reading and understanding the parameters for
each ES method, then writing the necessary enumerations by hand.

We have already started sorting through the ES 2.0 profile but, as you
can understand, we won't be able to complete this effort in the
immediate future without help.
