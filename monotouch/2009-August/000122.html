Hey,

On 8/7/09, Frank Krueger <fak at kruegersystems.com> wrote:
> I would like to use an assembly of utility functions that I have created
> over the years. Also, I have some 3rd party assemblies that I would like to
> reference (protobuf-net for example).
>
> Is this supported by MonoTouch right now?

This particular exception has been fixed. Now, it's definitely not
recommended to re-use third party libraries that have not been
compiled with the MonoTouch SDK.

For instance protobuf will likely be compiled against the .net 2.0
mscorlib and System.dll. We provide different versions of those
assemblies, with some types and some methods removed. So even if the
compiler and mtouch will be ok with that, you could face at runtime
some MethodMissingException for instance.
