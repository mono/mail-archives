A new build of MonoDevelop is now available:
http://go-mono.com/archive/MonoDevelop-MonoTouch-Preview-20090806-2.dmg

This fixes the following issues:

  * CodeBehind generator did not handle MapKit types
  * New project added to existing solution had invalid output path
  * ibtool invocation failed on files with spaces in the full path
  * Several app bundle options were incorrectly stored by the options GUI
  * Additional mtouch args property was not stored by the GUI
  * Action names not ending with ':' were truncated in the generated code
