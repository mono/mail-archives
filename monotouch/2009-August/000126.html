Morning all,

Just to chime in with some tips. We develop 1 platform of code on many different .NET platforms be it NETCF, NETFX and Silverlight and as a rule of thumb the best way to go about this is to just code-share the source files in various projects. That way you can ensure your compiling for all supported API's or using precompiler directives to chunk out platform differences.

This has worked out really well for us in the past. Allows us to maintain a large chunk of compatible code.

These kinds of setups will be important as people start targeting multiple SDK's like MonoTouch, Moonlight/Silverlight, .NET etc.

This particular exception has been fixed. Now, it's definitely not
recommended to re-use third party libraries that have not been
compiled with the MonoTouch SDK.

For instance protobuf will likely be compiled against the .net 2.0
mscorlib and System.dll. We provide different versions of those
assemblies, with some types and some methods removed. So even if the
compiler and mtouch will be ok with that, you could face at runtime
some MethodMissingException for instance.
