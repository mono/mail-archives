On 8/7/09, Frank Krueger <fak at kruegersystems.com> wrote:
> I get the following runtime error when trying to instantiate an object whose
> type is a constrained generic argument:
>
> Unhandled Exception: System.MissingMethodException: Method not found:
> 'Default constructor not found...ctor() of Transit.Names'.
>
> This is the function:
>
> public void c<T>(ref T x) where T : Content, new() {
> ...
> var r = new T();
> ...
> }

Hm. So it's definitely the linker removing the default constructor,
because it's not referenced in the metadata anywhere.

We have code in the linker to detect some situations, for instance:

class Foo {}

Foo f = null;
c<Foo> (ref f);

In this case, there's a direct instantiation of T with Foo, whose
default constructor will be preserved.

For more complex cases, we may not detect this case, and we may
actually be unable to do so.

Could you detail a bit more how `c' is used ?
