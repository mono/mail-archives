On Thu, Aug 6, 2009 at 7:12 AM, Mark Beaton<mark at beaton.co.nz> wrote:
> Hi all,
>
> Firstly - congrats with the initial release, I've been looking forward
> to this for a long time...
>
> I've just created a new MonoTouch solution in MonoDevelop, but when I
> attempted to build it I got an "ibtool returned error code 1" error.
>
> I've poked around a bit, and it seems it's due to my solution path
> containing spaces... I can successfully run ibtool from the command
> line to work around this problem.

Yes, the ibtool invocation was missing quotes. Fixed in svn:
http://anonsvn.mono-project.com/viewvc?view=rev&revision=139502

> I tried to dig around the solution/project options in MonoDevelop to
> find where the ibtool command was being initialised, but to no avail -
> is there some sort of build task I can edit to solve this without
> having to manually compile the xib each time? In the meantime, I've
> moved the solution to avoid the space-in-path issue...

Yes, you can add a custom prebuild command (project options -> general
-> custom commands). Have this invoke a bash script or makefile to
compile the xibs. If they are up-to-date, MD will not try to compile
them

Or you could build MD with the iPhone addin from SVN, or wait for our next drop.

http://monodevelop.com/Developers/Articles/Building_MonoDevelop_on_OS_X

> That aside - I've successfully (and pretty much painlessly, apart from
> the above) deployed & run the shell of an app to both the emulator &
> an iPhone 3G.
