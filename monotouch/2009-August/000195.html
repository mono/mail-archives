On Fri, 2009-08-07 at 21:18 -0400, Jonathan Pryor wrote:
> On Fri, 2009-08-07 at 18:15 -0400, Frank Krueger wrote:
> > As I'm trying to share code between MonoTouch and Windows, I ran into
> > a descrepency wrt OpenTK and the Math classes.
> 
> > In MonoTouch, Vector2 is in the namespace OpenTK.
> > On Windows, Vector2 is in OpenTK.Math.
> ...
> 
> > Is there any rational behind this?
> 
> Yes.  The rationale is that we're tracking OpenTK trunk, and OpenTK has
> changed the namespace from OpenTK.Math to OpenTK.
> 
> 
> (Why?  I have no idea.  It was done in r1899 by the_fiddler with comment
> "Removed OpenTK.Math namespace. Math functions now go directly into the
> OpenTK namespace.")

The OpenTK.Math namespace clashed with the System.Math class, something
that is against the library design guidelines
(http://msdn.microsoft.com/en-us/library/ms229042.aspx). Breaking
changes like this will occur, if necessary, until the API is declared
stable.
