> After I do that, I get the following exception on XDocument.Load("my-xml-string");
>  "Unhandled Exception: System.IO.FileNotFoundException: Could not load file or assembly 'System.Xml, Version=2.0.5.0, Culture=neutral, PublicKeyToken=7cec85d7bea7798e' or one of its dependencies."
> Any ideas? (Previously I had added the "-nolink" mtouch arg so the WebClient() object would work.)

This is another manifestation of MonoTouch not resolving indirect
references from the main assembly, which will be fixed in the next SDK
drop.

Try adding /Developer/MonoTouch/usr/lib/mono/2.1/System.Xml.dll to
additional mtouch args in MD, as this will explicitly add it to the
assembly list for inclusion in the .app. You'll have to remove this
after mtouch is fixed.

Also, be aware that the additional mtouch arguments setting is
per-configuration - see the comboboxes at the top of the settings
panel - so you'll have to set it for each configuration+platform you
want to use, e.g. Debug|iPhone, Debug|iPhoneSimulator, Release|iPhone,
Release|iPhoneSimulator
