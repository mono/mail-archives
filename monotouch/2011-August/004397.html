This bug has been there for sometime now, at least in the 2.6 betas. You'll

created, i.e. Selected the folder. If you right click on a folder and create

I'm seeing a strange issue with Monotouch and Monodevelop. I am using MT
4.0.4.1 and MD 2.6b3. I'm not quite sure which is causing this problem. When
I create an iPhone View with a controller, I am seeing the files associated
with the xib multiple times. This only occurs when I first create the new
view and controller. Here is the scenario:
1. in an iPhone project, create a new iPhone View with Controller.
2. Take a look at the solution file. Notice that the .xib, the .xib.cs, and
the .xib.designer.cs file are all displaying in the project.
3. Exit monodevelop.
4. Open monodevelop and open the project that you just had open.
5. Notice that the solution only has the .xib file in the root of the
solution. To view the other files, you have to expand the xib file and then
you can see the other files.

Like I said, I'm not sure if this is a MT or MD issue. For me, the lack of
consistency in the display is a bug. Thoughts?
