My *guess* would be that perhaps the GC thread is getting in the way
here, trying hard to clean up all the removed objects.

 

A common trick to work around this problem is to try and re-use your
heavier objects used in this scenario to try and limit the amount of
work the GC has to do. How hard this is to do depends on how your code
works/was designed etc.

 

It's possible that the SGen GC would be more performant in this case
(MonoTouch still uses the Boehm GC), but until we get SGen ported to
MonoTouch and work out any kinks it may have we'll never know :-(

 

I hope you figure out a solution,
