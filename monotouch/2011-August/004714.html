That's very interesting and will make it easier to investigate. Please

> so definitely something wrong with mono develop, rather than the device.

The error comes from the device, i.e. the application was transfered
but could not be installed, but it does not mean it's a error on the
device (e.g. lack of space). It's even less likely if you can
duplicate this using another device.

> Any idea's on how to get around it so I can continue working?

I've not seen this particular error code before. However similar
errors, i.e. from AMDeviceInstallApplication, occurs when the
certificate/provisioning profile are incorrect or when the data from
Info.plist does not match the devices (e.g. MinimumOSVersion). Those
have a different error codes but your error is likely around the same
conditions.

> Should I
> uninstall mono/dev/touch and reinstall everything?

Re-installing MonoDevelop won't help (it's not part of the application
transfer) and it's also unlikely that re-installing MonoTouch would
help. You could try to build/install another application to make sure
about the later.
