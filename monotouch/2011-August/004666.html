Hey Nic,

  Here are some stats:

Mac

512bit - 250ms
1024bit - 1300ms
2048bit - ~18000ms

Slower PC running the exact same code with .Net 4.0 so internally
RSACryptoServiceProvider uses Microsoft's CryptoAPI:

512bit - < 50ms
1024bit - < 500ms
2048bit - ~2000ms


So there is a major discrepancy going from 1024 to 2048 in managed code. 
This is due to how the primes are generated in the RSAManaged class vs how
CryptoAPI does it with native code.

I will check out the MYCrypto lib, thanks for the link!

Here is the insanely simply test code:

using System;
using System.Diagnostics;
using System.Security.Cryptography;

namespace TestCryptoCli
{
    class MainClass
    {
        public static void Main(string[] args)
        {
            int keyLength = 2048;
            
            if (args.Length > 1)
                keyLength = Convert.ToInt32(args[0]);
                
            Stopwatch sw = Stopwatch.StartNew();
            
            RSACryptoServiceProvider rsa = new
RSACryptoServiceProvider(keyLength);
            Console.WriteLine(rsa.ToXmlString(true));
            
            sw.Stop();
            
            Console.WriteLine(string.Format("Runtime: {0}ms",
sw.ElapsedMilliseconds));
        }
    }
}
