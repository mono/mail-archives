Working with *big* integers is something that can use a lot of different optimizations, algorithms and implementations - each with different limitations.

E.g. iOS generates up to 4096bits keypairs and _can_, not sure if, work with "fixed" big integers. Mono implementation works up to 16384bits (to cover the same _space_ as MS implementation) and yes the iPhone5 will be out sooner than your iPhone4 can generate those ;-) OTOH you could use such a key (generated on your desktop computer) using MonoTouch, while you could not do it from iOS itself.

The biggest difference, in this case, is that you're comparing managed code with hand-crafted assembly language (MS CryptoAPI) between different platforms/architectures. It does not change the solution (going native) but don't expect such performance from your iOS devices :-)
