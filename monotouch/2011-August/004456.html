Actually yes there is a difference between MD 2.4.2 / 2.6RC1 and MD 2.8a projects, MD 2.8a will ask you to upgrade projects to the new format, but it will allow you to backup the project in case you want to rollback.

this is needed due to MD 2.8a has experimental Xcode's 4 IB integration, so if you do not use Interface Builder then you can stay safe on MD 2.4.2 or 2.6RC1

as Miguel said you have these 3 options


> (a) Stable: MonoTouch 4.0.5 with MonoDevelop 2.4.2
> (b) Beta: MonoTouch 4.1 with MonoDevelop 2.6RC1
> (c) Alpha: MonoTouch 4.9 with MonoDevelop 2.8 Alpha 1

Choose (a) if you want to stay on the safe and stable side, Choose (b) if you want to go wild and try new beta features like usb debugging etc. and in order to try new iOS 5 features and Xcode 4 integration you need option (c)

you can go here to read a general overview of new features of each new release http://blog.xamarin.com/ 

and for detailed features of each MonoTouch versions check here

http://ios.xamarin.com/Releases/MonoTouch_4/MonoTouch_4.0.5

http://ios.xamarin.com/Releases/MonoTouch_4/MonoTouch_4.1.0

http://ios.xamarin.com/Releases/MonoTouch_4/MonoTouch_4.9.0

hope this helps
