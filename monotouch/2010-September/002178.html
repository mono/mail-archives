Hi,

I have a MonoTouch application with a window, inside a navigationcontroller and inside a tableviewcontroller.

In the constructor I have the following code:

        // Constructor invoked from the NIB loader
        public MyTableViewController (IntPtr p) : base (p)
        {
			btnAdd = new UIBarButtonItem(UIBarButtonSystemItem.Add);
			btnEdit = new UIBarButtonItem(UIBarButtonSystemItem.Edit);
			btnDone = new UIBarButtonItem(UIBarButtonSystemItem.Done);	
			
			btnAdd.Clicked += HandleBtnAddClicked;
			btnEdit.Clicked += HandleBtnEditClicked;
			btnDone.Clicked += HandleBtnDoneClicked;
			
			NavigationItem.LeftBarButtonItem = btnEdit;
			NavigationItem.RightBarButtonItem = btnAdd;
        }

        void HandleBtnDoneClicked (object sender, EventArgs e)
        {
        	Editing = false;
			NavigationItem.LeftBarButtonItem = btnEdit;
			NavigationItem.RightBarButtonItem = btnAdd;
        }

        void HandleBtnEditClicked (object sender, EventArgs e)
        {
			Editing = true;
			NavigationItem.LeftBarButtonItem = null;
			NavigationItem.RightBarButtonItem = btnDone;
        }

        void HandleBtnAddClicked (object sender, EventArgs e)
        {
			Console.WriteLine("Add button clicked");
        }

When I run it on the iPhone simulator both buttons are shown and clickable.
So clicking on edit changes the buttons to only show btnDone on the right and clicking on add shows "Add button clicked" on console.
Clicking on done restore the buttons to the first state.

However when I run it on the iPad simulator only the left one is clickable.
Clicking on add does nothing, clicking on edit changes the buttons and clicking on done does nothing.
