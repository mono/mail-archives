Pre MonoTouch I was able to keep iPod music playing after my app has
launched with this bit of code:

- (void)setAudioCategory {
AudioSessionInitialize (NULL, NULL, NULL, NULL);
UInt32 sessionCategory = kAudioSessionCategory_AmbientSound;
AudioSessionSetProperty(kAudioSessionProperty_AudioCategory, sizeof
(sessionCategory), &sessionCategory);
AudioSessionSetActive (true);
}



This was a long time ago, first gen iPhone days. Does this still work for
iOS 3.2 and later? I have tried the equivalent MonoTouch code, but it
doesn't work. Once my app launches, the iPod shuts off:

private void SetAudioCategory() {
AudioSession.Initialize();
AudioSession.Category = AudioSessionCategory.AmbientSound;
AudioSession.SetActive(true);
 }


For both the old and current app, I used OpenAL for my app's sound.


Anyone out there done this? I can't find much info out there regarding
AudioSession and MonoTouch, and all the ObjC samples I find match my
original objC code.
