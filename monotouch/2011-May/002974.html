WellŠ.. I found the problemŠ..

Inspired on Miguel's tip to do an UIAlert/MessageBox that waits for user
button press, I becae greedy and implemented a "modal" view controller, like
a modal Form in Windows, where you can open a dialog, the user work and when
finish, the parent form receive the response of the child form because it's
modal and continues the execution, so parent form stop the code execution
until child closeŠ

So when I open a view controller like:

// myView is declared "globally" inside the class
myView = new MyView();
myView.ModalTransitionStyle = UIModalTransitionStyle.FlipHorizontal;
myView.ModalPresentationStyle = UIModalPresentationStyle.FullScreen;
this.PresentModalViewController (myView, true);

After I have the inspired codeŠ:

while (!childView.isClosed)
{
NSRunLoop.Current.RunUntil (NSDate.FromTimeIntervalSinceNow (0.5));
}

SoŠ My modal-mutant-child view runs and parent stay stopped until child
closes. But then, all animation of bounces in web views, map views, image
viewsŠ All animation became crepy, like heavy and bad animatedŠ. Shame
myself ^^Š don't be greedy with the dark force.

Now I have to look for a way or the correct way to when a child view
dismiss, then must execute things like update info on parent, etcŠ All
different than windows, my sin.

If I continue with this path soon I'll able to write a book about 100 things
to do NOT do with MonoTouch hehe.Š Or maybe a MonoTouch uncyclopediaŠ
