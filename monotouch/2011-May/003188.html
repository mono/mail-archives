I would also add that at least using Sqlite.net in Windows, that if you don't
emclose an insert, update or delete in an explicit transaction it will
create one for you each time. If you loop through someting say 100 times, it
will create and commit 100 individual transactions and take approximately
200 times longer to execute. If you enclose the whole thing in 1 transaction
and commit at the end, it is extremely fast. Yes I did discover this the
long hard way. :) I'm not sure if this applies equally to the core database
stuff on the phone as well since I have only read from my iPhone app, not
updated yet.
