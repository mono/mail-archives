We have been using SvnBridge (http://svnbridge.codeplex.com/) successfully
for over a year now with fairly large projects. We're using TFS2008
currently, so I can't vouch for TFS2010 yet. There are multiple developers
on Macs using the same TFS project.

It makes TFS 'look like' Subversion... MonoDevelop can then do
source-control operations against it.
I would, however, consider another tool like Versions (
http://versionsapp.com/) as well because I find it a lot more 'TFS-like' in
terms of being able to do file diffs, merges, etc.
