Hi everybody.

in my application I have an UIWebView which should be open a "https"-Site.
The site has a self-signed certificate.

I've add the certificate to the system and it will be also registered inside
the application:

Code:

URI uri = new URI(url_of_the_website);

X509Certificate certificate = null;
certificate = new X509Certificate2("path_n_name_of_the_certification",
"pwd");
ServicePointManager.ServerCertificateValidationCallback = (sender, cert,
chain, ssl) => true;
HttpWebRequest req = (HttpWebRequest)WebRequest.Create(uri);

if(certificate != null)
{
   req.ClientCertificates.Add(certificate);
}

this.KWebView.LoadRequest(new NSUrlRequest(new NSUrl(url_of_the_website)));

this.KWebView.LoadFinished += delegate {
    indicator.StopAnimating();
};
}
......


If I will open a website with a public certification it is no problem. The
site will be loaded without any problems. But if I will load my site, which
has a self-signed certificate, the view will hang by loading the site.

Does anybody know how to show websites with self-signed certificate in a
UIWebView?

I am looking forward to your answers.
