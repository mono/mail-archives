I got an idea/feature request that would make life easier for developers
trying to do cross platform mobile development.

It would be nice if MonoTouch shipped with a Visual Studio 2010 plugin that
enables basic support for building MonoTouch projects in Visual Studio. I
know you can build MonoTouch projects if you manually edit the cproj files,
and copy the MonoTouch.dll over, so I'm only asking for a "smoother"
experience for something already possible. No more "unsupported project
type" when opening in VS2010, a nice MonoTouch icon on the class lib project
file etc.

The benefit would be when opening up projects with shared class libraries
linking files. It would be easier to get an overview of the entier solution,
and we could use tools like the "Project Linker" to keep shared class
libraries in sync (
http://msdn.microsoft.com/en-us/library/ff921108(v=pandp.20).aspx).

Would this be interesting for the MonoTouch team? Or is this something we as
a community should try to build using the VS2010 SDK's and extensibility
model?

- Jonas
