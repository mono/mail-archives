A few options
1. UIImageView - bind to TouchUp/TouchDown events
2. Create a UIButton and make the UIImage it's background - then bind to
TouchUp/TouchDown events
3. Do some subclassing

I prefer the latter for many situations.  Let's say you want to encapsulate
the whole thing as a single View

1. Subclass UIView into something usable
   public class MyImageRotator : UIView {}

2. In the constructor, add your elements
    MyImageView = new UIImageView(...);
    MyLeftButton = new UIButton(...)
    MyRightButton = new UIButton(...)
    AddSubview(MyImageView) // this will contain an image
    AddSubview(MyLeftButton) // this will be shown or hidden based on
touches
    AddSubview(MyRightButton) // this will be shown or hidden based on
touches
    MyLeftButton.Hidden = true;
    MyRightButton.Hidden = true;
    MyLeftButton.TouchUpInside += LeftButtonHandler;
    MyRightButton.TouchUpInside += RightButtonHandler;


2. Override TouchesBegan 
    In TouchesBegan
        MyLeftbutton.Hidden = false;
        MyRightButton.Hidden = false;

3. Override TouchesEnded
   In Touches Ended
        MyLeftButton.Hidden = true;
        MyRightButton.Hidden = true;

4. Handle left button
    private void LeftButtonHandler(Object sender, EventArgs e)
        Presumably switch the image or whatever

5. Handle right button
    private void RightButtonHandler(Object sender, EventArgs e)
        Presumably switch the image or whatever

Now, if you didn't want to use touch triggers on buttons, you could just
make the left/right buttons images and detect touch in TouchesBegan like:

public override void TouchesBegan(UITouches touches, UIEvent event)
{
    base.TouchesBegan(touches, event);

    PointF pt = (touches.AnyObject as UITouch).LocationInView(this);

   if(LeftButtonView.Frame.Contains(pt))
       LeftButtonHandler();
   else if(RightButtonView.Frame.contains(pt))
      RightButtonHandler();
}

It's a bit more complicated, but by creating a single view that handles
everything you want to have happen within it (via subclassing and
overrides), then you have a far more "portable" object.
