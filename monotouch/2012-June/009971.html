Using 'mtouch --xcode' is not a supported option and might not work in
future versions of MonoTouch.

In particular it is not meant to produce output you can insert in any
Xcode project, and this *will* fail in future versions of MonoTouch.

As I've mentioned before, the supported way of using both ObjectiveC
and managed code in the same app is to create static libraries of the
ObjectiveC code and bind those libraries in MonoTouch.
