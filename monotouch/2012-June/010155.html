It looks like the error is related to hitting that call too many times too
quickly. I've managed to prevent it by caching the reverse geodecoded
information, so if the user tries to do the operation again, it compares the
current GPS coords of the device to the last GPS coords that was geodecoded,
and if the distance between the two is < X, then use the cached geodecoded
information.
