> Is it intentional that the only way to set the runtime version to "Mono for
> iPhone" is to create a new Monotouch project? I'm finding a lot of year-old

Yes.

> HOWTOs (e.g. this one on JSON.NET) that talk about opening existing projects
> and recompiling them for iPhone, but there doesn't seem to be any way to do
> that without creating a new project (or hacking the project files by hand).

It's because of the way IDEs interact with the MSBuild project model -
you have the same restriction in VS with other project types that
target a different runtime, such as Silverlight, Compact Framework,
etc.
