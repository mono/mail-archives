> Thanks Michael - any comment on what this is used for? I didn't think
> we could get (non xcode) debug info out of release builds....

System.Reflection uses this metadata.  

So anytime you use reflection directly (typeof (...), GetType (),
MethodInfo, MemberInfo, etc).

For example, when you have an enum value, and you do "ToString()" this
uses Reflection to figure out what the value is.   Or to determine
whether a value should be printed out as a bitfield or not.

Or used for callbacks from web services (we look up method names and
properties that way).
