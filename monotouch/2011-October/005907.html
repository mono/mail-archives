Ah, yes, that could be it. We *think* that's a bug in Gtk+ (the GUI
toolkit we use) on Mac, but we've got some talented Gtk+ developers
working on fixing bugs we report to them. From a quick search in
Bugzilla, this may be the bug you are witnessing:

http://bugzilla.xamarin.com/show_bug.cgi?id=1371

Looks like Kris has a patch for that so we should make sure to include
that in our next MonoFramework release (MonoFramework is what bundles
Gtk+ and Gtk# for MonoDevelop's use).

I'll send a reminder to the team to include that patch in the next
cycle of internal MonoFramework builds so that QA can start testing
that patch and making sure it works and doesn't cause any other
regressions.
