I try to use the emulator as very little as possible.  I write a lot of unit
tests so that I'm not hitting the emulator to do manual tests of my business
and UI logic.  This is true for me whether I'm deving android or anything
else.

I sometimes use the old-timey method of writing runtime debug info with
Log.Write and looking at it with adb logcat (ala javascript 'alert'
debugging or php 'echo' debugging).  I don't use a device because I mostly
work on my 64-bit desktop, and my lousy phone doesn't have a 64 bit driver.
