On Jul 29, 2011, at 9:44 PM, bitjockey wrote:
> Just wondering if this could be the result of a bug in AndroidGameView or if
> I am just using it incorrectly.

It is very probably a bug in AndroidGameView; we know of an issue in which the underlying GraphicsContext disappears on e.g. screen rotation, which tends to make OpenTK crash. Oddly, targeting API level 7 instead of API level 8 makes things more stable (but not entirely stable).

Please file a bug so that we can keep tabs on this and improve matters. In the meantime, I would suggest targeting API level 7 for now.
