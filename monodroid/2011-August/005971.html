> The problem is frustrating because I don't have any more information on the
> problem.  Is there any other logging information that can be enabled?

First, try Mono for Android 1.0.3. That will hopefully fix your problem.

If that fails, you can enable additional logging, which will _hopefully_ provide crash output. You can do this by setting the `debug.mono.trace` system property to a value suitable for use with `mono --trace` [0]. For example, you could do:

	adb shell setprop debug.mono.trace E:all

and information about all thrown exceptions will be printed to the Android Debug Log.
