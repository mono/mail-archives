> Ok that's what I'm looking for sort of, now I'm stuck with trying to get Toast to work inside your example and how to call it from my background
> thread?   I don't seem to understand how to pass the UI into and out of non UI threads.  In Java, all I had to do was add a handler to the service class and pass that handler to the thread and then I could call the handler from the background thread

In Java, non-static inner classes can implicitly reference members of their enclosing type. In C#, that's not the case, so you merely need to be more explicit:

	class MyHandler : Handler {
		ContextWrapper self;

		public MyHandler (ContextWrapper self)
		{
			this.self = self;
		}

		public override void HandleMessage (Message msg)
		{
			Toast.MakeText (self.BaseContext, msg.Data.GetString("MSG1", ToastLength.Short).Show ();
		}
	}
