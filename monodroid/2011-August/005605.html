On Aug 1, 2011, at 11:10 AM, Narcís Calvet wrote:
> Re.: The current top-voted answer is the correct one -- you need an
> //application/@android:name attribute to cause Android to create a
> MyApplication instance. You should not need to ever explicitly create an
> android.app.Application instance.
> 
> But the same user at StackOverflow says I can't have more than one
> application tag in the AndroidManifest.xml. Is that correct? If so, how show
> I address that?

You can't have more than one <application/> element within AndroidManifest.xml, at least I don't believe that it worked when I last tried it. The docs also imply that there's only one:

	http://developer.android.com/guide/topics/manifest/manifest-element.html

	"[The <manifest/> element] must contain an <application> element."

Notice it doesn't say "one or more <application/> elements."

In short, don't have more than one 1 <application/> elements, and you should be fine.
