The class name used in JNIEnv.FindClass() must match an actual Java class packaged with your app. In this case, warren wrote a com.Exterminator.Game.FacebookHelper type, so the JNIEnv.FindClass() call needs to use the ~same value.

> The package of the facbookhelper.java is in my case "myapp.facebook".
