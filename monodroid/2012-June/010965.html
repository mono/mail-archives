Hi, thanks for the report. It gave us another inputs on binding 
implementation.

I tried it this weekend, and noticed that it complains about missing "R" 
class, which noticed me that R class is *not* included in the library 
project jar and hence fails for incomplete type resolution.

This needs to be "fixed" as there'll be similar library projects in 
missing "R" issue. (We used facebook SDK for this "Android library 
project" support which didn't hit this issue).

We'll deal with it in the same general "continue with missing Java 
types" approach.
