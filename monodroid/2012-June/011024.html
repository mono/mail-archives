huawei a400.  I think It's the stock cheap phone that a large (like 5-10
percent at least I would say) have. It runs 2.1 release something. 2.2
stuff runs fine on it, however. It's not hard to get the drivers so you can
adb it, either.  I've written several apps that run smoothly on it, so
although it has low hardware, it is possible to write fluid apps for it.

Another popular one is the skyrocket for a mid range phone. However, if it
works on the huawei then it'll run on the skyrocket fine. What I've heard
grumbles about are phones that have their own launcher (htc). When I was
writing java apps I came across this error that htc's launcher will do
something with either context or dialog and it can cause a null pointer.
Also, Launcher pro is a very popular launcher replacement.

Anyway, I'm glad someone cares about writing good apps so feel free to ask
me questions or I'll even test your app on my huawei. I also have a samsung
indule with a qwerty.
