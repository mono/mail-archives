I've been working around this issue for some time.  Since the designer file is generated on every build, I don't version the Resource.Designer.cs file.

Workaround steps:  close the solution, open the Source Control Explorer, and delete the Resource.Designer.cs file from there.  Then, create a new file with the same name (Resource.Designer.cs) in Windows Explorer and re-open your solution.  You will still be able to build, but it won't fail if you don't have Resource.Designer.cs checked out in TFS since it's not versioned in TFS anymore.

I've been meaning to report this issue, but I found this workaround and haven’t gotten around to filing a bug for it.  It would be nice if this was fixed, however.
