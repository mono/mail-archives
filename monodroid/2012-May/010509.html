I've noticed a slightly annoying issue since 4.2.1 came out (it's still
happening in 4.2.2), when building an android app from Visual Studio 2010.

If I make just a code change and then build the project and then start it
(without debugging) the changes don't get included. 

The only way to get round this is to "clean" the project first of all, and
then do a build. If I change one of the axml files or other resources then
this problem doesn't occur. Just when I make code changes.

I'm pretty sure this wasn't happening before 4.2 (though I could be wrong).

Any ideas why? I'm guessing it has something to do with trying to speed up
the build process. It's a lot slower after having done a clean.
