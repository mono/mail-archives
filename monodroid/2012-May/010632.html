I am evaluating Mono For Android with Visual Studio 2010 (on Windows 7 x64),
and am using the latest stable build (Mono For Android 4.2.2).

I have created a Hello World style "Android Application" project, which
builds fine.  When I start debugging, the "Select Device" window is
displayed with no running devices listed.  I click "Start emulator image"
and the "Start Emulator" dialog is displayed with my available images
listed.  I select my "MonoForAndroid_API_14" AVD image, click OK, and the
device starts in a new window (titled "5554:MonoForAndroid_API_14").  The
device window gets past the "Preparing SD card" message.

However, the "Select Device" window never updates to display the device. 
The Visual Studio status bar displays "Deploy started...", but the output
pane contains nothing.  List still not updated and output still contains
nothing after waiting 10+ minutes.

Any ideas how to get this working, or find out why it it not working?

My AVD image has the following settings:

- CPU/ABI: armeabi-7a
- Target: Android 4.0 (API level 14)
- Skin: WVGA800
- SD Card: 512M
- hw.lcd.density: 240
- hw.cpu.model: cortex-a8
- vm.heapSize: 24
- hw.ramSize: 512

Thanks.
