Hi, after the update to mono develop 3.01  all my code that involve custom
list view items
make the applications crash.
It seems to be impossible to have a custom control like this (text box
inside a relative layout)

<?xml version="1.0" encoding="utf-8"?>
<RelativeLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:id="@+id/container_item"
    android:orientation="vertical"
    android:layout_width="fill_parent"
    android:layout_height="fill_parent">
    <TextView
        android:id="@+id/textItem"
        android:textSize="44sp"
        android:layout_width="fill_parent"
        android:layout_height="wrap_content" />
</RelativeLayout>

If i remove the relativelayout the application run without problems but with
a relative layout parent
it always crashes without handling any exception.

My problem is that in most applications i have list views with custom
controls that has more than only one textview...so i need a relaivelayout
parent of all my controls (like the example below)
