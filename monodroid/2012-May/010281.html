hmm - from what I can find on the 'net this is an Android 2.2 and earlier
issue. my bad for testing on devices/emulators with 2.3 or newer...

the solutions seem to be:
* use newer emulator/device (2.3 or higher)
* move the file to /raw/ and adjust the code accordingly
* disable compression in the build, in Visual Studio Project Properties
Application tab there is a box for 'Leave the following resource extensions
uncompressed:', add ".xml" (no quotes) there
* if you are just testing, you could cut out a huge chunk of the
SeedData.XML so that it is less than 1Mb (just be sure it remains valid XML
- I suggest chopping half the records off the end)

I'll look into this a bit further to see what the best option is to update
the project on github, but for now perhaps one of those options will help
you? sorry for the inconvenience.

thanks
craig
