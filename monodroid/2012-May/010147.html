> So mandroid.exe is winexe - for remote command execution from android (device or emulator) on Windows?

The `mandroid.exe` program _was_ the "driver" of .apk creation, responsible for creating Android Callable Wrappers, compiling Java source into .class files, compiling .class files into a .dex file, generating the .apk, etc.

Most of this has since been moved into MSBuild tasks in order to speed up the build process.

The `mandroid.exe` program is a /target:winexe app so that it doesn't pop up Console windows whenever you build, as that's annoying. :-)

Generally, you should ignore it; you should never invoke it manually, unless you want to grab the version information from `mandroid --version`, and the new Version file will be an easier way to obtain that information.
