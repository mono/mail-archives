> to using this syntax:

> I see M4A 4.2 has support for annotations but the generated ACW doesn't seem
> to contain any 'ReportsCrashesAttribute'.
In short this seems to be a recent bug in 4.2.x (not sure when started). 
We'll fix the issue hopefully by the next release (4.2.5).

The reason why it fails is: when you try to bind acra-4.2.3.jar, you'd 
notice that there is a binding failure message, saying:

 > Class Org.Acra.Annotation.ReportsCrashes has unknown base type

Our binding tool rejects to bind this type because the type description 
xml does not actually tell the base class ("extends*" elements). It is 
however the Java bytecode does not tell base class (which 
is...java.lang.Object?) because(?) it is an @interface.

Now, back to the binding generator, it used to allow invalid base types 
which somehow allowed annotation types like this. Then we have improved 
our binding generator  to reject/ignore those "invalid" types, this kind 
of annotations became ignored, which is not good (since any annotations 
likely fall into the same result).

Thanks for the feedback.
