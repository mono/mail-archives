> On Feb 17, 2011, at 1:42 PM, Wally McClure wrote:
> > One strange thing I am seeing with the threadpool in Mono for Android is that this code works:
> > ThreadPool.QueueUserWorkItem(o => SlowMethod());
> > 
> > This code generates a compile error:
> > ThreadPool.QueueUserWorkItem(new WaitCallback(SlowMethod));
> 
> Because SlowMethod isn't compatible with WaitCallback.
> 
> WaitCallback [0]
> 
> public delegate void WaitCallback(object state)
> 
> Your method:
> 
> void SlowMethod()
> {/*...*/}
> 
> WaitCallback requires a parameter, while SlowMethod() doesn't accept any. To support `new WaitCallback(SlowMethod)`, SlowMethod() needs to take a single 'object' parameter.
> 
> - Jon
