I've been stabbing in the dark with bug 666610 and I seem to have it
narrowed down to my use of SQLiteOpenHelper.  If I don't use this (or a
class derived from it), then I do not get the build error.  It's weird
though: I have a repository class that inherits AndroidSqliteBase, and
AndroidSqliteBase is an abstract class that inherits SQLiteOpenHelper.

I register repository classes with TinyIoC like so:

        private static TinyIoCContainer RegisterTypes()
        {
            var container = TinyIoCContainer.Current;

            container.Register<Context>(Context);
            container.Register<IStockDataProvider,
DummyStockDataProvider>().AsMultiInstance();     // works
            container.Register<IPortfolioService,
PortfolioService>().AsMultiInstance();            // works
            container.Register<IPortfolioRepository,
AndroidSqlitePortfolioRepository>();     // error
            container.Register<IConfigRepository,
AndroidSqliteConfigRepository>().AsMultiInstance();               // error

            return container;
        }

If I comment those last two container.Register(...) lines then I don't get
the error when building.  If I remove AndroidSqliteBase from being inherited
by those classes, then I don't get the error when building (but I lose all
Sqlite functionality of course).  Is there something going on with
SQLiteOpenHelper during the build process that might cause this error?
