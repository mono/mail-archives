> Hi,

> Compiling correct mono code I have some errors. In the specific:
> 
> - RegexOptions.Compiled is not defined
> - InvalidDataException is not defined

The first is #if'd out of the 2.1 profile, which is why it's not in Mono
for Android.  The second is defined as a 2.0 profile item in mono.

Mono for Android is based on the 2.1 profile.  In general, I would say
that if it's available in the WP7 API we probably will want to expose it
in Mono for Android.  We will not be shipping the entire desktop mono
class library set though.
