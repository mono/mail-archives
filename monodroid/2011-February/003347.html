On Feb 26, 2011, at 7:03 PM, Dave Goughnour wrote:
> I am fairly new to MonoDroid and before going too far I had two questions that someone on this list might be able to answer quickly.
>  
> 1)      I assume MonoDroid can launch and interact with other Android activities similar to a traditional Android app?

Yes. Should be the exact same APIs, give or take a case change. :-)

> 2)      Is there any chance a MonoDroid app can utilize the Google Maps API?

There's a sample:

	https://github.com/mono/monodroid-samples/tree/master/GoogleMaps

Note, however, that this isn't ideal (by any means!), as it currently requires writing custom Java code which is merged into the final package. This _may_ change for 1.0 to support C# use of the GoogleMaps API, but MonoDroid 1.0 will not support using arbitrary Java libraries. Consequently, if you want to use a Java type outside of android.jar, some custom Java code will be required with 1.0, and the GoogleMaps sample shows how such Java code can be integrated and used from a MonoDroid application.
