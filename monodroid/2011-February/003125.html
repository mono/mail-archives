> On Feb 15, 2011, at 10:52 PM, Nathan Blevins wrote:
>> You are correct!  Why in the world are there 2 different locations for what it 1 set of constants in Android?  I am sure there is good reason, but it is confusing...
>
> The reason is enumerations and improved IDE support.
>
> The Android/Java approach is to use 'int' for constants, so if there is a restricted set of constant values you don't know what that set is without reading the documentation; the IDE and compiler can't help you.
>
> Phrased another way, which would you prefer to use?  This:
>
>         public MotionEventActions Action {get; set;}
>
> or this:
>
>         public int Action {get; set;}
>
