Apologies if I have missed something, but I wanted to throw this out and see what peeople's thoughts are.
 
I'm not quite sure when this changover happened, so apologies if this happened before p13.  I have some simple code that creates some menu items.  The code is:
menu.Add(MenuGroup, menuItemId + 20, menuItemOrder + 2, "Menu item 1");
It now looks like I need to do this:
menu.Add(MenuGroup, menuItemId + 10, menuItemOrder + 1, str0);
Java.Lang.ICharsequence str = new Java.Lang.String("Menu item 1");
The difference being that the ability to directly add a string as menu item has been removed and now I need to add these items based on adding an object that implements ICharSequence if I want to manually add a string.  I'm guessing that thischange is due to being true to the platform and that a string in the java world is different than a .NET string.  My concern is that this is a new thing to learn for a .NET developer coming to the platform and won't be obvious.  I think an overload should be added that allows a develop to directly add a string (didn't see one at this point).  This will make it easier for .NET developers targeting the platform.
