It seems like you already answered your own question.  Cast the enum
value to an int.  The docs for DialogInterface.OnClickListener contain
this:

"which	The button that was clicked (e.g. BUTTON1) or the position of the
item clicked."

So it looks like just an overzealous conversion of an int to an enum by
us, since it's apparently not always a button index.  We just need to
back out that conversion.  Can you file a bug please to make sure it
doesn't get lost?
