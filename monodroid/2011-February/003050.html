The KeyListener API did change in P12.  The listener callback OnKey
returns a boolean, and we have decided to follow the monotouch approach
of generating strongly typed delegates with property "attachers" instead
of EventHandler<EventArgs> and events for these for both consistency
across the platforms and for technical reasons.

Therefore, the KeyListener API is P12 is now:
                
public virtual void SetOnKeyListener (Android.Views.View.IOnKeyListener)

or

public delegate bool KeyHandler (Android.Views.View v, int keyCode, Android.Views.KeyEvent e);
public Android.Views.View.KeyHandler KeyPress { get; set; }


There were similar changes to LongClick and the other non-void
listeners.  There is some information about the current overall approach
here:

http://monodroid.net/Documentation/API_Design#Events_and_Listeners
