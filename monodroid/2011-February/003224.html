Again, the "problem" (such as it is) is that in Java only the builtin types (char/short/int/long/float/double) are value types, and everything else is a reference type. C# and .NET learned from Java's "mistakes" (oversight?) and added a "generic" value type mechanism, allowing the creation and use of "custom" value types (structs).

Consequently, value types can be used in MonoDroid code (and OpenTK uses them), but underlying Android/Java types _cannot_ be value types. This thus becomes an "education" issue, as not all Point/PointF/Rect/RectF types are alike (struct vs. class), and it becomes very important to know which one you're dealing with.
