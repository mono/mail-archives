On Feb 22, 2011, at 11:13 AM, Narcís Calvet wrote:

> I'm afraid I hit a bug with Android.Graphics.Point.



Java doesn't have value types, only reference types. Consequently, 
Android.Graphics.Point is a reference type, and thus `p1` aliases `p2`, 
which is why changes made to `p2` are visible through `p1`.



This is not a bug, this is Android.



If you want a copy, use the Point(Point) constructor:



   var p1 = new Android.Graphics.Point(0, 0);

   var p2 = new Android.Graphics.Point(p1);



   // these don't change p1

   p2.X = 1;

   p2.Y = 1;
