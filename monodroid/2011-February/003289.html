> Maybe? There is no documented one (that I could quickly find), but a search for [0] found [1], which implies a ~1MB asset size limit...in 2009. Whether that's still the case I couldn't tell you without reading the Android sources.

..and it looks like it hasn't been fixed in Android 2.2 (tested on Nexus One). My test was to add a 4000000-byte file [0] as an Asset and read it's contents [1]. Result:

	D/asset   (  403): Data exceeds UNCOMPRESS_DATA_MAX (4000000 vs 1048576)
	E/mono    (  403): [0xafd4a828:] EXCEPTION handling: Java.IO.IOException: Exception of type 'Java.IO.IOException' was thrown.
	E/*jonp*  (  403): Error reading from stream
	E/*jonp*  (  403): java.io.IOException
	E/*jonp*  (  403): 	at android.content.res.AssetManager.readAsset(Native Method)
	E/*jonp*  (  403): 	at android.content.res.AssetManager.access$700(AssetManager.java:36)
	E/*jonp*  (  403): 	at android.content.res.AssetManager$AssetInputStream.read(AssetManager.java:574)

In short, Android doesn't like large assets. Someone else mentioned the possibility of extracting the file into $HOME (aka $APPDIR/files), but how do you store + extract it? You're certainly not going to use the Assets API for that...

One possibility is to grab a .zip library, load your .apk, and use the zip library to read/extract the contents from the .apk [2]. MonoDroid does this for assembly loading (storing the assemblies uncompressed within the .apk and loading them during app startup).

 - Jon

[1] code, placed into Activity.OnCreate():
	var s = Assets.Open ("BinaryData.dat");
	try {
		if (s.CanSeek)
			Log.Info ("*jonp*", "Stream Size: {0} bytes", s.Length);
		int c = 0, n;
		byte[] data = new byte[1024];
		while ((n = s.Read (data, 0, data.Length)) > 0) {
			c += (int) n;
		}
		Log.Info ("*jonp*", "Read {0} bytes from the stream.", c);
	}
	catch (Java.Lang.Throwable e) {
		Log.Error ("*jonp*", e, "Error reading from stream");
	}
	catch (Exception e) {
		Log.Error ("*jonp*", "Unknown exception caught: {0}", e);
	}

[2] Use Context.ApplicationInfo.SourceDir to get the filename of your app's .apk:
