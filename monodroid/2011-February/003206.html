On Feb 19, 2011, at 10:02 PM, Wally McClure wrote:
> Can I access the application class so that I can modify the application's lifecycle events?

You should be able to.

Side note: You can use the Context.ApplicationContext property [0] to access the single Application instance from any Context subclass.

>  I have a simple class that I created that looks like the code below.  I am getting a VS error telling me that there is an unsupported exception.

As always, could you please provide the relevant `adb logcat` output so that we know what the exception is? Is it an Android error (found in `adb logcat`)? Is it some other error? Additional context would be useful.

However, based on the source you provided, the problem is that you're missing a ThisApplication(IntPtr) constructor:

	[Application]
	public class ThisApplication : Application {
		public ThisApplication(IntPtr handle)
			: base (handle)
		{
		}

		// other members as before
	}

This should have been mentioned in the `adb logcat` output, and I had hoped that the error message would have been sufficiently useful. Obviously I was wrong.

Application subclasses still require the (IntPtr) constructor; see also [1, 2].
