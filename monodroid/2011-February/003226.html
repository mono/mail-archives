On Feb 22, 2011, at 1:18 PM, Morten Nilsen wrote:
> I agree, the monotouch object should define the constructor as
>  
> Point(Point from) : this(from.X, from.Y) { }
>  
> To allow for parity with .NET on Windows

Uh, what? There is no System.Drawing.Point(System.Drawing.Point) constructor [0], so I'm at a loss for what parity this would provide with .NET on Windows. Furthermore, System.Drawing.Point doesn't require a Point(Point) constructor anyway, as it's a value type, and thus the assignment operator is more than sufficient on all platforms (MonoDroid included).

It's only Android.Graphics.Point that needs (and has) a Point(Point) constructor.
