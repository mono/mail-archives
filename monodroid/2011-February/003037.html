Is not so clear if you talk about local or remote databases.

If you talk about local database I think that you need to use sqlite
database using the native android interface (I suppose that the Greg's
sample application is on this way).
Otherwise, if you talk about remote database, so you can have a real
database server, I suppose that you can use sqlserver database backend
because is in the mono code.
Moreover I modify the the npgsql driver for PostgreSQL to work on Monodroid.

You can find my sample application here:

http://code.google.com/p/pgdroid

In this application the Npgsql postgresql provider is compiled in the
solution to simplify the debug step but I hope to release a library
version at soon as possible.



Hoping that this can be useful to you, best Regards, Torello
