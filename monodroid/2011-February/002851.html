The main biggest problem is that I would like to port some of my existing
market-published apps to monodroid. The market doesn't allow changing
package names, so if I'm required to publish the new version with a
different package name, that breaks upgrades. Additionally, if other apps
create intents that reference the package name, then they will be broken.

By enforcing extra rules on package names, that neither android nor the
android sdk require, you are preventing people from being able to move their
apps to Monodroid. Again, I wish to reiterate that I've used mixed case
package names for every android app I've written, both published and
unpublished apps used ant, Eclipse, and IntelliJ , and I've never had any of
the case problems you've expressed concerns about.
