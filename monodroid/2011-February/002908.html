On Feb 6, 2011, at 7:14 AM, Philipp Pätzold wrote:
> The deploying and debugging process in VS2010 is painfully slow:
> 
> 1) Building the application takes about >= 30 sec. to complete.
>     Compared to building a standard c# app thats pretty slow...

This is because the Build operation generates the .apk in addition to compiling your .dll's.

I believe that Preview 12 will be changing the Build command to just build the .dll and will add a separate command for creating the .apk.

> 2) Uploading the application to a running simulator is slow (>= 1 Min.)

This is due to two things:

 1. MonoDroid .apk files are, comparatively speaking, large.  The linker helps with this, but the linker slows down the build process (hence your > 30s build times).
 2. adb has gotten slower: http://code.google.com/p/android/issues/detail?id=13066

For Preview 13, we are planning on changing the developer strategy so that .apk's are smaller, which should help with build/deploy/debug turnaround times, but this can't fully work around adb needing ~10x the time to transfer data as previous Android SDK releases.

> 3) Waiting for the application to start and hit the first breakpoint in 
> OnCreate(...) breakpoint is the biggest pain though...

I believe that, to a large extent, this is due to the emulator being slower than molasses in January.  Try debugging on actual hardware; it should be much nicer.
