On Feb 24, 2011, at 12:48 AM, Paul Camacho wrote:
> There seems to be a size limit, as it's working for smaller .HTML files but not for large ones. Is there a size limit?

Maybe? There is no documented one (that I could quickly find), but a search for [0] found [1], which implies a ~1MB asset size limit...in 2009. Whether that's still the case I couldn't tell you without reading the Android sources.

Also realize that to render HTML the browser needs to do (potentially) lots of work -- parsing into whatever intermediate representation it uses, constructing a DOM and JavaScript parser (if necessary), CSS support, etc., etc. It's not terribly surprising that a moderately sized HTML document could bring the web view to it's knees with excessive memory use.

 - Jon

[1] http://osdir.com/ml/Android-Developers/2009-10/msg00320.html
