At System.Reflection.MonoCMethod.Invoke(System.Reflection.Binder binder, System.Object[] parameters, System.Globalization.CultureInfo culture)[0x0012d in /home/jon/Development/mono-droid/mono/mcs/class/corlib/System.Reflection/MonoMethod.cs:536…..
> 
> I'm guessing this is the WCF dynamic proxy generation — do I need to go the route I had to on MonoTouch and use the .net 2.0 proxies?

MonoTouch and MonoDroid share the same WCF infrastructure, it's just that MonoTouch doesn't support runtime codegen while MonoDroid does.

So anything that works with MonoTouch _should_ work on MonoDroid.

That said, I'd still expect the "normal" WCF mechanism to work...as long as it also works on desktop mono and/or Moonlight (which it might not, as mono's WCF implementation is still in-progress). Is there any way you could test this WCF code under Mono 2.10 for reference purposes? It's also possible that it's a linker issue; you might try disabling the linker and seeing if that changes anything (assuming a non-debug build, as in P13 debug builds no longer link).
