Hello,

 

I'm afraid I hit a bug with Android.Graphics.Point. Using code below p1 will
print as Point(1,1) instead of Point(0,0). That's not how a standard
WinForms applications works, it would return Point(0,0).

 

    [Activity(Label = "My Activity", MainLauncher = true)]

    public class Activity1 : Activity

    {

        protected override void OnCreate(Bundle bundle)

        {

            base.OnCreate(bundle);

 

            Android.Graphics.Point p1 = new Android.Graphics.Point(0, 0);

            Android.Graphics.Point p2 = p1;

            p2.X = 1;

            p2.Y = 1;

 

            var tv = new TextView(this);

            tv.Text = p1.ToString();

 

            SetContentView(tv);

        }

    }

 

I’m using latest Monodroid preview available with Visual Studio 2010.
