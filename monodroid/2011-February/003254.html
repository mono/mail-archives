On Wed, 2011-02-23 at 17:54 +0100, Christopher Ireland wrote:

> Although I do see Android.Graphics.Color derives from System.ValueType where 
> the equivalent Java class doesn't. Maybe we are talking about different 
> levels of binding pain here.

Android.Graphics.Color is just a static helper class in java to produce
integer color values.  There are no API elements which receive or return
an Android.Graphics.Color instance.  

This gave us some latitude to implement it as we chose, and we chose to
essentially copy in the named color set from System.Drawing.Color with a
manual implementation, since it is more extensive than the android set.
