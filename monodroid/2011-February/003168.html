On Feb 20, 2011, at 4:39 AM, Gareth Bradley wrote:
> 1) Is it a good idea to learn the native Andriod SDK/Java lang first before jumping on Monodroid.

Yes, and somewhat.

MonoDroid exposes the Android SDK types more or less directly (with some "interpretation" [0], e.g. get/set methods are turned into properties, events are added, etc.). Other pieces are are used directly, e.g. the XML UI description files. Consequently, familiarity with the Android SDK is a must.

Some familiarity with Java is useful in order to port and interpret existing Android samples.
 
> 2) Can i release MOnodroid Apps on the Android Market at the moment?

Not at this time, and see: http://monodroid.net/FAQ#Licensing

> 3) This is curiosity really on how Mono works. I know the C#/CLI/JIT are ECMA standards, but what about the .NET Framework.

The .NET Framework is also an implementation of the corresponding ECMA standards...with lots of extensions. Mono has lots of extensions as well, many to support compatibility with the .NET framework, and others as useful extension in their own right.

>  Were the namespaces and their corresponding classes/methods etc just interpreted for Mono or were the team given access to MS code?

For most of the framework the Mono team had no access to Microsoft source code and implemented the types based on public documentation and unit tests. The only exceptions to this are sources that Microsoft released under the MS-PL and were included into Mono unchanged, such as mcs/class/System.Web.Mvc.
