On Feb 2, 2011, at 11:55 AM, Brian Long wrote:
> On a related note, am I right in thinking that since MapView is not part of the Android core, that it won't be part of MonoDroid?

It won't be part of MonoDroid 1.0, that's correct.  Supporting ~arbitrary "third party" .jar files is a priority after 1.0.

We don't want to bite off too much at once (and often it feels like we've done that already).

Consequently, for those looking to use it now, the current (experimental?) solution is to use Build Action: AndroidJavaSource custom Java sources (if necessary for inheritance reasons) and Android.Runtime.JNIEnv to create and invoke methods on those types.  This is far from ideal, but (hopefully) it should be workable.
