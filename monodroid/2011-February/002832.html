On Jan 31, 2011, at 10:10 AM, Dan Russell wrote:
> So, it'll not be like Adobe Flash and Air where the runtime must be installed first?

As per the "Shared Runtime Changes" thread, we will not be supporting the shared runtime for deployment via the Android Market in MonoDroid 1.0, so the end user will not see any shared runtime to install or otherwise manage.  The shared runtime, for 1.0, will be a developer aid (to shrink package size and thus speed up deployment turnaround time).
