We would need to:

- Parse and understand android resource files.
- Override the default C# editor to:
   - Parse C# and pick out android string resources
   - Override Go To Definition to go to the resource file instead of the 
generated Resource.Designer.cs for string resources we find.

It would be a lot of work for a useful, but minor feature.  Likely there 
are other features that would give better bang for the buck.
