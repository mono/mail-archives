On Feb 23, 2011, at 10:01 AM, Christopher Ireland wrote:
> Can you please tell me if MONODROID's implementation of C# conforms to the 
> "Standard ECMA-334 C# Language Specification"?

Define "conform"...

ECMA-334 4th Edition (the current version at the site) is otherwise known as C# 2.0. The C# compiler bundled with MonoDroid supports C# 3.0 (which hasn't been standardized yet), and C# 3.0 adds a ton of stuff over C# 2.0 (LINQ, LINQ, and more LINQ in the form of extension methods, anonymous types, lambda delegates, Expression<T> code-as-data support...).

So, does a compiler implementing C# 3.0 conform to a standard for C# 2.0? I'm not entirely sure how to answer that...

Let's call it a superset. :-)

(There's also the "compiler extensions" that Microsoft CSC.EXE provides but aren't standardized, which mono's C# compiler supports for compatibility...)

Then there's the MonoDroid class library, which (kinda, sorta) conforms to ECMA-335 5th Edition. For example, System.Action through System.Action`8 are defined in mscorlib.dll (which conforms to .NET 4.0 and ECMA-335 but not Silverlight 4.0), though the assembly versions are "wrong" (ECMA-335 has <AssemblyVersion>4.0.0.0</AssemblyVersion>, while MonoDroid's mscorlib.dll is 2.0.5.0). Then there's the whole Parallel library (System.Threading.Parallel), which is defined in ECMA-335 but ~nobody provides)...
