On Feb 24, 2011, at 11:13 AM, Jonathan Pryor wrote:
> In short, Android doesn't like large assets. Someone else mentioned the possibility of extracting the file into $HOME (aka $APPDIR/files), but how do you store + extract it? You're certainly not going to use the Assets API for that...

Actually, you can, you just  can't store the asset in compressed form. Store the asset as an uncompressed file, and it works.

In the Visual Studio IDE, edit your Project Properties, and on the Application tab there is a "Leave the following resource extensions uncompressed" text input area. Add the appropriate file extensions (e.g. ".dat" for my test below), and the file will be stored in the .apk uncompressed, and you won't get an exception reading the contents.
