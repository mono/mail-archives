> I have a fairly extensive MonoTouch library that contains no iPhone specific code and I would like to use it in MonoDroid. The problem is that I can't reference that project in my solution since it tells me that it has an "incompatible target framework: vIPhone".
>
> The only way to make this work seems to be to create a new workspace and manually add all my source files. However I would like to keep that project in sync between my MonoTouch and MonoDroid solutions. Any idea how to do this? Visual Studio seems to have a way of doing this with the "Project Linker Extension" (http://www.gregshackles.com/2010/12/shared-libraries-for-windows-phone-7-monodroid-and-beyond/) but I need to do this on MonoDevelop.

That's the tool. I've been using it since 2010-09 and it is so far the 
best tool for code reuse.
You should determine smallest_common_denominator (subset), this is for 
me WP7 and
start from there.
I divide projects according to lasagne principle (layers: BusinessLogic, 
PesentationLayer...)
This increases number of projects, because You have Cartesian product 
Layers x Platforms.
But with project linker it is quite simple.

Relation is something like this

     WP7 < MDrd < Desktop/Server Version (there is sometimes RichClient 
Silverlight Project)

WP7 is source project every code file added to it is added as link in 
linked project
to file in source project.
To add source project, right-click on project > link project. Dialog 
asking source project appears,
which is usually subset project.

Tool gives possibility to exclude linking: if the code is WP7 specific 
(subset, let say reading XAP files)
then name the file XAP.Silverlight.cs and this source will not be linked 
in other projects.
Other exclusion extension words are WPF (for desktop) etc. This can be 
customized too, AFIK, but
haven't played with that...

UnitTesting. Due to the fact that last projects in the chain (supersets) 
are usually desktops, most
of the code can be tested with nunit or VS testing tool.
For platform specific unit testing havent got that far... My next todo.

Biggest problems for me were

    * Project and Namespace naming to find some useful and meaningful
      sort order in Solution Explorer...
    * determining relations between projects subsets/supersets
          o samples
                + ObservableCollection in WP7 but not in MDr => Mdr < WP7?
                + Networking in MDr there is IpAddress and WP7 no => Mdr
                   > WP7?
    * subtle differences in netfx
          o WP7
            System.IO.IsolatedStorage.IsolatedStorageFile.GetFileNames();
          o all other
            System.IO.IsolatedStorage.IsolatedStorageFile.GetFileNames(string
            pattern);
