I also ran into this issue developing a proof on concept application a few weeks ago.  I simply utilized the native Java.Net DatagramSocket/DatagramPacket classes and it worked fine (emulator only, not real hardware).  I wonder what the Davik VM uses that allows UDP whereas Mono cannot.



Although in a interesting twist, I couldn't get the native Android TCP Socket classes to work at all (couldn't access InputStream in Java.Net.Socket class when reading, not sure if a bug or I was just using it wrong) and had to use the standard .NET System.Net.Sockets for my TCP communication (which worked fine).
