Because "duplicate functionality" is also confusing, as developers don't know which one to use.  Thus, moving (if appropriate) is better than duplicating, especially before 1.0. (After 1.0, we want stability, so if things are to be moved, it must be done prior to 1.0.)

The question then becomes, if it's to be moved, then where to?  The whole reason it's in AndroidEnvironment is because I had a brainfart and had mentally merged System.Environment and System.AppDomain (oops), and I was somewhat thinking of the System.AppDomain.UnhandledException event.

In retrospect, I suspect that moving the event into Java.Lang.Throwable makes more sense, as the the event would primarily be responsible for mapping non-Throwable Exception types onto a Throwable type to be raised in Java when returning from JNI.  (I'm of course open to better locations...)
