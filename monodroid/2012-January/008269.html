On Nov 13, 2011, at 2:08 PM, Wally McClure wrote:

He thinks that an apk file is cpu independent due to java apps being cross
platform.  I'm not sure if thats true or not.  Anyway, questions:


No. The .apk file is a ZIP package that can contain native shared
libraries, and libmonodroid.so is a native shared library. Just because
it's a .apk does not mean it's cross platform (just as being Java does not
make it cross-platform; see also `native` methods and the fact that
`android.jar` doesn't run elsewhere yet ;-)
