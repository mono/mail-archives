JNI_OnLoad() occurs before managed code runs, unless you're calling Java.Lang.JavaSystem.LoadLibrary() for another library or you have some custom Java code which is using java.lang.System.loadLibrary(). Assuming you're not using any custom Java code and not using JavaSystem...I have no idea what would be causing this. :-(
