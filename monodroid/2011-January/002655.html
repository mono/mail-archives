Hi all,

Couple of things:

1) The generated resource class in a MonoDroid project is now called
Resources, but the system 'equivalent' is called Android.Resource. Should
that not be Android.Resources for consistency?

2) The equivalent of Java's ContextMenu interface is called
IContextMenuContextMenuInfo. That seems a little clumsy. Is this
intentional or can it be trimmed to IContextMenuInfo?

3) Is it likely to remain the case that adding resources and strings will
not automatically update the project's Resources class (unlike Eclipse,
which updated R.java when you save strings.xml, for example)? I realise it
updates when you do a build, but that means after adding a new string you
have to do a build in order to reference the string id currently.
