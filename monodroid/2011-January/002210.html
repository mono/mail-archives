On Jan 3, 2011, at 11:47 PM, Nathan Blevins wrote:
> Interesting.  If is directly insert the value "display_name" rather than that gigantic property name it works without a hitch.   

I think the real question here is, What Would Java Do?

The answer: http://developer.android.com/resources/samples/ApiDemos/src/com/example/android/apis/app/AlertDialogSamples.html

                String[] projection = new String[] {
                        ContactsContract.Contacts._ID,
                        ContactsContract.Contacts.DISPLAY_NAME,
                        ContactsContract.Contacts.SEND_TO_VOICEMAIL
                };
                Cursor cursor = managedQuery(ContactsContract.Contacts.CONTENT_URI,
                        projection, null, null, null);

Notice that Java is using ContactsContract.Contacts.DISPLAY_NAME, which is ContactsContract.ContactsColumns.DISPLAY_NAME, which _does_ have the value "display_name":

	http://developer.android.com/reference/android/provider/ContactsContract.ContactsColumns.html#DISPLAY_NAME

Thus, I think the problem is your use of ContactsContract.CommonDataKinds.StructuredName.DisplayName; it should instead by ContactsContract.ContactsColumnConsts.DisplayName (which I can't find on docs.monodroid.net, but I do see in my local build, so I hope it's in the current preview).
