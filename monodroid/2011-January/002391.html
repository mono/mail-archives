I have not been able to get my device to take a USB connection using the Google USB driver as Windows 7 fails to recognize to combo. That would save me a lot of grief. Here are the steps I performed though:

1.) Installed aLogcat and read in the device logs that it was a signing issue preventing my apk from installing.
2.) Fake deploy from VS2010 and grab the apk that has "signed" after the namespace name. (Is there a way to get this signed apk without deploying?)
3.) Tried to install and found out (via aLogcat) I need the monodroid runtime as well.
4.) Search and fail to find an apk for it in the SDK or on the interwebz.
5.) adb shell to my emulator and explore the filesystem.
6.) Find the apk in /data/app and adb pull it to local machine.
7.) Copy to device via USB FS mount and install.
8.) Voila, it all works.

Now the painful question: Short of USB deploy is there an easier way to do this? When I deploy via VS2010 it looks for devices and finds none so my only choice is deploy to emulator and then snag the files via those methods. Pain.
