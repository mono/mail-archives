> > Most likely any language that targets the CLR/Mono: VB.NET

> Alas, VB.NET is a brain-dead language which can't do anything without Microsoft.VisualBasic.dll (which contains the actual brains). You used to be able to build VB.NET apps w/o referencing Microsoft.VisualBasic.dll in .NET 1.0, but those days appear to be over.
> 
> We are investigating including Mono's Microsoft.VisualBasic.dll in the next release for VB.NET support.
> 
> > IronPython, IronRuby.
> 
> Dynamic languages pose another problem (in addition to the fact that we're not currently shipping a System.Core.dll/etc. which includes DLR support, though you could always include your own as was done with .NET 3.5):
> 
> http://monodroid.net/Documentation/Limitations#Limited_Dynamic_Language_Support
> 
> At present, if you want to inherit from any Android type (e.g. Activity, and remember that Java is very inheritance-heavy), you must use a "static" .NET language such as VB.NET (next release), C#, Boo, F#, C++/CLI (pure), etc. Anything involving runtime code generation will only have limited support.
> 
> This is very similar to the MonoTouch language restrictions (just for different reasons; see above URL for details).
