> public class AbsListViewScrollListener :

Does it help if you subclass Java.Lang.Object when implementing this
listener?  This is the recommended way to expose "java interface"
implementors.  It gives you the Handle implementation that you stubbed
in your implementation.  It also avoids instantiation of an intermediary
object when you pass it to a java method.

Based on the exception you are seeing, I believe you are hitting a known
problem.  The good news is that the next preview is going to deliver a
new GC synchronization mechanism and these sorts of issues should go
away.  We are testing it internally now.
