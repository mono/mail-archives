Unfortunately, with the latest 2.3 release of Google's Android SDK, 
copying a file to the emulator now takes 10 times what it did in 2.2 on 
Windows.  We have filed a bug with Google, but have not seen any 
resolution on it yet.

You might want to try a 2.2 emulator, sometimes that helps, sometimes it 
does not.  Also, deploying to a physical device is much speedier, around 
10-15 seconds for me.

We currently do not intend to move Mono.Android.dll to the shared 
runtime, as we anticipate it will be a fast moving target and we also 
believe that going forward we will have different versions of 
Mono.Android.dll for different target versions of Android.

What will mitigate this is when we begin linking Mono.Android.dll, which 
will strip out the parts you don't use, which results in large savings. 
  The first version of linking *should* appear in the next preview.

Hope this helps!
Jonathan
