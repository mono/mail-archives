Hey all, 

I am not sure if this is a reported bug or not, but I found a minor issue in setting the permissions for the application via the Visual Studio interface.  When I open my Application properties and selection the application tab, I have the option of setting Required Permissions.  When I navigate to READ_CONTACTS, select that option, and save, the application seems to save my permission.  However, when I open my properties back up, the checkbox is no long selected and my Android Manifest is not updated.  However, if I add <uses-permission android:name="android.permission.READ_CONTACTS" /> to my manifest and then open the properties, the checkbox is checked.  On my pc, it does not seem like the VS tools are saving changes made from the properties window for permissions.


Anyone else experience this?


Nathan
