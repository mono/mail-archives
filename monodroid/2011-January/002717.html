On Jan 26, 2011, at 2:03 PM, justin coon wrote:
> Is there a way to use a control created in Java?

Use Android.Runtime.JNIEnv:

	http://docs.monodroid.net/index.aspx?link=T:Android.Runtime.JNIEnv

The methods have the same semantics as the equivalently named JNI functions except the JNIEnv pointer isn't needed:

	http://download.oracle.com/javase/1.5.0/docs/guide/jni/spec/functions.html

For example, instead of:

	jclass c = (*env)->FindClass (env, "java/lang/String");

you'd do:

	IntPtr c = JNIEnv.FindClass ("java/langString");
