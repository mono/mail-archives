>From an enterprise development standpoint it is definately viable.
Enterprise applications are going mobile and MonoDroid and MonoTouch allow
us to target our internal mobile customers with our existing enterprise
development staff.  So for my world it does not matter what the size is but
I can see the concern for a market app.  However one excellent benefit that
might significantly outweigh any pure size increase is the ability with a
good MVC model allow you to more effectively reuse your code and target
different markets quicker.  You still need to have a platform specific
UI/View but the rest of your code will be fairly common.
