Hey all,

Not sure if this is a bug report, but I am trying to query the contacts
content provider using a managed query.  In the query, I am using a
selection string & args to limit the query to all contacts whose display
name is "test".  Here is the code that I was trying to use.

ManagedQuery(ContactUri, new string[] { },
ContactsContract.CommonDataKinds.StructuredName.DisplayName + "=?",
new string[] { "Test" }, "");


When I attempt this code, I am getting an error that says, column named data
2 does not exist.  When I reflect upon the structured name object, I see the
following values.

[Register("android/provider/ContactsContract$CommonDataKinds$StructuredName")]
			public sealed class StructuredName : Object
			{
				[Register("CONTENT_ITEM_TYPE")]
				public const string ContentItemType = "vnd.android.cursor.item/name";
				[Register("DISPLAY_NAME")]
				public const string DisplayName = "data1";
				[Register("FAMILY_NAME")]
				public const string FamilyName = "data3";
				[Register("GIVEN_NAME")]
				public const string GivenName = "data2";
				[Register("MIDDLE_NAME")]
				public const string MiddleName = "data5";
				[Register("PHONETIC_FAMILY_NAME")]
				public const string PhoneticFamilyName = "data9";
				[Register("PHONETIC_GIVEN_NAME")]
				public const string PhoneticGivenName = "data7";
				[Register("PHONETIC_MIDDLE_NAME")]
				public const string PhoneticMiddleName = "data8";
				[Register("PREFIX")]
				public const string Prefix = "data4";
				[Register("SUFFIX")]
				public const string Suffix = "data6";

				public StructuredName(IntPtr handle);

				protected override IntPtr ThresholdClass { get; }
				protected override Type ThresholdType { get; }
			}

Rather than giving me the appropriate value for the Display name (which
would be "DISPLAY_NAME" ) for the query, I am getting the "data1" string.
Seems like data 1-9 is not the appropriate values for these object members.
 Is this a bug?
