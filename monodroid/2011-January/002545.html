>>
>>> I thought I'd drop a little tip in here for other folks. If you are
trying
>>> to debug on certain devices (noticed it on a moto droid), it appears
that
>>> the wifi interface on the device likes to shut itself off or go to
sleep.
>>> This makes it really dodgy trying to attach the debugger. To get around
>>> this, you can set up a constant ping to the device. This seems to keep
the
>>> wifi "awake".
>>>
>>> Here's the windows command I ran:
>>>
>>> ping -t [IP of your device]
>>>
>>> After setting that up, I have a LOT more success device debugging.
>>>
