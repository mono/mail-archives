@Flo
> I would guess that image drawing functionality is a quite common task
> and there should be an official solution. I hope Mono is not running
> into an issue at that front, isn't Mono's goal to provide base classes
> which allow to share or build your code on many different target
> systems. Graphics certainly is a difficult topic, but I would have
> guessed a partial approach should be possible.

Which one? Microsoft has already fragmented in this area with
WPF/Silverlight. System.Drawing attaches to the immediate mode GDI+
graphics subsystem (ie. WinForms), while the System.Windows.Media
classes go to a retained mode rendering system.

See for example 
http://msdn.microsoft.com/en-us/library/system.drawing.bitmap.aspx
and
http://msdn.microsoft.com/en-us/library/system.windows.media.imaging.bit
mapsource.aspx

It would be *really* nice if UI code was portable between every .NET
platform, but that's not presently the case, even with Microsoft's
implementations. Some platforms (eg. Windows Phone 7) don't have
System.Drawing at all.
