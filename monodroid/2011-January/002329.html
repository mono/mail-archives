On Jan 7, 2011, at 4:28 AM, Niels Henriksen wrote:
> How big difference is there on MonoDroid and MonoTouch?

Gigantic.  That's why they have different names (HHOS). :-)

The "core class libraries" are the same between them -- mscorlib.dll, System.dll, etc.

However, what makes MonoTouch...MonoTouch, is monotouch.dll [0], which contains classes such as MonoTouch.UIKit.UIButton, and everything else that is used to make the UI, plus integration with the underlying CocoaTouch frameworks.  None of this is available on MonoDroid.

Likewise, what makes MonoDroid...MonoDroid, is Mono.Android.dll [1], which contains classes such as Android.Widget.Button, and everything else that is used to make the UI, plus integration with the underlying Android platform.  None of this is available on MonoTouch.

Consequently, a straight "port" is ~impossible, as the entire UI+contacts+maps+html+etc. APIs are completely different.

Instead, if you want to share code, you need to carefully design your app (e.g. via MVC/MVVM/etc.) so that platform-specific code is separated from platform-neutral code.  Core data structures, algorithms, file I/O, networking, etc. should all Just Work, if you do things properly.

 - Jon
