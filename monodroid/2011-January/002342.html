Hi folks,

Having trouble with a WCF method on MonoDroid that has a nullable parameter.

That same service is currently being used in Silverlight and Windows
Phone 7 so it works, hence must be something that MonoDroid (or the
emulator) isn't picking up.

My  WCF method: void GetTasksAsync(DateTime? DateTime.Now);

In the proxy Reference.cs it generates:
GetTasksAsync(System.Nullable<System.DateTime> runDate, bool runDateSpecified)

Seems that it generates runDateSpecified for backward compatibility.

Tried it with both true or false for the runDateSpecified parameter
and both attempts returned that above error message.

E/mono    (  298): [0xafd48828:] EXCEPTION handling:
System.InvalidOperationException: IsNullable may not be 'true' for
value type System.Boolean in member 'runDateSpecified'

or

E/mono    (  298): [0xafd48828:] EXCEPTION handling:
System.InvalidOperationException: IsNullable may not be 'false' for
value type System.Boolean in member 'runDateSpecified'

Seems that MonoDroid cannot handle a nullable
DateTime?  If that's the case, I could file a bug report if necessary.
