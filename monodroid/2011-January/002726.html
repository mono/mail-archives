On Jan 27, 2011, at 12:23 PM, Brian Long wrote:
> However I guessed the wrong place to do this and errantly set the Name
> property of my ActivityAttribute to some descriptive string, e.g. "My nice
> app", as in:
> 
> [Activity(Label = "@string/app_name", MainLauncher = true, Name = "My nice app")]
> public class Notepadv2 : ListActivity
> ...

Name is used for something ~completely different:

	http://docs.monodroid.net/index.aspx?link=P:Android.App.ActivityAttribute.Name

Quoting:

	The name of the class that implements the activity.

"My nice app" is NOT a valid Java class name, hence the error's you're getting.  You shouldn't set the ActivityAttribute.Name property unless you have a _really_ good reason (and this isn't one), and if you do need to set it, it needs to contain a fully-qualified Java class name, e.g.:

	[Activity(Name="my.Notepad")]
	public class Notepadv2 : ListActivity {...}

> than zero.

> Has this one come up before? A more helpful error would help track this down
> in a larger system

	http://monodroid.net/Documentation/Troubleshoot#monodroid.exe_or_aresgen.exe_exited_with_code_1

then provide the full message + stack trace from the Output pane.
