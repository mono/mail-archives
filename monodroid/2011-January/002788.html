On Jan 31, 2011, at 1:25 PM, Piotr Kryger wrote:
> Is there any way I can stop Monodroid from removing application files (from /data/data/* directory and from external storage)
> on each deploy to emulator?

There is currently no way to do this through the IDE, but from the "any way" department you can uninstall and install the packages using adb instead of through the IDE:

	adb install -r path/to/your-Signed.apk

`adb install -r` will "reinstall" your app, keeping the data directory.

	adb uninstall -k package.name

`adb uninstall -k` will remove the app, keeping the data directory.

Given that I don't see a "Create Signed Package" command within the VS IDE, the easiest solution is probably to manually run `adb uninstall -k package.name` before you use the VS IDE to install the app.
