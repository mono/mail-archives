Hello,

Yes there is a plan and some working code that takes arbitrary jar library,
but it does not mean it should work on *any* library yet. There are not a
few constraints that blocks importing (e.g. there are constraints on
ItemizedOverlay in our Google Maps binding).
And compatibility.jar is a beast, lots of features that are prevented by
those constraints. Maybe it won't be able to import it by MfA 2.2 timeframe,
and/or it will need not a little effort to get a working binding even in
some limited form.

Atsushi Eno
