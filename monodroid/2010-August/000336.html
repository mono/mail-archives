On Sun, 2010-08-22 at 19:54 -0500, William Smith wrote:
> The internal storage mechanism via the Android API Docs seems to work
> without directory involvement.

Directories can be used.  See e.g. ContextWrapper.GetDir()[0] and
ContextWrapper.GetFileStreamPath()[1] along with File.AbsolutePath [2].
These directories will be located within your application's directory,
e.g. underneath /data/data/YOUR_APP_NAME/files in the emulator.

(At least, this is how it works with Java code, so this is how it
_should_ work with MonoDroid; if it doesn't, that's a bug.)

> I'm assuming at this point that there aren't wrappers in place to make
> System.IO work with that.

I'm not entirely sure what a System.IO wrapper would do, as you can
obtain full path names from the Android API.  (I suppose
Environment.GetSpecialFolder() could be modified to use these folders,
but I don't know how feasible that actually is.)
