I guess one drawback with the approach I proposed  is that would be
that it's not 100% transparent to the user.  If the application that
provides the intent is missing, the user must take some action to
download it.  Your application can initiate the download/install
process, but the user must still give permission to install the
application.  Likewise if you update the application - the user will
be notified of the update and permit the install.

If you wanted the user to be completely oblivious to the whole affair,
it seems to me that the custom layout format/dynamically build the UI
would be the way to go.  Sounds like it would be fun to do, but a lot
of work.
