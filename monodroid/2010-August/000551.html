> On 8/25/2010 11:32 AM, Jonathan Pobst wrote:
> > Good news everyone!
> >
> > We are happy to release the second preview of MonoDroid (r7813),
> > addressing many of the issues encountered and reported so far.
> >
> > http://go-mono.com/monodroid-download/monodroid7813.msi
> >
> > Note this version comes with a new Mono shared runtime package, however
> > there is not yet a mechanism to automatically update the runtime on your
> > emulator/device.  You will need to uninstall the old runtime, and the
> > new runtime will be copied over automatically.
> >
> > To do this on the device:
> > Settings->Applications->Manage Applications->MonoDroid Runtime->Uninstall
> >
> > To do this from the your PC:
> > adb uninstall com.novell.monodroid.runtimeservice
> >
> > Fixes and enhancements in this version:
> > 632507 - Refresh Devices takes two clicks in VS
> > 632650 - changing the android sdk path setting still requires a VS2010
> > restart to apply the change
> > 632647 - MonoDroid installer over-writes previous install's android sdk
> > path setting
> > 632788 - Missing AttributeSet class (creating new View)
> > 633334 - No scrollbar in application properties window in VS2010
> > 632636 - CharSequence.ToString does not provide expected results
> > 632631 - HashMap<>  key is Java.Lang.Object instead of K
> > 632575 - generator.exe should make final classes sealed
> > 632869 - NRE adding null to LinkedList
> > 632968 - NRE passing null to Log.I
> > 633054 - R.cs generator does not validate resource names
> > 632470 - Error creating nested subclasses
> > 633675 - Compiling a ArrayList adapter produces incorrect intermediate
> > java code
> > Update the project template to match tutorials
> > Add Class Library Project template
> > Add OpenTK Project template
> > Add an explicit reference to mscorlib so that ReSharper will work on new
> > projects
> > Fix a warning in Visual Studio when adding an assembly reference
> > Fix an issue where minSdkVersion could get written multiple times
> > Fixed illegal instruction (SIGILL) on ARMv6 devices.
> > WebServices should not work
> > OpenTK projects should now work
> >
> > Please test it out and continue to provide us with great feedback!
> > http://monodroid.net/Support
> >
> > Thank you!
