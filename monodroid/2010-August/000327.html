> As a purveyor of the C5 libraries, this is something that I deal with often
> and easily. Many of the C5 collections collide with
> System.Collections.Generic. As part of the standard template for C5, we
> recommend the following using directives:
>
> using SCG = System.Collections.Generic;
> using C5;
>
> Now, whenever you want to use a non-C5 collection, you just prefix it with
> the SCG alias, as in SCG.ICollection<T>. This pattern can be applied to the
> Java namespaces as well, e.g.:
>
> using System;
> using JL = Java.Lang;
