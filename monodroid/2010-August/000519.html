Hello Amir,

 

1. Is there an easy way to have an obfuscator handle this file before it's
packed?

2. Do you have an idea if an obfuscated DLL would work on Android? (I will
run some tests myself soon).

3. If possible, allow the DLL to be encrypted or password protected.
Currently it's too easy to fetch.

 

Simple obfuscation will work with MonoDroid, but more advanced obfuscation
that is available on .NET will merely make your application break randomly.
Advanced obfuscators depend on quirks, bugs and special behavior of the .NET
runtime that is not part of the spec, and as such, is incredibly hard to
support.       

 

If you use any of the available .NET tools to obfuscate and report a bug, we
will likely just close the bug without even opening it.   

 

That being said, we have some plans to protect the assemblies, the question
is how important is this to our users.
