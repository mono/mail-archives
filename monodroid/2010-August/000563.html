Hi,

 

I'm trying to set a Tag for a View, but it seems to be overly complicated.

 

For example, in Java this code works:

Button button;

button.setTag("2");

 

But in MonoDroid it fails because:

1. There is no implementation of setTag(object) but only of
<http://developer.android.com/reference/android/view/View.html#setTag(int,
java.lang.Object)> setTag(int key,
<http://developer.android.com/reference/java/lang/Object.html> Object tag)

2. It won't work even if I use a key: button.SetTag(0, "2"). The compiler
complains that "2" isn't a Java.Lang.Object but it actually is an object!

 

I'd like to use any type of C# object as a Tag. Is there a quick fix or a
recommended way to handle this annoyance? 

 

Thanks,

Amir
