This is not a result of not having the 3.5 framework, this is a result 
of not giving monodroid.exe all the parameters it needs to work with.

If you really want to run monodroid.exe yourself (which isn't 
recommended), I would build in VS or MSBuild first so you can see what 
all gets passed to monodroid.exe.  (Set MSBuild verbosity in VS.)
