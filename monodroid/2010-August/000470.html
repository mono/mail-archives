I never did understand why we have to write code like:

Button my_button = (Button) findViewId(R.id.my_button);

Could MonoDroid be smarted than this and save us typing?
Why not generate member variables of correct types the way XAML does?
Or at least an inflatable "View" class containing public read-only
properties/variables?

class MyView
{
   public MyView(Context x)
   {
        // inflate the view
        // save ui refs in member variables
   }

   // ...

   public Button
   {
        get { return m_button; }
   }
}
