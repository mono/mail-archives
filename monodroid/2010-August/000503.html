On 8/25/2010 2:49 PM, Jonathan Pobst wrote:
> I haven't looked at the issue much, but one potential problem is that
> resources are only 'findable' after you have called SetContentView.
>
> So this works:
> SetContentView (R.layout.main);
> var button = FindViewById (R.id.myButton);
>
> This does not:
> var button = FindViewById (R.id.myButton);
> SetContentView (R.layout.main);
>
> Also, these methods are instance methods on the Activity class, so they
> can only be called if you have an instance of Activity, not in a
> generated resource class.

One option is to figure out a model for partial designer classes, like 
XAML/ASP.NET/MonoTouch etc.

For example, a variant of the "activity template" could add a 
FooActivity.xml layout file with a FooActivity.designer.cs file 
containing a partial activity subclass with all the accessors, and a 
partial FooActivity.cs file with the activity partial class and a ctor 
that called SetContentView. There are several problems with this - 
primarily, there isn't a simple, direct relationship between a layout 
and an activity.

Another possibility could be to implement the designer class for custom 
views/widgets, so there would be a "widget" template that would create a 
view resource with a custom class. This would be more straightforward to 
keep coherent.

Of course, with any of these models, the devil is in the details.
