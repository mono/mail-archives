>> files get related to resources on R.cs (or R.java).  I'd be really
>> surprised if Android allow you to dynamically load an XML layout file
>> and the "code-behind binary file".  Your best bet might be to ask on
>> the Android mailing lists to see if this is possible, though.
>>
>> What you could do is break your application up into multiple
>> applications, holding the UI parts.  So, you have one
>> "main-application" that holds your launcher, and the
>> "sub-applications" holding the various UIs.   You could then start up
>> the UI parts from the other "sub-applications" using intents.  Share
>> data between all the applications with your own content provider.
>> Then when you needed to change a UI (or add a UI part), just download
>> a new apk to update (or provide) the new activities and such.  Would
>> that handle the scenario you had in mind?
