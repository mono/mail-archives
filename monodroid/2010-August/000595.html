Now sure how everyone else is doing it, but I found this combination to work
very well:

- xUnit as the unit testing library: http://xunit.codeplex.com
- TestDriven.net as the runner in the desktop. http://testdriven.net
- Test project is a MonoDroid class library, which tests another class
library.
- All references (direct or indirect) used by the library under test must be
added as references on the test project
- They all must have CopyLocal=true
- Must run as administrator the strong name verification skipping once: sn
-Vr *,31bf3856ad364e35

With this, the monodroid assemblies will load on the desktop, and the tests
will be run happily by TDD.NET :)
