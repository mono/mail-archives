I am try to execute the HelloGridView example and I think I have found a
bug..

http://monodroid.net/Tutorials/Hello_Views/Grid_View

I have followed the steps of the example and added an ImageAdapter class.

The first error in the example was immediately found by VS2010 since the
code on the webpage for the ImageAdapter does not override the GetView,
GetItemId, GetItem, and Count with the override keyword.

After fixing that problem and placing the override keyword on the
aforementioned methods the compilation succeeds, but the build fails with
the message

"Error 1 "monodroid.exe" exited with code 1. C:\Program Files
(x86)\MSBuild\Novell\Novell.MonoDroid.Common.targets 208 2 HelloGridView
"

In the build output that you can find at the bottom of this email I have
highlighted what the build process thinks is the problem. It is

"helloGridView.ImageAdapter is not abstract and does not override abstract
method getView(int,android.view.View,android.view.ViewGroup) in
android.widget.Adapter"

The interesting fact is that I do override public abstract
View<http://www.aisto.com/roeder/dotnet/Default.aspx?Target=code://Mono.Android/Android.Views.View>
*GetView<http://www.aisto.com/roeder/dotnet/Default.aspx?Target=code://Mono.Android/Android.Widget.BaseAdapter/GetView(Int32,Android.Views.View,Android.Views.ViewGroup):Android.Views.View>
*(int<http://www.aisto.com/roeder/dotnet/Default.aspx?Target=code://mscorlib:2.0.5.0:7cec85d7bea7798e/System.Int32>position,
View<http://www.aisto.com/roeder/dotnet/Default.aspx?Target=code://Mono.Android/Android.Views.View>convertView,
ViewGroup<http://www.aisto.com/roeder/dotnet/Default.aspx?Target=code://Mono.Android/Android.Views.ViewGroup>parent);

"""

        // create a new ImageView for each item referenced by the Adapter
        override  public View  GetView (int position, View convertView,
ViewGroup parent)
        {
          ...
"""


But if you look carefully the error says I don't override getView <- notice
the lower cased character. Thus the error is correct, I have not overridden
getView, but I have overridden GetView as I should have according
to Android.Widget.BaseAdapter (Assembly: Mono.Android, Version=0.0.0.0 )

I think this is a bug..
