I've been using XmlDocument on MonoDroid all morning.

It would make sense to keep it on MonoDroid as there is already precedent
for it in native Android, i.e. Java's own "Document" DOM implementation,
although there's been plenty of times in native Android where we've had to
fall back to using the XML Pull Parser because we keep running out of
memory.

It is a total pain not having a DOM on iOS...
