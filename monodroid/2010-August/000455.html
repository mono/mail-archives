.Net first tries to run an assembly with the runtime (mscorlib) it was 
compiled against.  In this case, the runtime is 2.0, since 2.0, 3.0, and 
3.5 all share the same runtime.

(You have to remember that 3.0 and 3.5 did not come with a new runtime 
or mscorlib, it basically just added some new assemblies.)

I think there are three options:

- We compile monodroid.exe against 4.0, which means it wouldn't run on 
Mono until we get a new release of Mono that supports 4.0.
- You remove the 2.0 framework from you computer.  When 2.0 is no longer 
available, .Net will try to run it on the latest framework available: 4.0.
- We simply say 3.5 is required if you have 2.0 installed.
