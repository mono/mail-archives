On 2010-08-20, at 4:02 PM, Johannes Roith wrote:

> Hello, everyone!
> 
> Will the mono runtime be shared on the devices when multiple MonoDroid
> apps are installed from the android store?

The shared runtime is shared between all applications yes.
> 
> Will it be necessary to include it in each package or will there be a
> separate Mono application on the store that could be used as kind of a
> dependency?

We will likely provide both options in the final product in some for or other, but we cannot to commit what this will look like at this time.

> 
> Can somebody elaborate on the Java/Mono shared GC that was mentioned
> on the website? How will it be shared and what will be the effect? How
> is this currently done? That is, when do objects that may be collected
> on either side (Java/Mono) become available for collection currently?

Currently its possible for cross-vm boundaries to cause a GC deadlock; we will be fixing this in a future preview.

> 
> Finally, are there currently any android devices that do not allow
> applications involving native code (which in the monodroid case would
> then be incompatible)? Are vendors required to allow this (would be a
> plus, of course)?

To be certified as a android device you must support the NDK from 1.6 onwards.  We do not currently have support for x86 in p1, but we will add this.
