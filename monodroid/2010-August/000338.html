Part of this is because the Android community is largely derived from the
Java community, where we never really had a lot of "designer demand" going
on. Remember, the Java space went to Web apps fairly early in its adoption
arc (by 1998 we're all writing servlets and thinking we're so smart for
doing so), and then the emphasis was on MVC apps (which somehow got
translated into "drag-and-drop UIs suck", even though I can't find anybody
who can elucidate why I can't unit-test an MVC-based UI where the view is
drag-and-drop-created).

Dare I say it? The Java guys kinda don't know what they're missing; if
Monodroid could give a good designer experience without losing the
unit-testing facilities that the Java community finds absolutely critical,
it'd be all kinds of win.
