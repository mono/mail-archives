Hi guys! The integration with Visual Studio is beyond fantastic! I'm really psyched!

Anyhow, I just ran into the same problem when I did my initial port of Mono to Android. Basically, if you create a .NET Thread, do some allocations, and then System.GC.Collect is called (doesn't matter from which thread), the application will hang/crash. I tracked this down to the pthread_kill ing GC_suspend_all (libgc) failing with an EPERM error when trying to stop anything but the main thread (which was created in dalvik land). I tried a few other signals for SIG_SUSPEND, to no avail.

Here is exactly where the failure occurs:
http://github.com/mono/mono/blob/master/libgc/pthread_stop_world.c#L342

Here is a sample application you can use to easily reproduce the problem:
http://gist.github.com/539485
