I suspect what you are referring to as "package signing" is the
combination of several steps, several of which aren't performed on debug
builds.  One of the big ones there is linking, where we reduce the
packaged code down to what is necessary for your program to run.  This
is a fairly big operation, and is most likely what you are sensing as
extra slowness.

You have some control over the amount of linking that is done via the
project options, but the less linking, the bigger the end product.  We
realize this is a bit of a bottleneck and we intend to improve the
performance of the release build packaging logic in future releases.
