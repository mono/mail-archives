I'm using the same code for background threads on Windows Mobile 5 to 6.5,
on iOS and on AndroidŠ, made in 2009. This was inspired on threads explained
on book Mobile Development Handbook, wrote by Andy Wigley, Daniel Moth and
Peter Foot, chapter 11, pages 405 to 435. I think that is a good read.

I don't know if today there is better ways, after all, this book is for
native compact .NET that came on Windows mobile 6.5, that is 2.0 version.
But it runs exactly equal on WM, iOS and Android. Following there is a
sample of what I make with that:

- After write user and password, a thread is opened for control and wait
- It opens a worker thread that do the connection to a webservice and
download all data, showing a progress bar for each thing downloaded
- If some problem appear, the worker thread ends with status of error, or
success if runs to the end and can be canceled (clicking on a cancel button)
by user without killing thread, but checking for user trying to cancel.
- At the end, control thread analyze the status of worker thread, to see if
was ok of had a problem, to decide what to do and to show to the user
