On Apr 12, 2011, at 8:11 PM, Hugo Gomes wrote:
> does this mean i should not call run on ui thread when i'm already running on ui thread? is there any way to know if i'm already running on the ui thread?

Could you please file a bug with a solution that reproduces this issue? I can't reproduce it for the trivial example of:

		protected override void OnCreate (Bundle bundle)
		{
			base.OnCreate (bundle);

			// Set our view from the "main" layout resource
			SetContentView (Resource.Layout.Main);

			// Get our button from the layout resource,
			// and attach an event to it
			Button button = FindViewById<Button> (Resource.Id.myButton);
			RunOnUiThread(() => button.Text += " [via UI thread!]");
			
			button.Click += delegate {
				button.Text = string.Format ("{0} clicks!", count++);
				RunOnUiThread (() => button.Text += " [from UI thread!]");
			};
		}
