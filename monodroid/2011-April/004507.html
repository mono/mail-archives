Presumably you want to call AlertDialog.Builder.SetOnKeyListener(), right?

	
http://docs.mono-android.net/index.aspx?link=M%3aAndroid.App.AlertDialog.Bui

Unlike SetItems(), there are no SetOnKeyListener() methods that take a
delegate parameter, nor do I see any AlertDialog.Builder properties or
events that would be useful either, so I believe that you would need to
implement the Android.Content.IDialogInterfaceOnKeyListener interface:

	
http://docs.mono-android.net/index.aspx?link=T%3aAndroid.Content.IDialogInte
rfaceOnKeyListener

For example:

	class EnterListener : Java.Lang.Object,
IDialogInterfaceOnKeyListener {
		public  bool OnKey (IDialogInterface dialog, int keyCode,
Android.Views.KeyEvent e)
		{
			// TODO
		}
	}

Then pass an instance of EnterListener() to
AlertDialog.Builder.SetOnKeyListener().

> Can you explain a bit more what you mean by Java allowing 'anonymous'
implementations ? and what is the preferred way of declaring an
onkeylistener in c#/mono?

Java allows "anonymous inner classes" -- classes that have a
compiler-generated name instead of an explicit name. Your original Java code
had an example of one:

> AlertDialog.Builder builder = new AlertDialog.Builder(this);
> builder.setTitle("Pick a color");
> builder.setItems(items, new DialogInterface.OnClickListener() {
>    public void onClick(DialogInterface dialog, int item) {
>        Toast.makeText(getApplicationContext(), items[item],
Toast.LENGTH_SHORT).show();
>    }
> });


Here, `new DialogInterface.OnClickListener() {...}` is an anonymous inner
class.

C# does not have anonymous inner classes. Consequently, to port such
examples to C# you need to convert the anonymous inner class into an
explicit class that implements the IDialogInterfaceOnClickListener interface
(as I suggested in an earlier email).

Turning anonymous inner classes into "normal" classes tends to be painful,
which is why many will prefer the anonymous delegate/lambda based
implementation instead, when possible.

 - Jon
