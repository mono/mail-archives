On Apr 27, 2011, at 3:23 PM, Scott R. Dahl wrote:
> I'm not that familiar with Java programming. The examples in java have a function definition like so:
> Quote:
> protected void onProgressUpdate(Integer... progress)
> What the heck is the ellipses? I have no clue how to translate that into C#. 

The ellipses is Java syntax for C# params arrays. `progress` is an Integer[] array.

> I'm also getting errors when trying to send Void as an argument to the AsyncTask<String,Void,Void> creation. 

This isn't possible, as System.Void isn't a type that can be used with generics (sadly). Instead, use `object`, and just use null and/or ignore the parameters.

> Any suggestions on how I could work around creating a background task in monodroid would be greatly appreciated.

As Greg suggested, there's always ThreadPool.QueueUserWorkItem(). There's also System.Threading.Tasks.Task (the Task Parallel Library), both of which would promote greater portability with other platforms.
