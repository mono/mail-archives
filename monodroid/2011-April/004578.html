> I've not done any XAML work before.

Just to clarify, Mono for Android doesn't use XAML, it uses .axml files (Android XML), which is what Android uses for layout XML. It's conceptually similar (-ish), but not exactly like XAML. (One difference among many: the treatment of element attributes. In XAML, element attributes are just properties; in Android XML, they're collected into an IAttributeSet collection and passed to the type constructor.)
