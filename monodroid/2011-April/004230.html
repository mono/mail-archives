Actually, when I type login and password on emulator, the keyboard is
shown. When I click on OK button to start validation and database data
download and schema mount, the keyboard stay visible...

There is 2 threads working on that, one opened as a control thread, and
another as a worker thread. The first thread begin and open the worker and
join(), to wait for worker thread termination to manage errors or success.
So it runs without theoretically freezing UI main thread. At least it
works on Windows Mobile and MonoTouch.

I think maybe need to buy a Samsung S (the same of client) to make real
tests, to see if execution is more quickly and if things runs better...

Samples: 
- HTC Touch (200Mhz) on Wifi with SQL CE making Pulls take 1:20 minutes to
download database.
- iOS emulator takes 40 seconds to do the same but the Pull is from
webservice
- iPad takes about 60 seconds to do this
- Android emulator are taking between 15 and 20 minutes.... (crying voice
