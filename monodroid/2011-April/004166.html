On Apr 7, 2011, at 11:27 AM, Michal Sartoris wrote:
> Does anybody have any idea how to build documentation (i.e. chm or html) for library (dll) build with Mono for Android?

Mono for Android assemblies are ye normal assemblies (modulo assembly references and assembly version numbers, e.g. they reference mscorlib.dll 2.0.5.0).

I know that Mono's mdoc works on them (as we use mdoc to generate docs.mono-android.net). I would assume that .NET Sandcastle works, though I haven't actually tried, and you should certainly be able to use inline XML documentation under .NET (though, sadly, this won't currently work with smcs on OSX)...
