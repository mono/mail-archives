> The strange this now is when I subclass AsyncTask I also have to include
> RunInBackground, otherwise it wouldn't compile. So I have both
> RunInBackground and DoInBackground, and they both fire.

RunInBackground is a generic version of the DoInBackground method that
does the conversion to the Java objects exposed by the non-generic base
class method.  If you are implementing an AsyncTask<,,> you only need to
implement the abstract RunInBackground, not the already overridden
virtual DoInBackground method.

The docs for that class were imported from the java docs and therefore
don't do a good job of describing the difference required to expose this
class as a C# generic class.  Sorry for the confusion.
