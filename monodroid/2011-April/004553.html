Hi again,

I looked at the Mono implementation of

XmlConvert.ToDateTime(string,XmlDateTimeSerializationMode)

and found, that Mono *requires* that the date string is parsable by
"localDateTimeFormats". The exception observed probably is due to the fact,
that my date time string is in UTC format.

This behaviour, however, is different to the documented (and implemented)
Microsoft .NET framework behaviour where the parameter
XmlDateTimeSerializationMode is documented as:

"One of the XmlDateTimeSerializationMode values that specify whether the
date should be converted to local time or preserved as Coordinated Universal
Time (UTC), if it is a UTC date."

The behaviour expected by my code thus is, that the stored UTC date time is
parsed back and converted to a local date time. This works in .NET, but
produces the given exception on Mono.

Looks like a small Mono/MS.NET incompatibility to me.
