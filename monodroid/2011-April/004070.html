> Following on from this, while i was trying to work things out I was
> checking the docs for FileInputStream and it covers two things:
> 
> 1) It urges me to use a Java.Io.FileReader to read characters, which
> looks good. However the FileReader does not appear to accept a
> FileInputStream

We do not recommend using the java io streams.  We have added support to
convert java streams to System.IO.Stream under the hood in most places.
Some places may have leaked through because they exposed subclasses
instead of the base stream types which are used for the overwhelming
majority of exposed streams.  There should be no reason to use a
java.io.FileReader.  If you find yourself forced to do that, we've
missed something and a bug report is reasonable.

> 2) The code snippet suggests passing it to a BufferedInputStream to
> add buffering. However I cannot find BufferedInputStream in the
> MonoDroid docs (despite it being around since API Level 1), nor in the
> libraries.

Much of the java.io package is not exposed in our binding. That is
intentional.  We encourage you to use System.IO for StreamReaders.
