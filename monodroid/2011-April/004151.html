> One thing I just noticed when I was checking over my code that I hadn't
> noticed before is that Intents implement IDisposable.  I currently do not
> treat them as such.  This may or may not be a serious problem for my app,
> but before I go refactor crazy, is it a major problem if these Intents go
> un-disposed?

All Monodroid types that bind a Java object implement the IDisposable
interface to give you control regarding the strong reference that is
kept from the C# world to the Java world.

Although IDisposable is used culturally for objects that hold
references to expensive objects like files, or images, in Monodroid's
case this is extended to all Java object wrappers so you can not
assume that just because it implements IDisposable the object needs to
have its Dispose method called right away.   You should dispose
objects that you know hold precious resources like file descriptors,
database connections, image pointers and other heavy objects like that
and ignore them otherwise, the GC will take care of them.
