For the benefit of others - this causes an error 
  System.Xml.Linq.XDocument xdoc = new XDocument();
                xdoc.Add( new XDeclaration("1.0", "utf-8", "yes"));
                
But this compiles and functions fine and dandy 
System.Xml.Linq.XDocument xdoc = new XDocument();
                xdoc.Declaration=   new XDeclaration("1.0", "utf-8", "yes");
                
Also for the benefit of anyone else stumbling through this - most of the
examples show the construction of the xdocument nested thus 
XDocument xDoc = new XDocuement();
xDoc.Add(new XElement("fieldname","Fieldcontent"),new
XElement("secondfieldname","more content"));

I needed to put my additions of fields in a loop from an unknown length
array 
I discovered that it is perfectly workable to do this (all the examples fail
to tell you :-) ) 

Var rootele= new XElement("Root","");
rootele.SetAttributeValue("generated", DateTime.Now.ToString());

for (int n = 0; n < (myCrew.Count()); )
                     {
                       var crewele = new XElement("crew","");
                         if (myCrew.Ui[n] == null)
                         {
                             myCrew.Ui[n] = UID.ToString() + n.ToString();
                         }
                         crewele.Add("Ui",myCrew.Ui[n].ToString());
///etc etc
				n++;
				}
                rootele.Add(crewele);
                xdoc.Add(rootele);
                xdoc.Save(filename);

Generally, you should prefer System.IO over Java.IO, as there are fewer
cross-VM issues (and your code will be more portable to other platforms, and
things like XDocument expect System.IO, not Java.IO...).
