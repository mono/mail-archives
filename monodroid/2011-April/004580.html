On 27/04/11 22:29, "Jonathan Pryor" <jpryor at novell.com> wrote:

>On Apr 27, 2011, at 8:28 PM, Dean Cleaver wrote:
>> I presume any axml I can find (even if it's java sample code) will work
>>in MonoDroid?
>
>Yes. Note, though, that on Android it's just .xml. We renamed it to .axml
>so that we could provide some code completion support.
>
>> And could I then enumerate the attribute sets?
>
>Sadly, no. The actual set of supported attributes is defined by Java
>source code; there is no XML schema/DTD/etc. (The one we have was
>painstakingly created by grepping Java source.)
>
>Furthermore, due to it's design, any View subclass can also be used
>within XML, and any subclass can add support for additional attributes.
>There is no way to declaratively determine the full set of valid XML
>attributes.
>
>(For this and other reasons, I vastly prefer XAML as Microsoft defined
>it.)
>
>> Like you mentioned "RelativeLayout.LayoutParams.AddRule()" - does the
>>LayoutParams have a Rules collection I can read etc?
>
>You can read the Java source ;-)
>
>Again, there is no Android mechanism to programmatically/declaratively
>determine the supported set of XML attributes. This includes
>RelativeLayout.LayoutParams.AddRule() & co.
>
>If you want to play around a bit, you could investigate xamldroid:
>
>    https://github.com/atsushieno/xamldroid
>
>As the README says, it doesn't work yet, but it's an effort at allowing
>Android Views to be used with XAML. I would love to get something like
>this into Mono.Android.dll proper (no roadmap, no timeline, no
>announcements yet!), but as part of such work we'd need to find all the
>bizarro "corner cases" such as RelativeLayout.LayoutParams.AddRules() and
>provide a XAML-callable form. xamldroid may be the ideal place to store
>such information, e.g. provide a XamlRelativeLayout.LayoutAlignRules
>property which would call AddRules() in the setter.
>
>I have no idea how feasible/practical this is, but it would make an
>interesting "research project" if someone had the time. :-)
>
