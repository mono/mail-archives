On Apr 2, 2011, at 12:18 AM, Matthew Groves wrote:
> According to logcat, it looks like "ERROR: Unable to detach current thread from the Java VM!" 
...
> I have no idea what this means, or how to fix it.  I can't reproduce it manually, but I am a far more gentle user than a monkey (I hope).  Any ideas?

We believe that the message is relatively harmless; it happens because JavaVM::DetachCurrentThread() returns a failure, which (as per Dalvik source) would be because we never called JavaVM::AttachCurrentThread(), which makes no sense (as we attach all Mono-created threads, and if AttachCurrentThread() fails we abort the process...).

We would love to have a test case that demonstrates the issue so that we can investigate further. :-)
