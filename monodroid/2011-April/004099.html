It's already been discussed before, try a search on this mailing list. The
debugger is still slow. 

For now if you want to make it quicker, don't use the Watch feature at all.
When you do use it, remove all expressions before taking another step. I
wouldn't leave there more than one expression.

As for timeouts, put your expression's value in a variable before
compilation so you can check its value during runtime. I couldn't find any
other way around this.

 

Btw, I'm with you on this for making it run the way we're used to.
