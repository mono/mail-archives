On Apr 7, 2011, at 2:03 AM, Miguel de Icaza wrote:
> You should dispose
> objects that you know hold precious resources like file descriptors,
> database connections, image pointers and other heavy objects like that
> and ignore them otherwise, the GC will take care of them.


Note, though, that disposing the Intent (and any other Java object) isn't "bad," and will allow the Java GC to collect the object faster than it would otherwise (so early disposal could be useful in lowering overall memory consumption). You just don't need to be quite as "anal" for most types.

This is also mentioned in:

	http://mono-android.net/Documentation/GC#Object_Cycles
