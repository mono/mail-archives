No. The [BroadcastReceiver] and [IntentFilter] custom attributes serve to generate the appropriate XML within AndroidManifest.xml, so all you actually need is:

	[BroadcastReciever]
	[IntentFilter(new[] {Android.Content.Intent.ActionBootCompleted},
			Categories = new[]{Android.Content.Intent.CategoryDefault})]
	public class OnBootBroadcastReceier : BroadcastReceiver {
		// ...
	}

and an appropriate body for the OnBootBroadcastReceiver type, e.g.

	public override void OnReceive(Context context, Intent intent)
	{
	}

See the BroadcastReceiver documentation to see which methods you may want to override:

	http://docs.mono-android.net/index.aspx?link=T%3aAndroid.Content.BroadcastReceiver%2f*

You also need (which I forgot to mention last time) is this XML fragment in your AndroidManifest.xml:

	<uses-permission android:name="android.permission.RECEIVE_BOOT_COMPLETED" /> 

You can do that within the IDE by viewing your Project Properties, going to the Android Manifest tab, and in the Required permissions section selecting RECEIVE_BOOT_COMPLETED.
