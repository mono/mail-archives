On Apr 2, 2011, at 12:09 AM, Daniel Bullington wrote:
> If I do a spinner.getselecteditem(pos), I get back a javaobject...how do I get the underlying clr object that I can see as property in debugger as .instance, having expected clr type?

At present, you don't, not directly.

If you use e.g. ArrayAdapter<T> [0], there are methods provided to convert from/to a desired type e.g. ArrayAdapter<T>.Add(T)[1] and ArrayAdapter<T>.GetItem(int)[2], but for the general base-class case this functionality is currently missing.

Could you please file a feature request providing the scenario you're interested in? Is this for e.g. BaseAdapter[4] and similar?

For that matter, would using Convert.ChangeType() work for your intended scenario? This won't work now, but I can imagine adding support for this:

	T value = (T) Convert.ChangeType(adapter.GetItem(0), typeof(T));
