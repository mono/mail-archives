On Apr 21, 2011, at 6:50 AM, Olaf Bartelt (AMTANGEE AG) wrote:
> Update 2: I was right - after copying "mandroid" from the "bin" directory to "lib/mandroid" as "mandroid.exe", everything works fine - so @ Novell: please check the Novell.MonoDroid.Build.Tasks.ResolveSdks method - it seems to be broken/incomplete... 

Further exploration has narrowed it down to a bug in the installer -- it wasn't replacing existing files in /Library/Frameworks/Mono.framework/External.

Consequently any new users would get the proper xbuild support, while anyone upgrading from a previous release would have the xbuild support present in the earlier release.

Oops.

This will be fixed in the next release.

In the meantime, as I can't recall what all changed in the MSBuild support, I would suggest that any OSX developers who upgraded to 1.0 from a prior release perform the following commands within Terminal:

	cd /Library/Frameworks/Mono.framework/External
	sudo unzip -o /Developer/MonoAndroid/usr/share/External.zip

Thanks,
