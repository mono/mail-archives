We should improve that error message; it provides no useful context...

That said, you have an assembly that is somehow referencing System.ComponentModel.ImmutableObjectAttribute. Unfortunately that's a desktop .NET type which isn't included in the Mono for Android profile. The logical implication is that your project is referencing an assembly that wasn't built against the Mono for Android profile assemblies.

A "simple" way to determine this is:

	for a in `find $ProjectDir -name \*.dll` ; do echo $a ; monodis --assemblyref $a ; done

Then look for Version=2.0.0.0 or Version=4.0.0.0 in the output; Mono for Android SDK assemblies have a version of 2.0.5.0.

(Yes, this is rather specific to OS X. I'm sure there's something equivalent for Windows, right?)
