Rationale? Activity.MoveTaskToBack() keeps the Activity on the Activity stack, meaning that if the user hits the Multitasking key in ICS, the Activity will still be present & displayed as an app that can be switched to. If it shouldn't be possible to move to that Activity again (e.g. it was a login dialog), then it shouldn't appear in the activity stack at all, in which case Activity.Finish() seems the saner solution, no?
