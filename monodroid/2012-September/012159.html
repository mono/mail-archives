If a lot of your activities display data in tables or such it would be
wise to use something like MonoTouch.Dialog to create these parts.
Though if you have a lot of custom views where you manually draw stuff
it will be a bit more work.

I did some custom controls displaying some graphs where the logic was
almost 100% reusable while I just called the MonoTouch equivalent
methods to draw things, which worked almost flawlessly.

Timewise it is hard to exactly tell you how long it is going to take,
but depending on the complexity it would take about 5-10 workhours per
activity, a lot less for the less complex ones. What will take some
time would be to get used to how iPhone works.
