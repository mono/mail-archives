On Apr 7, 2012, at 4:57 PM, Mike Tajmajer wrote:
> Could Mono.Data.Sqlite.dll use my new .so?  Or, would it need to be rebuilt?

No, as Mono.Data.Sqlite.dll P/Invokes "sqlite3", which is dllmap'd to /system/lib/libsqlite.so (and there's no way I know of to override the dllmap).

You'd need to alter the Mono.Data.Sqlite sources to use a new native library name (matching the library that you bundle with your app):

	https://github.com/mono/mono/tree/mono-2-10/mcs/class/Mono.Data.Sqlite
