On Apr 20, 2012, at 9:16 AM, Jonathan Vazquez Hernandez wrote:
> I want to get the dpi programmatically. I used this code to get it:
> 
> DisplayMetrics metrics = new DisplayMetrics();
> WindowManager.DefaultDisplay.GetMetrics(metrics);
> 
> int dpi = metrics.DensityDpi;
> 
> I got this: 
> 	Tablet (Acer ATab500 10") = 160dpi
> 	Cell (Galaxy S 4G) = 240dpi

These don't seem to be horribly wrong: the Acer is apparently 149ppi [0], while the Galaxy S 4G is 233ppi (assuming those sites are actually correct...).

So I think the API calls are correct.

> Accordingly Google doc, the devices with the xlarge screen (like tablet with 10" screen) must be xdpi (up to 300dpi).

I'm not sure which docs you're referring to. Consider:

	http://developer.android.com/guide/practices/screens_support.html#testing

This provides a table of screen size vs. screen density. The Acer would be an "Extra Large screen" with "medium density", while the Galaxy would be a "Normal screen" with "High density."

The Android device specifiers (screen size, dpi, etc.) are _ranges_, not absolutes. (Plus, I don't think there exists a device that conforms to xlarge and xhdpi; the new iPad would only be xlarge and hdpi, and there is no Android device with anything close, display-wise...).
