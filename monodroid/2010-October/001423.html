Has any consideration been given to supporting VS2008?

It would seem to make sense, considering VS2010 only supports Windows
Phone 7+ and not Windows Mobile. Since Windows Phone is crippled by a
lack of support for native code, many enterprise applications will
need to stay on the "old" Windows Mobile platform. Having to switch
back and forth between VS2010 and VS2008 would significantly reduce
the practicality of using Mono to extend such applications to the
Android platform.
