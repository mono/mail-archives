On Thu, 2010-10-14 at 00:11 +0300, Henadzi Sabaleuski wrote:
> Any progress with this bug? I have same problem, can you help me to
> resolve it?

As a guess:

        W/ResourceType(   53): Bad string block: last string is not
        0-terminated

        W/PackageParser(   53): /data/app/vmdl53538.tmp (at Binary XML
        file line #12): Requires newer sdk version #8 (current version
        is #7)

So your app isn't being installed.  This is possibly because you're
using resource constructs that don't work with Android v7.

To further diagnose this, you can do two things:

 1. Install the Android v7 SDK (via the `android` tool)
 2. Explicitly provide the SDK Version to use with monodroid.

I don't know how to do (2) within the IDE, but you can edit your .csproj
file and add:

  <PropertyGroup>
    <MonoDroidExtraArgs>--sdk-platform=7</MonoDroidExtraArgs>
  </PropertyGroup>
