I've got the Samsung Galaxy S I9000M (canada, but basically identical to the
I9000), and I'm running the JPK firmware (so froyo), rooted, with the one
click lag fix applied.


I hadn't been able to get debugging working on device, but the first time on
a fresh run of visual studio, if I went to debug, I would get a blank screen
on my phone.  I had to manually quit debugging, and also Force quit the app
on the device.

If I launched the app from the device after this, it would launch fine.  If
I debugged from visual studio subsequent times, I got the "only one usage of
each socket..." message, but then I could still manually launch the app on
device and it had been updated.


So it seems if the debugger fails to connect, it is still hanging in limbo
and you can't bind the debugger again the next time you try, however the app
still compiles fine and gets installed fine on device.

Now, in Preview 5, debugging* IS WORKING* for me.  Of course, you need wifi
turned on, but it's working for me now.  I can hit breakpoints and inspect
variable values, etc.

So, it looks to me like the debugging routine needs to be a bit more
resilient in figuring out if it can't connect, and then terminate if that's
the case.  I'd actually like to see an option to just build and push the app
to device, without debugging as well.
