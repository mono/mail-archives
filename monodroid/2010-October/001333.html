On Wed, 2010-10-13 at 15:18 -0400, Wally McClure wrote:
> probably a dumb question, but I'll ask it anyway.  I created some test
> projects in preview 4.  Do I need to create new projects for preview 5
> or will my existing ones work?

I don't know how your test projects work. :-)

The core issue is that the interaction between libmonodroid.so (bundled
in the .apk) and the shared runtime changed, so if you use the new
shared runtime you need a new libmonodroid.so in your app.  The easiest
way to do that is to just rebuild your app.

If you're not generating a .apk that contains libmonodroid.so, it
shouldn't be an issue.
