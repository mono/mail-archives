We've had several reports of issues related to using android/java API
from non-java-originated threads.  This is very likely the same thing.

Right now, using java APIs, even the ones intended to get you back to
the UI thread, are broken from System.Threading originated threads.  We
are aware of the problem and will hopefully have a solution soon.  

I have had success with java.lang.Thread and RunOnUIThread, though as
you are aware, there are other issues which you have filed bug reports
for that will stop a timer based app, or any object that creates a lot
of objects in a hurry.  We are working on this as well.
