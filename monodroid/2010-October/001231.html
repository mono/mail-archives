We won't be supporting such changes to the resource layout system - at 
least, not without a lot of careful consideration, and it's not a 
priority at this time.

However, note that you can remap a resource project item's filename to a 
resource "path" using <LogicalName>, similarly to .NET 
EmbeddedResources: 
http://monodroid.net/Documentation/Build_Process#Resources

For your own projects you could easily write a custom MSBuild build hook 
to set the LogicalName item property at build time based on whatever 
folder naming pattern you want.

- Michael
