FYI: This extension currently looks for ".Silverlight" or ".WPF" in the
file names to ignore the specific files.  Though it has no UI to
configure this, you can go and modify the .csproj file to add/change the
filters.

 

Just in case you want to do a "Foo.MonoDroid.cs" file and have it
ignored by the Project Linker

	I knew I had seen something to solve this very issue!!!

 

This sounds fabulous!    Should we do a blog entry for this?

 

	 

	Go to the Extension Manager, search the Online Gallery for
"Project Linker". 
	

	So right when you start, with your empty project, create the
two, three, whatever platform-specific projects you need. All of them
empty. Add whatever references each one needs, etc.

	 

	Now, on each of the "secondary" projects (the ones that will
have linked files), right-click, select "Add project link..." and select
the "main" project. 

	 

	From that point on, that extension will track item
add/rename/delete and replicate the action in the linked projects, but
over *linked* files :). Yes, that means it will preserve also the folder
structure as you create it on the main project.
