  As far as I read android sources, there isn't any xsd files (neither 
in the runtime nor in the sdk sources), and 
android.widget.LayoutInflator[*1] only uses reflection i.e. 
Context.getClassLoader().loadClass().

Since users can use their own classes that for each derive from View, 
simple xml schemas wouldn't apply. But for the foundation classes, it 
would be possible to generate incomplete xsds from the API definition, 
or as an easier choice, reflection from Mono.Android.dll.

I haven't practically used VS for years, so I don't know what and how VS 
helps you with xsds though.

[*1] 
http://android.git.kernel.org/?p=platform/frameworks/base.git;a=blob;f=core/java/android/view/LayoutInflater.java;h=e5985c179e9d4394c53e7f58bd2c38d54c7b208e;hb=HEAD
