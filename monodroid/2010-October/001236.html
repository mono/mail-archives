  I don't think XAML is easy and usable. First, our System.Xaml.dll 
(i.e. xaml 2009) support is still in infancy and not for productive use 
(I'm hacking it while my primary task is WCF). VS-only feature which is 
based on .NET cannot be brought to MonoDevelop.
And even with working xaml library, you'll have to attach several XAML 
attributes in the "modeled" library. In this case, it is 
Mono.Android.dll and it's not an option. System.Xaml is not in the 
mobile profile. AFAIK Microsoft did not make good System.Xaml usage 
scenario for Silverlight either.
You can still use xaml 2009 library with some trick like having some 
intermediate library, but it won't make building a designer easier.

Or, if you have any concrete idea, beyond ambiguous assumption, on how 
to build the designer with XAML, I'd like to hear. I'd like to know 
about XAML library use cases :)

Atsushi Eno
