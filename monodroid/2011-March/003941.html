On Mar 25, 2011, at 11:29 AM, Wally McClure wrote:
> I read where Google is going to hold the source code for Honeycomb (did not see this one coming, but am not surprised given recent lack of announcements).  How does this effect Mono for Android?  Does it effect the ability to target Honeycomb?

It means you will only be build API level <= 10 with Mono for Android, which means no use of Honeycomb-native APIs until the first major release after Mono for Android 1.0. Your apps will still run on Honeycomb, and your apps will probably be able to fill the entire screen (depending on how you define your UI); you just won't be able to use the newly added UI elements.
