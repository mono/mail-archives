Hi folks

 

I've noted that the AsyncTask class is apparently currently considered
unusable, and it is unlikely to be ready before launch.  If someone can
provide a suggestion to the best approach to substitute for this within
.NET then this would be really useful, or if people are interested in
this functionality then I can continue grabbing the AsyncTask source
code (and onward references) to create an open source port.  Are there
any suggestions, or is there any interest for this conversion as a
project?  FYI, I have converted about 2000 lines of source code in an
evening towards this, but I am fearful of the number of onward class
references that will be required to be converted too, so I would rather
garner some interest and support, or an alternate approach, before I end
up wasting a load of time attempting to convert this.  

 

If anyone is interested in the andengine port to monodroid, then it is
available here:
https://github.com/jamesburton/AndEngine.net

 

We welcome any willing collaborators, and I am gaining experience with
merges now so please ask me if you get stuck on such issues.
