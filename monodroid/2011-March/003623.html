Jonathan's suggestion of uninstalling and reinstalling the plugin 
unfortunately didn't work (it just installed to C:\VS_DIR again) so I 
tried manually copying the files into the Visual Studio install.  That 
didn't have any effect either, until I found a web site that said I 
needed to run "devenv /InstallVSTemplates" after putting stuff into the 
template folders.

Now all the templates, menus, and such show up where I expect them, and 
I'm even able to build and run the Hello World sample.  So it seems to 
more or less work now, except that I expect a certain amount of trouble 
when it comes time to uninstall this version of MonoDroid :-)

Are there any installer logs or anything I can look at to try to figure 
out why it's getting confused about the install directory?  I thought it 
might be because I had the Express version of VC# 2008 installed before 
I installed the Pro trial of VS 2010 last night, but I tried 
uninstalling that as well before I installed the plugin this time, and 
it didn't help.
