On Feb 28, 2011, at 1:04 PM, justin coon wrote:
> Atsushi I did see that, however it sounds like MD is waiting on something from Google, but I thought Google has released the final SDK for Honeycomb/3.0....is that incorrect?

They have, but they haven't updated everything. :-)

So, background: Mono.Android.dll is created via a code generator which parses an XML description of the Java API to bind. Google has previous versions of the XML description at [0], but they have not provided XML descriptions for API level 10 (Android 2.3.3) or API level 11 (Android 3.0). Without these files, we can't generate an updated Mono.Android.dll, and the Google SDK doesn't include them.

This is also the reason why MonoDroid 1.0 is only binding android.jar: because that's the only .jar file that we have an XML description for. Our ~first project after 1.0 will be to write a .jar-to-XML generator so that we can generate an XML description for 3rd party .jar files, thus allowing use of 3rd party .jars, and packaging/cleaning up the source generator for external use.

- Jon

[0] http://android.git.kernel.org/?p=platform/frameworks/base.git;a=tree;f=api;hb=HEAD
