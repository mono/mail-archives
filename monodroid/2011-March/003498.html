I have two project in the same solution in Visual Studio + Preview 13.

 

When I set MonoDroid to be the default project, it deploys to the device,
and everything works as expected.

 

Bug #1 When I set another project (e.g Windows Forms) to be the default one,
I get MonoDroid's 'Select Device' dialog.

 

Bug #2 Another annoyance with this dialog is when you have two screens
connected to the same computer, the dialog opens in the *active* screen
instead of the one that shows Visual Studio. For example, when you press F5,
switch to another application on a secondary screen, then the dialog opens
in the *wrong* screen and may hide behind whatever application that is
defined as 'always on top' (in my case, I have such application running on a
secondary screen. MonoDroid's dialog is hidden behind that app).
