Hello,

I had a look at DroidText (as I whated to know what blocked you), and 
found that there are many dependencies on Java collection types that we 
don't support, as well as many packages we don't support bindings such 
as w3c dom. I have to say, we have no plan to add those types in our 
Mono.Android.dll.

It might be still possible to reduce bound API (maybe significantly) to 
get working bindings for minimum required feature set. Also, Java 
interoperability is not only about automatic bindings - with good old 
manual JNI bindings(!) if you want...
http://docs.xamarin.com/Android/Guides/Advanced_Topics/Java_Integration_Overview/Working_With_JNI

Atsushi Eno
