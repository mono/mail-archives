On Mar 13, 2013, at 12:18 PM, Dimitar Dobrev <dpldobrev at yahoo.com> wrote:
>     OK, this makes sense. So, is it then some kind of a bug in the Android SDK? I don't see a reason for the android.jar on the device (or on the emulator in my case) to be different.

This is not a bug in the Android SDK. If you read the Android source much, you will see the @hide javadoc-comment tag. This does (at least) two things: 

1. It removes the type/member from the generated JavaDoc HTML output.
2. It will cause the type/member to be removed from the android.jar that is in the Android SDK.

This prevents developers from "accidentally" referencing implementation details.

This is not an Android SDK bug, it's a feature.
