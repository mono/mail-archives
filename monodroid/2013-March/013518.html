It also doesn't "allow packages to be created in native code." It allows assemblies to be distributed in native libraries. It's basically `mkbundle`ing assemblies into a shared library instead of an app; otherwise, same basic idea.

> I've downloaded the latest NDK and read the docs, but it looks as if there isn't a way to install and use the NDK system wide.
> 
> Is there a way (on a Mac) to install system wide rather than project by project?

This confuses me. The NDK that Xamarin.Android uses _is_ system wide (for all Xamarin.Android projects, anyway). You can specify the location in Xamarin Studio Preferences > Projects / SDK Locations / Android > Android NDK.
