The single parameter constructor for XmlTextWriter takes a
System.IO.TextWriter object as a parameter, not a string:
http://msdn.microsoft.com/en-us/library/9zecxb09.aspx

If you don't want to create the TextWriter yourself, there is another
constructor that will take a string and an encoding that you can use:
http://msdn.microsoft.com/en-us/library/5366y21d.aspx
