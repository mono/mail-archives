I am using SetTcpKeepAlive method
(http://msdn.microsoft.com/en-us/library/system.net.servicepointmanager.settcpkeepalive.aspx)

long time to complete and when there is a network failure.

bit strange, since I can see it on github -
https://github.com/mono/mono/blob/master/mcs/class/System/System.Net/ServicePointManager.cs#L278. 

One way or another, is there some alternative way to use TCP's KeepAlive
packets on Android?
