Just to follow up on this, since android didn't provide a default
implementation of the interface method in their BaseAdapter class, I'm
not inclined to provide one in our binding, especially from a position
of ignorance over what it's for, since the docs are pretty thin.  I
could see it being useful in a sorted adapter, or filtering adapter,
perhaps, but maybe not within the context of BaseAdapter.

Might be worth following up with the android project if someone has the
itch.  If they add a default implementation to the java class, we'll get
it automatically when we update to the next API level.
