Interesting, I've never got such a problem with my application in MonoTouch.

I got it in Mac OS X and some Linux distributions (no problem in Windows because I use the System.Data.Sqlite namespace, not Mono.Data.Sqlite, and it includes the library embedded in the .NET assembly), and so, I download the source from sqlite's homepage, compile it and supply with my application.

On iOS it worked in 4.1 and all 4.2 releases up to GM.

If nothing changed in MonoDroid, putting the .so in the same folder as the main executable should just work, and not increase the app more than ~300Kb.
