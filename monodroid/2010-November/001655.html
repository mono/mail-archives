On Tue, 2010-11-16 at 12:53 -0800, Andy Shah wrote:
>  I have one question... Will the code base be almost the same across
> MonoTouch & MonoDroid ?
> 
>  in the sense, if we develop applications using one, can it compile on
> the other... without changes ?

No.

There are two sets of assemblies, the "core" assemblies and the
platform-specific assemblies.

The "core" assemblies are mscorlib.dll, System.dll, etc. -- the things
you'll find on most any .NET implementation.  Code using the core
assemblies WILL be portable between MonoDroid and MonoTouch (and
probably WP7 and Silverlight and desktop .NET and...).

The platform-specific assemblies wrap and (more or less directly) expose
the underlying platform; this consists of monotouch.dll for MonoTouch,
and Mono.Android.dll for MonoDroid.

Code using platform-specific assemblies WILL NOT be portable.
Consequently, all UI code written on MonoTouch WILL NOT be directly
usable on MonoDroid (and vice versa) [0].

The solution, of course, is to carefully design your app so that
platform-specific code is kept separate (MVC, anyone?), thus allowing
greater reuse of your core algorithms and data structures.

 - Jon

[0] Except for OpenTK, which we are trying to keep consistent.  However,
most apps will not be written entirely in OpenTK.
