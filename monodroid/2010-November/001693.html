Hi,
i've solved the problem now by further analyzing the adb logcat output I 
foudn the following line:
E/PackageParser(  274): parsePackageLite error: <manifest> specifies bad 
package name "bbingodroid": must have at least one '.' separator
I've then changed the package name to contain a '.' and it works again. 
I guess during the renaming I've done to only use lower case characters 
this problem came up.

It would be great if you can integrate a feature to show such error 
message in a VS window to give programmers direct feedback of what is 
going wrong.
