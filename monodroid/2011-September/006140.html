My situation is identical (except I'm Cali and they're east coast). I put
the signed .apk on an IIS Web Server to which all testers have access. Part
of our application checks the web site for the availability of a newer
version, and offers the client the ability to start the download (or, they
can just navigate to our .apk file with their browser).

The only tricky part is the IIS server needs the following MIME type added.

.apk 
application/vnd.android.package-archive
