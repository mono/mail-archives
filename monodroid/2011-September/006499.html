On Sep 28, 2011, at 1:14 AM, Tyson Swing wrote:
> I'm impressed with the whole Android implementation for C#.  I find it much more concise than dealing with the Java equivalent using Eclipse.  I was reading up on how to use a MapView, which is a bit cumbersome if you activity heavily depends on it, and in the thread it mention something about it being on the future roadmap yet when I checked the current roadmap I don't see it on there.  Is this in the future of mono droid?  

It's on the roadmap:

	• Support for exposing arbitrary Java APIs to .NET
	• Support for the Android 3.x APIs

We'll have the order backwards, though. 2.0 will support Android 3.x (the beta 1.9 release already does), and the following 2.2 release should support using arbitrary Java APIs, which would include maps.jar.
