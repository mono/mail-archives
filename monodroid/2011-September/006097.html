On Sep 1, 2011, at 9:44 AM, Nick Randolph wrote:
> The classes that I'm deserializing to are just standard .NET classes, so don't think they inherit from Java.Lang.Object.

To the best of my knowledge, types which do not inherit from Java.Lang.Object should be usable via DataContractJsonSerializer.

However, if you have linking enabled, and you don't [Preserve] your types and members, things may break.

If you're not linking (linking is disabled by default in Debug builds), and you're certain that your types aren't inheriting from Java.Lang.Object, please file a bug so that we can investigate.
