My guess is that it's showing a GREF overflow:

	http://support.xamarin.com/customer/portal/articles/141166-unexpected-nullreferenceexceptions

> So could it mean that there is not enough memory like in our case?

The problem with API Demo isn't memory use per-se (e.g. OutOfMemoryException), it's Dalvik Global Reference use. It's still  related to memory, but has nothing to do with the size of the GC heap, and more to do with how many instances you can have allocate at once (even if those instances are tiny, like `new Java.Lang.Object()`).
