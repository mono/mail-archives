On Sep 9, 2011, at 5:31 PM, emalamisura wrote:
> I consistently get this error: F/MonoDroid(19561): ERROR: Unable to detach current thread from the Java VM!

(I made that a Fatal error level? What was I thinking...)

Ignore it. There's nothing you can do about it (and we should probably remove it in a future release). I've tried to figure it out before, and had no luck -- it comes from the JNI DetachCurrentThread() returning an error, but we always only call it on threads that we called AttachCurrentThread() on...

It bears further research, but it's not an actual "error"-level event.
