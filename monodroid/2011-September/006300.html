On Sep 17, 2011, at 8:48 AM, dgoughnour wrote:
> The question is what is the best way to 'reach into' the managed code to get values and data needed to display current information on the map.

Using Android Callable Wrappers is probably not the best way to do this, largely because they're very limited. They're generated during every package operation, and the types are visible within e.g. obj\Debug\android\src. They only contain methods that are overridden from base methods, or methods that come from Java-side interfaces.

This makes it rather difficult to send and retrieve data.

An (untested, theoretically) easier way to go would be to instead use a ContentProvider. ContentProviders were explicitly designed to share data across application boundaries, which means they can also be used to share data across the Java/managed boundary.

	http://docs.mono-android.net/index.aspx?link=T%3aAndroid.Content.ContentProvider
	http://docs.mono-android.net/index.aspx?link=T%3aAndroid.Content.ContentProviderAttribute
