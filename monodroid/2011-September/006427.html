On Sep 26, 2011, at 5:33 AM, was wrote:
> Thanks Glen. However, the issue I have is that my dates are always formatted
> according to the Android device's 'Settings>Language & keyboard>Select
> Locale' setting. The 'Settings>Date & time' format has no effect. I can't
> figure out how to get the  'Settings>Date & time' format to override the
> locale format if it's different.

This is probably a Mono for Android bug, as we set the LANG environment variable at startup to the Locale setting, but we don't set any environment variable overrides that the user might set. (Meaning your default output will be based on the default locale, but not on any specific overrides.)

Please file a bug with instructions on how to reproduce (e.g. what to set Settings > Date & time to, and what the in-app output should be).
