That's a core Android bug. You can work around it in Xamarin.Android (by not compressing, or using another mechanism entirely such as EmbeddedResources), but you can't _fix_ it, anymore than you can fix it in Java (i.e. you can't fix it from Java either, it's an OS bug, requiring an OS update to actually "fix").
