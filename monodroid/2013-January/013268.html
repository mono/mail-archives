Hi,

 

we encounter a problem building a Monodroid solution with Team Build / TFS2010.

 

The error message appearing in the build log is "!!!!!!!!!! tis.truckbox.muxer.lib.jar" (10 exclamation marks, no further information...)

 

-          The file "tis.truckbox.muxer.lib.jar" is a 3rd party AndroidJavaLibrary included in one of our solutions projects.

 

-          Build via Visual Studio 2010 works fine on the same machine with the same sources, the error only appears in the automated build.

 

-          Monodroid-Version on the buildserver is the latest 4.4.55.104956787. Error occurred with the previous version as well.

 

-          We found out that the error is thrown within the build-target "_CheckDuplicateJavaLibraries" in C:\Program Files (x86)\MSBuild\Novell\Novell.MonoDroid.Common.targets. We removed the CheckDuplicateJavaLibraries-task from the local Novell.MonoDroid.Common.targets-file and the error disappeared.
