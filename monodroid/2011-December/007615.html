On Dec 6, 2011, at 6:23 PM, Daniele Salvatore Albano wrote:
> There is a way to use Mono for Android to compile some external C#
> application (that doesn't need to access the android stuff)? And use
> the mono for android installed on the phone to execute it?

Maybe? You could certainly compile your app against the Mono for Android BCL assemblies (mscorlib.dll, etc.) using CSC.EXE (.NET) or /Developer/MonoAndroid/usr/bin/smcs (OS X) in order to get a .exe out of it. As for running on the device, you might be able to use Assembly.Load() from a "normal" Mono.Android app and then execute the .exe...

However, I'm not sure how stable this will actually be. This isn't a tested configuration, and the BCL contains a number of changes to support Android devices, for example stdout and stderr are copied to the Android Debug Log:

	https://github.com/mono/mono/blob/master/mcs/class/corlib/System/Console.cs#L181

System.Net.ServicePointManager calls into Mono.Android.dll in order to delegate certificate validation to Android:

	https://github.com/mono/mono/blob/master/mcs/class/System/System.Net/ServicePointManager.cs#L420

and numerous other changes (grep for MONODROID).

In short, if it breaks, you're unlikely to get any significant support unless it also breaks "normal" apps.
