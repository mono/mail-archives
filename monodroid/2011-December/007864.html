On Dec 14, 2011, at 6:40 PM, klimaye wrote:
> The only part I can see where I could possibly make a change is the json parsing.

Mono for Android provides System.Json.dll, which is 24KB in size (unlinked). Compare to Netwonsoft.Json.MonoDroid.dll, which is 358KB. You might want to consider System.Json.

As for SQLite, instead of using Mono.Data.Sqlite you may consider using Sqlite-net, though this won't support existing ADO.NET code (a feature in this case, as it doesn't depend on System.Data.dll):

	http://code.google.com/p/sqlite-net/

TweetStation uses Sqlite-net.
