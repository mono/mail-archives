> On Dec 7, 2011, at 5:35 AM, Daniele Salvatore Albano wrote:
>> Understood, and if i compile it normally with mono can i run it on mono for android?
>
> No, if you use mcs/gmcs/dmcs you'll be referencing the desktop assembly set, which are incompatible with the Mono for Android profile. (For example, Mono for Android doesn't have System.Configuration.dll, and is lacking lots of other assemblies, types, and members present in the "full desktop" profile.)
>
> You need to use (unreleased) mono 2.12 mcs against the Mono for Android assemblies, or use the smcs cmpiler included with Mono for Android, or use .NET CSC against the Mono for Android assemblies.
