On Dec 6, 2011, at 8:13 AM, Tossing, Chris wrote:
> I am able to launch and run our application, but I am getting a Java.Lang.NoClassDefFoundError exception when trying set the items of an AlertDialog using AlertDialog.Builder.SetItems(..).
>  
> Here is our code:
>             var builder = this.CreateDialog(title, null)
>                 .SetItems(items, (s, e) => { CallResultHandler(resultHandler, (int)e.Which); })
>                 .SetNeutralButton(defaultText ?? "Cancel", (s, e) => { CallResultHandler(resultHandler, null); });

That's a new one (I usually see that with IInputFilter). This should should be fixed in the next release. In the meantime, you should be able to fix this by placing the following statement prior to the above statement:

	Android.Runtime.TypeManager.RegisterType ("java/langCharSequence", typeof (Java.Lang.ICharSequence));
