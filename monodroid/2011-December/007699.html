After have installed the OS of my PC for 4 times, finally I find and
confirmed this bug.
Envirement: Win7 x64 + VS 2010
1. I have a virtual PC with MonoDroid 4 installed, and everything goes well.
2. For my real PC, "Hello Word" demo can work correctlly.
3. Create a solution with two project one is Android class library and the
other is Android application, in the class library project, add several
complicated class, and use them in Android application project.
4. For the solution in point 3, Mondroid in my *VIRTUAL* PC works well
4. For the solution in point 3, Mondroid in my *REAL* PC always fail to
package the application and the VS output is "mondroid exit with error code
-1", it seems that it faild build the solution with the sign option.

How this happens?
After compare and search in google, I find that this bug is because that:
in my real pc, I changes the location of "My Documents", "My Pictures" and
so on (all the windows default folder for person) and move them to another
hard disk for example "driver E:". I find there are two ".android" folders
are created, one is in C: and the other is in E:
but for Virtual PC I just install win 7 without any personal setting
changes.
