Hello,

Sorry to say, this is a bug in MfA. We created enum type for 
android.text.InputType constants as Android.Text.InputTypes[*1], but we 
forgot to update the type of the property you were trying to use.

So far, please use this enum and cast to int before assigning to the field.

We are in the process of reviewing the entire API in some new process 
and picking up a bunch of methods and constants so that we don't miss 
lots of this sort of mismatches. Things will become much better in some 
next releases.

[*1] http://docs.mono-android.net/?link=T%3aAndroid.Text.InputTypes
