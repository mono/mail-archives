On Dec 6, 2011, at 5:23 PM, Wally McClure wrote:
> What is "fast deployment" in the project properties?  What does it do for me?

It's intended to speed up turnaround time for the (hopefully) "common" Debug build case.

Assemblies are no longer embedded into the .apk, they're instead placed into a directory on the device.

IFF you don't add any new Java.Lang.Object subclasses, don't override any new methods, don't implement any new interfaces (i.e. no  changes are required to the Android Callable Wrappers), don't add any assets or resources (i.e. no other changes are required in the .apk), then we'll skip rebuilding the .apk entirely and instead just sync the assemblies into a directory on the device.

If you're just adding statements to log something/etc. while debugging, this could greatly speed things up, as for large apps the packaging process can be quite slow, and this manages to bypass a large chunk of it.

See the Fast Deployment section in the release notes:

	http://android.xamarin.com/Releases/Mono_for_Android_4/Release_4.0.0

In particular, note the "Code change, build and install" time difference: 3.9s vs. 14.7s (fast deployment vs. normal deployment).
