<div class="im">On Mar 18, 2011, at 12:43 PM, Aaron Knabb wrote:<br>
&gt; However, if I am creating and disposing of a large bitmap every time my activity is recreated (say in an ImageView), I run out of memory before long.<br>
<br>
</div>The problem -- which is present on every GC system -- is that Mono's GC doesn't know how big the Dalvik-allocated bitmap is (just as NET's GC doesn't know that you just called Marshal.AllocHGlobal() to allocate 4GB of memory...). Furthermore, Mono's GC _can't_ know how big the Dalvik-allocated bitmap is, and even if you knew how much memory it took GC.AddMemoryPressure() isn't supported.<br>

<br>
Thus, in this (and similar) cases you need to help the GC out, as you know things the GC doesn't, and thus the GC.Collect() call is appropriate.<br>
