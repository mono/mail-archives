    &nbsp;&nbsp;&nbsp; <br>
    That's the tool. I've been using it since 2010-09 and it is so far
    the best tool for code reuse.<br>
    You should determine smallest_common_denominator (subset), this is
    for me WP7 and<br>
    start from there.<br>
    I divide projects according to lasagne principle (layers:
    BusinessLogic, PesentationLayer...)<br>
    This increases number of projects, because You have Cartesian
    product Layers x Platforms.<br>
    But with project linker it is quite simple.<br>
    <br>
    Relation is something like this<br>
    <br>
    &nbsp;&nbsp;&nbsp; WP7 &lt; MDrd &lt; Desktop/Server Version (there is sometimes
    RichClient Silverlight Project)<br>
    <br>
    WP7 is source project every code file added to it is added as link
    in linked project <br>
    to file in source project.<br>
    To add source project, right-click on project &gt; link project.
    Dialog asking source project appears,<br>
    which is usually subset project.<br>
    <br>
    Tool gives possibility to exclude linking: if the code is WP7
    specific (subset, let say reading XAP files)<br>
    then name the file XAP.Silverlight.cs and this source will not be
    linked in other projects.<br>
    Other exclusion extension words are WPF (for desktop) etc. This can
    be customized too, AFIK, but<br>
    haven't played with that...<br>
    <br>
    UnitTesting. Due to the fact that last projects in the chain
    (supersets) are usually desktops, most<br>
    of the code can be tested with nunit or VS testing tool.<br>
    For platform specific unit testing havent got that far... My next
    todo.<br>
    <br>
