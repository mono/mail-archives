<br>
Sorry if this is not a really &#39;new&#39; idea, but it crossed my mind these days<br>
and I would like some feedback from you.<br>
<br>
The problem is: how to build an app that could work on Android, iOS (the<br>
mainstream) and perhaps other platforms? Ok, we have Mono for iOS and Mono<br>
for Android, but what about the UI? They are really different among<br>
platforms.<br>
<br>
On the other side we have a &#39;partial&#39; solution: WebApps written in any Web<br>
Framework (Sencha Touch, JQuery Mobile, etc) and PhoneGap to build a<br>
&#39;native&#39; app. The problem with this approach is that the JavaScript is<br>
limited and if you want to introduce anything new, you have to create a<br>
native class for iOS, Android, etc. So, you are maintaing many different<br>
sources again.<br>
<br>
I thought if we could do something like this:<br>
<br>
- The View: The application would be a WebApp, like those we would create<br>
with any Web Framework, running in the WebView component. Ok, it&#39;s not<br>
&#39;native&#39;, but changing the CSS file it can look very close to native under<br>
any platform, with the same objects.<br>
<br>
- &#39;MonoGap&#39;: it would be amazing if we could create a class library for iOS<br>
and other for the Android called for example &#39;MonoGap&#39; that we could<br>
reference in our application. This library would give access to a &#39;MonoGap&#39;<br>
namespace in the WebApp with calls for Accelerometer, GPS, Camera, etc.<br>
<br>
- The Controller: pure C# classes we write under Mono that gets accessible<br>
to the WebApp through a custom namespace. Actually, it&#39;s like PhoneGap does,<br>
but it would be our custom code.<br>
<br>
- The Model: also, pure C# classes to handle the data under Mono, whatever<br>
data source it is, local, remote, object, relational, etc, etc.<br>
<br>
With this kind of approach, we could really build powerful applications,<br>
because all the application logic (calculations, data management,<br>
communications, etc), could be done with C# code, under Mono, shared among<br>
all platforms, and the UI could also be the same. Debugging would be easy,<br>
as all the complex logic would be under Mono.<br>
<br>
What are your thoughts?<br>
