<html><body><div style="color:#000; background-color:#fff; font-family:times new roman, new york, times, serif;font-size:12pt"><div><span>Hi Tim,</span></div><div><span></span>&nbsp;</div><div><span>I've found much of the same thing in my playing around with Mono for Android. I've mostly adopted the Java approach for client applications on the Android platform.&nbsp;Web/Restful Services are implemented in .NET/WCF/C#. Although there are different stacks available, I've adopted Android/MOTODEV Studio -&gt; Restlet -&gt; Jackson for accessing and serializing/deserializing to/from POJO (plain old Java objects).</span></div><div>&nbsp;</div><div>Regarding differences between Eclipse and Visual Studio, I've found the following:</div><div>&nbsp;</div><div>1) Eclipse offers better refactoring capabilities. Visual Studio offers the very basics.</div><div>2) Intellisense/Code Completion is much better in Visual Studio</div><div>3) Code formatting is much better
 in Visual Studio. Eclipse provides more options, but there are some scenarios where the options don't allow what I'd like to do. This is especially so with XML formatting. Visual Studio just works.</div><div>4) Device management is much better in the Android/MOTODEV Studio stack. You have options like starting and stopping devices/emulators, take screen shots, change startup options, etc., which make for a more pleasant experience.<br>5) When porting applications to Mono for Android, I've found some Java code constructs that simply do not have equivalents in C# like anonymous interface implementations. You can get the job done otherwise, but in more code is required. Also, you'll find some Android platform items not implemented at all. In cases like this, it's not just a simple port, you have to find another way to do it. It's not that it cannot be done. It just require more work.</div><div>&nbsp;</div><div>Best

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;">If anyone is interested, we're doing a proof of concept with
C# and Java/Eclipse to determine if we can meet business requirements with VS/C#.&nbsp;
We prefer to use C# and take advantage of cross platform development.&nbsp; Overall,
both products offer the same capabilities and functionality so what I'll
discuss here is what we've noticed and are from my notes for our design
meeting.&nbsp; I thought I'd share them here for others.</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;"> &nbsp;</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;"> &nbsp;</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;">1.&nbsp; Debugging an application in Java is faster and easier
because breaks and restarts are noticeable quicker.&nbsp; In VS/C#, it is almost
impossible to use a breakpoint and step through the application because it too
painfully slow.&nbsp; Java, stops and steps through like you'd expect.</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;"> &nbsp;</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;">2. Monitoring the event log in Eclipse is much better than VS/C#.&nbsp;
Eclipse does a continual logging so you can watch the logs as events appear.&nbsp;
In VS you have to refresh which is painfully slow.</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;"> &nbsp;</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;">3. Making a program auto start cannot be done in the
emulator for C# however, in Java it works fine.&nbsp; Supposedly, if you deploy the C#
application to a real device this behavior is not seen.&nbsp; I cannot confirm this
and do not understand why the Java App works as expected and the C# doesn't.</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;"> &nbsp;</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;">4. Running the application takes 5 times longer using VS/C#
than in Eclipse/Java.&nbsp; If you press the run button in Eclipse the application
starts relatively fast however on VS there's time for coffee.</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;"> &nbsp;</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;">5. Sizes of the programs are different.&nbsp; The C# application
is 732kb while the Java application is 104k.&nbsp; Maybe this is why starting and
building the application in the development environment is at least 5 times
longer.</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;"> &nbsp;</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;">6.&nbsp; There are less examples and discussions on MonoDroid
versus Java.&nbsp; So if&nbsp; you are looking to do something, you'll be hard pressed to
find an example.&nbsp; However, in Java you'll find lots of examples and people claiming
to be experts.&nbsp; Worst case, you'll be left trying to figure out how to convert
it to the C# world.</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;"> &nbsp;</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;">7.&nbsp; We did find a few things in C# not working in MonoDroid
that C# supports.&nbsp; For example C# supports network hardware calls.&nbsp; In our
business requirements we need to get the MAC Address and even though C#
supports it, the MonoDroid has issues with certain calls, basically enumeration
of generic interfaces is not supported.</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;"> &nbsp;</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;">8.&nbsp; After an application service runs for a while it just
stops running whereas Java it does not.&nbsp; Not sure what or why this happens.&nbsp;
Not sure if this is an emulator problem, but it doesn't happen with Java in the
emulator.</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;"> &nbsp;</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;">9.&nbsp; MonoDroid requires two Mono libraries.&nbsp; "Mono
Android-8 Support"-12KB and "Mono Shared Runtime"-18MB get
loaded and seem to be needed at least for development.&nbsp; Not sure how this works
for a production application.&nbsp; But, the same application would be
"huge" if these are also deployed with each application.&nbsp; Not a real
concern, but if the user deletes these libraries then the application breaks.&nbsp;
So, instead of having just 1 file to worry about, you now have framework files.</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;"> &nbsp;</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;"> &nbsp;</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;">Overall, this is what I've got so far. We did not try to
deploy anything so I cannot speak to deploying applications and what size
applications would be or how they&nbsp; perform.&nbsp; In the emulator, the Java program
starts faster, but they both respond the same once started.</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;"> &nbsp;</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;">The only concern we have so far are #3 and #8 which would be
show stoppers for us.&nbsp; The bigger files and libraries are a little concerning,
but not that much.&nbsp; </span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;"> &nbsp;</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;">The huge positive is if this works we'd have an easy
portable platform for Windows7 phones and iPad (which is a most have!).</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;"> &nbsp;</span></font></div> 

<div class="yiv53292444MsoNormal"><font size="2" face="Arial"><span style="font-family: Arial; font-size: 10pt;">Anyway, just my 2¢</span></font></div>
