Yes, there is support for WCF, REST, and direct database connectivity in MonoDroid.&nbsp; The stories are:<br><ul><li>WCF.&nbsp; Connecting to a WCF Service works the same as it does in Monotouch with one addition.&nbsp; the one addition is that the WCF proxy can be generated for you in MDr.&nbsp; This isn't possible in MonoTouch due to the runtime limiations in the iPhone.&nbsp; WCf support is still listed as experimental, so there are a lot of WS* features that aren't there.&nbsp; Basic http binding works properly as far as I know.&nbsp; I've done async and it works properly.<br></li><li>REST.&nbsp; Callling REST services works.&nbsp; I've got examples using GET and POST.&nbsp; These examples use XML and System.JSON.&nbsp; LINQ to XML and "LINQ to JSON" is working allow you to easily turn text into objects.&nbsp; I've done asyncon this as well and it works well.<br></li><li>Database support is a little more spotty.&nbsp; This is due to the requirements of having a data provider running in MDr.&nbsp; There is a Mono.Data.TDS assembly that is used for Sql Server connectivity and is included in the box so that you can use.&nbsp; I've used it to connect to a sql server database.&nbsp; I;ve pulled data back in a datareader.&nbsp; Jon Pryor says that there is a bug that keeps datasets from working.&nbsp; Getting to antoher db is harder.&nbsp; I think that here is support for some other database using some custom providers, but these are rare right now.&nbsp; This is listed as experimental, so I'm sure that there are some features that don't work perfectly, but you should be able to get at the basics.&nbsp; If you want to connect to oracle, db/2, or other database, in the absence of a provider, you can connect to the sql server and then used a linked server in sql server to connect up remotely to the database.</li></ul><br>Hope that helps some. :-)<br><br>&gt; From: tom@opgenorth.net<br>&gt; Date: Mon, 7 Mar 2011 21:49:57 -0800<br>&gt; To: monodroid@lists.ximian.com<br>&
