    Even google admits it that emulator is slow.<br>
    <br>
    You can <br>
    <ul>
      <li>speed up startup (but I don not believe this is an issue) with</li>
      <ul>
        <li>the use of new snapshot feature of the emulator</li>
        <li>skip the wonderfull animation with -no-boot-anim&nbsp; <br>
          emulator.exe -avd AndroidVirtualDeviceTestName -no-boot-anim
          -cpu-delay 0<br>
        </li>
      </ul>
      <li>try speeding up emulator (unproven - we haven't seen too much
        improvements)<br>
      </li>
      <ul>
        <li>pass arguments to qemu (last arg on CLI)<br>
          emulator.exe -avd AndroidVirtualDeviceTestName -no-boot-anim
          -cpu-delay 0 -qemu -icount auto<br>
          <br>
          -icount auto <br>
          enables virtual instruction counter with 2^N clock ticks per
          instructionSnippet
        </li>
        <li>scaling down the emulator</li>
        <ul>
          <li>on emulator, settings-display -&gt; disable screen
            orientation</li>
          <li>Shrink the display size to the smallest you can get away
            with (Jonathan Pryor few weeks ago)<br>
          </li>
        </ul>
        <li>
          use SSD</li>
      </ul>
    </ul>
