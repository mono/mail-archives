<blockquote class="gmail_quote" style="margin: 0pt 0pt 0pt 0.8ex; border-left: 1px solid rgb(204, 204, 204); padding-left: 1ex;"><div>On Jan 31, 2011, at 1:25 PM, Piotr Kryger wrote:<br>
&gt; Is there any way I can stop Monodroid from removing application files (from /data/data/* directory and from external storage)<br>
&gt; on each deploy to emulator?<br>
<br>
</div>There is currently no way to do this through the IDE, but from the &quot;any way&quot; department you can uninstall and install the packages using adb instead of through the IDE:<br>
<br>
        adb install -r path/to/your-Signed.apk<br>
<br>
`adb install -r` will &quot;reinstall&quot; your app, keeping the data directory.<br>
<br>
        adb uninstall -k <a href="http://package.name" target="_blank">package.name</a><br>
<br>
`adb uninstall -k` will remove the app, keeping the data directory.<br>
<br>
Given that I don&#39;t see a &quot;Create Signed Package&quot; command within the VS IDE, the easiest solution is probably to manually run `adb uninstall -k <a href="http://package.name" target="_blank">package.name</a>` before you use the VS IDE to install the app.<br>
