<p class=MsoNormal>Hi Wayne,<br>
&nbsp;I'm working on a basic OpenGL ES demo that currently compiles on MT and<br>
MD as well as supporting OpenGL ES 1.1 and ES 2.0 on both platforms.<br>
So far everything compiles and works on MT, just waiting for the magicians<br>
at Novell to add the ability to support ES 2.0 more easily in MD, and I<br>
think it should work on that as well.<br>
<br>
I've abstracted the rendering as follows...<br>
I have an abstract base rendering class ( I know everyone loves<br>
interfaces, but since this is game code, i think abstract classes should be<br>
slightly faster over all ).<br>
Then an OpenGL 1.1 class that inherits from this base class and also an<br>
OpenGL 2.0 class that does the same.<br>
Therefore the 1.1 class renders the rectangle using fixed function<br>
pipeline calls, and the 2.0 class uses shaders, while sharing the vertices<br>
code and colours ( they are in the base class ).<br>
Then when the View's FrameBuffer ( either iPhone or Android ) is created I<br>
just instantiate the appropriate OpenGL rendering class and call its<br>
UpdateFrame and RenderFrame methods at the right time. The 1.1 code path<br>
compiles and runs on both MT and MD.<br>
<br>
I'm happy to share the code if you or anyone else is interested.<br>
If I have time I'll upload something to github, with the hope that it<br>
makes it into both the MonoTouch or MonoDroid samples.<br>
<br>
Dominique.<br>
