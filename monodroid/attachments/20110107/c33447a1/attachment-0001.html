&gt; How big difference is there on MonoDroid and MonoTouch?<br>
<br>
Gigantic.  That&#39;s why they have different names (HHOS). :-)<br>
<br>
The &quot;core class libraries&quot; are the same between them -- mscorlib.dll, System.dll, etc.<br>
<br>
However, what makes MonoTouch...MonoTouch, is monotouch.dll [0], which contains classes such as MonoTouch.UIKit.UIButton, and everything else that is used to make the UI, plus integration with the underlying CocoaTouch frameworks.  None of this is available on MonoDroid.<br>

<br>
Likewise, what makes MonoDroid...MonoDroid, is Mono.Android.dll [1], which contains classes such as Android.Widget.Button, and everything else that is used to make the UI, plus integration with the underlying Android platform.  None of this is available on MonoTouch.<br>

<br>
Consequently, a straight &quot;port&quot; is ~impossible, as the entire UI+contacts+maps+html+etc. APIs are completely different.<br>
<br>
Instead, if you want to share code, you need to carefully design your app (e.g. via MVC/MVVM/etc.) so that platform-specific code is separated from platform-neutral code.  Core data structures, algorithms, file I/O, networking, etc. should all Just Work, if you do things properly.<br>
