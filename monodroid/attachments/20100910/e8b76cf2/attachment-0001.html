Most of the types we added over Silverlight fall into three categories:<br>
<br>
 1. Feature requests (System.Xml.dll, numerious System.dll additions)<br>
 2. OpenGLES support via OpenTK.dll<br>
 3. SQLite support via Mono.Data.Sqlite (which brings in <a href="http://ADO.NET" target="_blank">ADO.NET</a>)<br>
<br>
(1) were added during the MonoTouch betas and release processes, as<br>
people attempted to port their existing code to MonoTouch.  (Apparently<br>
they either hadn&#39;t tried to port to Silverlight or didn&#39;t care about<br>
Silverlight at the time.)  We added these types and assemblies as a<br>
porting aid.<br>
<br>
(2) is because we don&#39;t have a native XNA implementation (though there&#39;s<br>
an XNA implementation which is built atop OpenTK, iirc).  This is<br>
basically required.<br>
<br>
(3) is because (originally) iPhoneOS 3.x included SQLite, as does<br>
Android, so SQLite is part of the common platform.  We felt that<br>
developers would find an <a href="http://ADO.NET" target="_blank">ADO.NET</a> provider for SQLite useful (and<br>
superior to needing to write/include their own SQLite interop code or<br>
skip the feature outright).<br>
<br>
Then there&#39;s Mono.Android.dll and monotouch.dll, which are required for<br>
native platform access.  Skipping them isn&#39;t really an option.<br>
