I&#39;d add that I&#39;ve mostly opted to find .net ways around using service bindings.. It won&#39;t be the best in all cases, but a simple static event living in the service class that passes a reference to itself to the event handler can go a long way :)<div>
<br></div><div>Then you make your Activity register and unregister for the event on its resume and pause overrides... Simple, and I don&#39;t think the performance or memory implications are terrible.<br><div><br clear="all">

<br>
</div>Binding to a service and using AIDL are related but not identical.<br>
<br>
You _can_ start a Service using Context.StartService()[0], bind to a service using Context.BindService()[1] , unbind using Context.UnbindService()[2], and use the service. We even have a sample [3, 4]. :-)<br>
<br>
Where AIDL comes in is when you want to use Inter-Process Communication to a Service hosted in another process, as AIDL is used to describe the messages that can be sent to the Service. This is not currently &quot;natively&quot; supported by Mono for Android, but it should be possible to use Mono for Android&#39;s Java interop features [5, 6] to run the `aidl` program as a pre-build step to generate .java sources, then implement and include those .java sources into your build so that your app and other apps may use them.<br>
