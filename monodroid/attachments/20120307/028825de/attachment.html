<br></div><div class="im">ClickOnce service is broken on Windows 8 and since setup.exe is a ClickOnce bootstrapper, it makes our installer to
<br>appear broken. The last line from the log above means that ClickOnce attempts to start the actual installer, but the
<br>service it uses (dfshim) for some unknown reason fails to do so. You can still use the installer by finding the
<br>directory where CLickOnce put the downloaded application. Unfortunately it's not possible to tell where it is exactly,
<br>it changes from system to system. You need to find directory with WindowsInstaller.exe along with
<br>Xamarin.Web.Installer.dll - the two might be in several directories, choose the one which has lots of dlls and files.
<br>Run WindowsInstaller.exe from there.
