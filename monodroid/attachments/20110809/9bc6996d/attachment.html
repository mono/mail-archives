Dear Simon,<div><br></div><div>This issue has recently been brought up in Stackoverflow. It seems like the MonoDevelop Android plug-in relies on a 4.0 version of the Mono.Posix library, whereas only the version 2.0 version is available in the Gtk# runtime. Obtaining the 4.0 Mono.Posix version from a recent Mono package (2.10.3 for example), copying the DLL to your personal MonoAndroid add-in folder and re-starting MonoDevelop should do the trick. At least it has worked for me. For more detailed information, see this answer on StackOverflow: <a href="http://stackoverflow.com/questions/6202820/mono-for-android-not-appearing-in-the-monodevelop-options/6565231#6565231">http://stackoverflow.com/questions/6202820/mono-for-android-not-appearing-in-the-monodevelop-options/6565231#6565231</a> .</div>
