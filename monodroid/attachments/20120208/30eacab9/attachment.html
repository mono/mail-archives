This is one of those interesting areas.  I have never had an end user complain about application size.  The only user type that has ever complained about an app size is another developer user, which isn't accurate view of your userbase.  There is a legitamate worry with apk size, there is a limit on the download size over cell networks.  I think the generally accepted download size is 20 megs over cell and 100 megs over wifi, but that may only be apple.  <BR> <BR>Why is the market reporting 12 megs, I'm not sure.  Never have paid attention to it.<BR> <BR>The application size is not something to <strong><em>*typically*</em></strong> worry about regarding performance.  This is a myth in computer programming, assuming that the size is due to features.  Typically size is associated with features.  A more worrisome issue is something called a working set size of an application - <a href="http://en.wikipedia.org/wiki/Working_set_size">http://en.wikipedia.org/wiki/Working_set_size</a>.  Working set size is more a product of the use of good algorithms instead of programming language.  I remember writing applications in VB that performed better than C++ apps in the late 90s.  Now, this isn't true in every situation, and no, I am not the awesome programmer that knows everything.  I do tend to obsess over my algorithms.  Picking a good algorithm tends to overcome the language used.<BR> <BR>Wally<br><BR><div><hr id="stopSpelling">From: john@murray.gb.com<br>To: monodroid@lists.ximian.com<br>Date: Wed, 8 Feb 2012 09:35:01 +0000<br>Subject: [mono-android] size of apk<br><br>
