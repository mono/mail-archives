</div>This is a bug.<br>
<br>
Android doesn't like PascalCased resource names for filename-backed resources, and @layout/ActivityAthleteTwopane is (presumably) file backed as Resources\layout\ActivityAthleteTwopane.axml.<br>
<br>
Mono for Android adds support for PascalCased resource names by lowercasing all the resource names + references as part of the build process (see the files in e.g. obj\Debug\res). Unfortunately I didn't know about the <item/> element, or that the <item/> element could contain a resource reference. Result: <item/> isn't fixed, and contains @layout/ActivityAthleteTwopane instead of @layout/activityathletetwopane.<br>

<br>
The workaround? Use the "internal" name, @layout/activityathletetwopane.<br>
