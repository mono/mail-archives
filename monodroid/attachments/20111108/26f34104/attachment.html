        reference said project from either platform.</div>

    Our team does it with single solution and projects for each platform
    and shares on source<br>
    code level with links to source files (cs, xaml etc) not IL (by
    referencing assemblies).<br>

    PL with<br>
    assemblies per platform... Controlers and ViewModels in MVC and MVVM
    can be <br>
    moved to those assemblies...<br>
    <ul>
      <li>Business logic (BL) per platform linked with Project Linker. <br>      </li>
      <ul>
        <li>Starting with WP7x project as "smallest common denominator -
          subset" <br>
          (because of various restrictions - Android Mono4Android and
          iOS <br>
          MonoTouch are still in front).</li>
        <li>all projects have WP7 project as "linked source project"
          (Project Linker terminology)<br>
          meaning code in them/their folders is just a link to cs file
          in WP7x prj<br>
        </li>
      </ul>
      <li>Presentation Layer PL dlls (User Control library project
        types)<br>
      </li>
      <ul>
        <li>our team uses them for reusing <br>
        </li>
        <ul>
          <li>some UI code if it can be shared across projects<br>
          </li>
          <li>xaml controls (write up is on the way).</li>
        </ul>
