</div>This should work. I suspect that what's happening is that the resource name isn't your filename (very often the EmbeddedResource will have a namespace prefix).<br>
<br>
If you're on OS X you can verify this with `monodis`:<br>
<br>
        $ monodis --manifest *.dll<br>
        Manifestresource Table (1..1)<br>
        1: public 'Bxc8410.Resources.AboutResources.txt' at offset 0 in current module<br>
<br>
That was from setting the Build action for AboutResources.txt to "EmbeddedResource"; note that it has a namespace + folder prefix, not just the filename.<br>
<br>
`ildasm` will also provide you the name of embedded resources.<br>
