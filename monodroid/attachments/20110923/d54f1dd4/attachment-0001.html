<blockquote class="gmail_quote" style="margin:0 0 0 .8ex;border-left:1px #ccc solid;padding-left:1ex;"><div class="im">On Sep 23, 2011, at 2:40 AM, Mittchel Van Vliet wrote:<br>

<br>
</div>If you&#39;re careful, it is possible to share the same assembly between MonoTouch and Mono for Android. You will need to take care to only reference assemblies common to both platforms (e.g. don&#39;t reference monotouch.dll, OpenTK.dll, Mono.Android.dll), and not use features that MonoTouch doesn&#39;t have (no System.Reflection.Emit), but otherwise it&#39;s possible to share the same assembly between MonoTouch and Mono for Android.<br>

<br>
The one problem is that there is currently no Project you can create that can be used by both MonoTouch and Mono for Android. What you would instead need to do is create a e.g. MonoTouch Library project, and then reference the output assembly from your Mono for Android Application project.<br>

<br>
