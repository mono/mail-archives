    dlls as such only lay on MacOS filesystem. Mono (.net) runtime takes
    care to load and interpret <br>
    them in sense it knows what to do with them. So it does not have too
    much to do with MacOS, <br>
    rather MonoTouch.<br>
    <br>
    To see how Your dll is coping with Mono<br>
    <ul>
      <li>install mono on Windows, <br>
      </li>
      <li>install MoMA (Mono Migration Analyzer)</li>
      <li>open up this dll with MoMA and it will tell You how far from
        Mono You are</li>
      <ul>
        <li>MonoTouch is suportting 2.10.5 from recently and this should
          cover a lot of<br>
          new features from .net 3.5 and 4 if You are using them.</li>
        <li>fix the differences while on desktop/asp.net side, cos it is
          easier to debug</li>
        <li>migrate code to MonoTouch<br>
        </li>
      </ul>
      <li>database is a different problem<br>
      </li>
