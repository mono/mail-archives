However, we can probably create our own convenience method(s) that will<BR>
do that:<BR>
<BR>
public void StartActivity (string activity);<BR>
public void StartActivity (Type activity);<BR>
<BR>
Note that you do not create an instance of your activity to activate<BR>
like you would with Winforms. &nbsp;StartActivity is more like Process.Start,<BR>
so you pass it a type instead of an instance, which we would convert to<BR>
a string, which would get a default Intent created and passed to<BR>
StartActivity.<BR>
<BR>
Please file a bug for this enhancement!<BR>
