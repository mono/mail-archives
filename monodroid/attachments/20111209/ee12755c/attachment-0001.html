<div>I am preparing our app for release. The app talks against several sharepoint webservices. And it was behaving perfectly in debug mode, when I switched it to release mode it stopped working. </div><div><br></div><div>To potentially spare other people the same frustrations I experienced. <div>
<br></div><div>I want to share a small checklist</div><div>1) on the build tab check &quot;Generate serialization assembly&quot; is set to off (<a href="http://stackoverflow.com/questions/5792193/error-building-a-release-version-when-referencing-a-web-service">http://stackoverflow.com/questions/5792193/error-building-a-release-version-when-referencing-a-web-service</a>)</div>
<div>2) do you have the INTERNET permission (from this list <a href="http://mono-for-android.1047100.n5.nabble.com/WebServices-not-working-in-Release-Mode-Signed-apk-td4362833.html">http://mono-for-android.1047100.n5.nabble.com/WebServices-not-working-in-Release-Mode-Signed-apk-td4362833.html</a>)</div>
<div>3) on the mono for android tab the linking setting if you have followed <a href="http://docs.xamarin.com/android/tutorials/Preparing_Package_for_Android_Marketplace">http://docs.xamarin.com/android/tutorials/Preparing_Package_for_Android_Marketplace</a>, you probably have the setting linking set to &quot;sdk and user assemblies&quot; like i did. In my case that was not such a great idea, my webservice did not return to much results anymore. Try to set the linking to &quot;sdk assemblies only&quot;. Perhaps the screenshot from the Xamarin site should be changed, after reading up it looks like linking also user assemblies seems more risky.</div>
</div><div><br></div><div>Looking back it looks like a small list but it kept me busy 6 hours ;), so i hope it will help somebody else...</div>
