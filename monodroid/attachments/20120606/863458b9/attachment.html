</div>The problem is getting the method parameter names: Java .class files contain type and member names but they don't contain parameter names, i.e. the .jar is useless for parameter names.<br>
<br>
Where do we get parameter names from? JavaDoc. Our .jar binding tool has support to parse JavaDoc HTML to get decent parameter names. Unfortunately, there doesn't appear to be a good way to specify this within the IDE, and there is a great deal of variance in javadoc output, all of which we don't support.<br>

<br>
Consequently, you'll need to do two things:<br>
<br>
1. Download the JavaDoc for the library you're binding. (This may or may not be possible.)<br>
2. Edit the .csproj and add a <JavaDocPaths/> property:<br>
<br>
        <PropertyGroup><br>
                <JavaDocPaths>Path\To\JavaDocs</JavaDocPaths><br>
        </PropertyGroup><br>
<br>
3. Clean and rebuild the project.<br>
<br>
Once this is done, assuming the HTML scraper does its job properly, the generated api.xml should contain useful parameter names.<br>
