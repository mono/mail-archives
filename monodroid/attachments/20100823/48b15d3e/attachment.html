              &gt; will compile in MonoTouch for iOS and MonoDroid for

            As I believe we've mentioned elsewhere, this is currently
            impossible<br>
            (and will likely remain the case). &nbsp;You'll be able to share
            *some* code<br>
            between iOS/Android/WP7 (hopefully lots, though that depends
            upon the<br>
            design of your app), but not all. &nbsp;In particular, the GUI
            code will be<br>
            using different types depending on whether you're using
            MonoTouch<br>
            (MonoTouch.UIKit), MonoDroid (Android.Views), or WP7
            (System.Windows).<br>
            Unless/until Novell writes Moonlight support for MonoTouch
            and MonoDroid<br>
            (which may never happen), there will be no cross-platform UI
            toolkit.<br>
            <br>
            Furthermore, I'm not convinced that such a thing is useful;
            see:<br>

            hardware Menu<br>
            button, which neither iOS nor WP7 provide. &nbsp;I haven't

            up to the<br>
            Menu button, and the probable result would be a WP7-looking
            app on<br>
