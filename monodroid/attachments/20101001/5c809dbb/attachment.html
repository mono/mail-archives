</div>Maybe.  Preview 4 does not contain the DLR, so any dynamic languages are<br>
out.  Consequently, if you have a language which can be made to depend<br>
only on the assemblies shipped with MonoDroid [0], that language should<br>
work.<br>
<br>
Thus, it *should* be possible (in theory [1]) to get Visual Basic.NET<br>
working, as long as you remove all dependencies on<br>
Microsoft.VisualBasic.dll.<br>
<br>
(Nemerle is slightly more problematic, as iirc they can&#39;t easily make<br>
their compiler target profile assemblies that differ from the profile<br>
the compiler was built against.  F# could plausibly work...if you can<br>
(again) remove all dependencies on the FSharp*.dll assemblies OR<br>
recompile those assemblies and bundle them with your app [1].)<br>
<br>
In short, Try It And See.  If the language can target Silverlight,<br>
there&#39;s a high likelihood that it&#39;ll work on MonoDroid (though<br>
Silverlight includes Microsoft.VisualBasic.dll while MonoDroid doesn&#39;t,<br>
so see above advice on VB support).<br>
