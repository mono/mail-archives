> I have problem with my app: it crashes randomly (but pretty frequently). Before crash the message:<br>
> Thread 0x0 may have been prematurely finalized<br>

<br>
</div>The problem is that most mono messages are printed to stdout/stderr, which Android dumps on the floor. (Only a few mono messages make it to logcat.)<br>
<br>
The quick fix is to use an API16 (Jelly Bean) device, which changes logcat behavior so that it also captures stdout and stderror, so you should (in theory) get a full mono crash dump.<br>
<br>
The downside is that, afaik, API16 doesn't currently have an x86 emulator image, so you're either using the (slow) ARM emulator, or you need an API16 device.<br>
