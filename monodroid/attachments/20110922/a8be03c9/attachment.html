      <pre wrap="">The bulk of our business these days is Android ports of iOS apps.
There are many "cross platform" solutions, but they all tend to be a
pain and don't really work as well as one would hope.  What I've been
trying to do is find a cross platform solution that would simply be a
shared code block.  Like a black box logic processor that you could
deploy on either platform, and call into it to do calculations and
business logic.  The UI, and simpler code would be handled in whatever
the native platform preferred.</pre>
    </blockquote>
    &nbsp;&nbsp;&nbsp; There would be 2 concepts:<br>
    <ol>
      <li>sharing on binary level (PInvoke, callable wrappers (COM,
        .net, android, ...)</li>
      <li>sharing on source level with MonoTouch/MonoForAndroid/Mono*<br>
      </li>
    </ol>
    Cons for 1. <br>
    <ul>
      <li>You still need experts in underlying tech that would maintain
        that code if&nbsp; problems<br>
        arise in that part.</li>
    </ul>
    <br>
    Pros for 1.<br>
    <ul>
      <li>Less to do. Just wrap and go...</li>
    </ul>
    <br>
    Pros for 2.<br>
    <ul>
      <li>For BusinessLayer only .net programmers are needed...</li>
      <li>Core can be shared accross projects<br>
        In our case 3 apps currently wp7, monodroid, starting monotouch
        (iOS), silverlight, <br>
        desktop (WF and WPF), asp.net.</li>
      <li>sharing not only cs code<br>
        just writing small writeup how to share XAML code accros WP7,
        Silverlight and WPF<br>
      </li>
      <li>unit testing</li>
      <ul>
        <li>take desktop version in solution add unit tests and You can
          cover a lot, really a lot<br>
        </li>
      </ul>
    </ul>
    Cons for 2.<br>
    <ul>
      <li>need to get used with new tools Project Linker/Productivity
        Tools etc...<br>
        Microsoft came out few years ago with excellent tool Project
        Linker for sharing<br>
        SL and WPF code.<br>
        Today thanks to Mono* you can use it for far more!<br>
        I can rember sharing SL code with desktop by tweaking
        assemblies! <br>
        (let's say binary level, no thanks)<br>
      </li>
      <li>need to find smallest common denominator for tools/features<br>
        for us WP7 (WP7 project is source for linked cs files all other
        projects)<br>
      </li>
      <ul>
        <li>going from superset app (desktop) to subset (mobile,
          silverlight) rewrite all <br>
          code to async, sync versions are kept only in desktop and
          asp.net versions.<br>
          I know that was mistake at the beginning, but at that point
          that was the fastest<br>
          solution, never thought I will program mobile apps.</li>
      </ul>
      <li>faster growing number of projects in solution<br>
        #platforms_targeted * #architectural_layers<br>
        Most superset projects (desktop, asp.net) are almost empty</li>
      <li>quite a bit knowledge about diffs between .net and mono...<br>
        remember when we hit the wall by using Observable collection for
        WP7, SL,<br>
        Then MonoDroid came in. <br>
        Cannot rember which mono base was then, but Observable
        Collection was not there)<br>
        But everything can be done: <br>

href="http://www.java2s.com/Open-Source/CSharp/2.6.4-mono-.net-core/System.Collections/System/Collections/ObjectModel/ObservableCollection.cs.htm"><br>
http://www.java2s.com/Open-Source/CSharp/2.6.4-mono-.net-core/System.Collections/System/Collections/ObjectModel/ObservableCollection.cs.htm</a><br>
        Voila...<br>
        <br>
      </li>
    </ul>
