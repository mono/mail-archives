</div>logoscreen.xnb is ~4MB in size. Android < 2.3 does not support decompressing compressed assets that are larger than 1MB in size:<br>
<br>
        <a href="http://developer.android.com/about/versions/android-2.3-highlights.html" target="_blank">http://developer.android.com/about/versions/android-2.3-highlights.html</a><br>
<br>
                "There is no longer a limit on the size of compressed .apk assets that can be read."<br>
<br>
        <a href="http://stackoverflow.com/q/2860157/83444" target="_blank">http://stackoverflow.com/q/2860157/83444</a><br>
<br>
There are two fixes:<br>
<br>
1. Require Android 2.3+ (thus preventing execution on your GS1 device). (OK, this isn't a fix...)<br>
<br>
2. Do not compress logoscreen.xnb. This can be done by opening Project Options, going to the Application tab, and adding .xnb to the Leave the following resource extensions uncompressed textbox. This also means that your .apk size will increase by ~4MB, as it won't be compressed.<br>

<br>
An alternate 3rd solution is to not use Android Assets/Resources for the .xnb. It's apparently a XNA data file, so there's no actual need for it to be an Android Asset. You can instead set its Build action to EmbeddedResource, and use the System.Reflection APIs to grab it:<br>

<br>
        System.Reflection.Assembly.GetExecutingAssembly ()<br>
                .GetManifestResourceStream ("logoscreen.xnb");<br>
