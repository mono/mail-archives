On Dec 12, 2011, at 2:49 PM, Neal Culiner wrote:<br>
&gt; Should I have both armeabi and aremabi-v7a checked in the application<br>
properties?  I can&#39;t find anything in the MD docs to advise me.<br>
<br>
The largest differences are in hardware floating point support and SMP<br>
(multi-CPU) support.<br>
<br>
armeabi supports neither, so if you use lots of floating point math you&#39;ll<br>
be using a software FPU, and if you run on an multi-CPU device such as the<br>
Xoom or the Samsun Galaxy S II you&#39;ll be tied to one core.<br>
<br>
armeabi-v7a supports both, allowing faster floating point execution and the<br>
use of all available cores.<br>
