<p>I work in a company where most of the applications are written in C#, . NET, all these are written for desktop use. Now the company is looking into getting on the mobile market for some of the applications. What better would be to write native apps reusing a lot of the already existing code. Re-writing the code to each platforms native tongue would be a lot of work. Re-writing the code to spit out html could probably work for some things, but things such as accessing the device hardware becomes VERY hard.</p>

<p>Just my 2 cents. </p>
