Android.Graphics.Point.     <br />     <br /> Java doesn't have value types, 
only reference types. Consequently, Android.Graphics.Point is a reference 
type, and thus `p1` aliases `p2`, which is why changes made to `p2` are 
visible through `p1`.     <br />     <br /> This is not a bug, this is 
Android.     <br />     <br /> If you want a copy, use the Point(Point) 
constructor:     <br />     <br /> &nbsp; &nbsp;var p1 = new 
Android.Graphics.Point(0, 0);     <br /> &nbsp; &nbsp;var p2 = new 
Android.Graphics.Point(p1);     <br />     <br /> &nbsp; &nbsp;// these 
don't change p1     <br /> &nbsp; &nbsp;p2.X = 1;     <br /> &nbsp; 
&nbsp;p2.Y = 1;     <br />     <br /> &nbsp;- Jon     <br />     <br />
