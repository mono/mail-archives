<p class="MsoNormal">Again, the &quot;problem&quot; (such as it is) is that in Java only the builtin types (char/short/int/long/float/double) are value types, and everything else is a reference type. C# and .NET learned from Java's &quot;mistakes&quot; (oversight?) and added a
 &quot;generic&quot; value type mechanism, allowing the creation and use of &quot;custom&quot; value types (structs).<br>
<br>
Consequently, value types can be used in MonoDroid code (and OpenTK uses them), but underlying Android/Java types _cannot_ be value types. This thus becomes an &quot;education&quot; issue, as not all Point/PointF/Rect/RectF types are alike (struct vs. class), and it
 becomes very important to know which one you're dealing with.<o:p></o:p></p>
