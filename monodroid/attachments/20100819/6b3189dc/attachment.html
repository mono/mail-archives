Anyhow, I just ran into the same problem when I did my initial port of Mono to Android. Basically, if you have a .NET Thread running and the garbage collector is started (explicitly or automatically, does not matter), the application will hang/crash. I tracked this down to the pthread_kill in GC_suspend_all (libgc) failing with an EPERM error when trying to stop anything but the main thread (which was created in dalvik land). I tried a few other signals for SIG_SUSPEND, to no avail.</div>
