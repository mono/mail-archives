&gt; However, if I am creating and disposing of a large bitmap every time my activity is recreated (say in an ImageView), I run out of memory before long.<br>

</div>The problem -- which is present on every GC system -- is that Mono&#39;s GC doesn&#39;t know how big the Dalvik-allocated bitmap is (just as .NET&#39;s GC doesn&#39;t know that you just called Marshal.AllocHGlobal() to allocate 4GB of memory...). Furthermore, Mono&#39;s GC _can&#39;t_ know how big the Dalvik-allocated bitmap is, and even if you knew how much memory it took GC.AddMemoryPressure() isn&#39;t supported.<br>



<br>
Thus, in this (and similar) cases you need to help the GC out, as you know things the GC doesn&#39;t, and thus the GC.Collect() call is appropriate.<br>
