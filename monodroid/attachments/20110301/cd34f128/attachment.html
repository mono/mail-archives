<br>They have, but they haven&#39;t updated everything. :-)<br><br>So, background: Mono.Android.dll is created via a code generator which parses an XML description of the Java API to bind. Google has previous
 versions of the XML description at [0], but they have not provided XML descriptions for API level 10 (Android 2.3.3) or API level 11 (Android 3.0). Without these files, we can&#39;t generate an updated Mono.Android.dll, and the Google SDK doesn&#39;t include them.<br>
<br>This is also the reason why MonoDroid 1.0 is only binding android.jar: because that&#39;s the only .jar file that we have an XML description for. Our ~first project after 1.0 will be to write a .jar-to-XML generator so that we can generate an XML description for 3rd party .jar files, thus allowing use of 3rd party .jars, and packaging/cleaning up the source generator for external use.<br>
