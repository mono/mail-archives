      The problem though is the UserSignIn_Async returns void and you
      can't have Task<void>. Changing the return type of the
      method to Task causes a problem at the caller line</blockquote>
    <br>
    First convention is Async w/o _ (this should not matter)<br>
    Async functions can have return types:<br>
    <ul>
      <li>void</li>
      <li>Task</li>
      <li>Task<T></li>
    </ul>
    <p>If the return type is void then new function should return void
      too. Actually compiler makes this<br>
      magic for us if async function returns string or int You can write
      <br>
          return some_string;<br>
      or<br>
          return some_int;<br>
      compiler will wrap it into Task<string>, Task<int>. So
      it should leave <br>
          return; <br>
      (or lack of it) as it is or wrap it in non-generic task.<br>
    </p>
    <p>Why don't  You try<br>
          var getLoginTask = MyEvents.UserSignIn_Async(username,
      password);<br>
    </p>
