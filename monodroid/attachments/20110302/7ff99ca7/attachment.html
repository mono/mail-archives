<br>
In fact:<br>
<br>
        <a href="http://android.git.kernel.org/?p=platform/frameworks/base.git;a=blob;f=tools/aapt/XMLNode.cpp;h=c0d74275a8a6a6883fedeed008ab6e993e4658cc;hb=HEAD#l49" target="_blank">http://android.git.kernel.org/?p=platform/frameworks/base.git;a=blob;f=tools/aapt/XMLNode.cpp;h=c0d74275a8a6a6883fedeed008ab6e993e4658cc;hb=HEAD#l49</a><br>

<br>
If an XML namespace starts with RESOURCES_PREFIX, aka RESOURCES_ROOT_NAMESPACE, aka &quot;<a href="http://schemas.android.com/apk/res/" target="_blank">http://schemas.android.com/apk/res/</a>&quot;, then `aapt` considers the namespace to be &quot;public&quot; and treats it specially, which is your actual problem.<br>

<br>
Consequently, if I change the above xml namespace declaration to:<br>
<br>
        xmlns:binding=&quot;<a href="http://com.cartopac.android.binding" target="_blank">http://com.cartopac.android.binding</a>&quot;<br>
<br>
then `aapt` builds the Eclipse project fine, so I believe the actual problem is that you chose a terrible xml namespace ID.<br>
