[0] except for gigantic app sizes and user complexity [1]... The Mono.Android.DebugRuntime-debug.apk package, containing BCL + ARMv5, ARMv7, and x86 runtimes is "only" 25MB in size. We already have requests for MIPS support, which will make that larger still, and then there's the per-API level Mono.Android.Platform.apk, which is another 12-16MB. Assuming the end user only downloads apps which require a single API level, you're still requiring that they download 37+MB just to run your app, as opposed to the ~2+MB size that is currently required.<br>
