On Oct 13, 2011, at 5:40 AM, Nicklas Møller Jepsen wrote:

Answering things out of order...

> What can I do to successfully install a Mono app?

If you can debug the app in VS, then the app _has_ been installed onto the device, and you should be able to launch your app from the Android launcher and have it load. If that isn't the case, something is _very_ screwy, as VS is using normal Android interfaces to launch Activities...

Also, are you sure you're installing the Release build and not the Debug build? Alternatively, does your Release build have the shared runtime _disabled_ (as it is, by default)?

> When I start my application I get java.lang.ClassNotFoundException:  android.widget.ItemClickImplementor
> This happens ONLY when I manually install the APK – not when debugging in VS.
> I’m running in Release mode with linking turned off.

This looks like a shared runtime mismatch, e.g. your (Debug/shared runtime) app was built against Mono for Android 1.2.0 but you have the 1.9.x shared runtime installed on your device (or vice versa; things are getting hazy in my memory). Are you using the shared runtime? If so, is VS using the same version as is present on your device?
