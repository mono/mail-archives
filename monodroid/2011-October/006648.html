On Oct 10, 2011, at 9:33 PM, Nirban Dutta wrote:
> Could you please suggest, what needs to be done to consume this web service.

I'm not sure why it's failing. A workaround would be to set the ServicePointManager.ServerCertificateValidationCallback property to a delegate which will validate the certificate (or just skip validation entirely), e.g.:

	System.Net.ServicePointManager.ServerCertificateValidationCallback += (sender, cert, chain, errors) => {
		return true;
	};

That should disable the certificate check, and allow things to validate. If that doesn't work, something else is going wrong. :-/
