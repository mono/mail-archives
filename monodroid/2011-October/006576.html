I figured it out.

Using "android:id=@android:id/tabhost" instead of "android:id="@+id/tabhost"
made everything work fine. But it only works if TabActivity is subclassed.
It doesn't work in ActivityGroup subclass because tab host must be
initialized with FindViewById<TabHost> and "@android:id" entries are missing
from the Resource class. Is this a bug or by design?
