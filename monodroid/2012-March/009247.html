On Feb 24, 2012, at 8:53 PM, Zach Bean wrote:
> I've got a project with target and minimum SDK values that aren't the same (13 and 7, respectively).

Is this with Mono for Android 4.0.3 or later?

	http://android.xamarin.com/Releases/Mono_for_Android_4/Release_4.0.3#Changes_since_4.0.1

What TargetFrameworkVersion are you targeting? (In MonoDevelop: Project > ProjectName Options ; Build / General "tab"; Target Framework drop-down)

TargetFrameworkVersion should be Android v2.1 if you want the minimum SDK to be 7.

Debug builds, with the shared runtime enabled, should work (and should greatly speed up developer turnaround).
