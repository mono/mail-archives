Firstly thanks for fixing bug #635565, this seems to work the way it is
supposed to now.

As previously mentioned, I've created a simple cross-platform OpenGL
example that supports OpenGL ES 1.1 and 2.0.

When compiling the code under Android, I found that there is a slight
difference between how OpenTK MonoTouch handles getting Shader information
compared to MonoDroid.


Under MT, this works...
                string pszInfoLog = string.Empty;
                GL.GetShaderInfoLog(shader, i32InfoLogLength, ref
i32CharsWritten, pszInfoLog);

But under MD, I need to change the pszInfoLog variable to being a
StringBuilder object....
                StringBuilder pszInfoLog = new StringBuilder();
		GL.GetShaderInfoLog(shader, i32InfoLogLength, ref i32CharsWritten,
pszInfoLog);

Not a massive issue, but it would be nice if they worked identically under
both devices. The fewer IFDEFs the better IMHO. My preference is for them
to both be strings, rather than a StringBuilder objects, but at the end of
the day, they just need to be consistent.

Should I create a bug report for this?

Other than that, the example compiles and works on both simulators now.

Last question, does the MonoDroid compiler have a built in compiler
variable that I can use to IFDEF my shared code between MonoTouch and
MonoDroid,
Or should that be handled at Project level?
