> I made my own attempt but it’s by no means fully thought out and I’m not
> decided on the implicit vs. explicit. I rely on the IDE to help me knead
> the code so I like to capture as much of the semantics as I can. (oops,
> I used a field instead of a property, so I’m sloppy).
>
> struct clr {
>
> public static clr Red = new clr(0xff, 0, 0);
>
> public static clr Green = new clr(0, 0xff, 0);
>
> public static clr Blue = new clr(0, 0, 0xff);
>
> public static clr Cyan = new clr(0, 0xff, 0xff);
>
> byte R;
>
> byte G;
>
> byte B;
>
> public clr(byte R, byte G, byte B) {
>
> this.R = R;
>
> this.G = G;
>
> this.B = B;
>
> }
>
> public clr(int rgb) {
>
> this.R = (byte)((rgb >> 16) & 0xff);
>
> this.G = (byte)((rgb >> 8) & 0xff);
>
> this.B = (byte)(rgb & 0xff);
>
> }
>
> public int color { get { return (R << 16) | (G << 8) | B; } }
>
> public static implicit operator int(clr color) {
>
> return color.color;
>
> }
>
> }

> This is a great idea. Unfortunately, we discussed it a bit and I don't
> think it's feasible to fully implement this. The use of int for colors
> is too ingrained in Android for us to fully replace it (methods,
> properties, constants, resource files).
>
> What we can do is beef up the Android.Graphics.Color class. Currently,
> it basically is a static class with static methods to convert RGBA to
> ints. I can make it an instance class with the traditional constructors
> like Color (int r, int g, int b, int a) and properties for each color
> component. I can also add the implicit cast to int, so you can do
> something like this:
>
> Color c = new Color (255, 255, 255, 255); Paint p = new Paint ();
> p.Color = c;
>
> The only missing part is "Paint.Color" will still be an int, so
> Intellisense won't point you to the Color class. Which sucks, but looks
> unavoidable.
>
> Additionally, I can add the usual "Named" colors to the Color class,
> like Color.Aquamarine, etc. Android provided 9 named color constants
> that I moved to Android.Graphics.Colors, but .Net developers are used to
> having the full box of Crayola at their disposal. ;)
>
> Please let me have any feedback about how useful this proposed change
> would be, and absolutely let me know of any other places where we can
> make your life easier!
>
> Thanks!
