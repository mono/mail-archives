OK, greater C# has multiple meanings. One is that it is greater than Java
but I don't want to visit that. In this case by "greater" I mean the way the
language is used as in using typed objects instead of integers whenever
feasible.

 

It would be nice to have a Color class structure instead of using int. Part
of this is that I haven't find where named colors are in the libraries but
also because it's the right thing to do. I wrote my own but it's pretty
straightforward with the option of implicit conversion to int if that's
useful.

 

Of course it's the start of a slippery slope towards using reflection
instead of resource files for colors and other standard objects. So let the
slide begin.
