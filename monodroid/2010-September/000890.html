Oh, before I forget, just realized I forgot to point out the "(no authors)"
issue related to the authors file used by git-svn and svn2git (definitely go
with svn2git) when dealing with Google Code (and other) SVN repositories
which has rev 0 set as (no author).  You need to add an entry to the top of
the authors file for the repository you're importing to avoid getting and
"Author: (no author) not defined in .. file" error.

e.g. /path/to/authors.txt would look like:

(no author) = marcgravell <marcgravell at github.com>
xmlhacker = mdavid <mdavid at github.com>
marc.gravell = marcgravell <marcgravell at github.com>

... which will fix the above error when importing the SVN repository.
