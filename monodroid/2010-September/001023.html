On Sat, 2010-09-25 at 13:59 -0400, Miguel de Icaza wrote:
>         Is there a plan to support the dynamic keyword that was
>         introduced in C# 4.0?
>
> Odd, I could swear we had just released with support for it.

We just released based on mono 2.8 (instead of 2.6), which is not the
same thing as using C# 4.0.

Specifically, our profile code is still based on the MonoTouch-esque
Silverlight 2.1 + extensions set of types & assemblies, which has lots
of types in the "wrong" places for .NET 4 support, e.g.:

  - System.Action is in System.Core.dll, not mscorlib.dll
  - Ditto System.Func (all "overloads").

Then there's all the missing types and namespaces:

  - The System.Dynamic namespaces doesn't exist in System.Core.dll
  - Nor does System.Linq.Expressions.BlockExpression, etc.

Then there's the (perpetual) size issue: in the .NET 4.0 profile,
mscorlib.dll is 2.8MB and System.Core.dll is 796KB.  In the currently
shipping MonoDroid profile, mscorlib.dll is 2.4MB and System.Core.dll is
256KB.  Things will be bigger, but enough bigger to prevent it?  I can't
say.

Finally, we should note that while Silverlight 4.0 does have
System.Dynamic, Windows Phone 7 does *not* have System.Reflection.Emit
(or so I've been told by those who've tried to use it; MSDN isn't very
helpful here), so I imagine it's also missing System.Dynamic (as I can't
imagine how you'd implement System.Dynamic w/o SRE).

 - Jon
