Because my main line development path is XNA on the Windows desktop.

My target platforms are:  Windows Desktop, WP7, Zune HD, iPhone and now
Android.

OpenTK forces makes it very difficult for me to maintain a consistent type
system (XNA compatible types) across all those different platforms.  I end
up copying data around unnecessarily.  On top of that, I don't much care for
trying to present OpenGL in a "friendly" OOP form.  It's nice for some
people, but the API differences mean it is harder to port existing code and
I have to mentally switch gears more often.

I know that there is an effort to layer XNA on top of OpenTK, but that just
seems backwards.  Much better to have a cross platform XNA library that
talks directly to the underlying native API.

Don't get me wrong, I think OpenTK is the right choice for lots of people.
 Just not for me.

-chris
