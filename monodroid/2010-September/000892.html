> Oh, before I forget, just realized I forgot to point out the "(no authors)"
> issue related to the authors file used by git-svn and svn2git (definitely go
> with svn2git) when dealing with Google Code (and other) SVN repositories
> which has rev 0 set as (no author).  You need to add an entry to the top of
> the authors file for the repository you're importing to avoid getting and
> "Author: (no author) not defined in .. file" error.
>

I should have just written a script and posted that instead to ensure that
all contributors and variations thereof (for some reason Google Code SVN
repositories seem to have a lot of variations in usernames representing the
same contributor) are found and a proper author file generated. You may want
to edit the resulting file to better match the desired user name and email
on git, but this provides the basic starting template:

PROJECTNAME=protobuf-net mkdir -p ~/.svn2git && echo "(no author) =
name-of-replacement-for-no-author email-for-no-author at example.com" >
~/.svn2git/authors && for name in `svn log http://$
PROJECTNAME.googlecode.com/svn | grep -E "r[0-9]+" | grep "|" | grep -v "(no
author)" | awk '{print $3}' | sort | uniq`; do echo "$name = $name <$
name at gmail.com>" >> ~/.svn2git/authors; done

.. will generate ...

(no author) = name-of-replacement-for-no-author
email-for-no-author at example.com
marc.gravell = marc.gravell <marc.gravell at gmail.com>
xmlhacker = xmlhacker <xmlhacker at gmail.com>
xmlhacker at gmail.com = xmlhacker at gmail.com <xmlhacker at gmail.com@gmail.com>

... at ~/.svn2git/authors with svn2git will then automatically use if it
finds it when importing an SVN repository.  The generic gist is @
http://gist.github.com/590911
