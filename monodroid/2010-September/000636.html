Just an FYI on hand editing the AndroidManifest.xml for anyone else that may
need this information in the future, since it took me a couple minutes to
figure out the correct setting of

<activity android:name="..">

Let's say you have a type defined as:

// file: SampleMenusActivity.cs
namespace SampleMenusApplication
{
    public class SampleMenusActivity : Activity
    {
       public SampleMenusActivity (IntPtr handle) : base ( handle )
        {
        }

        protected
        override
        void
        OnCreate (Bundle savedInstantState)
        {
            base.OnCreate ( savedInstantState );

            SetContentView ( R.layout.main );
        }
    }
}

The default AndroidManifest.xml in the Properties folder of your solution
will then look like this..

<?xml version="1.0" encoding="utf-8"?>
<manifest xmlns:android="http://schemas.android.com/apk/res/android"
package="com.example.sampleMenusApplication" android:versionCode="1"
android:versionName="1.0">
  <application android:label="SampleMenusApplication">
  </application>
  <uses-sdk android:minSdkVersion="7" />
</manifest>

If say you want to change the icon or label of the application or register
the activities intent filter to handle specific actions, etc.. then modify
the AndroidManifest.xml to look like this..

<?xml version="1.0" encoding="utf-8"?>
<manifest xmlns:android="http://schemas.android.com/apk/res/android"
package="com.example.sampleMenusApplication" android:versionCode="1"
android:versionName="1.0">
  <application
        android:icon ="@drawable/icon"
        android:label="Sample Menus "
    >
    <activity
        android:name="sampleMenusApplication.SampleMenusActivity"
        android:label="Sample Menus Application"
      >
      <intent-filter>
        <action   android:name="android.intent.action.MAIN" />
        <category android:name="android.intent.category.LAUNCHER" />
      </intent-filter>
    </activity>
  </application>
  <uses-sdk android:minSdkVersion="7" />
</manifest>
