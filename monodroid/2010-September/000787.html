I'm not sure how this would be possible, as monodroid.exe is using
System.Diagnostics.Process to execute javac/jar/etc.  Even if
monodroid.exe were a 32-bit process and javac were a 64-bit process,
they'd be in two separate address spaces, so there shouldn't be a
problem.

That said, I actually have both the 32-bit and 64-bit JDKs installed, so
I don't know which is actually being used.  (I originally tried using
the 64-bit one, but iirc the then-current Android SDK required the
32-bit JDK.  Alas, that was months ago, so I forget the details.)
