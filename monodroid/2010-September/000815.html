Just noticed that MonoDroid does two-way name normalization:

*         Any Java stuff (like package name, getters/setters, etc.) are cleaned up for .Net. (We know about this)

*         Any .Net stuff is transformed into the [horrible] Java naming convention. So even though my class/namespace is "SourceCode.TaskDroid.Authenticator.Authenticator" MonoDroid exports it as "sourceCode.taskDroid.authenticator.Authenticator".

I couldn't find this in the docs, so I thought I would share it. Keep it in mind when you are developing services and whatnot.
