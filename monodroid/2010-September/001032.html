Hi Miguel,

 

There's a problem with dynamic that sometimes it seems as if it's supported
when in fact it's not.

 

 

In order for it to work, it required a reference to Microsoft.CSharp. WP7
doesn't have it, and MonoDroid doesn't too... Any suggestion?

 

Here's a code example of what dynamic can do (operator overloading for
generics). Unfortunately, it doesn't compile on either MonoDroid or WP7. It
does compile on Silverlight and .NET 4.0.

 

    class Test<T>

    {

        public T Data;

 

        public Test(T data)

        {

            Data = data;

        }

 

        public T Add(Test<T> test2)

        {

            dynamic d1 = Data;

            dynamic d2 = test2.Data;

 

            dynamic result = d1 + d2;

 

            return result;

        }

    }

 

The line that won't compile is:

 dynamic result = d1 + d2;

However, it WILL compile on Silverlight, or whenever there's a proper
reference to Microsoft.CSharp
