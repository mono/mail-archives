On 9/2/2010 1:00 AM, Amir (GMail) wrote:
> What's the safest way of doing it using Class Libraries?

I think your best bet is to use the new MonoDroid Class Library template 
that shipped with the 2nd preview.

It will of course work in MonoDroid.

It should work in MonoDevelop/MonoTouch once MonoDroid for MonoDevelop 
ships.

WinPhone7 won't let you add a project reference to it, but it will let 
you add an assembly reference to it.  It's not ideal, but at least you 
are still sharing the same assembly.
