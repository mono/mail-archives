This is already (somewhat) provided through the
ActivityAttribute.MainLaucher property:

        http://docs.monodroid.net/index.aspx?link=P:Android.App.ActivityAttribute.MainLauncher
        http://monodroid.net/Tutorials/Working_with_AndroidManifest.xml#Launchable_from_Application_Chooser

Which allows:

        [Activity (MainLauncher=true)]
        class MyActivity : Activity {}

Beyond that, I'm not sure what more should be done. For activities
without ActivityAttribute.MainLauncher or IntentFilterAttribute, why
should monodroid generate any //intent-filter elements?  I doubt that
Intent.CategoryDefault for some generated activity makes sense, and
further raises the question of how to *prevent*
generating //intent-filter elements if none are desired.

 - Jon
