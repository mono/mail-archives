On Wed, 2010-09-08 at 14:13 -0600, Tom Opgenorth wrote:

> So I did something like this in C#:
> 
> public abstract class ChangeScoreListener: Android.Views.View.IOnClickListener
> {
> }
> 
> I notice that IOnClickListener has an extra method in the interface:
>         public IntPtr Handle
>         {
>             get { throw new NotImplementedException(); }
>         }
> 
> What is this for? What should I be returning?

It's there to make it easier to discern that it's a java interface being
implemented, and to support efficiency in the binding when passing
interface references to the native API.  The suggested way to implement
interfaces which derive from IJavaObject is to subclass
Java.Lang.Object, which gets you the Handle property for free.

However, in the next preview, the View.IOnClickListener will be removed
from the API.  The View.Click event is the preferred, and soon to be
only, way to receive notification of clicks on a view.  We are moving to
an EventHandler<> based event system to expose all java *Listener API.
