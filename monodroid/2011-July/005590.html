Hello,

 From the structural aspect...

I believe this Q should be first directed to android platform itself:

- At run time, Mono for Android basically runs on top of android
   framework and apps are in dalvik VM bytecode. There is no
   point that Java 7 affects here.
- Dalvik bytecode (dex) is compiled from Java classes by some
   android SDK tool.
- Whether those Java .class files can be compiled from .java
   using Oracle Java7 is Android's matter. Mono for Android often
   generates java sources for corresponding C# classes, but
   whether Java 7 javac compiles those sources or not is almost
   the same as whether it compiles typical Android apps or not.

While the new compiler should work like earlier versions, there
are some projects that are known to fail to build, so I wouldn't
be surprised if it sometimes fail to compile.

For the runtime, Android platform source (AOSP) has a *fixed*
dependency of either Java 6 (gingerbread and later) or Java 5
(froyo or earlier). I don't think android dev. team will soon
replace dependency.
