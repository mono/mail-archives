On Jul 26, 2011, at 8:30 AM, Wally McClure wrote:
> To add to this, I have also seen issues where something happens in the try part of the try-catch and nothing is caught. 

The explanation for this is straightforward: the mono runtime aborted. No exception is thrown for most of mono's abort conditions.

Eventually, running within the debugger should allow you to get the stack trace for when mono aborts, but that doesn't help if you're not debugging. However, it's entirely plausible that mono will log the stack trace/etc. to `adb logcat`; it would be interesting if someone could verify this.

As always, if you do get a repro for this, please file a bug.
