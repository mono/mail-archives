On Aug 29, 2012, at 1:36 AM, SRI <srid68 at gmail.com> wrote:
>        Is something wrong with my understanding of Minor Version in MonoDroid. Is it because of Alpha/Beta.
> 
>        I am already running mono-android-4.2.5.195540401.msi (from 7/27/2012) from the Alpha Chanel and now today morning was prompted for the Beta but the Version Numbers are
> mono-android-4.2.5.114017294.msi which seams to be less. (114017294 < 195540401)

That's not the minor version value, that's the revision value:

	http://msdn.microsoft.com/en-us/library/kstyaxxh.aspx
	major.minor.build.revision

The revision value is currently "meaningless;" it's the base-10 git abbreviated commit hash that we built from. As such, it's "random", and certainly not always incrementing.

Yes, you should upgrade. :-)
