On Dec 2, 2010, at 10:46 PM, Greg Shackles wrote:
> Does MonoDroid support service bindings yet, both in creating your own and also in binding to someone else's AIDL?

This is something we've put no thought into, and will be a post 1.0 feature.

For those needing to support it "now" (well, in P9) "by hook or by crook," you will be able to do so by:

 1. Using the normal Android toolchain to generate .java files/etc.
 2. Bundle those Java sources (and any additional custom Java files you need to write) in your .apk:

	http://monodroid.net/Releases/Previews/Preview_6#Java_Source_Support

 3. Using Android.Runtime.JNIEnv to create instance of any needed Java types and invoke methods on those types.

(1) and (2) can be done now, (3) will require the next release.

 - Jon
