Checkout the microsoft project linker extension. It allows you to use the
original c# desktop targeted projects, and then create a new, monodroid
(or silverlight, etc.) specific project and link the code from the
original into there.

Of course, that assumes you have the source for what you want to reference.

>On Dec 9, 2010, at 8:32 AM, Lyra` wrote:
>> hm, i want to reference huge, existing DLLs (wich run under mono for
>>linux)
>> is that possible
>
>No.  The MonoDroid profile uses assemblies with different version numbers
>than the "desktop" .NET assemblies you compiled against.  (The same is
>true for Silverlight and WP7 -- the assemblies use different version
>numbers.)
>
>Specifically, MonoDroid mscorlib.dll is version 2.0.5.0, while .NET 2.0
>uses version 2.0.0.0.
>
>The reason for the difference is that there are a number of significant
>differences between the profiles -- several members are missing, for
>example, and other types and members have been added.  You cannot safely
>take an existing assembly and run it unchanged on a different profile
>(just as you can't take an assembly built against .NET 2.0 and safely run
>it under .NET 1.0).
