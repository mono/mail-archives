Tried your sample with the code on my machine (should be vaguely close to P9).  It no longer fails with a NullReferenceException, it fails with a SecurityException.  If you read the logs carefully, you'll see:

	E/DatabaseUtils( 1828): Writing exception to parcel
	E/DatabaseUtils( 1828): java.lang.SecurityException: Permission Denial: reading 
		com.android.providers.contacts.CallLogProvider uri content://call_log/calls 
		from pid=1821, uid=10033 
		requires android.permission.READ_CONTACTS

Consequently, you'll need to edit your Properties\AndroidManifest.xml file and add the /manifest/uses-permission element:

	<uses-permission android:name="android.permission.READ_CONTACTS" />

Other changes may be required; see the Android documentation for more information:

	http://developer.android.com/guide/topics/manifest/manifest-intro.html#perms
