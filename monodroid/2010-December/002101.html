>From my understanding of how MonoTouch works, this is true, as Managed code
must be compiled to native code for the iOS platforms. i.e. There is no Mono
JIT to handle the IL to native conversion. However, I believe, there is a
Mono Runtime + JITter for MonoDroid, so, as long as your assemblies are
valid assemblies you can include them into the MonoDroid package ( .apk )
for deployment. I can say this, because one of my projects references
CShapCode.SharpZipLib as a compiled IL Assembly, and it works on MonoDroid
just fine, without having to recompile it specifically in MonoDroid.
