On May 5, 2011, at 11:19 PM, Steve Sharrock wrote:
> I haven't done this before. Does this mean use Bugzilla to file a "Bug
> Report and Enhancement Request" following the link "How to make a good bug
> report"?

Yes, which also means you'll need a bugzilla.novell.com account.

> The link talks about command line mcs and such, can I just zip up a sample
> VS2010 solution and attach that?

Absolutely, you can just zip up a sample solution and attach it.

The key point is that it should be simple for mere mortals/moron (me) to reproduce. If it requires that I setup a WCF server "somewhere", change configuration strings, etc., etc., then that significantly increases the complexity of reproducing the bug (and decreases the chances of fixing it in a timely fashion). The closer it is to "build & run", the better.

If specific steps need to be done within the app to cause the error, please provide those steps as well.
