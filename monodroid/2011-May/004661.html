I seem to be unable to add custom HTTP headers to REST WCF calls using the widely accepted pattern of wrapping with an instance of OperationContextScope((IContextChannel)_MyWcfProxy).

Per the below code, "headers" winds up being unexpectedly NULL, so no headers can be added to the call.

            System.ServiceModel.OperationContext operationContext = OperationContext.Current;
            WebOperationContext webContext = new WebOperationContext(operationContext);
            System.Net.WebHeaderCollection headers = webContext.OutgoingRequest.Headers;

I noticed this was previously a problem on MonoTouch, but was eventually resolved.
