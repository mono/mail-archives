On May 3, 2011, at 9:28 AM, Wally McClure wrote:
> I have tried to add a custom view in an axml file.

You mean a custom Android.Views.View subclass? The GLCube sample does that:

	https://github.com/mono/monodroid-samples/tree/master/GLCube
	https://github.com/mono/monodroid-samples/blob/master/GLCube/Resources/layout/main.xml
	https://github.com/mono/monodroid-samples/blob/master/GLCube/PaintingView.cs

Note that main.xml uses a <mono.samples.glcube.PaintingView/> element; this corresponds to the PaintingView type.

>  Unfortunately, I am getting an error on this.

What's the error?

> I was able to add the view manually, so that was ok, but I was wondering if a custom view is supported in an axml file at this point in time.

Yes, it should work (as demonstrated by the GLCube sample).

> I thought I would need to use the <view /> tag, but this generated an error during compile.

No, you need to use an element that is the fully-qualified (Java-side) View subclass to use.
