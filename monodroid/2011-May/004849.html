Craig and Liam answered the immediate question.

Another point, though, is that you should avoid using ApplicationAttribute.Name unless you really need it (and you don't need it here). This is also true for ActivityAttribute.Name, BroadcastReceiverAttribute.Name, etc.

The reason being that, by default, Mono for Android tries to do string manipulations to determine the Java-side type from the managed type name (lowercase the namespace, etc.). If you set the Name property to a different value, Mono for Android _still_ tries the string manipulation first (it's faster than using Reflection), catches the resulting Java ClassNotFoundError, then uses reflection to determine the value of the Name property. This is obviously slower than just using the string value in the first place.

The only time you should use the Name property is when you have to use a specific name (e.g. for backward compatibility reasons).
