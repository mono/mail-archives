On May 3, 2011, at 4:14 PM, Mike Tajmajer wrote:
> I'm calling a static method in a Java class and have found that JNIEnv.CallStaticBooleanMethod *always* returns true.
> 
> public static Boolean testBoolFalse()
>        {
>            Boolean ret = false;
>            IntPtr methodId = JNIEnv.GetStaticMethodID(_helperClass, "testBoolFalse", "()Ljava/lang/Boolean;");

java.lang.Boolean != boolean. :-)

CallStaticBooleanMethod is for methods which return Java `boolean` (the value type). java.lang.Boolean is a reference type (inheriting, eventually, from java.lang.Object). The two are not the same. :-)

If your method is returning `boolean`, you should use JNIEnv.CallStaticBooleanMethod() (and your GetStaticMethodID() signature should use 'Z' for the type, e.g. "()Z"). If your method is returning `java.lang.Boolean`, then you should use JNIEnv.CallStaticObjectMethod().
