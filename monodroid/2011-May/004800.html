On May 7, 2011, at 3:06 PM, Matthew Groves wrote:
> For testing the accelerometer on the Android emulator, there is a great sensor simulator available, but it requires including a Jar in your project, which leaves Mono for Android developers out in the cold.

You can include .jar files in your project. Just set the Build action to AndroidJavaSource (which is valid for both .jar and .java files, or it can include a directory which will be recursively traversed to find all .java & .jar files).

The downside is that you need to use JNI to interact with any of the types in the .jar.

A similar-but-not-quite-exact sample is:

	https://github.com/mono/monodroid-samples/tree/master/GoogleMaps

This uses an existing library that must be present on the device (instead of including the .jar into the final .apk), but otherwise it's close to what you'd need to do.
