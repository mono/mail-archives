On May 4, 2011, at 3:13 PM, Madhuri Mittal wrote:
> 1 ) Putting a breakpoint into the code and debugging does not always work....do I need to do 'Attach to process' ? If so , is it the adb.exe or some other?

I don't think "Attach to process" will work, given that the process you want to attach to isn't even on the same machine...

If the breakpoint doesn't work, please file a bug (and/or wait for the next release, which will have more debugger fixes, and if it still doesn't work file a bug). We know there are issues, but it's useful to know what specific issues remain.

> 2) Do the emulator devices cache the last deployed binary?

The emulator has persistent storage, and will preserve all installed apps from one "boot" to the next.

> Why does n't the modified new binary get deployed right away when I hit F5 or 'Start Debugging' ?

The new binary isn't deployed immediately because it can't; the only way to update the app is to build a new package (.apk) and install the new package (plus an intermediate package uninstall). Package creation is not an instance process, hence the delay.

We intend to improve the package creation experience in the future, but there's only so much that can be done...
