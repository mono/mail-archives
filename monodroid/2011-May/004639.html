On Sun, May 1, 2011 at 3:29 PM, Jonathan Pryor <jpryor at novell.com> wrote:

> On Apr 30, 2011, at 11:38 AM, Fabrice wrote:
> > Well, I need to return a Java.Lang.Object ... and of course it's not
> possible as my custom class doesn't inherit from this.
> >
> > So, does someone have a tip for me, as I suppose it's something I'll
> often need to deal with
>
> Use an intermediate wrapper type:
>
>        class ObjectRef<T> : Java.Lang.Object {
>                public ObjectRef (T value)
>                {
>                        Value = value;
>                }
>
>                public T Value { get; private set; }
>        }
>
> Then use this in your GetGroup() method:
>
>        public override Java.Lang.Object GetGroup (int groupPosition)
>        {
>                 return new ObjectRef<DeviceGroup>(Groups [groupPosition]);
>        }
>
> You will of course need to use ObjectRef<DeviceGroup> in other methods
> instead of DeviceGroup.
>
> You may also consider filing an enhancement request for a
> BaseExpandableListAdapter<T> which does the value wrapping for you.
>
