I would suggest looking at the Android Debug Log output to see if it mentions anything relevant:

	http://mono-android.net/Documentation/Guides/Android_Debug_Log

Which brings us to the Android source [0, 1], and the (hidden[2]!) "documentation" for PackageManager.INSTALL_FAILED_CONTAINER_ERROR:

    /**
     * Installation return code: this is passed to the {@link IPackageInstallObserver} by
     * {@link #installPackage(android.net.Uri, IPackageInstallObserver, int)} if
     * a secure container mount point couldn't be accessed on external media.
     * @hide
     */
    public static final int INSTALL_FAILED_CONTAINER_ERROR = -18;

What's a "secure container mount point"? I have no idea. I'd suggest making sure that your emulator has an SD card attached, though. (Use the `android` program to edit the emulator options and use the radio button to enable the SD card.)

At this point I'd resort to reading the PackageManaagerService source [3] (why? because it's the only other file that matched `grep -r INSTALL_FAILED_CONTAINER_ERROR`), which has lots of matches which seem to relate to cache paths. Again, I have no idea why a cache path would be involved in package installation...but you have more info to track things down now. :-)
