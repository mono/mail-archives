Hey Miggy,

Yep, we have talked about this before. jluke said that we would loose
alot of the ECMA docs by doing this. However, as I have expressed
before, I think that the risk of loosing docs that we can copy/paste is
much smaller than the problem of *not* having alot of api docs. It is
really hard to tell what one is able to do with Mono. Right now, I often
use nautilis browsing /devel/mcs/class/ as my `monodoc'. That is a
pretty bad situation.

And editing makes this change easier -- a developer can easily check a
class for documenation loss in 30 minutes, and copy /paste any lost
docs.

Also, I think we should use this time to update all classlib docs.
