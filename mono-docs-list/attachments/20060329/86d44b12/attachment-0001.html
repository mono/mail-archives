As requested, I've patched monodocer to (optionally) notice the presence of attributes derived from System.ComponentModel.DescriptionAttribute, and the use descriptions they contain for member summaries instead of the default &quot;To be added.&quot;&nbsp; When updating old documentation, and the old docs already contain a summary, the attribute value overwrites the old summary (this could be changed if needed).&nbsp; This feature is off by default, and can be enabled via a new &quot;-usedescriptions&quot; flag on monodocer.&nbsp;
