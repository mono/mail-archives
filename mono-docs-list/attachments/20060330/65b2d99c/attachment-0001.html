I've patched monodocer to optionally look for member attributes that are instances of System.ComponentModel.DescriptionAttribute, and use the descriptions found there for the contents of the &quot;summary&quot; element instead of the default &quot;To be added&quot;.&nbsp; If docs are being updated, and the old docs already have a summary, it gets overwritten (this could be changed if needed).&nbsp; This behaviour is disabled by default, and is turned on via a new &quot;usedescriptions&quot; flag to monodocer (this strikes me as not necessarily the best flag name... suggestions for a better one are welcome).&nbsp;
