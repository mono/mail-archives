These are good questions.

The monodoc contributions are processed by me currently, since the tool
to merge the changes is kind of primitive (you can see it in
monodoc/browser/admin.cs) so I can work around the various buglets it
might have.

As the tool evolves, I would not mind giving more "admin" privileges to
people to approve contributions, but right now its too buggy, too
unstable, and is missing too many features.

I will try to process some of the contributions tomorrow (we got quite a
few!).

About the documentation: I think we would have to make new releases of
the documentation with the updates for them to be useful, I am not sure
how often I will make one, its something we can discuss.
