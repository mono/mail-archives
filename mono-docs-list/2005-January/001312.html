Shane Landrum wrote:
> Ah, great. So the <a href> syntax is for the .html files and the 
> <see cref="hlagh"> syntax is for the XML. Gotcha.

This is because <see/> tags in ECMA docs are transformed into <a/> tags 
so that the HTML widget can render them as links, whereas there aren't 
transformations (or not many, I forget) taking place on Monkeyguide docs.

 > (And it's available in full detail at 
http://www.go-mono.com/monodoc-xml.html

That document has some tags in there that don't actually exist (doc, 
event, field, method, property, and maybe seealso).  The descriptions of 
some things are okay, but taken as a whole it's not correct.  The 
definitive reference (because I wrote it based on what Monodoc actually 
expects) is apparently only available now in the Google cache:

http://64.233.161.104/search?q=cache:boIzr6U9xKUJ:www.nullenvoid.com/mono/wiki/index.php/ECMAStyleDocumentation%3Fversion%3D4+tauberer+ecma+xml&hl=en
