Hey guys,

    In the past few months we have done a lot of work on the API
documentation for Gtk# and the results are starting to show: content and
layout are constantly improving, but there is still plenty to be
done.    See the attached screenshot.

    The documentation is also very good: people who have been doing the
documentation have not only copied the text from the Gtk developer site,
but have also added samples to the tricky APIs as well as annotating the
data with information that has been found on the Web or discussed on the
mailing lists about different APIs.  

    If you have struggled with a class, it would be great if you could
take on and write a bit about it on the documentation, and contribute
it, here is a quick overview of where we need help:

	Atk:    No documentation available
	Gdk:    50
	Glade:  10
	Gnome:  80
	Gtk:    80 
	GConf:  10 
	Glib:   20 
	Pango:  100 

    Some are classes, some are interfaces, and some are enumeration
values.  

    Please check our Wiki page:

http://www.nullenvoid.com/gtksharp/wiki/index.php/ApiDocs?PHPSESSID=eff8caf326d30a60fdc572add1a02fa1

Extra points: the MIT X11 or the BSD license are best in this case, just
to stay clear of the current FDL debate ;-)
