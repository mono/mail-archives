Hello,

At debtags (http://debtags.alioth.debian.org) we categorize Debian
packages and we're recently discussing mono[1].  We created a category
for the C# language, but then since there's more than C# in .NET, we're
discussing a category for .NET in general.

An obvious one would be devel::dotnet, which would be widely recognized.
However we aren't sure if there are trademark issues, nor if in the
Linux world we prefer a different name.  If there are not problems with
devel::dotnet, we would go with that one.

Alternatives were devel::cli (discarded because it would be confused
with "command line interface") and devel::cli-platform.

What would you suggest?


Ciao,
