That constructs the name of the foo_get_type name, but glade-3 doesn't try
to bind it to a real function until glade_util_get_type_from_name.

And that function will fail if the module that contains the function has not
yet been opened.  It works with all gtk+ widgets, as we're anyway linked to
gtk+, so that's not a problem.  But we should explicitely open external
modules if want to get their *_get_type function.
