I've just submitted http://bugzilla.gnome.org/show_bug.cgi?id=408441 but 
after some more testing, it *looks* like it's not a problem tied to 
glade-3.1.x, but in fact affects all versions - or possible none.

My refined description of the problem is:

If you add a page to a notebook, but don't put anything in it, the 
previous page and its children don't get constructed properly. Adding 
*any* widget to the page ( apart from in the tab thing at the top ) 
fixes the problem.

The reason I'm posting here about it is that I'm not really sure where 
the bug belongs now. Is it in Glade? Glade appears to render things 
correctly regardless. Is it in libglade? I'm using Gtk2::GladeXML to 
assemble my window from glade files. Perhaps the problem is there.
