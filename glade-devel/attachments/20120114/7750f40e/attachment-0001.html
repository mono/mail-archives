  You do not need a conversion tool to load a file created with Glade 2.<br>
<br>
While Glade 3 is a completely new code base, it is built upon the same<br>
xml format as Glade 2 was generating.<br>
<br>
What you should have, is the latest version of Glade 3.8 stable, Glade 3.10<br>
forward will no longer recognize the old libglade format.<br>
<br>
With Glade 3.8 you should be able to load your old Glade file created with<br>
Glade 2 and then use the menubar: "Project->Properties" to get the project<br>
dialog and change the target format to be GtkBuilder instead of libglade.<br>
<br>
This action will do the conversion, after which you might get some warnings<br>
about usage of widgets that are deprecated or unsupported by GtkBuilder.<br>
<br>
Ideally, Glade 3.8 is intended to eventually cover the full GtkBuilder feature<br>
set offered by GTK+ 2.24 (there may remain a couple unsupported widgets<br>
but I think it's mostly covered)... if you can load your project in the latest<br>
Glade and there is no usage of deprecated widgets, then you should be<br>
able to load that file in Glade 3.10 (if you wish to make the switch to GTK+-3).<br>
<br>
That was a lot of information, the key point though is you should be able<br>
to load your older file created with Glade 2 using Glade 3.8.x without any<br>
problems (if you hit a snag though, please let us know).<br>
