</div><div><div><br></div><div>After the whopping release of Juan Pablo's Drag and Drop support,<br>this new release was originally planned to support editing of files<br></div><div>using the new <template> syntax, to support Composite Widget Templates,<br>
</div><div>but we couldn't resist stepping it, so here are some highlights:<br></div><div><br><div>  o Autosave Support (with configuration in preferences dialog)<br><div>  o Backup previous revision on save (also in preferences)<br>
</div></div>  o Configure what types of errors to display at save time (in preferences)<br><div>  o Inspector now shows warnings for any widgets which might cause<br></div><div>     warnings, for use of deprecated elements or elements not available<br>
</div><div>     in the currently set target version.<br></div><div><br></div><div>And last but not least, more DoggFooding !<br>The following elements are now designed in Glade and use<br>Composite Widget Templates to define the UI, not to mention they've<br>
had a major facelift:<br></div><div>  o GladePreferences (the preferences dialog)<br></div><div>  o GladeProjectProperties (the project properties dialog)<br></div><div>  o GladeEditor (the main property editor)<br></div>
<div>  o GladeActivatableEditor (the editor for GtkActivatable widgets)<br></div><div>  o GladeButtonEditor<br>  o GladeEntryEditor<br>  o GladeImageEditor<br>  o GladeLabelEditor<br>  o GladeWidgetEditor (this shows up in the "Common" tab now instead<br>
</div><div>     of a boring list of properties).<br></div><br>For more information on Composite Widget Templates, see this blog post:<br>    <a href="http://blogs.gnome.org/tvb/2013/04/09/announcing-composite-widget-templates/">http://blogs.gnome.org/tvb/2013/04/09/announcing-composite-widget-templates/</a><br>
</div><br>What is <span class="">Glade</span>?<br>
============<br>
<span class="">Glade</span> is a RAD tool to enable quick & easy development of user<br>
interfaces for the GTK+ toolkit and the GNOME desktop environment.<br>
The user interfaces designed in <span class="">Glade</span> are saved as XML, and by using<br>
the GtkBuilder GTK+ object these can be loaded by applications<br>
dynamically as needed.<br>
<br>
By using GtkBuilder, <span class="">Glade</span> XML files can be used in numerous<br>
programming languages including C, C++, C#, Vala, Java, Perl,<br>
Python, and others.<br><br>============<br>Glade 3.15.1<br>============<br>        - Much UI fixing and refactoring (Juan Pablo Ugarte)<br>        - Added new --css and --screenshot options to glade-preview (Juan Pablo Ugarte)<br>
        - Fix main window shortcuts (bug 695866 Juan Pablo Ugarte)<br>        - Implemented Project Properties dialog with Glade as a composite tempalte (Juan Pablo Ugarte)<br>        - Support editing the GtkWidget:name property (useful for CSS theming)<br>
        - Improved project inter-widget dependancy calculation and project serialization ordering,<br>          object output ordering (and signal ordering) is now consistent and sorted.<br>        - GtkButton: response-id for dialogs is now optional (allowing adding dialog buttons that<br>
          have no dialog response)<br>        - GtkImage: Support the new "resource" property<br>        - GTK+ catalog updated to cover widgets in GTK+ 3.8<br>        - Now show warnings in the UI for deprecated properties and signals<br>
        - Now show warnings in the Inspector view if any deprecated widgets, signals<br>          or properties are used (or if anything is used which is not available in<br>          the project's target version).<br>
        - Optional properties are now always saved if they are enabled, allowing<br>          to explicitly set them to the default value (eg. GtkWidget:hexpand/vexpand) <br>        - Inspector search entry: Restore completions to work nicely again and filter<br>
          the treeview with a search-in-text algorithm.<br>        - Inspector: Add a feature to expand-all<br>        - Property editor title-bar is now optional in the View menu<br>        - Fixed many deprecation warnings in Glade's compile<br>
        - Added Glade's first 2 ever unit tests<br>        - Fixed Devhelp doc search for properties (Juan Pablo Ugarte)<br>        - Make Drag'n'Drop work on regular clicks without entering Drag Mode<br>          and restored old Drag/Resize functionality when SHIFT is pressed (Juan Pablo Ugarte)<br>
        - Improved Drag/Resize which has been rather broken for a while<br>        - Added Autosave and Automatic backup of previous file features, configurable<br>          in Glade's preferences Dialog<br>        - Support Composite GtkBuilder template editing (mark a toplevel GtkWidget as<br>
          the "composite template" widget in the Project Properties dialog or<br>          property editor).<br>        - Support editing of the project domain in the project properties dialog (Juan Pablo Ugarte)<br>
        - Properties in the Project Properties dialog are now undoable<br>        - GladeSignalEditor: Automatically expand rows which have signal callbacks assigned when<br>          loading a widget<br>        - Add options to decide what types of warnings should be presented at project save time,<br>
          configurable in Glade's preferences dialog.<br>        - Added new property editor components GladePropertyLabel, GladePropertyShell and<br>          GladeEditorSkeleton, allowing us to create custom property editors using Glade.<br>
        - Add "Grid" option to "Add parent" menu (Dmitrijs Ledkovs)<br>        - Added Gladeui catalog for IDE's and embedders to design Glade frontends with Glade<br>        - Added private catalog to allow the GTK+ plugin to embed editors that it provides<br>
        - Added new GladeEpropCheck boolean property editor, to be used in some cases for<br>          boolean properties.<br>        - Added new adaptor methods GladeWidgetAdaptor->write_widget_after() and<br>          GladeWidgetAdaptor->destroy_object()<br>
        - Automatically generate gladeui's enum types with glib-mkenums<br>        - Modified properties now show up in italic letering instead of bold, bold<br>          lettering is reserved for title labels in custom property editors.<br>
        - New components that now use GtkBuilder Composite Templates (and have had<br>          a basic review and rework of their UIs): GladePreferences (Glade's preferences<br>          dialog), GladeEditor (the property editor), GladeProjectProperties (the project<br>
          properties dialog) and custom property editors: GladeActivatableEditor,<br>          GladeButtonEditor, GladeEntryEditor, GladeImageEditor, GladeLabelEditor, GladeWidgetEditor<br>        - And... many various bugfixes and UI fixes by Juan Pablo and myself.<br>
