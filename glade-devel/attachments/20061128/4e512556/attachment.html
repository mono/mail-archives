<DIV><SPAN class=026482818-28112006><FONT face=Arial size=2>Bug 345602 describes 
a situation where items cannot be graphically added to a preferred location in a 
table.&nbsp; So, as far a I know, in order to add an item to a table, one must 
increase the size of the table, add the new item to the end, then manually 
jigger up the .glade file, in order to put the new item&nbsp;in the correct 
position.&nbsp; In addition, table descriptions in .glade files are written out 
in (what is probably) creation order and so are not sorted in display 
order.&nbsp; This makes the "jiggering"&nbsp;process consists of moving the new 
item around in the .glade file then renumbering every item in the table.&nbsp; 
Even if the table is of some reasonable size, it can&nbsp;be quite tedious and 
time-consuming.&nbsp; I would like to know if there is any other, more 
reasonable way to add an element to a table.</FONT></SPAN></DIV></BODY></HTML>
