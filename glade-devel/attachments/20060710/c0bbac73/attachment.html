With regards to adding the gnome-db component to glade3 and getting<br>
it built properly... this is the proceedure (like the gnomeui catalog does):<br>
&nbsp;</div><a href="http://configure.in">configure.in</a> needs a block like this:<br>
dnl ================================================================<br>
<br>
dnl ================================================================<br>
dnl Check for optional gnome libs<br>
dnl ================================================================<br>
AC_ARG_ENABLE(gnome,<br>
[&nbsp; --disable-gnome&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; disable gnome catalog],<br>
&nbsp;&nbsp;&nbsp; check_gnome=$enableval, check_gnome=yes)<br>
<br>
if test x&quot;$check_gnome&quot; = x&quot;yes&quot;; then<br>
&nbsp; PKG_CHECK_MODULES(GNOME, libbonoboui-2.0 libgnomeui-2.0,<br>
&nbsp;&nbsp;&nbsp; [have_gnome=yes],[have_gnome=no])<br>
&nbsp; AC_SUBST(GNOME_LIBS)<br>
&nbsp; AC_SUBST(GNOME_CFLAGS)<br>
else<br>
&nbsp; have_gnome=no<br>
fi<br>
<br>
AM_CONDITIONAL(BUILD_GNOME, test x&quot;$have_gnome&quot; = &quot;xyes&quot;)<br>
<br>
dnl ================================================================<br>
<br>
Ofcourse, <br>
&nbsp;&nbsp;&nbsp; you'll be substituting 'GNOME' here for 'GNOMEDB' and checking for the <br>
appropriate libraries with PKG_CHECK_MODULES...<br>
<br>
You'll also need to mention the gnomedb catalog in src/Makefile.am like the<br>
gnome ui catalog does:<br>
###################################################################<br>
if BUILD_GNOME<br>
gladegnome_LTLIBRARIES = <a href="http://libgladegnome.la">libgladegnome.la</a><br>
gladegnomedir = $(glade_modulesdir)<br>
libgladegnome_la_SOURCES = glade-gnome.c<br>
libgladegnome_la_CFLAGS = -g -Wall $(GNOME_CFLAGS)<br>
libgladegnome_la_LDFLAGS = -module -avoid-version $(GNOME_LIBS)<br>
libgladegnome_la_LIBADD = <a href="http://libgladeui-1.la">libgladeui-1.la</a> $(GLADE_LIBS)<br>
endif<br>
###################################################################<br>
<br>
And thats all you have to do for the source code...<br>
<br>
The <a href="http://gnomedb.xml.in">gnomedb.xml.in</a> file goes into $(glade3)/widgets/ ... look at<br>
the $(glade3)/widgets/Makefile.am and adjust it for the newly added<br>
catalog.<br>
