I am trying to package Glade-3 for Fedora, which till now had Glade-2
only. While doing that, I have come across a few issues, which I would
like to clarify with the Glade developers.

+ Glade installs a few libraries, namely:
/usr/lib/glade3/modules
/usr/lib/glade3/modules/libgladegnome.so
/usr/lib/glade3/modules/libgladegtk.so
/usr/lib/libgladeui-1.so.7
/usr/lib/libgladeui-1.so.7.0.1

Are all those libraries meant for external programs which want to
embed Glade-3 and need to be in the ldconfig search path? eg., Anjuta.

+ rpmlint thinks that /usr/lib/libgladeui-1.so.7.0.1 is needlessly
linked to /usr/lib/libpangocairo-1.0.so.0, /usr/lib/libpango-1.0.so.0,
/lib/libdl.so.2, /lib/libgthread-2.0.so.0, and /lib/librt.so.1:

glade3-libgladeui.i386: W: unused-direct-shlib-dependency
/usr/lib/libgladeui-1.so.7.0.1 /usr/lib/libpangocairo-1.0.so.0
glade3-libgladeui.i386: W: unused-direct-shlib-dependency
/usr/lib/libgladeui-1.so.7.0.1 /usr/lib/libpango-1.0.so.0
glade3-libgladeui.i386: W: unused-direct-shlib-dependency
/usr/lib/libgladeui-1.so.7.0.1 /lib/libdl.so.2
glade3-libgladeui.i386: W: unused-direct-shlib-dependency
/usr/lib/libgladeui-1.so.7.0.1 /lib/libgthread-2.0.so.0
glade3-libgladeui.i386: W: unused-direct-shlib-dependency
/usr/lib/libgladeui-1.so.7.0.1 /lib/librt.so.1

Is this alright?
