It seems that you can trace with full runtime debug symbols...........is it
means that your current glade project is set to link with a DEBUG MT
runtime?
If yes then you can try to use RELEASE MT runtime in your project. It
doesn't matter with intl.dll or iconv.dll (as well as all gtk+ related
.dlls, libxml2, etc.) because most of them distributed around the Internet
are already using MSVCRT.DLL.
