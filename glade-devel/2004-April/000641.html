Hello Jonathan,

> Could you please define REST for me?

  As I wrote before a REST-style format design simply
means that you care about your XML format. If you use
a REST-style design approach than your XML UI format
used by your forms designer (e.g. Glade) is not just
an afterthought but is central. (See
http://article.gmane.org/gmane.comp.gnome.glade.devel/796
for an example on what's wrong with the Glade XML UI
format.)  
 
  A good example of a REST-style XML format besides
HTML is RSS or Atom for example. For more about REST
see the Web Architecture document @
http://www.w3.org/TR/webarch

  - Gerald 

PS: REST stands for REpresentational State Transfer
(REST). See
http://www.ics.uci.edu/~fielding/pubs/dissertation/top.htm
for how it all started.
