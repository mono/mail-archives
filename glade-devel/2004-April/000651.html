A new functionnality is added into CVS version.<br>
Now, a psedo stand alone script is created. As this time, it's really a
toy... not very well implemented. I use shareutils to create a sh
script, afterward i rescan the script generated to modify some things. <br>
The script generated is executable, create a new temporary directory
into /tmp (for linux only hu) and extract everythings. Then it goes to
this directory, execute the script and wait. Finaly, it erase this
temporary directory, and quit.<br>
<br>
The main use of this tool is to generate an "easy to give" executable
file to your clients without thinking about where and how they will
execute it. <br>
This generated script is really interressing because it can be call by
click into browsers for example ! mozilla, konqueror, etc... can run
the script :) <br>
The only "problem" is that the client have to have php and gtk
installed on his computer... so that's why i have another the following
idea:<br>
I don't know if it's possible to include directly php and librairie
needed into it (this is to generate a psedo-real-executable file ;) )
