Hello,


Here a two little bugs I've found in Glade 2.0.0.  They might seem a little cosmetic to some, but they're still problems which should be dealt with.  Here they are:


1.  When doing a "make install" of an existing project, pixmaps (from the "<projectname>/pixmaps" directory) are installed in "/usr/local/share/pixmaps/<projectname>".  This is normal.  But when a "make uninstall" is performed (say, if somebody wants to uninstall <projectname>, the pixmaps are NOT removed from "/usr/local/share/pixmaps/<projectname>" and the "<projectname>" directory is NOT deleted from "/usr/local/share/pixmaps".  This uses unnecessary disk space.

2.  After Glade 2.0.0 has been running for some time, when double-clicking on a certain window name in the main Glade 2.0.0 window, the clicked window does NOT appear (so it can be edited and worked on), as it was previously the case with Glade 0.6.4.  The user is forced to use the widget tree, right click on the desired window name and select "redisplay" to get the desired window to appear.  I can't figure out the exact conditions for this problem to show, but it seems to appear after many windows have been opened/closed several times.  If this can be of any help, I am using Enlightenment 0.16.5.
