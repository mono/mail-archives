Well its much easier to popup another emacs buffer to the ChangeLog
than run "cvs diff -u|less". Anyways what I'm suggesting is to follow
something of the sorts of the "GNU ChangeLog Guidelines"
(see the GNU Coding Standards info page for that).

Also in ChangeLog's rather than say something like "moved code logic to
use inheritance", its better if you add per function change specific change
info.
