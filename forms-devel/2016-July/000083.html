Hi guys,

 

There is one way of using the hypothetical RepeaterView control which doesn’t require a backing StackLayout or Grid.

Consider the scenario where you have a StackLayout inside ScrollView already containing several controls and let’s assume that you want to inject a couple of controls somewhere inside. In this case RepeaterView takes its parent as a host container and simply injects elements from its ItemsSource using the specified ItemTemplate at the position where the RepeaterView is located (itself invisible).

 

So we have to differ between simple replacement for a ListView from an injectable repeater.
