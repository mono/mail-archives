<p dir="ltr">I'm writing to discuss about "Pressed" and "Released" events on Button. I believe that it is useful to application developers who want to detect when a button is first pressed and ends when it is released. These events work across platforms (android, iOS and UWP).</p>
<p dir="ltr">In android, users can use OnTouchListener.onTouch(View v, MotionEvent event) as following.<br>
  button.setOnTouchListener(new OnTouchListener() {<br>
     @Override<br>
     public boolean onTouch(View v, MotionEvent event) {<br>
        if(event.getAction() == MotionEvent.ACTION_DOWN) {<br>
          // Pressed<br>
        } else if (event.getAction() == MotionEvent.ACTION_UP) {<br>
          // Released<br>
        }<br>
    }<br>
};</p>
<p dir="ltr">In iOS, we can use addTarget:action:forControlEvents: method in UIControl class as below.<br>
  [button addTarget:self action:@selector(buttonDown):forControlEvents:UIControlEventTouchDown]; // Pressed<br>
  [button addTarget:self action:@selector(buttonUp): forControlEvents:UIControlEventTouchUpInside]; // Released</p>
<p dir="ltr">In UWP, as you may already know, there is Click event and ClickMode property in ButtonBase class. <br>
<Button x:Name="btn1" Content="Press to Click" Click="OnClick1" ClickMode="Press"/> <br>
<Button x:Name="btn2" Content="Release to Click" Click="OnClick2" ClickMode="Release"/></p>
<p dir="ltr">I could do a PR for</p>
<p dir="ltr">a) Adding the Button.Pressed and Button.Released events. (In this case, events occur in the following order. pressed -> released -> clicked)</p>
<p dir="ltr">or</p>
<p dir="ltr">b) Adding the Button.ClickMode (Press/Release) property similar to UWP way.</p>
<p dir="ltr">or </p>
<p dir="ltr">c) Adding a new GestureRecognizer (e.g. TouchGestureRecognizer).</p>
