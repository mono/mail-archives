<p>Hello all and Adam,<o:p></o:p></p>
<p>Thank you for you opinion.<o:p></o:p></p>
<p>I agree that the way using ListView you mention can be another way to provide a multiple selection of items.
<br>
I also agree that this approach can provide better UX for ListView.<o:p></o:p></p>
<p>However, how about considering the scenario that the number of item is one or two?<br>
If the number of item is one or two and app user can select one status between 'Checked' and 'Unchecked' in each item,
<br>
Should app developer provide this UX with ListView?<br>
The development process of an app that this UX is provided with ListView takes more time than the procedure using a simple checkbox.
<br>
The app should include more code line than an app using a simple checkbox. <o:p></o:p></p>
<p>As you already know, checkBox have function to indicate two states which are 'checked' and 'unchecked' with text<br>
as well as function to provide multiple selections of items.<br>
So, checkBox could be useful control for  providing user's selection between 'checked' and 'unchecked' about one or two items with text.<o:p></o:p></p>
<p>Also, is this link(<a href="https://na01.safelinks.protection.outlook.com/?url=https%3a%2f%2fmaterial.google.com%2f&data=01%7c01%7cjas%40microsoft.com%7cb3c1d44ecf3042c9df2308d3c2863d6f%7c72f988bf86f141af91ab2d7cd011db47%7c1&sdata=4Q%2b7aBplhu%2bp8mercu%2b%2bNUx1aIzyNpnbyIsJiFQ%2fNjg%3d">https://material.google.com/</a>)
 Google Material Design Guidelines you mention on a previous email?<br>
Actually, it is a little hard to find some opinion that guideline specially discourage checkbox.
<br>
Instead, this design guideline describes well that checkbox is one of controls for allowing the user to select options.
<br>
I'd like to share two link about checkbox in Google Material Design Guidelines. <br>
-<a href="https://na01.safelinks.protection.outlook.com/?url=https%3a%2f%2fmaterial.google.com%2fcomponents%2fselection-controls.html%23&data=01%7c01%7cjas%40microsoft.com%7cb3c1d44ecf3042c9df2308d3c2863d6f%7c72f988bf86f141af91ab2d7cd011db47%7c1&sdata=uuf%2ferYU9MNszNei%2bznP66uRhu%2fud0LuPvVmV59R3mY%3d">
 https://material.google.com/components/selection-controls.html#</a><br>
-<a href="https://na01.safelinks.protection.outlook.com/?url=https%3a%2f%2fmaterial.google.com%2fcomponents%2flists-controls.html%23&data=01%7c01%7cjas%40microsoft.com%7cb3c1d44ecf3042c9df2308d3c2863d6f%7c72f988bf86f141af91ab2d7cd011db47%7c1&sdata=KfVnIYNo1G7rgkZDPT1QhtIEgC8NAxF9wEAl%2fp3MBLU%3d">
 https://material.google.com/components/lists-controls.html#</a><br>
These two link describe definitely how checkbox can be useful for providing various UX to user.
<o:p></o:p></p>

<p class="MsoNormal">Since Checkboxes are natively a part of Android and UWP, I think it could be good to expose them. iOS would probably need a toggle with a different image, or a button with a different selected state as I don't know of a native checkbox.<o:p></o:p></p>
<div>
<p class="MsoNormal"><o:p> </o:p></p>
</div>
<div>
<p class="MsoNormal">Rina, while checkboxes can be good the specific scenario you mention for their use, such as a multiple select of items in a list is normally discouraged due to UX issues such as if you are selecting the item or ticking the checkbox. For
 this reason Google Material Design Guidelines specifically discourages them and I think other guideline do as well.<o:p></o:p></p>
</div>
<div>
<p class="MsoNormal"><o:p> </o:p></p>
</div>
<div>
<p class="MsoNormal">Might I suggest when you select an item in the ListView or list of some kind you show a tick next to the item, or highlight the row in the list. This approach tends to go over better in a mobile UI and doesn't confuse selecting the list
 row, vs a control in the list.<o:p></o:p></p>

Hello,<br>
I'd like to add a CheckBox for allowing the user can select multiple<br>
options in the list.<br>
And a CheckBox is a specific control that has two states which are<br>
'checked' and 'unchecked'.<br>
<br>
This CheckBox will have the following properties.<br>
- FontAttributes : A value that indicate whether the font of the checkbox<br>
is bold, italic or neither.<br>
- FontFamily : A value that indicate font family to which the font for<br>
checkbox text belongs.<br>
- FontSize : A value that indicate font size of checkbox.<br>
- IsChecked : A value that indicate whether this checkbox is checked or<br>
not.<br>
- Text : A text displayed as the content of the checkbox.<br>
- TextColor : A value that indicate the color for the text of checkbox.<br>
<br>
Also, this CheckBox will have an event.<br>
- Checked : This event occurs when checkbox is checked.<br>
<br>
Just like other controls, the CheckBox is a very common control.<br>
Xamarin will be able to provide richer function in many application, if<br>
this control is added on xamarin.<br>
I will look forward to hearing your opinion.<br>
Thank you.<br>
