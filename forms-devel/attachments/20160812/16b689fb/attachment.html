<p dir="ltr">Hello all and Adam,</p>
<p dir="ltr">Thank you for you opinion.</p>
<p dir="ltr">I agree that the way using ListView you mention can be another way to provide a multiple selection of items. <br>
I also agree that this approach can provide better UX for ListView.</p>
<p dir="ltr">However, how about considering the scenario that the number of item is one or two?<br>
If the number of item is one or two and app user can select one status between 'Checked' and 'Unchecked' in each item, <br>
Should app developer provide this UX with ListView?<br>
The development process of an app that this UX is provided with ListView takes more time than the procedure using a simple checkbox. <br>
The app should include more code line than an app using a simple checkbox. </p>
<p dir="ltr">As you already know, checkBox have function to indicate two states which are 'checked' and 'unchecked' with text<br>
as well as function to provide multiple selections of items.<br>
So, checkBox could be useful control for  providing user's selection between 'checked' and 'unchecked' about one or two items with text.</p>
<p dir="ltr">Also, is this link(<a href="https://material.google.com/">https://material.google.com/</a>) Google Material Design Guidelines you mention on a previous email?<br>
Actually, it is a little hard to find some opinion that guideline specially discourage checkbox. <br>
Instead, this design guideline describes well that checkbox is one of controls for allowing the user to select options. <br>
I'd like to share two link about checkbox in Google Material Design Guidelines. <br>
-<a href="https://material.google.com/components/selection-controls.html#"> https://material.google.com/components/selection-controls.html#</a><br>
-<a href="https://material.google.com/components/lists-controls.html#"> https://material.google.com/components/lists-controls.html#</a><br>
These two link describe definitely how checkbox can be useful for providing various UX to user. </p>
<p dir="ltr">Thank you for sharing your opinion about checkbox again.</p>

I'd like to add a CheckBox for allowing the user can select multiple<br>
options in the list.<br>
And a CheckBox is a specific control that has two states which are<br>
'checked' and 'unchecked'.<br>
<br>
This CheckBox will have the following properties.<br>
- FontAttributes : A value that indicate whether the font of the checkbox<br>
is bold, italic or neither.<br>
- FontFamily : A value that indicate font family to which the font for<br>
checkbox text belongs.<br>
- FontSize : A value that indicate font size of checkbox.<br>
- IsChecked : A value that indicate whether this checkbox is checked or<br>
not.<br>
- Text : A text displayed as the content of the checkbox.<br>
- TextColor : A value that indicate the color for the text of checkbox.<br>
<br>
Also, this CheckBox will have an event.<br>
- Checked : This event occurs when checkbox is checked.<br>
<br>
Just like other controls, the CheckBox is a very common control.<br>
Xamarin will be able to provide richer function in many application, if<br>
this control is added on xamarin.<br>
I will look forward to hearing your opinion.<br>
