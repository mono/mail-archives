<p class="MsoNormal"><span style="font-size:11.0pt;font-family:Calibri">Hello,<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:Calibri"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:Calibri">My take is that high-level controls like the CalendarView belong to an add-on, and not to the core platform.<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:Calibri"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:Calibri">The reason is that these controls can have so many configuration options and features, that whatever we end up adding to the core will be deemed insufficient over time.   The API will be
 insufficient, will have design limitations, and people do take specific dependencies on the UX.   Putting in on the core means that the code will be subject to the API stability rules of the rest of the runtime core and some UX will have to be frozen in time
 as people take dependencies on it.<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:Calibri"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:11.0pt;font-family:Calibri">This is a perfect example of something that is best done outside the core as a separate control.   Very much like chart controls, data-entry controls and so on.<o:p></o:p></span></p>

<p class="MsoNormal">Hello All,<br>
<br>
I have noticed that there is no simple way for application developers to provide calendars for users on Xamarin.Forms - there is no CalendarView.<br>
I am thinking about a control where users can interact with a calendar by selecting a date or by browsing by month.<br>
This picture from the Windows APIs shows what I have in mind: <a href="https://na01.safelinks.protection.outlook.com/?url=https%3a%2f%2fi-msdn.sec.s-msft.com%2fdynimg%2fIC790864.png&data=02%7c01%7cmiguel%40microsoft.com%7c65688618a6d54baadbc808d3d591ffc9%7c72f988bf86f141af91ab2d7cd011db47%7c1%7c0%7c636086795450738903&sdata=Za2ST9nsjAnw91YidnXOZRBAtESL9f9eAj6sCL%2bwiJU%3d">
https://i-msdn.sec.s-msft.com/dynimg/IC790864.png</a> . Perhaps something like the month view.<br>
<br>
What does everyone think about providing such a control to application developers in Xamarin.Forms?<br>
<br>
The following native controls could be used:<br>
Windows - <a href="https://na01.safelinks.protection.outlook.com/?url=https%3a%2f%2fmsdn.microsoft.com%2fen-us%2fwindows%2fuwp%2fcontrols-and-patterns%2fcalendar-view&data=02%7c01%7cmiguel%40microsoft.com%7c65688618a6d54baadbc808d3d591ffc9%7c72f988bf86f141af91ab2d7cd011db47%7c1%7c0%7c636086795450738903&sdata=piSMpAz7bGtVhOLTu%2fMFfD9JCb%2bQ9VVEfFqB6XgwGB0%3d">
https://msdn.microsoft.com/en-us/windows/uwp/controls-and-patterns/calendar-view</a><br>
Android - <a href="https://na01.safelinks.protection.outlook.com/?url=https%3a%2f%2fdeveloper.xamarin.com%2fapi%2ftype%2fAndroid.Widget.CalendarView%2f&data=02%7c01%7cmiguel%40microsoft.com%7c65688618a6d54baadbc808d3d591ffc9%7c72f988bf86f141af91ab2d7cd011db47%7c1%7c0%7c636086795450738903&sdata=wNB7HiuixfAQcxJ95MMEF2f1bBoUvTd6VDRKxYOx8cY%3d">
https://developer.xamarin.com/api/type/Android.Widget.CalendarView/</a><br>
iOS - As far as I know, iOS does not provide a native counterpart. We would have to create something... Can we provide a Xamarin.Forms control if there is no native counterpart?<br>
<br>
It would allow only one date to be selected at a time. Initially, we could have the following properties:<br>
- Date: The current selected date<br>
- FirstDayOfWeek: The first day of the week<br>
- MaxDate: The last date to display<br>
- MinDate: The first date to display<br>
And an event for whenever the user selects a date - EventArgs would be the selected date.<br>
<br>
What does everyone think? I look forward to getting your thoughts regarding CalendarView.<br>
