<div dir="ltr">Hello Xamarin.Forms Community,<br>
<br>
Why doesn't Xamarin.Forms support custom Font's for various controls on the Android Platform?<br>
<br>
As a recap the current situation.<br>
(<a href="https://developer.xamarin.com/guides/xamarin-forms/user-interface/text/fonts/" target="_blank">https://developer.xamarin.com/guides/xamarin-forms/user-interface/text/fonts/</a>):<br>
iOS: Use the Font Name (Custom fonts that should be loaded are defined in info.plist).<br>
Android: Create a Custom renderer  or use the font name for a system font.<br>
WP: Use filelocation # fontname for a custom font, or use the font name for a system font.<br>
<br>
Something that could be implemented for Android so custom fonts is a supported feature across all platforms.<br>
<br>
We could parse the FontFamily property to determine if it's an filelocation/filename [myawesomefont.ttf/myawesomefont.otf]. If it is, we should load the font from that location, otherwise it's a normal font and we should use the current way of loading a font.<br>
<br>
It seems to be all that has to be changed is this:<br>
<a href="https://github.com/royvou/Xamarin.Forms/blob/android-customfont/Xamarin.Forms.Platform.Android/Renderers/FontExtensions.cs" target="_blank"></a><a href="https://github.com/royvou/Xamarin.Forms/blob/android-customfont/Xamarin.Forms.Platform.Android/Renderers/FontExtensions.cs" target="_blank">https://github.com/royvou/Xamarin.Forms/blob/android-customfont/Xamarin.Forms.Platform.Android/Renderers/FontExtensions.cs</a><br>
<br>
<br>
This shouldn't have a big impact on the codebase and not break existing apps.<br>
<br>
