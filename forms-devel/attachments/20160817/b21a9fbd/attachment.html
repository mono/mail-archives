<p dir="ltr">Hello, </p>
<p dir="ltr">I'd like to share a draft spec for CheckBox in the mailing list. <br>
And then, I will make a PR for CheckBox based on other opinions in this mailing list. </p>
<p dir="ltr">* The CheckBox will inherit View and IFontElement and have a public constructor for initializing a new instance of the CheckBox class. </p>
<p dir="ltr">* The CheckBox will have the following public fields for bindable properties:<br>
- public static readonly BindableProperty FontAttributesProperty : Identifies the FontAttributes bindable property.<br>
- public static readonly BindableProperty FontFamilyProperty     : Identifies the FontFamily bindable property.<br>
- public static readonly BindableProperty FontSizeProperty       : Identifies the FontSize bindable property.<br>
- public static readonly BindableProperty IsCheckedProperty      : Identifies the IsChecked bindable property.<br>
- public static readonly BindableProperty TextProperty           : Identifies the Text bindable property.<br>
- public static readonly BindableProperty TextColorProperty      : Identifies the TextColor bindable property.<br></p>
<p dir="ltr">* The CheckBox will have the following public properties.<br>
- public Font   FontAttributes : A value that indicates whether the font of the checkbox is bold, italic or neither.<br>
- public String FontFamily     : A value that indicates the font family which the checkbox font belongs to.<br>
- public Double FontSize       : A value that indicates font size of the checkbox.<br>
- public bool   IsChecked      : A value that indicates whether this checkbox is checked or not.<br>
- public String Text           : A value for the text that is displayed as the content of the checkbox.<br>
- public Color  TextColor      : A value that indicates the color for the text of checkbox.<br></p>
<p dir="ltr">* The CheckBox will have an event. <br>
- public event EventHandler Clicked : This event occurs when the checkbox is checked.<br></p>
<p dir="ltr">If you have any opinion about the draft spec for CheckBox, please, could you share your opinion?<br>
I look forward to hearing any opinions about this draft spec.<br>
Thank you.</p>

<p>I agree that the way using ListView you mention can be another way to provide a multiple selection of items.
<br>
I also agree that this approach can provide better UX for ListView.<u></u><u></u></p>
<p>However, how about considering the scenario that the number of item is one or two?<br>
If the number of item is one or two and app user can select one status between 'Checked' and 'Unchecked' in each item,
<br>
Should app developer provide this UX with ListView?<br>
The development process of an app that this UX is provided with ListView takes more time than the procedure using a simple checkbox.
<br>
The app should include more code line than an app using a simple checkbox. <u></u><u></u></p>
<p>As you already know, checkBox have function to indicate two states which are 'checked' and 'unchecked' with text<br>

So, checkBox could be useful control for  providing user's selection between 'checked' and 'unchecked' about one or two items with text.<u></u><u></u></p>

 Google Material Design Guidelines you mention on a previous email?<br>
