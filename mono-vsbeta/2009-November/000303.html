Eric Hexter wrote:
> I created a package for a website and the interface if very clunky and 
> time consuming.  When I selected build package and then was prompted 
> with the project list from my solution, I selected the web project and 
> expected that the packaging would just run through my project and add 
> all of the content files in the project.  When that did not happen I 
> tried using the Add Directory, but when you select the root directory of 
> the web application, it pulls in every single file, including the code 
> files, project files, and obj files. 
>  
> I talked with Jackson and Miguel at the monospace conference and it was 
> clear that the packaging worked well for non-web projects.   Those are 
> just some of my thoughts after using that feature.

A workaround is to use VS's Publish command to publish a copy of your 
website to another location on disk, and then do Add Directory on that 
location.

We'll be looking at how to make packaging easier for future releases.

Thanks for the feedback!
