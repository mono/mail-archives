Ok, thanks, I found it (manually), it lacks the namespace prefix name. It
seems it is not using the next web.config directive:
      <namespaces>
        <add namespace="dbmlib"/>
      </namespaces>

(dbmlib is the namespace where the class Config is)

This Web.config file is in the Views folder of a MVC project, (ASP.NET is
working OK without namespace prefix).
