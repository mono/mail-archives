Hey Antonio,

Antonio Anzivino wrote:
>
> Hello,
>
> I noticed a couple of things in the Run/Debug Remotely window. The
> first is that the list box on the left is always empty. Does MonoVS
> broadcast the LAN to search for active VMs?
>

Yes, this is how it finds the VM. The list box should be filled with
auto discovered VMs / MonoVS server side components, unless it was
unable to discover them. This can happen either because the VM is on a
separate network, or it's behind VMware's NAT. If VMware is in bridged
mode, then it finds it fine (that's normally how we test).

> Or can I bookmark my favourite hosts? How?
>

Not that I know of.

> Second, what happens if I check “Use always this host”? Will I be able
> in the future to restore that window? I think an options panel should
> be created somewhere.
>
Yes, you can restore it using the MonoVS options page which is in Tools
/ Options / Mono. The option is in the main page and it's called "run
host selector at startup." If this is checked, then you'll see the
dialog. If it's not, then it'll take the saved value.
