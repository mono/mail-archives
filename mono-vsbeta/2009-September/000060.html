Ok, here is how to recreate quite simply. 

Create a console application, add a reference to Db4objects.Db4o.dll and
a corresponding using . Add these three lines of code to the main method-

Console.WriteLine("1");
IObjectContainer ioc = Db4oFactory.OpenFile("x.db");
Console.WriteLine("2");

Put a breakpoint on the second console.writeline.  Debug remotely in
mono.  It never gets to the breakpoint.  The output window also shows a
continuous stream of threads ending, that does not happen if you are not
debuging.

Interestingly, if you put a breakpoint on the first console.writeline
and then continue, the second breakpoint is hit just fine and the stream
of threads does not happen.
