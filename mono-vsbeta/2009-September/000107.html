>The nic being absent is because MonoVS requires a bridged connection and we
cannot define a nic without specifying which nic it's bridged to.  Older
versions of of VPC would automatically add you a nic if 
>you didn't have one defined so it worked most of the time.  Newer versions
assume you meant not to have one.  This is all irrelevant though because the
modus operandi of the VPC world is to give you only 
>a .vhd file and you have to configure the VM yourself.  I have not found
any VPC images anywhere that gave you more than just the .vhd.  We are the
first.

>From what I understand each VM maker emulates the hardware (interface) of
one or more well-known network adapter classes that the user can choose
from. However the OS (or whatever other low level s/w) running in the VM
should be able to detect the hardware and have an OS (usually kernel mode)
driver for it.
