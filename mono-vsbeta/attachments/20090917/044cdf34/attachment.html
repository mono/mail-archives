Dinis,<div><br></div><div>Mono as a Framework has never been a 1 for 1 copy of any specific Microsoft .NET Framework.</div><div><br></div><div>The Mono Team tends to add compatibility with the published specifications at any given time based on their own needs and feedback within the Community.</div>

<div><br></div><div>This is not necessarily a bad thing.  A lot of times you will find 100% compatibility with the most used features of a given Microsoft .NET Framework but then some of the more obscure stuff may not be implemented yet.</div>

<div><br></div><div>For example, Microsoft posts the official ECMA Specifications for C# 3.0 (or whatever). They themselves haven&#39;t even beta versions of the Framework that implements these new specifications but the Mono Team may find something that looks really good so they will add it to Mono.</div>

<div><br></div><div>Members of the Mono Team can correct me if I am wrong but while Mono might not be 100% compatible with Microsoft .NET 3.5 SP1, it already includes some features from the 4.0 Microsoft .NET Specifications that are still in CTP (i.e. beta).</div>

<div><br></div><div>I believe the latest version of Mono is 100% MS .NET 2.0 compatible but may still be missing some parts of 3.5.</div><div><br></div><div>Somebody please correct me on that.</div><div><br></div><div>To help the Mono Team, they create MoMA which stands for the &quot;Mono Migration Analyzer&quot;. You can read more about it here:</div>

<div><br></div><div><a href="http://www.mono-project.com/MoMA">http://www.mono-project.com/MoMA</a></div><div><br></div><div>This is available as a standalone application but <b>is also <i>included</i><span class="Apple-style-span" style="font-weight: normal;"> in the Mono Tools for Visual Studio as an integrated add-in.</span></b></div>

<div><br></div><div><b><span class="Apple-style-span" style="font-weight: normal;"></span></b>With MoMA you can analyze your applications binaries (EXE, DLL, etc) for compatibility issues and then supply the report to the Mono Team to help them complete compatibility with Microsoft&#39;s version.</div>

<div><br></div><div>NOTE: This is built-in with the Mono Tools for Visual Studio and should actually run automatically after a build</div><div><br></div><div>The output of this analyzer is very important to the team so make sure to submit issues through Bugzilla.</div>

<div><br></div><div>If you have been following this list you will find that the Mono Team typically will point such reports to Bugzilla on the Mono Site for reporting such issues as they are outside the scope of this beta which is for the Mono Tools for Visual Studio and not Mono itself.</div>
