Hi,

this happens when Ssdp either cannot determine your computer's IP
address or it fails to bind to the broadcast address.

Not being able to bind to the broadcast address usually happens if
you're not connected to the internet and thus don't have a default
route.

I had this problem when running Linux inside VMware and ethernet
connection was down, but I still had the vmware network interface.
